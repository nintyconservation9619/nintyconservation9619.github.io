詳解

ホスト・ターゲット間ソケット通信の名前空間です。

機能概要

HTCS とは、Host Target Communication Socket の略です。
HTCS ライブラリでは、ソケットインターフェイスによるホストとターゲット間の通信手段を提供します。
HTCS ライブラリの機能はビルドタイプ Release の場合、標準では使用できないようになっています。
ビルドタイプ Release で使用したい場合は NN_ENABLE_HTC を定義するようにしてください。

使用方法

ターゲット側での基本的な使用手順はソケットと同じです。
ホスト側では Winsock などのソケットインターフェイスを使用してください。
またホストとの接続を行うためにはあらかじめ TargetManager.exe を起動しておく必要があります。
詳細な接続手段についてはマニュアルの接続シーケンスの項を参照してください

注意
以下の引数のみサポートされます。
  • command == nn::htcs::HTCS_F_GETFL
  • (command == nn::htcs::HTCS_F_SETFL) && (value == 0 || value == nn::htcs::HTCS_O_NONBLOCK)
詳細
ソケットの動作設定フラグを制御します。
command == nn::htcs::HTCS_F_GETFL の場合は現在の動作設定フラグの設定値を返します。
command == nn::htcs::HTCS_F_SETFL の場合は value の値を動作設定フラグに設定します。
