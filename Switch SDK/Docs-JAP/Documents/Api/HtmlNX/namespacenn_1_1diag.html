詳解

プログラム診断用ライブラリの名前空間です。

詳細
プログラムの動作を診断するためのライブラリです。 プログラム診断用ライブラリの定義するマクロは、マクロの一覧 から参照できます。

型定義詳解

typedef void(* nn::diag::AbortObserver) (const AbortInfo &abortInfo)

アボートオブザーバの型です。

引数
[in]abortInfoアボートに関する情報

詳細
アボートオブザーバは、アボート時に呼び出されるコールバック関数です。

複数のスレッドで同時に呼び出される可能性がありますので、再入可能である必要があります。

例外安全でない関数で呼び出される可能性がありますので、例外を発生させてはいけません。

関数内では、ログの出力などの処理を行うことが想定されています。

アボートオブザーバが、return されない場合の挙動は未定義です。

注意
アボートオブザーバは、製品環境でも無効化されません。
typedef AssertionFailureOperation(* nn::diag::AssertionFailureHandler) (const AssertionInfo &assertionInfo)

NN_ASSERT 系列のマクロに与えた条件が満たされない場合に呼び出されるコールバック関数の型です。

引数
[in]AssertionInfoAssertion に関する情報
戻り値
関数から返った後に指定する処理を nn::diag::AssertionFailureOperation で返します。

詳細
戻り値で、関数から返った後の処理を指定します。通常は、Abort を選択してください。 Continue を選択する場合、NN_ASSERT 系列のマクロに与えた条件が満たされない状態で実行が継続されるため、十分な注意が必要です。

複数のスレッドで同時に呼び出される可能性がありますので、再入可能である必要があります。

例外安全でない関数で呼び出される可能性がありますので、例外を発生させてはいけません。

関数内では、ログの出力などの処理を行うことが想定されています。

typedef void(* nn::diag::LogObserver) (const LogMetaData &metaInfo, const LogBody &logBody, void *argument)

詳細
ログが diag ライブラリのバッファに収まりきらないときは、 ログが分割されて、ログオブザーバが複数回呼び出されます。

引数

int nn::diag::GetBacktrace ( uintptr_t *  pOutArray,
int  arrayCountMax 
)

呼び出したスレッドのバックトレースを取得します。

引数
[out]pOutArrayアドレスを格納する配列
[in]arrayCountMaxpOutArray に格納できる最大個数
戻り値
pOutArray に格納したアドレスの個数
事前条件
  • pOutArray != nullptr
  • arrayCountMax > 0

詳細
pOutArray には、バックトレースにある関数のリターンアドレスが、 直近に呼びだされた順に格納されます。

戻り値が arrayCountMax と等しい場合、古いスタックフレームのアドレスは 切り捨てられている可能性があります。

完全なバックトレースを取得するには、十分に大きな配列を渡してください。

pOutArray には、SDK が内部実装する関数のリターンアドレスも 含まれることに注意してください。

各種例:
BreakpadSimple.cpp, CapstoneSimple.cpp, OsUserExceptionHandlerSample.cpp.

int  arrayCountMax,
uintptr_t  fp,
uintptr_t  sp,
uintptr_t  pc

レジスタの情報からバックトレースを取得します。

引数
[out]pOutArrayアドレスを格納する配列
[in]arrayCountMaxpOutArray に格納できる最大個数
[in]fpバックトレースを取得するスレッドのフレームポインタ
[in]spバックトレースを取得するスレッドのスタックポインタ
[in]pcバックトレースを取得するスレッドのプログラムカウンタ
戻り値
pOutArray に格納したアドレスの個数

  • arrayCountMax > 0

詳細
ユーザー例外ハンドラから、例外が発生したスレッドのバックトレースを取得する用途を想定しています。

ユーザー例外ハンドラの、例外情報から例外が発生したスレッドのレジスタ情報を渡して呼び出してください。

pOutArray[0] には、プログラムカウンタ(pc) の値が格納されます。

pOutArray[1] 以降は、バックトレースにある関数のリターンアドレスが、 直近に呼びだされた順に格納されます。

戻り値が arrayCountMax と等しい場合、古いスタックフレームのアドレスは 切り捨てられている可能性があります。

完全なバックトレースを取得するには、十分に大きな配列を渡してください。

pOutArray には、SDK が内部実装する関数のリターンアドレスも 含まれることに注意してください。

（Windowsプラットフォームのみ）:
Windows 環境では本 API は使用できません。

uintptr_t nn::diag::GetModulePath ( char *  outPathBuffer,
size_t  pathBufferSize,
uintptr_t  address 
)

引数に指定したアドレスを含むモジュールを作成するのに使用された nss ファイルのパスを取得します。

引数
[out]outPathBufferパスを格納するバッファ。
[in]pathBufferSizeパスを格納するバッファのバイトサイズ。
[in]addressモジュールを検索するアドレス。
戻り値
モジュールのアドレス。モジュールが見つからない場合は 0 を返す。
事前条件
  • outPathBuffer != nullptr
  • pathBufferSize > 0
  • address > 0

詳細
outPathBuffer には、取得したモジュールのパスが格納されます。outPathBuffer はヌル終端されます。
パスが、pathBufferSize より長い場合は、pathBufferSize - 1 バイトまで格納されます。
パスの文字コードは、UTF-8 です。
モジュールがシステムモジュールの場合は、パスではなくモジュール名が取得されます。

size_t nn::diag::GetRequiredBufferSizeForGetAllModuleInfo ( )

ロード済み全モジュールの情報取得に必要なバッファサイズを計算して取得します。

戻り値
バッファのバイトサイズ。失敗したときは 0 を返します。

詳細
本関数でバッファサイズを取得してから、 GetAllModuleInfo の実行が完了するまでに、 モジュール構成が変更されるような操作（DLL のロード／アンロード）を行うと、すべてのモジュールの情報が取得できない可能性があります。

各種例:
BreakpadSimple.cpp, OsUserExceptionHandlerSample.cpp.
int nn::diag::GetAllModuleInfo ( ModuleInfo **  pOutModules,
void *  buffer,
size_t  bufferSize 
)

ロード済みの全モジュールの情報を取得します。

引数
[out]pOutModulesbuffer 内の ModuleInfo 構造体配列の先頭要素を指すポインタ。
[in]bufferモジュールの情報を格納するバッファ。
[in]bufferSizeモジュールの情報を格納するバッファのバイトサイズ。
戻り値
情報を取得したモジュールの個数。失敗したときは 0 を返します。
事前条件
  • pOutModules != nullptr
  • buffer != nullptr
  • bufferSize > 0

詳細
本関数で必要なバッファのサイズは、 GetRequiredBufferSizeForGetAllModuleInfo で取得してください。
バッファのサイズが足りないときは、バッファに収まる個数だけ、モジュールの情報が取得されます。

バッファには、ModuleInfo 構造体の配列と、構造体のメンバから参照する情報が格納されます。

各種例:
BreakpadSimple.cpp, OsUserExceptionHandlerSample.cpp.
