詳解

鍵サイズが 128bit の AES の CCM モードによる認証付き復号化を利用するためのクラスです。

詳細
このクラスは以下の状態を持ちます。 インスタンスの生成直後は未初期化状態です。

  • 未初期化状態
  • 追加認証データ処理可能状態
  • 復号化処理可能状態
  • データ入力完了状態
  • 完了状態

鍵サイズが 128bit の AES-CCM モードでの認証付き復号化の処理を初期化します。

引数
[in]pKey鍵データへのポインタ。
[in]keySize鍵データのバイトサイズ。
[in]pNonceNonce データへのポインタ。
[in]nonceSizeNonce データのバイトサイズ。
[in]aadSize追加認証データのバイトサイズ。
[in]dataSize復号化の対象となるデータのバイトサイズ。
[in]macSizeメッセージ認証コードのバイトサイズ。
事前条件
  • keySize == 16
  • nonceSize >= 7 && nonceSize <= 13
  • macSize >= 4 && macSize <= 16 && macSize % 2 == 0
  • aadSize >= 0
  • dataSize >= 0
  • (nonceSize == 7) ? dataSize <= INT64_MAX : dataSize < (1LL << ((15 - nonceSize) * 8))
事後条件
  • aadSize が 0 の場合、インスタンスは復号化処理可能状態になる。
  • aadSize が 0 でない場合、インスタンスは追加認証データ処理可能状態になる。

詳細
各パラメータのサイズなど詳しくは CcmDecryptor::Initialize を参照してください。

void nn::crypto::Aes128CcmDecryptor::UpdateAad ( const void *  pAad,
size_t  aadSize 
)

鍵サイズが 128bit の AES-CCM モードによる追加認証データを入力します。

引数
[in]pAad追加認証データが格納されているバッファへのポインタ。
[in]aadSize入力する追加認証データのバイトサイズ。
事前条件
  • インスタンスは追加認証データ処理可能状態である。
  • UpdateAad によって入力されるデータの合計が Initialize() で指定された aadSize を超えない。
事後条件
  • 追加認証データとして入力されたサイズが aadSize に達したとき、Initialize() で指定された dataSize が 0 でない場合、インスタンスは復号化処理可能状態になる。
  • 追加認証データとして入力されたサイズが aadSize に達したとき、Initialize() で指定された dataSize が 0 の場合、インスタンスはデータ入力完了状態になる。

詳細
追加認証データとは暗号化の対象とならず MAC の対象にのみなるデータです。 pSrc から指定のサイズ分のデータを読み取り、MAC を計算します。

この関数は複数回に分けて呼び出すことができます。 ただし、入力されたデータはすべてが連結されたひとまとまりのデータとして扱われるため、入力する順序が意味を持つことに注意してください。

すべての追加認証データの入力は Update() の処理を開始する前に完了させる必要があります。

size_t  dstSize,

鍵サイズが 128bit の AES-CCM モードによる認証付き復号化を行います。

引数
[out]pDst復号化されたデータを受け取るバッファへのポインタ。
[in]dstSizepDst が指すバッファのバイトサイズ。
[in]pSrc復号化するデータが格納されているバッファへのポインタ。
[in]srcSize復号化するデータのバイトサイズ。
戻り値
実際に復号化されたバイト数
事前条件
  • インスタンスは復号化処理可能状態である。
  • Update によって入力されるデータの合計が Initialize() で指定された dataSize を超えない。
  • dstSize >= srcSize
  • pDst, dstSize で指定する領域と pSrc, srcSize で指定する領域に重なりがない。
事後条件
  • 復号化データとして入力されたサイズの合計が Initialize() で指定された dataSize に達した場合、インスタンスはデータ入力完了状態になる。
  • pDst に復号化された結果が書き込まれる。

詳細
CCM モードではバイト単位で復号化処理が行われるため 1 回の呼び出しで必要になる dstSize は srcSize と同じだけ必要です。

dstSize の合計として必要になるバッファサイズも srcSize の合計と同じだけ必要になります。

void nn::crypto::Aes128CcmDecryptor::GetMac ( void *  pMac,
size_t  macSize 
)

鍵サイズが 128bit の AES-CCM モードによる最終的な MAC を取得します。

引数
[out]pMacMAC を格納するバッファへのポインタ。
[in]macSizepMac で指定されたバッファのバイトサイズ。
事前条件
  • インスタンスはデータ入力完了状態または完了状態である。
  • macSize が Initialize() で指定された macSize 以上である。
事後条件
  • pMac に MAC が書き込まれる。
  • インスタンスは完了状態になる。

詳細
これまでに入力された追加認証データと暗号文データから計算された MAC を取得します。

暗号化では計算した MAC を暗号文、追加認証データとあわせて送信します。 復号化では計算した MAC と受信した MAC との一致を確認することで暗号文、追加認証データに改ざんがないことを検出できます。
