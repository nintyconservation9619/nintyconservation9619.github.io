ストリーミング伸長ライブラリを用いたサンプルプログラムの解説です。

概要

ここでは、ストリーミング伸長ライブラリを用いて圧縮データの ストリーミング伸長を行うサンプルプログラムの説明をします。

ストリーミング伸長ライブラリの使い方については、 ユーティリティライブラリの関数リファレンス も併せて参照して下さい。

ファイル構成

本サンプルプログラムは Samples/Sources/Applications/UtilStreamingDecompression 以下にあります。

128 void DisplayWithStreamingDecompressionZlib(const uint8_t* pComressedData,
129  size_t compressedDataSize)
130 {
131  // 元の圧縮されたデータを差すポインタ
132  const uint8_t* pCompressedDataPtr = pComressedData;
133  // 圧縮データの残りサイズ
134  size_t compressedDataRemainSize = compressedDataSize;
135 
136  const size_t SrcBufSize = 32; // 入力バッファのサイズ
137  char srcBuf[SrcBufSize]; // 一時的に圧縮データを格納する入力バッファ
138  char* pSrcBufCurrent = nullptr; // 入力バッファ上の位置を示すポインタ
139  size_t remainSrcBufSize = 0; // 入力バッファの残りサイズ
140 
141  const size_t DstBufSize = 64; // 出力バッファのサイズ
142  char dstBuf[DstBufSize]; // 一時的に伸長データを格納する出力バッファ
143 
144  size_t dstConsumedSize; // 出力されたデータ量
145  size_t srcConsumedSize; // 伸長に使用されたデータ量
146 
147  // ストリーミング伸長用コンテキストをヒープから取得
148  std::unique_ptr<nn::util::StreamingDecompressZlibContext>
149  pContext(new nn::util::StreamingDecompressZlibContext);
150  NN_ABORT_UNLESS_NOT_NULL(pContext);
151 
152  // ストリーミング伸長用コンテキストを初期化
153  nn::util::InitializeStreamingDecompressZlibContext(pContext.get());
154 
155  // ストリーミング伸長処理のループ
156  for(;;)
157  {
158  // 入力バッファが空になったら圧縮データの続きから読み込む
159  if(remainSrcBufSize == 0)
160  {
161  // 入力バッファサイズもしくは残りの圧縮データサイズの分だけ読み込む
162  size_t inputSize = std::min(compressedDataRemainSize, SrcBufSize);
163 
164  // 簡単のために本サンプルでは最初からメモリ上に配置された圧縮データを読み込む
165  std::memcpy(srcBuf, pCompressedDataPtr, inputSize);
166 
167  pCompressedDataPtr += inputSize;
168  compressedDataRemainSize -= inputSize;
169  pSrcBufCurrent = srcBuf;
170  remainSrcBufSize = inputSize;
171  }
172 
173  // 入力バッファが空になるか出力バッファが満杯になるまでストリーミング伸長
174  // 文字列としてログ出力するために出力バッファサイズは終端文字の分減らして渡す
175  bool isSuccess = nn::util::StreamingDecompressZlib(&dstConsumedSize, &srcConsumedSize,
176  dstBuf, (DstBufSize - sizeof(dstBuf[0])),
177  pSrcBufCurrent, remainSrcBufSize, pContext.get());
178 
179  // ライブラリからの返り値をチェック
180  if(!isSuccess)
181  {
182  // ライブラリが false を返した場合は入力データが壊れているか圧縮形式が異なる
183  NN_LOG("Streaming Decompression (zlib): NG.\n");
184  NN_LOG(" (the input data is corrupted or no supported format.)\n");
185 
186  return ;
187  }
188 
189  // 入出力ともバッファが伸長に使用されていなければ処理は終了
190  if(dstConsumedSize == 0 && srcConsumedSize == 0)
191  {
192  return ;
193  }
194 
195  // 出力バッファに伸長データが入っていればログに出力
196  if(dstConsumedSize > 0)
197  {
198  // 文字列としてログ出力するために得られた伸長結果の後ろに終端文字を挿入
199  NN_ABORT_UNLESS(dstConsumedSize < DstBufSize);
200  dstBuf[dstConsumedSize] = '\0';
201 
202  // 今回のストリーミング伸長で得られた文字列をログ出力

204  }
205 
206  // 続きから入力するために入力バッファへのポインタとサイズを更新
207  pSrcBufCurrent += srcConsumedSize;
208  remainSrcBufSize -= srcConsumedSize;
209  }

サンプルプログラムの解説

サンプルプログラムではある文字列を zlib 形式で圧縮したデータである SampleData のストリーミング伸長を行います。
ストリーミング伸長ではサイズの小さい入力および出力バッファに、 それぞれ圧縮データと伸長結果を分割して格納しながら逐次的に伸長を行うことができます。
ストリーミング伸長ライブラリが対応している他の圧縮形式には gzip 形式と raw deflate 形式があります。

サンプルプログラムの処理の流れは以下の通りです。

  • 伸長元データである SampleData のサイズを表示
    • SampleData の圧縮元データは 圧縮 で圧縮した文字列
    • 圧縮 では gzip 形式で圧縮しているため圧縮データのサイズは異なる
  • ストリーミング伸長用コンテキスト pContext のメモリ領域確保
    • コンテキストの型は nn::util::StreamingDecompressZlibContext
  • ストリーミング伸長ライブラリの nn::util::InitializeStreamingDecompressZlibContext() を呼び出して pContext を初期化
  • ストリーミング伸長のループ処理を実行
    • 入力バッファ srcBuf は 32 Byte 用意
    • 出力バッファ dstBuf は 64 Byte 用意

ストリーミング伸長のループ処理の流れは以下の通りです。

  • srcBuf が空になっているかどうかを判定
    • 空の場合は srcBuf に圧縮データである pComressedData の続きを読み込み
  • ストリーミング伸長ライブラリの nn::util::StreamingDecompressZlib() を呼び出してストリーミング伸長
    • 渡された入力バッファが空になるか出力バッファが満杯になるまで伸長
    • 伸長に使用した入力データサイズ srcConsumedSize と出力データサイズ dstConsumedSize を出力
  • srcConsumedSize と dstConsumedSize が両方とも 0 であるか判定
    • 両方とも 0 の場合はストリーミング伸長完了であるためループを終了
  • dstBuf に格納された伸長データを文字列として表示
  • pSrcBufCurrent と remainSrcBufSize を更新
    • pSrcBufCurrent は srcBuf 上の次に伸長を開始する位置を示すポインタ
    • remainSrcBufSize は srcBuf 内でまだ伸長されていないデータのサイズ

このサンプルプログラムでは簡単のために、ストリームではなく最初からメモリ上に配置された SampleData を std::memcpy() で srcBuf に分割しながら読み込んでいます。
圧縮データのファイルなどを入力バッファに読み込む場合には、該当の記述を変更してください。
