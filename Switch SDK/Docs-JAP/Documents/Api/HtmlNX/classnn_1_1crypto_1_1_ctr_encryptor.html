詳解

template<typename BlockCipher>
class nn::crypto::CtrEncryptor< BlockCipher >

CTR 暗号利用モードによる暗号化を行うクラスです。

テンプレート引数
BlockCipher対象とするブロック暗号の実装クラス

詳細
CTR は NIST SP 800-38A で規定される暗号利用モードです。 このクラスによる実装ではカウンタ値のインクリメント関数として単純なインクリメントを採用しています。

このクラスはブロック暗号の実装クラスをテンプレート引数とするテンプレートクラスです。 実際の使用には Aes128CtrEncryptor などを利用してください。

テンプレートとして指定するブロック暗号クラスは以下の条件を満たす実装になっている必要があります。 各関数の引数など詳細な説明については AesEncryptor クラスを参照してください。

  1. BlockSize という処理するブロックサイズが定義された定数を持つ。
  2. EncryptBlock という単一ブロックを暗号化する関数を持つ。

このクラスは以下の状態を持ちます。 インスタンスの生成直後は未初期化状態です。

  • 未初期化状態
  • 処理可能状態

CTR モードで暗号化の処理を初期化します。

引数
[in]pBlockCipherブロック暗号の実装クラスのインスタンス。
[in]pIvカウンタの初期値データへのポインタ。
[in]ivSizeカウンタの初期値データのバイトサイズ。
事前条件
  • pBlockCipher で指定するインスタンスは EncryptBlock を実行できる。
  • ivSize == IvSize
事後条件
  • インスタンスは処理可能状態になる。

詳細
pBlockCipher には暗号化に使用するブロック暗号を実装するインスタンスを指定します。 あらかじめ鍵の設定など必要な初期化を行い、EncryptBlock が実行できる状態になっている必要があります。

pBlockCipher で指定したインスタンスはコピーされません。 一連の処理が完了するまで対象 pBlockCipher に対する処理は行わないでください。

CTR モードでは IV はカウンタの初期値として扱われます。 カウンタの初期値は内部にコピーされます。

size_t nn::crypto::CtrEncryptor< BlockCipher >::Update ( void *  pDst,
size_t  dstSize,
const void *  pSrc,

)

CTR モードで暗号化を行います。

引数
[out]pDst暗号化されたデータを受け取るバッファへのポインタ。
[in]dstSizepDst が指すバッファのバイトサイズ。
[in]pSrc暗号化するデータが格納されているバッファへのポインタ。
[in]srcSize暗号化するデータのバイトサイズ。
戻り値
実際に暗号化されたバイト数
事前条件
  • インスタンスは処理可能状態である。
  • dstSize >= srcSize
  • pDst, dstSize で指定する領域と pSrc, srcSize で指定する領域に重なりがない。
事後条件
  • pDst に暗号化された結果が書き込まれる。

詳細
CTR モードではバイト単位で暗号化処理が行われるため 1 回の呼び出しで必要になる dstSize は srcSize と同じだけ必要です。

dstSize の合計として必要になるバッファサイズも srcSize の合計と同じだけ必要になります。
