詳解

振動ファイルを再生することで振動値の発生源としての機能を提供する振動ノードです。

詳細
VibrationPlayer は、あらかじめメモリ上に展開された振動ファイルのバイナリデータを ロードして再生することによって振動を発生させる機能を提供する振動ノードです。
内部的には、 nn::hid::ParseVibrationFile 関数を利用して振動ファイルをパースし、 nn::hid::RetrieveVibrationValue 関数を利用して振動ファイルから振動値を順次取り出します。

振動ファイルから取り出された振動値は、 nn::hid::VibrationNode::Update が呼ばれるたびに 接続先の振動ノードに伝えられます。

void nn::hid::VibrationPlayer::Play ( )

VibrationPlayer にロードされた振動ファイルの再生を開始します。

事前条件
  • IsLoaded() == true
  • IsPlaying() == false
  • 0 <= GetCurrentPosition() < GetFileInfo().sampleLength
  • IsLoop() == true の場合には、 0 <= GetLoopStartPosition() < GetLoopEndPosition() <= ロードされている振動ファイルのサンプル長
事後条件
  • IsPlaying() == true

詳細
VibrationPlayer にロードされた振動ファイルを CurrentPosition から順に デコードして振動値を生成します。
VibrationPlayer が別の振動ノードに接続されている場合には、 nn::hid::VibrationNode::Update が呼ばれるたびに 接続先の振動ノードに対して生成された振動値が伝えられます。

IsLoop() == false の場合には、 振動ファイル内の振動サンプル末尾まで再生が完了した時点で自動的に再生を停止して、 IsPlaying() == false かつ GetCurrentPosition() == 0 となります。
IsLoop() == true の場合には、 明示的に Stop() を呼ばない限り振動ファイルがループ再生され続けます。

static void nn::hid::VibrationNode::Update ( )
staticinherited

振動値フローの有向グラフ全体を更新して、 それぞれの振動ノードが接続先に振動値を伝えます。

詳細
この関数が呼ばれると、すべての振動ノードの内部状態が更新されます。
その際、隣接している振動ノード間では、接続元から接続先に対して振動値が伝えられます。
接続元の更新処理は接続先の更新処理よりも先に行われます。
この関数は任意のタイミングで実行することができますが、 タイマー等を利用して、DefaultVibrationSampleInterval 間隔で 実行することが推奨されます。

本関数はスレッドセーフではありません。
本関数と本クラスおよび派生クラスのコンストラクタおよびデストラクタを同時に呼ばないように排他制御をしてください。
