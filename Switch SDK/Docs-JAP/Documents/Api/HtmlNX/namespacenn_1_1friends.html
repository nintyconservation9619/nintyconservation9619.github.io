enum nn::friends::PresenceStatus

プレゼンスの状態を表します。

詳細
プレゼンスの状態は、以下のように遷移します。

  • ユーザーアカウント Open 時
    選択したユーザーアカウントのプレゼンスの状態が、オンラインに遷移します。
  • ユーザーアカウント Close 時
    選択したユーザーアカウントのプレゼンスの状態が、オフラインに遷移します。
  • オンラインプレイ開始の宣言時
    選択したユーザーアカウントのプレゼンスの状態が、オンラインプレイ中に遷移します。
  • オンラインプレイ終了の宣言時
    選択したユーザーアカウントのプレゼンスの状態が、オンラインに遷移します。
  • ネットワーク接続時
    Open 中のユーザーアカウントのプレゼンスの状態が、オフラインであればオンラインに遷移します。
  • ネットワーク切断時
    全ユーザーアカウントのプレゼンスの状態が、オフラインに遷移します。
    オンラインプレイの開始宣言は、自動的に取り下げられます。（＝終了を宣言）
列挙値
PresenceStatus_Offline 

オフライン。
フレンドからは、ゲームで遊んでいないように見えます。

PresenceStatus_Online 

オンライン。
フレンドからは、ゲームで遊んでいるに見えます。

PresenceStatus_OnlinePlay 

オンラインプレイ中。
フレンドからは、ネットワーク対戦や協力プレイ等、オンラインで遊んでいるように見えます。

nn::Result nn::friends::GetFriendList ( int *  outCount,
nn::account::NetworkServiceAccountId *  outAccountIds,
const nn::account::Uid &  uid,
int  offset,
int  count,
const FriendFilter &  filter 
)

フレンドリストを取得します。（ネットワークサービスアカウント ID のみ）

引数
[out]outCount取得した数。
[out]outAccountIdsフレンドリスト。
[in]uidユーザーアカウント。
[in]offsetオフセット。
[in]countフレンドリストの要素数。
[in]filterフレンドリスト取得のフィルター。
戻り値
処理結果。
事前条件
  • outCount != nullptr
  • outAccountIds != nullptr
  • offset >= 0
  • count > 0
  • 指定したユーザーアカウントが Open 状態である。

詳細
指定したオフセットから指定した数のフレンドのネットワークサービスアカウント ID を取得します。
フィルターを指定することで、条件に合致したフレンドのみ取得することができます。

本関数は本体ストレージ上のキャッシュからフレンドリストを取得します。
そのため、ネットワーク未接続時でもフレンドリストを取得することができます。

フレンドサーバーと通信可能な状態であれば、本体ストレージ上のキャッシュは定期的に更新されます。

各種例:
FriendsPresence.cpp, FriendsSimple.cpp.

nn::Result nn::friends::DeclareOpenOnlinePlaySession ( const nn::account::Uid &  uid)

オンラインプレイの開始を宣言します。

引数
[in]uidユーザーアカウント。
戻り値
処理結果。
事前条件
  • 指定したユーザーアカウントが Open 状態である。

詳細
オンラインプレイの開始を宣言した場合、プレゼンスの状態が PresenceStatus_Online から PresenceStatus_OnlinePlay に遷移します。
プレゼンスの状態が PresenceStatus_Offline の場合、 PresenceStatus_OnlinePlay には遷移しません。

プレゼンスの状態が PresenceStatus_Online になるのは、ユーザーアカウントを Open したタイミングです。
ただし、プレイ状況を非公開にしている場合、 PresenceStatus_Online にはなりません。

オンラインプレイの終了を宣言したい場合、 DeclareCloseOnlinePlaySession を使用してください。

オンラインプレイの開始の宣言とプレゼンスの更新を確実に同時に行いたい場合、 UserPresence::DeclareOpenOnlinePlaySession を使用してください。

参照
  • PresenceStatus
  • UserPresence::DeclareOpenOnlinePlaySession

nn::Result nn::friends::AddPlayHistory ( const nn::account::Uid &  uid,
const PlayHistoryRegistrationKey &  key,
const InAppScreenName &  inAppScreenName,
const InAppScreenName &  myInAppScreenName 
)

いっしょに遊んだユーザーを登録します。

引数
[in]uidユーザーアカウント。
[in]key相手の登録キー。
[in]inAppScreenName相手のアプリ内表示名。
[in]myInAppScreenName自分のアプリ内表示名。
戻り値
処理結果。
事前条件
  • 指定したユーザーアカウントが Open 状態である。

詳細
アプリケーションは、以下の手順でいっしょに遊んだユーザーの登録を行います。

  1. 登録キーの取得
    GetPlayHistoryRegistrationKey で自身の登録キーを取得します。
  2. セッションの構築
  3. 登録キーとアプリ内表示名の配布
    最初に取得した登録キーとアプリ内表示名をセッション内の全ユーザーに配布します。
  4. 登録キーとアプリ内表示名の登録
    本関数を利用し、セッション内の全ユーザーの登録キーとアプリ内表示名を登録します。

オンラインプレイの場合、セッションとは "P2P" セッションを指します。（マッチメイクセッションではありません）
これは、実際に遊べるのは P2P セッションを構築したユーザー同士であり、マッチメイクセッションを構築した段階では遊べるかどうか分からないためです。
