メモリ―フェンスについて

一部のプロセッサでは、プログラムの高速実行を実現するために、 プログラムの命令順序を動的に入れ替えて実行するアウトオブオーダーの 仕組みを搭載しているものがあります。

命令実行順序の入れ替えによって、プログラム中のメモリアクセス順序も 入れ替えられて実行されますが、マルチプロセッサ環境においては、 各プロセッサコアによってこのメモリアクセス順序が違って観測される という現象が発生します。

メモリフェンス機能は、このようなメモリアクセス命令の実行順序の入れ替えを 制限するためのものです。実行順序の入れ替えが制限されたメモリアクセスは、 他のプロセッサコアからも同じメモリアクセス順序で観測されるようになります。

本 OS ライブラリでは以下に示す 9 つのメモリフェンス API を提供しています。

  • nn::os::FenceMemoryStoreStore()
  • nn::os::FenceMemoryStoreLoad()
  • nn::os::FenceMemoryStoreAny()
  • nn::os::FenceMemoryLoadStore()
  • nn::os::FenceMemoryLoadLoad()
  • nn::os::FenceMemoryLoadAny()
  • nn::os::FenceMemoryAnyStore()
  • nn::os::FenceMemoryAnyLoad()
  • nn::os::FenceMemoryAnyAny()

各 API は nn::os::FenceMemoryXxxxYyyy() という名前になっており、 Xxxx および Yyyy には Store, Load, Any のメモリアクセスの種類を表す名前が 入ります。ここで、Any は Store と Load のいずれでも良いことを示しています。

Xxxx はメモリフェンス API よりも前に配置されたメモリアクセスの種類を、 Yyyy はメモリフェンス API よりも後ろに配置されたメモリアクセスの種類を 表しており、上記の API はこの両者のメモリアクセス順序が入れ替えられて しまうことを制限し、そのメモリアクセス順序を保証します。

例えば nn::os::FenceMemoryLoadStore() は、メモリフェンス API よりも 前に配置されたメモリロード命令が、メモリフェンス API よりも後ろに 配置されたメモリストア命令よりも後に実行されないように制限をかけます。 言い換えると、前に配置されているメモリロード命令は、後ろに配置されている メモリストア命令よりも先に実行を完了することが保証されます。

これら 9 つの API は、各ターゲットプロセッサに応じて最適な機械語命令を 使って実装されています。API の名前に完全に一致する機械語命令が用意 されていない場合は、より強いメモリアクセス制限を行なう機械語命令を 使用するため、API が示すメモリアクセス順序は正しく保証されます。
