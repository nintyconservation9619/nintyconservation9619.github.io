詳解

template<size_t KeySize, typename HashFunction>
class nn::crypto::RsaPssSigner< KeySize, HashFunction >

RSA-PSS 署名の生成を行うクラスです。

テンプレート引数
KeySize使用する鍵のバイトサイズ
HashFunction使用するハッシュ関数の実装クラス

詳細
RSA-PSS 署名は PKCS #1 で規定される RSA 署名アルゴリズムを利用した署名方式です。

このクラスは鍵サイズとハッシュ関数の実装クラスをテンプレート引数とするテンプレートクラスです。 実際の使用には Rsa2048PssSha256Signer などを利用してください。

鍵のサイズは 1 バイト (8 ビット) 単位で最大 512 バイト (4096 ビット) まで指定できます。

テンプレートとして指定するハッシュ関数クラスは以下の条件を満たす実装になっている必要があります。 各関数の引数など詳細な説明については Sha1Generator クラスなどを参照してください。

  1. BlockSize という処理するブロックサイズが定義された定数を持つ
  2. HashSize というハッシュ値の出力サイズが定義された定数を持つ
  3. Initialize という処理を開始する関数を持つ
  4. Update という入力でハッシュ値を更新する関数を持つ
  5. GetHash という最終的なハッシュ値を取得する関数を持つ

このクラスは以下の状態を持ちます。 インスタンスの生成直後は未初期化状態です。

  • 未初期化状態
  • 処理可能状態
  • 完了状態

対象のデータのハッシュ計算はクラスの内部で行われます。

この関数は複数回に分けて呼び出すことができます。 ただし、入力されたデータはすべてが連結されたひとまとまりのデータとして扱われるため、入力する順序が意味を持つことに注意してください。

template<size_t KeySize, typename HashFunction >
bool nn::crypto::RsaPssSigner< KeySize, HashFunction >::Sign ( void *  pOutSignature,
size_t  signatureSize,
const void *  pSalt,
size_t  saltSize 
)

署名を生成します。作業領域としてスタックを利用します。

引数
[out]pOutSignature署名データが格納されるバッファへのポインタ。
[in]signatureSizepOutSignature が指すバッファのバイトサイズ。
[in]pSaltSalt データへのポインタ。
[in]saltSizeSalt データのバイトサイズ。
戻り値
署名の生成に成功したかを返します。
失敗した場合は不正なパラメータが与えられています。
事前条件
  • インスタンスは処理可能状態である。
  • signatureSize == SignatureSize
  • 0 < saltSize && saltSize <= (SignatureSize - HashSize - 2)
事後条件
  • pOutSignature に署名が格納される。
  • インスタンスは完了状態になる。

詳細
これまでに入力されたデータに対して RSA-PSS 署名を生成します。

Salt データには署名の生成ごとに異なる乱数を指定してください。 DefaultSaltSize のサイズが推奨されています。

ワークバッファとしてスタックから RequiredWorkBufferSize バイトが利用されます。
