シーズンパス

追加コンテンツの機能を使ってシーズンパスを実現する方法を示すサンプルの解説です。

はじめに

ここでは、追加コンテンツの機能を使ってシーズンパスを実現する方法を示すサンプルの解説します。
追加コンテンツ開発機能のモジュールは nn::aoc の名前空間にあります。

以下の手順でシーズンパスを実現することができます。

  1. シーズンパスの販売開始時には、将来配信するコンテンツにはダミーのデータを作成する。
  2. シーズンパスとしてのデータの追加配信は、ダミーとして登録していたコンテンツを本物のデータに更新することで実現する。

詳しくは、「Nintendo Switch eコマースガイド」の「5.2.1. シーズンパスの実現方法」をご参照ください。

シーズンパスのサンプルは、「販売開始時」(上記 1.)と「コンテンツ配信時」(上記 2.)の二つの構成となっており、以下の内容となっています。

フォルダ名 内容
01_SalesStart シーズンパスを利用するソフト販売開始時のサンプルプログラム
02_Deliveryシーズンパスを利用するコンテンツ配信時のサンプルプログラム
