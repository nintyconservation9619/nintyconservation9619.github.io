nn::fontll::ScalableFontEngine::ScalableFontEngine ( )

コンストラクタです。

詳細
実質的な初期化はInitializeで行われます。ここではメンバ変数の初期化のみ行っています。

本クラスは複数スレッドからの利用を考慮した排他制御を内部で行いません。
複数スレッドから同時に利用する必要がある場合、ユーザコード側で排他制御を行うかスレッド毎にインスタンスを作成するようにしてください。

関数詳解

int nn::fontll::ScalableFontEngine::Initialize ( void *  pHeap,
uint32_t  heapSize 
)

初期化を行います。

引数
[in]pHeapワークメモリへのポインタ
[in]heapSizeワークメモリのサイズ

詳細
ワークメモリは以下の用途に利用されます。

・読み込まれたフォントのリスト
・読み込まれたフォント情報
・スケール処理された、フォント情報とキャッシュ
・グリフ（文字）ビットマップのキャッシュ

メモリが不足した場合は、自動的に内部キャッシュを開放して柔軟にやりくりをします。
完全にメモリが枯渇した場合には、201(nn::fontll::ErrorMemoryAllocationFail)のエラーコードが返ります。 ワークメモリの適切サイズは利用状況（フォントの種類、同時利用文字数、フォントサイズ）によるので一概に記述しにくいです。
64ピクセル程度のサイズなら 最低 100kバイト以下でも動作します。
パフォーマンスを実測しながら試行錯誤されることをお勧めします。

戻り値
リザルトコードです。
成功すると Success が返ります。

フォントを読み込みます。

引数
[in]pNameBuffer内部管理用のnull終端されたフォント名文字列を出力するバッファです。
[in]pMemフォントファイルデータへのポインタ。
[in]indexフォントの番号。ttc の場合、使用するフォントのインデックスを指定します。その他の場合は必ず 0 を指定します。
[in]nameBufferLenフォント名文字列を出力するバッファ（pNameBuffer）の長さです。最小で2、最大で FontNameLengthMax を指定します。

詳細
対応するスケーラブルフォントの種類は以下です。

  • 本体内蔵フォント(bfttf)
  • TrueType(ttf)
  • TrueType Collection(ttc)
  • OpenType(otf)


bfttf ファイルを読み込む場合は、ScalableFontEngineHelper::Decode でデコードしてから渡すようにしてください。

void nn::fontll::ScalableFontEngine::SetAutoHint ( bool  isAutoHint)

ヒンティングを強制するかどうかを設定します。

通常は常に false に設定されています。 特定のフォントで描画が崩れたりクラッシュする場合、 AcquireGlyphmap メソッドの直前で true に設定すると直ります。 ただし true にすることで、描画結果がピクセル単位で若干変わります。 本来フォントが持つヒント情報に基づいて フォント描画の計算していますが、そのときの演算の丸め誤差が原因して 特定のフォントでクラッシュすることが確認されています。 このメソッドに true を渡すことで ヒント情報をランタイム側で計算するようになり、 これらのクラッシュを回避することができます。

引数
[in]isAutoHintヒンティングを強制するかどうかです。
