詳解

通知キューを扱うクラスです。

詳細
本クラスは、ユーザーアカウントごとに作成する通知キューの管理を行います。

通知キューは最大 NotificationCountMax 個の通知を保持します。
通知キューが満杯の時、 NotificationType_FriendPresenceUpdated の通知はすべて削除され、代わりに NotificationType_FriendListUpdated の通知 1 つに置き換わります。

同一内容の通知は 1 つにまとめられます。
例えば、あるフレンドが複数回プレゼンスを更新しても、通知キューに貯まる通知は 1 つです。
アプリケーションが NotificationQueue::Pop で通知を取得すると、それと同一内容の通知が再び追加されるようになります。

各種例:
FriendsPresence.cpp.

nn::Result nn::friends::NotificationQueue::Initialize ( )

初期化します。（ユーザーアカウント自動選択）

戻り値
処理結果。
事前条件
  • 未初期化である。
  • Open 状態のユーザーアカウントが 1 人である。

詳細
本関数は、Open 状態のユーザーアカウントが 1 人の場合のみ、自動的にそのユーザーを選択します。
Open 状態のユーザーアカウントが 1 人以外の場合、ASSERT して失敗します。

ユーザーアカウントを指定しないこと以外は、標準の NotificationQueue::Initialize と同じです。

void nn::friends::NotificationQueue::Finalize ( )

終了します。

詳細
本関数は、システムイベントの破棄を行います。
本関数の呼び出し以降、 NotificationQueue::GetSystemEvent で取得したシステムイベントを使用してはいけません。

終了処理後、本オブジェクトは未初期化状態になります。

各種例:
FriendsPresence.cpp.

nn::Result nn::friends::NotificationQueue::Pop ( NotificationInfo *  outInfo)

通知を取得します。

引数
[out]outInfo通知。
戻り値
処理結果。
戻り値一覧:
ResultNotificationNotFound 指定したユーザー宛ての通知はありません。
事前条件
  • outInfo != nullptr

詳細
通知キューに貯め込まれた通知を 1 つ取得します。

1 回のシステムイベントの Signal で複数の通知が追加されている場合があります。
そのため、 ResultNotificationNotFound が返るまで本関数を繰り返し呼び出すことを推奨します。
