nn::Result nn::nfc::AttachActivateEvent ( nn::os::SystemEventType *  pEvent,
const DeviceHandle &  deviceHandle 
)

タグの発見を通知するイベントを設定します。

引数
[in,out]pEventタグ発見通知に使用するイベントを指定します。
[in]deviceHandleNFC デバイスのハンドル
戻り値
処理の結果を表す nn::Result 値を返します。
  • ResultSuccess 設定に成功しました。
  • ResultNfcDeviceNotFound 正しいステートの NFC デバイスが見つかりません。ListDevices() による NFC デバイスのハンドル取得からやり直す必要があります。
  • ResultNfcDisabled NFC 機能が無効になっています。NFC 機能が有効になった後、再試行する必要があります。
事前条件
  • pEvent はアクセス可能なアドレスを指している必要があります。
  • GetState() == State_Init
事後条件
指定した NFC デバイスにて、タグ発見時に pEvent がシグナルされるようになります。

詳細
NFC ライブラリの初期化後、NFC デバイス毎に StartDetection() でタグの検知を開始する前に実行してください。 指定した NFC デバイスにて、タグが検出されると、この API で設定したイベントがシグナル化されます。

イベントは API 内部で初期化されるため、ユーザー側で初期化を行う必要はありません。 イベントは自動クリアモードで初期化されます。

イベントが不要になった場合は nn::os::DestroySystemEvent() を呼び出して必ず破棄してください。

void nn::nfc::AttachAvailabilityChangeEvent ( nn::os::SystemEventType *  pEvent)

NFC デバイスの利用可/不可の変化を通知するイベントを設定します。

引数
[in,out]pEvent通知に使用するイベントを指定します。
事前条件
  • pEvent はアクセス可能なアドレスを指している必要があります。
  • GetState() == State_Init
事後条件
NFC デバイスの利用可/不可が変化した場合に pEvent がシグナルされるようになります。

詳細
NFC ライブラリの初期化後に実行してください。
以下のタイミングで、この API で設定したイベントがシグナル化されます。

  • NFC デバイスが切断された
  • NFC デバイスが接続された
  • フォーカスのあるアプリケーションが変わった
  • 本体がスリープから復帰した
  • NFC 機能が無効になった
  • NFC 機能が有効になった

シグナルされたときに ListDevices() や GetDeviceState() を呼ぶことで、効率的に NFC デバイスの状態を確認することができます。
イベントは API 内部で初期化されるため、ユーザー側で初期化を行う必要はありません。 イベントは自動クリアモードで初期化されます。

イベントが不要になった場合は nn::os::DestroySystemEvent() を呼び出して必ず破棄してください。

void nn::nfc::Finalize ( )

NFC ライブラリを終了します。

事前条件
  • ライブラリが Initialize() にて初期化済みの状態である必要があります。
  • GetState() == State_Init
事後条件

詳細

nn::Result nn::nfc::ReadMifare ( MifareReadBlockData *  pOutBlockData,
const DeviceHandle &  deviceHandle,
const MifareReadBlockParameter *  pBlockParameter,
size_t  blockCount 
)

MIFARE タグから任意の 1 または複数のブロックを読み込みます。

引数
[out]pOutBlockDataデータの取得先バッファです。blockCount 分必要です。
[in]deviceHandleNFC デバイスのハンドル
[in]pBlockParameter対象となるブロックの情報です。blockCount 分必要です。
[in]blockCount読み込むブロックの数です。
戻り値
処理の結果を表す nn::Result 値を返します。
  • ResultSuccess 読み込みに成功しました。
  • ResultNfcDeviceNotFound 正しいステートの NFC デバイスが見つかりません。ListDevices() による NFC デバイスのハンドル取得からやり直す必要があります。
  • ResultNfcDisabled NFC 機能が無効になっています。NFC 機能が有効になった後、StartDetection() によるタグの検知からやり直す必要があります。
  • ResultAccessError 読み込みに失敗しました。タグへのアクセス方法（鍵やアクセスするアドレス）が正しいか確認し、StartDetection() によるタグの検知からやり直す必要があります。
  • ResultNeedRestart 読み込みに失敗しました。読み込み対象のタグを喪失した可能性があります。StartDetection() によるタグの検知からやり直す必要があります。
  • ResultNotSupported MIFARE タグではありません。使用しているタグを確認する必要があります。
事前条件
  • pOutBlockData はアクセス可能なアドレスを指している必要があります。
  • pBlockParameter はアクセス可能なアドレスを指している必要があります。
  • 0 < blockCount
  • GetState() == State_Init
  • 認証に使う全ての鍵の MifareKeyValueFormat が同じである必要があります。

詳細
MIFARE タグから任意の 1 または複数のブロックを読み込みます。
タグが検知済みで喪失されていない必要があります。(状態が正しくない場合、ResultNfcDeviceNotFound が返ります。)

  • ResultNfcDisabled NFC 機能が無効になっています。NFC 機能が有効になった後、StartDetection() によるタグの検知からやり直す必要があります。
  • ResultAccessError 送信に失敗しました。タグへのアクセス方法（コマンドの内容やレスポンス最大待ち時間など）が正しいか確認し、StartDetection() によるタグの検知からやり直す必要があります。
  • ResultNeedRestart 送信に失敗しました。送信対象のタグを喪失した可能性があります。StartDetection() によるタグの検知からやり直す必要があります。
  • ResultNotSupported サポートしている NFC タグではありません。使用しているタグを確認する必要があります。
事前条件
  • pOutBuffer はアクセス可能なアドレスを指している必要があります。
  • pOutSize はアクセス可能なアドレスを指している必要があります。
  • 0 < bufferSize
  • pData はアクセス可能なアドレスを指している必要があります。
  • 0 < dataSize
  • GetState() == State_Init

詳細
タグにコマンドをパススルー送信します。
タグが検知済みで喪失されていない必要があります。(状態が正しくない場合、ResultNfcDeviceNotFound が返ります。)
実際に取得できたレスポンスデータのサイズを超えるレスポンスデータバッファの中身は保証されません。参照しないでください。

nn::Result nn::nfc::KeepPassThroughSession ( const DeviceHandle &  deviceHandle)

パススルー送信で使用するセッションの維持を行います。

引数
[in]deviceHandleNFC デバイスのハンドル
戻り値
処理の結果を表す nn::Result 値を返します。
  • ResultSuccess セッション維持に成功しました。
  • ResultNfcDeviceNotFound 正しいステートの NFC デバイスが見つかりません。ListDevices() による NFC デバイスのハンドル取得からやり直す必要があります。
  • ResultNfcDisabled NFC 機能が無効になっています。NFC 機能が有効になった後、StartDetection() によるタグの検知からやり直す必要があります。
  • ResultNeedRestart セッション維持に失敗しました。取得対象のタグを喪失した可能性があります。StartDetection() によるタグの検知からやり直す必要があります。
事前条件
  • GetState() == State_Init

詳細
タグが検知済みで喪失もセッション維持もされていない必要があります。(状態が正しくない場合、ResultNfcDeviceNotFound が返ります。)
本 API を実行していない状態で、SendCommandByPassThrough() を複数回呼び出した場合、各コマンド送信で利用されるセッションは同一であるとは限りません。
同一であることを保証する必要がある場合は、本 API を実行した後、SendCommandByPassThrough() を呼び出してください。
例えば、事前にパスワード認証が必要なコマンドを送るような場合、
タグの検出後、本 API を実行し、「パスワード認証のコマンド」、「パスワード認証が必要なコマンド」の順に送信するような使い方を想定しています。
本 API を実行したのち、セッションの維持が不要になった際には、
ReleasePassThroughSession() を実行して、セッション維持をしない状態に戻してください。
セッション維持をしない状態と比べ、セッション維持中は、消費電力の増加、および、タグの喪失を通知するイベントが発生しないという制限がありますので、セッション維持は必要最小限にとどめてください。
