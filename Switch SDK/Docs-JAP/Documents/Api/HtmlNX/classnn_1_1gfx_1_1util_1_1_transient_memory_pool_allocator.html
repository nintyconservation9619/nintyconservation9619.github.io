詳解

一時的に使用するメモリプール上のメモリを管理するアロケータです。

詳細
描画処理で一時的に使用するメモリプール上のメモリを動的に確保し、 描画の完了後にまとめて解放するための機能を提供します。

関数詳解

void nn::gfx::util::TransientMemoryPoolAllocator::Initialize ( nn::gfx::MemoryPool *  pMemoryPool,
ptrdiff_t  baseOffset,
size_t  size,
size_t  allocatableAlignmentMax,
int  historyLengthMax 
)

メモリプールと管理対象にするオフセットの範囲を指定して初期化します。

引数
[in]pMemoryPoolメモリプールです。
[in]baseOffset管理するメモリの先頭のオフセットです。
[in]size管理するメモリのサイズ（バイト）です。
[in]allocatableAlignmentMaxこのアロケータから確保するメモリに要求する最大のアライメントです。
[in]historyLengthMax記録する履歴の最大数です。
事前条件
IsInitialized() == false
pMemoryPool は有効なメモリプールのポインタでなければなりません。
baseOffset >= 0
baseOffset % allocatableAlignmentMax == 0
size >= 1 && size <= SizeMax
allocatableAlignmentMax <= AlignmentMax
allocatableAlignmentMax は 2 の累乗でなければなりません。
historyLengthMax >= 1
SetMemory() によって必要な大きさとアライメントを満たすメモリブロックが設定されていなければなりません。
事後条件
IsInitialized() == true
IsRecording() == false
GetHistoryLength() == 0

詳細
メモリプール上の指定した範囲のメモリ領域をこのオブジェクトの管理対象とします。 メモリ領域は連続した有効な範囲を指定する必要があります。

baseOffset は allocatableAlignmentMax の整数倍でなければなりません。 allocatableAlignmentMax で指定した値以下のアライメントを Allocate() に指定できます。

同一インスタンスに対するこの関数の呼び出しはスレッドセーフではありません。

void nn::gfx::util::TransientMemoryPoolAllocator::Free ( )

このオブジェクトが記録している履歴のうち最も古いものに記録されているメモリを解放します。

事前条件
IsInitialized() == true
GetHistoryLength() > 0
事後条件
最も古い履歴に記録されているメモリを解放します。
GetHistoryLength() の返す値が 1 減少します。

詳細
メモリの解放は Begin() の呼びだしから End() の呼び出しまでを 1 単位として行われます。 各々の Allocate() で確保したメモリを個別に解放することはできません。
解放されたメモリは直ちに確保可能な状態になります。
同一インスタンスに対するこの関数の複数回の呼び出しはスレッドセーフではありません。 同一インスタンスに対するこの関数とメモリの確保を行う関数の同時呼出しはスレッドセーフです。

End() の呼び出しにより履歴が 1 つ増加します。 Free() の呼び出しにより履歴が 1 つ減少します。 この関数が GetHistoryLengthMax() と同じ値を返す場合、 Begin() は失敗します。

void nn::gfx::util::TransientMemoryPoolAllocator::FillHistory ( )

履歴を確保数 0 の履歴で埋めます。

事前条件
IsInitialized() == true
IsRecording() == false
事後条件
GetHistoryLength() == GetHistoryLengthMax()

詳細
履歴がいっぱいになるまで確保数 0 の履歴を追加します。 以下のコードと等価です。

TransientMemoryPoolAllocator transient;
while(transient.GetHistoryLength() < transient.GetHistoryLengthMax())
{
transient.Begin();
transient.End();
}

既に履歴が記録されている場合、この関数呼び出しによって追加される履歴がより新しい記録となります。
同一インスタンスに対するこの関数の呼び出しはスレッドセーフではありません。
