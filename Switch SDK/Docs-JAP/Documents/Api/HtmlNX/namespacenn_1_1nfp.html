nn::Result nn::nfp::StartNicknameAndOwnerSettings ( DeviceHandle *  pOutDeviceHandle,
bool *  pOutIsRegistered,

const TagInfo &  tagInfo,

ニックネームとオーナーの設定を行う amiibo 設定を開始します。

引数
[out]pOutDeviceHandleタグを更新した NFC デバイスのハンドルを格納するバッファへのポインタを指定します。
[out]pOutIsRegistered更新後の登録情報の有無を格納するバッファへのポインタを指定します。
[out]pOutRegisterInfo更新後の登録情報を格納するバッファへのポインタを指定します。pOutIsRegistered にて登録情報が「有り」を示す場合のみ参照してください。
[in]startParamamiibo 設定開始時必要な共通パラメータです。
[in]tagInfo更新の対象となるタグを示す情報です。
[in]registerInfo本 API 呼び出し時点での登録情報です。GetRegisterInfo 関数で取得した値を指定してください。
事前条件
  • GetState() == State_Init
  • pOutDeviceHandle はアクセス可能なアドレスを指している必要があります。
  • pOutIsRegistered はアクセス可能なアドレスを指している必要があります。
  • pOutRegisterInfo はアクセス可能なアドレスを指している必要があります。
戻り値
処理の結果を表す nn::Result 値を返します。
  • ResultSuccess タグの更新に成功しました。
  • ResultNotUpdated タグは更新されていません。
詳細
ニックネームとオーナーの設定を行う amiibo 設定を開始します。本 API は、amiibo 設定が完了するまで戻りません。
ResultNotUpdated が返ってきた場合、pOutDeviceHandle 、pOutIsRegistered および pOutRegisterInfo には有効な情報が格納されていませんので、参照しないでください。
registerInfo は、 amiibo 設定開始後に改めてタグを読み込むことなく、設定前のニックネームとオーナーを参照するために利用されます。登録情報が登録されている場合には、必ず指定してください。
登録情報が未登録の場合は、本 API ではなく、RegisterInfo を引数に指定しない
StartNicknameAndOwnerSettings(DeviceHandle* pOutDeviceHandle, bool* pOutIsRegistered, RegisterInfo* pOutRegisterInfo, const AmiiboSettingsStartParam& startParam, const TagInfo& tagInfo) を利用してください。

処理の結果を表す nn::Result 値を返します。
  • ResultSuccess 作成に成功しました。
  • ResultNfcDeviceNotFound 正しいステートの NFC デバイスが見つかりません。ListDevices() による NFC デバイスのハンドル取得からやり直す必要があります。
  • ResultNfcDisabled NFC 機能が無効になっています。NFC 機能が有効になった後、StartDetection() によるタグの検知からやり直す必要があります。
  • ResultNeedRestart 作成に失敗しました。作成対象のタグを喪失した可能性があります。StartDetection() によるタグの検知からやり直す必要があります。
事前条件
  • 0 < createInfo.initialDataSize && createInfo.initialDataSize <= アプリケーション専用領域サイズ

詳細
この API によって、既に作成済みのアプリケーション専用領域のデータとアクセス ID を変更することができます。ただし、変更前のアクセス ID にて、
OpenApplicationArea() を呼び出してしておく必要があります。(状態が正しくない場合、ResultNfcDeviceNotFound が返ります。)
変更を確定するには、Flush() を呼ぶ必要があります。
なお、アクセス ID が変更されるため、変更後のアクセス ID を知らないアプリケーションからはそのタグのアプリケーション専用領域にアクセスできなくなります。
ユーザーの意思に反してアクセスできなくなることがないように、ユーザの意思を確認する文言を表示したうえで、本 API を使用してください。
