CPU 及び GPU パフォーマンス計測サンプルの解説

概要

nn::perf::LoadMeterCenter を使用して CPU 及び GPU のパフォーマンス計測を行うサンプルです。
初期化や処理計測には nn::perf のマクロを利用しています。

ファイル構成

本サンプルプログラムは Samples/Sources/Applications/PerfSimple 以下にあります。

必要な環境

画面表示が利用可能である必要があります。

操作方法

特にありません。

注意事項

Debug ビルド版では 60fps に収まらずに処理落ちしてしまう場合があります。
これは処理落ちした場合の挙動を示すことを意図しているため不具合ではありません。

実行手順

サンプルプログラムをビルドし、実行してください。

解説

サンプルプログラムの解説

このサンプルではメインスレッドと 3つのサブスレッドを使用して、赤緑青黄の 4つのオブジェクト群を描画します。
また、CPU 及び GPU のパフォーマンス計測を行い、その結果をコンソール及び画面に出力します。

3つのサブスレッドはそれぞれ赤緑青のオブジェクト描画コマンドを生成し、メインスレッドでは黄オブジェクトとその他の描画コマンドを生成します。
各スレッドは以下のように特定の CPU コア上で動作するように設定しています。

  • メインスレッド(黄・その他) → コア0
  • サブスレッド(赤) → コア0
  • サブスレッド(緑) → コア1
  • サブスレッド(青) → コア2

サンプルプログラムの処理の流れは以下の通りです。

  • 以下を初期化 – グラフィックスフレームワーク – 赤青緑オブジェクト用コマンドバッファ – プリミティブレンダラ – デバッグフォント – スレッド – イベント – LoadMeterCenter – 描画オブジェクトの位置
  • 各スレッドの開始
  • ループ開始 – プリミティブレンダラを更新 – 各スレッドで赤青緑のオブジェクト描画コマンド作成 – メインのコマンドを作成 – コマンドの実行
  • ループ開始に戻る
  • 各種オブジェクトを破棄
