詳解

管理領域と確保・解放領域を分離して持つことのできる整数アロケータのクラスです。

詳細
nn::mem::NumberLineAllocator は 0 から最大 0xFFFFFF までのインデックスで表される整数領域から、任意の領域の割り当てを行うことのできるクラスです。
NumberLineAllocator の管理領域は専用のコールバックを用いて管理し、 NumberLineAllocator で扱う領域と分離できます。

型定義メンバ詳解

typedef void(* nn::mem::NumberLineAllocator::UserCallback) (void *userPtr)

確保された領域を解放するときに呼び出されるコールバック関数の型です。

引数
[in]userPtrユーザから渡されるポインタ

詳細
Alloc() 時にこのコールバックを引数として渡した場合、確保した領域が Free() される際に呼び出されます。

bool nn::mem::NumberLineAllocator::AddRange ( int  first,
int  last 
)

指定した領域を確保可能にします。

引数
[in]first確保可能にしたい範囲の先頭インデックス
[in]last確保可能にしたい範囲の末尾インデックス
戻り値
確保可能に設定できた場合は true を、そうでない場合は false を返します。
事前条件
0 <= first < 0x01000000U
0 < last <= 0x01000000U
first < last
[first, last) が確保不可能領域である必要があります。
事後条件
[first, last) が確保可能領域になります。

詳細
指定した領域を確保可能にします。
first から last の範囲が確保可能領域を含む場合は false が返ります。
コンストラクタ時、もしくは Initialize() 時に isThreadSafe 引数に true を渡した場合、本 API はスレッドセーフです。
isThreadSafe 引数に false を渡した場合、もしくは isThreadSafe 引数を指定しないコンストラクタか Initialize() を呼び出して初期化した場合は、スレッドセーフではありません。

int nn::mem::NumberLineAllocator::GetTotalFreeSize ( )

アロケータに存在する空き領域の合計を取得します。

戻り値
空き領域の合計サイズが返ります。
事前条件
アロケータが初期化されている必要があります。

詳細
アロケータで管理している空き領域の合計を取得します。
空き領域が複数存在する場合、実際に確保可能なサイズは合計サイズよりも小さくなります。
実際に確保可能なサイズを取得するには GetAllocatableSize() を使用してください。

コンストラクタ時、もしくは Initialize() 時に isThreadSafe 引数に true を渡した場合、本 API はスレッドセーフです。

int nn::mem::NumberLineAllocator::GetAllocatableSize ( )

アロケータから確保可能な最大サイズを取得します。

戻り値
確保可能なサイズが返ります。
事前条件
アロケータが初期化されている必要があります。

詳細
アロケータから確保可能な最大サイズを取得します。
空き領域が複数存在する場合、その中で最大の領域のサイズを取得します。

本 API で返却されたサイズは必ず確保可能であることを保証しますが、アルゴリズム上、本 API で返却されたサイズ以上の領域が確保可能な場合があります。

コンストラクタ時、もしくは Initialize() 時に isThreadSafe 引数に true を渡した場合、本 API はスレッドセーフです。

void nn::mem::NumberLineAllocator::Dump ( uint64_t  offset,
int  factor 
)

アロケータ内部の情報を表示します。

引数
[in]offset表示時のインデックスの開始番号のずれ
[in]factor区間の長さ 1 を定数倍するための定数
事前条件
アロケータが初期化されている必要があります。

詳細
アロケータの内部状態を出力します。
offset と factor を指定すると、インデックスの開始番号を offset 、アロケータで扱う区間の長さ 1 を factor とした場合のアロケータ内部の情報を表示します。
offset に 0 、 factor に 1 を指定して呼び出すと、アロケータ内部そのままの情報が表示されます。

出力されるフォーマットの詳細については NintendoSDK ドキュメントの NumberLineAllocator の説明を参照してください。
