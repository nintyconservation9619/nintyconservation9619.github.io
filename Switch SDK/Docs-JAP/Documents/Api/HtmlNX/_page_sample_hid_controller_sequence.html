コントローラーサポートアプレットの呼び出し方のサンプルアプリケーションの解説です

概要

コントローラーサポートアプレットの呼び出し方の一例となるサンプルアプリケーションの解説です。

ファイル構成

本サンプルプログラムは Samples/Sources/Applications/HidControllerSequence 以下にあります。

必要な環境

事前に実機とコントローラーをペアリングしてください。

操作方法

サンプルアプリケーションを実行してください。

コントローラーが接続されていない場合、起動直後にコントローラーサポートアプレットが呼び出されます。
コントローラーが接続されている場合、スタート画面が表示されます。
接続中のいずれかのコントローラーでボタン入力をするとタイトル画面に移動します。

タイトル画面では上下の方向ボタン、もしくはスティック操作で項目を移動する事が出来ます。
Aボタンを押すと、』カーソルがあっているボタンに表示されたプレイヤー人数でゲームを開始します。
その際、プレイヤー人数に対してコントローラーの接続数が不足する場合、コントローラーサポートアプレットが呼び出されます。

ゲーム画面では左右の方向ボタンもしくはスティック操作でプレイヤーを操作する事が出来ます。
また、プレイヤーのコントローラーで([Plus] or [Minus])を押すと中断ダイアログが表示されます。
中断ダイアログで「Return to Title」を選択するとタイトル画面に戻ります。

注意事項

コントローラーは十分に充電した状態でお使いください。

実行手順

サンプルアプリケーションをビルドし、実行してください。

解説

サンプルアプリケーションは以下のシーンから構成されます。

  • スタート画面
  • プレイ人数の選択を行うタイトル画面
  • ゲーム画面
  • ダイアログで表示される画面

スタート画面でのコントローラーサポートアプレットの呼び出し方
タイトル画面でのコントローラーサポートアプレットの呼び出し方
ゲーム画面でのコントローラーサポートアプレットの呼び出し方

アプリケーションは、コントローラーサポートアプレットを使わない場合でも、
起動時、復帰時および動作中にコントローラーが不足した際、
独自にコントローラーの準備を案内することを推奨します。

サンプルアプリケーションの全体のフロー図です。
