詳解

template<size_t KeySize, typename HashFunction>
class nn::crypto::RsaOaepEncryptor< KeySize, HashFunction >

RSA-OAEP による暗号化を行うクラスです。

テンプレート引数
KeySize使用する鍵のバイトサイズ
HashFunction使用するハッシュ関数の実装クラス

詳細
RSA-OAEP は PKCS #1 で規定される RSA 暗号アルゴリズムを利用した暗号化方式です。

このクラスは鍵サイズとハッシュ関数の実装クラスをテンプレート引数とするテンプレートクラスです。

鍵のサイズは 1 バイト (8 ビット) 単位で最大 512 バイト (4096 ビット) まで指定できます。

テンプレートとして指定するハッシュ関数クラスは以下の条件を満たす実装になっている必要があります。 各関数の引数など詳細な説明については Sha1Generator クラスなどを参照してください。

  1. BlockSize という処理するブロックサイズが定義された定数を持つ
  2. HashSize というハッシュ値の出力サイズが定義された定数を持つ
  3. Initialize という処理を開始する関数を持つ
  4. Update という入力でハッシュ値を更新する関数を持つ
  5. GetHash という最終的なハッシュ値を取得する関数を持つ

このクラスは以下の状態を持ちます。 インスタンスの生成直後は未初期化状態です。

  • 未初期化状態
  • 処理可能状態
  • 完了状態

詳細
ラベルは暗号化対象のメッセージとは別に付加することができるデータです。

この関数は複数回に分けて呼び出すことができます。 ただし、入力されたデータはすべてが連結されたひとまとまりのデータとして扱われるため、入力する順序が意味を持つことに注意してください。

すべてのラベルの入力は Encrypt() の処理をする前に完了させる必要があります。

ラベルのハッシュ値があらかじめわかっている場合はこの関数のかわりに SetLabelDigest() で入力することができます。

事前条件
  • インスタンスは処理可能状態である。
  • cipherSize == KeySize
  • rawMessageSize <= KeySize - 2 * HashSize - 2
事後条件
  • インスタンスは完了状態になる。
  • pOutCipher に暗号化された結果が書き込まれる。

詳細

Seed データには暗号化を行うごとに異なる HashSize バイトの乱数を指定してください。

ワークバッファとしてスタックから RequiredWorkBufferSize バイトが利用されます。
