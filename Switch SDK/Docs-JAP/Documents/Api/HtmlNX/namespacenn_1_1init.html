void nn::init::InitializeAllocator ( void *  address,
size_t  size,
bool  isCacheEnable 
)

メモリアロケータを初期化します。

引数
[in]addressメモリアロケータが使用するメモリ領域の先頭アドレス
[in]sizeメモリアロケータが使用するメモリ領域のサイズ
[in]isCacheEnableスレッド毎のメモリキャッシュ機能を有効にするか
事前条件
  • プログラムの中で本 API の呼出しが初めてである
  • size > 0

詳細
メモリアロケータを初期化します。

SDK のデフォルト状態では new や malloc は、この関数で初期化されるアロケータ を使って実際のメモリ獲得を行ないます。このため、この関数に指定する size に応じて、new や malloc で獲得できるメモリサイズが決定します。

メモリアロケータでは、その内部実装に nn::mem::StandardAllocator を使用 しています。そのため、本 API の引数で指定する address, size および isCacheEnable は、 以下の API を参考に指定してください。

  • void nn::mem::StandardAllocator::Initialize(void* addr, size_t size, bool isCacheEnable);

なお、デフォルトの nninitStartup() が適用された場合、 確保可能なメモリをすべてメモリヒープとして取得したうえで、本 API を使ってメモリアロケータを初期化します。
デフォルトの nninitStartup() ではスレッド毎のキャッシュ機能は有効 (isCacheEnable には true が指定) となります。
メモリアロケータで使用するバッファのサイズを変更したい場合は、 nninitStartup のサンプルを参照して下さい。

一方、独自のメモリ管理を行なう場合には以下のようにしてください。

  • 独自の nninitStartup() を定義してください。
  • 独自の malloc(), free(), calloc(), realloc(), aligned_alloc() を定義してください。
  • InitializeAllocator() を呼び出さないでください。

上記のいずれかを満たさない場合、ビルド時にリンクエラーとなります。 詳細は nninitStartupWithMalloc を参照して下さい。

（Windowsプラットフォームのみ）:
  • Windows 環境では本 API は何も行なわずにリターンします。
  • Windows 環境では malloc(), free(), calloc(), realloc() は Windows 標準の API がそのまま呼ばれます。
  • aligned_alloc() は VisualStudio のバージョンによっては使用できない 場合があります。
nn::mem::StandardAllocator* nn::init::GetAllocator ( )

メモリアロケータの内部実装クラスオブジェクトを取得します。

事前条件
  • InitializeAllocator() によりメモリアロケータが初期化されている
戻り値
メモリアロケータの内部実装クラスオブジェクトが返ります。

詳細
メモリアロケータでは、その内部実装に nn::mem::StandardAllocator を使用 しています。本 API ではメモリアロケータが利用している nn::mem::StandardAllocator オブジェクトを返します。

（Windowsプラットフォームのみ）:
  • Windows 環境では本 API は 常に NULL を返します。
