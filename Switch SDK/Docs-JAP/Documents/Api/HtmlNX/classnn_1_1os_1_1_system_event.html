詳解

システムイベントを扱うためのクラスです。

詳細
システムイベントは SDK がイベントの通知に使用する同期オブジェクトです。 詳細は関連ドキュメントを参照して下さい。

nn::os::SystemEvent クラスは、nn::os::SystemEventType オブジェクトを使用する同期制御 API の呼び出しをラッピングした ユーティリティクラスです。
本クラスが提供するメンバ関数は、nn::os::SystemEventType 型を 引数にとる nn::os::WaitSystemEvent() などの API を発行する形で 実装されています。

各種例:
AccountApplication.cpp, AccountApplicationAuthorization.cpp, AudioCpuRenderer.cpp, AudioDevice.cpp, AudioEchoback.cpp, AudioEffect.cpp, AudioEffectAuxUtility.cpp, AudioMemoryPool.cpp, AudioMultipleRenderer.cpp, AudioOut.cpp, AudioOutWithResampler.cpp, AudioPerformanceMetrics.cpp, AudioRenderer.cpp, AudioSink.cpp, AudioSplitter.cpp, CodecAacDecoder.cpp, CodecOpusCodecProcessingTime.cpp, CodecOpusDecoder.cpp, CodecOpusEncoder.cpp, CodecOpusMultiStreamDecoder.cpp, FriendsProfileImage.cpp, FriendsSimple.cpp, HtcConnectionEvent.cpp, OeDisplayResolution.cpp (計24項目).

構築子と解体子

nn::os::SystemEvent::SystemEvent ( )

システムイベントオブジェクトを未初期化で用意します。

詳細
コンストラクタです。
内部のシステムイベントオブジェクトは未初期化状態になります。 この後に Attach() することで、 システムイベントが使用できるようになります。

nn::os::SystemEvent::~SystemEvent ( )

デストラクタです。

詳細
システムイベントオブジェクトを破棄します。
詳細は nn::os::DestroySystemEvent() を参照して下さい。

ただし、本クラスでは nn::os::DestroySystemEvent() とは異なり、 システムイベントオブジェクトが未初期化の場合、何もしません。

void nn::os::SystemEvent::Wait ( )

システムイベントがシグナル状態になるまで待機します。

詳細
システムイベントがシグナル状態になるまで待機します。
システムイベントが nn::os::EventClearMode_AutoClear の場合、 シグナル状態を検知すると同時にシステムイベントは 非シグナル状態にクリアされます。
詳細は nn::os::WaitSystemEvent() を参照して下さい。

各種例:

bool nn::os::SystemEvent::TryWait ( )

システムイベントがシグナル状態かをポーリングします。

戻り値
システムイベントのポーリング成否を返します。

詳細
システムイベントがシグナル状態かどうかをポーリングします。
システムイベントがシグナル状態なら true を、 そうでなければ false を返します。
システムイベントが nn::os::EventClearMode_AutoClear の場合、 シグナル状態を検知すると同時にシステムイベントは 非シグナル状態にクリアされます。
詳細は nn::os::TryWaitSystemEvent() を参照して下さい。

各種例:
AudioDevice.cpp, FriendsPresence.cpp.
