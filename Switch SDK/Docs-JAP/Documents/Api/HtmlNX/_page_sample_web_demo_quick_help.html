ゲームをポーズしてクイックヘルプを表示することを想定したサンプルオフライン HTML

概要

ゲームをポーズしてクイックヘルプを表示することを想定したオフライン HTML のサンプルです。
本サンプルでは下記の機能を使っています。

  • 背景にアプリケーションのキャプチャー画像を利用する
  • アプリケーションの状態を URL パラメータでオフライン HTML へ渡す
  • ズーム機能を無効にする
  • 空間ナビを独自に制御する

ファイル構成

本サンプルオフライン HTML は Samples/Sources/Applications/WebDemo/html_document/sample/quick_help.htdocs 以下にあります。

必要な環境

本サンプルは NX 実機環境のみサポートしています。

操作方法

アプリケーションの操作は下記の通りです。

入力 動作
B ボタン メニューへ戻ります。
+/- ボタン ゲーム中にポーズしたことを想定して、オフライン Web アプレットを起動してオフライン HTML を表示します。

オフライン HTML の操作は下記の通りです。

入力 動作
L スティックまたは上下左右ボタン 空間ナビを移動します。
タッチ ヘルプを表示する対象を選択します。
B ボタン、+/- ボタン オフライン Web アプレットを終了して、アプリケーションへ戻ります。

注意事項

特にありません。

実行手順

WebDemo のサンプルプログラムをビルドし、実行してください。
メニューから本サンプルを選択してください。

解説

本サンプルの流れは下記の通りです。

  • メニューから本サンプルを選ぶと、アプリケーションへ遷移します。
  • アプリケーションはゲーム中を想定した画像を表示します。
  • +/- ボタンを押すと、ゲーム中にポーズしたことを想定して、オフライン Web アプレットを起動してオフライン HTML を表示します。
  • オフライン HTML の表示を終了すると、アプリケーションは再びゲーム中を想定した画像を表示します。

背景にアプリケーションのキャプチャー画像を利用する

nn::web::ShowOfflineHtmlPageArg::SetBackgroundKind() の引数に OfflineBackgroundKind_ApplicationCapture を指定します。

アプリケーションの状態を URL パラメータでオフライン HTML へ渡す

オフライン Web アプレットを起動する際に、URL パラメータでゲーム中の情報を渡します。
ゲームをポーズした時に画面内に存在する対象物のカテゴリー、種類、位置、大きさを渡します。
オフライン HTML は、渡されたパラメータを使ってページを構築します。

ズーム機能を無効にする

meta 要素の viewport 属性を使って、ズーム機能を無効にしています。

空間ナビを独自に制御する

keydown イベントでコントローラーの方向ボタンが押されたことを検知します。
preventDefault() を呼び出してデフォルトの空間ナビ制御を無効にしつつ、 押された方向にあるフォーカス可能要素を探索して次にフォーカスすべき要素へと空間ナビを移動します。
