詳解

ホスト・ターゲット間通信の名前空間です。

機能概要

HTC とは、Host Target Communication の略です。
HTC ライブラリでは、ホストとやり取りを行う手段を提供します。
HTC ライブラリの機能はビルドタイプ Release の場合、標準では使用できないようになっています。
ビルドタイプ Release で使用したい場合は NN_ENABLE_HTC を定義するようにしてください。

使用方法

HTC ライブラリの機能を使うためにはあらかじめ TargetManager.exe を起動しておく必要があります。
詳細な接続手段についてはマニュアルを参照してください。

関数詳解

void nn::htc::Initialize ( )

HTC ライブラリを初期化します。

詳細
HTC ライブラリを初期化します。
本関数はライブラリが初期化済の状態でも呼ぶことができますが、その場合は初期化処理は行いません。

事後条件
ライブラリは初期化済の状態になります。
各種例:
HtcConnectionEvent.cpp.
void nn::htc::Finalize ( )

HTC ライブラリを終了します。

詳細
HTC ライブラリを終了します。
nn::htc::Initialize() と同数だけ本関数が呼ばれたとき、本関数はライブラリの終了処理を実行し、未初期化状態にします。

事前条件
ライブラリは初期化済の状態である必要があります。
事後条件
ライブラリの終了処理が行われた場合、ライブラリは未初期化の状態になります。
各種例:
HtcConnectionEvent.cpp.

Result nn::htc::GetTargetEnvironmentVariableLength ( size_t *  pOutValue,
const char *  variableName 
)

ターゲット環境変数の値を格納するのに必要なバッファサイズを取得します。

引数
[out]pOutValue取得したバッファサイズ
[in]variableNameターゲット環境変数名
戻り値
処理の結果が返ります。
戻り値一覧:

ResultSuccess 成功しました。

ResultConnectionFailure ホストとの通信またはターゲット環境変数定義ファイルの読み込みに失敗しました。

ResultNotFound 該当するターゲット環境変数が見つかりませんでした。

事前条件
  • ライブラリが初期化済の状態である
  • 設定されているターゲット環境変数定義ファイルのフォーマットが正しい
  • pOutValue != nullptr
  • variableName != nullptr
  • variableName の文字列長が 64 以下

詳細
取得するターゲット環境変数名はヌル終端である必要があります。
ターゲット環境変数の値が文字列型で定義されている場合は、ヌル終端を含む必要なバッファサイズが返ります。
この関数はホストとの接続がない実機環境では常に ResultConnectionFailure を返します。

引数

[out]outBuffer取得した値を格納するバッファ
[in]bufferSize取得した値を格納するバッファのサイズ
[in]variableName取得するターゲット環境変数名
戻り値
処理の結果が返ります。
戻り値一覧:

ResultSuccess 成功しました。

ResultConnectionFailure ホストとの通信またはターゲット環境変数定義ファイルの読み込みに失敗しました。

ResultNotFound 該当するターゲット環境変数が見つかりませんでした。

事前条件
  • ライブラリが初期化済の状態である
  • 設定されているターゲット環境変数定義ファイルのフォーマットが正しい
  • pOutSize != nullptr
  • outBuffer != nullptr
  • variableName != nullptr
  • variableName の文字列長が 64 以下
  • bufferSize が GetTargetConfigurationValueLength() で取得できる値以上の値

詳細
取得するターゲット環境変数名はヌル終端である必要があります。
取得された文字列はヌル終端されて pOutValue に格納されます。
ヌル終端を含む最大 2048 バイト分の文字列が取得可能です。ターゲット環境変数定義ファイルで 2048 バイトを超える文字列が定義されていた場合は、2048 バイト目以降の値が切り捨てられヌル終端された値が取得されます。
この関数はホストとの接続がない実機環境では常に ResultConnectionFailure を返します。

ResultConnectionFailure タイムアウト時間内にターゲット環境変数定義ファイルが設定されませんでした。

この関数はホストとの接続がない実機環境では即座に ResultConnectionFailure を返します。

void nn::htc::BindHostConnectionEvent ( nn::os::SystemEvent *  pOutValue)

ホストとの通信が確立されたことを通知するイベントオブジェクトにバインドします。

引数
[out]pOutValueイベントオブジェクトを指すポインタ
事前条件
  • ライブラリが初期化済の状態である
  • pOutValue != nullptr
  • *pOutValue は未初期化状態
事後条件
  • *pOutValue は初期化状態

詳細
ホストとの通信が確立されたことを通知するイベントオブジェクトにバインドします。
イベントは読み取り専用で手動クリアに設定されます。
ホストと接続中は常にシグナル状態で、切断時にクリアされます。
イベントはシステム側でクリアするため、ユーザ側でクリアしないでください。

各種例:
HtcConnectionEvent.cpp.
