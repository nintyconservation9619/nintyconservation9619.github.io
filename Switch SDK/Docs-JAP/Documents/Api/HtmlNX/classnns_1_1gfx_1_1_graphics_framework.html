enum nns::gfx::GraphicsFramework::MemoryPoolType

フレームワークで管理するメモリプールのタイプです。

詳細
最適化のため、できる限り用途に合ったタイプを選択することを推奨します。
MemoryPoolType_Others は速度やメモリ効率を考慮しなくてよい場合に限り使用するようにしてください。

列挙値
DepthMode_Near_Is_Minus_W 

Depth -1 - 1.

DepthMode_Near_Is_Zero 

Depth 0 - 1.

enum nns::gfx::GraphicsFramework::FrameworkMode

このクラスで対応するフレームワークのモードです。

列挙値
FrameworkMode_DeferredExecution 

vsync の前にコマンドの提出を行い、提出後 vsync を待ってから GPU が処理を始めるモードです。
Generic と NX で挙動が変わります。
コマンドバッファのマルチバッファリング数を 2 以上にする必要があります。

FrameworkMode_DeferredSubmission 

vsync の後にコマンドの提出を行い、提出後から GPU が処理を始めるモードです。
GL との互換をとりやすいモードです。
コマンドバッファのマルチバッファリング数を 2 以上にする必要があります。

FrameworkMode_Immediate 

コマンドの生成、提出、実行を１フレームで行うモードです。
表示の遅延が最小となりますが、コマンド生成から提出までの間 GPU がアイドル状態になります。
コマンドバッファのマルチバッファリング数を 2 以上にする必要はありません。
