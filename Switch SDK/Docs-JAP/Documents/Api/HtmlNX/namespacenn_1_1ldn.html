enum nn::ldn::AcceptPolicy

アクセスポイントの接続受付に関するポリシーです。

詳細
アクセスポイントの接続受付に関するポリシーです。

デフォルトでは AcceptPolicy_AlwaysAccept が使用されます。 アクセスポイントの開始時点でステーションの接続を制限したい場合、 OpenAccessPoint 関数でアクセスポイントモードを開始した後、 CreateNetwork 関数でネットワークを構築する前に SetStationAcceptPolicy 関数を呼び出して接続を禁止してください。

CreateNetwork 関数でネットワークを構築した後に SetStationAcceptPolicy 関数を実行し、 ステーションの接続受付ポリシーを変更することも可能です。

列挙値
AcceptPolicy_AlwaysAccept 

接続要求を全て受け入れます。

AcceptPolicy_AlwaysReject 

接続要求を全て拒否します。

AcceptPolicy_BlackList 

フィルタに登録されているノードの接続を拒否します。

AcceptPolicy_WhiteList 

フィルタに登録されているノードのみが接続できます。

enum nn::ldn::DisconnectReason

ステーションの切断理由です。

詳細
ステーションの切断理由です。GetDisconnectReason() で取得できます。

LDN ライブラリが初期化されてから一度もステーションとして切断されていない場合、 もしくはネットワークに接続中であれば DisconnectReason_None です。

ステーション側のアプリケーションが Disconnect() を呼び出して切断した場合は DisconnectReason_DisconnectedByUser が、 システムの通信開始やスリープなどシステムの要求で切断した場合は DisconnectReason_DisconnecetdBySystem が設定されます。

アクセスポイント側のアプリケーションが DestroyNetwork() を呼び出してネットワークを破棄した場合は DisconnectReason_DestroyedByUser が、 システムの通信開始やスリープなどシステムの要求でネットワークを破棄した場合は DisconnectReason_DestroyedBySystem が設定されます。

また、Reject() を使ってステーションを切断した場合は DisconnectReason_Rejected が、 通信環境が不安定なためにステーションがネットワークから離脱した場合は DisconnectReason_SignalLost が設定されます。

ただし、アクセスポイントから切断されたことを示す以下の 3 つの要因については 正確な切断理由を確実に取得できることを保証していません。

  • DisconnectReason_DestroyedByUser
  • DisconnectReason_DestroyedBySystem
  • DisconnectReason_Rejected

切断理由の取得に失敗した場合、代わりに DisconnectReason_Unknown が取得できます。 切断理由の失敗は特に通信環境が不安定な場合に多く発生します。 GetDisconnectReason() を繰り返し実行しても結果は変わりません。

列挙値
DisconnectReason_Unknown 

切断理由は不明です。

DisconnectReason_None 

切断されていません。

DisconnectReason_DisconnectedByUser 

アプリケーションの要求によってネットワークから切断しました。

DisconnectReason_DisconnectedBySystem 

システムの要求によってネットワークから切断しました。

DisconnectReason_DestroyedByUser 

アプリケーションの要求によってアクセスポイントがネットワークを破棄しました。

DisconnectReason_DestroyedBySystem 

アクセスポイントがシステムの要求によってネットワークを破棄しました。

DisconnectReason_Rejected 

アクセスポイントがステーションを切断しました。

DisconnectReason_SignalLost 

通信環境が不安定なため切断されました。

enum nn::ldn::SecurityMode

ローカル通信のセキュリティ・モードです。

詳細
製品機上のローカル通信では SecurityMode の指定は無視され、 常に SecurityMode_Product が指定されたものと仮定して、 全てのデータがシステムによって暗号化された状態で送受信されます。

一方で、ローカル通信ライブラリが開発機上で動作する場合に限定して、 アプリケーションから通信保護のポリシーを指定することができます。 これはアプリケーションの通信機能のデバッグを目的としたもので、 CreateNetwork 関数に渡す SecurityConfig の securityMode に SecurityMode_Debug を指定することでデータ通信の暗号化が解除されます。 ただし、暗号化が解除されるのはアプリケーションが送受信するデータのみで、 システムが制御用に交換する情報の暗号化は解除されません。

また、通常、ステーションは SecurityMode が一致するアクセスポイントにしか接続できませんが、 SecurityMode_Any を指定した場合には SecurityMode_Debug, SecurityMode_Product のいずれのアクセスポイントにも接続することができます。 この設定も開発機でのみ有効で、製品機では SecurityMode_Any や SecurityMode_Debug の指定は無視されます。

列挙値
SecurityMode_Any 

セキュリティ・モードを指定しません。

SecurityMode_Product 

製品用のセキュリティ・モードです。

SecurityMode_Debug 

デバッグ用のセキュリティ・モードです。

ステーションの接続を制限するフィルタにエントリを追加します。 フィルタの役割は SetStationAcceptPolicy 関数で指定した AcceptPolicy で決まります。

AcceptPolicy_AlwaysAccept あるいは AcceptPolicy_AlwaysReject を使用している場合、 フィルタは意味をもちません。 AcceptPolicy_BlackList を使用している場合、フィルタに登録されたステーションは、 このネットワークに接続することができなくなります。 AcceptPolicy_WhiteList を使用している場合、フィルタに登録されたステーション以外は、 このネットワークに接続することができなくなります。

なお、 SetStationAcceptPolicy 関数でフィルタの役割を変更したり、 この関数でフィルタに新しいエントリを登録したりしても、 その時点でネットワークに参加しているステーションが自動的に切断されることはありません。 必要に応じて Reject 関数を併用して不要なステーションを切断してください。

登録可能なエントリの最大数は AcceptFilterEntryCountMax で、 上限を超えた場合には古いエントリから順に削除して上書きします。 登録したエントリは CloseAccessPoint 関数や Finalize 関数でアクセスポイントとしての動作を停止した時点で全て削除されます。 明示的にエントリを削除したい場合は ClearAcceptFilter 関数を使用してください。

デフォルトのローカル通信識別子です。

詳細
IntentId の localCommunicationId に DefaultLocalCommunicationId を指定すると、 アプリケーション管理データの先頭に登録されているローカル通信識別子を使用します。 アプリケーション管理データでローカル通信識別子の設定を行っていない場合、 自身のアプリケーション ID がローカル通信識別子として登録されているため、 DefaultLocalCommunicationId を指定するとアプリケーション ID を ローカル通信識別子として使用することになります。

複数のローカル通信識別子を使い分ける必要がある場合には DefaultLocalCommunicationId を使用せず、明示的にローカル通信識別子を指定してください。

nn::ldn::NodeCountMax = 8

ローカルネットワークに接続できるノードの最大数です。

詳細
ローカルネットワークに接続できるノードの最大数です。 この数字にはアクセスポイントが含まれるため、 ローカルネットワークには最大で NodeCountMax - 1 台のステーションが存在することになります。 なお、ステーションの最大数は StationCountMax として定義されています。

アクセスポイントはローカルネットワークに接続できるステーションの最大数を、 必要に応じてさらに少なく制限できます。 そのためには、 CreateNetwork 関数に渡す NetworkConfig 構造体の nodeCountMax メンバに NodeCountMax よりも小さい値を指定します。

各種例:
LdnAccessPoint.cpp, LdnStation.cpp.
nn::ldn::StationCountMax = NodeCountMax - 1
