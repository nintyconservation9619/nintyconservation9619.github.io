対になっている「構造体」と「クラス」の使い分け

NintendoSDK において、ある機能が「構造体＋関数」および「クラス」の対で提供されている場合、 構造体は POD (Plain Old Data) であることが保証されています。 一方「クラス」形式の API は、「構造体＋関数」をラップしたユーティリティ的な位置づけになっています。

POD であるオブジェクトを利用したい場合は「構造体＋関数」形式の API を、 そうでない場合は「クラス」形式の API を利用してください。

このような対となる機能を提供している構造体とクラスは、次のような命名規則が適用されています。

種別命名規則例
構造体XXXType 構造体nn::os::EventType 構造体
クラスXXX クラスnn::os::Event クラス
