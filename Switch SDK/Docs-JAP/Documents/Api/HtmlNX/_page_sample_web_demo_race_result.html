レースゲームのリザルトシーンを想定したサンプルオフライン HTML

概要

レースゲームのリザルトシーンを想定したオフライン HTML のサンプルです。
本サンプルでは下記の機能を使っています。

  • Web Fonts で独自のフォントを表示する
  • CSS で文字を装飾する
  • アプリケーションの状態を URL パラメータでオフライン HTML へ渡す
  • システムによってオフライン Web アプレットが終了した時に復元する
  • JS 拡張を使ってコントローラー操作をカスタマイズする

ファイル構成

本サンプルオフライン HTML は Samples/Sources/Applications/WebDemo/html_document/sample/race_result.htdocs 以下にあります。

必要な環境

本サンプルは NX 実機環境のみサポートしています。

操作方法

入力 動作
A ボタン オフライン Web アプレットを終了して、アプリケーションを経由してメニューへ戻ります。
X ボタン フォントを切り替えます。

注意事項

特にありません。

実行手順

WebDemo のサンプルプログラムをビルドし、実行してください。
メニューから本サンプルを選択してください。

解説

メニューから本サンプルを選ぶと、アプリケーションを経由してオフライン Web アプレットを起動し、本サンプルオフライン HTML を表示します。

Web Fonts で独自のフォントを表示する

レースゲームのデザインに合った独自のフォント（document_sample_font.ttf）で、全ての文字を表示しています。
X ボタンで標準のフォントに切り替えて確認することが出来ます。

CSS で文字を装飾する

画像データを使わずに CSS で文字を装飾して、様々な表現を行っています。

アプリケーションの状態を URL パラメータでオフライン HTML へ渡す

オフライン Web アプレットを起動する際に、URL パラメータでレース結果の情報を渡します。
ステージ、コース、車種、順位、ドライバー、タイムなどの情報をオフライン HTML へ渡します。
順位は、アプリケーションがランダムで決めます。
オフライン HTML は渡されたパラメータに応じて、表示する内容を変更します。

システムによってオフライン Web アプレットが終了した時に復元する

システムによってオフライン Web アプレットが終了した時、アプリケーションは再びオフライン Web アプレットを起動し、オフライン HTML を復元して表示します。
復元するかどうかを URL パラメータでオフライン HTML へ渡します。
復元して表示する場合、オフライン HTML は順位の表示などのアニメーションを省略します。

JS 拡張を使ってコントローラー操作をカスタマイズする

JS 拡張 window.nx.footer.setAssign() を使って、指定されたキーの機能を変更します。
