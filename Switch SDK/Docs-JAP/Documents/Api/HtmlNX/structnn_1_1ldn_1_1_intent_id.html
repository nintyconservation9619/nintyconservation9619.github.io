詳解

ネットワークの目的を表す識別子です。

詳細
ネットワークの目的を表す 128 bit の識別子です。NetworkId の前半部分に相当します。 IntentId はローカル通信識別子 (localCommunicationId) とシーン識別子 (sceneId) から構成されます。

ローカル通信識別子はアプリケーションタイトルやシリーズを識別する目的で使用します。 原則として、ローカル通信識別子にはアプリケーション ID を使用します。 ただし、他のアプリケーションと通信を行う場合、 通信するアプリケーション間で共通のローカル通信識別子を使用する必要がありますので、 どちらか一方のアプリケーション ID を双方で使用することができます。 アプリケーションが使用するローカル通信識別子は全てアプリケーション管理データに登録しておく必要があり、 登録されていない場合には CreateNetwork() や Connect() が事前条件違反で失敗します。 なお、アプリケーション管理データでローカル通信識別子の設定を行っていない場合、 自身のアプリケーション ID がローカル通信識別子として登録されています。 アプリケーションが複数のローカル通信識別子を使い分ける必要が無い場合は localCommunicationId に DefaultLocalCommunicationId を使用できます。

これに対して、シーン識別子は「通信対戦」「データ交換」のようなアプリケーションの シーンを識別することを目的としており、アプリケーション開発者が自由に設定することができます。 上記のような特性から、同じ用途でローカルネットワークを構成する場合、 同じ IntentId が何度も使用されることになります。

IntentId を生成する必要がある場合には IntentId 構造体に直接値を設定するのではなく、 必ず MakeIntentId 関数を使用してください。
