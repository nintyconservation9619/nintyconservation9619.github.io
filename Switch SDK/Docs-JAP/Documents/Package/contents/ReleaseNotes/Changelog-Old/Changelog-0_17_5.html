NintendoSDK NX Addon 0.12.5 (based on Generic 0.17.5) の変更点

改善

13-6S01: (TOOL) テクスチャコンバータで、float_bc6 フォーマットにエンコードする際の品質の改善

テクスチャコンバータ（%NINTENDO_SDK_ROOT%\Tools\Graphics\GraphicsTools\TextureConverter.exe）で、GPU で float_bc6 フォーマットにエンコードする際の品質を改善しました。

不具合修正

13-6S02: (TOOL) テクスチャコンバータで、サイズの大きい画像を astc フォーマットに変換する際にクラッシュする場合がある不具合の修正

テクスチャコンバータ（%NINTENDO_SDK_ROOT%\Tools\Graphics\GraphicsTools\TextureConverter.exe）で、サイズの大きい画像を GPU で astc フォーマットにエンコードする際にクラッシュする場合がある不具合があり、これを修正しました。

NintendoWare NX Addon 0.12.5 (based on Generic 0.17.5) の変更点

機能追加

13-6W01: (レイアウト-UI2D) LayoutArchiver に、テクスチャのタイリング方法を指定するオプションを追加

LayoutArchiver に、テクスチャのタイリング方法を指定するオプション（tile-optimize と tile-size-threshold）を追加しました。仕様は、 TextureConverterの同じオプションに準拠しています。サイズが小さく、２のべき乗ではないサイズのテクスチャでサイズの差が出ます。Samples\Sources\Resources\Ui2d\cannonButton\CannonButton_00.flyt の全テクスチャで tile-optimize=size_auto を指定して比較した場合、tile-size-threshold=5 の場合、96.1 KBで、tile-size-threshold=100 の場合 72.1 KB となりました。tile-size-thresholdは大きな値ほど、サイズを優先して変換をおこなうようになります。

不具合修正

13-6W02: (3D-G3D) nn::g3d::mii::AssignCharModel() でバウンディング球半径が正しく計算されない不具合の修正

nn::g3d::mii::AssignCharModel() でバウンディング球半径が正しく計算されていない不具合が修正されました。本修正により nn::g3d::ResShapeData::pRadiusArray が保持する半径に正しい値が入るようになりました。

13-6W03: (エフェクト-VFX) VFX ランタイムで、パーティクルの Z ソートがローカル座標系で計算されていた不具合を修正

VFX ランタイムで、パーティクルの Z ソートがローカル座標系で計算されていたため、カメラとの位置関係によっては正しくない見た目になる不具合があり、これをワールド座標系で計算するように修正しました。

13-6W04: (エフェクト-VFX) 利用する fmdb ファイルの構成によっては、頂点ステートバッファが足りなくなる不具合を修正

利用する fmdb ファイルの構成によっては、頂点ステートバッファが足りなくなり、アサートする不具合を修正しました。
