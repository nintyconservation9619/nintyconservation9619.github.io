NintendoSDK NX Addon 1.3.0 (based on Generic 1.3.0) の変更点

互換性の失われる仕様変更

★1-3-0S01: (General) SD カードのファイル構成変更

SD カードのファイル構成が NX Addon 1.1.0 から 1.2.0 までの構成内容と変わっています。

NX Addon 1.1.0 から 1.2.0 で使っていた SD カードを NX Addon 1.3.0 以降のパッケージで初期化した本体で使おうとすると、ホームメニューのエントランスに入った後に読み込みに失敗するというダイアログが出るようになります。
DevMenuCommand sdcard cleanup を実行するか、 DevMenuCommand sdcard format を行うことで、再び SD カードを使うことが出来るようになります。
なお、この変更は NX Addon 1.0.0 以前で利用していた SD カードには影響しません。

機能追加

★1-3-0S02: (GFX) HLSL から GLSL へ変換するためのツールを追加

HLSL から GLSL へ変換するためのツール SLConverter を追加しました。
詳細は SLConverter を参照してください。

1-3-0S03: (FS) マッチングサイズを調整する --minimum-matching-size オプションを追加

AuthoringTool でのパッチ作成時に、マッチングが効きすぎてファイルが断片化しすぎることの対処方法として、最小マッチングサイズを指定できるようにしました。

    • minimum-matching-size で 16 以上の 2 のべき乗を指定してください。未指定の場合は従来同様 32 になります。


1-3-0S04: (TOOL) AuthoringTool に getproperty コマンドを追加

AuthoringTool に getproperty コマンドで、nsp に含まれるタイトルのダウンロードサイズやプレイに最低限必要なセーブデータサイズを表示できるようになりました。

1-3-0S05: (TOOL) AuthoringTool の list, extract コマンドのパッチ対応

AuthoringTool の list, extract コマンドにオプションを指定することで、パッチを適用した状態のファイルに対して情報表示・抽出できるようになりました。また、list コマンドではパッチにのみ含まれるファイルの情報を表示できるようになりました。

改善

★1-3-0S06: (General) NX Addon 1.3.xで作成したアプリの古いファームでの起動制限

NX Addon 1.3.xで作成したアプリをNintendoSDK Firmware for NX 1.0.0 系では起動できないようにしました。アプリを起動すると「本体更新が必要です」と表示されます。この組み合わせは市場では本来起こりえない組み合わせですので、NX Addon 1.3.xで作成したアプリを動作確認する場合にはNintendoSDK Firmware for NX 2.0.0以降のファームでご確認ください。

1-3-0S07: (Display) CEC 機能の接続性の改善

CEC 機能の接続性を改善し、ワンタッチプレイやシステムスタンバイ等の各種機能がより確実に動作するようにしました。

なお、すべての CEC 対応 TV において CEC の機能が確実に動作することを保証するものではありません。

1-3-0S08: (ERR) nn::err::ShowApplicationError 呼び出し時の事前条件の記載漏れの修正

nn::err::ShowApplicationError の呼び出し時に、引数に与える nn::err::ApplicationErrorArg に言語が設定されていない場合呼び出しに失敗することを追記しました。

1-3-0S09: (FRIENDS) 通信処理を伴う API のドキュメントの改善

事前にインターネット利用要求を行い受理される必要がある旨を追加しました。

1-3-0S10: (FS) FsAccessLogChecker を version 4.0 に更新

FsAccessLogChecker (%NINTENDO_SDK_ROOT%\Tools\FsAccessLogChecker\FsAccessLogChecker.exe) version 4.0 にて、以下の対応を行いました。

  • nn::fs::MountSaveDataReadOnly() のアクセスログに対応
  • セーブデータアクセスの分類を "Nand" に変更
  • 不正なアクセスログの解析失敗メッセージを改善


1-3-0S11: (FS) サンプルデモのエラーハンドリングに対するコメントを追加

fs のサンプルデモに対して、各関数のエラーハンドリングをどのように行うべきかコメントを追加しました。

1-3-0S12: (NVN) nvn の改善

glslc に以下の改善を行いました。

  • Added error if applications try to use assembly dumps ("outputAssembly" option) in conjunction with multi-threading ("enableMultithreadCompilation" option). This is currently a limitation of GLSLC.

  • GLSL shaders using NV_fragment_shader_interlock now consume less scratch memory than similar shaders that do not use the feature.

  • Made various programming guide updates:
    • Updated section "Comparing compiled programs for functional equality" with new entries.
    • Clarify that atomic_uint functionality is unsupported.
    • Document the GLASM limits in the programming guide.


1-3-0S13: (TOOL) InitializeSdev.exe のファームウェア更新のエラーハンドリング方針変更

InitializeSdev.exe で初期化する各種領域のうち直接の動作に影響しない領域の初期化に失敗した場合はエラーではなく警告で実行を継続するようにしました。

不具合修正

1-3-0S14: (AUDIO) PC 環境で 2 チャンネル以上の音声再生時にアサートの誤った条件違反が発生する不具合の修正

PC 環境で 2 チャンネル以上の音声再生時にアサートの誤った条件違反が発生する不具合があり、これを修正しました。
本不具合は、 nn::audio::AudioRenderer を使用して再生している時のみ発生します。nn::audio::AudioOut を使用して音声再生を行っている場合は発生しません。
また、ビルドターゲットが Release の場合、アサートは有効化されませんので、不具合が発生することはありません。

1-3-0S15: (DevMenu) DevMenuCommand の network コマンドの修正

DevMenuCommand の network コマンドを実行すると socket ライブラリの多重初期化により動作しないようになっていましたが、これを修正しました。

1-3-0S16: (DevMenu) パッチをインストールするとアプリケーションを起動できなくなる不具合を修正

パッチをインストールすると、パッチ対象のアプリケーションを起動できなくなる不具合を修正しました。

1-3-0S17: (Display) CEC を有効にして特定の TV に出力中、TV 側のチャンネルを切り替えて電源を切ると開発機がハングすることがある不具合の修正

CEC を有効にして特定の TV に出力中、TV 側のチャンネルを切り替えて電源を切ると開発機がハングすることがある不具合を修正しました。

1-3-0S18: (Display) ドックイン・ドックアウトで本体がハングアップすることがある不具合の修正

稀にドックイン・ドックアウトで本体がハングアップすることがある不具合を修正しました。

1-3-0S19: (Display) 据置モードでスリープ・ウェイクを多数繰り返すとハングする不具合の修正

据置モードでスリープする際に少量のメモリリークが発生するバグがあり、数千回程度繰り返すことでハングする不具合がありました。これを修正しました。

1-3-0S20: (HID) InputDirector 経由で Npad が操作できない不具合の修正

InputDirector 経由で接続された XInput 対応のコントローラーで Npad が操作できない不具合を修正しました。
InputDirector の Npad の正式対応は将来のSDKで予定しています。

1-3-0S21: (NFC) nn::nfp::StartDetection() 中に他の NFC デバイスの切断が発生した場合に nn::nfp::ResultNfcDeviceNotFound が返る不具合の修正

ある NFC デバイスを指定して nn::nfp::StartDetection() 中に他の NFC デバイスの切断が発生した場合に nn::nfp::ResultNfcDeviceNotFound が返る不具合がありましたので、これを修正しました。

1-3-0S22: (NFP) NFC 機能が無効の状態で nn::nfp::AttachActivateEvent() や nn::nfp::AttachDeactivateEvent() が nn::nfp::ResultNfcDeviceNotFound を返す不具合の修正

NFC 機能が無効の状態で nn::nfp::AttachActivateEvent() や nn::nfp::AttachDeactivateEvent() が nn::nfp::ResultNfcDisabled ではなく nn::nfp::ResultNfcDeviceNotFound を返す不具合がありましたので、これを修正しました。

1-3-0S23: (NFP) NfpMulti にて nn::nfp::ResultNfcDeviceNotFound が要因の restart 操作を複数の NFC デバイスで同時に行うと異常終了することがある不具合の修正

NfpMulti にて nn::nfp::ResultNfcDeviceNotFound が要因の restart 操作を複数の NFC デバイスで同時に行うと異常終了することがある不具合がありましたので、これを修正しました。

1-3-0S24: (NFP) 複数台の NFC デバイスに同時にアクセスした場合に nn::nfp::ResultNeedRetry や nn::nfp::ResultNeedRestart が返る不具合の修正

複数台の NFC デバイスに同時にアクセスした場合に nn::nfp::ResultNeedRetry や nn::nfp::ResultNeedRestart が返る不具合がありましたので、これを修正しました。

1-3-0S25: (NIFM) スリープからの復帰に時間がかかることがある不具合の修正

USB Ethernet アダプターが接続されていない場合に、スリープからの復帰に時間がかかることがある不具合を修正しました。

1-3-0S26: (PowerManagement) PC と開発機を直結した状態から抜いても充電中アイコン表示が消えないことがある不具合の修正

TypeA-C 変換ケーブル等を使用して PC と開発機を直結し、その後ケーブルを抜くと、充電中であることを示すアイコンの表示が充電中のまま消えない不具合がありました。
またこの症状が発生した状態で開発機をスリープすると暗転状態でハングしていました。
この不具合を修正しました。

1-3-0S27: (PowerManagement) スリープ時に電源ボタンや HOME ボタンを連打するとスリープから復帰できなくなる不具合の修正

スリープ時に電源ボタンや HOME ボタンを連打するとスリープから復帰できなくなる不具合を修正しました。

1-3-0S28: (PowerManagement) 稀にスリープから復帰できなくなる場合がある不具合の修正

スリープから復帰しようとしても、暗転状態のまま復帰できなくなる場合がある不具合を修正しました。

1-3-0S29: (Socket) nn::socket::Close() が処理を返さないことがある不具合の修正

稀に nn::socket::Close() が処理を返さなくなる不具合がありましたが、これを修正しました。

1-3-0S30: (Socket) nn::socket::SendTo() が返らなくなる不具合の修正

高頻度で nn::socket::SendTo() を呼び出すと、API が返らなくなる問題を修正しました。

1-3-0S31: (VSI) VSI でビルドする際に不要なビルドが行われることがある問題の修正

VSI でビルドする際に不要なビルドが行われることがある問題があり、これを修正しました。

1-3-0S32: (アカウント) サンプルで文字を出力する際に終端しない場合がある不具合の修正

サンプルで出力する文字列を終端せず意味のないデータを表示することがある不具合を修正しました。

NintendoWare NX Addon 1.3.0 (based on Generic 1.3.0) の変更点

機能追加

1-3-0W01: (3D-G3D) nn::g3d::ResShaderProgram::IsInitialized() の追加

nn::g3d::ResShaderProgram が初期化済みであるかどうかを取得する nn::g3d::ResShaderProgram::IsInitialized() を追加しました。

不具合修正

1-3-0W02: (3D-G3D) G3dDemo サンプルの Edit シーンにおける 3DEditor 接続時にアサートに失敗する不具合の修正

G3dDemo サンプルの Edit シーンにおいて、接続した 3DEditor からモデルやシェーダーが送信されると下記メッセージのアサートに失敗してしまう不具合があり、これを修正しました。

Can't find appropriate shader type

1-3-0W03: (3D-G3D) nn::g3d::viewer::ViewerServer::QueueClearMaterialSelectionCommand() 実行時にアサートに失敗する不具合の修正

nn::g3d::viewer::ViewerServer::QueueClearMaterialSelectionCommand() 実行時にアサートに失敗してしまう不具合があり、これを修正しました。

1-3-0W04: (レイアウト-UI2D) ユーザが設定したグローバル行列が nn::ui2d::Pane::CalculateGlobalMatrix を呼ぶと上書きされる不具合の修正

ユーザが設定したグローバル行列が nn::ui2d::Pane::CalculateGlobalMatrix を呼ぶとデフォルトのグローバル行列で上書きされる不具合を修正しました。

1-3-0W05: (SPY) AtkSpy の利用中に Spy.exe が異常終了する不具合を修正

Spy.exe をアプリケーションと接続し、nn::atk を使って複数のシーケンスサウンドを再生したときに、Spy.exe が異常終了する問題がありましたが、これを修正しました。

1-3-0W06: (SPY) Spy.exe で１時間１１分以上の記録ができない不具合の修正

Spy.exe をアプリケーションと接続し、１時間１１分経過するとデータが更新されなくなったり例外が起きるなど Spy.exe が正常に動作しなくなり、データファイルに不正な内容が記録される不具合を修正しました。

これまでに記録された不正なデータファイルは、本リリースにおいても正常に開くことはできません。
（これについて修正の予定はありません）

1-3-0W07: (SPY) Spy.exe のデータファイルのサイズが2Gバイトを超えていた場合に例外が発生する不具合の修正

Spy.exe のデータファイルのサイズが2Gバイトを超えていた場合に、通信を切断、または Spy.exe を閉じたときに例外が発生する不具合を修正しました。
