NintendoSDK NX Addon 0.12.15 (based on Generic 0.17.15) の変更点

機能追加

13-16S01: (General) 統合サンプルの追加

NintendoSDK の複数の機能を同時に使用する Samples\Sources\Applications\Demo1 サンプルの追加。（以下、統合サンプル）
統合サンプルをビルドし、実行すると動作します。

＜説明＞

統合サンプルは以下の機能から構成されています。

  • nns::gfx::PrimitiveRenderer ライブラリを用いたプリミティブ描画
  • nn::hid ライブラリを用いたヒューマンインターフェイスデバイスの状態取得
  • オーディオレンダラを用いたオーディオ再生
  • nn::fs ライブラリを用いたファイルの読み出し
  • nn::fs ライブラリを用いたユーザーセーブデータの書き込み
  • nn::time::StandardUserSystemClock ライブラリを用いた現在時刻の表示と時刻操作の検知
  • Npad を用いた SixAxisSensor の状態取得
  • Npad を用いた IrSensor の状態取得
  • ソフトウェアキーボードで入力された文字列の表示


改善

13-16S02: (FS) FsAccessLogChecker を version 3.0 に更新

FsAccessLogChecker (%NINTENDO_SDK_ROOT%\Tools\FsAccessLogChecker\FsAccessLogChecker.exe) を version 3.0 に更新しました。以下の変更が含まれます。

  • NintendoSDK Firmware for NX 2.0.0-3 以後のアクセスログ SD カード出力フォーマットに対応
  • 入力ファイルに複数の開始ログがある場合に、それぞれを解析し、複数のログが含まれる旨を警告する
  • アクセスログのタイムスタンプが時系列ではない場合にエラーとする
  • "FS_ACCESS:" のログが完結していない場合にエラーとする
  • "FS_ACCESS:" の "result:" がエラーなものは読み込み・書き込み操作としてカウントしない


13-16S03: (TOOL) 追加コンテンツ作成時のエラー条件追加

追加コンテンツの nmeta ファイルに Tag 要素が指定されておらず、且つ --error-unpublishable が指定されている場合エラーが発生するようになりました。

不具合修正

13-16S04: (DevMenu) PROD MODE だと DevMenu でアプリケーションをインストールできない不具合を修正

開発機を PROD MODE で初期化した場合に DevMenu の Appllication ページから SD カード内の nsp ファイルを本体にインストールできない不具合を修正しました。

13-16S05: (APPLET) アプリケーションから複数のライブラリアプレットを同時に起動しようとした後にシステムが停止してしまうことがある不具合を修正

アプリケーションから複数のライブラリアプレットを同時に起動しようとした後にシステムが停止してしまうことがある不具合を修正しました。この問題は特に以下のようなシーケンスで発生することがありました。

  • アプリケーションから、何かしらのライブラリアプレット起動とほぼ同時にエラービューアも起動（ただし、エラービューアは起動待ちとなる）
  • この状態で HOME メニューへ遷移
  • HOME メニューから何かしらのライブラリアプレットを起動（してハマる）


13-16S06: (TOOL) ファームウェア種類選択コンボボックスで +Prod のリビジョン番号が表示されない不具合の修正

ファームウェア種類選択コンボボックスで +Prod (および +ProdWithLog) のリビジョン番号が表示されない不具合を修正しました
