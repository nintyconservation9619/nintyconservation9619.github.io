NintendoSDK NX Addon 1.2.0 (based on Generic 1.2.0) の変更点

互換性の失われる仕様変更

★1-2-0S01: (FS) キャッシュバッファ引数の無い nn::fs::MountAddOnContent() の廃止

廃止予定としていた、キャッシュバッファ引数の無い nn::fs::MountAddOnContent(const char * name, nn::aoc::AddOnContentIndex targetIndex) を廃止しました。
キャッシュバッファ引数を指定する nn::fs::MountAddOnContent(const char * name, nn::aoc::AddOnContentIndex targetIndex, void * pCacheBuffer, size_t cacheBufferSize) に移行してください。

★1-2-0S02: (OE) 廃止予定にしていた性能 (nn::oe::SetPerformanceConfiguration) の定義の削除

nn::oe::SetPerformanceConfiguration で設定できる性能の中で、廃止予定としていた以下の設定を定義から削除しました。

  • nn::oe::SetPerformanceConfiguration_Cpu1020MhzGpu307MhzEmc1600Mhz
  • nn::oe::SetPerformanceConfiguration_Cpu1020MhzGpu384MhzEmc1600Mhz


機能追加

1-2-0S03: (APPLET) 新たなフォーカスハンドリングモードを追加

新たなフォーカスハンドリングモードとして「インフォーカス単独モード」を追加しました。本モードに変更する場合には nn::oe::SetFocusHandlingMode() に nn::oe::FocusHandlingMode_InFocusOnly を指定して下さい。詳細はマニュアルの「プログラムの通知メッセージ」の章にある「フォーカス状態変更のハンドリング」を参照して下さい。

1-2-0S04: (AUDIO) nn::audio::ReverbType における 6ch サポートの追加

nn::audio::ReverbType においてチャンネル数 6 のサポートを追加しました。

1-2-0S05: (DevMenu) Accounts ページの Firmware settings メニューに "Suspend background activity temporarily" ボタンを追加

Accounts ページの Firmware settings メニューに "Suspend background activity temporarily" 項目を追加しました。
この項目は、アカウントに関する次のバックグラウンド処理を一時停止します。

  • プロフィール更新時の、ネットワークサービスアカウントとの即時の同期処理


一時停止は次の期間有効です。

  • 同項目を Disabled の状態にするまで
  • DevMenu の場合: Reboot や開発機をリセットするまで
  • DevMenuApp の場合: DevMenuApp を終了するまで
  • DevMenu と DevMenuApp を併用する場合: それぞれの一時停止処理が無効になるまで


1-2-0S06: (DevMenu) DevMenu で、テレビとの電源連動 (CEC) の有効・無効を変更できる機能を追加

DevMenu で、テレビとの電源連動 (CEC) の有効・無効を変更できる機能を追加しました。Device ページの "Consumer Electronics Control (CEC)" の項目で設定可能です。

1-2-0S07: (DevMenu) DevMenuCommand にユーザー時計の自動補正 ON/OFF 切り替え機能の追加

DevMenuCommand に nn::time::StandardUserSystemClock がネットワーク経由で自動的に補正されるかどうかを切り替える以下のコマンドを追加しました。

  • time set-user-clock-auto-correction <enable|disable>
  • time get-user-clock-auto-correction


1-2-0S08: (DevMenu) アプリケーション起動中に起動元ストレージを表示

アプリケーションの起動中に、アプリケーションおよびパッチの起動元ストレージを表示するようにしました。

1-2-0S09: (General) スリープ中に定期的にシステム処理を行う仕組みの追加

スリープ中に定期的にネットワークに接続してシステムがバックグラウンド処理を行う仕組みを追加しました。アプリケーションを開発する上では特に考慮すべきことはありません。

1-2-0S10: (GFX) 実機環境にて Vulkan を利用するサンプルアプリケーションを追加

実機環境にて Vulkan を利用するサンプルアプリケーション Samples/Applications/Sources/VkSimple を追加しました。

1-2-0S11: (OS) nn::os::SetMemoryPermission() に nn::os::MemoryPermission_None を指定できるように拡張

nn::os::SetMemoryPermission() に nn::os::MemoryPermission_None を指定できるようになりました。このメモリアクセス権に変更されたメモリ領域に対する Read/Write アクセスはできなくなります。

1-2-0S12: (TOOL) DevMenuCommand にオンカードパッチを書き込む機能を追加

DevMenuCommand の gamecard コマンドを利用し、オンカードパッチが付与された開発カードを作成できるようになりました。詳しくは、パッチのビルド、実行方法を参照してください。

1-2-0S13: (TOOL) シェーダコンバータで、ダンプ出力への glslc のオプションフラグの情報の追加

シェーダコンバータ (%NINTENDO_SDK_ROOT%\Tools\Graphics\GraphicsTools\ShaderConverter.exe) で、--ApiType=nvn かつ --dump-directory オプション指定時、ダンプされる出力ファイルに glslc のオプションフラグの情報を含めるようにしました。

1-2-0S14: (TOOL) シェーダコンバータに、glslc のオプションフラグを指定するオプションの追加

シェーダコンバータ (%NINTENDO_SDK_ROOT%\Tools\Graphics\GraphicsTools\ShaderConverter.exe) に、glslc のオプションフラグを指定するためのオプション --glslc-option-flags を追加しました。
このオプションの中に、各フラグをサブオプションとして同様のオプション記法で記述します。
指定できるサブオプションは GLSLCoptionFlags の以下のメンバです。

  • spillControl
  • tessellationAndPassthroughGS
  • enableFastMathMask
  • optLevel
  • unrollControl
  • errorOnScratchMemUsage
  • enableCBFOptimization
  • enableWarpCulling

以下に例を記述します。

    • glslc-option-flags="--optLevel=GLSLC_OPTLEVEL_NONE --errorOnScratchMemUsage=1 --spillControl=NO_SPILL --unrollControl=GLSLC_LOOP_UNROLL_NONE"


1-2-0S15: (TOOL) シェーダコンパイラでの、指定できる glslc のオプションフラグの追加

シェーダコンパイラ (%NINTENDO_SDK_ROOT%\Tools\Graphics\GraphicsTools\ShaderCompiler.dll) で、nngfxToolShaderCompilerCompileOptionNvn に以下を追加しました。それぞれ GLSLCOptionFlags に指定する場合と同じ値を指定できます。

  • tessellationAndPassthroughGS
  • prioritizeConsecutiveTextureInstructions
  • enableFastMathMask
  • optLevel
  • unrollControl
  • errorOnScratchMemUsage
  • enableCBFOptimization
  • enableWarpCulling

★1-2-0S16: (FRIENDS) スリープ時にオンラインプレイの終了宣言が自動的に行われるように変更

1-2-0S17: (エフェクト-VFX) VfxSimple サンプルにてパーティクル用ソートバッファサイズを増強

不具合修正

1-2-0W18: (オーディオ-ATK) 64 bit 環境において nn::atk::SoundDataManager::GetRequiredMemSize() が正しい値を返さない不具合の修正

64 bit 環境において nn::atk::SoundDataManager::GetRequiredMemSize() が実際に消費するメモリサイズから 4 バイト少ない値を返す不具合を修正しました。

1-2-0W19: (オーディオツール) SoundMaker でダウンサンプルが有効の時にビットレートが 0 になる場合がある不具合の修正

SoundMaker でダウンサンプルが有効の時にビットレートやサンプルレートの値が 0 になる場合がありましたが修正しました。

1-2-0W20: (オーディオツール) SoundMaker のタブにドラッグするとエラーになる場合がある不具合の修正

SoundMaker のパネルのタブの領域にサウンドアイテムをドラッグするとエラーになる場合がありましたが修正しました。

1-2-0W21: (オーディオツール) SoundMaker の一部のヘルプメニューからドキュメントが開かない不具合の修正

一部のヘルプメニューからドキュメントが開かない不具合がありましたが修正しました。
また、メニューの文面を「ビューアのヘルプ」から「AtkPlayer のヘルプ」に変更しました。

1-2-0W22: (3D-G3D) nn::g3d::ResShaderFile::IsValid() が常に true を返す不具合の修正

nn::g3d::ResShaderFile::IsValid() が常に true を返す不具合を修正しました。

1-2-0W23: (3D-G3D) ビューアライブラリにおけるシェーダーアタッチ時に正しく SSBO の変更が反映されない不具合の修正

ビューアライブラリにおいて、シェーダーアタッチ時に正しく SSBO の変更が反映されない不具合があり、これを修正しました。

1-2-0W24: (3D-G3D) ビューアライブラリにおけるシェーダーアタッチ時に未初期化シェーダープログラムをアプリケーションが参照する場合がある不具合の修正

ビューアライブラリにおいて、シェーダーアタッチ時に未初期化のシェーダープログラムをアプリケーションが参照してしまう場合がある不具合があり、これを修正しました。

1-2-0W25: (3Dツール) 3D シェーダーコンバーターおよび 3D バイナリーコンバーターで --silent オプションを付けた場合でも正常系のログが出力される問題の修正

3D シェーダーコンバーター (%NINTENDO_SDK_ROOT%\Tools\Graphics\3dTools\3dShaderConverter.exe) および 3D バイナリーコンバーター (%NINTENDO_SDK_ROOT%\Tools\Graphics\3dTools\3dBinaryConverter.exe)で --silent オプションを付けた際に正常系のログが出力されないように修正しました。

1-2-0W26: (3Dツール) 3D シェーダーコンバーターで SSBO の type アノテーションが省略されたシェーダーをコンバートした場合にエラーが発生する不具合の修正

3D シェーダーコンバーター (%NINTENDO_SDK_ROOT%\Tools\Graphics\3dTools\3dShaderConverter.exe) で、SSBO のアノテーション type が省略されたシェーダーをコンバートした場合に内部エラーが発生する不具合を修正しました。
なお、 SSBO のアノテーション type を省略した場合、type には "none" が自動で設定されます。

1-2-0W27: (3Dツール) 3DEditor とアサインユーティリティのマテリアル参照で、子マテリアルの値が未指定になっているときに、親マテリアルの値が反映されない問題を修正

3DEditor とアサインユーティリティ (%NINTENDO_SDK_ROOT%\Tools\Graphics\3dTools\3dIntermediateFileAssignUtility.exe) のマテリアル参照で、子マテリアルの値が未指定になっているときに、親マテリアルの値が反映されない問題を修正しました。
子マテリアルのサンプラ変数、頂点属性が未指定になっていて、マテリアル参照設定の「値」が「親マテリアルの値を参照する」のときに、親マテリアルの値が反映されていませんでした。

1-2-0W28: (3Dツール) 3dEditorBridge の処理がモデル転送中にブロックされる不具合を修正

3dEditorBridge を実行したときに 3DEditor がモデルを転送していると、転送が終わるまで、3dEditorBridge の処理が開始されない不具合があり、これを修正しました。

1-2-0W29: (3Dツール) 中間ファイルオプティマイザーのポリゴンリダクション機能におけるテクスチャー座標、頂点座標計算の不具合修正

中間ファイルオプティマイザー(%NINTENDO_SDK_ROOT%\Tools\Graphics\3dTools\3dIntermediateFileOptimizer.exe)のポリゴンリダクション機能において、テクスチャー座標、頂点座標の計算に不具合があり、これを修正しました。

1-2-0W30: (レイアウト-UI2D) nn::ui2d::ResourceTextureInfo のコンストラクタで未初期化の変数がある不具合の修正

nn::ui2d::ResourceTextureInfo のコンストラクタで未初期化の変数がある不具合を修正しました。

1-2-0W31: (レイアウト-UI2D) コピーコンストラクタでパーツペインを含むレイアウトをコピーした際にコピーされたパーツレイアウトが正しく管理されていない不具合を修正

コピーコンストラクタでパーツペインを含むレイアウトをコピーした際に、コピーされたパーツレイアウトがコピー元のレイアウトのパーツリストで管理されており、不正なポインタにアクセスすることがあったため修正しました。

1-2-0W32: (レイアウトツール) LayoutEditor からビューアに転送するフォントの生成失敗時にエラーが表示されない不具合の修正

LayoutEditor からビューアに転送するフォントの生成に失敗した時にエラーが表示されない不具合があり、これを修正しました。

1-2-0W33: (レイアウトツール) LayoutEditor の、ツール環境設定で、「リニアガンマ特性モード」をオフにすると、「色パラメータをリニア特性で指定する」がオフになる不具合を修正

LayoutEditor の、ツール環境設定で、「リニアガンマ特性モード」をオフにすると、「色パラメータをリニア特性で指定する」がオフになる不具合があり、これを修正しました。（「色パラメータをリニア特性で指定する」の設定は、非リニアガンマモードでは、常にオンにしておくべき設定です。）従来も、ツールを再起動すると、正しい状態に戻っていましたが、切り替えた瞬間だけ不正な状態になっていました。

1-2-0W34: (SPY) タイムラインパネル nn::audio ボイス数が実際の数値よりも大きい値で表示されてしまう不具合を修正
