NintendoSDK NX Addon 0.12.4 (based on Generic 0.17.4) の変更点

互換性の失われる仕様変更

★13-5S01: (FS) 廃止が予定されていた API の削除

廃止が予定されていた以下の API を削除しました。

  • Result nn::fs::MountRom(const char* name) NN_NOEXCEPT;
  • void nn::fs::SetRomRoot(const char* rootPath) NN_NOEXCEPT;


★13-5S02: (HID) Xinput 方式と DirectInput 方式のコントローラの混在に対応

これまでは XInput 方式のコントローラも DirectInput 方式と見做して処理していましたが、安定性を向上させるため、 XInput 方式のコントローラは XInput API で処理するよう変更しました。 これにより、 nn::settings::DebugPadGenericPadMap 構造体と nn::settings::BasicXpadGenericPadMap 構造体の解釈が変わります。
詳しくは HID エミュレーション設定 (%NINTENDO_SDK_ROOT%\Documents/Package/contents/Pages/Page_89983199.html) をご確認ください。

機能追加

★13-5S03: (BuildSystem) AuthoringTool で nsp を作成する時に nrr に登録された nro の情報をチェックする機能を追加

AuthoringTool で nsp ファイルを作成する時に nrr ファイルに登録された nro ファイルが --nro オプションで指定したディレクトリ以下に存在することをチェックする機能を追加しました。
RO ライブラリで nro ファイルを利用する場合に AuthoringTool の --nro オプションに nro ファイルが存在するディレクトリを指定する必要があります。
RO ライブラリの機能を必要としない場合、この手順は省略可能です。

★13-5S04: (NVN) NVN_DEVICE_FLAG_DEFERRED_FIRMWARE_MEMORY_RECLAIM_BIT の追加

従来、nv::InitializeGraphics() で与えたファームウェアメモリのうち、nvnMemoryPoolInitialize() によって GPU ページテーブルに使用された領域は nvnDeviceFinalize() を呼び出すまで解放されませんでした。これを nvnMemoryPoolFinalize() 呼び出し時に他のメモリプールと共有していない GPU ページテーブルは解放するように変更しましたが、NVN_DEVICE_FLAG_DEFERRED_FIRMWARE_MEMORY_RECLAIM_BIT を nvnDeviceBuilderSetFlags() に渡すことで従来挙動に戻すことが可能です。この機能は将来的に削除される予定のため、サポート担当者から案内された場合に限り使用するようにしてください。

13-5S05: (DevMenu) microSD カード内の全 nsp ファイルの一括インストール機能を追加

microSD カード内の全ての nsp ファイルを一括でインストールする機能を追加しました。
Application ページにて ｢Install from SD｣ -> ｢Install All｣ を選択すると一括インストールを開始することができます。

13-5S06: (FS) nn::fs::IsSaveDataExisting() の追加

ユーザアカウントセーブデータの存在を確認する手段として nn::fs::IsSaveDataExisting() を追加しました。

13-5S07: (HID) Npad の複数の機能を切り替えて操作できるサンプルを追加

以下の Npad 関連の機能を切り替えて操作できるサンプル HidNpadIntegrate を追加しました。

  • パッドの入力状態の表示
  • IR センサの状態の表示
  • 振動子の制御
  • 6 軸センサの状態の表示


13-5S08: (TOOL) 「電力消費量の大きい性能」の有効・無効を切り替えるコマンドの DevMenuCommand への追加

DevMenuCommand (%NINTENDO_SDK_ROOT%\TargetTools\NX-NXFP2-a64\DevMenuCommand\Release\DevMenuCommand.nca) に「電力消費量の大きい性能」の有効・無効を切り替えるコマンドを追加しました。

  • 使用方法等は {{DocLink=107328760}}開発機の制御／DevMenuCommand{{DocLink}} を参照してください。
  • 「電力消費量の大きい性能」については {{DocLink=107339250}}OE ライブラリ／マニュアル／機能説明／性能選択{{DocLink}} を参照してください。


13-5S09: (TOOL) meta ファイルの編集を行うための GUI ツール AuthoringEditor を追加

meta ファイルの作成、編集用のツール AuthoringEditor を追加しました。
作成した meta ファイルは、 nsp, nspd ファイルの生成に使用できます。
ツールは %NINTENDO_SDK_ROOT%\Tools\AuthoringEditor\AuthoringEditor.exe にあります。
詳細情報は ツールリファレンス／AuthoringEditor をご確認ください。

13-5S10: (TOOL) 電池残量のエミュレーションコマンドの ControlTarget への追加

電池残量のエミュレーションコマンドを ControlTarget に追加しました。
製品および EDEV では電池残量僅少時にクレードルを接続すると、動作モードが据置モードかつ性能モードがノーマルモードという状態になります。
この状態を SDEV で動的にエミュレーションできます。
詳細は 開発機の制御／ControlTarget を参照してください。

改善

★13-5S11: (HID) 市場環境においても nn::hid API からキーボードの状態を取得できるよう変更

下記の API は用途をデバッグ目的に限定して提供していましたが、 市場環境でも USB 接続されたキーボードの状態を取得するためにもご利用いただけるようになりました。

  • nn::hid::InitializeKeyboard()
  • nn::hid::GetKeyboardState()
  • nn::hid::GetKeyboardStates()

ただし、文字入力用途であれば nn::swkbd API の使用がより適切であるため、 合わせてこちらもご確認ください。

13-5S12: (AUDIO) システムイベントの事後条件に関する API リファレンスの修正

nn::os::SystemEvent を引数として持つ nn::audio::OpenAudioRenderer などの関数について、クローズ時にシステムイベントが未初期化状態に戻るという記述がありましたが、実際には未初期化状態に戻らないため、 API リファレンスから該当の記述を削除しました。
また、これに伴って audio, codec, spy のサンプルにおいて、終了時に利用したシステムイベントを明示的に破棄するように変更を行いました。

具体的には、以下の関数の API リファレンスの記述が対象となります。

  • nn::audio::OpenAudioRenderer
  • nn::audio::OpenDefaultAudioIn
  • nn::audio::OpenAudioIn
  • nn::audio::OpenDefaultAudioOut
  • nn::audio::OpenAudioOut


13-5S13: (DIAG) Log Viewer をドラッグ＆ドロップによるファイルオープンに対応

Log Viewer のアイコンおよびウィンドウに、.nxbinlog 形式のログファイルをドロップして、開けるようになりました。

13-5S14: (FRIENDS) アプリ内表示名の言語コードの妥当性検証処理の追加

アプリ内表示名を引数に取る API （例：nn::friends::AddPlayHistory）を実行する際、アプリ内表示名の言語コードの妥当性検証を行うようにしました。

13-5S15: (NIFM) 公開 Result の変更

ネットワーク接続管理に関して、実行時にハンドリングが必要な Result のみを公開するように修正しました。

13-5S16: (TIME) nn::time::InputPosixTimeMin の値を修正

nn::time::InputPosixTimeMin を 2000/1/1 00:00:00(UTC) から 1999/12/31 00:00:00(UTC) が指す値に変更しました。
本体のタイムゾーン設定によっては、nn::time::ToCalendarTime() の動作保証範囲の最小値である nn::time::InputPosixTimeMin よりも過去の時刻を設定可能であり、nn::time::StandardUserSystemClock::GetCurrentTime() が nn::time::InputPosixTimeMin に満たない時刻を返すことがあったためです。

13-5S17: (TOOL) GUI 版セットアップツールで開発機初期化以外の機能で発生した例外のログ出力対応

GUI 版セットアップツール (InitializeSdevWin・InitializeEdevWin) で開発機の初期化以外の機能で例外が発生した場合にログ確認ができるようになりました。

このログ確認のためには Microsoft 社の DebugView が必要です。

NintendoWare NX Addon 0.12.4 (based on Generic 0.17.4) の変更点

互換性の失われる仕様変更

★13-5W01: (レイアウト-UI2D) LayoutConverter で、アルファ情報を持たない tga ファイルで、BC5 などの２チャンネル形式のフォーマットを指定した場合の挙動を修正

LayoutConverter で、アルファ情報を持たない tga ファイルで、BC5 などの２チャンネル形式のフォーマットを指定した場合の挙動を修正しました。以前はアルファ値に、255 が採用されていましたが、カラー情報と同じ値を採用するようにしました。この修正によって、過去のプラットフォームの挙動が一致しデーター移行が容易になります。仕様変更の影響を受けるファイルを列挙するユーティリティツールの サンプルコード が用意されていますので参照してください。

機能追加

★13-5W02: (エフェクト-VFX) テクスチャミップマップの利用枚数を制限する機能を追加

テクスチャミップマップの利用枚数を制限する機能を追加しました。EffectMaker 上、エミッタタブ、テクスチャに「ミップマップを制限」のチェックを有効にすることで、利用するミップマップテクスチャの枚数を２枚に制限することができます。この機能追加によりエフェクトバイナリのバージョンが変更されています。

13-5W03: (3Dツール) 3DEditor での最適化シェーダ生成時のオプションの指定機能の追加

3DEditor のチーム設定またはプラットフォーム設定ファイルに最適化シェーダ生成時のシェーダコンバータのオプション <ShaderConverterAdditionalArgsForOptimizedShader> を指定する機能を追加しました。詳細は、チーム設定ファイルフォーマットをご覧ください。

13-5W04: (レイアウト-UI2D) ペイン全体に投影を、ウィンドウペイン以外にも指定できるように修正

ペイン全体に投影を、ウィンドウペイン以外にも指定できるように修正しました。従来は、ウィンドウペインにしか指定できませんでした。

13-5W05: (エフェクト-VFX) EmitterSet が保持する Emitter が一つでもフェードされているかどうかを検知する EmitterSet::IsAnyEmitterFadeRequest() の追加

カスタム処理のコールバックなどから、 Emitter::Fade() を通して、エミッタ単位でのフェードができますが、EmitterSet から、一つでも配下のエミッタがフェードされているかどうかを検知するための API である、 EmitterSet::IsAnyEmitterFadeRequest() を追加しました。
配下のエミッタがフェード中かどうかのチェックは、 EmitterSet の計算の最後に行われるため、 Emitter::Fade() が呼ばれたのと同フレーム内で検知することはできない点にご注意ください。

改善

13-5W06: (3D-G3D) nn::g3d::viewer::ViewerServer::QueueAttachModel の機能改善

nn::g3d::viewer::ViewerServer::QueueAttachModel において、同じ nn::g3d::ResModel から作られた複数の nn::g3d::ModelObj がアタッチできるように改善しました。
アタッチしたすべての nn::g3d::ModelObj に 3DEditor による編集が反映されます。

13-5W07: (レイアウト-UI2D) LayoutConverter で、テクスチャが見つからなかった場合に詳しいエラーメッセージを表示するように修正

LayoutConverter で、テクスチャが見つからなかった場合に詳しいエラーメッセージを表示するように修正しました。具体的には、マテリアルが参照しているテクスチャーや、アニメーション区間タグが参照しているテクスチャーが、レイアウトのテクスチャリストから発見できなかった場合に、テクスチャ名や、レイアウトファイル名などを含めてエラーを表示するようにしました。

不具合修正

13-5W08: (オーディオ-ATK) SoundSystem の終了処理で落ちる不具合の修正

nn::atk::SoundSystem::Finalize でメモリプールのデタッチが完了する前に破棄しようとして落ちる場合がある不具合を修正しました。

13-5W09: (オーディオ-ATK) メモリプールのアタッチ・デタッチ関数とレンダラ更新処理が排他制御されない不具合の修正

nn::atk::SoundSystem::AttachMemoryPool, nn::atk::SoundSystem::DetachMemoryPool 関数が atk のサウンドシステムが使用するオーディオレンダラの更新処理と排他制御されない不具合を修正しました。

13-5W10: (オーディオ-ATK) リリース時優先度固定の機能が無効化できない不具合の修正

SoundMaker でリリース時優先度固定のオプションを無効に設定しても、リリース時の優先度が最低にならない不具合を修正しました。

13-5W11: (FONT) nn::font::TextureCache::Finalize() で、不正なメモリ解放処理を修正

nn::font::TextureCache::Finalize() で、不正なメモリ解放処理を修正しました。メモリープールの終了処理を完了する前に、管理しているメモリを開放していたので、解放されたメモリが、終了処理が完了する前に再利用された場合に問題が起こる可能性がありました。

13-5W12: (FONT) OpenType フォントの改行時のプロポーショナルメトリクス情報の計算の不具合の修正

OpenType フォントの改行時のプロポーショナルメトリクス情報の計算の不具合を修正しました。

複数行に渡るテキストを描画した場合、改行した行と文末の行とで行の幅が変わることがあったため、これを修正しました。

13-5W13: (レイアウト-UI2D) LayoutEditor で、部品ペインの上書き時に、カラー補完だけ上書きをした場合、データーによってはUIが正しく初期化されずツールがヌル参照で異常終了する不具合を修正

LayoutEditor で、部品ペインの上書き時に、カラー補完だけ上書きをした場合、データーによってはUIが正しく初期化されずツールがヌル参照で異常終了する不具合があありこれを修正しました。

13-5W14: (レイアウト-UI2D) nn::ui2d::DrawInfo::SetInvisiblePaneCalculateMtx() を使用した場合に非表示のペインが描画されることがある不具合の修正

nn::ui2d::DrawInfo::SetInvisiblePaneCalculateMtx() を使用した場合に非表示のペインが描画されることがある不具合を修正しました。

13-5W15: (レイアウトツール) LayoutEditor でグリッドに吸着が有効にならない不具合の修正

LayoutEditor で表示メニューのグリッドに吸着が有効にならない不具合があり、これを修正しました。

13-5W16: (エフェクト-VFX) VFX ランタイムで、GPU+SOエミッタで速度 0 のパーティクルに「速度スケールY」を適用すると表示されない不具合の修正

GPU+SOエミッタで速度 0 のパーティクルに「速度スケールY」を適用すると表示されない不具合を修正しました。速度 0 の場合は大きさが変わらないのが正しい挙動です。

13-5W17: (エフェクト-VFX) フレームレート 0 で nn::vfx::System::Calculate() を実行したときに、エミッタ位置が正しく反映されなくなる不具合の修正

フレームレート 0 で nn::vfx::System::Calculate() を実行したときに、EmitterSet が内部で保持する DirtyFlag が消されてしまうため、 nn::vfx::EmitterSet::SetPos() などで変更したエミッタ位置が正しく反映されなくなる不具合を修正しました。

13-5W18: (SPY) Spy.exe で収集したデータをファイル保存できない不具合の修正

Spy.exe のメニューにある保存ボタンが常に押せないために、収集したデータをファイル保存できない不具合がありましたが、これを修正しました。
