NintendoSDK NX Addon 1.4.0 (based on Generic 1.4.0) の変更点

機能追加

1-4-0S01: (DevMenu) イベント用モード(exhibition mode) の追加

詳細は 開発機の制御／DevMenu の「自動起動アプリケーションの設定」 を参照してください。

1-4-0S02: (MII) DevMenu に Mii 開発支援を行う機能の追加

詳細につきましては、開発機の制御／DevMenu の Mii Testing Mode 以下を参照してください。

1-4-0S03: (MOVIE) Decoder および Extractor スレッドのアフィニティを指定するAPIの追加

Decoder および Extractor 利用時には movie ライブラリ内部でスレッドを作成します。
これらのスレッドをどの CPU コア上で作成するかを指定する movie::SetCoreMask() を追加しました。

1-4-0S04: (MOVIE) MovieDecoderPlayer へDecoder::Reset の呼び出し例の実装を追加

MovieDecoderPlayer サンプルに対して movie::Decoder::Reset の使用例の実装を追加しました。
その実装を適用するにはサンプルの起動時に -reset を渡す必要があります。

1-4-0S05: (OS) 指定されたアドレス空間のメモリがロックされているかどうかを検査する機能を追加

アプリケーション開発時のデバッグ専用の機能として、指定されたアドレス空間のメモリがロックされているかどうかを検査する nn::os::IsMemoryLocked() という API を追加しました。この機能は製品機では使用できません。

1-4-0S06: (TOOL) AuthoringEditor に nmeta または nsp ファイル同士の比較機能を追加

AuthoringEditor 上で、2 つのファイルの差分確認が行えるようになりました。
比較をサポートするのは次のファイルの組み合わせです。

  • アプリケーション nmeta ファイル同士
  • アプリケーション nsp ファイル同士
  • 追加コンテンツ nmeta ファイル同士
  • パッチ nsp ファイル同士
  • パッチ nsp ファイルとアプリケーション nsp ファイル


1-4-0S07: (TOOL) AuthoringEditor にパッチ nsp ファイルのサポートを追加

AuthoringEditor 上からパッチ nsp に関する以下の操作が行えるようになりました。

  • パッチ nsp の新規作成 (AuthoringEditor 起動直後の画面より操作)
  • パッチ nsp の内容確認


1-4-0S08: (TOOL) AuthoringEditor に追加コンテンツ nsp ファイルのサポートを追加

AuthoringEditor 上から追加コンテンツ nsp に関する以下の操作が行えるようになりました。

  • 追加コンテンツ nmeta ファイルの新規作成
  • 追加コンテンツ nsp の新規作成
  • 追加コンテンツ nsp の内容確認


改善

1-4-0S09: (Compiler) Clang for NX 1.2.3 への更新

Clang for NX のバージョンを 1.2.3 に更新しました。
変更内容は以下の通りです。

アプリケーションのビルド時に、リンクに失敗したり、リンク時間が以前と比較して非常に長くなるケースを改善するために、コンパイラがデフォルトオプションとして指定していたデバッグ情報の出力オプションを -fno-standalone-debug に戻しました。

1-4-0S10: (DevMenu) DevMenu, DevMenuCommand からの起動時、初版アプリケーションとパッチの組み合わせが間違っている場合は起動させない機能の追加

初版アプリケーションとパッチの組み合わせが間違っているとき、従来は起動した後に fs のエラーでアプリケーションが fatal していました。
しかしこの挙動だと fatal の原因が分かりづらいため、DevMenu や DevMenuCommand からの起動時に初版アプリケーションとパッチの組み合わせが正しいか確認し、間違っていたら「The application is not one that is used to create the patch」というエラーメッセージを表示して起動させないようにしました。
ただし HomeMenu ではこのチェックは働きません。

1-4-0S11: (DevMenu) SDK ドキュメント Accounts ページの説明で Open, Close 状態の表示について言及

SDK ドキュメント「開発機の制御 > DevMenu」にて、 DevMenu の Accounts ページに関して次の説明を追記しました。

* ユーザーアカウント一覧で表示される情報
* このうち Open 状態に関する説明において、アカウントガイドについての言及の作成

1-4-0S12: (MOVIE) MovieDecoderPlayer で行われていた余分な memcpy を削除

movie::Decoder から取得したデコード結果を NVNTexture へコピーするまでに行っていた memcpy の回数を減らしました。

不具合修正

1-4-0S13: (DevMenu) アプリケーションのタイトルにマルチバイト文字が含まれていると DevMenu 起動時に Fatal エラーが発生することがある不具合の修正

アプリケーションのタイトルが以下の条件を満たしていると、DevMenu 起動時に Fatal エラーが発生することがある不具合を修正しました。

  • 終端文字を含むタイトル名が 64 バイト以上である
  • タイトルにマルチバイト文字が含まれている


1-4-0S14: (MOVIE) MovieDecoderPlayer で動画再生開始時にカクついてしまう不具合の修正

動画の再生直後に絵や音が途切れてしまうことがありましたが、これが起こらないように修正しました。

1-4-0S15: (OS) ファイバ関数内のスタックフレームのアライメントが正しくなかった不具合を修正

nn::os::InitializeFiber() で指定したファイバ関数内におけるスタックフレームのアライメントが正しくなかった不具合を修正しました。これにより、例えばファイバ関数内の自動変数に対して NX32 では 8byte 以上、NX64 では 16byte 以上のアライメント制約を付加したものが正しいアライメント位置に配置されなかった不具合が解消されます。

1-4-0S16: (TOOL) AuthoringEditor 上で入力したファイルパスに関する仕様変更

AuthoringEditor 上で nmeta ファイル に入力したファイルパスは、保存時に書き出した nmeta ファイルパスからの間接参照（相対パス）へ変換されます。
異なるドライブレター、もしくはネットワーク上の場所を指し示すパスの場合のみ、絶対参照（絶対パス）のまま nmeta ファイルに保存されます。

1-4-0S17: (TOOL) AuthoringTool の NX カード向けパラメータチェックの不具合の修正

AuthoringTool による replace, makepatch コマンド実行時に、特定の条件を満たすと NX カード向けパラメータチェックで不当にエラーを検出してしまい処理が失敗する問題を修正しました。

1-4-0S18: (TOOL) シェーダコンパイラで、Windows7 環境での実行時に応答がなくなることがある不具合の修正

シェーダコンパイラ (%NINTENDO_SDK_ROOT%\Tools\Graphics\GraphicsTools\ShaderCompiler.dll) とシェーダコンバータ (%NINTENDO_SDK_ROOT%\Tools\Graphics\GraphicsTools\ShaderConverter.exe) で、Windows7 環境でシェーダをコンパイルした際に確率的に応答がなくなることがある不具合への対策を行いました。

1-4-0S19: (TOOL) テクスチャコンバーターで、キューブマップの DDS ファイルから変換すると +Z 方向と -Z 方向の画像が逆になる不具合の修正

テクスチャコンバーター（%NINTENDO_SDK_ROOT%\Tools\Graphics\GraphicsTools\TextureConverter.exe）で、ミップマップを持たないキューブマップの DDS ファイルから変換すると +Z 方向と -Z 方向の画像が逆になる不具合があり、これを修正しました。

NintendoWare NX Addon 1.4.0 (based on Generic 1.4.0) の変更点

互換性の失われる仕様変更

★1-4-0W01: (エフェクトツール) EffectMaker が出力するバイナリのバージョンを更新します

今後リリースする SDK とのバイナリ互換性を可能な限り維持するため、バイナリ領域の調整を行いました。
これにより、以前のバージョンの EffectMaker で作成したバイナリは互換が切れますので、リソースの再コンバートが必要です。

機能追加

1-4-0W02: (オーディオツール) SoundProjectConverter にアイテムフィルタ機能を追加

コンバートのタイミングでサウンドプロジェクトのアイテムや、バンクのインストルメントをフィルタして、サウンドアーカイブに含めるかどうかを制御できるようになりました。

この機能を利用するには、SoundProjectConverter の --exclude-list, --include-list オプションを指定します。
詳しくは、SoundProjectConverter の解説ページを参照ください。

※この機能は評価中のため、予告なく仕様変更が入る可能性があります。

1-4-0W03: (エフェクト-VFX) nn::vfx::Emitter::GetCustomShaderSwitch() メソッドを追加

nn::vfx::Emitter::GetCustomShaderSwitch() メソッドを追加しました。
カスタムシェーダのラジオボタンの押下状態を取得できます。

1-4-0W04: (エフェクト-VFX) nn::vfx::System::Calculate( nn::vfx::EmitterSet* pEmitterSet ) メソッドを追加

nn::vfx::System::Calculate( nn::vfx::EmitterSet* pEmitterSet ) メソッドを追加しました。EmitterSet 単体で計算処理を行いたい場合は、こちらのメソッドを利用していただくことで、System クラスがカウントしている再生中エミッタセット数などのデバッグ情報も更新されるようになります。

改善

1-4-0W05: (エフェクト-VFX) vfx ランタイムが出力する警告メッセージ内容を改善

vfx ランタイムが出力する警告メッセージ内容を改善し、警告に関する ドキュメント を整備しました。

1-4-0W06: (GFX) libnns_gfx で使用するシェーダーバイナリを生成するプロジェクトを libnns_gfx プロジェクトへ統合

これまで libnns_gfx で使用するシェーダーバイナリを生成するプロジェクトが NnsGfxShader プロジェクトとして分離されていましたが、このプロジェクトを libnns_gfx へ統合しました。libnns_gfx で使用するシェーダーバイナリは libnns_gfx ビルド時に自動的にビルドされるようになります。

不具合修正

1-4-0W07: (オーディオ-ATK) atk ライブラリのリビルドに失敗する不具合の修正

ビルドに必要なヘッダが不足していて、atk ライブラリのリビルドに失敗する不具合を修正しました。

1-4-0W08: (3D-G3D) 依存するシェーダーパラメーターへのポインタの書き込み先が間違っていた不具合の修正

シェーダーパラメーターにシェーダーアノテーション depend を記述した場合、ランタイム時に依存するシェーダーパラメーターへのポインタをシェーダーパラメーターの末尾にライブラリが書き込んでいますが、この書き込み先が間違っていました。4バイト後ろにずれた位置に書き込んでいたため、正しく依存するシェーダーパラメーターへのポインタが参照できない、後ろに配置されている別のシェーダーパラメーターが壊れているなどの問題が発生していました。

depend アノテーションで自身を指定している場合、ライブラリはポインタの書き込みを行わないため、この問題は発生していませんでした。

depend アノテーション記述時の挙動については、 シェーダーパラメーターを参照してください。

1-4-0W09: (レイアウト-UI2D) 詳細コンバイナを利用時にテクスチャのマッピング設定が正しく描画されない不具合の修正

LayoutEditor で詳細コンバイナを設定し、テクスチャのマッピング設定を UV マッピング以外にすると、ランタイム側で正しく描画されない不具合があり、これを修正しました。

1-4-0W10: (レイアウトツール) LayoutEditor でプロジェクトルートにプロジェクト設定ファイルがない状態でテクスチャが読み込めない不具合の修正

LayoutEditor でプロジェクトルートにプロジェクト設定ファイルがない状態で LayoutEditor を起動するとテクスチャが読み込めなくなる不具合あり、これを修正しました。

1-4-0W11: (レイアウトツール) LayoutEditor でワークスペースの復元を行うと色見本の項目が増加する不具合の修正

LayoutEditor でワークスペースの復元を行うと色見本の項目が増加する不具合があり、これを修正しました。

1-4-0W12: (レイアウトツール) LayoutEditor の部品・コントロール設定でペイン名や区間タグが変更できない不具合の修正

LayoutEditor の部品・コントロール設定で機能ペインの一覧のペインの列や機能アニメーションの一覧の区間タグの列の項目が変更できない不具合があり、これを修正しました。

1-4-0W13: (エフェクト-VFX) nn::vfx::viewer::ViewerSystem::UpdateModelInfo 呼び出し時不具合を修正

nn::vfx::viewer::ViewerSystem::UpdateModelInfo にてアプリ内モデル情報更新時に、既にビューアランタイムに登録されているアプリ内モデル情報を削除できていなかった不具合を修正しました。

1-4-0W14: (エフェクト-VFX) チャイルドエミッタのリアルタイム編集が動作しない不具合を修正

EffectMaker を利用して、EffectViewer や アプリ で再生中 のチャイルドエミッタをリアルタイム編集した際に、一部のパラメータが正しく編集できない不具合を修正しました。

1-4-0W15: (エフェクトツール) EffectMaker エミッタセットファイルを大量にまとめてコンバートするとコンバートに失敗する問題を修正

EffectMaker において、エミッタセットファイルを 1400 個程度以上まとめてコンバートするとテクスチャコンバートエラーになる問題を修正しました。

1-4-0W16: (エフェクトツール) EffectMaker エミッタの描画を入れ替えるとビューア上の表示が崩れる問題を修正

EffectMaker において、プリミティブやテクスチャが設定されたエミッタのノード順を入れ替えて描画順を変更したとき、ビューア上での表示が崩れる問題を修正しました。

1-4-0W17: (エフェクトツール) EffectMaker カスタムアクションのコールバック ID が書き換わる問題を修正

EffectMaker において、カスタムアクションユーザー定義の数が eset ファイルを保存したときより増えていた場合、eset ファイルを開いたときにカスタムアクションコールバック ID が「コ－ルバックを利用しない」から他の ID に書き換わる問題を修正しました。
