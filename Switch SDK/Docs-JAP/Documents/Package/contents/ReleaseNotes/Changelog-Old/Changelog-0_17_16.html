NintendoSDK NX Addon 0.12.16 (based on Generic 0.17.16) の変更点

機能追加

★13-17S01: (PCM) 開発機内部の消費電力を計測する開発用機能 (PCM ライブラリ) の追加

開発機内部の消費電力を計測する開発用機能 (PCM ライブラリ) を追加しました。
本機能を使用することで、アプリ実行中の開発機内部での消費電力を計測することができます。
この値の変化をシーンごとにチェックすることで、電池の消費が相対的に激しい箇所を確認することができます。

注意事項

  • 計測される電力の絶対値は、個体によってばらつきがあります。相対的に消費電力の大きなシーンを確認するといった使い方に留めるようにしてください。
  • 消費電力の計測は SDEV でのみ可能です。
  • 本ライブラリ API は製品環境では使用できません。


詳細は NintendoSDK ドキュメントの PCM ライブラリ を参照してください。

この変更は NX Addon 0.12.15 に含まれていましたが、変更履歴から漏れていました。

13-17S02: (NVN) nvn の新機能

nvn に以下の機能を追加しました。

  • Added new device flag NVN_DEVICE_FLAG_DEFERRED_FIRMWARE_MEMORY_RECLAIM_BIT to control when firmware memory used to store GPU page table entries is freed

  • Added new API functions to obtain total queue memory usage from queue commands since the queue was last initialized or the usage counters were reset.
    • Added the functions nvnQueueGetTotalComputeMemoryUsed, nvnQueueGetTotalControlMemoryUsed, and nvnQueueGetTotalCommandMemoryUsed to get the total queue compute, control, and command memory used (in bytes).
    • Added function nvnQueueResetMemoryUsageCounts to reset the usage counters for tracking queue memory usage.


この変更は NX Addon 0.12.10 で含まれていましたが、変更履歴から漏れていました。

改善

★13-17S03: (CODEC) nn::codec::HardwareOpusDecoder 利用の非推奨化

nn::codec::HardwareOpusDecoder 利用時に、システムスリープ状態にし、ドックインを行うと、システムが停止する不具合が見つかり、これに対するアプリ側での回避策がないため、利用を非推奨としました。

13-17S04: (FS) FsAccessLogChecker を version 3.2 に更新

FsAccessLogChecker (%NINTENDO_SDK_ROOT%\Tools\FsAccessLogChecker\FsAccessLogChecker.exe) を version 3.2 に更新しました。以下の変更が含まれます。

  • ファイルの上書き確認に対応
  • Invalid timestamp order を誤検出する問題を修正


13-17S05: (FS) FsAccessLogChecker をファイルの上書き確認に対応

FsAccessLogChecker (%NINTENDO_SDK_ROOT%\Tools\FsAccessLogChecker\FsAccessLogChecker.exe) version 3.1 にて、結果をファイルに出力する場合に出力ファイルが既に存在していたら上書きするか確認するように変更しました。
また、上書き確認をしないようにする -f,--force オプションを追加しました。

13-17S06: (General) nmeta ファイルの追加

サンプルのリソースに nmeta ファイルが追加されました。
また、VSI のウィザードを利用してアプリケーションを生成する際には、 nmeta ファイルがプロジェクトにコピーされるようになります。

13-17S07: (VSI) NX Addon 0.12 系 NX-VSI, Target Manager の互換性向上

VSI for NX 1.1.1 Hotfix2 をインストールしておくことで、NX Addon 1.1.x <-> NX Addon 0.12.16 の切り替えの際に VSI を更新する必要がなくなりました。
NX Addon 0.12.16 以降であれば問題ありませんが、NX Addon 0.12.15 以前を使用される場合は、各パッケージに含まれる NX-VSI をインストールしてください。

不具合修正

13-17S08: (FRIENDS) GetProfileList に渡すネットワークサービスアカウント ID リストをすぐに破棄した時、予期しない結果になる問題の修正

GetProfileList に渡すネットワークサービスアカウント ID リストをすぐに破棄した時、予期しない結果になる問題を修正しました。

13-17S09: (FS) FsAccessLogChecker が Invalid timestamp order を誤検出する問題を修正

FsAccessLogChecker (%NINTENDO_SDK_ROOT%\Tools\FsAccessLogChecker\FsAccessLogChecker.exe) が誤って Invalid timestamp order を検出し、解析エラーとしてしまう問題があり、これを FsAccessLogChecker version 3.2 で修正しました。

13-17S10: (SWKBD) NX32 で TextCheckCallback から渡される文字列が正しくない不具合の修正

NX32 で TextCheckCallback から渡される文字列が正しくない不具合を修正しました。

13-17S11: (TOOL) 追加コンテンツ生成時のエラーチェックの強化

AuthoringTool にて追加コンテンツを生成するときのエラーチェックが強化されました。下記のケースでエラーが発生するようになります。

  • Index の値が 1 以上 2000 以下を満たさない場合
  • 1つの nmeta 内で Index が重複して指定されている場合


NintendoWare NX Addon 0.12.16 (based on Generic 0.17.16) の変更点

機能追加

13-17W01: (エフェクト-VFX) EffectMaker の一部のストライプへのZ軸回転機能の追加

一部のストライプについて、Z軸回転が作用するように機能拡張を行いました。Z軸回転は EffectMaker の「回転」タブからパーティクルの回転と同様に設定可能で、ストライプの進行方向を軸に、ストライプの帯をねじるような挙動になります。
対象となるストライプは、以下の設定がされたストライプです。それ以外の組み合わせについては対応していません。

  • 「履歴式ストライプ」で、「計算方法」が「エミッタ行列ストライプ」または「エミッタ上下に伸ばす」のもの
  • 「履歴式ストライプ2」で、「計算方法」が「エミッタ行列ストライプ」または「エミッタ上下に伸ばす」のもの


13-17W02: (エフェクト-VFX) EffectMaker の一部のストライプへの遮蔽チェック機能の追加

一部のストライプについて、「遮蔽チェック」機能が作用するように機能拡張を行いました。遮蔽チェックは EffectMaker の「カラー」タブからパーティクルと同様に設定可能で、ストライプの先端が遮蔽された場合に、ストライプ全体がアルファフェードするという挙動です。
対象となるストライプは、以下の通りです。それ以外のストライプについては対応していません。

  • 「履歴式ストライプ」
  • 「履歴式ストライプ2」
