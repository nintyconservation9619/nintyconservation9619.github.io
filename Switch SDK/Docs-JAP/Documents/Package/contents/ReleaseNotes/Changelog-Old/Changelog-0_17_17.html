NintendoSDK NX Addon 0.12.17 (based on Generic 0.17.17) の変更点

互換性の失われる仕様変更

★13-18S01: (IRSENSOR) モーションIRカメラを複数台使用したアプリで、アプレット遷移して上限台数を超えるコントローラを接続した状態でアプリに戻った際にアボートすることがある不具合を修正

モーションIRカメラを複数台使用する場合、例えば、4 台中 2 台でモーションIRカメラを同時利用するといったケースで、メニューなどに遷移してコントローラを追加接続し、5 台接続状態でアプリに戻った際に、アプリ自体は制限台数を守っていてもアプリがアボートしてしまう不具合がありました。
これを回避するため、モーションIRカメラの接続台数制限を違反した場合にデータ取得関数を呼ぶと、ライブラリ内で「Hid Resource Error」とログ出力し、 nn::irsensor::ResultIrsensorNotReady のエラーを返すよう仕様変更を行いました。
こちらのエラーは、これまでと同様にデータが来ない状態となりますので、特にハンドリングポリシーを変える必要はありません。
本修正によってアボートは回避できますが、同問題のシーケンスに入った際には、上限台数を守っていないと、アプリ復帰後も 1 台しかモーションIRカメラが動作しない状況になりますので、複数台利用する場合の上限台数設定には注意が必要です。
将来的には、同問題のシーケンスに入った場合でもエラーを返さないよう修正する予定です。

★13-18S02: (OE) 廃止予定にしていた性能 (nn::oe::SetPerformanceConfiguration) の定義の削除

nn::oe::SetPerformanceConfiguration で設定できる性能の中で、廃止予定としていた以下の設定を定義から削除しました。

  • nn::oe::SetPerformanceConfiguration_Cpu1020MhzGpu307MhzEmc1600Mhz
  • nn::oe::SetPerformanceConfiguration_Cpu1020MhzGpu384MhzEmc1600Mhz


機能追加

13-18S03: (TOOL) nmeta の DataPath で環境変数を指定できる機能を追加

追加コンテンツ作成時の nmeta の項目の DataPath に対して環境変数を指定する機能を追加しました。デフォルトでは無効状態です。
有効にするには、DataPath の属性に UseEnvironmentVariable="true" を指定する必要があります。詳細は「AOC ライブラリ／追加コンテンツの作成方法」を参照してください。

改善

★13-18S04: (General) InitializeSdev で初期化できる対象機材から SDEV1.6 を削除

InitializeSdev で SDEV1.6 を初期化できなくなります。
UI 上では初期化する対象の機材として選択することができてしまいますが、実際に初期化をすることはできませんのでご注意ください。

13-18S05: (BuildSystem) ウィザードの SDK ルート選択リストボックスの表示順変更

「NintendoSDK アプリケーション」「NintendoSDK ライブラリ」プロジェクト作成用ウィザードの SDK ルート選択リストボックスの表示順を逆順 (インストールした日が新しい順) に変更しました。

13-18S06: (TOOL) AuthoringTool への get-unpublishable-error オプションの追加と --error-unpublishable オプションのチェック項目の追加

AuthoringTool で --error-unpublishable を指定した際のチェック項目に DebugAPI の使用の有無を追加しました。
また、 AuthoringTool で get-unpublishable-error オプションを指定することで DebugAPI の使用の有無を確認できる機能を追加しました。

13-18S07: (TOOL) InitializeEdev の冗長な処理の改善

InitializeEdev で今は不要な本体再起動処理がありましたので省略するようにしました。

不具合修正

13-18S08: (BuildSystem) ユーザー名に括弧を含む環境で SDK をインストールすると失敗する不具合の修正

ユーザー名に括弧を含むユーザーでログインした状態で SDK をインストールすると ウィザードのインストールが失敗する不具合を修正しました。

13-18S09: (General) nspd 起動時にオフライン HTML が参照できない不具合の修正

nmeta ファイルにオフライン HTML などのパスを指定したときに指定した内容が反映されない不具合を修正しました。

13-18S10: (TOOL) GUI 版セットアップツールで例外が発生する不具合の修正

GUI 版セットアップツール (InitializeSdevWin・InitializeEdevWin) で例外ウィンドウが表示される場合がありましたので対策を入れました。

13-18S11: (TOOL) InitializeEdev が途中で失敗する場合がある不具合の修正

InitialiazeEdev が "Parse GetSerialNumber output failed." というエラーログを出力させて失敗する場合がある不具合がありましたので修正しました。

13-18S12: (TOOL) セットアップツールで初期化実行後 Target Manager のデフォルトターゲットが変わる不具合の修正

セットアップツール (InitializeSdev・InitializeEdev) による初期化実行後に Target Manager のデフォルトターゲット設定が初期化した開発機に変わる仕様なのですが、変わっていなかった不具合を修正しました。

NintendoWare NX Addon 0.12.17 (based on Generic 0.17.17) の変更点

互換性の失われる仕様変更

★13-18W01: (FONT) RectDrawer の ワークメモリサイズの計算が間違っていて、正しいアラメント制約を満たしていないメモリを渡すと、オーバーランすることがある不具合の修正

nn::font::RectDrawer::Initialize() にて、ワークメモリサイズの計算が間違っていて、正しいアラメント制約を満たしていないメモリを渡すと、オーバーランすることがある不具合がありこれを修正しました。新しいバージョンでは、ワークバッファにアラメント制約を要求するように仕様変更をし、誤った、アライメントのメモリが設定された場合にアサートに失敗します。 必要なアライメント値は、nn::font::RectDrawer::GetWorkBufferAlignment() で取得するようにしてください。不具合は、1524byte 以上アライメント調整が行われる場合におこり、メモリ破壊や、意図しないメモリ属性の上書きによる、メモリ属性違反エラーが発生していました。以前(NintendoSDK v0.16よりも前)から、存在していた不具合なので、実際に問題が顕在化する状況は少なかったと予想しています。

不具合修正

13-18W02: (エフェクトツール) EffectMaker で、ストライプの「エミッタに完全追従」を設定すると、法線が正しく計算されていなかった不具合の修正

EffectMaker で、「履歴式ストライプ」「履歴式ストライプ2」の設定で「エミッタに完全追従」を有効にすると、法線が正しく計算されていなかった不具合を修正しました。
