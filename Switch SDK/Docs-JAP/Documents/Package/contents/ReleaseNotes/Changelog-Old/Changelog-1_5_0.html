NintendoSDK NX Addon 1.5.0 (based on Generic 1.5.0) の変更点

互換性の失われる仕様変更

★1-5-0S01: (VI) viの一部API廃止

NN_DEPRECATEDとして設定されていた下記のAPIをvi_Display.h, vi_Layer.hから削除しました

  • nn::vi::SetDisplayEnabled
  • nn::vi::GetDisplayResolution
  • nn::vi::CreateLayer(Layer*, Display, int, int, PixelFormat)
  • nn::vi::SetLayerAlpha


機能追加

1-5-0S02: (MII) デバッグ用のデータベースファイルの追加

開発の利便性のため、いくつかの似顔絵データベースのサンプルファイルを追加しました。
詳細については デバッグ用サンプルデータベース をご覧ください。

1-5-0S03: (NGC) nn::ngc::ProfanityFilter クラスと nn::ngc::CountNumbers() に UTF-16 文字列を入力に取れるインターフェイスを追加

以下の API に UTF-16 文字列を入力に取れるインターフェイスを追加しました。

  • nn::ngc::ProfanityFilter::CheckProfanityWords()
  • nn::ngc::ProfanityFilter::MaskProfanityWordsInText()
  • nn::ngc::CountNumbers()


改善

1-5-0S04: (GFX) nvn 版で GetGpuAddress の結果をキャッシュするように変更

nvn 版で nn::gfx::Buffer::GetGpuAddress が呼ばれたときに nvn をコールしないようにし、事前取得した結果を返すようにしました。
これにともなって、nn::gfx::Buffer のサイズが 8 バイト増加しています。

1-5-0S05: (TOOL) AuthoringEditor 上でのレーティング情報の入力項目を改善

レーティング情報の入力がより分かりやすくなりました。
AuthoringEditor 上で仕向け地または販売方法を選択すると、その販売形態に必要なレーティング団体の入力項目のみが表示されます。

1-5-0S06: (TOOL) AuthoringTool の --error-unpublishable オプションのチェック項目を追加

AuthoringTool で --error-unpublishable を指定した際のチェック項目にソフトリーガル情報の整合性確認を追加しました。

不具合修正

1-5-0S07: (GFX) nns_gfx ライブラリ がビルドできない不具合を修正

nns_gfx ライブラリ が NintendoWare パッケージに依存してしまい、NintendoSDKパッケージ単体ではビルドできない不具合を修正しました。

1-5-0S08: (MOVIE) SoftAAC2内のメモリリークの修正

SoftAAC2内で確保されていたメモリが、動画の再生終了時に一部正しく解放されていない不具合があり、これを修正しました。

NintendoWare NX Addon 1.5.0 (based on Generic 1.5.0) の変更点

互換性の失われる仕様変更

★1-5-0W01: (3D-G3D) バイナリーリソースファイルのフォーマット更新

今後バイナリーリソースファイルの互換性を可能な限り維持するため、 bfres ファイルおよび bfsha ファイルのバイナリーフォーマットの調整を行いました。 これにより、以前のバージョンの 3D バイナリーコンバーターおよび 3D シェーダーコンバーターで作成したバイナリーファイルとの互換が切れますので、リソースの再コンバートが必要です。

機能追加

1-5-0W02: (オーディオツール) SoundMaker のコマンド機能にプロジェクトの保存を追加

SoundMaker のコマンドラインオプションで指定できるコマンド機能にプロジェクトの保存ができるコマンドを追加しました。

不具合修正

1-5-0W03: (オーディオツール) SoundProjectWaveExporter でダウンサンプルに失敗する不具合を修正

SoundProjectWaveExporterにダウンサンプルを有効化したプロジェクトファイル (.fsp) を指定すると、必ずダウンサンプル時に失敗してエラー終了してしまう問題がありましたが、これを修正しました。
