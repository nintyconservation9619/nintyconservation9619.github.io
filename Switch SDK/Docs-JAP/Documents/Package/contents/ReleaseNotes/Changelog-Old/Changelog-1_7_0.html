NintendoSDK NX Addon 1.7.0 (based on Generic 1.7.0) の変更点

互換性の失われる仕様変更

★1-7-0S01: (FS) デバッグ用セーブデータをマウントする nn::fs::MountSaveData(const char* name) の削除

デバッグ用セーブデータをマウントする nn::fs::MountSaveData(const char* name) を削除しました。
デバッグ用セーブデータをマウントする場合は nn::fs::MountSaveDataForDebug() を利用してください。

★1-7-0S02: (General) InitializeSdev で初期化できる対象機材から SDEV1.8 と preMP を削除

InitializeSdev で SDEV1.8 と preMP を初期化できなくなります。
UI 上では初期化する対象の機材として選択することができてしまいますが、実際に初期化をすることはできませんのでご注意ください。

改善

★1-7-0S03: (TOOL) ゲームカード容量内のアプリケーションが使える容量の増加

ゲームカード容量の内、アプリケーションへの割当てが保証された領域を超える場合の許容量を 128 MB から 144 MB に変更しました。

1-7-0S04: (TOOL) nmetaのタッチスクリーン使用設定の削除

nmetaで指定したタッチスクリーン使用設定を削除しました。
こちらの設定を本体機能上で使わない方針になり、
そのためアプリのnmetaでの設定は不要になりました。

不具合修正

1-7-0S05: (TOOL) AuthoringEditor のレーティング入力フォームにおける PEGIPortugal レーティング区分の不足を修正

AuthoringEditor のレーティング入力フォーム内の PEGIPortugal レーティング区分について不足があったため追加しました。
追加したのは PEGIPortugal レーティング区分の 年齢値 3 および 7 の項目です。

1-7-0S06: (TOOL) AuthoringEditor 上で異なるメジャーバージョンの SDK でビルドされた nsp が変更できる不具合の修正

AuthoringEditor が含まれる SDK とは異なるメジャーバージョンの SDK でビルドされたアプリケーション nsp について、AuthoringEditor で開いた際に編集が可能となっていた不具合を修正しました。
SDK のメジャーバージョンが異なる nsp は常に読み取り専用で開かれます。

1-7-0S07: (TOOL) InitializeEdev で EDEV 複数台接続時に違う本体が初期化される場合がある不具合の修正

InitializeEdev で EDEV 複数台接続時に違う本体が初期化される場合がある不具合を修正しました。
