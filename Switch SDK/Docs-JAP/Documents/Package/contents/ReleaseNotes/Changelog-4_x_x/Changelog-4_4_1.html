NintendoSDK NX Addon 4.4.1 (based on Generic 4.4.1) の変更点

機能追加

★4-4-1S01: (TOOL) EDEV 本体更新ツール (SystemUpdateEdev) を追加

PC から EDEV の本体更新を行うためのツール SystemUpdateEdev を追加しました。

従来の初期化ツール (InitializeEdev) に比べて各種設定が保持されたままファームウェアのバージョンアップができるという特徴があります。
使い方は SystemUpdateEdev のコマンド解説のページ 等を参照して下さい。

また SDEV 向けの本体更新ツール SystemUpdateSdev もありますのでそちらもご活用ください。

4-4-1S02: (DevMenu) ゲームカードに nsp ファイルを書き込む機能を追加

microSD カードの nsp ファイルをゲームカードに書き込む機能を追加しました。Fatal エラーが発生する既知の不具合があるため、既知の不具合ページも合わせて参照してください。

4-4-1S03: (FS) nn::nfp のアクセスログ出力追加と FsAccessLogChecker version 11.0 でのチェックに対応

fs アクセスに関わる nn::nfp API でアクセスログ出力を追加し、FsAccessLogChecker (%NINTENDO_SDK_ROOT%\Tools\FsAccessLogChecker\FsAccessLogChecker.exe) version 11.0 にて nn::nfp 経由の fs アクセスをチェックするよう対応しました。詳細は FSライブラリ／デバッグ機能／アクセス頻度確認 を参照してください。

不具合修正

★4-4-1S04: (TOOL) AuthoringTool がパッチ生成に失敗する不具合を修正

初版アプリケーションが特定のデータパターンのとき、パッチ生成でエラーが発生する不具合があったため、これを修正しました。

4-4-1S05: (FS) FsAccessLogChecker version 10.8 でリードアクセスの重なり判定を修正

FsAccessLogChecker (%NINTENDO_SDK_ROOT%\Tools\FsAccessLogChecker\FsAccessLogChecker.exe) で、複数のパターンでリードアクセスが重なる場合に重なりを見逃す不具合があり、これを FsAccessLogChecker version 10.8 で修正しました。
