4-0-0S15: (NVN) nvn の新機能

nvn に以下の機能を追加しました。

  • Added new API entry points nvnCommandBufferDownsampleTextureView and nvnCommandBufferTiledDownsampleTextureView to support using texture views for level/layer selection on the source texture for downsampling operations. These functions compliment the existing NVN texture downsample functions.

  • Added API entry point nvnDeviceIsExternalDebuggerAttached to allow applications to query whether a debugger tool is attached to the driver while running an NVN application.

  • Added new API entry points nvnTextureReadTexelsStrided and nvnTextureWriteTexelsStrided to allow applications to specify row and image strides of pitch-linear application data when reading from or writing to textures, respectively.

  • Added new API entry points to support copying debug marker strings into the command buffers.
    • Added the function nvnCommandBufferInsertDebugMarkerStatic to make a shallow copy of the marker annotation string pointer to the command buffer which can be used with statically allocated strings.
    • Added the function nvnCommandBufferInsertDebugMarkerDynamic to make a deep copy of the contents of the marker annotation string into the command buffer which can be used with dynamically allocated strings.

  • Added new API entry points to use the texture LOD snap optimization with samplers.
    • Added the function nvnSamplerBuilderSetLodSnap to set the texture LOD snap value for the sampler builder.
    • Added the function nvnSamplerBuilderGetLodSnap to query the texture LOD snap value from the sampler builder.


glslc に以下の機能を追加しました。

  • Added API functions to perform specialized multi-threaded compilations.
    • Added the API function glslcCompileSpecializedMT which is the thread-safe version of glslcCompileSpecialized. This function can be called on multiple threads using the same shared GLSLCcompileObject.
    • Added the API function glslcFreeSpecializedResultsMT which must be used to free the GLSLCresults returned from glslcCompileSpecializedMT.

  • Added ability to specialize statically and dynamically indexed uniforms at all optimization levels.
    • In some cases the compiler will be able to correctly specialize dynamically indexed uniforms at the "default" optimization level, but will fail to specialize when optimizations are disabled. If the compiler can not perform the specialization, an error will be emitted.

インゲームビューアの機能を利用して、ゲーム内リソース と EffectMaker で編集中リソースをバインドした際に、EffectMaker 側 に バインドマーク が表示されない不具合を修正しました。

4-0-0W105: (エフェクトツール) EffectMaker で特定のテクスチャが正しくプレビューされない問題を修正

EffectMaker にて float_bc6、ufloat_bc6 テクスチャを選択した際、テクスチャ表示部分に Error と表示される問題を修正しました。
また、snorm フォーマットの場合、EffectMaker のテクスチャ表示と、ビューアの見え方が異なるため、該当テクスチャを設定した際、警告文を表示するように対応しました。

4-0-0W106: (エフェクトツール) EffectMaker プリミティブファイルのパスが検索パス内にあっても警告が出る問題を修正

EffectMaker において、エミッタにプリミティブファイルをセットしたとき、そのプリミティブファイルが検索パス内に配置されていたとしても誤検出により警告が表示されてしまう問題を修正しました。

4-0-0W107: (エフェクトツール) EffectMaker 編集済みのカラータブの色が未編集の状態になる不具合の修正

エミッタのカラー値を変更後、他エミッタを選択し、元のエミッタを再選択すると、カラータブが未編集状態の表示になる問題を修正しました。

4-0-0W108: (GFX) PrimitiveRenderer の終了処理で設定する初期化フラグの値を修正

PrimitiveRenderer の終了処理で初期化フラグを true に設定していたのを、false に設定するように修正しました。

4-0-0W109: (SPY) Spy.exe で正しいフレームから再生が開始されない不具合の修正

アプリケーションとの通信の切断後、またはファイルを開いた後の最初の再生時に、予期しないフレームから再生が始まる不具合を修正しました。

また、カレントフレームが末尾にあるときに再生を行うとすぐに停止してしまう場合がある不具合を修正しました。

4-0-0W110: (SPY) モジュールを複数回登録すると正常に動作しなくなる不具合の修正

モジュールを SpyController に複数回登録すると正常に動作しなくなる不具合を修正しました。
