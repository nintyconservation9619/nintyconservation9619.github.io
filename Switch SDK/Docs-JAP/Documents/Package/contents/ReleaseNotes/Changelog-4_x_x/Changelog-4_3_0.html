NintendoSDK NX Addon 4.3.0 (based on Generic 4.3.0) の変更点

互換性の失われる仕様変更

★4-3-0S01: (IRSENSOR) プロセッサ起動中にモード遷移を伴わない Run 関数を発行した後に、しばらく過去のデータが取得されてしまう不具合を修正

これまで、プロセッサ起動中にモード遷移を伴わない Run 関数を実行した際に、内部で設定が反映されるまでの間、パラメータ設定前の過去データが取得されていました。
Run 関数を発行後に正しいデータが取得できるまでの間は nn::irsensor::ResultIrsensorNotReady のエラーを返すよう修正しました。
本変更により、NXAddon 4.0.0 ~ 4.2.0 の SDK で作成されたアプリとの互換がなくなります。
NXAddon 4.3.0 でリビルド対応をしてください。

★4-3-0S02: (VSI) Target Manager の互換性のない変更

  • nn::tm::TargetHandle, nn::tm::DiscoveryHandle の定義を変更しました
    • nn::tm::TargetHandle, nn::tm::DiscoveryHandle の定義を、相互に暗黙な変換が行われないよう変更しました。これにより、誤用がコンパイル時に検出されるようになります。
  • TMAPI.h のサポートが NX Addon 5.1 から打ち切られます
    • NX Addon 4.1 から、Target Manager の機能を使用できる TM ライブラリが正式対応されました。これに伴い、NX Addon 5.1 をめどに、今まで C:\Program Files\Nintendo\Oasis\include\TMAPI.h に定義されていた関数が使用できなくなります。今後は C:\Program Files\Nintendo\Oasis\include\tm.h をお使いください。
  • Target Manager の JIT デバッグ有効化オプションが、デフォルトでは無効となりました
    • JIT デバッグを使用する場合は、「Target Manager > ツール > オプション > JIT デバッガを有効にする」をチェックしてください。


機能追加

4-3-0S03: (DevMenu) SD カードのキャッシュストレージデータに対応

Save Data ページで SD カードに保存したキャッシュストレージデータが表示されるようになりました。

4-3-0S04: (FS) パッチの生成時間を改善する機能を追加

AuthoringTool でパッチ生成時間を高速化する、--cache-directory オプションを追加しました。
このオプションを指定することで、パッチ作成用のキャッシュを作成し、次回以降のパッチ生成時間が改善されるようになります。
オプションの詳細は パッチのビルド、実行方法 を参照してください。

4-3-0S05: (MOVIE) 色空間及び H264 動画のサポート対象範囲を返す機能の追加

色空間及び H264 動画のサポート対象範囲を返す機能を追加しました。

4-3-0S06: (OE) アプリケーションの起動パラメータ取得に関するサンプルを追加

アプリケーションの起動パラメータ取得に関するサンプルを追加しました。nn::oe::TryPopLaunchParameter() の使い方と、起動パラメータのフォーマットの定義方法に関する注意事項などを実装例と共に解説しています。

4-3-0S07: (OE) 動画撮影を有効化したアプリの動画をファイルに書き出す API を追加

動画撮影を有効化したアプリの動画をファイルに書き出す API として nn::oe::RequestToSaveRecordingForDebug() を追加しました。
この API を発行すると、キャプチャーボタンを長押ししたのと同じ効果があります。
ただし、この API は製品機では使用できませんのでご注意下さい。
開発時のデバッグ用途でのみご利用頂けます。

4-3-0S08: (TOOL) InitializeEdev で EDEV の名前を指定して初期化する機能を追加

InitializeEdev で EDEV の名前を指定して初期化する機能を追加しました。

4-3-0S09: (TOOL) 開発機更新ツールおよび NX 開発カードや microSD で開発機を更新できるツールの追加

開発機の更新において、以下のツールを追加しました。
1. SystemUpdateSdev(SystemUpdateSdev) : 本体更新機能と同等の手順で設定等を保持しながら更新するためのコマンドラインツール
2. DevKitUpdater*.nsp : 1 と同等の開発機の更新を NX 開発カードや microSD で更新するための開発機で動作するツール
ただし、現時点ではこの更新方法にはいくつかの制約がありますので、SDK ドキュメントを参照してからご利用ください。

4-3-0S10: (TOOL) 統合コマンドラインツール（ TargetShell ）に新機能を追加

統合コマンドラインツールに下記の機能を追加しました。

  • アプリケーション、パッチ、AoC のインストール/アンインストール
  • ミュート設定の切り替え
  • メニューの切り替え
  • ショップの切り替え
  • セーブデータの削除

詳しくはマニュアルをご確認ください。

4-3-0S11: (VSI) VSI, Target Manager の新機能

  • VSI
    • ROM 提出可能でない場合にビルドエラーとするオプションの追加
  • Target Manager
    • Windows の権限不足などにより、HostFs でファイル操作をしようとして失敗した場合、Target Manager に警告を出す機能の追加
    • 複数台に対する 電源オン・オフ, LED 点滅メニューの追加

Web アプレット / オフライン Web アプレットで Web Audio の有効 / 無効を設定する機能を追加しました。

4-3-0S14: (WEB) WebCallerCommand に Web アプレット・オフライン Web アプレットで WebAudio を有効にするコマンドを追加

4-3-0S16: (WEB) ページキャッシュの有効 / 無効を設定する機能を追加

4-3-0S17: (キーコード処理) キーボードのキーコードと対応する文字コードを取得する機能を追加

本体設定に基づいて USB HID Keyboard/Keypad Page Usage ID を文字コード (UTF-16) に変換する機能を追加しました。詳しくはこちらをご確認ください。

デッドキー処理やローマ字入力など、複数のキー入力から構成される文字入力を受け付けて文字コード列を組み立てる機能を追加しました。詳しくはこちらをご確認ください。

改善

4-3-0S19: (AUDIO) アプリケーションへの DSP リソースの割り当て量の増加

従来、アプリケーションへの DSP リソースの割り当て量は 70 % でしたが、これを 75 % に変更しました。

アプリケーションへの DSP リソースの割り当て量に関する詳細は、DSP のリソース制限 を参照してください。

4-3-0S20: (DevMenu) DevMenu を利用時にも動画撮影結果がオーバーレイ通知されるように変更

DevMenu 利用時にも動画撮影を行った際に動画の保存結果がオーバーレイ通知されるようになりました。通知は英文テキストのみです。サムネイル表示および SE の再生は行われないため、製品環境での動作を確認する際には HOME メニューに切り替えて確認してください。

4-3-0S21: (DIAG) LogViewer が、エスケープシーケンスによる前景色および背景色の 24 bit 指定に対応

LogViewer が、エスケープシーケンスによる前景色および背景色の 24 bit 指定に対応しました。

4-3-0S22: (FS) FsAccessLogChecker version 10.2 でアクセスログを時系列にソートしてチェックするよう改善

アプリケーションのスレッド構成によっては fs アクセスログが時系列に出力されないため、FsAccessLogChecker (%NINTENDO_SDK_ROOT%\Tools\FsAccessLogChecker\FsAccessLogChecker.exe) version 10.2 でアクセスログを時系列に並び直し、チェックするよう改善しました。

4-3-0S23: (FS) FsAccessLogChecker version 10.3 でエラーのアクセスログをチェックから除外するよう改善

アプリケーションにエラーが返ったアクセスログでは、ストレージへのアクセスは発生していないことが想定されるため、FsAccessLogChecker (%NINTENDO_SDK_ROOT%\Tools\FsAccessLogChecker\FsAccessLogChecker.exe) version 10.3 で result: 0x00000000 以外のアクセスログはチェックから除外するよう改善しました。

4-3-0S24: (FS) FsAccessLogChecker version 9.6 で PASS した読み出し頻度の値を確認できるよう対応

FsAccessLogChecker (%NINTENDO_SDK_ROOT%\Tools\FsAccessLogChecker\FsAccessLogChecker.exe) version 9.6 で、読み出し頻度に PASS した場合も SameAddressRead 項目に上位の値を出力するよう対応しました。これに伴い、出力の体裁を変更しました。

4-3-0S25: (FS) nn::bcat のアクセスログ出力追加と FsAccessLogChecker version 10.0 でのチェック対応

fs アクセスに関わる nn::bcat API でアクセスログ出力を追加し、FsAccessLogChecker (%NINTENDO_SDK_ROOT%\Tools\FsAccessLogChecker\FsAccessLogChecker.exe) version 10.0 にて nn::bcat 経由の fs アクセスをチェックするよう対応しました。詳細は FSライブラリ／デバッグ機能／アクセス頻度確認 を参照してください。

4-3-0S26: (General) 動画撮影を有効化したアプリケーションに対する性能への影響の安定化

NX Addon 4.2.0 で導入した動画撮影機能を有効化すると、アプリケーションは CPU や GPU 処理時間の面で不定期に影響を受けていましたが、この不定期性を緩和し、負荷を安定化しました。
詳細は こちら を参照してください。

4-3-0S27: (TOOL) AuthoringTool でソフトリーガル情報の改変を検知するように改善

AuthoringTool 実行時に Software Legal Information System（SLIM）から取得したソフトリーガル情報の改変を検知した場合にエラーにするようにしました。

4-3-0S28: (TOOL) AuthoringTool の --legal-information 、 --html-document 、 --accessible-urls 、 --icon 、 --nx-icon オプションを廃止

AuthoringTool の下記のオプションを廃止しました。

  • --legal-information
  • --html-document
  • --accessible-urls
  • --icon
  • --nx-icon


今後は nmeta ファイルの下記の要素を使用してください。

  • LegalInformationFilePath
  • HtmlDocumentPath
  • AccessibleUrlsFilePath
  • IconPath
  • NxIconPath


詳細は 機能＞ビルド＞アプリケーションイメージの作成＞nmeta ファイルの書き方 を参照してください。

4-3-0S29: (TOOL) nmeta ファイルのロゴデータの設定値から DistributedByNintendo を削除

nmeta ファイルのロゴデータの設定値から DistributedByNintendo を削除しました。
ロゴデータに設定できる値については 機能＞ビルド＞アプリケーションイメージの作成＞nmeta ファイルの書き方 を参照してください。

4-3-0S30: (VSI) VSI, Target Manager の改善

    • Target Manager で nsp のパスを指定して実行する場合、" でくくられていても正しく解釈するよう改善しました。


4-3-0S31: (VSI) インストール時にエラーのように見えるメッセージを抑制

Visual Studio 2013 や 2015 がインストールされていない PC に対して、 VSI for NX パッケージをインストールすると、赤い文字でエラーのように見えるメッセージが表示されていましたが、これを表示しないようにしました。
Visual Studio 2013 / 2015 / 2017 のいずれかがインストールされていれば NX Addon 4.2.0 以前も NX Addon 4.3.0 以降も、VSI for NX をインストールし利用することが可能です。

2 スレッド以上かつ、一度の NN_LOG, NN_VLOG 呼び出しで、 761 文字以上のログを出力すると、Log Viewer が応答しなくなる不具合を修正しました。

4-3-0S36: (Display) 液晶の輝度の最適化
