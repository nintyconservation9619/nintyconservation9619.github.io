NintendoSDK NX Addon 4.4.0 (based on Generic 4.4.0) の変更点

機能追加

★4-4-0S01: (DevMenu) 開発機の初期化時に DevMenuApp もインストールされるように変更

今まで、別途インストールが必要だった DevMenuApp が初期化時にインストールされるようになりました。
この変更は ProdMode でも有効になっており、アプリケーションのインストールを容易に行えます。

★4-4-0S02: (OE) クラッシュレポート作成機能の有効・無効を一時的に変更

クラッシュレポート作成機能の有効・無効を一時的に切り替える API を追加しました。クラッシュレポートを作成してはいけないシーンでは無効化するようにしてください。詳細はドキュメント、及びガイドラインを参照してください。

★4-4-0S03: (TOOL) 市場で発生したクラッシュ情報の取得機能を追加

アプリケーションが市場でクラッシュした際の情報を取得できるようになります。詳細は SDK ドキュメントの機能／システム／クラッシュレポートを参照してください。

4-4-0S04: (DevMenu) TV アンダースキャン値の設定機能を追加

DevMenu および DevMenuCommand に TV アンダースキャン値の設定機能を追加しました。DevMenu は Device ページ、DevMenuCommand は settings コマンドで利用可能です。

4-4-0S05: (MOVIE) MovieDecoderPlayer 及び MoviePlayerSimple サンプルで、デフォルト描画処理のネイティブテクスチャモードへの変更

MovieDecoderPlayer 及び MoviePlayerSimple サンプルで、デフォルトの描画処理をネイティブテクスチャモードに変更しました。

4-4-0S06: (MOVIE) MovieDecoderPlayer 及び MoviePlayerSimple サンプルの対応最低サンプリングレートの変更

MovieDecoderPlayer 及び MoviePlayerSimple サンプルが対応する最低サンプリングレートを 16KHz に変更しました。16KHz 以下の周波数は再生されません。

4-4-0S07: (MOVIE) MoviePlayerSimple サンプルに複数の言語トラック対応の追加

MoviePlayerSimple サンプルを複数の言語トラックに対応するように更新しました。サンプルは、本体の言語設定に合致したオーディオを再生します。

4-4-0S08: (TOOL) AuthoringEditor でクラッシュレポートに関する設定項目を追加

AuthoringEditor 上で次のクラッシュレポートの nmeta 項目 (CrashReport) の編集が行えるようになりました。
クラッシュレポートについての詳細は NintendoSDK ドキュメント＞機能＞システム＞クラッシュレポートを参考にしてください。

4-4-0S09: (TOOL) AuthoringEditor にセーブデータ関連項目を追加

AuthoringEditor 上で次のセーブデータに関連する nmeta 項目の編集が行えるようになりました。

  • UserAccountSaveDataSizeMax
    • ユーザーアカウントセーブデータのデータ保存領域の拡張上限
  • UserAccountSaveDataJournalSizeMax
    • ユーザーアカウントセーブデータのジャーナリング領域の拡張上限
  • TemporaryStorageSize
    • 一時ストレージサイズ
  • CacheStorageSize
    • キャッシュストレージのデータ保存領域サイズ
  • CacheStorageJournalSize
    • キャッシュストレージのジャーナリング領域サイズ


4-4-0S10: (TOOL) AuthoringEditor 上で nmeta または nsp に対して別の nmeta ファイルをインポートする機能を追加

AuthoringEditor 上で、編集中のアプリケーション nmeta もしくは nsp ファイルに対して、別の nmtea ファイルをインポート（上書き設定）する機能を追加しました。

4-4-0S11: (キーコード処理) 組み立て中文字コード列から未確定部分を削除する機能を追加

組み立て中文字コード列から未確定部分を削除する機能として
nn::kpr::KeyCodeComposer::Remove() メソッドを追加しました。詳しくはマニュアルをご確認ください。

★4-4-0S12: (NVN) NVIDIA Graphics Driver におけるサポート GPU の追加

★4-4-0S13: (TOOL) InitializeEdev で EDEV の初期化が失敗する問題の改善

InitializeEdev で EDEV の初期化が失敗する問題の改善を行いました。
InitializeEdev 実行中に EDEV の電源が OFF になってしまったり、EDEV と ホスト PC の USB 通信が切断されてしまったりする場合に、InitializeEdev を再実行することで初期化が行えるように対策を行いました。
なお、この変更は EDEV を NX Addon 4.4.0 以降の InitializeEdev で初期化した状態で、更に InitializeEdev で初期化を行う場合に有効になります。
NX Addon 4.4.0 より前の InitializeEdev で初期化した状態で、NX Addon 4.4.0以降 の InitializeEdev を実行する際には効果がありません。

4-4-0S14: (DevMenu) セーブデータ一覧表示までの時間を改善

サイズの大きなセーブデータが存在する場合に、セーブデータ一覧が表示されるまでの時間を短縮しました。

4-4-0S15: (DevMenu) メモリ情報の修正が必要な SDEV にはそのことを表示するよう変更

メモリ情報の修正が必要な SDEV では、DevMenu の Debug ページに「Needs Memory Repair」と表示するよう変更しました。
詳細はDevMenu／Memory Mode／メモリ情報の修正が必要な SDEV の場合を参照してください。

4-4-0S16: (FS) FsAccessLogChecker version 10.7 で読み込み頻度の領域を確認できるよう対応

FsAccessLogChecker (%NINTENDO_SDK_ROOT%\Tools\FsAccessLogChecker\FsAccessLogChecker.exe) version 10.7 で、SameAddressRead 項目に読み込み領域を出力するよう対応しました。

4-4-0S17: (OS) デフォルトユーザ例外ハンドラで表示されるメモリ情報を改善

デフォルトユーザ例外ハンドラで表示されるメモリ情報に以下の 2 つを追加しました。

  • Uncached を示す 'U'（nv メモリプールなど）
  • nn::os::IsMemoryLocked() == true であることを示す 'L'


4-4-0S18: (TOOL) AuthoringEditor の PEGIPortugal の年齢区分を修正

AuthoringEditor のレーティング情報入力フォーム内の PEGIPortugal について、年齢区分から "4歳以上" を削除しました。 今後は年齢区分として "3歳以上" をご利用ください。

4-4-0S19: (TOOL) AuthoringEditor 上の「LogoType」から「DistributedByNintendo」を削除

AuthoringEditor 上でアプリケーション nmeta を編集する際に設定できる「ロゴデータの設定」について、「DistributedByNintendo」を削除しました。

4-4-0S20: (TOOL) DevMenuCommand でのメモリモード取得

DevMenuCommand にメモリモードを取得する debug get-memory-mode オプションを追加しました。

4-4-0S21: (TOOL) nspd に含めるリソースデータファイルのフィルタリング機能の追加

nsp 作成時にリソースデータファイルをフィルタアウトする機能と同様のことを、nspd 作成時にも行えるようにしました。

4-4-0S22: (TOOL) 開発機本体更新ツールのモード違いファームからの実行を不可にする対応

開発機本体更新ツール (SystemUpdaterSdev) のモード違いファームからの実行をできないようにしました。
モードが違っている場合は初期化ツールを使って該当するモードに初期化するようにして下さい。

4-4-0S23: (TOOL) 統合コマンドラインツール（ TargetShell ）のオプションを一部変更及び機能追加

統合コマンドラインツールのオプションの指定方法を一部変更しました。

  • ターゲット指定ありオプションの指定方法を一部変更しました。
  • 設定ファイルの Export、Import、Reset を行う機能の追加をしました。

詳しくはマニュアルをご確認ください。

★4-4-0S24: (DevMenu) PROD MODE で DevMenuApp を起動できない不具合を修正

★4-4-0S25: (TOOL) AuthoringEditor 上で nsp 編集中に nmeta インポート機能が使用できない不具合の修正

アプリからライブラリアプレット起動時にアプリの音が停止しないことがある不具合を修正しました。

4-4-0S27: (AUDIO) nn::audio::GetActiveAudioDeviceChannelCountForOutput() の不具合修正

nn::audio::GetActiveAudioDeviceChannelCountForOutput() で正しくチャンネルするが取得されない不具合がありました。これを修正しました。
具体的には、サラウンド出力中にも関わらず返り値として 2 が返ってくる、という症状が発生していました。
ただし、その際、発音自体は正しくサラウンド出力されます。

この不具合は Nintendo SDK 4.1 から 4.3 のリリースで発生します。

4-4-0S28: (AUDIO) 設定したデバイスボリュームがオーディオレンダラに適用されない不具合の修正

設定したデバイスボリュームがオーディオレンダラに適用されないことがある不具合があり、これを修正しました。
具体的には、nn::audio::StartAudioRenderer() を呼び出してから約 5 ms 以上経過するより前にデバイスボリュームを設定した場合、
オーディオレンダラから出力される音声に正しくデバイスボリュームが適用されないことがありました。

  • DevMenu から起動前ユーザー選択を伴うアプリケーションを起動

4-4-0W02: (レイアウトツール) LayoutEditor 上でフォントのサイズスケールを維持したままフォントを差し替える機能の追加
