NintendoSDK NX Addon 4.5.0 (based on Generic 4.5.0) の変更点

機能追加

4-5-0S01: (DevMenu) DevMenu に音声出力先・チャンネル数を表示する機能を追加

DevMenu に音声出力先・チャンネル数を表示する機能を追加しました。

4-5-0S02: (DevMenu) ソフトウェアキーボードでのユーザーネーム入力機能を追加

ソフトウェアキーボードを使用してユーザーネームを自由に入力できるようにしました。

4-5-0S03: (Sample) 対応言語取得サンプルの追加

Samples/Sources/Applications/OeGetLanguage にプログラムから対応言語を取得するサンプルを追加しました。

改善

4-5-0S04: (DevMenu) DevMenu 利用時の起動パラメータのサイズ上限を 16 KiB に拡張

DevMenu 利用時の起動パラメータのサイズ上限を、4 KiB から 16 KiB に拡張しました。

起動パラメータについては、「 機能／基本機能／OE ライブラリ／マニュアル／機能説明／起動パラメータ取得」を参照してください。

4-5-0S05: (TOOL) AlbumDownloader に関して利便性の観点から一部機能を改善

AlbumDownloader に関して利便性の観点から以下の機能改善を行いました。

  • AlbumDownloader をアプリの裏で同時動作できるように改善（RunOnTarget.exe で起動するときに --suppress-auto-kill オプションが必要）
  • ダウンロード中にターゲット側で動画ファイルなどの保存中のファイルがあった場合、ファイルの保存が終了するまで自動的に待機するように改善
  • 上記待機が 60 秒経っても終了しない場合、そこでダウンロードを諦めて終了し、ダウンロード失敗のログを出力するように改善


4-5-0S06: (TOOL) AuthoringTool にパッチ間差分情報の表示機能を追加

AuthoringTool に、パッチ間のファイル単位の差分情報を表示する diffpatch コマンドを追加しました。

4-5-0S07: (TOOL) AuthoringTool のパッチ生成処理時間の改善

AuthoringTool が行っているデータ変換処理を最適化し、特にパッチ生成における処理時間を改善しました。

4-5-0S08: (TOOL) パッチ作成時のセーブデータ・一時ストレージ・キャッシュストレージのサイズ更新内容のチェックを改善

パッチによるセーブデータ・一時ストレージ・キャッシュストレージのサイズ更新を行う場合に、一部のケースでパッチ作成時に不正な値の入力や更新ができてしまっていたため、チェックの内容を改善しました。

4-5-0S09: (TOOL) 追加コンテンツ作成時にタグ名の文字コードをチェックする機能の追加

AuthoringTool で追加コンテンツを作成する時に、 nmeta ファイルの Tag に 指定できない文字が設定されている場合にエラーを発生させるようにしました。
詳細は 機能／システム／AOC ライブラリ／追加コンテンツの作成方法 を参照してください。

4-5-0S10: (TOOL) 追加コンテンツ作成時にタグ名の文字列長をチェックする機能の追加

AuthoringTool で追加コンテンツを作成する時に、 nmeta ファイルの Tag に 128 文字を超える文字列を指定するとエラーを発生させるようにしました。
詳細は 機能／システム／AOC ライブラリ／追加コンテンツの作成方法 を参照してください。

不具合修正

4-5-0S11: (DevMenu) インストール対象の nsp ファイルのチェック機能の改善

DevMenuCommand で指定したインストール対象の nsp ファイルが適切であるかのチェックを強化しました。
絶対パスでない場合や nsp ファイルを指定していない場合などに適切なログが表示されるようになりました。

4-5-0S12: (DevMenu) メモリ情報の修正が必要な SDEV の表示不具合を修正

メモリ情報を修正済みの SDEV にもかかわらず、InitializeSdev で初期化直後にのみ Debug ページに「Needs Memory Repair」が表示される不具合を修正しました。
本不具合の修正に合わせて、「Needs Memory Repair」が表示されるのが DevMenuApp のみに変更しました。DevMenu では表示されません。

4-5-0S13: (DevMenu) 多数の追加コンテンツがインストール済みだと Fatal エラーが発生する不具合を修正

多数の追加コンテンツがインストール済みだと DevMenu 内のコンテンツ管理画面を Manage ボタンから開こうとすると Fatal エラーが発生する不具合を修正しました。
また、本修正に合わせて多数の追加コンテンツを含む nsp のインストール時間を改善しました。

4-5-0S14: (FS) FsAccessLogChecker を version 12.3 に更新

FsAccessLogChecker (%NINTENDO_SDK_ROOT%\Tools\FsAccessLogChecker\FsAccessLogChecker.exe) を version 12.3 に更新しました。以下の変更が含まれます。

  • 解析結果の "FS Access Log Details" が入る位置を修正
  • 改行コード CR に対応
  • マルチバイト文字を含む Shift-JIS に対応
  • インデックス付きキャッシュストレージ機能とnn::fs::GetSaveDataSize() のチェックに対応
  • SameAddressRead 項目のファイルに関連しない読み込み操作の出力形式を変更
  • オプション --warning_ratio を追加し、SameAddressRead 項目に閾値を超えていない上位の値を常時出力するよう対応


4-5-0S15: (FS) SD カード上のキャッシュストレージの拡張時の不具合修正

SD カード上のキャッシュストレージがパッチで拡張される際に SD カードが容量不足で拡張に失敗するとキャッシュストレージを開けなくなる不具合を修正しました。

4-5-0S16: (IRSENSOR) ポインティングモードの出力値が正しく正規化されていない不具合を修正

これまで、nn::irsensor::GetPointingProcessorStates 関数で取得される、 nn::irsensor::PointingProcessorState の position データとして、画面左上を (0, 0)、画面右下を (320.0, 240.0) とした正規化されていないデータを返してしまっていましたので、これを、画面中心を (0, 0)、　画面右上を (1.0, 1.0) 、アスペクト比を4:3 とした、正規化された値を返すように修正しました。

4-5-0S17: (MOVIE) MatroskaおよびWebMファイルの再生時にメモリ使用量が徐々に増える問題の修正

MatroskaおよびWebMファイルの再生時にメモリ使用量が徐々に増える問題を修正しました。

4-5-0S18: (MOVIE) 動画のない AAC オーディオのみの Matroska ファイルが再生できない不具合の修正

動画のない AAC オーディオのみの Matroska ファイルが再生可能になりました。

4-5-0S19: (TOOL) AuthoringTool でフィルタ記述ファイル指定時に createnspd が失敗する場合がある不具合を修正

AuthoringTool のサブコマンド createnspd で、-o オプションで指定した出力ディレクトリ以下にあるファイルやディレクトリを --program オプション等に指定し、かつ、フィルタ記述ファイルを指定していた場合に、.nspd ディレクトリ以下に program0.ncd/data ディレクトリが既に存在すると、nspd の作成に失敗する不具合があり、これを修正しました。

4-5-0S20: (TOOL) AuthoringTool の list コマンドで削除ファイルが表示されない不具合の修正

AuthoringTool の list コマンドの --patched-only オプションでパッチに含まれるファイルのみを表示する際に、パッチで削除されたファイルが "Listing removed files from original" として表示されない場合が有る不具合を修正しました。

4-5-0S21: (TOOL) AuthoringTool の replace コマンドでファイルサイズが変わるような編集を行った場合にファイル読み込みが遅くなる場合がある不具合の修正

AuthoringTool の replace コマンドでアプリに含まれるリソースデータ及び Html ドキュメントのファイルをファイル単位で編集し、
ファイルサイズが元のサイズから変更された場合、アプリ及びそのアプリを元にして作成しされたパッチにおいて、
リソースデータ及び Html ドキュメントのファイルの読み込みが本来の速度より遅くなる不具合があり、これを修正しました。

またこの修正により、replace コマンドを使用した場合にパッチサイズが意図せず大きくなる事もなくなりました。

4-5-0S22: (TOOL) ControlTarget 電源制御関連コマンドの挙動が変わっていた問題の修正

ControlTarget power-off, power-on, reset の挙動が、NX Addon 4.3 で変わってしまっていた問題を修正しました。

4-5-0S23: (TOOL) Nintendo NX CPU Profiler で、サンプリングできない期間がある問題を回避

Nintendo NX CPU Profiler で、サンプリングできない期間がある問題を回避するため、サンプリング方法を旧方式に戻しました。旧方式では、複数コアにアフィニティが設定されているスレッドは観測できないため、プロファイラ利用中は単一コアのみにアフィニティを設定して利用してください。

4-5-0S24: (TOOL) SystemUpdateEdev で存在しないシリアルナンバーを指定してもエラーログが出力されない不具合の修正

EDEV 更新ツール (SystemUpdateEdev) で存在しないシリアルナンバーを指定してもエラーログが出力されていませんでしたが、出力するようにしました。

4-5-0S25: (TOOL) Visual Studio をアンインストールして Help Viewer が残っている場合に Document パッケージのインストールが失敗する不具合の修正

Microsoft Visual Studio をアンインストールして Microsoft Help Viewer が残っている状態で Document パッケージをインストールしようとすると失敗する不具合を修正しました。

NintendoWare NX Addon 4.5.0 (based on Generic 4.5.0) の変更点

機能追加

4-5-0W01: (レイアウト-UI2D) LOD Bias が設定できる機能の追加

nn::ui2d::GraphicsResource::Setup で LOD Bias が設定できる機能を追加しました。

詳しい使い方はドキュメントの「拡大縮小時にテクスチャが滲まないようにするには」のページをご覧ください。

不具合修正

4-5-0W02: (レイアウトツール) LayouEditor のカーブエディタのテキストボックスでショートカットが効かない不具合を修正

LayouEditor のカーブエディタのテキストボックスでショートカットが効かない不具合があり、これを修正しました。

4-5-0W03: (レイアウトツール) LayoutEditor で、「色パラメータをリニア色空間で指定する」のチェックを外した際、詳細コンバイナーのコンスタントカラーが編集画面で正しく表示されない不具合を修正

LayoutEditor で、「色パラメータをリニア色空間で指定する」のチェックを外した際、詳細コンバイナーのコンスタントカラーが編集画面で正しく表示されない（ビューアーやゲームプログラムにおいての表示と食い違う）不具合があり、これを修正しました。

4-5-0W04: (レイアウトツール) LayoutEditor でスケーラブルフォントを使用したテキストボックスペインが表示されない不具合を修正

LayoutEditor でスケーラブルフォントを使用したテキストボックスペインが表示されない不具合を修正しました。

4-5-0W05: (レイアウトツール) LayoutEditor でフォントの読み込みに失敗した状態で関連部品ボタンを押したときに応答が無くなる不具合を修正

LayoutEditor でフォントの読み込みに失敗した状態で、レイアウトウインドウの関連部品を表示するボタンを押したときに応答が無くなる不具合があり、これを修正しました。

4-5-0W06: (レイアウトツール) LayoutEditor に引数を指定してレイアウトファイルを開いたときにエラーが発生する不具合を修正

LayoutEditor でレイアウトウインドウを最小化した状態で、レイアウトファイルを閉じた後に、そのレイアウトファイルを引数に指定して、LayoutEditor を起動したときに D3D デバイスの作成に失敗する不具合があり、これを修正しました。

4-5-0W07: (レイアウトツール) LayoutEditor のフォルダ以下に対する一括処理時に部品が正しく読み込まれない不具合を修正

LayoutEditor の「部品ルート以下のレイアウトファイルを再出力」、「部品フォルダ以下のレイアウトファイルを一括同期」、「指定フォルダ以下のサムネール画像を更新」を実行したときに、参照する部品が正しく読み込まれない不具合があり、これを修正しました。
