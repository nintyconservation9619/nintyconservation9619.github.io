NintendoSDK NX Addon 3.5.2 (based on Generic 3.5.2) の変更点

機能追加

3-5-2S01: (TOOL) AuthoringEditor 上で nmeta または nsp に対して別の nmeta ファイルをインポートする機能を追加

AuthoringEditor 上で、編集中のアプリケーション nmeta もしくは nsp ファイルに対して、別の nmtea ファイルをインポート（上書き設定）する機能を追加しました。

改善

3-5-2S02: (TOOL) AuthoringTool でソフトリーガル情報の改変を検知するように改善

AuthoringTool 実行時に Software Legal Information System（SLIM）から取得したソフトリーガル情報の改変を検知した場合にエラーにするようにしました。

3-5-2S03: (TOOL) AuthoringTool の createnspd コマンドに --error-unpublishable オプションを追加

AuthoringTool の createnspd コマンドに --error-unpublishable オプションを追加しました。
詳細は 「機能／ビルド／アプリケーションイメージの作成／ツールリファレンス／AuthoringTool」を参照してください。

3-5-2S04: (TOOL) AuthoringTool の廃止予定オプション指定時に警告を表示するように変更

AuthoringTool 実行時に、今後廃止予定の以下のオプションを指定した時に警告を表示するようにしました。

  • --accessible-urls
  • --html-document
  • --legal-information
  • --icon
  • --nx-icon


詳細は 機能／ビルド／アプリケーションイメージの作成／ツールリファレンス／AuthoringTool を参照してください。

3-5-2S05: (TOOL) nmeta ファイルのロゴデータの設定値から DistributedByNintendo を削除

nmeta ファイルのロゴデータの設定値から DistributedByNintendo を削除しました。
ロゴデータに設定できる値については 機能＞ビルド＞アプリケーションイメージの作成＞nmeta ファイルの書き方 を参照してください。

不具合修正

★3-5-2S06: (IRSENSOR) モーメントモード、クラスタリングモード使用時に、システムによって確率的にアプリが指定したバッファ外の領域が不正に書かれる不具合の修正

これまで、Get 関数でデータを取得する際に、アプリが指定したバッファの1構造体分となりの領域を不正に書き換えてしまう可能性がありましたので、これを修正しました。
例えば、通常 nn::irsensor::GetMomentProcessorState 関数に nn::irsensor::MomentProcessorState 構造体を1つ分アプリ側で確保して関数に渡しますが、その構造体領域の1つ隣の領域、合わせて構造体2つ分の領域をシステム側が確率的に書き換えて破壊してしまうリスクがありました。
この修正により、今後、構造体1つ分を余分に確保するというワークアラウンドは必要なくなります。

3-5-2S07: (FS) FsAccessLogChecker を version 10.4 に更新

FsAccessLogChecker (%NINTENDO_SDK_ROOT%\Tools\FsAccessLogChecker\FsAccessLogChecker.exe) を version 10.4 に更新しました。以下の変更が含まれます。

  • チェック時間間隔の境界にあるアクセスログ、および1分以上間の空いたアクセスログがあった場合の集計の不具合を修正
  • 1分以上間の空いたアクセスログがあったときの次のチェック時間範囲がずれる不具合を修正
  • 同一時間に複数のアクセスログがあった場合に一部のログを見落とす不具合を修正
  • リードアクセスの重なり判定を修正
  • オプション --time_range 指定で小数点の値を指定すると例外が発生する不具合を修正
  • PASS した読み出し頻度の値を確認できるよう対応
  • nn::bcat 経由の fs アクセスをチェックするよう対応
  • アクセスログを時系列にソートしてチェックするよう改善
  • エラーのアクセスログをチェックから除外するよう改善


3-5-2S08: (VSI) NintendoSDK のバージョンダウン時に VSI が不正な状態になる不具合の修正

NX Addon 4.3.0 以降に対応した VSI 4.4.0 以降から、それよりも前のバージョンの NintendoSDK に対応した VSI をインストールした場合に、VSI が正常動作しなくなる不具合を修正しました。
