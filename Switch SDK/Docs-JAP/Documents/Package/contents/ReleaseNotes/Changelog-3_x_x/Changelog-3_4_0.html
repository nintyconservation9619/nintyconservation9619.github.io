NintendoSDK NX Addon 3.4.0 (based on Generic 3.4.0) の変更点

互換性の失われる仕様変更

★3-4-0S01: (BCAT) Result 定義の修正

以下の Result 定義を修正しました。

  • nn::bcat::ResultTooManyHandle → nn::bcat::ResultTooManyHandles
  • nn::bcat::ResultTooManySyncRequest → nn::bcat::ResultTooManySyncRequests


★3-4-0S02: (FS) デバッグ用セーブデータをマウントする nn::fs::MountSaveData(const char* name) の削除

デバッグ用セーブデータをマウントする nn::fs::MountSaveData(const char* name) を削除しました。
デバッグ用セーブデータをマウントする場合は nn::fs::MountSaveDataForDebug() を利用してください。

★3-4-0S03: (General) InitializeSdev で初期化できる対象機材から SDEV1.8 と preMP を削除

InitializeSdev で SDEV1.8 と preMP を初期化できなくなります。
UI 上では初期化する対象の機材として選択することができてしまいますが、実際に初期化をすることはできませんのでご注意ください。

機能追加

3-4-0S04: (APPLET) DevMenuCommand にショップの切り替え機能追加

DevMenuCmmand で開発用にショップと DummyShop を切り替えることができる機能を追加しました。

3-4-0S05: (DevMenu) DevMenuCommand に TV 設定、オートスリープ設定系のコマンドを追加

DevMenuCommand に以下の TV 設定、オートスリープ設定系のコマンドを追加しました。

  • テレビ解像度の設定
  • RGB レンジの設定
  • テレビ画面焼け軽減機能の設定
  • CEC 機能の設定
  • オートスリープ設定


詳細は DevMenuCommand のマニュアル を参照してください。

3-4-0S06: (DevMenu) キャッシュストレージデータのバックアップ・リストア機能を追加

DevMenu にキャッシュストレージデータのバックアップ・リストア機能を追加しました。

改善

★3-4-0S07: (AUDIO) SystemEvent を扱う audio 機能についての注意事項のドキュメントへの追記

SystemEvent を扱う audio 機能に関する注意事項を以下の API について追記しました。アプリの動作停止に繋がりうるものなので、必ずご確認ください。

* nn::audio::OpenAudioIn()
* nn::audio::OpenDefaultAudioIn()
* nn::audio::OpenAudioOut()
* nn::audio::OpenDefaultAudioOut()
* nn::audio::OpenAudioRenderer()

★3-4-0S08: (Graphics) GPU クラッシュ発生時に記録されるログの内容について解説するドキュメントの追加

GPU クラッシュ発生時に記録されるログの内容について解説するドキュメントを追加しました。
詳細は NVNライブラリ／マニュアル以下の「GPU エラーガイド」を参照してください。

★3-4-0S09: (TOOL) nmeta ファイルの設定項目から UserAccountSaveDataOperation を削除

nmeta ファイルの設定項目から UserAccountSaveDataOperation を削除しました。
nmeta ファイルに UserAccountSaveDataOperation タグを記載している場合、 AuthoringTool によるアプリケーションイメージの作成が失敗します。

★3-4-0S10: (TOOL) ゲームカード容量内のアプリケーションが使える容量の増加

ゲームカード容量の内、アプリケーションへの割当てが保証された領域を超える場合の許容量を 128 MB から 144 MB に変更しました。

★3-4-0S11: (WEB) ウェブページを表示した結果の情報のうち最後に開いていたページの URL を取得する API の前提条件を変更

nn::web::WebPageReturnValue::GetLastUrl() の前提条件としてnn::web::WebPageReturnValue::GetWebExitReason() で取得した値が 、nn::web::WebExitReason::WebExitReason_CallbackUrlReached であること。を追加しました。

nn::web::OfflineHtmlPageReturnValue::GetLastUrl() の前提条件としてnn::web::OfflineHtmlPageReturnValue::GetOfflineExitReason() で取得した値が、nn::web::OfflineExitReason::OfflineExitReason_CallbackUrlReached であること。を追加しました。

3-4-0S12: (DevMenu) インストール中に自動でスリープに入る問題の改善

インストールに時間がかかった場合、自動スリープが有効だとスリープに入ってしまっていましたが、スリープに入らないよう改善しました。

3-4-0S13: (FS) AuthoringTool のリソースデータフィルタアウト機能にフィルタリング対象エントリ指定ルールを追加

フィルタ記述ファイルにフィルタリング対象エントリ指定ルールを記述することで、エントリの大半を除外する場合においてより効率よくフィルタリング処理を実行できるようになりました。
詳しい利用方法は、オーサリングツールのドキュメント を参照してください。

3-4-0S14: (TOOL) AuthoringEditor で読み込んだ nmeta を保存する際、含まれる XML 要素を可能な限り保持するように修正

AuthoringEditor で nmeta ファイルを読み込み保存した際、nmeta ファイルにもともと含まれていた XML 要素を可能な限り保持するように修正しました。

3-4-0S15: (TOOL) AuthoringTool でアプリケーションイメージを作成する際のチェック項目を追加

AuthoringTool でアプリケーションイメージを作成する際に、下記の条件でエラーが発生するようになりました。

  • nmeta ファイル内の Title と Icon の言語設定に不整合がある
  • nmeta ファイル内の BcatDeliveryCacheStorageSize と BcatPassphrase のどちらか一方しか設定されていない


3-4-0S16: (TOOL) nmetaのタッチスクリーン使用設定の削除

nmetaで指定したタッチスクリーン使用設定を削除しました。
こちらの設定を本体機能上で使わない方針になり、
そのためアプリのnmetaでの設定は不要になりました。

3-4-0S17: (WEB) Web Api サンプルを追加

Web Api を利用したサンプルプログラムを追加しました。

不具合修正

★3-4-0S18: (MOVIE) MovieDecoderPlayer サンプルがフルレンジ RGB に対応

MovieDecoderPlayer サンプルで、フルレンジ YCbCr がフルレンジ RGB に変換するようにシェーダーを更新しました。

★3-4-0S19: (TOOL) nsp 作成時に --filter オプションが反映されない不具合の修正

nsp 作成時に --filter オプションを指定してもフィルタリングが一切行われない不具合があり、これを修正しました。

3-4-0S20: (AUDIO) nn::audio::Resampler 関連のヘッダ不備の修正

nn::audio::Resampler 機能に関連するヘッダファイルがパッケージに含まれていない不具合を修正しました。

3-4-0S21: (AUDIO) オーディオレンダラのスタート時に音声にノイズが混じることがあることがある不具合の修正

オーディオレンダラのスタート時に音声にノイズ(5 ms ～ 40 ms 程度) が再生が混じることがあることがある不具合があり、これを修正しました。

3-4-0S22: (DIAG) CapstoneSimple の不具合修正

CapstoneSimple で使用している cs_disasm_iter() の使用方法に間違いがありましたので、これを修正しました。

3-4-0S23: (DIAG) 開発機をホスト PC に接続していないと SD カードログが記録されない不具合を修正

NX Addon 3.0.0 以降のファームウェアを使用した際に、開発機をホスト PC に接続していないと SD カードログが記録されない不具合を修正しました。

3-4-0S24: (MEM) nn::mem::StandardAllocator にて仮想メモリ管理機能を利用した際の GetTotalFreeSize() と GetAllocatableSize() が物理アドレスがマッピングできないサイズを返すことがある不具合を修正

nn::mem::StandardAllocator にて仮想メモリ管理機能を利用した際に GetTotalFreeSize() と GetAllocatableSize() は仮想アドレス空間に対するサイズを返していましたが、物理アドレスがマッピングできる（物理アドレスとして空き領域の）サイズを返すように修正しました。

3-4-0S25: (MOVIE) MovieDecoderPlayer サンプルが ASSERT を抑制したビルドで動作しなかった不具合の修正

MovieDecoderPlayer サンプルが ASSERT を抑制したビルドで動作しなかった不具合を修正しました。

3-4-0S26: (MOVIE) MovieDecoderPlayer サンプルから Type.h の削除

MovieDecoderPlayer サンプルから、未使用ファイル Type.h を削除しました。

3-4-0S27: (MOVIE) MovieDecoderPlayer サンプルで最終フレーム後すぐに終了しない不具合の修正

MovieDecoderPlayer サンプルで、再生終了後、動画の最終フレームが少しの間表示され続けることがなくなりました。

3-4-0S28: (SWKBD) swkbd の文字列設定関数のうち、 UTF-8 版のみ最大文字数で指定をすると FATAL エラーが発生する不具合の修正

nn::swkbd::SetOkTextUtf8、 nn::swkbd::SetHeaderTextUtf8、 nn::swkbd::SetSubTextUtf8、 nn::swkbd::SetGuideTextUtf8、 nn::swkbd::SetInitialTextUtf8 といった UTF-8 版の文字列設定 API において、最大文字列ちょうどで指定すると FATAL が発生する不具合がありましたので、これを修正しました。

3-4-0S29: (TOOL) AuthoringEditor のパッチウインドウ内「直前のパッチ (nsp)」にパスを与えると、常にパッチ前後関係のチェックが走る不具合の修正

AuthoringEditor のパッチ作成ウインドウ内にある「直前のパッチ (nsp)」のパス入力欄にパッチ nsp のパスが与えられていると、「直前のパッチを使用する」チェックボックスを外していても、常にパッチ前後関係のチェックが走り、警告メッセージが表示される不具合を修正しました。
この不具合は警告メッセージの表示に関するもので、パッチの作成処理自体に影響はありません。

3-4-0S30: (TOOL) AuthoringEditor のレーティング入力フォームにおける PEGIPortugal レーティング区分の不足を修正

AuthoringEditor のレーティング入力フォーム内の PEGIPortugal レーティング区分について不足があったため追加しました。
追加したのは PEGIPortugal レーティング区分の 年齢値 3 および 7 の項目です。

3-4-0S31: (TOOL) NDICmd で NintendoSDK Documents for NX パッケージをインストールできない不具合の修正

NDICmd で NintendoSDK Documents for NX パッケージをインストールしようとすると、終了コード 0 を返し、正常にインストール処理が終了しているように見えるにもかかわらず、インストールできていない (＝ NDICmd.exe list environments で NintendoSDK Documents for NX パッケージが列挙されない) 不具合がありましたが、これを修正しました。

3-4-0S32: (VSI) VSI for NX をインストールできない不具合を修正

PSMODULEPATH 環境変数に複数の値が含まれている場合に、VSI for NX をシステムにインストールできない不具合がありましたが、これを修正しました。

NintendoWare NX Addon 3.4.0 (based on Generic 3.4.0) の変更点

互換性の失われる仕様変更

★3-4-0W01: (エフェクト-VFX) VFX ランタイムのマニュアル放出 API のオプション機能の廃止

nn::vfx::EmitterSet::EmitParticle 関数について、 NX Addon 3.0.0 より、
内部的な空きパーティクル検索の方法を調整するオプション（任意）を第3引数で指定可能にしていましたが、特定のケースで意図通りに動作しないことが分かり、これを廃止させていただきます。API と挙動は 1.x.x 相当の仕様に戻ります。
明示的に第3引数を指定していない限り、互換性は問題なく維持されます。第3引数を指定している場合は、そのまま第3引数を削除してください。
