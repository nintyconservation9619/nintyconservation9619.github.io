NintendoSDK NX Addon 3.6.0 (based on Generic 3.6.0) の変更点

互換性の失われる仕様変更

★3-6-0S01: (General) アプリケーションの動画撮影機能の追加

アプリケーションの動画撮影機能が追加されました。動画撮影機能が有効なアプリケーションではシステムによって常に自動的にメモリ上に過去最大 30 秒分の動画データが溜められており、キャプチャーボタンを長押しすることでこの動画データがファイルに保存されます。本バージョンの SDK でビルドされたアプリケーションを Firmware version 4.0.0 以降がインストールされた本体で起動した場合、この動画撮影機能がデフォルトで有効化されます。
保存される動画のフォーマット等の詳細に関してはこちらなども参照してください。

動画撮影機能が有効化されたアプリケーションでは、NX64 環境では、アプリケーションが使用可能なメモリがこれまでより 96MiB 減少し、この分がシステムによって使用されます(NX32 環境では使用可能メモリに変化はありません)。また、アプリケーションの性能への影響があります。詳細は こちら を参照してください。

これらが許容できない場合や、企画上の都合など何らかの理由がある場合には、nmeta の VideoCapture の項目を Disable にすることで動画撮影機能を無効化することができます。また、AuthoringEditor によっても設定をすることができます。
動画撮影 の項目を参照してください。

動画撮影機能を有効化したアプリケーションにおいて、一時的に動画撮影を停止することができます。詳しくは新規追加された nn::oe::EnableRecording() および nn::oe::DisableRecording() を参照してください。

★3-6-0S02: (TOOL) VSI for NX と Target Manager のパッケージ分離

VSI for NX と Target Manager に関するパッケージ (zarf) の構成を変更しました。

* 変更前：「VSI for NX パッケージ」に NX Visual Studio Integration と Target Manager が含まれていました。
* 変更後：「VSI for NX パッケージ」に NX Visual Studio Integration が、「Target Manager パッケージ」に Target Manager が含まれます。

「Target Manager パッケージ」は NX 開発機でアプリケーションを実行する場合などに必要になります。
「(新しい) VSI for NX パッケージ」は、Visual Studio を利用してアプリケーションをビルドしたりデバッグ実行する際に必要になります。

用途に応じて必要なパッケージをインストールするようにしてください。

いずれも NDI でのインストール時に Windows システムへのインストール処理も行われます。 これ以外のタイミングでシステムへインストールしたい場合は、下記のファイルを「管理者権限」で実行してください。

* 「VSI for NX」は Installers\Vsi\Install.bat
* 「Target Manager」は Installers\TargetManager\Install.bat

★3-6-0S03: (VSI) Target Manager の互換性のない変更

  • TMAPI.h のサポートが NX Addon 5.1 から打ち切られます
    • NX Addon 4.1 から、Target Manager の機能を使用できる TM ライブラリが正式対応されました。これに伴い、NX Addon 5.1 をめどに、今まで C:\Program Files\Nintendo\Oasis\include\TMAPI.h に定義されていた関数が使用できなくなります。今後は C:\Program Files\Nintendo\Oasis\include\tm.h をお使いください。
  • Target Manager の JIT デバッグ有効化オプションが、デフォルトでは無効となりました
    • JIT デバッグを使用する場合は、「Target Manager > ツール > オプション > JIT デバッガを有効にする」をチェックしてください。


機能追加

3-6-0S04: (AOC) 追加コンテンツの機能を使ったシーズンパスのサンプル追加

追加コンテンツの機能を使用しシーズンパスを実現する方法についてサンプルを追加しました。

3-6-0S05: (FS) nn::nfp のアクセスログ出力を追加し FsAccessLogChecker を version 11.0 に更新

fs アクセスに関わる nn::nfp API でアクセスログ出力を追加し、FsAccessLogChecker (%NINTENDO_SDK_ROOT%\Tools\FsAccessLogChecker\FsAccessLogChecker.exe) を version 11.0 に更新しました。以下の変更が含まれます。

  • 入力ファイルの文字コード解析を修正
  • 読み込み頻度の領域を確認できるよう対応
  • リードアクセスの重なり判定を修正
  • nn::nfp 経由の fs アクセスをチェックするよう対応


3-6-0S06: (General) アプリケーションの動画撮影の一時停止・再開 API を追加

動画撮影機能を有効化したアプリケーションにおいて、一時的に動画撮影を停止・再開するための機能として以下の API を追加しました。

  • nn::oe::EnableRecording()
  • nn::oe::DisableRecording()


動画撮影の有効化は、nmeta の VideoCapture の項目によって行います。詳細は「動画撮影」の項目を参照してください。
また、保存される動画フォーマット等の詳細に関しては こちら を参照してください。
アプリケーション性能への影響については、こちら を参照してください。

3-6-0S07: (IRSENSOR) モーションIRカメラにポインティング機能を追加

モーションIRカメラを利用して、画面のポインティングを行う機能を追加しました。
本機能を利用するためには、専用のハードウェアが必要となるためご注意ください。
詳細については、IRSENSOR ライブラリのマニュアルをご覧ください。

3-6-0S08: (Sample) 対応言語取得サンプルの追加

Samples/Sources/Applications/OeGetLanguage にプログラムから対応言語を取得するサンプルを追加しました。

3-6-0S09: (TOOL) 市場で発生したクラッシュ情報の取得機能を追加

アプリケーションが市場でクラッシュした際の情報を取得できるようになります。詳細は SDK ドキュメントの、機能／システム／クラッシュレポートを参照してください。

3-6-0S10: (TOOL) 中国語（簡体字）、中国語（繁体字）、韓国語対応アプリの開発をサポート

中国語（簡体字）、中国語（繁体字）、韓国語対応アプリを作成できるようにしました。
OEライブラリ／マニュアル／機能説明／言語取得に対応言語取得方法の追記と、対応言語設定取得サンプルを追加しました。
また、AuthoringToolで中国語（簡体字）、中国語（繁体字）、韓国語対応アプリ作成時に--error-unpublishableオプション時にエラーにならないように修正しました。

3-6-0S11: (VSI) VSI, Target Manager に新機能を追加

  • VSI
    • スレッドを個別に一時停止・再開する機能の追加
    • コンパイル済みヘッダを暗黙的にインクルードする機能の追加
      • 今までは強制インクルードファイルに指定する必要がありましたが、NX プラットフォームではこの操作が不要になりました。
      • これに付随して、コンパイル済みヘッダの内容もインテリセンスに反映されるようになりました。
    • ROM 提出可能でない場合にビルドエラーとするオプションの追加
  • Target Manager
    • スクリーンショットを撮る機能の追加
      • ターゲットを右クリックし、「スクリーンショットを撮る」メニューを選択してください。
    • ターゲット毎に Log View を表示できる機能の追加
      • ターゲットを右クリックし、「ログを表示」メニューを選択してください。
    • アプリをインストールする機能、およびインストールされたアプリを起動する機能の追加
      • ターゲットを右クリックし、「アプリのインストール」または「アプリの起動・管理」メニューを選択してください。
    • ダンプファイルの生成機能の追加
      • ターゲットを右クリックし、「ダンプ」メニューを選択してください。
    • ゲームカードへの書き込み機能の追加
      • ターゲットを右クリックし、「アプリのインストール」メニューを選択してください。
    • Target Manager 単体で使用可能にする機能の追加
      • 今までは Target Manager が依存するファイルのコピーも必要でしたが、NintendoTargetManager.exe だけで使用することが可能になりました。
    • 複数台に対して、同時にリセットする機能、およびプログラムの終了をする機能の追加
  • その他
    • 互換性のサポート
      • Target Manager の再インストールなしに、古い FW や NX Addon 4.1 以降で提供された RunnerTools を使えるようになります。
    • Target Manager の一部機能を使用するホスト側ライブラリのサポート
    • 拡張子 nsp の関連付け
      • アイコンが固有のものになり、ダブルクリックで Target Manager の既定のターゲット上で実行されるようになりました。
    • Windows の権限不足などにより、HostFs でファイル操作をしようとして失敗した場合、Target Manager に警告を出す機能の追加
    • 複数台に対する 電源オン・オフ, LED 点滅メニューの追加

3-6-0S12: (DevMenu) NX Addon 4 系向けの変更を反映

NX Addon 4 系向けに行った DevMenu の一部の変更を NX Addon 3 系に反映させました。以下に該当する変更を列挙します。詳細は各バージョンの変更履歴を参照してください。

◆ NX Addon 4.0.0

  • ユーザー詳細画面でボタン押下時に FATAL エラーが発生することがある不具合を修正


◆ NX Addon 4.1.0

  • DevMenuCommand に、本体のタイムゾーンを参照、変更する機能を追加
  • DevMenu におけるネットワーク接続時のエラーハンドリングの改善


◆ NX Addon 4.2.0

  • ネットワーク利用要求提出時に 2110-111 エラーが表示される問題の修正
  • サーバーから正しい現在時刻を取得し、ネットワーク時計を補正する機能を追加
  • NX 環境で使用不可の SD カードが挿入されていた場合にエラーが表示されるように変更


◆ NX Addon 4.3.0

  • nn::friends::ShowUserDetailInfo で表示した詳細画面からアプリケーションが起動できない問題を修正


◆ NX Addon 4.4.0

  • TV アンダースキャン値の設定機能を追加
  • ユーザー選択を伴うアプリケーション起動不具合を修正
  • セーブデータ一覧表示までの時間を改善


3-6-0S13: (TOOL) AuthoringEditor 上の「LogoType」から「DistributedByNintendo」を削除

AuthoringEditor 上でアプリケーション nmeta を編集する際に設定できる「ロゴデータの設定」について、「DistributedByNintendo」を削除しました。

3-6-0S14: (VSI) VSI, Target Manager の改善

  • Target Manager
    • ターゲット未接続状態で nsp を Target Manager にドロップした場合に、自動でターゲットに接続するよう改善しました。
    • Target Manager で nsp のパスを指定して実行する場合、" でくくられていても正しく解釈するよう改善しました。


不具合修正

★3-6-0S15: (TOOL) AuthoringEditor 上で nsp 編集中に nmeta インポート機能が使用できない不具合の修正

AuthoringEditor 上でアプリケーション nsp の編集中に nmeta インポートの機能が使用できない不具合を修正しました。
これにより nsp を開いている最中にも既存の nmeta ファイルの内容をインポートして反映できます。

3-6-0S16: (BuildSystem) nspd の作成時に指定するパスに空白が入っていると失敗する不具合の修正

AuthoringTool で nspd を作成する際に、パスに空白が入っているとビルドに失敗する不具合を修正しました。
この不具合は、 VSI でアプリケーションをビルドするときに、プロジェクトの TargetName プロパティに空白が入っていると、同様に発生します。

3-6-0S17: (TOOL) AuthoringEditor、AuthoringToolのマスターROM提出不可エラーの誤検出を修正

libcurlを使用したnspをAuthoringEditorで開いた場合やAuthoringToolでROM提出不可エラー取得時に、「使用申請が必要な機能 - Bundle_Private_NetworkMiddleware_Generic」が誤検出される問題がありました。これを修正しました。

3-6-0S18: (TOOL) Microsoft Visual C++ 2015 ランタイムのない環境でセットアップが失敗する不具合の修正

Microsoft Visual C++ 2015 ランタイム(Microsoft Visual C++ 2015 再頒布可能パッケージ等) がインストールされていない環境で SDK をインストールしようとするとエラーが発生する不具合を修正しました。

3-6-0S19: (VSI) VSI, Target Manager の不具合を修正

  • VSI
    • インテリセンスが不正にエラーを出す問題の修正
    • ステップ実行に時間がかかる問題の修正
    • 一部の NatVis 構文が正しく機能しなかった問題の修正
    • シンボルファイル読み込み時の UI がおかしかった問題の修正
      • VSI for NX 4.1.2 より前は nss ファイルのロードに「シンボルの読み込み情報」メニューを使用する必要がありましたが、VSI for NX 4.1.2 からは「シンボルの読み込み」メニューを使用してください。
    • メンバ関数に対する関数ポインタが正しく表示されなかった問題の修正
    • nro に対するいくつかのプロジェクトプロパティが指定できなかった問題の修正
    • nmeta ファイルを VSI で開けないことがある問題の修正
    • Visual Studio のメモリウインドウで、値の変更が行えなかった問題の修正
    • NX32 の逆アセンブリを開いた場合でも、NX64 のレジスタとして表示されていた問題の修正
    • ライブラリプロジェクトをデバッグ実行した場合に、ロードから先に進まなかった問題の修正
  • Target Manager
    • アプリの実行とリセットを繰り返していると、Target Manager から EDEV がリセットできなくなる問題の修正
    • スリープ時に htcs ポートが残ってしまう問題の修正
    • スリープ時にファイルの Read/Write が失敗することがある問題の修正
    • スリープを挟むと Target Manager が不安定になる問題の修正
    • 多数台接続時に挙動が不安定だった問題の修正
    • EDEV 接続時または切断時に、エラー画面に陥る場合があった問題の修正
    • Target Manager 接続前に起動されていたアプリを、Target Manager から終了させることができなかった問題の修正
    • 二つ以上の Log Viewer を実行した場合、一つ目にしかログが表示されなかった問題の修正
    • Target Manager が起動する前に Log Viewer を起動していた場合、ログが表示されなかった問題の修正
    • ある程度長いログを複数のスレッドから同時に送出した場合、片方のログが表示されなかった問題の修正
    • アプリ未実行時に、tm::GetTargetProgramState が失敗を返していた問題の修正
    • Target Manager 起動直後にターゲットの登録削除を行っても、登録が残る場合があった問題の修正
    • Target Manager から起動されたアプリをデバッグやダンプした場合、誤った例外コードが表示されていた問題の修正
  • その他細かな不具合修正
