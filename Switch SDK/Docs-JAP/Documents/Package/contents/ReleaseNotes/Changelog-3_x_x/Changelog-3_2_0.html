NintendoSDK NX Addon 3.2.0 (based on Generic 3.2.0) の変更点

互換性の失われる仕様変更

★3-2-0S01: (AUDIO) nn::audio::AuxType に適用する I3dl2Reverb エフェクトのユーティリティの追加

nn::audio::AuxType で取得されたバッファに対して、nn::audio::I3dl2ReverbType と同等のエフェクト処理を行うユーティリティ、 nn::audio::AuxI3dl2ReverbType を追加しました。

具体的な利用方法は AudioEffect サンプルを参照してください。

またこの機能追加に伴い、既存の nn::audio::I3dl2ReverbType と共用される nn::audio::I3dl2ReverbParameterSet が追加されました。
この構造体は I3dl2ReverbType に対して、一括で設定を更新するために利用されます。
この構造体の追加により nn::audio::LoadI3dl2ReverbPreset() の引数に変更が入っていますので、ご注意ください。

★3-2-0S02: (FS) セーブデータサイズ拡張時の制限違反チェックを追加

パッチや nn::fs::ExtendSaveData() の実行によるセーブデータサイズ拡張で指定するサイズに制限を追加し、制限違反の場合はプログラムを停止するようにしました。詳しくは FS ライブラリ／マニュアル／機能説明／セーブデータ／セーブデータ (NX Addon) を参照してください。

機能追加

★3-2-0S03: (AUDIO) 音声出力先・入力元デバイスのイベント発生を通知するイベントを取得する API の追加

音声出力先・入力元デバイスのイベント発生を通知するイベントを取得する nn::audio::AcquireAudioDeviceNotificationForOutput() および
nn::audio::AcquireAudioDeviceNotificationForInput() を追加しました。
また、これに伴い、 nn::audio::AcquireAudioDeviceSwitchNotification() の使用が非推奨となりました。
詳しくは API リファレンスをご確認ください。

★3-2-0S04: (GFX) HlslCrossCompiler の追加

HLSL から GLSL に変換する HlslCrossCompiler.exe ( %NINTENDO_SDK_ROOT%\Tools\Graphics\GraphicsTools\HlslCrossCompiler.exe ) を追加しました。詳細は グラフィックス／ツール／HlslCrossCompiler を確認して下さい。

★3-2-0S05: (TOOL) グラフィックスリソースファイル用コンバーターを NintendoSDK GraphicsConvertersKit パッケージに分離

グラフィックスリソースファイル用コンバーターと Photoshop プラグインを NintendoSDK パッケージから分離して、NintendoSDK GraphicsConvertersKit パッケージに移動しました。
アプリケーションのパッチを作成する際、初版アプリケーションと同一のバージョンの GraphicsConvertersKit パッケージを使用することでリソースファイルの差分を減らしてパッチサイズを小さくすることができます。

3-2-0S06: (APPLET) アプリケーションから本体のアルバムにアクセスする API を追加

本体のアルバムにアクセスする機能を提供する ALBUM ライブラリが追加されます。アプリケーションが生成した画像を画面写真として保存する機能とアプリケーションから SNS へ投稿する機能が提供されます。

3-2-0S07: (APPLET) アプリケーションへの起動パラメータを取得する API の追加

アプリが起動する際に、起動元から渡された起動パラメータを取得するためのアプリ向け API として nn::oe::TryPopLaunchParameter() を追加しました。ただし、本体機能（HOME メニューなど）からどのようなケースで起動パラメータが渡されるのかについては本体機能の仕様に従って下さい。

3-2-0S08: (APPLET) スクリーンショット撮影画像に回転方向を設定する機能を追加

キャプチャーボタン短押しによるスクリーンショット撮影機能に関して、保存される撮影画像に回転方向の情報を設定するための以下の API を追加しました。

  • nn::oe::SetScreenShotImageOrientation()


3-2-0S09: (AUDIO) サンプルレート変換のための API の追加

nn::audio::AudioOut と組み合わせて利用することを目的とした、サンプルレート変換 API を追加しました。詳細は下記 API のリファレンスを参照してください。これらの API の内部処理はすべて CPU で行われます。

  • nn::audio::GetRequiredBufferSizeForResampler()
  • nn::audio::GetResampledOutBufferSize()
  • nn::audio::InitializeResampler()
  • nn::audio::ProcessResamplerBuffer()


3-2-0S10: (DevMenu) DevMenu に動作モード・性能モードに関する設定切り替え機能を追加

DevMenu の開発支援機能に以下の項目の切り替え機能を追加しました。

  • Operation Mode Policy
  • Performance Mode Policy
  • CPU Overclock


使用方法および諸注意については DevMenu のマニュアル を参照してください。

3-2-0S12: (DevMenu) アプリケーション管理機能を追加

Application ID 単位でコンテンツ（アプリケーション実体、追加コンテンツ、パッチ）を管理する機能を追加しました。

  • Application ID に紐付く全コンテンツの削除
  • コンテンツ単位での削除
  • コンテンツ単位での ROM ID ハッシュの表示
  • 必須バージョンのリセット


詳細は DevMenu のマニュアル を参照してください。

シェーダーコンバーター (%NINTENDO_SDK_ROOT%\Tools\GraphicsTools\ShaderConverter.exe) で、文字列を指定するオプションに '%' が入った文字列を渡すと Internal Error となってクラッシュする不具合を修正しました。

3-2-0S69: (TOOL) シェーダーコンパイラーで、ソースが SPIR-V のときに不正なダンプ結果が出力される不具合の修正

シェーダーコンパイラー (%NINTENDO_SDK_ROOT%\Tools\Graphics\GraphicsTools\ShaderCompiler.dll) で nngfxToolShaderCompilerCompileOptionCommon::dump に nngfxToolShaderCompilerDump_NoDump 以外の値を指定しており nngfxToolShaderCompilerCompileArg::shaderSourceFormat に nngfxToolShaderCompilerShaderSourceFormat_SpirV の値をしている場合、またはシェーダーコンバーター (%NINTENDO_SDK_ROOT%\Tools\Graphics\GraphicsTools\ShaderConverter.exe) で --dump-directory オプションを指定しており --source-format オプションに SpirV を指定している場合、ダンプ結果に文字化けした不正なテキストが出力される不具合を修正しました。

NintendoWare NX Addon 3.2.0 (based on Generic 3.2.0) の変更点

互換性の失われる仕様変更

★3-2-0W01: (レイアウトツール) --degamma-parameter を指定している場合に、フォントの影カラーや、文字毎の変換のカラーが正しくガンマ変換処理されない不具合を修正

LayoutConverter で、degamma-parameter オプションを指定している場合に、フォントの影カラーや、文字毎の変換のカラーが正しくガンマ変換処理されない不具合があり、これを修正しました。不具合に合わせて、正しく表示されるようにレイアウトデーター側で調整している場合、不具合が修正されることによって逆に表示が正しく行われなくなる可能性がありますのでご注意ください。（degamma-parameter は、移植作品など、過去に SRGB ガンマワークフロー前提で作成されたレイアウトデーターをそのまま利用する用途で指定されるオプションです。）

★3-2-0W02: (レイアウトツール) ピクチャペインの形状設定に「四角形」以外が指定されているときのみ中間ファイルへ形状設定のデータを出力するように変更

これまでピクチャペインの形状設定にかかわらず全ての形状に関するパラメータが中間ファイルへ出力されていましたが、形状設定が「四角形」以外の時のみそれらのパラメータの関連タグを出力するように変更しました。
この変更により「四角形」の状態で中間ファイルを保存すると、以前は保存されていてた「四角形」以外の形状設定のパラメータがリセットされるようになります。
