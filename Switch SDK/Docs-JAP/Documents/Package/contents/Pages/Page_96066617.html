CPU処理の最適化

CPU処理の最適化について説明します。
CPUで行われる処理は大きく２つに分けられます。

  1. エミッタ 処理
  2. パーティクル処理( CPUエミッタ時 )

エミッタ

日々データ作成の上でパーティクル数には気を使っているかと思いますが、再生エミッタ数もネックになる場合があります。
特にプロジェクトにて独自のカリング処理などを行っていない場合、同時に再生されるエミッタ数が膨大になり問題となる場合があります。

エミッタで行われる処理

エミッタでは以下の処理 ( CPUエミッタ/GPUエミッタ共通 ) が行われます。

  • エミッタ時間アニメ処理 ( 利用時 )
  • エミッタマトリクス更新処理
  • パーティクル放出処理
  • シェーダに送るエミッタデータのパッキング処理
  • 描画処理( ステート設定 / テクスチャ設定 / エントリ処理 など )

数の上では、パーティクル数と比較になりませんが、処理の上では、１エミッタは１パーティクルより大きいCPUコストが必要になります。
同時再生されるエミッタ数が数百の単位になると、無視できないCPUコストになる場合もあります。
