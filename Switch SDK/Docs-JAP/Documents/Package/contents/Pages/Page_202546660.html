NintendoSDK 1.2.0

本バージョンの既知の不具合は以下の通りです。

  • 全般
    • OS のシステム ロケールをデフォルト値以外に変更した場合の動作は保証外
    • API リファレンスの関数などのアルファベット一覧が正しく表示されない
  • FS
    • ドライブのルートディレクトリに nn::fs::ReadDirectory() を実行した際、1つ目のエントリが列挙されない場合がある
  • HID
    • PC 環境で振動が発生しない
  • libcurl
    • 1 台の PC で複数のアプリケーションからの HTTPS が実行できない
  • Socket
    • nn::socket::Recv() で MSG_DONTWAIT フラグを使用できない
    • nn::socket::Select() もしくは、nn::socket::Poll() が EBADF を返すことがある
  • PERF
    • 特定の PC の環境において、サンプル PerfSimple が停止する
  • SSL
    • nn::ssl::Initialize() と nn::ssl::Finalize() は、メインスレッドで呼んでください
  • オーディオツール
    • NintendoWare SoundMaker, SoundProjectWaveExporter でサウンドの波形出力をした際に、波形の頭に無音が入ることがある
    • NintendoWare SoundMaker でインポートしたサウンドプロジェクトをコンバートすると、不正なサウンドアーカイブを出力することがある
    • NintendoWare AtkPlayer でサウンドアーカイブのリロードを行うとアプリケーションが停止することがある
    • nn::atk::viewer ライブラリを使ったインゲーム編集中に、シーケンスサウンドのファイルパスを変更するとプログラムが停止することがある
    • nn::atk::viewer ライブラリを使ってインゲーム編集のストリームサウンドを再生中に稀にプログラムが停止する
    • nn::atk::SoundSystem::SetVoiceCountMax によって最大ボイス数を設定できない
  • レイアウト-UI2D
    • アルファテストが機能しない
    • LayoutViewer で、一部のグラフィックスドライバ環境においてテクスチャ表示が正しく行われない
  • 3Dツール
    • 3DEditor とランタイムを接続してから接続先プラットフォームを変更すると正しくシェーダーが送信されない
  • 3D-G3D
    • G3dDemo サンプルの Town シーンの実行中に不正アクセスが発生することがある
    • G3dDemo サンプルの Edit シーンに 3DEditor からモデルやシェーダーをロード、アタッチするとアサートに失敗する
  • エフェクト-VFX
    • エフェクトデータのチャイルドエミッタの継承設定「パーティクルにエミッタ割当」が OFF の状態で「履歴式ストライプ」「履歴式ストライプ2」のいずれかを設定すると VFX ランタイムが停止する
  • Spy
    • Spy.exe で１時間１１分以上の記録ができない
    • Spy.exe の Atk シーケンス変数パネルで例外が発生することがある

全般

OS のシステム ロケールをデフォルト値以外に変更した場合の動作は保証外

OS のシステム ロケールをデフォルト値以外に変更した場合の動作は保証外です。(例えば日本語版 Windows 上でシステム ロケールとして“日本語(日本)”以外を選択した場合)

API リファレンスの関数などのアルファベット一覧が正しく表示されない

API リファレンスで、「名前空間」→「名前空間メンバ」 の下の、「全て」 と 「関数」 と 「列挙値」で表示されるアルファベット一覧を選択しても正しいリストが表示されません。

 

FS

ドライブのルートディレクトリに nn::fs::ReadDirectory() を実行した際、1つ目のエントリが列挙されない場合がある

nn::fs::MountHost()/MountHostRoot() 経由でドライブ直下を nn::fs::OpenDirectory/ReadDirectory() した際に、対象がリムーバブルメディアなどの場合に、1つ目のエントリが列挙されない不具合があります。

HID

PC 環境で振動が発生しない

症状

NintendoSDK 0.15.0 以前を利用したことがある PC 上で、NintendoSDK 0.16.0 以降を利用して振動機能を利用する際に、振動が発生しない不具合があります。

HidVibrationBasic や HidVibrationPlayer のような振動機能を利用するプログラムを起動した際に、この不具合を確認できます。 

回避策

以下のファイルを手動で削除してください。

C:\Users\<ユーザー名>\AppData\Local\Temp\Nintendo_SystemSaveData_SystemSettings

その後、該当プログラムを PC 上で再度起動すると、上記のファイルが自動的に再生成されて、振動が発生するようになります。

修正予定

次回以降のリリースで修正予定です。

Socket

nn::socket::Recv() で MSG_DONTWAIT フラグを使用できない

症状

nn::socket::Recv() で MSG_DONTWAIT フラグをセットしても、ノンブロックモードになりません。

回避策

nn::socket::fcntl() で O_NONBLOCK をセットすることで同等の動作にすることができます。

修正予定

次回以降のリリースで修正予定です。

回避策

回避策はありません。

修正予定

特定の PC の環境においてサンプル PerfSimple がアサートで停止する不具合があります。

SSL

nn::ssl::Initialize() と nn::ssl::Finalize() は、メインスレッドで呼んでください

症状

nn::ssl::Initialize() と nn::ssl::Finalize() をメインスレッド以外のスレッドで使用すると、動作が停止します。

回避策

nn::ssl::Initialize() と nn::ssl::Finalize() はメインスレッドで使用するようにしてください。

修正予定

次回以降のリリースで修正予定です。 

オーディオツール

NintendoWare SoundMaker, SoundProjectWaveExporter でサウンドの波形出力をした際に、波形の頭に無音が入ることがある

NintendoWare のツール SoundMaker, SoundProjectWaveExporter でサウンドの波形出力を行うと、出力波形の頭に無音が入ることがあります。

この問題は、今後のリリースで修正予定です。

NintendoWare SoundMaker でインポートしたサウンドプロジェクトをコンバートすると、不正なサウンドアーカイブを出力することがある

SoundMaker で NW4C / NW4F のサウンドプロジェクト（コンバート用のキャッシュファイルが残った状態）をインポートして（通常）コンバートすると、NintendoSDK 向けのバイナリサウンドアーカイブファイル (.bfsar) に、NW4C / NW4F 向けのバイナリリソースファイルが混ざってしまい、nn::atk / nw::snd ランタイムで正しく読み込めない問題が発生します。

 サウンドプロジェクトをインポートした後は、必ず再コンバートすることで、古いキャッシュファイルを利用しなくなるため、この問題を回避できます。

NintendoWare AtkPlayer でサウンドアーカイブのリロードを行うとアプリケーションが停止することがある

NintendoWare SoundMaker でストリームサウンドを編集した後、サウンドアーカイブファイルのコンバートを行い、AtkPlayer でリロードを行うとアプリケーションが停止することがあります。

本件は次回以降のリリースで改善予定です。

nn::atk::viewer ライブラリを使ったインゲーム編集中に、シーケンスサウンドのファイルパスを変更するとプログラムが停止することがある

インゲーム編集の対象にしたシーケンスサウンドのファイルパスを変更すると、アサートが発生してプログラムが停止することがあります。現状以下のアサートが発生することを確認しています。

SDK Assertion Failure: 'velocity >= 0 &amp;&amp; velocity <= 127' in nn::atk::detail::driver::SequenceTrack::NoteOn()

本件は次回以降のリリースで改善予定です。

nn::atk::viewer ライブラリを使ってインゲーム編集のストリームサウンドを再生中に稀にプログラムが停止する

インゲーム編集対象のストリームサウンドの再生を繰り返したり、インゲーム編集でパラメータを高速に変更し続けたりしていると稀に以下のアボートが発生してプログラムが停止することがあります。 

Abort: 'delimiter != nullptr' in nn::fs::detail::GetMountName()

本件は次回以降のリリースで改善予定です。

nn::atk::SoundSystem::SetVoiceCountMax によって最大ボイス数を設定できない

症状

nn::atk::SoundSystem::SetVoiceCountMax を用いてサウンドライブラリが使用する最大ボイス数を設定しようとした場合、

その設定は無視されて nn::atk::SoundSystem::SoundSystemParam::voiceCountMax で設定した値が使用されます。

回避策

nn::atk::SoundSystem::SetVoiceCountMax は非推奨な機能であるため、推奨される機能である nn::atk::SoundSystem::SoundSystemParam::voiceCountMax を用いることをご検討ください。詳しくは nn::atk::SoundSystem::SoundSystemParam::voiceCountMax のリファレンスを参照ください。

修正予定

修正予定はありません。

3Dツール

3DEditor とランタイムを接続してから接続先プラットフォームを変更すると正しくシェーダーが送信されない

症状

3DEditor とランタイムを接続し、シェーダーをリアルタイム編集、もしくは最適化シェーダーを生成後、接続先プラットフォームを変更すると、変更前のプラットフォームのシェーダーが接続先に送信されてしまう不具合があります。

回避策

接続先プラットフォームを切り替える際に 3DEditor を再起動することで回避できます。

修正予定

本件は次回以降のリリースで修正予定です。

エフェクト-VFX

エフェクトデータのチャイルドエミッタの継承設定「パーティクルにエミッタ割当」が OFF の状態で「履歴式ストライプ」「履歴式ストライプ2」のいずれかを設定すると VFX ランタイムが停止する

症状

エフェクトデータのチャイルドエミッタの継承設定「パーティクルにエミッタ割当」が OFF の状態で「履歴式ストライプ」「履歴式ストライプ2」のいずれかを設定すると、VFX ランタイム内で不正アクセスが発生する不具合があります。

回避策

エフェクトデータの設定の「パーティクルにエミッタ割当」を ON に設定することで回避できます。

修正予定

本件は次回以降のリリースで修正予定です。

Spy

Spy.exe で１時間１１分以上の記録ができない 

症状

Spy.exe をアプリケーションと接続し、１時間１１分経過するとデータが更新されなくなったり例外が起きるなど、Spy.exe が正常に動作しなくなります。

また、このとき記録されたデータファイルは不正な内容になり、Spy.exe で正常に開くことができません。

回避策

回避策はありません。

修正予定

本件は次回以降のリリースで修正予定です。

ただし、すでに記録済みの不正なデータファイルは、修正後の Spy.exe でも正常に開くことはできません。

Spy.exe の Atk シーケンス変数パネルで例外が発生することがある

症状

Spy.exe  をアプリケーションと接続し、複数のシーケンスサウンドを再生したときに Atk シーケンス変数パネルを表示していると例外が発生することがあります。

回避策

複数のシーケンスサウンドを再生しないことで回避できます。

もしくは、NintendoSDK 1.1.0 に含まれる Spy.exe を使うことでも回避できます。

修正予定

本件は次回以降のリリースで修正予定です。
