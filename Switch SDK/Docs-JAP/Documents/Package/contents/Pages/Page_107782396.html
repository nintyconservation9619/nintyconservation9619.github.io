従来の開発環境との違い

NintendoSDK では、以下の項目が従来の開発環境から変更されます。

API

  • 従来の開発環境の API から、API の名前や機能が変更されます。
  • 今後、Nintendo が提供するプラットフォーム間で API が共通化されます。
  • 対象プラットフォームと同一の API を使用可能な Windows プログラムの開発環境が用意されます。

コンパイラおよびデバッガ

  • コンパイラツールチェインがプラットフォーム間で共通化されます。
  • デバッガソフトウェアがプラットフォーム間で共通化されます。
  • VisualStudio を前提とした環境となります。
    • デバッガソフトウェアは Visual Studio Integration （VSI）形式のみとなります。
    • make 等によるビルドシステムは提供されません。
    • Windows プログラムの開発環境ではコンパイラツールチェインおよびデバッガソフトウェアとして VisualStudio を使用します。

リリース

  • パッケージは Nintendo Dev Interface (以下、NDI) と呼ばれる専用 Windows アプリケーションを介して配布されます
    • NDI では、ネットワーク経由でのパッケージインストールと、
      あらかじめ必要ファイルをダウンロードした上でのオフラインインストールが可能です
    • 従来のように、パッケージごとに分割された zip ファイルの配布は行いません
