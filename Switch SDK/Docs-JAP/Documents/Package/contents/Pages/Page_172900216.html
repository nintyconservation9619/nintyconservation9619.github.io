ASTC12x12
 0.89  ASTC フォーマットの中では最もちいさいファイルサイズですが、品質は極端に落ちます。
  • 圧縮フォーマットです。
  • 表にある14種類の圧縮品質から、きめ細かく「圧縮率」と「圧縮品質」のバランスを選択することができます。
  • 1チャンネル（L, A フォーマット）～ 4チャンネル（RGBA）まですべての形式で利用できます。
  • ソース画像の特徴から自動判定して、圧縮最適化を行いノイズを減らせます。
    • RGBがすべて同じ値 ⇒ グレースケールと判断
    • アルファがすべて１ ⇒ アルファ無し画像と判断
    • 32-bit tga 画像を使って、用途に合わせて適切に上記を設定します

例えば ASTC8x8 のとき、以下のように、アルファがあるときと、アルファが白のときでRGB チャンネル側の品質が変わります。

アルファがあるとき、ややノイズが目立ちます

 アルファが白(1)のとき、RGB 側が高品質になります
参考：
「bpp」という単位は「bits per pixel」の略で、1ピクセルの色を表現するのに必要とするビット数を表しています。
例えば、8bpp の場合、1 ピクセルにつき 8 ビットの色情報を仕様していることになります。

注意すること

  • 圧縮に時間がかかる（特に大きな画像で注意が必要。数分かかることもある）
  • PCでは表示できない。
    • 品質やサイズの確認は実機で行う
    • LayoutEditor でPCプレビューをする際は、制限を回避するため強制的に BC3 に変換して表示している

利用方法

プロジェクト設定で、AstcTextureEnabled を true にすることで、ASTCフォーマットを利用することができます(デフォルトは false で無効になっています)。詳細は「プロジェクト設定ファイルフォーマット」を参照してください。
