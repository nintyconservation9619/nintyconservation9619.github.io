CharModel

CharModel

概要

nn::mii::CharModel は顔モデルを構成する各パーツのシェイプやテクスチャなどのデータを保持するクラスです。

顔モデルを構成する各パーツに関しては、顔モデル構造の顔モデルのページを参照してください。

CharModelInfo

nn::mii::CharModelInfo は顔輪郭テクスチャ、マスクテクスチャの解像度、座標系などの顔モデルの生成のために必要な情報を保持します。

動的テクスチャに関する設定

顔輪郭テクスチャやマスクテクスチャは、任意のテクスチャフォーマットや解像度、ミップマップ数を指定することが可能です。

以下の関数を使用してそれぞれ指定してください。

  • nn::mii::CharModelInfo::SetDynamicTextureFormat()
  • nn::mii::CharModelInfo::SetDynamicTextureMipCount()
  • nn::mii::CharModelInfo::SetDynamicTextureResolution()

nn::mii::CharModelInfo::SetMaskCount() 関数を使用することでマスクテクスチャの枚数を指定することも可能です。CharModel に複数の表情用のマスクテクスチャを持たせたい場合は、表情分のマスクテクスチャを用意してそれぞれのテクスチャに各表情のマスクをレンダリングするようにしてください。

顔モデルの生成条件

作成する顔モデルの種類をフラグの組み合わせで指定できます。顔モデルの種類に関しては顔モデル構造の顔モデルのページを参考にしてください。

nn::mii::CharModelInfo::SetCreateFlag() 関数には nn::mii::CreateFlag の論理和を渡してください。nn::mii::CreateFlag_Normal、nn::mii::CreateFlag_Hat、nn::mii::CreateFlag_FaceOnly を 1 つ以上、nn::mii::CreateFlag_NoseNormal、nn::mii::CreateFlag_NoseFlatten を 1 つ以上指定する必要があります。

座標系

顔モデルの初期化の際に使用されるデフォルトの座標系は頭の上が Y 軸正の向き、前が Z 軸正の向きです。

アプリケーションが使用する座標系がデフォルトの座標系と異なる場合は、描画時に nn::mii::CharModelInfo::SetCoordinateUp() 関数と nn::mii::CharModelInfo::SetCoordinateFront() 関数を使用して座標系を指定することが可能です。アプリケーションで使用する座標系に合わせて、顔モデルの向きをあらかじめ回転させておくことができます。

SetCoordinateUp() 関数と SetCoordinateFront() 関数に渡す値は、直行するような組み合わせを設定する必要があります。 座標系変換を行うため、座標系がデフォルト以外の場合は、初期化処理時間が増えます。

スケーリング

顔モデルの初期化の際には使用される顔モデルのスケール値は 1.0 に設定されています。

アプリケーションが使用するスケールと異なる場合は、nn::mii::CharModelInfo::SetScale() 関数を使用してスケールを設定することが可能です。
アプリケーションで使用するスケールに合わせたサイズで顔モデルを生成します。

スケール変換を行うためスケールがデフォルト以外の場合は、初期化処理時間が増えます。

初期化と終了処理

初期化/終了時に注意点に関して説明します。各関数の API リファレンスも合わせて参照してください。

初期化処理

nn::mii::CharModel::Initialize() 関数を使用して初期化を行います。

初期化用のメモリのサイズとアライメントは　nn::mii::CharModel::CalculateMemorySize() と nn::mii::CharModel::CalculateMemoryAlignment() 関数を使用することで取得できます。これらの関数に渡す CharModelInfo は初期化時に渡すものと同じものを渡す必要があります。

CharModel は初期化時に 2 種類のメモリプールを渡します。

1 つは CharModel が持つレンダーターゲットとして扱われるテクスチャ（顔輪郭テクスチャとマスクテクスチャ）を初期化するためのメモリプールです。動的テクスチャを初期化するためメモリプールなので、動的メモリプールと呼ぶこととします。

動的メモリプールのサイズとアライメントは nn::mii::CharModel::CalculateDynamicMemoryPoolSize() と nn::mii::CharModel::CalcualteDynamicMemoryPoolAlignment() 関数を使用することで取得できます。これらの関数に渡す CharModelInfo は初期化時に渡すものと同じものを渡す必要があります。動的メモリプールのプロパティには、nn::gfx::MemoryPoolProperty_CpuCached と nn::gfx::MemoryPoolProperty_Compressible を指定してください。

もう 1 つは CharModel が持つレンダーターゲットとして扱われるテクスチャ以外のテクスチャやシェイプを初期化するためのメモリプールです。これらのテクスチャやシェイプを保持するメモリプールは初期化されてから変更されることがありません。このメモリプールを静的メモリプールと呼ぶこととします。

静的メモリプールのサイズとアライメントは nn::mii::CharModel::CalculateStaticMemoryPoolSize() と nn::mii::CharModel::CalcualteStaticMemoryPoolAlignment() 関数を使用することで取得できます。これらの関数に渡す nn::mii::Resource は初期化時に渡すものと同じものを渡す必要があります。静的メモリプールのプロパティには、nn::gfx::MemoryPoolProperty_CpuCached を指定してください。nn::gfx::MemoryPoolProperty_Compressible は必要ありません。

参考：

CharModel の初期化は 1 つのメモリプールで行うことも可能です。

1 つのメモリプールで CharModel を初期化する場合は、初期化用のメモリプールのサイズとアライメントは nn::mii::CharModel::CalculateMemoryPoolSize() と nn::mii::CharModel::CalcualteMemoryPoolAlignment() 関数を使用することで取得できます。これらの関数に渡す CharModelInfo や nn::mii::Resource は初期化時に渡すものと同じものを渡す必要があります。CharModel の初期化時に渡すメモリプールのプロパティには、nn::gfx::MemoryPoolProperty_CpuCached と nn::gfx::MemoryPoolProperty_Compressible を指定してください。

1 つのメモリプールで初期化を行う場合、 nn::gfx::MemoryPoolProperty_Compressible が設定されたメモリプールのサイズが大きくなります。nn::gfx::MemoryPoolProperty_Compressible が設定されたメモリプールは初期化時のオーバーヘッドも大きくなるため、2 種類のメモリプールを渡して CharModel を初期化する方法を推奨します。

Resource の初期化時の設定によっては、Resource が保持するソースのデータは圧縮されたままの場合があります。圧縮されたままのリソースの場合、CharModel の初期化時に必要なリソースのみ圧縮の展開を行います。そのため、圧縮展開用の一時バッファが必要となります。圧縮展開用の一時バッファは専用の構造体 nn::mii::TemporaryBuffer として定義されています。

初期化時に受け取った Resource と nn::mii::CharInfo と CharModelInfo から顔モデルを描画するためのデータを構成します。初期化した内容は終了処理を呼ぶまで変更できません。

初期化時に利用したメモリ、メモリプールは終了処理を呼ぶまで保持する必要があります。一時バッファ（nn::mii::TemporaryBuffer）は初期化後に破棄することができます。

注意：

プラットフォームのメモリプールの制限を考慮した実装になっているため、Win/Nvn 環境では初期化に使用するメモリプールのサイズとアライメントが他のプラットフォーム（Win/Generic や NX 開発機）に比べて大きくなっています。具体的には、以下の関数が影響を受けます。

  • nn::mii::CharModel::CalculateStaticMemoryPoolSize()
  • nn::mii::CharModel::CalcualteStaticMemoryPoolAlignment()
  • nn::mii::CharModel::CalculateMemoryPoolSize()
  • nn::mii::CharModel::CalcualteMemoryPoolAlignment()

Win/Nvn と他のプラットフォームを同じ実装で動作させる場合には注意してください。

終了処理

nn::mii::CharModel::Finalize() 関数を呼ぶと終了処理を行います。終了処理が行われた後は初期化に使用したメモリ、メモリプールの破棄を行うことができます。

TextureView の取得

初期化済みの CharModel から顔モデルの描画に必要な nn::gfx::TextureView を取得できます。CharModel から取得できる TextureView は列挙体 nn::mii::CharModel::TextureType と関連しています。nn::mii::CharModel::GetTextureView() 関数を使用すると指定した TextureType の TextureView のポインタが取得できます。

初期化時に渡した CharInfo に設定されたパラメータによっては、TextureView が存在しない TextureType もあります。そのような TextureType を引数として渡した場合は、nullptr が返ってきます。例えば、メガネをかけていない Mii は TextureType_Glass を引数に指定した場合、nullptr が返ってきます。GetTextureView() 関数から nullptr が返ってきた場合は無視するようにしてください。

複数のマスクテクスチャを持つ CharModel の場合、マスクのインデックス番号を指定して各マスクテクスチャの TextureView を取得することができます。初期化時に使用した CharModelInfo に設定したマスクの枚数分の TextureView が取得可能です。

DescriptorSlot の設定

CharModel から取得した nn::gfx::TextureView をデスクリプタプールにセットし、取得したデスクリプタスロットを再び CharModel にセットします。セットには nn::mii::CharModel::SetTextureDescriptorSlot() 関数を使います。取得した TextureView の nn::mii::CharMolde::TextureType と同じタイプを指定してセットしてください。nn::mii::CharModel::TextureType_Mask の場合は、マスクのインデックス番号も合わせて指定してセットしてください。

顔輪郭テクスチャ

初期化済みの CharModel から顔輪郭テクスチャが取得できます。顔輪郭テクスチャの取得には nn::mii::CharModel::GetFacelineTexture() 関数を使用してください。取得したテクスチャに対して Faceline を使用して顔輪郭テクスチャをレンダリングしてください。

マスクテクスチャ

初期化済みの CharModel から初期化時に指定した枚数分のマスクテクスチャが取得できます。マスクテクスチャの取得には nn::mii::CharModel::GetMaskTexture() 関数を使用してください。取得したテクスチャに対して Mask を使用してマスクテクスチャをレンダリングしてください。

GetMaskTexture() 関数は引数として取得するマスクテクスチャのインデックスを指定できます。CharModel に複数の表情のマスクテクスチャを持たせる場合、適宜インデックスを指定してテクスチャを取得するようにしてください。

DrawParam の取得

初期化済みの CharModel から Mii の顔モデル描画するための DrawParam を取得できます。CharModel から取得できる DrawParam は列挙体 nn::mii::CharModel::DrawType と関連しています。顔モデルの描画は列挙体の順番通りに行ってください。

nn::mii::CharModel::GetDrawParam() 関数を使用すると指定した DrawType に合わせた DrawParam のポインタが取得できます。初期化時に渡した nn::mii::CharInfo に設定されたパラメータによっては、描画の必要のない DrawType も存在します。そのような DrawType を引数として渡した場合は、nullptr が返ってきます。例えば、あご髭のない Mii の場合は、DrawType_Beard を引数に指定した場合、nullptr が返ってきます。GetDrawParam() 関数から nullptr が返ってきた場合はその DrawType は描画しないようにしてください。

GetDrawParam() 関数には DrawType に加えて、顔モデルと鼻のタイプを指定する必要があります。引数に渡す顔モデルと鼻のタイプは、初期化時に使用した CharModelInfo に設定してあるタイプの必要があります。

CharModel に複数の表情のマスクテクスチャを持たせている場合、DrawType_Mask を取得する際には適切なインデックスを指定して DrawType を取得するようにしてください。

バウンディングボックス

初期化済みの CharModel から顔モデルのバウンディングボックスを取得できます。nn::mii::CharModel::GetBoundingBox() 関数を使用して取得することができます。

パーツトランスフォーム

パーツトランスフォームは Mii に装飾物をつける場合に、どの位置につければよいかの情報を保持します。nn::mii::PartsTransform 構造体として定義され、初期化済みの CharModel の nn::mii::CharModel::GetPartsTransform() 関数を使用することで取得できます。
