起動方法

NintendoSDK パッケージの以下のフォルダに含まれる、LogViewer.exe をダブルクリックして起動してください。

NintendoSDK\Tools\Log\LogViewer

画面構成

ウィンドウ構成、タブ構成およびタブごとのフィルタ式は、Log Viewer 終了時に保存されます。

①ログ表示欄

開発機から受信したログを表示します。

ログにマウスオーバーすることで、ログに付加されたメタデータを参照できます。

フィルタ式を入力していると、フィルタに適合したログのみが表示されます。

②フィルタ式入力欄

表示するログの条件を、フィルタ式で入力します。フィルタ式はタブごとに入力できます。

③タブ

ログ表示欄とフィルタ入力欄を切り替えるためのタブです。

タブを右クリックして「Rename」を選択することで、タブ名を変更できます。

タブに、ログを表示する開発機や、スレッドの名前を付けると、ログの分類に便利です。

④新規タブ

新たなタブを開きます。

⑤メニュー

ログを保存したり、サブウィンドウを開いたりできます。

サブウィンドウを閉じても、メインウィンドウが残っている限り、Log Viewer は動作し続けます。

サブウィンドウにはメニューバーがないため、メインウィンドウと見分けられます。

フィルタリング

フィルタ式を入力することで、表示されるログをフィルタリングできます。

バックグラウンドでは、フィルタに適合しないログも受信されます。そのため、フィルタ式の変更すれば、フィルタに適合しなかった、過去のログも表示できます。

演算子

フィルタ式に利用できる演算子です。

演算子 説明
( ) 括弧
! 否定
>, <, <=, >= 大小関係の比較
==, != 同値比較
=~, !~ 正規表現マッチ
&&, || AND, OR
参考：

文字列の部分一致でログを抽出したい場合は、 正規表現マッチを利用します。

同値比較は、文字列全体の一致を調べるため、他の文字列が含まれる場合は、ログを抽出できません。

オペランド

フィルタ式に利用できるオペランドです。

オペランド 説明
キー ログのデータのキーを入力します。
数値リテラル 10 進数表記のみ対応しています。
文字列リテラル ダブルクォートで囲みます。例：”文字列"
正規表現リテラル スラッシュで囲みます。例：/正規表現パターン/

フィルタリングに利用できるデータ

フィルタ式のオペランドとして利用できる、ログのデータです。

キー 型 説明
PeerName 文字列 ログを出力した開発機の名前
ProcessId 数値 ログを出力したプロセスの ID
ThreadId 数値 ログを出力したスレッドの ID
Severity 数値 ログの重大度 (割り当てる方法が未提供です）
Verbosity 数値 ログの冗長度 (割り当てる方法が未提供です）
LineNumber 数値 ログを出力した行の番号
FileName 文字列 ログを出力したファイルの名前
FunctionName 文字列 ログを出力した関数の名前
ModuleName 文字列 ログを出力したモジュールの名前 (割り当てる方法が未提供です）
ThreadName 文字列 ログを出力したスレッドの名前
TextLog 文字列 ログ本体

ログの保存

すべてのログを保存する

メインウィンドウのメニューより保存します。フィルタは適用されず、すべてのログが保存されます。

フィルタを適用してログを保存する

適用したいフィルタ式を入力したタブの、右クリックメニューより保存します。フィルタ式の条件に適合するログのみ保存されます。

ログの保存形式

プレーンテキスト形式

ログ API で出力した文字列を、プレーンテキスト形式で保存します。メタデータは含みません。

JSON 形式

ログ API で出力した文字列に加えて、ログに付加されたメタデータを、JSON 形式で保存します。

エスケープコードによるコンソール制御

文字色の変更のみ対応しています。

色指定形式は、ANSI カラーのみ対応しています。0~256 のインデックスカラー、24 bit カラーには対応していません。
