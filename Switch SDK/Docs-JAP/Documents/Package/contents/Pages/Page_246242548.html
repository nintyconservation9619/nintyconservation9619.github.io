旧 UDD ファイルからの乗り換え

UDDファイルからの移行について

NintendoSDK 0.9.0 以前にカスタムアクションを利用されていた場合は、UDD ファイルから ユーザーデータ定義ファイル に乗り換えていただく必要があります。

移行にあたっては、UDD で定義していたデータメンバーと同じ内容のユーザーデータ定義ファイルを利用するとスムーズです。
以下に、UDD ファイルにおいて全 UI を表示した状態と同等になるユーザーデータ定義ファイルを示します。このファイルの UILayout タグ内で、表示名や順番などを調整すれば、既存のUDD と同じような運用が可能です。 

UDD ファイルによるカスタムアクション付き eset を読み込む際には、あらかじめ上記のようなユーザーデータ定義を作成して読み込んでおくことで、データを引き継ぐことができます。"flags", "paramFloat0"~"paramFloat7", "paramInt0"~"paramInt5" という名前のメンバを定義しておくことで、該当するデータがマッピングされます。
