BackupSaveData

概要

BackupSaveData は開発機 SDEV / EDEV に保存されているセーブデータをバックアップおよびリストアするためのコマンドラインツールです。

パッケージ

BackupSaveData は NintendoSDK NX Addon パッケージに含まれています。

配置

$NINTENDO_SDK_ROOT/Tools/CommandLineTools/BackupSaveData.exe

なお動作のためにはこの exe ファイル以外にも依存するファイルがあります。

コマンド定義

オプション形式

BackupSaveData.exe <command> directory-path target-options [other-options]

command:
    export                        開発機のセーブデータをPCにエクスポートします
    import                        PCにあるエクスポートデータを開発機にインポートします

directory-path:                   エクスポートデータの存在するディレクトリのパスを指定します

target-options:
    --target <IP address | serial number | MAC address | name>  初期化するターゲットのIPアドレス・シリアル番号・MACアドレスもしくは名前を指定します
                                                                (ただし EDEV の場合は IP アドレス・MAC アドレスは利用できません)

other-options:
    --export-title <Application Id>  エクスポートするセーブデータを指定したアプリケーションのもののみに限定します。エクスポート時のみ有効。
    --verbose                        ログを詳細に出力します
    --version                        ツール自身のバージョンを表示します

さらに NX Addon 4.0.0 からはこちらのオプションも追加しました。

注意：

それ以前のバージョンでは利用できませんのでご注意ください。

other-options:
    --follow-exist-account       インポートする時にユーザーアカウントセーブデータを開発機上に存在するアカウントIDに変更してインポートします。インポート時のみ有効。
    --exist-account-only         エクスポートするユーザーアカウントセーブデータについて開発機に存在するアカウントのもののみに限定します。エクスポート時のみ有効。

終了コード

ツールの結果成否は終了コードで確認することができます。

0   - 正常終了
非0 - エラー終了

使い方

以下に利用ケースに応じた BackupSaveData の実行例を示していきます。

開発機に保存されているセーブデータを PC の C:\dir 以下にバックアップする

BackupSaveData export C:\dir --target 169.254.0.1

IP アドレスが 169.254.0.1 である SDEV に保存されているセーブデータを PC の C:\dir 以下にエクスポートします。
ここでは IP アドレスを指定しましたが、これ以外にシリアルナンバー・MAC アドレス・名前 を指定することが可能です。(EDEV の場合はシリアルナンバー・名前のみ指定できます)

PC の C:\dir 以下にエクスポートしたデータを指定した開発機にリストアする

BackupSaveData import C:\dir --target 169.254.0.1

PC の C:\dir 以下にエクスポートしたデータを IP アドレスが 169.254.0.1 である SDEV に書き込みます。

開発機に保存されているセーブデータのうちアプリケーションIDが 0x0100000000000000 のもののみバックアップする

BackupSaveData export C:\dir --target 169.254.0.1 --export-title 0100000000000000

エクスポートするセーブデータを特定のアプリケーションのもののみに絞りたい場合は --export-title <Application Id> をつけます。なお Application Id には 0x をつけないことに御注意ください。

エクスポートしたデータのユーザーアカウントセーブデータを開発機に存在するアカウントIDに紐づけながらリストアする

BackupSaveData import C:\dir --target 169.254.0.1 --follow-exist-account

--follow-exist-account オプションをつけると、エクスポートするセーブデータのうちユーザーアカウントセーブデータをインポート先の開発機に存在するアカウントのものに置き換えながらインポートします。次章の「ユーザーアカウントが異なる場合はインポートしてもセーブデータにアクセスできません」に対する対応ができるようになります。

開発機に保存されているセーブデータのうち利用可能なアカウントのセーブデータのみバックアップする

BackupSaveData export C:\dir --target 169.254.0.1 --exist-account-only

--exist-account-only オプションをつけると、ユーザーアカウントセーブデータに関して開発機に存在するアカウントのもののみエクスポートするようにします。
開発機には既に存在しないアカウントのセーブデータが残っている場合がありますので、それをエクスポート対象から外すことができるようになっています。
--follow-exist-account オプションを付けたインポートが失敗する場合はこの機能を利用してエクスポートしたデータをインポートすると改善することがあります。

仕様制約

ユーザーアカウントが異なる場合はインポートしてもセーブデータにアクセスできません

BackupSaveData ではエクスポート時とインポート時でユーザーアカウントが同一である必要があります。そのためエクスポート時のユーザーアカウント ID が存在しない状態でインポートを行うと、インポートには成功するものの(アクセスできるアカウントがないため)そのインポートしたセーブデータにアクセスできなくなるという状態になります。特にこのようなケースがそれにあたります。

  • エクスポート後に開発機を初期化した場合
  • 別の開発機でインポートを行った場合

このような場合は以下の手順でインポートすることで利用できるようになります。(ただし、この方法は今後廃止になる可能性もありますので、その点はご了承ください。)

  1. エクスポート時と同数のユーザーアカウントを作成する
  2. アプリケーションを起動してセーブデータを作成する
  3. BackupSaveData でセーブデータをエクスポートする
  4. 手順3で生成されたエクスポートデータにインポートしたいエクスポートデータを上書きする
  5. 手順4でできたエクスポートデータを使って BackupSaveData でインポートする
注意：

なお、以下の場合にインポートに失敗する不具合は改善しております。

  • ジャーナリング領域サイズよりセーブデータに保存されたファイルの総サイズの方が大きい場合
  • ジャーナリング領域サイズより大きいサイズのファイルがセーブデータ内に存在する場合

また NX Addon 4.0.0 からは上記手順を行わなくても --follow-exist-account オプションをつけながらインポートすることでも利用できるようになりました。

注意事項

  • このツールがバックアップできるのはアプリケーションのセーブデータです。システムの設定値はバックアップしません。それをバックアップしたい場合は SettingsManager をご利用下さい。
  • エクスポートしたデータを改変した場合の動作保障は致しません。
  • DevMenu でもセーブデータのバックアップ機能が提供されています。そちらを用いると開発機のみでバックアップ・リストアを行うことができます。
