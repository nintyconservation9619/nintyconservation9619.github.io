はじめに

本ドキュメントの目的

本ドキュメントでは、NintendoSDK に収録されている時刻を扱うライブラリ（以降、 TIME ライブラリ）について説明します。

対象読者

NintendoSDK を使ってアプリケーションを開発される方を対象としています。

関連資料

nn::time:: 名前空間の関数リファレンス と併せて参照してください。

利用上の注意 も必ず参照してください。

TIME ライブラリが提供する機能

TIME ライブラリは、NintendoSDK 環境における時刻を扱うための API を提供します。

TIME ライブラリを利用することによって、ユーザがデバイスに設定した時刻の取得や、ネットワークを使って補正された時刻の取得などを提供します。また、ある時点での時計のコンテキストをアプリケーションのセーブデータ等に保存し、後に比較することによって、アプリケーションのシャットダウンをまたいだ経過時間の計測や、時計操作の有無の検知を行うことができます。

さらに、POSIX 時間を人間が理解できるカレンダー表記と相互に変換する機能を持ちます。

現在時刻の取得機能を提供するクラス

  • nn::time::StandardSteadyClock クラス
    • SoC 外部に電源を持つ、一定のペースで単調増加する時計です。
    • このクラスから得られる nn::time::SteadyClockTimePoint を人間の理解できる日時に変換することはできません。
    • 時刻の変更・補正の影響を受けずに時間経過を計算することができます。
    • セーブデータ等を利用することで、アプリケーションのシャットダウンをまたいだ経過時間を計測することができます。
  • nn::time::StandardUserSystemClock クラス
    • ユーザーが本体設定を介して操作可能な時計を表す時計です。
      • 本体機能に表示される時刻は、このユーザー時計のものです。
    • このクラスから得られる nn::time::PosixTime を人間の理解できる日時に変換することができます。
    • 指し示す時刻が、実際の時間以上に進んだり、過去へ巻き戻ったりすることがあります。
      • 本体設定でユーザーが時刻を自由に操作できます。
      • 本体設定の「インターネットで時間をあわせる」がONの場合には、システムによる自動補正の影響を受け、ネットワークから自動補正される時計と同じ時刻に同期されます。
      • アプリケーションの実装の簡略化のため、任意のタイミングでこのクラスの時刻に揃えることができる nn::time::AdjustableUserSystemClock も用意しています。
    • セーブデータ等を利用することで、アプリケーションのシャットダウンをまたいでこの時計が操作されたかどうかを検知することができます。
  • nn::time::StandardNetworkSystemClock クラス
    • ネットワークから自動補正される時計です。デバイスがネットワークを利用可能なとき、自動的にシステムが補正を行います。未補正の状態では時刻を得ることができません。
    • このクラスから得られる nn::time::PosixTime を人間の理解できる日時に変換することができます。
    • システムによる自動補正の影響を受け、指し示す日時が実際の時間経過以上に進んだり過去へ巻き戻ったりすることがあります。
      • アプリケーションの実装の簡略化のため、任意のタイミングでこのクラスの時刻に揃えることができる nn::time::AdjustableNetworkSystemClock も用意しています。

現地時刻である nn::time::CalendarTime との変換機能

  • nn::time::ToCalendarTime
    • nn::time::PosixTime を nn::time::CalendarTime へ変換します。
  • nn::time::ToPosixTime
    • nn::time::CalendarTime を nn::time::PosixTime へ変換します。

時計の設定について

DevMenu での設定

時計の設定は DevMenu の Device ページ、 Time Test ページにて行うことができます。 Time Test ページについては 各種テスト機能 を参照してください。

nn::time::StandardUserSystemClock の設定

 以下の項目を設定できます。

  • 現在時刻
  • 自動補正の有効性
    • "Automatic Correction" のチェックボックスで指定してください。
    • チェック状態が自動補正有効、チェックされていない状態が自動補正無効です。

nn::time::StandardNetworkSystemClock の設定

ネットワークへの接続を契機に、自動的に有効化されます。

テスト目的で時刻を操作するには Time Test ページで行います。

タイムゾーンの設定

Time Zone 項目の "Edit" ボタンで、各タイムゾーンを設定できます。

DevMenuCommand での設定

DevMenuCommand でも各時計の日時を設定可能です。
