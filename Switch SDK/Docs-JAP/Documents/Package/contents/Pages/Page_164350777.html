ランタイムオーバービュー

プログラムの流れ

レイアウトライブラリをプログラムに組み込む方法は、プログラミングガイドの「クイックスタート」を参考にしてください。

諸注意

ペインのプログラムからの制御

ペインの各パラメータはレイアウトエディタから設定したり、アニメーションを付けたりすることができますが、プログラム側でも、ペインを表すクラスのメソッドを用いることによって制御することができます。
例えば、Paneクラスの SetTranslate によって、ペインの位置を動かすことができます。

これによって、例えばプレイヤーがタッチした位置にペインを動かす等のインタラクティブな挙動を実現することができます。

描画ステートについて

レイアウトライブラリは、ブレンド設定、ロジックオペレーション設定を内部で指定します。

それ以外は、Draw メソッドで描画する前に設定したものがそのまま使われますのでご注意ください。
最も影響を受ける設定はカリングとデプステストです。
標準的な方法では、カリングなし、デプステストなしで描画しますが、特殊な表現をしたい場合にはデプステストを使用する場合があります。

マルチスレッドでの動作

レイアウトライブラリは、ライブラリ内部でスレッド間の同期を取るような処理は行っていません。
つまり、内部でスレッドのロック/アンロックは行いません。

このため、レイアウトライブラリをマルチスレッドで動作させるためには、スレッド毎に別々のインスタンスを用いる必要があります。
例えば、別々の nn::ui2d::Layout クラスであれば、それぞれ別のスレッドで同時に Animate を行ったり、Draw を行ったりすることができます。
この 際、Draw に与える nn::ui2d::DrawInfo クラスも別々のインスタンスである必要があります。

例外として、nn::ui2d::GraphicsResource クラスのインスタンスは、異なるスレッドで共通で使うことができます。
たとえば、アプリ内でグローバルに一つの nn::ui2d::GraphicsResource インスタンスを持ち、それを複数の nn::ui2d::DrawInfoに SetGraphicsResource しても、正常に動作します。

レイアウトについてさらに知るためには

  • パフォーマンスガイドは最適化に関する有用な情報が記載されていますので、目を通すことをお勧めします。
  • LayoutEditor のマニュアルは、デザイナにはもちろん、レイアウトで何ができるかを把握しておきたいプログラマにとっても有用です。
