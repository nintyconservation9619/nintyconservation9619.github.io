ディスプレイ

概要

ディスプレイオブジェクトは、機能の問い合わせおよび物理ディスプレイで使用するレイヤの作成のためのインターフェースです。なお、このオブジェクトは複数の物理ディスプレイにマッピングできます。

デフォルトのディスプレイ

デフォルトのディスプレイが提供する機能は、使用するプラットフォームに応じて異なります。NX では、アプリケーションから利用できるディスプレイはデフォルトのディスプレイのみであり、コンテンツは必ずこの正しいディスプレイに送られます。可能な出力パス（HDMI と LCD）の間でコンテンツを切り替える操作は、アプリケーションが受け持つ役割の範囲にはありません。

参考：

アプリケーションでは、解像度が変更された場合のメリットを活用するのでなければ、動作モードが変更されたときにコンテンツを再作成する必要はありません。

ディスプレイのポリシー

アプリケーションが開くことができるディスプレイ上で使用できる機能には制限があります。アプリケーションが特に配慮を必要とする点は、ディスプレイプロセスに渡すことのできるレイヤの数です。さらに作成できるレイヤのサイズにも制限があります。

参考：

制限内容はディスプレイごとに異なります。

DisplayInfo 構造体にはディスプレイの制限事項が詳しく記述されています。レイヤの数が制限されている場合は、作成できるレイヤの最大数が　layerCountMax に記述されます。レイヤのサイズはlayerWidthPixelCountMax layerHeightPixelCountMax の値に制限されます。

これらの制限内容はプラットフォームのライフサイクルを通じて変更されることがない予定です。
