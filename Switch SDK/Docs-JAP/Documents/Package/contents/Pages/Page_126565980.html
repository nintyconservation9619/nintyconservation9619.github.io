atk の主な変更点

メモリプールへの対応

atk が管理する一部のメモリ領域について、AUDIO ライブラリが提供するメモリプール機能に対応しました。
ただ、一部のメモリ領域につきましては、アプリケーション側でメモリプールを管理してください。 

詳しくは、プログラミングガイドの メモリー管理 のドキュメントを参照ください。

atk がメモリプールを管理するデータ

メモリ領域の確保や破棄を atk のサウンドシステムで管理するデータについては、適宜メモリプールのアタッチ・デタッチが行われます。

  • サウンドヒープ、プレイヤーヒープ
  • WSD, STRM, SEQ の ADPCM コンテキスト情報

atk がメモリプールを管理しないデータ

アプリケーション側でメモリ領域の確保や破棄を管理するデータについては、メモリプールを管理してください。

  • ストリームバッファ
  • エフェクトバッファ
  • ウェーブサウンドの StartInfo に指定する波形データのアドレス
