プログラマ向け変更点

NintendoSDK 0.15.0 で行われた NintendoWare オーディオ (nn::atk) の変更点について説明します。

目次

  • 各チャンネルの音量を直接指定する API の追加
  • I3DL2 Reverb エフェクトの追加
  • レンダリング結果を取得する API の追加
  • エフェクトの有効・無効フラグの初期値の変更
  • ドキュメントの改訂

各チャンネルの音量を直接指定する API の追加

各チャンネルの音量にはパンニングのパラメータを指定し間接的に設定する方法のみ提供していましたが、各チャンネルの音量を直接追加する機能を追加しました。

詳しくは AtkMixMode サンプル やプログラミングガイドの ミックス - 各チャンネルの音量の直接指定 の項目、以下の API リファレンスをご確認ください。

  • nn::atk::SoundHandle::SetMixMode 
  • nn::atk::SoundHandle::SetMixVolume

I3DL2 Reverb エフェクトの追加

I3DL2 Reverb エフェクトを追加しました。また、 AtkEffect サンプル で適用するエフェクトに I3DL2 Reverb を加えました。 
詳しくは nn::atk::EffectI3dl2Reverb の API リファレンスをご確認ください。

レンダリング結果を取得する API の追加

オーディオレンダラのレンダリング結果を取得する機能を追加しました。

取得方法や取得したサンプルデータの使用方法など、詳しくはプログラミングガイドの クイックスタート - レンダリング結果の取得 の項目、以下の API リファレンスをご確認ください。

  • nn::atk::SoundSystem::ReadCircularBufferSink 
  • nn::atk::SoundSystem::GetCircularBufferSinkBufferSize 
  • nn::atk::SoundSystem::GetRendererSampleCount 
  • nn::atk::SoundSystem::GetRendererChannelCountMax 

エフェクトの有効・無効フラグの初期値の変更

これまではエフェクトの有効・無効フラグは有効で初期化されていましたが、
エフェクトを一度削除し、再度追加するようなフローにおいては、2 回目の追加時に明示的に有効化する必要があり、
1 回目の追加時と必要な初期化処理が異なっていため、常に明示的に有効化する必要があるようにフラグの初期値を無効に変更しました。

これまで、各エフェクトの SetEnabled() 関数で明示的に true を設定していなかった場合には、エフェクトがかからなくなる点にご注意ください。

ドキュメントの改訂

以下の項目を修正しました。

  • サウンドシステムマニュアルのサウンドパラメータ制御の動作の「サウンドパラメータの種類と処理方法の対応一覧」の表の内容に誤りがあったため修正しました。
  • nn::atk::SoundSystem::SetOutputMode の API リファレンスで、出力モードの初期値の内容に誤りがあったため修正しました。
