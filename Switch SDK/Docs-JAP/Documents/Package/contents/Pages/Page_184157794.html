ステーションの接続制限

概要

ネットワークの構築で説明した通り、アクセスポイントで nn::ldn::CreateNetwork() を使用してネットワークを構築するとき、nn::ldn::NetworkConfig の nodeCountMax に指定した値によってネットワークに接続できるステーションの数を制限することができます。

しかし、LDN ライブラリではより詳細にステーションの接続を制限できる機能を提供しています。

ステーションの接続受入ポリシー

既定では、ネットワークに接続できるステーション数が上限に達していない限り、全てのステーションからの接続を受け入れます。このポリシーを変更するためには nn::ldn::SetStationAcceptPolicy() を使用して nn::ldn::AcceptPolicy を設定します。この API は nn::ldn::OpenAccessPoint() を使用してアクセスポイントとしての役割を開始した nn::ldn::State_AccessPoint 状態、あるいは nn::ldn::CreateNetwork() でネットワークを構築した nn::ldn::State_AccessPointCreated 状態で使用できます。

以下では、nn::ldn::AcceptPolicy の詳細を説明します。

AcceptPolicy_AlwaysAccpet

特に設定しない限り、新しいネットワークでは nn::ldn::AcceptPolicy_AlwaysAccept が使用されます。このポリシーは、全てのネットワークに接続できるステーション数が上限に達していない限り、全てのステーションからの接続を受け入れることを意味します。

nn::Result result = nn::ldn::SetStationAcceptPolicy(nn::ldn::AcceptPolicy_AlwaysAccept);
if (nn::ldn::ResultDeviceNotAvailable::Includes(result))
{
    nn::err::ShowError(result);
}

AcceptPolicy_AlwaysReject

ある時点でネットワークへのステーションの接続を締め切りたい場合には nn::ldn::AcceptPolicy_AlwaysReject を使用します。このポリシーは、あらゆるステーションからの接続を拒否します。

nn::Result result = nn::ldn::SetStationAcceptPolicy(nn::ldn::AcceptPolicy_AlwaysReject);
if (nn::ldn::ResultDeviceNotAvailable::Includes(result))
{
    nn::err::ShowError(result);
}

AcceptPolicy_BlackList

特定のステーションの接続を禁止したい場合には nn::ldn::AcceptPolicy_BlackList を使用します。

nn::Result result = nn::ldn::SetStationAcceptPolicy(nn::ldn::AcceptPolicy_BlackList);
if (nn::ldn::ResultDeviceNotAvailable::Includes(result))
{
    nn::err::ShowError(result);
}

ブラックリストにステーションを登録するには nn::ldn::AddAcceptFilterEntry() を登録します。ただし、ブラックリストに登録した時点で接続中のステーションが自動的に切断されることはありませんので、必要に応じて nn::ldn::Reject() を併用してください。

nn::Result AddToBlackList(const nn::ldn::NetworkInfo& network, int index) NN_NOEXCEPT
{
    NN_ASSERT_RANGE(index, 1, nn::ldn::NodeCountMax);
    nn::Result result = nn::ldn::AddAcceptFilterEntry(network.ldn.nodes[index]);
    if (result.IsSuccess())
    {
        result = nn::ldn::Reject(network.ldn.nodes[index].ipv4Address);
    }
    return result;
}
 
nn::Result result = AddToBlackList(network, 1);
if (nn::ldn::ResultDeviceNotAvailable::Includes(result))
{
    nn::err::ShowError(result);
}
else if (nn::ldn::ResultNodeNotFound::Includes(result))
{
    // nn::ldn::Reject() 以前にステーションが切断されていました。
    // ブラックリストへの登録は成功しているため、以後、そのステーションが接続することはありません。
}

AcceptPolicy_WhiteList

特定のステーション以外の接続を禁止したい場合には nn::ldn::AcceptPolicy_WhiteList を使用します。ホワイトリストにステーションを登録するには nn::ldn::AddAcceptFilterEntry() を登録します。

例えば、以下のサンプルでは、ある時点でステーションの新規接続を打ち切って、その時点で接続しているステーションの再接続のみを許可します。

nn::ldn::NetworkInfo network;
nn::Result result = nn::ldn::GetNetworkInfo(&network);
if (result.IsSuccess())
{
    result = nn::ldn::SetStationAcceptPolicy(nn::ldn::AcceptPolicy_WhiteList);
    if (result.IsSuccess())
    {
        for (int i = 1; i < nn::ldn::NodeCountMax && result.IsSuccess(); ++i)
        {
            const auto& node = network.ldn.nodes[i];
            if (node.isConnected)
            {
                result = nn::ldn::AddAcceptFilterEntry(network.ldn.nodes[index]);
            }
        }
    }
}
 
if (nn::ldn::ResultDeviceNotAvailable::Includes(result))
{
    nn::err::ShowError(result);
}
