テクスチャコンバイナ、カラー合成設定に、比較 ( 暗 ) ,比較 ( 明 ) を追加しました。

新しいマテリアルブレンド「比較 : 暗」,「比較 : 明」が追加され、表現の幅が広がりました。

プロパティウインドウ内に追加された「比較 ( 暗 ) 」と「比較 ( 明 ) 」

比較 ( 暗 )

比較 ( 暗 ) は、基本色と合成色、2 つの重なり合う色を比較して、暗い色の方が適用される設定です。

  • 合成した色に比べて、基本色が暗い場合、基本色がそのまま残ります。
  • 合成した色に比べて、基本色が明るい場合、色が置き換えられます。
例を用いた、比較 ( 暗 ) の説明
A テクスチャ 0 画像 「テクスチャ 0 」に、黄色 1 色の絵を設定します。（基本色）
B テクスチャ 1 画像 「テクスチャ 1 」に、白～黒のグラデーションの絵を設定し、比較 ( 暗 ) を適用します。
C

結果画像

比較 ( 暗 )

 「テクスチャ 1 」の白色部分よりも「テクスチャ 0 」の黄色部分が暗いため、「テクスチャ 0 の黄色」がそのまま残ります。
「テクスチャ 1 」の黒色部分よりも「テクスチャ 0 」の黄色部分が明るいため、「テクスチャ 0 の黄色」が置き換えられます。

比較 ( 明 )

比較 ( 暗 ) は、基本色と合成色、2 つの重なり合う色を比較して、明るい色の方が適用される設定です。

  • 合成した色に比べて、基本色が暗い場合、色が置き換えられます。
  • 合成した色に比べて、基本色が明るい場合、基本色がそのまま残ります。
例を用いた、比較 ( 明 ) の説明
A テクスチャ 0 画像 「テクスチャ 0 」に、茶色 1 色の絵を設定します。（基本色）
B テクスチャ 1 画像 「テクスチャ 1 」に、白～黒のグラデーションの絵を設定し、比較 ( 明 ) を適用します。

「テクスチャ 1 」の黒色部分よりも「テクスチャ 0 」の茶色部分が明るいため、「テクスチャ 0 の茶色」がそのまま残ります。

テキストボックスの斜体傾きが、角度で指定できるようになりました !!

テキストボックスの斜体傾きが角度で指定できるようになり、より細やかな表現が可能となりました。
従来の比率設定より、より直感的に設定できるようになりました。

プロパティウインドウに追加された「斜体傾き」項目
斜体傾き ( 角度指定 ) : フォントを傾ける角度を入力します。
「 -60.00 ～ 60.00 」の範囲で設定できます。
( 比率 ) : フォントに傾きを与えることで変更された、比率を表示します。
「斜体傾き ( 角度指定 ) 」で設定した数値と連動しています。

例 : テキストボックスのフォントに「 30° 」の傾きを設定

今まで OpenType フォントが持つ GPOS テーブルのカーニング情報の読み込みには対応していませんでしたが、描画に反映されるようになりました。

 

BC7 フォーマットがサポートされるようになりました !!

BC7 フォーマットを利用できるように拡張されました。

LayoutEditor のテクスチャウィンドウから指定できます。

BC7 は、RGBA カラーフォーマットで、1ピクセルあたり、1バイトのテクスチャフォーマット(RGBA8 の1/4、BC1 の倍)となっています。
BC3 に比べると、圧縮ノイズが少なく高品質な表示となっています。
一方で、変換に長い時間がかかる（特に大きなサイズの場合）のでご留意ください。
