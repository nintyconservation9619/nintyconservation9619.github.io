パッケージ構成

NintendoSDK のパッケージ

NintendoSDK は複数のパッケージに分割され配信されています。

これにより、必要な開発環境のみを取得することができ、不必要な機能を使ってしまうリスクを回避することが出来ます。

例：

  • NintendoWare を使わないことを確実にするために、NintendoWare をインストールしない

パッケージ一覧

通常パッケージである NintendoSDK では以下のパッケージを提供しています。

基本
NintendoSDK

アプリケーションの開発に必ず必要なライブラリやツールです。

このパッケージが無い場合、アプリケーションをビルドすることはできません。
NintendoSDK Samples

NintendoSDK のサンプルです。
NintendoWare
NintendoSDK NintendoWare 3D・2Dグラフィックス、エフェクト、サウンドの開発ができるミドルウェアです。 詳しくは、 NintendoWare を参照してください。 NintendoSDK パッケージに依存しています。 
NintendoSDK NintendoWare Samples NintendoWare のサンプルです。

パッケージ内容

NintendoSDK の全てのパッケージは同じディレクトリにインストールされ、以下の構成になっています。

NintendoSDK/ Build/
アプリケーション開発用の Visual Studio プロジェクトテンプレートなどが格納されています。

Common/
アプリケーションのビルドに必要な情報が格納されています。 

Compilers/
特定環境で必要となるコンパイラが格納されています。

Documents/ API/

API リファレンスが格納されています。


Licenses/ NintendoSDK が依存している NintendoSDK 外のライブラリなどのライセンス表記が格納されています。


Package/ その他ドキュメントが格納されています。


index.html ドキュメントのインデックスページです。

Externals/
NintendoSDK が依存している NintendoSDK 外のライブラリなどが格納されています。

Include/
ヘッダーファイルが格納されています。

Installers/
特定環境で必要となるソフトウェアのインストーラーが格納されています。

Libraries/
ライブラリファイルが格納されています。

Resources/
特定の環境やサンプルで必要なリソースデータが格納されています。

Revisions/
NintendoSDK パッケージを区別するリビジョン情報が格納されています。

Samples/
NintendoSDK のサンプルが格納されています。

Sources/
ソースコード (提供可能なもののみ) が格納されています。
ソースコードを改編した場合、動作の保証はできません。改変したライブラリを製品で使用する場合は十分に検証を行ってください。

TargetTools/
特定環境で入る実機向けのツール（実行ファイル）が格納されています。

Tools/
ホスト PC およびターゲット開発機用のツールが格納されています。

NintendoSdkRootMark
NintendoSDK のルートディレクトリを示すファイルです。 NintendoSDK が提供するビルドシステム内で参照されます。
