3D ランタイムの主な変更点

アニメーションおよびテクスチャの関連付けを行う関数の返り値の変更

アニメーションおよびテクスチャの関連付けを行う関数の返り値を変更しました。今までの返り値は成功、失敗のみでしたが、新しい返り値では以下の状態を取得出来るようになります。

  • 完全に成功
  • 完全に失敗
  • 一部成功
  • 一部失敗


変更があった関数は以下になります。

  • nn::g3d::BoneVisibilityAnimObj::Bind()
  • nn::g3d::MaterialAnimObj::Bind()
  • nn::g3d::MaterialAnimObj::SubBind()
  • nn::g3d::ShapeAnimObj::Bind()
  • nn::g3d::ShapeAnimObj::SubBind()
  • nn::g3d::SkeletalAnimObj::Bind()
  • nn::g3d::SkeletalAnimObj::BindFast()
  • nn::g3d::ResBoneVisibilityAnim::PreBind()
  • nn::g3d::ResMaterialAnim::PreBind()
  • nn::g3d::ResMaterialAnim::BindTexture()
  • nn::g3d::ResSceneAnim::Bind()
  • nn::g3d::ResLightAnim::Bind()
  • nn::g3d::ResFogAnim::Bind()
  • nn::g3d::ResShapeAnim::PreBind()
  • nn::g3d::ResSkeletalAnim::PreBind()
  • nn::g3d::ResFile::BindTexture()
  • nn::g3d::ResMaterial::BindTexture()

ui2d/font ランタイムの主な変更点

nn::font の UTF-8 版機能の追加

今までの UTF-16 版機能はそのままに、新たに UTF-8 版の文字列に対応した機能を追加しました。

1 文字単位で操作する関数の引数や戻り値は、UTF-16 のサロゲートペア領域の文字にも対応するために、uint16_t から uint32_t に拡張する仕様変更を行っています。

詳しくは「ui2d/font の UTF-8 対応版機能について」のページを参照してください。

nn::font::TextureCache::CompleteTextureCache() の引き数変更

nn::font::TextureCache::CompleteTextureCache() が CommandBuffer 引数を要求しなくなりました。関数は、内部で描画コマンドを発行しなくなりましたので、呼び出しタイミングの制約がなくなりました。

また、この修正に関連して内部バッファの容量がほぼ半減しています。

ユーザーシェーダー更新コールバックにユーザーデータを渡す機能を追加

ユーザーシェーダー更新コールバックにユーザーデータを渡す機能を追加しました。
この変更により nn::ui2d::UpdateUserShaderCallback および nn::ui2d::DrawInfo::SetUpdateUserShaderCallback にユーザーデータ用の引数が増えていますのでご注意ください。

その他API 変更

以下の互換性の無いAPI 変更を行っています。ビルドエラーとなりますので、お手数ですが修正をお願いいたします。

  • スケーラブルフォントの TextureCache の初期化方法の変更

    • 複数のフォントを登録する際に、今までは可変長にしていたために初期化コードが複雑になっていましたが、固定長の配列にしたことで簡潔になりました。

    • 詳しくは、FontSimple サンプルおよび「ペアフォントおよびマルチスケーラブルフォントについて」のページを参照してください。

  • nn::ui2d::viewer::Viewer::Initialize() の引き数を InitializeArg に統合
