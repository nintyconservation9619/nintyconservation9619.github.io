WaveConverter

はじめに

WaveConverter は、任意の波形ファイル（*.wav, *.aiff）をリソースファイル（*.bfwav, *.bfstm）に変換するためのコマンドラインツールです。
WaveConverter で作成したリソースファイルは、 nn::atk ライブラリで利用することができます。

本ドキュメントでは、WaveConverter の使い方と機能について説明します。

WaveConverter の使い方

実行方法

WaveConverter を使うには、コマンドラインから以下の形式で実行します。

WaveConverter.exe  [オプション] <入力ファイルパス>

例）

WaveConverter.exe --format=bfwav --encoding=adpcm yoshi.wav
WaveConverter.exe --format=bfstm --encoding=pcm16 -o bgm.adpcm.bfstm bgm.txt

コンバートが成功すると、--output オプションで指定したファイルパスにリソースファイル（*.bfwav, *.bfstm）が出力されます。
--output オプションを省略した場合は、入力ファイルパスの拡張子違い（*.bfwav, *.bfstm に変更したもの）として出力されます。

コマンドライン引数

入力ファイルパス

入力波形ファイルパスを指定します。
-format=bfstm を指定した場合には、波形リストファイルパスを指定することでマルチトラックストリームバイナリファイルを出力することもできます。

波形リストファイルとは、下例のように波形ファイルパスを改行で区切ったテキストファイルです。

例）

piano.wav
bass.wav
drum.wav

コマンドラインオプション

オプション

説明

-h / --help

ヘルプメッセージを表示し、終了します。

-v / --verbose

細かい内部動作を表示します。
--format <format-type>

出力する波形バイナリファイルフォーマットを bstm, bfwav から指定します。

このオプションは必ず指定する必要があります。
--encoding <encoding-type>

出力エンコーディングを pcm16, adpcm から指定します。

このオプションは必ず指定する必要があります。
-o, --output <output-filepath> 出力ファイルパスを指定します。
省略した場合は、入力ファイルパスの拡張子違い（*.bfwav, *.bfstm に変更したもの）として出力されます。
--loop-start <frame>

ループ開始フレームを指定します。ループさせない場合は、このオプションを指定しないでください。
--loop-end <frame>

ループ終了フレームを指定します。--loop-start を指定して --loop-end を省略すると、

--loop-start に指定した位置から波形終端までのループを設定します。
--stream-prefetch ストリームプリフェッチバイナリファイルを生成します。–format=bfstm を指定した場合にのみ有効です。
--stream-buffer-scale <arg>

ストリームプリフェッチバイナリ向けのストリームバッファ倍率を指定します。
--format=bfstm かつ --stream-prefetch を指定した場合にのみ有効です。
