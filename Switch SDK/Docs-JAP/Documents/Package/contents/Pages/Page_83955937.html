DCC優先モード

DCC ツールのエクスポートプラグインでは、サンプラマージの設定を DCC ツール優先か、3DEditor 優先かを変更することができます。

参考：

DCC ツールのエクスポートプラグインのサンプラマージ設定については、別途用意されている 各DccPlugin のドキュメント内の[エクスポートプラグイン] → [コンフィグファイル]にある sampler_merge_priority を参照してください。

サンプラマージ設定が有効（DCCツール設定が優先）だった場合に、3DEditor ではマテリアルプロパティウィンドウのサンプラページの UI 仕様を変更することができます。

ここでは、サンプラマージ設定が有効の場合に対応した UI への変更方法と操作について説明します。

サンプラページの UI 仕様の変更方法

マテリアルプロパティウィンドウのサンプラページの UI 仕様の変更は、チーム設定ファイル （3dEditor.teamsettings) の <Preset> 内にある <FollowDccSamplerNameRule> の true / false で行います。

3dEditor.teamsettings は下記場所にあります。

"（NIntendoSDKインストール場所）\NintendoSDK\Tools\Graphics\3dEditor\3dEditor.teamsettings"

参考：

チーム設定ファイル （3dEditor.teamsettings) の <Preset> 内にある <FollowDccSamplerNameRule> のデフォルトは、false になっています。
DCC ツールのエクスポートプラグインで、サンプラマージの設定を DCC ツール優先にした場合は、 <FollowDccSamplerNameRule> を true に変更してください。

3dEditor.teamsettings 内の <Preset>　の詳細については、こちらを参照してください。

true サンプラマージ設定が有効な場合のUI 仕様
false 従来の UI 仕様

サンプラマージ設定が有効時の操作方法

ここでは、サンプラマージ設定が有効な場合のサンプラページの操作方法について説明します。

  • サンプラの新規追加 / サンプラ名の入力変更
  • DCC ツールで設定され同期されているサンプラの表示
  • サンプラマージ設定が有効時のコピー & ペースト
  • サンプラマージ設定が有効時（DCCツールが優先）の注意

サンプラの新規追加 / サンプラ名の入力変更

サンプラマージ設定が有効な場合のサンプラ名の入力、サンプラの新規追加について説明します。

A

サンプラの新規追加

追加ボタンを押して、サンプラの新規追加をすると、デフォルト名が sampler0 となります。2つ目以降は、sampler1, sampler2 ... と続きます。

_a1, _a2 のように名前の前にアンダーバーがつくものは追加されません。
B

サンプラ名を自由入力か、コンボボックスから設定　

サンプラ名の入力と変更

サンプラ名の入力と変更について説明します。

自由入力時 （ユーザーが任意にサンプラ名を付ける場合）

・　サンプラ名を自由に入力できますが、_ 始まりの名前は入力できません。（例：_a0 など）
・　DCC ツールからエクスポートされて、既に_a0 のようにアンダーバーから始まるサンプラ名で、ヒントが設定されているサンプラの名前を変更すると、ヒントを空にしてサンプラ名のアンダーバーが外れます。
（例：hint = albedo0、name = _a0 のサンプラ名を a0 に変更すると hint なし、name = a0 のサンプラになります）

コンボボックスから任意のサンプラ名を選択して変更

任意のサンプラ名入力候補をコンボボックスに設定して表示できます。

参考：

コンボボックスに表示されるサンプラ名は、チーム設定ファイル （3dEditor.teamsettings) の <Preset> 内にある <SamplerNamePresets> でサンプラ名候補を任意に設定できます。

関連リンク　3dEditor.teamsettings 内の <Preset>　の詳細については、こちらを参照してください。

DCC ツールで設定され同期されているサンプラの表示

DCC ツールで設定され同期しているサンプラには、一覧表示のサンプラの名前に DCC マークが表示されます。

参考：

・DCCツールと同期しているサンプラは、サンプラパラメータ以外は編集できません。
・DCCツールと同期しているサンプラ名には、名前の先頭に "_" ( アンダーバー ) が付きます。

サンプラマージ設定が有効時のコピー & ペースト

コピー & ペーストと、マテリアルインポート時のサンプラ割り当ての挙動は同じになります。

名前の先頭にアンダーバーが付いていて、ヒントがあるサンプラ （DCCツールと同期しているサンプラ） をコピーしてペーストする場合

【具体例】

サンプラ名 _a0 でヒントが albedo0 のサンプラをコピーしてペーストすると下記状態でコピーされます。

サンプラ名のアンダーバーが自動的に削除され、ヒントは空になります。また、DCC ツールとの同期も外れます。

アンダーバーを自動的に削除する際に、既に同じ名前のサンプラがあった場合は、サンプラ名をナンバリングしてペーストします。
上記の場合に、a0 というサンプラが既に存在していた場合は、a1 という名前でペーストされます。

参考：

ペースト先に同名で、同ヒントを持つサンプラがあった場合は、サンプラパラメータのみ上書きされます。

DCC ツールと非同期のサンプラをコピーしてペーストする場合

  • サンプラ名とヒント共に、そのまま維持してペーストされます。
  • サンプラ名が重複する場合は、テクスチャデータ / サンプラパラメータ ( wrap , filter , lod ) 共に上書きされます。
  • ペースト先に新規作成したサンプラのヒントが、既存サンプラのヒントと重複する場合は、新規作成したサンプラのヒントを空にします。
    ※既存サンプラのヒントを優先します。

サンプラマージ設定が有効時（DCCツールが優先）の注意

サンプラマージ設定が有効な場合に、オリジナルヒントが空でないテクスチャ SRT のシェーダーパラメータは編集できません。
