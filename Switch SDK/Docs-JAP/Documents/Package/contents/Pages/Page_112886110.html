SD カード

機能概要

デバッグ・開発用途限定で、 FAT または exFAT にフォーマットした SD カードをマウントする機能を提供します。

警告：
  • 本機能を製品環境で使用することはできません。
    • 詳細については FS ライブラリ／開発向けオーバービュー の「デバッグ API の利用について」を参照してください。
  • 名前にマルチバイト文字を含む SD カード上のファイルやディレクトリを取り扱うことはできません。

サンプルプログラム

サンプル解説書の「デバッグ用途の SD カードカードマウント機能」をご確認ください。

SD カードのマウント

nn::fs::MountSdCardForDebug() を呼び出すことで、 SD カードをマウントすることが出来ます。

ジャーナリング機能をサポートしません。nn::fs::Commit() を実行する必要はありません。

開発機環境での注意事項

NX プラットフォームでビルドしたアプリケーションでは、開発機の SD カードスロットに挿入された SD カードがマウントされます。

SD カードは FAT または exFAT でフォーマットされている必要があります。

Windows 環境での注意事項

Windows 環境では、アプリケーションの exe ファイルと同じディレクトリにある sdcard ディレクトリを代替領域としてマウントします。このディレクトリが無い場合、自動で作成されます。

ホスト PC に挿入された SD カードがマウントされるのではない点にご注意ください。

参考：

将来的に、代替とするディレクトリのパスは指定可能になる予定です。

パス

  • ディレクトリの区切り文字には '/' または '\' を使用します。
  • エントリ名に '<'、'>'、'*'、'?'、':'、'|'、'"' は使用できません。
  • タブや改行などの制御文字は使用できません。
  • 英字の大文字小文字は区別されません。
  • マルチバイト文字には対応していません。

ファイルのタイムスタンプ

参考：

SD カード上のファイルのタイムスタンプは NX Addon 4.1.0 以降から残るようなります。

FAT

作成日時のタイムスタンプ精度は1秒単位です。

最終更新日時のタイムスタンプ精度は2秒単位です。1秒以下の情報は捨てられます。

最終アクセス日時のタイムスタンプ精度は1日単位です。時、分、秒の情報は捨てられます。

exFAT

作成日時のタイムスタンプ精度は1秒単位です。

最終更新日時のタイムスタンプ精度は1秒単位です。

最終アクセス日時のタイムスタンプ精度は2秒単位です。1秒以下の情報は捨てられます。


参考：

SD カード上のファイルのタイムスタンプは精度が異なるため、最終更新日時や最終アクセス日時が作成日時より前の日時にみえる場合があります。

例えば、FAT フォーマットされた SD カードに、2018/1/1 1:23:45 にファイルを作成した場合、以下のようにタイムスタンプが残ります。

  • 作成日時 2018/1/1 1:23:45
  • 最終更新日時 2018/1/1 1:23:44
  • 最終アクセス日時 2018/1/1 0:00:00
注意：

NX プラットフォームでは、SD カードへの書き込みを抑制するために、読み込みアクセスをしても最終アクセス日時を更新しません。
