入力の「WindowsFont」タブ

WindowsFont

以下は「WindowsFont」タブの UI 説明です。
入力時のフォントファイルとして WindowsFont を利用できます。

A フォント名

変換元とするフォントを指定します。

一覧にはPCにインストールされているフォントが表示されます。

背景色がツールチップの背景色となっているものはラスタフォント(ビットマップフォント)で、通常の背景色のフォントはベクタフォント（アウトラインフォント）です。
B 本体内蔵フォント（bfttf）をインポート

本体内蔵フォント（bfttf）を指定します。

ファイル設定ボタンを押し、選択ダイアログで bfttf を選びます。
C サイズ

変換元フォントのサイズをピクセル単位で指定します。

実際に出力されるフォントのサイズは各フォントに依存するため、数値は参考程度に考えてください。
D 平均幅を指定

変換元フォントの平均幅を指定を有効にします。有効にすることにより平均幅を指定して縦横比率を変更できます。
E 平均幅

変換元フォントの平均幅をピクセル単位で指定します。数値は参考程度に考えてください。
F カーニング抽出元に nn::fontll を使う

｢bffnt の描画｣と ｢ttf/otf の描画」（例：ttfなどbffnt のソースフォントを nn::font、nn::fontll、nn::ui2d で描画する場合など）で、微妙な表示差異が問題になる場合に利用します。

デフォルトはOFFになっています。
G 階調数

出力する階調数を指定します。

括弧内にはbffntで出力する場合に使用されるテクスチャフォーマットを表しています。
H ソフトアンチエイリアス

多階調出力(2階調以外の出力)で行うグリフへにアンチエイリアス処理をよりやわらかくします。

フォントによっては出力されるグリフの品質が改善することがあります。
 I 文字幅 出力するフォントの文字幅をどのように算出するかを指定します。
関連リンク 詳細についてはこちらを参照してください。
 J 幅の指定

「文字幅」の指定で「全ての文字で同じ幅を使用する」を選択した場合のみ入力できます。

全ての文字で共通の幅をピクセル単位で指定します。
 K 太さ（ウェイト）

文字の太さを指定します。

0～900で指定し、標準は 400 で Bold は 700 です。
参考：

Unicode の0x0000～0ｘFFFDの中でフォントに存在する文字が出力側に渡されます。 出力側がImageの場合は文字順序ファイルにより出力文字が制限されますが、bffntの場合は文字フィルタを指定しないとフォントに存在する多くの文 字が出力されることになります。フォントによっては数万文字が出力されます

  • 階調数 出力する階調数を指定します。
    括弧内にはbffntで出力する場合に使用されるテクスチャフォーマットを表しています。
  • 非推奨オプション
    非推奨オプションでは、階調数が減少しますがフォントのメモリサイズは変わりません。
    後方互換性のために残されていますので、特別な狙いがない限りは、利用しません。

 

  • 文字幅
     出力するフォントの文字幅をどのように算出するかを指定します。
    • グリフ幅を文字幅とする
      文字幅がグリフ幅と等しくなるようにします。
      すなわちグリフの前後にスペースが入らないようにします。必然的にプロポーショナルになります。

      空白文字に関しては文字幅が0として出力され、そのまま使用すると空白文字が存在しないかのように表示されてしまいます。
      空白文字を使用する場合は、代わりに「グリフ幅を使用する (空白文字の保護)」設定で出力するか、一旦画像ファイルに出力のうえ、空白文字の文字幅を調整して使用します。
    • グリフ幅を使用する (空白文字の保護)
      基本的に「グリフ幅を文字幅とする」と同様ですが、空白文字については「フォントで指定される文字幅を使用する」を指定した場合と同じ文字幅になります。
    • フォントで指定される文字幅を使用する
      各フォントが持つ文字幅をそのまま使用します。結果、プロポーショナルのフォントであればプロポーショナルに等幅のフォントであれば等幅になります。
    • 全ての文字で同じ幅を使用する
      グリフの前後に適宜スペースを取る事で全ての文字で文字幅が同じになるようにします。
      この全ての文字で共通の幅は次の「幅の指定」で指定します。出力は等幅になります。
