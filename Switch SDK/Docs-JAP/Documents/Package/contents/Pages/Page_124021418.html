ターゲット環境変数（NX Addon）

概要

NX アプリケーションでも、Windows プラットフォームと同じターゲット環境変数を参照することができます。

ターゲット環境変数の利用方法

警告：

機能を利用する上で重要な制限があります。NintendoSDK のターゲット環境変数の「制限」も必ずご一読ください。

ターゲット環境変数定義ファイルの準備

NintendoSDK のターゲット環境変数に記載の方法でターゲット環境変数定義ファイルを準備します。

ターゲット環境変数定義ファイルを指定してアプリケーションを実行

NX アプリケーションでは、アプリケーション実行時にターゲット環境変数定義ファイルを指定する方法が次のようになります。
Windows アプリケーションと同様、実行されたアプリケーションで nn::htc::GetTargetEnvironmentVariableLength()、 nn::htc::GetTargetEnvironmentVariable() 等の API を使用して、ターゲット環境変数の値を取得できます。

VSI によるアプリケーションの実行

プロジェクトプロパティの デバッグ＞Target Environment Variable File に、準備したターゲット環境変数定義ファイルを指すパスを指定して実行します。

Target Manager による NX アプリケーションの実行

Target Manager のでアプリケーションを実行する際の、「実行ファイルのロード」ウィンドウの Target Environment Variable File に準備したターゲット環境変数定義ファイルを指す絶対パスを指定して実行します。

RunOnTarget による NX アプリケーションの実行

RunOnTarget の "--env" オプションに、準備したターゲット環境変数定義ファイルを指すパス（絶対パスでも相対パスでも可）を指定して実行します。

$ RunOnTarget.exe --env ./app.tdf ./app.nsp

インストールした NX アプリケーションの実行

インストールしたアプリケーションでのターゲット環境変数の利用は現時点ではサポートされていません。今後の更新でサポートされる予定です。

制限

NintendoSDK のターゲット環境変数に記載の制限に加え、以下の制限があります。

  • 本機能は Windows 及び NX 開発機でのみ利用可能です。NX 製品機では利用できません。
    • NX 製品機に置いては、ターゲット環境変数関連の API の実行はすべてエラーが返ります。
  • ターゲット環境変数の取得にはホスト PC との通信が必要になるため、処理にかかる時間に関する保証はありません。そのため、リアルタイム性が要求される場面でのターゲット環境変数の取得処理の呼び出しは避けてください。
