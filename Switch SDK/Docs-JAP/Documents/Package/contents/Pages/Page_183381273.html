ネットワーク識別子

ネットワーク識別子 (nn::ldn::NetworkId) は、ネットワークの目的を表す nn::ldn::IntentId と、セッション毎に生成される nn::ldn::SessionId から構成されています。

IntentId

nn::ldn::IntentId はネットワークの目的を表す 128 ビットの識別子であり、ローカル通信識別子 (LocalCommunicationId) とシーン識別子 (SceneId) から構成されています。周囲のネットワークをスキャンした際に、スキャン結果のフィルタリングに nn::ldn::IntentId を使用すれば、他のアプリケーションが作成したネットワークや、同じアプリケーションであっても異なるシーンで作成されたネットワークをスキャン結果から省くことができます。

ローカル通信識別子はアプリケーションタイトルやシリーズを識別する目的で使用します。原則として、ローカル通信識別子にはアプリケーション ID を使用します。ただし、アプリケーション ID が異なる他のアプリケーションと通信を行う場合、通信するアプリケーション間で共通のローカル通信識別子を使用する必要がありますので、どちらか一方のアプリケーション ID を双方で使用することができます。アプリケーションが使用するローカル通信識別子は全てアプリケーション管理データに登録しておく必要があり、登録されていない場合には nn::ldn::CreateNetwork や nn::ldn::Connect() が事前条件違反で失敗します。なお、特に設定を行っていない場合は、既定値として自身のアプリケーション ID がローカル通信識別子としてアプリケーション管理データに登録されています。

シーン識別子は「通信対戦」「データ交換」ようなアプリケーションの異なるシーンを区別することを目的としており、アプリケーション開発者が自由に設定することができます。

以下に示す通り、nn::ldn::IntentId は nn::ldn::MakeIntentId() を使用して生成します。

const nn::Bit64 localCommunicationId = 0x01004B9000490000; // Application ID
const nn::Bit16 sceneId = 0;
nn::ldn::IntentId intentId = nn::ldn::MakeIntentId(localCommunicationId, sceneId);


SessionId

nn::ldn::SessionId はアクセスポイントがネットワークを構築するたびにランダムに生成される 128 ビットの識別子であり、ネットワークが破棄されるまで変化することはありません。
