インストリスト

インストリスト

バンクタブは上下に分割されて表示されます。この上の領域をインストリストと呼びます。


インストリストの役割

インストリストはインストルメントの作成と、プログラム番号の指定を行います。インストリストは、シーケンスがバンクを参照する際の音色セットとして機能します。
リージョンをひとつしか持たないシングルリージョンのインストルメントについては、インストリストのみでインストルメントの編集も行うことができます。

インストルメントの作成

バンクタブがアクティブな状態で、[プロジェクト]→[アイテムの追加]→[インストルメント] を実行します。

インストリストで、右クリックを行うと現れるコンテキストメニューによる操作でも、インストルメントを新規作成することができます。


新規作成時のデフォルト状態でのオリジナルキーについて、指定した波形ファイルのファイル名や付加情報に基づいて、自動的に適切と思われる値に設定されます。
（例えば "Test.A3.aif" というファイル名の "A3" という部分）

ファイル名に書かれるノートの表記は、シーケンスコマンドの規則に従います。 設定される順序は、以下のようになります。

  1. 波形ファイル名に、キーを表す文字列があれば、それをオリジナルキーに変換します。
  2. キーを表す文字列が見つからない場合は、波形ファイル内のオリジナルキー情報を取得します。
  3. 2.に失敗したときは、cn4 とします。

エンベロープの適用対象の選択

エンベロープについての解説はインストルメントのパラメータを参照してください。

適用対象の設定は、インストルメントごとに個別に行うことができます。
切り替えは、インストリストのコンテキストメニューによって行います。

リージョン単位からインストルメント単位への切り替え

 

インストルメント単位からリージョン単位への切り替え

 

フォルダを開く・参照ファイルを開く

コンテキストメニューの「フォルダを開く」または「参照ファイルを開く」を選択することで、そのファイルに対するそれぞれの動作を行えます。

プログラム番号の振り直し

コンテキストメニューの「プログラム番号の振り直し」を選択すると、インストリストの選択されているアイテムについて、プログラム番号を振り直すことができます。

 

プログラム番号の振り直しダイアログが表示されるので、起点となる番号を入力します。

 

既に存在する番号と重複する場合には、その旨のアラートダイアログが表示されます。

「プログラム番号の振り直し」を行うことで、後から同系列のものを並べて見やすくしたり、変数を使ったプログラムチェンジ指定をする場合に、既に登録済みのインストを意図した並びにして行うことができます。

インストリストの編集

ここではインストリストの各定義要素の意味を解説します。

プログラム番号

シーケンスからバンクが参照される際に、プログラムチェンジの番号に対するインストルメントの割り当てを指定します。
MIDI のプログラムチェンジでは 128 個のプログラム番号しか指定できませんが、テキストシーケンスによる指定も考慮し、インストリストではこの制限を越えたプログラム番号を指定することができます。

名前

インストルメントの名前を指定します。

名前に使用できる文字には制限があり、以下の文字が使えます。

  • 先頭は、半角英字とアンダーバー [a-zA-Z_]
  • 先頭以外は、半角英数字とアンダーバー [a-zA-Z0-9_]

圧縮方式

コンバータが参照波形ファイルをコンバートする際の圧縮方法を指定します。

  • ADPCM
    波形ファイルを専用の ADPCM 形式にコンバートします。
  • PCM16
    16bit の PCM データにコンバートします。

ファイルパス

インストルメントが参照する波形ファイルを指定します。
シングルリージョンのインストルメントについてのみ、インストリストからこの指定を行うことができます。マルチリージョンのインストルメントの場合には、この項目は "*" と表示され、編集を行うことはできません。

ファイル名、およびパス名に使用できる文字には制限があり、以下の文字が使えます。

  • ファイル名の先頭は、半角英字とアンダーバー [a-zA-Z_]
  • ファイル名の先頭以外およびパス名は、半角英数字とアンダーバー [a-zA-Z0-9_]

再生時間

サウンドの再生時間（波形データ長の時間）を表示します。
表示単位は［分：秒．ミリ秒］で、「ミリ秒」は 3 桁目を四捨五入し、2 桁表示となります。

ループ付き波形データでは、波形そのものの長さでは無く、ループエンドまでの時間を表示します。
また、セルの先頭に "L" が表示されます。

ティック数

必要なノートコマンドの長さを表示します。
ティックは、テキストシーケンスでノートや休符の長さを指定するための単位です。

ティック数は、テンポや四分音符分解能の値に依存して値が変わりますが、テンポのデフォルト値 120 (1 分間あたり四分音符 120 回) および、テキストシーケンスの四分音符分解能のデフォルト値 48 (これはプロジェクト設定の「SMFの四分音符分解能」で設定するものとは異なります)で考えた場合、

  • 1 分 ＝ 120 * 48      ＝ 5760 ティック
  • 1 秒 ＝ 120 * 48 / 60 ＝   96 ティック

にそれぞれ相当します。

テキストシーケンスを書く際に、ティック数がわかると有用です。

ただし、インストの音程を変えた場合、実際の波形再生時間が長くなったり短くなったりしますが、これにあわせて実際の再生ティック数も変化してしまうので、注意が必要です。

ダウンサンプル

ダウンサンプルの有効、無効を指定します。

有効にすると元の波形ファイルを変更することなく「サンプルレート」に指定された値でダウンサンプルしてサウンドアーカイブに含めることができます。

サンプルレート

「ダウンサンプル」が無効の場合は、波形ファイルのサンプルレートを表示します。

「ダウンサンプル」が有効の場合は、ダウンサンプルする値を 2000 ～ 192000 Hz の範囲で指定します。

ビットレート

波形ファイルのビットレートを表示します。

量子化ビット数

波形ファイルの量子化ビット数を表示します。

チャンネル数

波形のチャンネル数が 1, 2 のときは Mono, Stereo と表示します。 チャンネル数が 3 以上の時は、数字をそのまま表示します。

ダウンミックス

ダウンミックスの有効、無効を指定します。

有効にすると元の波形ファイルを変更することなくステレオをモノラルにダウンミックスしてサウンドアーカイブに含めることができます。

データサイズ

圧縮変換後の波形データのサイズを表示します。
マルチリージョンのインストルメントなど、複数の波形ファイルを参照する場合は、参照ファイルの合計サイズを表示します。
表示内容は、コンバートのタイミングで更新されます。

オリジナルキー

参照する波形ファイルのオリジナルキー（鍵盤のどの位置で波形ファイルが元のピッチで鳴るか？）を指定します。
シングルリージョンのインストルメントについてのみ、インストリストからこの指定を行うことができます。マルチリージョンのインストルメントの場合には、この項目は "*" と表示され、編集を行うことはできません。

Env A

エンベロープのアタックタイムを 0 〜 127 で指定します。
インストルメント単位にエンベロープの適用対象を設定している時にのみ、インストリストからこの指定を行うことができます。適用対象がリージョン単位に設定されている時には、この項目は "*" と表示され、編集を行うことはできません。
この動作は、以降のエンベロープの各フェーズの指定も同様です。

Env H

エンベロープのホールドタイムを 0 〜 127 で指定します。

Env D

エンベロープのディケイタイムを 0 〜 127 で指定します。

Env S

エンベロープのサスティンレベルを 0 〜 127 で指定します。

Env R

エンベロープのリリースタイムを 0 〜 127 で指定します。

音量

インストルメントの音量を 0 〜 255 で指定します。

サウンドデータの音量を 127 にすると、元データと等倍の音量 (1.0倍) で再生されます。

ただし、音量は、いろいろな場面 (データやプログラム) で設定されます。 すべての設定値が掛け合わされた結果、元音量から比べて、0.0 倍 〜 2.0 倍にクランプされます。 詳しくは、サウンドシステムマニュアルの「サウンドパラメータ制御の動作」の章をご覧ください。

ピッチ（半音）

インストルメントのピッチを半音単位で指定します。

ピッチ（セント）

インストルメントのピッチをセント単位（半音の 1/100）で指定します。

補間タイプ

インストルメント単位で補間タイプを指定します。

ボイスの補間方法を設定し、「4 点補間」,「線形補間」が選べます。 デフォルト値は 4 点補間 です。

現状、こちらの補間タイプの設定を変更しても、補間方法には反映されませんのでご了承ください。
