LineFeed や LineFeedOffset での高さ計算の不具合の修正

LineFeed や LineFeedOffset で行間を変更していたときに、1 行のテキストを描画するときに高さの計算がおかしくなっていました。

左側は修正前で、右側が修正後です。
参考：

本修正によって、描画結果がこれまでと変わることがあります。

LineFeed や LineFeedOffset を変更していなかった場合は影響ありません。

また、2 行のテキストを描画している場合も影響ありません。

スケールを維持したままフォントを差し替える機能の追加

スケールを維持したままフォントを差し替える機能を追加しました !!

プロジェクト設定ファイルで以下を記述すると、フォントを差し替えたときにスケールが維持されるようになります。

<KeepingSizeScaleEnabled>true</KeepingSizeScaleEnabled>

サムネールの最大サイズを指定する機能の追加

サムネールの最大サイズを指定する機能を追加しました !!

プロジェクト設定ファイルで以下を記述すると、サムネール画像の最大サイズが 256x256 になります。

<SaveThumbnailImage>true</SaveThumbnailImage>
<ThumbnailMaximumWidth>256</ThumbnailMaximumWidth>
<ThumbnailMaximumHeight>256</ThumbnailMaximumHeight>

キャプチャテクスチャ機能を追加

フレームバッファやレイアウトのレンダリング結果をキャプチャして テクスチャとして使用する機能を追加しました!!

プロジェクト設定ファイルで機能を有効にすると、メインメニュー[ ペイン ] → [ ペイン作成 ]から「キャプチャペイン」が選択できます。
キャプチャペインを選択すると、「親子階層：ウィンドウに「 キャプチャペイン」が作られます。
「親子階層」ウィンドウで「 キャプチャペイン」階層下のペインは通常のペイン描画は行われません。
キャプチャペインが生成するテクスチャでのみ描画されます。

キャプチャペインが生成したテクスチャは「テクスチャ」ウィンドウに「█ 黄色帯」で通常テクスチャと区別して表示されます。
このテクスチャは、通常のテクスチャ同様、ペインの「プロパティウィンドウ」→「 テクスチャ」に登録して利用できます。

キャプチャペインプロパティ

A

レイアウトウィンドウ上「█キャプチャ領域」のフレームバッファをテクスチャとして取り込むかを設定します。
B

テクスチャを初回フレームに一度だけ生成するかを設定します。
「 キャプチャペイン」階層下のペインをアニメーションさせたい場合は「 OFF」にします。
C キャプチャペインの解像度を最大として、設定した割合でスケールをかけてテクスチャを生成します。
D

以下のケースでテクスチャカラーを設定します。

  • フレームバッファのキャプチャを行わない場合
  • キャプチャ領域がフレームバッファのテクスチャサイズからはみ出た場合

特にアルファ値に 0 を設定することで、生成するテクスチャのアルファ初期値が 0 となります。
その結果「 キャプチャペイン」階層下にあるペイン外側の透過部を、生成するテクスチャでは透明にできます。

キャプチャペインは、フレームバッファの特定領域をテクスチャとしてキャプチャできます。
マテリアルに「FramebufferCaptureTexture」を設定することでゲームアプリケーションのフレームバッファをキャプチャできます。
「FramebufferCaptureTexture」はマテリアルのテクスチャ選択リストに常に表示されています。

「FramebufferCaptureTexture」ではキャプチャペインのような細かな設定はできません。
マテリアルを使用しているペインの領域のフレームバッファテクスチャをキャプチャします。

LayoutEditor で「背景画像」を設定するとフレームバッファテクスチャとして扱われるため、キャプチャの挙動を疑似的に確認できます。

関連リンク 詳細は「キャプチャテクスチャについて」を参照してください。


以下はキャプチャペインを利用した表現の一例です。

LayoutEditor LayoutViewer

CapturePane : 「 キャプチャペイン」階層下に「 テキストボックスペイン」や「 ピクチャペイン」を配置した例です。

Use Texture : キャプチャペインによって生成されたテクスチャを「 ピクチャペイン」で利用しています。

以下は利用方法に関する解説です。

A 「 ピクチャペイン」の詳細コンバイナ設定で、テキストの中にテクスチャのパターンを流しています。

B 「ピクチャペイン」にマスク用テクスチャを登録し、テクスチャ座標をアニメーションすることでテキストを左右から消しています。

C フラグメントオペレーションで加算合成した結果をキャプチャし、「 ピクチャペイン」でマスクしています。
参考：

【親子関係に関する制限】
キャプチャテクスチャに依存関係が発生するような構造の親子関係は作成できません。
例えばキャプチャペインの子供にキャプチャペインで作成したテクスチャを使用しているピクチャペインを配置するような構造です。

【回転、スケールに関する制限】
キャプチャテクスチャを作成するペインに回転やスケールが設定されているとペインの形状に沿ってキャプチャすることができません。
