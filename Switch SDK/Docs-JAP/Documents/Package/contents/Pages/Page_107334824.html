概要

ControlTarget は開発機の操作と、開発機の情報取得を行うコマンドラインツールです。

RunOnTarget と組み合わせて自動テストに必要な開発機の制御に必要な機能を提供します。

パッケージ

ControlTarget は、NintendoSDK NX Addon で提供されます。

配置

$NINTENDO_SDK_ROOT/Tools/CommandLineTools/ControlTarget.exe

対応プラットフォーム

NX プラットフォームに対応しています。

共通の引数

-t, --target 引数

制御対象となるターゲットを指定します。

指定に使用する値は、 シリアル番号、IP アドレス、  set-name サブコマンドにより設定した名前です。

指定可能なターゲットは list-target サブコマンド、または、 detect-target サブコマンド で見つけることができます。

-v, --verbose 引数

コマンドの詳細な動作を出力します。

サブコマンド

reset

開発機をリセットし再起動します。

自動テストや開発機に異常が発生している場合に使用します。

--target オプションを使用しない場合は、デフォルトの開発機をリセットします。

--hard オプションを使用した場合、開発機と開発機を制御するシステムを合わせて再起動します。

電源の入っていない EDEV に対しては、このコマンドを使用することはできません。

power-on, power-off

開発機の電源を制御します。

自動テスト等で開発機の電源を制御を含んだテストを行う場合に使用できます。

EDEV に対する power-on は非サポートです。

terminate

開発機で実行中のアプリケーションを終了します。

set-default

指定された開発機をデフォルトの開発機として指定します。

ControlTarget および、RunOnTarget で開発機を明示的に指定しない場合はデフォルトとして指定された開発機を使用します。

get-default

デフォルトの開発機を取得します。

set-name

Target Manager 上での開発機の名前を変更します。
用途に応じた名前を付けることで識別しやすくすることが出来ます。

デフォルトでは、開発機のシリアル番号、または、MACアドレスが開発機の名前となっています。

以下のように名前が設定されます。

> ControlTarget.exe list-target
40-D2-8A-B4-73-4B       192.168.10.11

> ControlTarget.exe set-name SdevForAutoTest

> ControlTarget.exe list-target
SdevForAutoTest 192.168.10.11

また、名前の指定を無しに set-name を呼ぶことで、 MACアドレスか、シリアル番号の名前を再設定することができます。

> ControlTarget.exe list-target
SdevForAutoTest 192.168.10.11

> ControlTarget.exe set-name

> ControlTarget.exe list-target
40-D2-8A-B4-73-4B 192.168.10.11

connect, disconnect

Target Manager から開発機に接続と切断を行います。

RunOnTarget や ControlTarget は必要なときに自動的に接続に行くため、このコマンドを直接使用する機会はほとんどありません。

HTCS により提供されているサービスのみを使用するときなどに使用できます。

list-target

TargetManager に登録されている開発機をリストアップします。

ControlTarget や RunOnTarget で操作する開発機を探すときに使用できます。

list-target は detect-target とは異なり TargetManager に登録されているもののみをリストアップするため、普段から使用している開発機の名前を確認する場合に使用できます。

detect-target

ネットワーク内で見つかるすべての開発機をリストアップします。

ControlTarget や RunOnTarget で操作する開発機を探すときに使用できます。

detect-target は list-target とは異なり、ネットワーク内にあるすべての開発機をリストアップするため、新しく使用する開発機を探す場合に使用できます。

find-target

ネットワーク上のリミテッドブロードキャストが届く範囲内で見つかるすべての開発機をリストアップします。

ControlTarget や RunOnTarget で操作する開発機を探すときに使用できます。

ネットワークの構成に依存しますが、detect-target とは異なり、サブネットをまたいで設置されている開発機のリストアップに使用できます。

list-service

HTCS のサービスをリストアップします。

サービスを提供している開発機とサービスへアクセスするための IPアドレスとポート番号を取得できます。

% ControlTarget list-service
XAWF0111100014  @Log            127.0.0.1:52258
XAWF0111100014  @cs             127.0.0.1:52259
XAWF0111100014  @InputDirector  127.0.0.1:52260

register

Target Manager に開発機を登録します。

通常、Target Manager の GUI 上で行う操作のため、自明な用途はありません。
自動化するときに使用することが出来ます。

ControlTarget および、 RunOnTarget では、 "--target" 引数を用いて、登録されていない開発機を操作しようとした場合、TargetManager に開発機を自動で登録するため、このコマンドを使う必要はありません。

unregister

TargetManager に登録されている開発機の登録を解除します。

通常、Target Manager の GUI 上で行う操作のため、自明な用途はありません。
自動化するときに使用することが出来ます。

unregister-all

TargetManager に登録されているすべて開発機を登録解除します。

自動テスト等で多数の開発機を登録した後などのクリーンアップの処理として使用することができます。

wait-detect

開発機が検出されるのを待ちます。

--timeout オプションで指定した秒の間、開発機が現れるのを待ちます。

自動テストなどで ControlTarget の reset --hard コマンドを使用した場合に、対象の開発機が再びネットワーク上に現れるまで自動操作を待機する場合に使用できます。

set-ip

開発機のIPアドレスを変更します。

IPアドレスを静的に割り当てる

$ ControlTarget.exe set-ip --target 40-D2-8A-35-41-00 192.168.10.11 --netmask 255.255.255.0
Discovering target...
Network setting has been changed to:
    IP address:      192.168.10.11
    Subnet mask:     255.255.255.0
Change Internet Protocol Version 4 (TCP/IPv4) Properties to "Use the following IP address".
Discovering target...
Discovering target...
Done.

Change Internet Protocol Version 4 (TCP/IPv4) Properties to "Use the following IP address". というメッセージが表示されたら、PC のネットワークデバイスの IPアドレスを "次のIPアドレスを使う" に変更し、開発機と通信可能な IPアドレスを設定してください。

--netmask オプションを合わせて指定してください。

IPアドレスを自動取得する

$ ControlTarget.exe set-ip --target 40-D2-8A-35-41-00 auto
Discovering target...
Network setting has been changed to:
    IP address:      auto
    Subnet mask:     auto
    Default gateway: auto
Change Internet Protocol Version 4 (TCP/IPv4) Properties to "Obtain an IP address automatically".
Discovering target...
Discovering target...
Done.

Change Internet Protocol Version 4 (TCP/IPv4) Properties to "Obtain an IP address automatically". というメッセージが表示されたら、PC のネットワークデバイスのIPアドレスを "IPアドレスを自動的に取得する”に変更してください。

オプション

名前 説明 
--target <arg>

開発機の名前を指定します。

開発機の名前は ControlTarget.exe detect-target で確認できます。

 
--netmask <arg>
 開発機のサブネットマスクを設定します。IPアドレスを静的に割り当てる場合必須です。 
--gateway <arg>

開発機のデフォルトゲートウェイを指定します。

 
--primary-dns <arg>

開発機のプライマリDNSを指定します。

 
--secondary-dns <arg>

開発機のセカンダリDNSを指定します。

enable-cradle

disable-cradle コマンドで無効化された SDEV クレードルの接続状態を有効化します。

disable-cradle

SDEV クレードルの接続状態を無効化します。

無効化された状態では、SDEV クレードルが接続されていても接続されていないように扱われます。

enable-cradle コマンドを呼び出すことで再び有効化することができます。

また、SDEV の電源ケーブルを抜いた場合も有効な状態に戻ります。

press-power-button

Power ボタンを一定時間押して離す操作を行います。押してから離すまでの時間を引数に指定することができます。

こちらの機能を使うと、Power ボタンを1秒押してスリープにする、などの操作をエミュレーションすることが出来ます。

こちらの機能は SDEV のみ使用可能で、EDEV では使用できません。

set-battery-level

NintendoSDK NX Addon 0.12 以降で使用可能です。

電池残量レベルを変更します。引数 <BatteryLevel> に enough か no-boost を指定できます。初期状態は enough です。

no-boost を指定すると、電池残量が僅かで SDEV クレードルを接続してもブーストモードに遷移できない状態をエミュレーションすることができます。

この状態については 動作モード の「動作モードと性能モードの関係」および 性能選択 の「性能モード」を参照してください。

こちらの機能は SDEV のみ使用可能で、EDEV では使用できません。

take-screenshot

NintendoSDK NX Addon 5.2 以降で使用可能です。

現在実行中のアプリのスクリーンショットを保存します。
出力先のフルパスを --full-path で指定するか、出力先のディレクトリとファイル名を --directory, --file で指定してください。
出力される画像のフォーマットは 32bit bmp です。

create-target-program-dump

NintendoSDK NX Addon 5.2 以降で使用可能です。

現在実行中のアプリの nxdmp ファイルを作成、保存します。
出力先のフルパスと、ダンプの種類を指定してください。

種類として指定できるのは、full と quick です。

DevMenuCommand 系コマンド共通の引数

引数として、DevMenuCommand Arguments を指定可能な全てのサブコマンドは、内部では DevMenuCommand を呼び出しています。

機能の詳細については、DevMenuCommand を参照してください。

-t, --target 引数

DevMenuCommand 系のコマンドを実行するターゲットを指定します。

NintendoSdk 5.3 以降で追加されています。

-d, --devmenucommand 引数

DevMenuCommand.nsp へのパスを指定します。

-r, --runontarget 引数

RunOnTarget.exe へのパスを指定します。 

DevMenuCommand Arguments 引数

この機能は NintendoSdk 5.3 以降で追加されています。

DevMenuCommand には随時機能が追加されていきますが、ControlTarget ではサポートが追い付いていない機能がある場合があります。

このときも以下のような形式で追加の引数を渡すことで DevMenuCommand にある機能を使える場合があります。この例では、install-application コマンドに --force オプションを渡しています。

$ ControlTarget install-application SomeApplication.nsp -- --force

devmenu

この機能は NintendoSdk 5.3 以降で追加されています。

DevMenuCommand には随時機能が追加されていきますが、ControlTarget ではサポートが追い付いていない機能があります。

この場合にも、 devmenu サブコマンドを使用すると DevMenuCommand のすべての機能を RunOnTarget 等を介さずに呼び出すことができます。

例えば以下のようにコマンドを指定します。

$ ControlTarget devmenu -- application install D:/SomeApplication.nsp --force

firmware-version

ファームウェアバージョンの確認をします。

$ ControlTarget.exe firmware-version
NintendoSDK Firmware for NX 0.1.0
[SUCCESS]

list-application

 アプリケーションのリストアップをします。

$ ControlTarget.exe list-application
1 application(s) found.
    id: 0x1000000000010000, version: 0
[SUCCESS]

launch-application

アプリケーションの起動をします。

$ ControlTarget.exe launch-application 0x1000000000010000
[SUCCESS]

install-application

アプリケーションのインストールをします。

アプリケーションのインストール方法については、 NX アプリケーションを開発機にインストールして実行する もあわせて参照してください。 nsp ファイルについては、アプリケーションイメージの作成 や 環境構築ガイド を参照してください。

$ ControlTarget.exe install-application "C:\application.nsp"
Preparing to install C:\application.nsp...
[SUCCESS]

uninstall-application

アプリケーションのアンインストールをします。

$ ControlTarget.exe uninstall-application 0x1000000000010000
[SUCCESS]

list-savedata

セーブデータのリストアップをします。

セーブデータの作成については、FS ライブラリ／マニュアル／機能説明／セーブデータ や FS ライブラリのリファレンスを参照してください。

$ ControlTarget.exe list-savedata
id: 0x1000000000010000
[SUCCESS]

delete-savedata

セーブデータの削除をします。

$ ControlTarget.exe delete-savedata 0x1000000000010000
[SUCCESS]

delete-all-savedata

セーブデータの全削除をします。

$ ControlTarget.exe delete-all-savedata
delete id: 0x1000000000010000
[SUCCESS]

count-pairing-controller

コントローラペアリング設定のカウントをします。

$ ControlTarget.exe count-pairing-controller
2
[SUCCESS]

delete-all-pairing-controller

コントローラペアリング設定の削除をします。

$ ControlTarget.exe delete-all-pairing-controller
2 controller paring settings are deleted.
You need to restart the device to enable editing.
[SUCCESS]

batch-devmenu

コマンドのバッチ実行をします。

batch サブコマンドを使うことで、DevMenuCommand の各サブコマンド実行を一括で行なうことができます。
DevMenuCommand については、 開発機の制御／DevMenuCommand を参照してください。

まず最初に以下のような、DevMenuCommand のサブコマンド以降を列挙したテキストファイルを用意します。ファイル名は仮に batch.txt としておきます。 行頭に # もしくは // を置くことでコメント行とすることが出来ます。

// アプリケーションのインストール
application install "C:\application1.nsp"
application install "C:\application2.nsp"
application install "C:\application3.nsp"
application install "C:\application4.nsp"

上記のテキストファイルを batch コマンドで指定します。

$ ControlTarget.exe batch-devmenu c:\batch.txt
Preparing to install C:\application1.nsp...
Preparing to install C:\application2.nsp...
Preparing to install C:\application3.nsp...
Preparing to install C:\application4.nsp... 
[SUCCESS]
