作業サイクルを高速化したい

インゲーム編集ワークフローでは、ゲーム用のバイナリデータなどを再作成する手間を介さずに、エフェクトをゲーム内で直接確認、編集するためエフェクト作成の作業サイクルが大幅に短縮されます。

ゲームになじむエフェクト編集をしたい

インゲーム編集ワークフローでは、最初からゲーム内でエフェクトを 新規作成、編集 するため、エフェクトビューア上では不可能なゲームになじむエフェクトを作成することができます。

「インゲーム編集」は、エフェクトをゲームアプリケーション上で確認しながら作成・調整できる仕組みです。

これまでの一般的なワークフロー

一般的にこれまでのワークフローでは EffectViewer でエフェクトを確認しながら EffectMaker でデータを作成します。
ゲーム要素に合わせた確認はゲームアプリケーション上で行いますが、修正する場合は以下のフローを何度も往復しなければなりません。

EffectMaker と EffectViewer EffectViewer でエフェクトを確認しながら EffectMaker でデータを作成、調整します。
バイナリコンバート EffectMaker で一旦データを保存し、バイナリコンバートすることでゲームに反映させます。
この間、待ち時間が発生します。
ゲーム

エフェクトがゲーム上で正しく表示されているかを確認します。

インゲーム編集のワークフロー

「インゲーム編集」を利用したワークフローでは、 エフェクト作成時からゲームアプリケーション上で確認しながら調整できます。
 EffectMaker の編集内容は即ゲームアプリケーションに反映される為、効率的に短時間でエフェクトを完成させることができます。

EffectMaker と ゲーム

EffectViewer に代わり、ゲーム上でエフェクトを確認しながら EffectMaker でデータを作成、調整します。

インゲーム編集のメリット

「インゲーム編集」をゲームアプリケーションに実装することで、以下のメリットがあります。

  • ゲーム要素に合わせた調整ができる
  • 手戻りが楽

ゲーム要素に合わせた調整ができる

EffectViewer上で作業した場合と異なり、ゲームで確認しながらエフェクトデータの作成、編集ができる為、ゲームグラフィックス や ゲーム性に関わるゲーム要素 と合わせた調整が行えます。

グラフィックスに関わる要素 ゲーム性に関わる要素

  • ゲーム中のライティングをエフェクトに反映
  • グレアや被写界深度などのポストエフェクト
    が適用された状態のエフェクトを編集
  • エフェクトを表示、消去するタイミング
  • ゲーム性を意識したエフェクトの表示優先

手戻りが楽

EffectMaker の編集内容は、即ゲームアプリケーション上のエフェクトに反映される為、エフェクトの作成、調整、確認が1度で済み、その結果手戻りが楽になります。
以下にこれまでのワークフローを例に説明します。

  • ゲームへのコンバートに30秒
  • ひとつのエフェクトにつき修正によるイテレーションが100回
  • エフェクト素材は100個

上記の例では、30 秒 × 100 回 × 100 個 = 約 80 時間 のコストがかかります。
