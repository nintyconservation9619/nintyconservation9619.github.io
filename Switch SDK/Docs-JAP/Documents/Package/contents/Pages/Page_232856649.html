ALBUM ライブラリ

はじめに

本ドキュメントの目的

本ドキュメントでは、NintendoSDK に収録されている ALBUM ライブラリの機能について説明します。

対象読者

NintendoSDK を使って、アルバム機能を利用するアプリケーションを開発する方を対象とします。

関連資料

ALBUM ライブラリの関数リファレンスおよびサンプルプログラムと併せて参照してください。

動作環境 

ALBUM ライブラリは実機環境および Windows 環境でのビルドが可能です。

Windows 環境ではアルバム機能は利用できません。アルバム機能を利用する各種 API は常に nn::album::ResultNotSupported を返して失敗します。

静止画・動画ファイルの扱いについて

ALBUM ライブラリでは、以下に示す静止画ファイルおよび動画ファイルを扱うことができます。

  • 本体機能の「アルバム」に作成・保存される静止画ファイルおよび動画ファイル

それぞれのファイルは以下のような仕様となっています。

本体設定からのまとめて削除 可能 可能

ALBUM ライブラリの機能

ALBUM ライブラリは、以下の機能を提供します。

  • 画面写真機能
    • 画面写真の保存：アプリケーションが作成した画像を画面写真としてアルバムに保存する機能
    • 画面写真の保存＆投稿：アプリケーションが作成した画像を画面写真としてアルバムに保存して、 SNS に投稿する機能
  • アルバムファイルへのアクセス機能
    • アルバム静止画ファイルへのアクセス機能
      • キャプチャーボタン短押しや上記「画面写真機能」で保存したアルバム静止画ファイルのリスト取得機能
      • 上記静止画ファイルから、静止画イメージ画像、サムネイル画像を取得する機能
    • アルバム動画ファイルへのアクセス機能
      • キャプチャーボタン長押し等で保存されたアルバム動画ファイルのリスト取得機能
      • 上記動画ファイルから、静止画イメージ画像、サムネイル画像を取得する機能


参考：

NX Addon 3.x.x ではアプリケーションは画面写真の保存のみを行えます。

NX Addon 6.0.0 以降ではアルバムに保存されている画面写真や動画をアプリケーションが読み込むことが可能です。

製品機上で動作しているプログラムから、アルバムに保存されているファイルを削除することはできません。
