OpusEncoder

  • 概要
  • 使用方法
  • 注意事項

概要

OpusEncoder は 入力 PCM データを Opus フォーマットにエンコードする PC ツールです。Tools/Codec/AudioCodec/ 以下に配置されています。

このツールは以下の DLL に依存しています。これらは NintendoSDK に含まれるものです。

  • Tools/Codec/AudioCodec/AudioCodec.dll
  • Tools/Codec/AudioCodec/ToolFoundation.dll
  • Tools/Codec/AudioCodec/GameToolFoundation.dll

出力される Opus データのコア部分は一般的な opus フォーマットと同等ですが、ヘッダフォーマット等は任天堂が独自に定義するものです。

使用方法

OpusEncoder.exe: 波形ファイルを opus データに変換します。

使用方法: OpusEncoder.exe [-o<output-filepath>] [--bitrate=<bitrate>]
          [--framesize=<framesize>]
          [--bitrate-control=<bitrate-control>] [-v] <input-filepath>

使用可能な引数:
  -o, --output <output-filepath>
                             出力ファイルを指定します。指定しない場合、入力フ
                             ァイルの拡張子を .opus に変更したものになります。
  --bitrate <bitrate>        ビットレートを bps で指定します。<6000 - 510000>
                             [デフォルト値: 96000]
  --framesize <framesize>    フレームサイズをミリ秒で指定します。2.5, 5, 10,
                             20 が指定可能です。[デフォルト値: 20]
  --bitrate-control <bitrate-control>
                             ビットレート制御方法を指定します。cbr（固定ビット
                             レート）、vbr（可変ビットレート）、cvbr（制約付き
                             可変ビットレート）が指定可能です。[デフォルト値:
                             cvbr]
  --coding-mode <coding-mode>
                             符号化モードを指定します。celt（音楽音源向け符号
                             化モード）、silk（音声音源向け符号化モード）が指
                             定可能です。[デフォルト値: celt]
  -v, --verbose              詳細なログを出力します。
  <input-filepath>           入力ファイルを指定します。

マルチストリーム対応

3 チャンネル以上の音源をエンコードする機能です。Opus エンコードでは、ストリームという単位にエンコードされます。最大2 チャンネルの音源データが 1 ストリームにエンコードされます。3 チャンネル以上の音源をエンコードする場合、2 ストリーム以上の複数ストリームから構成される Opus データが作成されます。これをマルチストリーム Opus データと呼びます。

マルチストリーム Opus データは nn::codec::OpusMultiStreamDecoder によりデコードすることができます。使い方は opus マルチストリームデコーダサンプルを参照してください。

オプション

--bitrate

圧縮後の opus パケットのビットレートを指定します。

次の点に注意して下さい。

  • 独自のヘッダを付与しているため、実際のビットレートは指定したビットレートより高い値になります。
  • モノラル音源とステレオ音源とで指定可能な最大ビットレートが異なります。
    • 指定可能な最大値ビットレートは、モノラル音源、ステレオ音源それぞれに対して 300 kbps、510 kbps です。

--framesize

opus パケットのフレーム長をミリ秒単位で指定します。

次の点に注意して下さい。

  • nn::codec::HardwareOpusDecoder を使用する場合、nn::codec::HardwareOpusDecoder::DecodeInterleaved() 毎に呼び出しオーバーヘッドが発生します。
    より長いフレーム長でエンコードする方が、一度のデコードでより多くのサンプルを得られるため、デコード関数の呼び出しオーバーヘッドを抑えられます。
    • nn::codec::OpusDecoder はライブラリであり、呼び出しオーバーヘッドは軽微であるため、この限りではありません。

--bitrate-control

opus パケットのビットレート制御モードを指定します。

  • vbr (可変ビットレート)
    他のモードより効率的にエンコードします。
    平均ビットレートが指定ビットレートと一致するようにエンコードされます。
    フレーム毎のビットレートは指定ビットレートと必ずしも一致しません。
  • cvbr (制限付き可変ビットレート)
    MP3 や AAC で "CBR" と呼ばれているモードと同じモードです。
    平均ビットレートが指定ビットレートと一致するようにエンコードされます。
    フレーム毎のビットレートは指定ビットレートと必ずしも一致しません。
  • cbr (固定ビットレート)
    フレーム毎のビットレートが指定ビットレートと一致するようにエンコードされます。

更なる詳細は opus codec RFC の ドキュメント(RFC6716) の "2.1.8.  Constant/Variable Bitrate" を参照してください。

--coding-mode

opus パケットの符号化モードを指定します。

次の点に注意して下さい。

  • silk 符号化モードでは、エンコーダ内部の帯域が 16 kHz (ナイキスト周波数は 8kHz) に設定されます。
    • celt 符号化モードの帯域は、最大で 48 kHz (ナイキスト周波数は 20 kHz) です。

使用例

input.wav を output.opus にエンコードする

$ OpusEncoder.exe -o output.opus input.wav

ビットレートやフレームサイズを指定してエンコードする

$ OpusEncoder.exe --bitrate=48000 --framesize=10 input.wav

注意事項

  • エンコードした結果をデコードした際に、パラメータによってはクリッピングノイズが発生することがあります
    • 入力波形の振幅をあらかじめ小さくしておくことで、問題を回避することができます
