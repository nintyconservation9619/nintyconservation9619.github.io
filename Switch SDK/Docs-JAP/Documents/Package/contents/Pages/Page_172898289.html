HDR 設定ができるようになりました !!

LayoutEditor で、HDR カラーを出力できるようになりました !!

プロパティウインドウの「カラー補間設定」の中にある「HDR カラー」というチェックボックスで機能の有効無効を切り替えます。
チェックボックスを有効にすると、カラーの 255 を 1.0 とした数値入力に変化します。

カラー設定に 10.0 や 100.0 など 1.0 を超える値を設定することもできるようになりました。
その場合、プレビューでグレアエフェクトを確認することができます。
通常カラーと同様にアニメーションカーブを使用できますので、グレアエフェクトを変化させる等の表現も可能です。

プロパティウインドウのカラー補間設定にある HDR 設定画面
A HDR カラー チェックボックスで機能の有効無効を切り替えます。
B 値を設定 1.0 ( 255 ) を超える値も設定できます。
プレビュー画面でのグレアエフェクトの変化

ウインドウペインのデフォルトの状態です。

R:1.00 G:1.00 B:1.00
B

ウインドウペインのコンテンツ ( 内部 ) 部分の数値を設定しました。

グレアエフェクトが確認できます。

R:10.00 G:1.00 B:1.00
C

ウインドウペインのコンテンツ ( 内部 ) 部分の数値を設定しました。

グレアエフェクトが確認できます。

R:1.00 G:30.00 B:30.00
注意：

 この機能はデフォルトで無効状態になっており、有効にするためには設定を変える必要があります。

色見本に登録された色をコピーする際に RGB と A を別々にコピーできるようになりました !!

LayoutEditor の色見本に RGB と A を別々にコピーする機能が追加され、より便利になりました !!

 [ メニューバー ] → [ ウインドウ ] → [ 色見本 ]

色見本からカラー設定を行う時に、RGB カラーと A を別々にドラッグしてコピーできます。

コピー先の設定、「 RGB を変更する」や「 A を変更する」と合わせて使用することで、より自由度の高い作業が可能となりました !!

色見本に登録された色を RGB と A で別々にコピー
A 色見本の RGB カラーラベルの左側をドラッグすると RGB がコピーされます。
B 色見本の A ( アルファ ) カラーラベルの右側をドラッグすると A がコピーされます。
C 頂点カラーの設定 頂点カラーを設定する画面です。


レイアウト・フォントで ASTC フォーマットが使用できるようになりました !!

LayoutEditor で使用できるレイアウト・フォントに、新たに ASTC フォーマットが加わりました !!

「 ASTC フォーマット」は圧縮フォーマットで、11 種類のフォーマットから、きめ細かく「圧縮率」と「圧縮品質」のバランスを選択することができます !
1 チャンネル ( L, A フォーマット ) ～  4 チャンネル ( RGBA ) まですべての形式で利用できます。
また、ソース画像の特徴から自動判定して、圧縮最適化を行いノイズを減らせます !!

プロジェクト設定で、AstcTextureEnabled を true にすることで、ASTC フォーマットを利用することができます。
( デフォルトは false で無効になっています )

ASTC4x4
BC7、BC3、BC5 と同じサイズで、BC7を上回る位の高い品質です。
ASTC12x12

ASTC フォーマットの中では最もちいさいファイルサイズです。

品質は極端に落ちます。

レイアウトのサンプルに「しきい値によるアルファ補間」が追加されました !!

LayoutEditor のサンプルデータに、新たに「しきい値によるアルファ補間」が加わりました !!

「しきい値によるアルファ補完」は、高性能なアルファテスト（細かな制御ができ、境界が滑らか）として利用することができます。
また、アルファチャンネルをディスタンスフィールド化したテクスチャに適用することで、拡大・縮小した場合のテクスチャ境界のぼやけを軽減することができます。
( アルファチャンネルのディスタンスフィールド化は、 Nintendo Ware の Photoshop プラグインで行うことができます )

 

Samples/Sources/Resources/Ui2d/thresholdingAlphaInterpolation/thresholdingAlphaInterpolation.flyt

 「しきい値によるアルファ補間」を使用しない場合 ( 初期設定 )

テクスチャのカラーに「黒カラー補完」と「白カラー補完」の入力値で影響を与えます。

 

  • 「黒カラー補完」のアルファ値はテクスチャの透明部分の透明度に影響を与えます。
  • 「白カラー補完」のアルファ値はテクスチャの不透明部分の透明度に影響を与えます。
「しきい値によるアルファ補間」を使用した場合

「黒カラー補完」と「白カラー補完」で設定したアルファ値を「しきい値」として、アルファ値の保管を行います。

 

  • 「黒カラー補完」のアルファ値はテクスチャの透明部分の透明度に影響を与えます。
    ここで指定した数値をしきい値として、数値に満たないテクスチャのアルファ値を透明として扱います。
  • 「白カラー補完」のアルファ値はテクスチャの不透明部分の透明度に影響を与えます。
    ここで設定した数値をしきい値として、数値以上のテクスチャのアルファ値を不透明として扱います。
