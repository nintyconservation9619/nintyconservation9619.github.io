概要

ユーザーの追加、削除、ネットワークサービスアカウントの取得を行います。

使い方

アカウント管理で一覧に表示される情報

項目名 内容
ニックネーム 設定されているニックネームがダブルクオート(")に括られて表示されます。
プロフィール画像 設定されているプロフィール画像
UID ユーザーアカウントごとに自動で発番されるID
Open状態 ユーザーアカウントがOpen状態の場合、文字色が白く表示されます。
Close状態の場合、文字色がグレーで表示されます。
LastOpenedフラグ 最後にOpenされた（現在Open状態のものも含む）ユーザーアカウントでは、文字色が白く表示されます。
そうでない場合、 文字色がグレーで表示されます。
参考：

UIDは開発の便利のため表示しています。
製品では表示されず、ユーザーはこのIDを認識することはありません。

注意：

プラットフォームごとのアカウントガイドに、ユーザーアカウントのOpen, Close状態に関する推奨の制御方法が記載されています。
アプリケーション開発者は、作成したプログラムが意図通りの制御を行っているか、上記 「Open状態」や「LastOpenedフラグ」を利用して確認することができます。


Firmware settings

以下の項目を設定できます。

項目名 内容
Requires Nintendo Account for use of network service ネットワークサービスを使用するのにNintendo Accountとの連携が必要であるかを変更できます。
デフォルト値は ｢Enabled｣ です。
Enable Nintendo Account license verification ネットワークサービスアカウントへのログイン時に、資格情報の検査を行うかを変更できます。
デフォルト値は「Enabled」です。 (NX Addon 5.0.0以降)
Skip user account selection if possible アプリケーション起動時にユーザーアカウント選択を省略可にできるか否かを変更できます。
デフォルト値は ｢Disabled｣ です。
Suspend background activity temporarily ユーザーアカウントに関するシステムの自律動作 (プロフィールやNintendo Accountの状態の同期) を一時的に停止します。
デフォルト値は「Disabled」です。

Add a user account

ユーザーを最大8人まで追加できます。

Edit user information

追加したユーザーを選択すると、ユーザー情報の変更やユーザーの削除を行えます。Nintendo Service Account登録前後で編集できる情報が変わります。

NSA 登録前 NSA 登録後

NSA登録前後で共通の項目

View play data

参考：

NX Addon 4.4.0以降で使用可能です。

「View play data」ボタンより対象のアカウントが最近遊んだアプリケーションのプレイ時間を表示できます。

表示対象のアプリケーションはインストールしたアプリケーション及びゲームカードから起動したアプリケーションに限られます。RunOnTargetやTargetToolを使用してPCから起動したアプリケーションは対象になりません。

Edit nickname

「Edit Nickname」ボタンよりニックネームを変更できます。

DevMenuで設定できる文字列(UTF-8)は以下のとおりです。

ソフトウェアキーボードでカスタム入力

1～10文字の文字列を入力できます。

参考：

NX Addon 4.5.0以降で使用可能です。

ニックネームで使用可能な文字のみで構成
  • User (デフォルト値)
  • ユーザー
  • Benutzer
参考：

NX Addon 4.5.0 選択肢から削除されました。

NSA登録前の項目

Register NSA

NSAをユーザーに登録します。

参考：

フレンドプレゼンス機能を利用する場合、NSAの登録毎に「Upload notification token」を実行する必要があります。

Delete user account

ユーザーを削除します。

NSA 登録後の項目

Unregister NSA

ユーザーへのNSA登録を解除します。

Sync user profile

ユーザーのプロファイルをサーバーと同期します。

Upload notification token

NSA通知トークンをサーバーにアップロードします。

参考：

NSA通知トークンとは、NSAを登録したデバイス宛てに通知を届けるために必要なトークンです。
主に、フレンドプレゼンス機能で利用します。

参考：

事前に、ネットワークの設定ページでネットワーク接続を確立しておく必要があります。
1540413エラーが発生した場合、ネットワーク接続の状態を確認してください。

Force upload user profile

現在のプロフィールをサーバーにアップロードします。

Upload notification token

アカウント向けのプッシュ通知を受信するためにサーバーへの登録を行います。

Update NA info cache

Nintendo Account情報のキャッシュを更新します。

[Debug] Invalidate token cache

このユーザーに関する認証トークンのキャッシュを破棄します。

参考：

ユーザーを区別しない、開発機単位やアプリケーション単位の情報は破棄されません。

[Debug] Set user state to invalid

デバッグ機能です。ユーザーを無効化します。以下の状態を設定できます。
通常のアプリケーション開発において、これらを区別する必要はありません。

  • NetworkServiceAccount: "CredentialBroken"
  • NetworkServiceAccount: "Unmanaged"
  • NetworkServiceAccount: "Banned"
  • NintendoAccount: "LinkageBroken"
  • NintendoAccount: "StateOtherButInteractionRequired"
  • NintendoAccount: "StateDleted"
  • NintendoAccount: "StateBanned"
  • NintendoAccount: "StateSuspended"
  • NintendoAccount: "StateWithdrawn"
  • NintendoAccount: "StateTermsAgreementRequired"
  • NintendoAccount: "StateReauthorizationRequired"
