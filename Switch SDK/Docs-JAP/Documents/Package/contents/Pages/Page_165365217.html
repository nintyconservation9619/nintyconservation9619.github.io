「コールツリー」タブの概要

このタブには、プロファイリングされたサンプルデータに基づいて、検出されたコールツリーが表示されます。このタブは、コールスタックのデータを有効にしてプロファイルを取得した場合にのみ表示されます。

各項目の前に表示される単位を変更するには、リボンバーの「ホーム」タブにある「単位」ボタンを使用します。

「コールツリー」タブのツールバー

「コールツリー」タブのツールバーには以下の各種コントロールがあります。

プレフィックスの選択

各関数名の前のプレフィックスは、「コールツリー」ツールバーにある 3 つの「プレフィックス」切り替えボタンで指定します。デフォルトでは、合計 と セルフ という 2 つのプレフィックスがオンになっています。プレフィックスのオンとオフを切り替えるには、そのプレフィックスのボタンをクリックします。すべてのボタンをオフに設定すると、関数名のみが表示されます。以下のプレフィックスを選択できます。

  • 合計：各関数およびその関数が呼び出す関数で費やされた時間を示します。
  • 「セルフ」：各関数で費やされた時間を示します。その関数が呼び出す関数で費やされた時間は除外されます。
  • 「サブ」：この関数が呼び出した関数で費やされた時間を示します。

ツリーの各レベルの関数は、プレフィックスに基づいて並べ替えられます。合計、セルフ、またはサブを選択した場合は、左端のプレフィックスの降順に並べ替えられます。プレフィックスを選択していない場合は、関数名を基準にしてアルファベット順に並べ替えられます。

「すべて展開」ボタン

「すべて展開」ボタンは、ビューで展開できるすべての項目を展開します。

「すべて非表示」ボタン

「すべて非表示」ボタンは、ビューで非表示にできるすべての項目を非表示にします。

「自動展開」ドロップダウン

デフォルトでは、「関数」タブまたは「コールツリー」タブで選択されている関数、および「コールツリー」タブの下部にある「検索」テキストボックスで選択されてる関数に基づいて、コールツリーが自動的に展開されるか、自動的に非表示になります。

「自動展開」ドロップダウンボックスには、以下の 3 つのオプションがあります。

  • 「自動展開オフ」：「検索」で選択または選択解除されている関数や強調表示されている関数があっても、それらの関数に基づいてコールツリーが展開されたり、非表示にされたりすることはありません。
  • 「自動展開表示のみ」：「検索」で選択または強調表示されている関数に基づいてコールツリーが展開されますが、非表示にはなりません。
  • 「自動展開・非表示」：「検索」で選択または選択解除されている関数、または強調表示されている関数に基づいて、コールツリーが展開されるか、非表示になります。

また、「すべて展開」および「すべて非表示」の各ボタンを使用して、ツリー全体を一度に展開したり、非表示にしたりすることもできます。

注意：ゲームのコールツリーデータが非常に大きい場合（ノードが 50,000 を超える場合）、ツリー全体またはツリーの大部分を展開すると GUI がクラッシュする可能性があります。これは、WPF の Windows ツリーコントロールにおける既知のバグによるものです。このような状況に陥った場合は、ツリー全体を展開しないようにしてください。また、「コールツリー」タブが表示されているときには選択する関数が多くなりすぎないようにするか、「自動展開オフ」を選択するようにしてください。

「コピー」ボタン

「コピー」ボタンをクリックすると、現在の展開された状態または非表示の状態のまま、すべての項目がクリップボードにコピーされます。

「検索」機能

「コールツリー」タブ下部の「検索」テキストボックスを使用すると、「自動展開」ドロップダウンオプションがオンであっても、オフであっても、一致するすべての関数が自動的に展開され、強調表示されます。この動作は、「関数」タブの「フィルタ」テキストボックスの場合と同様です。

関数の選択

「関数」タブで関数を選択する場合と同様に、コールツリーで関数をクリックすると、その関数が強調表示されます。コールツリーでは、強調表示されている関数のすべてのインスタンスが強調表示されます。自動展開がオン（「自動展開・非表示」または「自動展開表示のみ」がオン）になっていると、選択した関数のすべてのインスタンスが表示されるようにツリーが展開されます。

右クリックによるコンテキストメニュー

「コールツリー」タブ内で右クリックすると、以下のオプションを持つコンテキストメニューが表示されます。

「以下展開」：ノードを右クリックしてこのオプションを選択すると、その場所からすべてのノードが再帰的に展開されます。

「以下非表示」：ノードを右クリックしてこのオプションを選択すると、その場所にすべてのノードが再帰的に非表示になります。

「関数名をクリップボードにコピー」：ノードを右クリックしてこのオプションを選択すると、関数名がクリップボードにコピーされます。

「アセンブリ表示」：「アセンブリ」タブにフォーカスが移動し、クリックした関数のアセンブリコードが表示されます。

「詳細表示」：新しい「関数の詳細」ウィンドウが開き、クリックした関数の詳細が表示されます。

部分コールツリー

関数を選択すると、「コールツリー」タブの下部にある部分コールツリーにもその関数が表示されます。

このサブウィンドウには、3 種類のビューとして「選択関数以降」、「選択関数から逆方向」、および「レガシーリバース」があります。

これら 3 種類のビューのそれぞれで、合計、セルフ、およびサブの各値が、その特定のツリービューが存在していたプロファイルの比率として計算されます。

「部分コールツリー」ウィンドウは、見出しを上下にドラッグすることでサイズを変更できます。また、見出しの右側の下向き矢印をクリックすると非表示にでき、上向き矢印をクリックすると展開できます。

選択関数以降

このビューでは、選択したすべての関数がルートとして扱われます。コールツリーの下位に移動できるよう各ルートが展開され、コールツリーと同様のビューが表示されます。コールツリーとの違いは、選択された関数と同じ関数がコールツリーに複数ある場合、このビューではそれらの関数が 1 つのエントリにまとめられ、「関数」タブと同様に、それら関数の時間がすべて合算される点です。これにより、選択した関数による呼び出しが見やすくなります。

選択関数から逆方向

このビューでは、選択したすべての関数がルートとして扱われます。コールツリーの上位へ移動できるように各ルートが展開されます。ただし、単にコールツリーの上位に移動できるだけではなく、呼び出された方法に基づいて関数が切り分けられ、グループ化されます。これにより、他の方法では把握しにくい関数どうしの関係がわかりやすくなります。例えば、ベクトル正規化関数がコールツリーに 10 回表示されていても、その呼び出し元は 3 つの異なる関数のみであることも考えられます。このような場合、そのベクトル正規化関数は、3 つの子ノード（それぞれが各呼び出し元関数に対応）と共に 1 回だけ表示されるようになります。

レガシーリバース

このビューは、「選択関数から逆方向」に似ています。違いは、コールツリーに出現する選択された関数のそれぞれについて固有のルートが存在し、それらのルートが費やされた時間に基づいて並べ替えて表示されることです。これは、以前のバージョンのプロファイラの動作を引き継いだものであり、一定の状況下で効果的です。
