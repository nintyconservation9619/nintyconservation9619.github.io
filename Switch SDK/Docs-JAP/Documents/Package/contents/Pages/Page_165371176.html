バッファ

プロファイルデータの格納に使用される開発機材上のメモリ。

コールツリー

個々のサンプルコールスタックから構築されたツリーであり、そのサンプルをプロファイリングしているときのゲームの呼び出し構造を示します。これは、サイクルがないため、コールグラフとは若干異なります。コールツリーでは、同じ関数が、呼び出された場所に基づいて複数回出現する場合があります。

コールスタック

アプリケーションのスタックに格納された関数コールのリスト。

カウンタ

CPU カウンタ（パフォーマンスカウンタ）のデータまたはプロファイラ API を使用して手動で記録したユーザーデータ。

ハートビート

プロファイラ API を使用して手動で記録できるか、プロファイラで自動的に推測できる、特定のコアでの定期的なイベント。ハートビートはフレームマーカに似ていますが、あらゆるタイプの定期的イベントを表現できるため、より汎用的な意味を持ちます。ハートビートマーカは「サンプルグラフ」で確認でき、リボンバーの「ハートビート」タブで選択できます。

アイシクルグラフ

アイシクルグラフは、サンプリングされたコールスタックを経時的に階層表示したもので、コールスタックのルートが最上部にあり、リーフ関数が最下部にあります。

ジッタ

コードのサンプリングされるセクションに変動を生じさせるための、サンプリング時間におけるオフセット。サンプリングレートは固定であるため、ジッタを正しく設定しないと、コードのセクションによってはサンプリング数が不均衡になります。

パフォーマンスカウンタによるサンプル

特定のパフォーマンスカウンタを使用して関数をサンプリングする方法です。

時間によるサンプル

指定のレートに基づいて関数をサンプリングする方法です。

サンプリング

定期的にゲームを停止し、現在の関数またはコールタックを記録すること。

サンプリングレート

サンプルを取得するレート。

サンプリングストラテジー

サンプルをいつ取得するかを決めるために使用する手法。

セルフ

ある関数で排他的に費やされた時間から、その関数が呼び出した関数で費やされた時間を除いた時間（セルフ = 合計 - サブ）。

サブ

この関数が呼び出した関数で費やされた時間（サブ = 合計 - セルフ）。

接続

開発機材と接続して ELF ファイルをロードすること。プロファイラが接続されるとプロファイルを取得できます。

システム分類

関数は、アニメーションシステム、グラフィックスシステム、物理システムなどの特定のゲームシステムに属するものとして分類されます。

合計

関数で費やされた時間、およびその関数が呼び出したすべての関数で費やされた時間の合計（合計 = セルフ + サブ）。

単位

関数で費やされた時間に使用する単位。パーセント、時間、平均値、またはサンプルを使用できます。

切断

開発機材から切断する処理。
