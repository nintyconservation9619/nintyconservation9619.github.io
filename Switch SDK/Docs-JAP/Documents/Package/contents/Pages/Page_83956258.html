ツール

ツールに関する操作を行います。

A チーム設定の設定内容を確認します。
B ユーザー設定の設定内容を確認します。
C オートアニメーションバインドモードの 有効 / 無効 を切り替えます。
D デバッグログウィンドウを表示します。


チーム設定

チーム設定ファイルの設定を確認できます。
関連リンク チーム設定ファイルについての詳細は、こちらを参照してください。

A チーム設定ファイルのファイル入出力についての内容が表示されます。
B チーム設定ファイルのプリセットの内容が表示されます。
C チーム設定ファイルのデフォルト値の内容が表示されます。
D チーム設定ファイルのエラー設定の内容が表示されます。
E チーム設定ファイルのプレビュー設定の内容が表示されます。
F チーム設定ファイルのカラー設定の内容が表示されます。
G チーム設定ファイルのプラットフォーム設定の内容が表示されます。
H

編集モードに切替えで、チーム設定のパラメーターを追加、編集できます。

編集の確定にはアプリケーションの再起動を伴うことがあります。

編集モードに切替え

「編集モードに切替え」をクリックして表示されるボタンから、チーム設定のパラメーターを追加、編集できます。


A 設定項目が複数存在する場合に、リスト内で選択して上下に移動して順序を変更します。
B リスト内で選択した設定を編集します。
C 設定を追加します。
D

リストから選択した設定を削除します。

A 必須項目を設定します。

C 編集内容をキャンセルしてウインドウを閉じます。

ユーザー設定

ユーザー設定の設定を確認、編集できます。

関連リンク ユーザー設定についての詳細は、こちらを参照してください。

A 追加参照パスの設定内容が表示されます。
B スタートアップパスの設定内容が表示されます。
C 再読み込み設定が確認できます。
D 編集履歴に対する設定内容の表示と、編集ができます。
E 外部プログラムの設定内容の表示と、編集ができます。
F 入出力の設定内容の表示と、編集ができます。
G

マテリアルプロパティウィンドウのシェーダページにあるスライダーコントロールの表示 / 非表示を切り替えます。

追加参照パス設定

追加参照パスを追加、編集できます。

関連リンク 追加参照パスについての詳細は、こちらを参照してください。

A 追加参照パスのリストです。追加参照パスの情報をリスト表示します。
B 追加参照パスが複数存在する場合に、リスト内で選択した追加参照パスを上下に移動して順序を変更します。
C 追加参照パスを追加します。
D リストから選択した追加参照パスを削除します。
E リストから選択した追加参照パスの名前を入力します。スタートアップパスからこの名前を参照できます。
F リストから選択した追加参照パスのフォルダパスを入力します。絶対パスまたは参照元ファイルからの相対パスを設定することができます。
G サブフォルダも検索の有効 / 無効を切り替えます。フォルダ階層下を検索するか否かの設定です。
H 編集内容を確定してウインドウを閉じます。
I 編集内容をキャンセルしてウインドウを閉じます。

編集履歴の編集

編集履歴の消去タイミングと、保持する編集履歴の項目数を設定します。

関連リンク編集履歴についての詳細は、こちらを参照してください。

A 編集履歴を消去するタイミングを選択します。
B 編集履歴に保持する項目数を設定します。
現在は、0 ～ 20000 の間で設定できます。

外部プログラム設定

シェーダのソースコードなどを編集する際に、3DEditor から起動できるテキストエディタを設定できます。

関連リンク 外部プログラム設定についての詳細は、こちらを参照してください。

A  現在設定されているテキストエディタへのパスが表示されます。
B  起動するテキストエディタを選択して設定します。
こちらのボタンを押すと、ファイル選択ウィンドウが開きますので、任意のテキストエディタを選択します。
C テキストエディタを呼び出す際のオプションを設定します。
D ユーザーコマンド終了時にログダイアログを自動で閉じるか閉じないか設定をします。
E 接続先アプリケーションのデバックログを出力するしないか設定をします。

入出力設定

ファイルの入出力に関する設定を行えます。

関連リンク ファイルの入出力についての詳細は、こちらを参照してください。

A モデル読み込み時にマテリアルの修正を自動で行うか設定します。
B ライトアニメーションの修正を自動で行うか設定します。
C すでに読み込まれているファイルを再度開こうとした時にファイルが更新されていた場合に、自動的に再読み込みするかどうか設定します。
D 【 プログラマ向けの機能 】シェーダ最適化時にコンパイル情報を出力するか設定します。
E コンパイル情報を出力する場合は、情報の出力先を指定します。
F [メインメニュー] → [ファイル] にある機能「最近使ったファイル」の表示数を変更できます。
G [メインメニュー] → [ファイル] にある機能「最近使ったプロジェクト」の表示数を変更できます。

オートアニメーションバインドモード

オートアニメーションバインドモードとは、同名のアニメーションファイルが読み込まれた場合に、自動的にファイル名でアニメーションセットを作成してモデルにバインドしてくれる機能です。
関連リンク オートアニメーションバインドモードは、ツールバーのアイコンでも有効 / 無効が切り替えることができます。

オートアニメーションバインドモード 説明 イメージ
有効 同名のアニメーションファイル毎に自動的にアニメーションセットが作成されて、モデルにバインドされます。

右のイメージ図では、PrjSmpMaleWalk と PrjSmpMaleRun という同名ファイルを複数読み込み、アニメーションセットが自動的に作成されて、モデルにバインドされています。
アニメーションセット以下は、 閉じた状態で表示されます。
有効
[ Ctrl ]キーを押しながらアニメーションファイルを読み込む		 同名のアニメーションファイル毎に自動的にアニメーションセットが作成されて、モデルにバインドされます。

[ Ctrl ] キーを押しながらアニメーションを読み込んだ場合は、アニメーションセット以下が開いた状態で表示されます。
無効 アニメーションファイル PrjSmpMaleWalk / PrjSmpMaleRun 共に既定アニメーションセットにすべて入っている状態になります。　

デバッグログウィンドウ

デバッグログウィンドウの表示/非表示を切り替えます。

主に問題の調査で使うウィンドウです。
