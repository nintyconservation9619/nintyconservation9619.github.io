ワークスペースファイル

ワークスペース上の作業状態を記録できる「ワークスペースファイル」について説明します。

ワークスペースファイルを開く

ワークスペースファイル（.work）を開くとダイアログが開きますので、編集中のファイルをすべて破棄するかどうか選択します。
破棄した場合はワークスペースファイル（.work）に記述されたファイルが開きます。

ワークスペースファイル（.work）を読み込んだ時、関連するファイルが見つからない場合はメッセージが表示されます。
見つからなかったファイルは除外されます。

ワークスペースファイル（.work）を開く場合、同時に別のワークスペースファイル（.work）や、その他のファイルを指定して開くことはできません。

注意：

ワークスペースファイル（.work）を開く場合、同時に別のワークスペースファイル（.work）や、その他のファイルを指定して開くことはできません。

ワークスペースファイルの保存

ワークスペース上の作業状態は、ワークスペースファイル（ .work）として保存できます。
保存したワークスペースファイル（.work）を開くことで、保存した時の状態へ復帰させることができます。

ワークスペースファイルは以下の情報を保持します。

  • プレビューノードの設定内容
  • ビューアの設定内容と背景用テクスチャファイル（ .ftxb）のリンク情報
  • モデルファイル（.fmdb）とアニメーションファイル（ .fskb）のリンク情報
  • エミッタセットファイル（ .eset）のリンク情報

関連ファイルの保存場所は、ワークスペースファイル（.work）と同じドライブ内の上位ディレクトリまでを指定できます。
それ以外の場所に保存した場合、ワークスペースファイル（.work）を開いた時に関連ファイルは読み込まれません。
