カラー の設定

ここでは「カラーコンバイナ」計算式で使用する Color0 や Color1 のカラーを設定する機能「カラー 0 挙動タイプ」「カラー 1 挙動タイプ」について説明します。

概要

「カラー 0 挙動タイプ」は、「カラーコンバイナ」計算式で使用する Color0 のカラーやカラーアニメーションが設定できる機能です。
「カラー 1 挙動タイプ」は、「カラーコンバイナ」計算式で使用する Color1 のカラーやカラーアニメーションが設定できる機能です。
以下は「 設定項目」の概要です。

カラー 0 挙動タイプ、カラー 1 挙動タイプ

「カラーコンバイナ」計算式の Color0 または Color1 として使用する挙動タイプを、以下のプリセットから選択します。

固定
アニメーションしない時の固定カラーは「 カラーのサムネイル」を選択し、カラー編集の UI を使用して設定します。
8キーアニメ 8 つまでキーフレームを登録してアニメーションできる UI 「グラデーションエディタ」 を使用してアニメーションを作成します。
ランダム
8 つまでのカラーが登録できるランダム設定の UI を使用してランダムテーブルを作成します。
「 チェックボックス ON」のカラーが適用されます。
パーティクルのカラーは、エミッタから放出される時に上記の テーブルからランダムに選択されます。

関連リンク 以下は関連ページへのリンクです。

カラー編集のUI 選択しているカラーやアルファの値を設定できる UI です。
グラデーションエディタ カラーやアルファにグラデーションでパーティクル時間のアニメーションを編集できる UI です。
ランダムカラー設定の UI

複数のカラーを登録できる UI です。
登録したカラーが放出時のパーティクルカラーとしてランダムに適用されます。

C ランダム

「ランダムカラーの UI」で 5 箇所に 3 色のカラーを設定してます。
「█ 赤」は 3 箇所に登録している為、適用確率が他のカラーより高くなります。

以下のように、カラー0、カラー1 で挙動タイプを「ランダム」に設定し、使用するカラーの数を合わせたとき、カラー0 と カラー1 で乱数値が揃います。

カラー0 カラー1

カラー 0 ,カラー 1 は共にランダムにするカラーを 2 色指定しています。

「カラー 0 , カラー 1」=「赤,黄」or「青,緑」の組合わせでランダムになります
