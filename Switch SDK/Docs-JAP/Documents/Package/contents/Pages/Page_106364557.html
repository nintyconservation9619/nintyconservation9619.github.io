カスタマイズとは

はじめに

標準で提供する機能をカスタマイズすることで、より高度なシェーダ表現を実現したり、ただ再生するだけではなくアプリとより強力に連動するようなエフェクト表現を実現することができます。
エフェクトランタイムでは、2通りのカスタマイズが可能です。

  • カスタムシェーダ
  • カスタムアクション

カスタムシェーダ

カスタムシェーダは、標準の頂点・ピクセル シェーダをカスタマイズすることで、エフェクト描画に対して、アプリ固有のライティング表現などを実装できます。

左：カスタムシェーダによるライティング表現です。
右：通常シェーダによる描画結果です。

カスタムアクション

エミッタの挙動やパーティクルの挙動や描画をカスタマイズできます。

たとえば、パーティクルを描画すべき位置にアプリ側で3Dモデルを描画するといったカスタマイズや、ゲーム内オブジェクトとパーティクル単位で衝突判定を行い、よりインタラクティブなエフェクトを実現などが可能になります。

ユーザデータ定義

これらのカスタマイズを実現するために、エフェクトメーカーには ユーザーデータ定義 とよばれる、ユーザー独自のバイナリ構造とエフェクトメーカーでそれを編集するためのＵＩを表示させる機能が実装されています。ここで編集したバイナリ構造はエフェクトランタイム上から参照できカスタマイズの実装に利用できます。

エフェクトメーカーには、機能をカスタマイズするにあたって必要となる　ユーザー独自のバイナリ構造とエフェクトメーカーでそれを編集するためのＵＩを表示させる機能が実装されています。ここで編集したバイナリ構造はエフェクトランタイム上から参照できカスタマイズの実装に利用できます。

コールバック関数

エフェクトランタイム処理内のさまざまなタイミングでコールバック関数を呼び出し、エミッタやパーティクルの挙動をカスタマイズすることができます。カスタムシェーダでは、ユーザーデータ定義を利用してアーティストがエフェクトメーカー上で編集・保存したパラメータをコールバック関数の中から参照し、コンスタントバッファとして頂点・ピクセルシェーダ内で利用することも可能です。
