Target Manager との接続・切断イベント取得

概要

Target Manager との接続・切断を通知するイベントを取得する機能です。

こちらの機能を使うことで Target Manager との接続や切断を待機することが可能になります。

なお、取得される同期待ちのイベントは SystemEvent クラスのオブジェクトです。SystemEvent クラスの詳細については以下のページを参照してください。

  • SystemEvent クラス

接続・切断イベントの利用方法

接続・切断の各イベントは以下の API で取得できます。

  • 接続イベントの取得
    • nn::htc::BindHostConnectionEvent
  • 切断イベントの取得
    • nn::htc::BindHostDisconnectionEvent

これらの API の利用には事前にHTC ライブラリの初期化が必要ですので nn::htc::Initialize() であらかじめ初期化しておいてください。

実際の利用方法についてはサンプルがありますので、以下を参照して下さい。

  • ホストとの通信状態を取得するサンプルプログラム

スリープ時の挙動について

TargetManager と接続中にスリープに入るだけでは接続・切断イベントはシグナルされません。
但し、ネットワーク断などでスリープ中に切断されていた場合には切断イベントがシグナルされます。

注意事項

こちらの機能で取得されるシステムイベントのクリアタイミングはシステム側で制御するようになっているため、ユーザ側でクリアしないようにしてください。
