プレゼンスグループ ID

プレゼンスにはアプリ固有の情報を設定・取得する機能があります。
通常、この情報は同一アプリケーションを起動しているフレンド間でしか共有されません。

しかし、バージョン違いのアプリケーションを複数リリースする・販売対象国によって ROM を分割する等、複数のアプリケーションに分かれるがプレゼンスは共有したいという場合があります。

この要求を満たすため、プレゼンスグループ ID は存在します。
アプリ固有の情報を共有する・しないは、プレゼンスグループ ID が一致するかどうかで判定されます。

パラメータの共有

プレゼンスグループ ID の一致・不一致によって、nn::friends::FriendPresence の各パラメータは以下のようになります。

関数名 プレゼンスグループ ID の影響 説明
GetStatus なし プレゼンスグループ ID によらず、プレゼンスの状態は取得可能です。
GetLastPlayedApplication なし プレゼンスグループ ID によらず、最後に遊んだアプリケーションは取得可能です。
GetLastUpdateTime なし プレゼンスグループ ID によらず、プレゼンスの最終更新時刻は取得可能です。
GetDescription あり プレゼンスグループ ID が一致しない場合、空文字列が返ります。
GetAppValue あり プレゼンスグループ ID が一致しない場合、空文字列が返ります。
IsSamePresenceGroupApplication あり プレゼンスグループ ID が一致しない場合、false が返ります。

プレゼンスグループ ID の設定

プレゼンスグループ ID の設定は、 nmeta の PresenceGroupId の項目に指定します。
デフォルトでは、アプリケーション ID と同一の値が設定されます。

プレゼンスを複数のアプリケーションで共有したい場合、代表となるアプリケーションを 1 つ決め、そのアプリケーションのプレゼンスグループ ID を他のアプリケーションの PresenceGroupId に設定します。
