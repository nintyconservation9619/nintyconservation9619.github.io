G3dバイナリを分割して出力する

エフェクトバイナリ内に含まれるG3dバイナリを個別ファイルとして出力し、ランタイム上でエフェクトバイナリとバインドする手順を説明します。

G3dバイナリを分割出力する

G3dバイナリを分割出力するには、バイナリコンバート　コマンドライン引数の --output-g3d-primitive-binary にて分割出力するG3dバイナリファイル名( *.bfres )を指定することで行えます。
分割出力されたG3dバイナリファイルは、bfres ファイル形式で出力され、G3dバイナリ単体としても機能します。

EffectConverter.exe --output effectBinary.ptcl --convert-list ConvertList.xml --output-g3d_primitive-binary effectG3dPrimitive.bfres ...

分割されたG3dバイナリとエフェクトバイナリにバインドする

分割出力されたG3dバイナリは、ランタイム上でエフェクトバイナリとバインドする必要があります。

// エフェクトテクスチャバイナリを初期化
nn::g3d::ResFile* pResFile = nn::gfx::ResTextureFile::ResCast( textureBinary );
pResTextureFile->Setup( pGfxDevice );

// 登録済みエフェクトリソースとG3dバイナリをバインドする
pVfxSystem->GetResource(resourceId)->BindExternalG3dResFile( pResTextureFile );

詳しくは、 VfxBinarySegmentation サンプルを参照ください。
