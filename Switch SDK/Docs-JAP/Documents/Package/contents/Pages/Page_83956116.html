マルチプラットフォーム対応について

本バージョンより エフェクトモジュールのマルチプラットフォーム対応を開始しています。
これまでは、Cafe版、Cafe( Windows版 ) が存在していましたが、Cafe( Windows版 ) が独立した Windows版へと変更になります。

参考：

NintendoWare Effect はマルチプラットフォーム対応中です。
今後のバージョンで仕様が変更される可能性があります。

エフェクトバイナリの変更点

バイナリフォーマットが以下のように変更されています。
バイナリのエンディアンがそのプラットフォームに合ったエンディアンで出力されるようになりました。

Ver 0.4.0 バイナリエンディアン グラフィックスライブラリ
Cafe 版 Big GX2
Windows 版 Big OpenGL
Ver 0.5.0 バイナリエンディアン グラフィックスライブラリ
Cafe 版 Big GX2
Windows 版 Little OpenGL

また、Cafeランタイム、Windowsランタイムで利用できるバイナリプラットフォームにも変更があります。

Ver 0.5.0 Cafe ランタイム Windows ランタイム
Cafe バイナリ ○ ○
Windows バイナリ ×
 ○

コマンドラインコンバートの変更点

マルチプラットフォーム対応により、コマンドラインコンバート時の引数にも変更があります。
以下例で示すように、 --platform オプションでコンバートするターゲットを指定する必要があります。

例)　Cafe向けのエフェクトバイナリを生成する
EffectMaker.Application.exe --convlist ConvertList.xml -o EffectBinary.ptcl --platform cafe

EffectMaker の変更点

EffectMaker にプラットフォーム指定コンボボックスが追加されました。
開発を行っているプラットフォームを正しく指定してビューアに接続する必要があります。
旧SDK( NW4Fを含む ) ユーザーは、プラットフォーム「Cafe」を選択し、接続先のビューアを「実機」、もしくは「PC」を選択していただければ大丈夫です。

このプラットフォーム選択を変更することで保存される中間ファイルに影響はありません。

EffectViewer の変更点

EffectMaker と接続する EffectViewer も以下のような変更があります。
プラットフォーム固有のビューアはそのプラットフォーム専用、Windowsビューアは、各プラットフォームのバイナリを再生するようになります。
Cafeプラットフォームで開発する場合は、CafeビューアとWindowsビューアを利用し、Windowsプラットフォームで開発する場合は、Windowsビューアのみが利用できます。

新しいエフェクト表現

範囲内ループ機能が追加されました！

パーティクルの表示を指定した「箱」の範囲内に納める「範囲内ループ」機能が追加されました！
パーティクルはその箱の範囲外に出たら、箱の反対側に戻り、繰り返し表示されます。

使用例

雨や雪、塵 など指定ワールド空間上に配置されるようなエフェクトに利用できます。
指定された箱の中でパーティクル挙動がループするため、少ない量のパーティクルを効果的に配置することができます。

A 下側をクリップ 雨粒を水面より下に表示させないといった表現に使用できます。
B 上側をクリップ 水中の泡を水面より上に表示させないといった表現に使用できます。

関連リンク 「範囲内ループ」については「範囲内ループ」を参照してください。

EffectMaker

起動時にツールのヒント！

毎回 EffectMaker 起動時には「今日のヒント」が表示されます。
新しくなった EffectMaker の操作が図説されています。

カラーの調整は一箇所でまとめて設定

カラーピッカーがプロパティウィンドウ内に入ったことにより、
頻繁に行うパーティクルのカラーの調整がよりかんたんになりました！

同種ノードの同時編集機能

複数の同種ノードをワークスペース上でまとめて選択、同時に編集できるようになりました！

画像の様に、3つのエミッタノードを選択、ひとつのノードでテクスチャを登録すると、他のノードにも登録されています。

変更が加えられた箇所がわかる機能

デフォルトから変更が加えられたタブやラベルにオレンジの色がつく様になりました。

画像の例では「パーティクル」タブと「テクスチャ」タブの内容が初期状態から変わっています。
テクスチャを登録するとラベル「ファイル設定」も黄色からオレンジ色に変化します。

便利な使い方としては、サンプルや他人が作成したエフェクトを参考にする際など、
新規エミッタの状態からどこを設定したら良いのかが一目でわかります。

また、タブ単位やグループ単位でデフォルト値に戻すこともできる様になりました。

画像の例では、タブ上でクリックで開くメニューです。
「タブ内容をデフォルトに戻す」を選択すると、新規エミッタ作成時の状態に戻すことができます。

エフェクトブラウザの機能向上

 「エフェクトブラウザ」ｊは EffectMaker 内で複数のエフェクトデータをWindows エクスプローラの様に一覧表示させることができます。
更に各エフェクトデータが使用するテクスチャやモデルファイルなどのアセットに対しても検索や表示させる機能も備えています。
この「エフェクトブラウザ」に以下の様な機能が追加されました。

検索ボックスで文字列の検索

Windowsエクスプローラの「検索ボックス」と同様に、検索ボックスに入力した文字列でファイル名を絞りコンでリストに一覧表示できます。

画像の例では、検索ボックスで「Fire」と入力することにより、「全項目数：310」の内、名前に Fire を含むファイル「1件」がリスト表示されています。

サムネイルの表示サイズを変更

ファイルの表示サイズを2段階で切り替えられる様になりました。

オプションの「 拡大 / 縮小ボタン」を　ON / OFF することで、サムネイルの表示サイズを切り替えています。

参考：

オプションの「 拡大 / 縮小ボタン」は、タブによって以下の様に結果が異なります。

コンピュータ / お気に入りタブ Textures または Primitives フォルダ内のファイルを大きく表示します。
検索タブ 検索の結果、表示されるファイルを大きく表示します。

どこでもラベルカラーやコメント入力

エミッタセット以外にテクスチャやモデルファイルにもラベルカラーやコメントを入力することができます。
また、ラベルカラーやコメントを入力できる箇所が増えています。

画像の例ではファイルをリスト表示した状態でラベルカラーを変更しています。
