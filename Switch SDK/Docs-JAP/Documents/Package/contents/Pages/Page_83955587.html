シェイプアニメーション

シェイプアニメーションは、モーファーモディファイアを適用し、ウェイト値をアニメーションさせて設定します。
法線のアニメーションは 3ds Max 上と NintendoWare 上とで表示が異なる場合があります。

頂点カラーのアニメーションは、対象プラットフォーム上でのみ確認できます。

シェイプアニメーションは、モディファイアからモーファーを適用し、設定します。

ベースオブジェクトとターゲットオブジェクトの頂点数や面数は同じにする必要があります。

参考：

モーファー適用後にベースオブジェクトおよびターゲットオブジェクトの頂点数や面数を変更するような操作を行わないでください。

参考：

ベースオブジェクトとターゲットオブジェクトとで、面の構成が異なるとシェイプアニメーションが正常に表示されない場合があります。
例えば、ターゲットオブジェクトを編集可能ポリゴンに変換している場合は、ベースオブジェクトも編集可能ポリゴンに変換し、必ず状態を合わせてください。

頂点の位置の変更はモーファー適用後でも可能です。

モーファー適用後のターゲットオブジェクトの頂点位置や法線、頂点カラーの変更は、ベースオブジェクトにて「自動的にターゲットを再ロード」にチェックがなくても 出力時に最新のターゲットオブジェクトの頂点情報を出力します。
その場合、3ds Max 上と表示が異なるように見えます。

シェイプアニメーション設定プラグインで、シェイプアニメーション再生時に更新する頂点属性を設定できます。

デフォルトでは頂点座標、法線、接線、従法線が更新されます（接線と従法線は法線マッピングまたは異方性シェーダを使用している場合のみ更新されます）。

参考：

更新する頂点属性が多いほど、データ量と対象プラットフォーム上の処理負担が増加します。

マテリアルが同じポリゴン群について、ベースシェイプとターゲットシェイプで変化しない頂点属性のキーシェイプデータは出力されません。
すべての頂点属性が変化しなければ、そのポリゴン群のシェイプアニメーションは出力されません。

オブジェクトの頂点属性の変化する範囲が狭い場合、その範囲の面のマテリアル ID を変えマルチマテリアルにすることで、データ量と NintendoWare 上の処理の負担が軽減できます。

ターゲットシェイプの頂点属性データはモデルファイル内に出力されます。
ターゲットシェイプの数・形状・頂点属性を変えた場合は、モデルファイルを再出力してください。

参考：

シェイプアニメーションファイルには、キーが打たれているか値が 0 でないウェイト値のみ、アニメーションカーブデータを出力します。

複数の頂点カラーを使用する場合、ベースシェイプとターゲットシェイプすべてに存在するカラーセットのみアニメーション可能です。

カラーセットの値の型と成分数はベースシェイプの設定で統一されます。

たとえば次の表で「○」のカラーセットが存在するとします。

頂点カラーチャンネル ベースシェイプ ターゲットシェイプ A ターゲットシェイプ B
nw4f_color0 ○ ○ ○
nw4f_color1 ○ × ○
nw4f_color2 ○ ○ ○
nw4f_color3 × × ○

この場合、color0 と color2 がアニメーションします。

color1 はアニメーションしない固定値として出力されます。color3 はベースシェイプに存在しないので出力されません。
