カラーターゲットビュー

概要

カラーターゲットビューは、テクスチャーをカラーターゲットとしてセットするための機能を提供します。
ビューとしての特性は基本的にテクスチャービューと同じです。テクスチャービューを参照してください。

カラーターゲットビューは、元のテクスチャーの部分を、配列範囲と特定のミップマップレベルによって表現します。

情報

カラーターゲットビューの情報は nn::gfx::ColorTargetViewInfo として定義されています。

次元、フォーマット、テクスチャーポインター

テクスチャービューとほぼ同じであるため、そちらを参照してください。

元となるテクスチャーのフォーマットはカラーターゲットとして利用可能である必要があります。

元となるテクスチャーは、GPU がアクセスする方法として GpuAccessFlag_ColorTarget を含んで初期化されている必要があります。

サブリソース範囲

EditArrayRange 関数により配列範囲を、SetMipLevel により特定のミップマップレベルを指定します。

オブジェクト

カラーターゲットビューのオブジェクトは nn::gfx::ColorTargetView として定義されています。

初期化と破棄

テクスチャービューの初期化と破棄はオブジェクトの規則に従います。

カラーターゲットビューのセット

nn::gfx::CommandBuffer クラスの SetRenderTargets に深度ステンシルビューと合わせて指定することにより、カラーターゲットとしてカラーターゲットビューをセットすることができます。このときカラーターゲットビューは配列としてセットされ、それぞれは順番に対応するマルチレンダーターゲットとして解釈されます。
セットするカラーターゲットのフォーマットはそれぞれ、レンダーターゲットステートのカラーターゲットのフォーマットと一致している必要があります。

カラーターゲットビューのクリア

カラーターゲットビューにはピクセルシェーダーを通して自由に書き込みを行うことができますが、カラーターゲットの全ピクセルについて一律に同じ値を書き込むためのクリア関数が用意されています。

カラーターゲットのクリアは、浮動小数点数、符号付き整数、符号なし整数フォーマットについて nn::gfx::CommandBuffer::ClearColorTarget を通して、または浮動小数点数フォーマット専用に nn::gfx::CommandBuffer::ClearColor を通して行われます。すべてのピクセルの赤要素、緑要素、青要素、アルファ要素それぞれについて、どの値でクリアするかを指定します。
このとき カラーターゲットにおける特定の配列範囲だけをクリアすることも可能で、それは pArrayRange に望む範囲を指定することで行われます。このパラメーターに NULL を指定した場合はすべての配列範囲をクリアします。

クリアにおける制限

クリア後はレンダーターゲットの設定が無効化されます。必要な場合は再度レンダーターゲットを設定してください。

マルチサンプルイメージの解決

もしカラーターゲットビューがマルチサンプルカラーターゲットであるならば、それは nn::gfx::CommandBuffer クラスの Resolve 関数を通して単一サンプルのテクスチャーへと解決することができます。

このとき、カラーターゲットにおける特定の配列範囲だけを解決することも可能で、それは pSrcArrayRange に望む範囲を指定することで行われます。このパラメーターに NULL を指定した場合はすべての配列範囲を解決します。
