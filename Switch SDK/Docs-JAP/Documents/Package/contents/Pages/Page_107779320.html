アプリケーションの開発機へのインストール


開発した NX アプリケーションは開発機にインストールして実行することでホスト PC を接続せずに動作確認することができます。

ここでは、開発した NX アプリケーションを開発機にインストールして実行する手順を説明します。

警告：

現在、多くのサンプルプログラムは以下の手順で開発機にインストールした場合に、正常に実行できないか、正常に実行できたとしても画面になにも表示されません。
サンプルプログラムを利用して以下の手順を確認する場合は、以下に示すサンプルプログラムをお使いください。

  • GfxShaderCompile
  • GfxSimple
  • NvnTutorial01GettingStartedWithNVN
  • HidSimple
参考：

インストールしたアプリケーションが実行できない場合

作成したアプリケーションが以下の条件を満たしているか確認してください。

  • 起動時にプログラム引数が与えられなくてもアプリケーションが想定通りに動作すること
    • 現在、以下の手順でインストールしたアプリケーションを実行する際には、プログラム引数が何も指定されません。
  • ホスト PC と通信する機能が利用できなくてもアプリケーションが想定通りに動作すること

インストール可能なアプリケーションイメージの作成

NX アプリケーションを開発機にインストールするためには、インストール可能なアプリケーションイメージの形式である nsp ファイルを作成する必要があります。

nsp ファイルは、NX アプリケーションの プロジェクトプロパティ＞構成プロパティ＞全般の アプリケーションプログラムフォーマットに Nsp を指定してビルドすることで出力されます。

参考：

nsp ファイルは AuthoringTool を使ってコマンドラインで作成することもできます。
コマンドラインを利用もしくは独自のビルドシステムを構築して nsp ファイルを作成する場合は、アプリケーションイメージの作成 や 独自のビルドシステムを構築する を参照してください。

アプリケーション ID について

以降の操作では、アプリケーションID の指定が必要になります。NX アプリケーションをビルドする際に指定する .nmeta ファイル（XML 形式）の NintendoSdkMeta/Core/ApplicationId のパラメータを適宜設定した上で操作してください。
複数のアプリケーションを同時にインストールしたい場合は、都度指定する .nmeta ファイルを変更して nsp ファイルを作成してください。

参考：

NX Addon 3.0 から nmeta ファイルのルートディレクティブのデフォルトが Meta から NintendoSdkMeta に変わりました。

Meta も引き続きご利用になれますが、 サンプルやドキュメントなどの nmeta ファイルは NintendoSdkMeta に置き換わっているのでご注意ください。


アプリケーションのインストール

アプリケーションのインストールには DevMenuCommand を利用します。

前項で作成した nsp ファイルをインストールするコマンドを次に示します。
ここで指定する nsp ファイルのパスは、絶対パスで書く必要がある点にご注意ください。

%NINTENDO_SDK_ROOT%\Tools\CommandLineTools\RunOnTarget.exe ^
%NINTENDO_SDK_ROOT%\TargetTools\NX-NXFP2-a64\DevMenuCommand\Release\DevMenuCommand.nsp ^
-- application install %NspFile%

-s オプションでインスール先のストレージを指定することも出来ます。
インストールしたアプリケーションは、DevMenu や DevMenuCommand を利用して実行することができます。

他にも、SDEV 本体や EDEV 本体にアプリケーションをインストールして、開発機単体でアプリケーションを実行することもできます。
詳しい手順は、DevMenu - アプリケーション管理 や以下を参照してください。

  • SDEV 取扱説明書
    • 「アプリケーションをインストールする」
  • EDEV 取扱説明書
    • 「アプリケーションをインストールする」

インストールしたアプリケーションの実行

インストールしたアプリケーションは DevMenu または DevMenuCommand で実行できます。
デバッグパッドとタッチでの操作に対応しています。現時点では、タッチ操作は一部未対応です。

アプリケーションを NX 開発カードと内部ストレージに書き込んでいる場合など、複数のストレージに同じ ApplicationId を持つアプリケーションが存在することがあります。
この場合、以下の優先度で利用するストレージを決定します。

  1. NX 開発カード
  2. SD カード
  3. 内部ストレージ

DevMenu で実行する

開発機の電源を入れると DevMenu が起動してインストールされているアプリケーションのリストが表示されます。

十字ボタンの右を押してアプリケーションの一覧画面にフォーカスを移動します。
起動したいアプリケーションを選択し、A ボタンを押すとアプリケーションが実行されます。

DevMenuCommand で実行する

インストールされているアプリケーションの ID を次のコマンドで確認します。

%NINTENDO_SDK_ROOT%\Tools\CommandLineTools\RunOnTarget.exe ^
%NINTENDO_SDK_ROOT%\TargetTools\NX-NXFP2-a64\DevMenuCommand\Release\DevMenuCommand.nsp ^
-- application list

次のコマンドで指定した ID （ここでは例として 0x0100 f001 0348 8000 としています）のアプリケーションが実行できます。

%NINTENDO_SDK_ROOT%\Tools\CommandLineTools\RunOnTarget.exe ^
%NINTENDO_SDK_ROOT%\TargetTools\NX-NXFP2-a64\DevMenuCommand\Release\DevMenuCommand.nsp ^
-- application launch 0x1000f00103488000
