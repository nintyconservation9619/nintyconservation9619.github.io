はじめに

このページについて

このページでは、Windows 向けのバイナリを生成する既存の Visual Studio プロジェクトに NX 開発機用のプロジェクト構成を追加するための具体的な手順について説明します。

このチュートリアルでは次のいずれの Visual Studio プロジェクトも対象としています。

  • NintendoSDK を使用しない一般的な Windows 用プロジェクト
  • NintendoSDK の共通パッケージを用いて作成された、NintendoSDK を使用した Windows 用プロジェクト

関連項目

このチュートリアルでは、NintendoSDK が提供する Visual Studio 拡張機能である「NintendoSDK 構成マネージャ」を使用します。NintendoSDK 構成マネージャの詳細は NintendoSDK 構成マネージャ を参照してください。

注意事項

このチュートリアルの手順やスクリーンショットは、使用している Visual Studio のバージョンや環境設定によって実際の手順やスクリーンショットと一部異なる可能性があります。

手順

事前準備

NDI を使用して NintendoSDK VSI for NX パッケージ、および NintendoSDK VsExtension パッケージをあらかじめインストールしてください。インストール後は、Visual Studio の再起動を行ってください。

NX 開発機用のプラットフォームを追加する

NintendoSDK VSI for NX をインストールした Visual Studio では、プロジェクトプラットフォームとして NX32 及び NX64 という値が選択できるようになります。これらはそれぞれ NX 開発機用の 32bit バイナリ及び 64bit バイナリをビルドするためのプラットフォームです。これらのプラットフォームをプロジェクトに追加します。

Visual Studio でプロジェクトを開き、構成マネージャーを開きます。例えば、メニューから [ビルド(B)] → [構成マネージャー(O)...] を選択します。

[アクティブ ソリューション プラットフォーム(P)] のプルダウンメニューを開き [<新規作成...>] を選択します。

32bit バイナリを生成する場合は [新しいプラットフォームを入力または選択してください(P)] に NX32 を、64bit バイナリを生成する場合は NX64 を選択して [OK] をクリックします。既存のビルド設定を再利用する場合は [設定のコピー元(S)] から適切なプラットフォームを選択します。[新しいプロジェクト プラットフォームを作成する(C)] にはチェックを入れてください。

NintendoSDK 構成マネージャを使用する  

NintendoSDK 構成マネージャを使用して、 NintendoSDK を使用するために必要な設定を行います。

NintendoSDK 構成マネージャを起動する

[ツール(T)] → [NintendoSDK] → [NintendoSDK Configurations...] メニューを選択し、 NintendoSDK 構成マネージャを起動します。

NintendoSDK を使用する設定を行う

NintendoSDK を使用したいプロジェクト構成に対して、以下の操作を行ってください。

  • 「NintendoSDK 使用」列のチェックボックスをチェックする
  • 「SDK パス」列の 「...」 ボタンをクリックするか既存の構成からコピー＆ペーストして、SDK のパスを入力する
  • 「API セット」列に、API セット NX を設定する
  • 「SDK ビルドタイプ」列に、使用したいビルドタイプを設定する

ビルドタイプについての詳細は、 ビルドターゲットとビルドタイプ を参照してください。

設定を反映する

「OK」ボタンを押すと、設定がプロジェクトに反映されます。

また、プロジェクトファイルと同じディレクトリにプロパティシート ImportNintendoSdk.props 及び設定値を保存するためのファイル <プロジェクト名>.nnsdk.xml が生成されます。これらの 2 ファイルは削除しないでください。またプロジェクトファイルを移動する場合は、これらのファイルも一緒に移動してください。

ビルド確認を行う

ソリューション プラットフォームに NX32 または NX64 を選択してビルドが行えるかどうか確認してください。

NintendoSDK では Windows 環境と開発機環境とで異なるコンパイラツールチェインを使用していますので、ソースコードの修正が必要となる可能性があります。ソースコードを修正しても正常にビルドが行えない場合は、次の箇所を確認してください。

開発機環境用のビルド設定が使用されていることを確認する

NX32 及び NX64 プラットフォームのビルド時にプロジェクト固有のプリプロセッサマクロの定義、インクルードディレクトリ、ライブラリディレクトリ等の設定が正しく使用されていることを確認してください。

  • WIN32 といったプリプロセッサマクロが定義されている場合は正しいコンパイル結果が得られない可能性があります。
  • ..\LibraryProject\Win32\Release といったライブラリディレクトリを指定している場合はリンク時に NX 開発機用のライブラリではなく Windows 用のライブラリが検索される可能性があります。..\LibraryProject\NX32\Release に変更するなどの見直しを行ってください。

このチュートリアルでプラットフォームを追加する以前から NintendoSDK を使用していた場合は、リンクするライブラリファイルに注意が必要です。

  • NintendoSDK が提供するライブラリファイルの構成は Windows 環境と NX 開発機環境とで異なっており、Windows 環境で提供されるライブラリファイルが NX 開発機環境では存在しない場合があります。両環境で提供されるライブラリについては ライブラリの提供形態と利用方法 を参照してください。
  • NX 開発機環境用の静的ライブラリファイルの拡張子は .a です。Windows 環境用の .lib とは異なりますのでご注意ください。

NintendoSDK のビルドシステムの設定を上書きしていないことを確認する

NintendoSDK のプリプロセッサマクロの定義、インクルードディレクトリ、ライブラリディレクトリ等の設定は NintendoSDK のプロパティシートによって行われますが、プロジェクト内でこれらの設定が上書きされている場合に正しくビルドが行えない可能性があります。設定が上書きされているかどうかは次の手順で確認できます。

プロジェクトのプロパティ ページを開きます。例えば、ソリューション エクスプローラー上でプロジェクトを右クリックし、[プロパティ(R)] を選択します。

適当な設定のプルダウンメニューを開いて [<編集...>] を選択します。ここではインクルードディレクトリを選択しています。

表示されるウインドウで [親またはプロジェクトの規定値から継承(I)] にチェックが入っていない場合は、設定が上書きされています。チェックを入れてください。
