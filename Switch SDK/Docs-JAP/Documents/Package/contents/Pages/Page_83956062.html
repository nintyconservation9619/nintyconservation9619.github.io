インダイレクトテクスチャのテクスチャフォーマットによる挙動の違い

インダイレクトテクスチャのテクスチャフォーマットによる挙動の違いを知りたい

インダイレクトテクスチャは、テクスチャフォーマットによって（正確には、フォーマットのチャンネル数によって）挙動が変わります。

テクスチャチャンネル数 フォーマット 挙動
1チャンネル BC4、L8、A8 インダイレクトでずらす値は、st共に同じ値が使われます。
2チャンネル BC5、LA8 インダイレクトでずらす値は、Rがs、Aがtとして使われます。
3チャンネル BC1、RGB565_INDIRECT インダイレクトでずらす値は、Rがs、Gがtとして使われ、Bの値はインダイレクトをかけたあとにアルファで抜くときの値として使われます。
4チャンネル BC2、BC3、RGBA8 インダイレクトでずらす値は、Rがs、Gがtとして使われ、Aの値はインダイレクトをかけたあとにアルファで抜くときの値として使われます。
Bの値は使われません。

インダイレクトテクスチャを使用したペインでマスクまたはアルファの処理を最適化

インダイレクトテクスチャを使用したペインにマスクやアルファを適用したい場合に マルチテクスチャでアルファチャンネル用テクスチャをさらに追加しないといけないケースがあります。

マルチテクスチャの枚数を追加すると描画負荷が大きく上がってしまうので、 テクスチャを追加せず、アルファチャンネル付きのインダイレクトテクスチャを作成することで 描画負荷を増加を抑えることができます。

インダイレクトテクスチャとして、３チャンネル(BC1)または４チャンネル(BC3, RGBA8)の テクスチャフォーマットを利用すると、それぞれ、３チャンネル目(B成分)、４チャンネル目(A成分)が アルファチャンネルとして使用されます。

圧縮テクスチャの特性上、BC1テクスチャでは想定したようにアルファチャンネルが 再現できない可能性があります。その場合、サイズは増えてしまいますが、 BC3テクスチャなど４チャンネルのフォーマットを利用してください。
