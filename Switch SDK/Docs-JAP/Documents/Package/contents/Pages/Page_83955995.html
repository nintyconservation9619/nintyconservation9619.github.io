フォントコンバータ（コンソール版）

フォントコンバータ (コンソール版)
FontConverterConsole.exe

以降は、コマンドライン版固有の事項に関してのみ記述されています。

概要

FontConverterConsoleはFontConverter （以下GUI版と表記）のコマンドライン版です。
機能的にはGUI版と同じです。
Makefile内での使用やバッチファイルでの処理に使用することを目的としています。

フォントのライセンスについて

FontConverter には、いかなるフォントのライセンスも付属しません。
フォントを利用したソフトを発売するには別途フォントメーカーとのライセンス契約が必要になります。

コマンドラインオプション

FontConverterConsole.exe -tile-mode <タイルモード> -i <入力フォーマット> <入力オプション>
 -o <出力フォーマット> <出力オプション> [-f <文字フィルタファイルパス>] [<その他のオプション>]
オプション 必須 備考
-tile-mode <タイルモード> Yes タイルモード。linear または、Addon がインストールされたプラットフォームの名前。

Windows / OpenGL 向けバイナリを出力する際は、linear　を指定します。

Windows / OpenGL 向け以外のバイナリを出力する際は、プラットフォーム固有の情報 を参照してください。
-p <対象プラットフォーム> Yes Win または Addon がインストールされたプラットフォーム。廃止予定のオプションです。-tile-mode を利用してください。
-i <入力フォーマット> Yes image, win, ffntのいずれか。
-o <出力フォーマット> Yes image, bffntのいずれか。
-f <文字フィルタファイルパス> - 省略時はフィルタなし。
-ignore-filter -

指定すると ffnt 内のフィルタを無視します。-f オプションで明示的に指定したフィルタは無視されません。

このオプションは LayoutEditor が内部的に使用するもので、ユーザが指定する必要はありません。

-gpu-encoding <type>

-

テクスチャー変換の際に、GPU でエンコードするかどうかを指定します。
type は、true, false, auto で指定します。省略時は true になります。

指定したオプションは、内部で呼び出される 3Dテクスチャーコンバーターに渡されます。
詳細は、3Dテクスチャーコンバーターマニュアル の gpu-encoding を参照してください。
  • <入力オプション>と<出力オプション>はそれぞれ<入力フォーマット>, <出力フォーマット>の指定で変化します(後述)。
  • スペースを含むパラメータはダブルクオーテーションマーク(")で囲む事で指定できます。
    ダブルクオーテーションマークで囲まれている内部ではダブルクオーテーションマークを使用できません。

タイルモード

出力するフォントのタイルモードを指定します。
linear を指定すると、リニアタイリングのフォントを出力します。
linear  以外を指定する場合、対象プラットフォーム向けの Addon がインストールされている必要があります。

対象プラットフォーム

（廃止予定のオプションです。-tile-mode を利用してください）

出力するフォントの対象プラットフォームを指定します。
Win を指定すると、Generic 向けのデータ（タイリングがリニア）を出力します。
Win 以外を指定する場合、対象プラットフォーム向けの Addon がインストールされている必要があります。

入力オプション

入力：Image ( -i image )

オプション 必須 備考
-if <画像ファイルパス> Yes -
-io <文字順序ファイルパス> Yes 文字順序ファイル
-ic <カラーフォーマット> Yes A4, A8, LA4, LA8, LA4Packed, LA4PackedNoCompress, RGB565, RGB5A1,RGBA4, RGB8, RGBA8, RGBA8_BC7, RGB565_SRGB, RGB5A3_SRGB, RGBA8_SRGB, RGBA8_BC7_SRGB のいずれか
内部で以下のように解釈
(A4,A8 => BC4) (LA4,LA8 => BC5)
(LA4Packed => BC4(明度とアルファを特殊な形式で格納。圧縮形式の BC4 で格納する。))
(LA4PackedNoCompress => A8(明度とアルファを特殊な形式で格納。非圧縮の A8 で格納する。サイズがLA4Packedの倍になる。))
(RGB565,RGB8 => BC1)
(RGB5A1,RGB5A3 => BC3) (RGBA8 => RGBA8)
(RGBA8_SRGB => RGBA8) (RGB565_SRGB => BC1) (RGB5A3_SRGB => BC3)

..._SRGB と付くのは リニアガンマ用 フォーマットです。指定すると実行時に、sRGBフェッチされるようになります。
-ia - 指定すると線形補間対策処理を行う
チェックをつけるとグリフに対して線形補間による周辺色にじみ問題の対策処理を行います。
この処理はアルファチャンネルをもつ「カラーフォーマット」の場合のみ効果があります。

例

文字順序ファイルxlor/cp1252.xlor に従ってＢＭＰファイル font.bmp を入力として読み込みます。 BMPのカラーはLA8フォーマット用に解釈されます。

-i image -if font.bmp -io xlor/cp1252.xlor -ic LA8

入力：Windowsフォント ( -i win )

オプション 必須 備考
-in <フォントパス、もしくはインストール済みフォントのフォントフェース名> Yes

フォントファイルのファイルパスか、インストール済みフォントのフォントフェース名(例：Tahoma)を指定します。指定できるファイル形式は ttf, otf, bfttf, bfotf です。
フォントフェース名で指定する場合、PCにフォントをインストールしておく必要があります。また、フォントパスで指定した場合でも PC に同じフォントフェース名のフォントがインストールされている場合はインストール済みのフォントが使われます。

一部の otf, bfotf の読み込みにはフォントフェース名が必要になります。もし読み込みエラーが発生した場合はエラー内容に従って -iface オプションによりフォントフェース名を指定してください。
-isysfont <本体内蔵フォント（bfttf）ファイルパス> - 入力に、本体内蔵フォントを使うときに指定可能です。本オプション指定時には、-inオプションは無視されます。本オプションは互換性のために残されているものですので、通常は本体内蔵フォントを読み込む場合にも -in オプションをお使いください。
-is <フォントサイズ> Yes ピクセル単位で指定。
-isw <平均幅> - ピクセル単位で指定。
-iwg <太さ> - 0 - 900 で指定。400 が標準。700 が Bold。
-ic <階調数> Yes A6 A8 LA6 LA8 のいずれか。輝度とアルファを別画像にしたい場合は LA を選ぶ。
A4 LA4 は 非推奨(階調数が減少しますがフォントのメモリサイズは変わりません)。
A6 LA6 は A8 LA8 を指定したのとまったく同じ動作。
A1, LA1は廃止。
内部で以下のように解釈。
(A4 A6 A68 => BC4) (LA4 LA6 LA8 => BC5)
-ia - 指定するとソフトアンチエイリアス有効
-isck -

指定すると カーニング抽出を nn::fontll を使って行い、nn::fontll との表示差異が出にくくなります。
｢bffnt の描画｣と ｢ttf/otf の描画」で、微妙な表示差異が問題になる場合に利用します。
-it <文字幅> - glyph keepsp char fixed のいずれか。省略時はchar。
-iw <等幅幅> - ピクセル値を指定。-it fixed 指定時に必須。それ以外の場合の指定は無効。
-iface <フォントフェース名> -

-in オプションにファイルパスを指定した場合、一部の otf, bfotf の読み込みにはフォントフェース名が必要になり、読み込みに失敗することがあります。読み込みエラーが発生した場合には本オプションによりフォントフェース名を指定してください。

例

Windowsフォント 「ＭＳ ゴシック」 をサイズ16 ｐｉｘｅｌ 、64階調アルファのみ(=A6)で読み込みます。文字幅にはグリフ幅を使います。

-i win -in "ＭＳ ゴシック" -is 16 -ic A6 -it glyph

出力オプション

出力：Image ( -o image )

オプション 必須 備考
-of <画像ファイルパス> Yes -
-oo <文字順序ファイルパス> Yes 文字順序ファイル
-oi <画像形式> - -
-ia - ext bmp tga tiff のいずれか。extなら画像ファイルの拡張子を元に判定。省略時はext。
-og - 指定するとグリッドを引かない。
-ow <セル幅> - 省略時は十分に大きな値と解釈。-or -ob とは同時に指定不可。
-oh <セル高> - 省略時は十分に大きな値と解釈。-or -ob とは同時に指定不可。
-ol <セル左マージン幅> - 省略時は0と解釈。
-or <セル右マージン幅> - 省略時は -ol と同じ値と解釈。-ow -oh とは同時に指定不可。
-ot <セル上マージン幅> - 省略時は0と解釈。
-ob <セル下マージン幅> - 省略時は -ot と同じ値と解釈。-ow -oh とは同時に指定不可。
-ocg <グリッド色> - 色を16進6桁の数で指定。省略時は000000。
-ocm <余白色> - 色を16進6桁の数で指定。省略時は99AA99。
-ocw <幅線色> - 色を16進6桁の数で指定。省略時はFF0000。
-ocn <　null色> - 色を16進6桁の数で指定。省略時はFF7F7F。

例

文字順序ファイル xlor/cp1252.xlor に従ってセルごとに上下に3ピクセルずつ、左右に1ピクセルずつのマージンを持たせ、グリッドなしでファイル font.bmp に出力します。

-o image -of font.bmp -oo xlor/cp1252.xlor -og -ol 1 -ot 3

出力：bffnt ( -o bffnt)

オプション 必須 備考
-of <出力ファイルパス> Yes 指定通りのファイル名で出力。
-oe <文字コード> - unicode sjis cp1252 のいずれか。省略時にはunicode。
-op <グリフグループファイル> - 廃止オプション。省略時はグループなし(無圧縮)となります。
-os <シートサイズ> - シートサイズをキロピクセル単位で指定。16以上8192以下の2の累乗。省略時には自動で決定。
-oa <代替文字> - 省略時は入力に応じたデフォルト値。
-oh <改行幅> - 省略時は入力に応じたデフォルト値。
-ol <デフォルト左スペース> - 省略時は入力に応じたデフォルト値。
-ow <デフォルトグリフ幅> - 省略時は入力に応じたデフォルト値。
-or <デフォルト右スペース>> - 省略時は入力に応じたデフォルト値。
-ok - カーニング情報を出力するかどうか。省略時は出力します(注意：1.11.2 より変更されました)。
指定した際、入力にカーニング情報が無い場合は警告を表示します。
-onok - カーニング情報の出力を禁止します。

例

シートサイズ256Kピクセル、代替文字 「?」、対応エンコーディング ShiftJISで font.bffnt に出力します。

-o bffnt -of font.bffnt -os 256 -oe sjis -oa ?

その他のオプション

オプション 必須 備考
-usedtd - DTD ファイルによる、入力ファイルのバリデーションを有効にします。
-silent - ログの出力を抑止します。
