ライブラリ

概要

PC 版 OpenGL ではヘッダファイルのみが提供されます。 ライブラリの静的リンクは不要で OpenGL API のエントリポイントはランタイムで WGL や GLEW 等を利用してグラフィックスドライバから関数ポインタを取得します。具体的な使用方法は利用されている PC の GPU ベンダによって異なりますので OpenGL を参照して下さい。

NX 版 OpenGL ではヘッダファイルに加えて、専用のライブラリ(EGL)が提供されます。

NX 版 OpenGL では非拡張の OpenGL ES 3.2 に存在する関数をエクスポートします。このため、OpenGL ES 3.2 をターゲットにしている場合は、これらの関数を静的にリンクすることができます。 別の OpenGL のバージョンをターゲットにしている場合は、必要な OpenGL 関数をすべて動的にリンクする必要があります。 OpenGL バージョンの不一致による関数定義を使用している場合、警告は表示されません。 このため、対象とする OpenGL バージョンのすべての関数を動的にリンクすることをお勧めします。

関数を動的にリンクするためには、まず eglGetProcAddress により関数ポインタを取得し、それから取得された関数ポインタを通して関数を呼び出す必要があります。この関数ポインタのロード作業は、下記で説明される GLEW ライブラリやその他の OpenGL Loading Library に任せることも可能です。OpenGL 版の gfx では、デバイスの初期化時に GLEW を使ってこのロード作業が行われます。

ヘッダファイル

NintendoSDK NX Addon は gl.h 等のヘッダを提供しています。

その他の必要なヘッダも合わせて、以下のディレクトリに配置されます。

$NINTENDO_SDK_ROOT/Include/EGL
$NINTENDO_SDK_ROOT/Include/GL
$NINTENDO_SDK_ROOT/Include/GLES2
$NINTENDO_SDK_ROOT/Include/GLES3
$NINTENDO_SDK_ROOT/Include/KHR

The OpenGL Extension Wrangler Library

NintendoSDK NX Addon は OpenGL でグラフィックスドライバから関数ポインタを取得するためのライブラリ The OpenGL Extension Wrangler Library (GLEW) を提供しています。

利用するために必要なヘッダ、ライブラリは以下のディレクトリに配置されています。

$NINTENDO_SDK_ROOT/Externals/glew

NX 版で GLEW を使うには libglew.a をリンクしてください。

注意：

GLEW では OpenGL 1.x から存在する関数（例：glPolygonMode, glLogicOp）を動的に取得しません。これらの関数を NX で利用したい場合は、eglGetProcAddress を使用して関数ポインタを取得してください。
