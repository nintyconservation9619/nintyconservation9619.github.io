 nn::g3d ニュース

バイナリーリソースファイルのフォーマット更新

NXAddon3.0.0 で bfres および bfsha に予約領域が追加されましたが、CPU負荷に影響を与えるケースが見られましたので、NXAddon3.1.0 で再度 bfres ファイルおよび bfsha ファイルの予約領域のサイズを調整しました。

これにより、以前のバージョンの 3D バイナリーコンバーターおよび 3D シェーダーコンバーターで作成したバイナリーファイルとの互換が切れますので、リソースの再コンバートが必要です。

中間ファイルコンバーターでモデル中間ファイルの --magnify オプションに対応

中間ファイルコンバーター にモデル中間ファイルを入力し --magnify オプションを指定することで、モデルの拡大・縮小ができるようになりました。

過去に利用していたモデル中間ファイルを新規プロジェクトで再利用する場合などに利用してください。

nn::vfx ニュース

フラグメントシェーダにて out 変数を利用するように変更

標準フラグメントシェーダにて gl_FragData の利用を辞め out 変数への置き換えを行いました。

カスタムシェーダ内で gl_FlagData を利用されている場合、シェーダコンパイルエラーとなる為、カスタムシェーダ側で out 変数 を利用するよう変更をお願いします。

 nn::ui2d ニュースえ

ピクチャペインに四角形以外のメッシュ形状を設定する機能を追加

 ピクチャペインの「形状」設定から四角形以外のメッシュ形状を設定する機能を追加しました。「角丸四角形」と「円」を設定することができ、それぞれの形状にはメッシュの分割数などで形状を調整することができます。

  機能を利用する場合、GpuBuffer 内の領域にメッシュ表示のための頂点バッファとインデックスバッファを確保しますので、GpuBuffer に設定する GpuAccessFlag へ nn::gfx::GpuAccess_ConstantBuffer に加え、nn::gfx::GpuAccess_VertexBuffer と nn::gfx::GpuAccess_IndexBuffer を設定してください。また、この機能追加により libnn_ui2d を使用する際は libnn_gfxUtil のリンクが必要になります。

nn::ui2d::GraphicsResource::Setup に 外部メモリ-プールを受け取る機能を追加

  nn::ui2d::GraphicsResource::Setup に 外部メモリ-プールを受け取るオーバーロード関数を追加しました。
利用の仕方については、Samples/Sources/Applications/Ui2dDemo/main.cpp の InitializeGraphicsResource() を参考にしてください。

 nn::font ニュース

nn::font::RectDrawer::Initialize に 外部メモリ-プールを受け取る機能を追加

nn::font::RectDrawer::Initialize に 外部メモリ-プールを受け取るオーバーロード関数を追加しました。
利用の仕方については、Samples/Sources/Applications/FontDemo/main.cpp の InitRectDrawer() を参考にしてください。

nn::font::WordWrapping::CalculateWordWrapping() で正しい折り返し位置が計算されないことがある不具合の修正

6 個の引数を受け取る nn::font::WordWrapping::CalculateWordWrapping() を使用すると、実機で正しい折り返し位置が計算されないことがある不具合を修正しました。

文字列長を計算する処理に不具合があり、渡した文字列の途中までしか解釈されなかったり、文字列の長さを超えて解釈される場合がありました。

文字列長を明示的に指定する 7 個の引数を受け取る同名のオーバーロード関数を使用していた場合は問題ありません。

NON-BREAKING-HYPHEN の代わりに HYPHEN-MINUS のグリフを使って描画できる機能の追加

一部のプラットフォームでは NON-BREAKING-HYPHEN のグリフが内蔵フォントに存在しないため、それらを正しく表示するために、NON-BREAKING-HYPHEN(U+2011) の代わりに HYPHEN-MINUS(U+002D) のグリフを使って描画できる機能を追加しました。

nn::font::ScalableFont::InitializeArg::isDrawingHyphenMinusInsteadOfNonBreakingHyphen を true にすることで、本機能は有効になります。

デフォルトで true になるため、NON-BREAKING-HYPHEN を使っていて表示の違いが問題になる場合には false に設定してください。
