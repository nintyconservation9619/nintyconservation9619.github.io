nmeta ファイルとは

nmeta ファイルはアプリケーションや追加コンテンツの設定を記述するためのファイルです。

XML 形式のフォーマットで、 BOM 付き UTF8 で記述します。

AuthoringEditor は GUI を用いて nmeta ファイルを生成、編集出来ます。


参考：

拡張子が .nmeta に変わりましたが、今まで通り .meta 拡張子もご利用いただけます。

現状は両対応しておりますが、将来的には .meta 拡張子のサポートを打ち切る予定ですので今のうちからファイル名の置換作業をしておくことをおすすめします。


NX Addon 3.0 から nmeta ファイルのルートディレクティブのデフォルトが Meta から NintendoSdkMeta に変わりました。

Meta も引き続きご利用になれますが、 サンプルやドキュメントなどの nmeta ファイルは NintendoSdkMeta に置き換わっているのでご注意ください。


アプリケーション用の nmeta ファイル

アプリケーション用の nmeta ファイルは、 npdm ファイルの作成と nca/nsp ファイルの作成時に利用されます。詳細は独自のビルドシステムを構築する (NX 開発機環境)を参照してください。

テンプレートとなる nmeta ファイルが次の場所に配置されています。各アプリケーションごとにコピーして編集してください。

アプリケーションの種類 nmeta ファイル
64bit アプリケーション %NINTENDO_SDK_ROOT%\Resources\SpecFiles\Application.aarch64.lp64.nmeta
32bit アプリケーション %NINTENDO_SDK_ROOT%\Resources\SpecFiles\Application.arm.ilp32.nmeta

Visual Studio を使用している場合は、編集したファイルを プロジェクトで指定できるパラメータ を参考にビルド時に参照するようにしてください。なお NX Addon 1.0.0 以降の場合、Visual Studio のウィザードから新規アプリケーションプロジェクトを作成すると、プロジェクトファイルと同じディレクトリにテンプレートとなる nmeta ファイルが自動作成され、ビルド時に参照するよう設定済みの状態となっています。

アプリケーション用の nmeta ファイルには、大きく分けて三つの設定項目があります。

  • Core
    • アプリケーションを起動・実行する際に必要な設定をします。
  • Application
    • アプリケーション管理データです。
    • アプリケーションが利用するシステムの設定をします。
  • CardSpec
    • アプリケーションを NX カードに書き込む際のパラメータ設定をします。
注意：

現在、 VSI のパラメータで「アプリケーションプログラムフォーマット」を RAWに設定している場合、アプリケーション管理データを反映させるために、AuthoringTool の createnspmeta コマンドを実行する必要があります。

$ AuthoringTool.exe createnspmeta -o sample.nspd --meta $NINTENDO_SDK_ROOT/Resources/SpecFiles/Application.aarch64.lp64.nmeta --type Application

このコマンドを「ビルド後のイベント」に登録しておくと実行し忘れがなくなります。

コマンドについての詳細は AuthoringTool にある「 nspd に配置するメタデータを作成する 」を参照してください。

この手順は将来修正される可能性があります。


Core の設定項目

アプリケーション ID

  • アプリケーション毎に割り当てられた固有の ID です。
  • 要素名: ApplicationId
  • 設定値
    • 64 bit ID 文字列
  • 設定する値はアプリケーション ID を参考にしてください。

アプリケーション ID は必須項目です。

メインスレッドのスタックサイズ

  • アプリケーションのメインスレッドが持つスタックのサイズを指定します。
  • 要素名: MainThreadStackSize
  • 設定値
    • UInt32 の数値を表す文字列
      • 16 進数で指定してください。
  • デフォルト値は 0x100000 です。

プロセスのアドレス空間

  • アプリケーションのアドレス空間を指定します。
    • アドレス空間サイズの違いや予約領域については アプリケーションの動作環境（開発機環境） を参照してください。
  • 要素名: ProcessAddressSpace
  • 設定値
    • AddressSpace32Bit
      • 32 bit アドレス空間を利用します。
      • Application.arm.ilp32.nmeta を利用するときに指定できます。
    • AddressSpace32BitNoReserved
      • 32 bit アドレス空間を利用します。
      • 予約領域をメモリヒープ用の領域として利用します。
      • Application.arm.ilp32.nmeta を利用するときに指定できます。
    • AddressSpace64Bit
      • 64 bit アドレス空間を利用します。
      • Application.aarch64.lp64.nmeta を利用するときに指定できます。
  • 利用する nmeta ファイルに最初から設定されています。
  • AddressSpace32BitNoReserved を指定するとき以外に、この設定項目を修正しないでください。

システムリソースサイズ

  • システムが利用できるメモリサイズを指定します。
    • この値を 0 より大きい値に設定するとアプリケーションが利用できるメモリを減らし、代わりにシステムが利用できるメモリを増やします。
    • アドレス空間が AddressSpace64Bit の場合にのみ利用できます。
  • 要素名: SystemResourceSize
  • 設定値
    • UInt32 の数値を表す文字列
      • 16 進数で指定してください。
      • 0x200000 の整数倍で指定してください。
  • デフォルト値は 0 です。

ファイルシステム API 利用権限

  • 特定のファイルシステム API の利用権限を設定します。
  • 現時点では以下の場合を除いて、特に設定項目の記述は必要ありません。
    • アプリケーションの動作確認のために FS ライブラリのデバッグ API を利用できない環境を準備するとき
      • 詳細については FS ライブラリ／開発向けオーバービュー の「デバッグ API の利用について」を参照してください。
    • 他のアプリケーションのセーブデータをマウントするとき
      • 詳細については セーブデータ (NX Addon) の「他アプリケーションのセーブデータのマウント」を参照してください。
      • 他社のアプリケーションのセーブデータを設定するには事前の申請が必要です。任天堂の窓口までご相談ください。

Application の設定項目

タイトル

  • 本体機能に表示されるアプリケーションのタイトル名です。
  • 複数の言語ごとにタイトルを設定することが出来ます。
  • 一つのタイトルの設定には、タイトルの名前、タイトルの言語、パブリッシャー名が含まれます。
  • 要素名: Title
  • 設定値
    • タイトルの名前 (要素名: Name)
      • 名前の最大長は 127 文字です。
      • タイトルの文字列にハングル、簡体字、繁体字を指定することはできません。
      • 本体機能ガイドの「共有データ ／ 内蔵フォント」にある日米欧フォントが対応している文字コード以外の文字は受け付けません。
    • タイトルの言語 (要素名: Language)
      • AmericanEnglish, BritishEnglish, Japanese, French, German, LatinAmericanSpanish, Spanish, Italian, Dutch, CanadianFrench, Portuguese, Russian, SimplifiedChinese, TraditionalChinese, Korean
      • タイトルを指定するときには必ずタイトルの言語を設定してください。
    • パブリッシャー名 (要素名: Publisher)
      • パブリッシャー名の最大長は 63 文字です。
      • パブリッシャーの文字列にハングル、簡体字、繁体字を指定することはできません。
      • 本体機能ガイドの「共有データ ／ 内蔵フォント」にある日米欧フォントが対応している文字コード以外の文字は受け付けません。
  • デフォルト値は全て空文字になります。
  • パッケージを提出するときにはすべての項目が設定されている必要があります。
  • タイトルに設定されていない言語が本体設定で設定されている場合、設定されている言語の中から適切なものが選択されて表示されます。
    • 選択される言語の優先度については、本体機能ガイドの「 表示言語の仕組み 」を参照してください。
参考：

中国語（簡体字）、中国語（繁体字）、韓国語は NX Addon 3.5.3 以降から利用することが出来ます。

参考：

NintendoSDK Firmware for NX 3系での動作確認が必要な場合は、 中国語（簡体字）、中国語（繁体字）、韓国語設定のみで動作させるとエラーになるので中国語（簡体字）、中国語（繁体字）、韓国語以外の言語設定も追加してください。


アイコン

  • 本体機能に表示されるアイコンです。
    • アイコンの詳細は アプリケーションイメージに含めるアイコンについて を参照してください。
  • 複数の言語毎にアイコンのパスを設定することが出来ます。
    • 選択される言語の優先度については、本体機能ガイドの「 表示言語の仕組み 」を参照してください。
  • タイトルに設定した言語毎に設定する必要があります。
  • 要素名: Icon
  • 設定値
    • アイコンの言語 (要素名: Language)
      • AmericanEnglish, BritishEnglish, Japanese, French, German, LatinAmericanSpanish, Spanish, Italian, Dutch, CanadianFrench, Portuguese, Russian, SimplifiedChinese, TraditionalChinese, Korean
    • 大きなアイコンのパス (要素名: IconPath)
      • nmeta ファイルがあるディレクトリからの間接参照パスか絶対パスを設定してください。
    • 小さなアイコンのパス (要素名: NxIconPath)
      • nmeta ファイルがあるディレクトリからの間接参照パスか絶対パスを設定してください。
      • 通常はこの項目を指定しないでください。
  • デフォルトは設定なしです。
  • IconPath, NxIconPath タグの属性値として UseEnvironmentVariable="true" を設定した場合、設定値に %環境変数名% の形式で環境変数を記述することができます。
参考：

中国語（簡体字）、中国語（繁体字）、韓国語は NX Addon 3.5.3 以降から利用することが出来ます。

参考：

NintendoSDK Firmware for NX 3系での動作確認が必要な場合は、 中国語（簡体字）、中国語（繁体字）、韓国語設定のみで動作させるとエラーになるので中国語（簡体字）、中国語（繁体字）、韓国語以外の言語設定も追加してください。

リリースバージョン

  • アプリケーションのリリースバージョンを設定します。
  • アプリケーション、パッチを Nintendo Developer Portal に提出する際には Nintendo Developer Portalで発行されたリリースバージョンをこちらに設定してください。
  • 要素名: ReleaseVersion
  • 設定値
    • UInt16 の数値を表す文字列を指定します。
  • デフォルト値は「0」です。


バージョン表記

  • システムに表示されるアプリケーションのバージョンです。
  • 要素名: DisplayVersion
  • 設定値
    • システムでは「Ver.」と表示されるので、それ以降の英数字(例:3.1.5b)を設定します。
    • ASCII 文字列 1文字以上、15 文字以下
  • 複数言語には対応していません。
  • 設定された文字列は、本体機能の本体設定内のデータ管理 ＞ソフトの詳細管理 でソフトを選ぶと詳細画面に表示されます
  • リリースバージョンが異なるアプリケーション、パッチには、必ず異なる文字列を設定してください。
  • デフォルト値は全て空文字になります。


起動時本体アカウント選択

  • アプリ起動時にシステムが本体アカウントを選択させるために使用します。
  • 要素名: StartupUserAccount
  • 設定値
    • Required
      • アプリが本体アカウントを選択しなくても起動時から本体アカウントが決定されています。アプリが本体アカウント選択をハンドリングする必要がなくなります。 
    • RequiredWithNetworkServiceAccountAvailable
      •   アプリが起動した時点で選択されている本体アカウントにニンテンドーアカウントが紐付いていることが保証されます。
      • ニンテンドーアカウントが紐付いていないアカウントからはアプリケーションが起動できなくなります。
    • None
      • 起動時に本体アカウントについてシステムは何も保証しません。
  • デフォルト値は「None」です。


ペアレンタルコントロール設定

  • アプリケーションが使用するペアレンタルコントロール要素を明示するために使用されます。
    • 要素を設定すると、ペアレンタルコントロールの設定画面にアプリケーションが列挙され、有効・無効の選択ができるようになります。
    • 実際に要素機能を有効・無効にする処理はアプリケーションで実装する必要があります。
    • 設定した内容が正しく実装に反映されているかはロットチェックのときに検証されます。
  • 要素名: ParentalControl
  • 設定値
    • FreeCommunication
      • 他の人との自由なコミュニケーションを利用します。
  • デフォルト値は「設定なし」です。

レーティング

  • システム が対象年齢に満たないユーザによるアプリの起動を制限するために使用されます。
  • 複数のレーティング団体を設定できます。
  • 一つのレーティング団体の設定には、レーティング団体名と年齢が含まれます。
  • 要素名: Rating
  • 設定値
    • レーティング団体名 (要素名: Organization)
      • レーティングを設定するときには必ず設定してください。
      • 以下の中から設定する値を指定してください。
        • CERO・ GRACGCRB ・ GSRMR ・ESRB・ ClassInd ・USK・PEGI・PEGIPortugal・PEGIBBFC・Russian・ACB・OFLC
    • 年齢 (要素名: Age)
      • レーティングを設定するときには必ず設定してください。

      • Int8 の数値を表す文字列を指定してください。

対応言語

  • 対応言語を選択しておくと、システムの言語設定が対応言語以外に設定されている場合に、アプリには特定の優先順位に従って対応言語の１つが設定されているように見えます。
    • 要素名: SupportedLanguage

    • 設定値

      • AmericanEnglish, BritishEnglish, Japanese, French, German, LatinAmericanSpanish, Spanish, Italian, Dutch, CanadianFrench, Portuguese, Russian, SimplifiedChinese, TraditionalChinese, Korean

    • nn::oe::GetDesiredLanguage() を用いると対応言語のうち、システムの言語設定で優先度の最も高い言語を取得することができます。

  • 必ず 1 つ以上の対応言語を設定する必要があります。   

  • 選択される言語の優先度については、本体機能ガイドの「 表示言語の仕組み 」を参照してください。


参考：

中国語（簡体字）、中国語（繁体字）、韓国語は NX Addon 3.5.3 以降から利用することが出来ます。

参考：

NintendoSDK Firmware for NX 3系での動作確認が必要な場合は、 中国語（簡体字）、中国語（繁体字）、韓国語設定のみで動作させるとエラーになるので中国語（簡体字）、中国語（繁体字）、韓国語以外の言語設定も追加してください。


プレゼンスグループ ID

  • 複数アプリケーション間でフレンドプレゼンスを共有するために使用される ID です。
    • プレゼンスグループ ID の詳細は プレゼンスグループ ID を参照してください。
  • 要素名: PresenceGroupId
  • 設定値
    • 64 bit ID の文字列を指定してください。
  • デフォルト値は「ApplicationId」です。


スクリーンショット撮影

  • ゲームのスクリーンショットを撮影可能かを設定できます。
  • 本設定もしくは API による制御のどちらかが「不許可」であれば撮影不許可になります。
    • API については、 動作環境制御機能 を参照してください。
  • 要素名: Screenshot
  • 設定値
    • Allow
      • ゲーム内のスクリーンショットを許可します。
    • Deny
      • ゲーム内のスクリーンショットを不許可にします。
  • デフォルト値は「Allow」になります。


動画撮影

  • アプリケーションの動画撮影を有効にするか否かを設定できます。
  • アプリケーションの動画撮影は以下に示す全ての条件が満たされる場合にのみ許可されます。
    • 本設定が Enable である
    • スクリーンショット撮影（"Screenshot"）の設定が Allow である
    • スクリーンショット撮影の API で撮影が許可されている
  • 要素名: VideoCapture
  • 設定値
    • Enable
      • アプリケーションの動画撮影を有効にします。
    • Disable
      • アプリケーションの動画撮影を無効にします。
  • デフォルト値は「Enable」になります。
  • 本設定で動画撮影を有効にすると以下のようになります。
    • NX64 環境の場合、アプリケーションで使用できるメモリ総量が 96MiB だけ減少します。
    • NX32 環境の場合、アプリケーションで使用できるメモリ総量は減少しません。

ロゴデータの設定

  • ホームメニューでロゴを表示する際に用いるライセンスの情報を設定します。
  • 要素名: LogoType
  • 設定値
    • LicensedByNintendo
      • 一般のアプリケーションのライセンスです。
    • Nintendo
      • 任天堂タイトルのライセンスです。
  • デフォルト値は「LicensedByNintendo」です。


アプリケーション開始時のロゴ表示の停止設定

  • アプリケーションの開始時に表示されるロゴ表示を自動で停止するか手動で停止するかを設定します。
  • 要素名: LogoHandling
  • 設定値
    • Auto
      • ロゴ表示を自動で停止します。
    • Manual
      • ロゴ表示を自動では停止しません。
      • 適切なタイミングで nn::oe::FinishStartupLogo() を呼ぶ必要があります。
  • デフォルト値は「Auto」になります。

アプリケーション終了時のデータ消失確認

  • アプリケーションを終了する際に、未セーブの内容が失われる旨の文言を表示するかを切り替えます。
  • 要素名: DataLossConfirmation
  • 設定値
    • Required
      • データ消失確認のダイアログが表示されます。
    • None
      • データ消失確認のダイアログが表示されません。
      • アプリケーションがデータ消失が発生しないように気を付ける必要があります。
  • デフォルト値は「None」になります。


アプリケーションエラーコードのカテゴリ

  • アプリケーションエラーコードを表示する場合のカテゴリに使用する値です。
    • アプリケーションエラーコードとは、アプリケーションがエラービューアを用いてそのアプリケーション独自のエラーを表示する際のエラーの識別子となるコードです。全体として「2-AAAAA-BBBB」というフォーマットをとり、AAAAA の部分がカテゴリに相当します。
  • エラービューアを用いてアプリケーション独自のエラーを表示するアプリケーションのみ設定が必要な項目です。
  • 要素名: ApplicationErrorCodeCategory
  • 設定値
    • ５文字の文字列。
    • 通常はアプリケーションのイニシャルコードを指定します。
    • 例外として、シリーズ共通で同一のカテゴリを使用したい場合には、他のアプリケーションのイニシャルコードを指定しても構いません。
  • デフォルト値は空文字列です。

ユーザーアカウントセーブデータのデータ保存領域サイズ

  • ユーザーアカウントセーブデータのデータ保存領域のバイトサイズを設定します。
  • 要素名: UserAccountSaveDataSize
  • 設定値
    • Int64 の数値を表す文字列を指定します。
      • 16 進数で指定してください。
  • デフォルト値は 0 です。この場合ユーザーアカウントセーブデータが作成されません。
  • ユーザーアカウントセーブデータのデータ保存領域サイズとジャーナリング領域サイズに特定の値以上を設定する場合は事前の申請が必要です。 セーブデータ を参照してください。
  • ユーザーアカウントセーブデータの詳細については セーブデータ (NX Addon) を参照してください。
  • NX Addon 1.x.x では、パッチ作成時にこの値を変更することはできません。変更した場合はパッチ作成時にエラーになります。

  • ユーザーアカウントセーブデータのデータ保存領域サイズとジャーナリング領域サイズに特定の値以上を設定する場合は事前の申請が必要です。 セーブデータ を参照してください。

インデックス付きキャッシュストレージのサイズ上限

  • この設定項目が関わる機能の利用には事前の申請が必要です。任天堂の窓口までご相談ください。
  • インデックス付きキャッシュストレージのデータ保存領域とジャーナリング領域の合計の上限バイトサイズを設定します。
  • 要素名: CacheStorageDataAndJournalSizeMax
  • 設定値
    • Int64 の数値を表す文字列を指定します。
      • 16 進数で指定してください。
  • デフォルト値は 0 です。この場合、インデックス付きキャッシュストレージを作成することができません。
  • インデックス付きキャッシュストレージの詳細については インデックス付きキャッシュストレージ を参照してください。

フィルタ記述ファイル

  • リソースデータファイルをフィルタアウトするためのフィルタ記述ファイルを指定します。
  • 要素名: FilterDescriptionFilePath
  • 設定値
    • フィルタ記述ファイルのパスを記述します。
      • 絶対パスの指定が可能です。
      • 相対パスは、AuthoringTool 実行時のディレクトリから相対パスとなります。
  • フィルタ記述ファイルの詳細は「AuthoringTool」を参照してください。
  • この要素を指定しなかった場合は、AuthoringTool 実行時に --filter オプションでフィルタ記述ファイルを指定することができます。
    • --filter オプションの指定と、nmeta での FilterDescriptionFilePath 要素の指定を同時に行う場合はエラーとなるので注意してください。

追加コンテンツの、アプリケーション実行時インストール制御

  • アプリケーションを実行している裏で新たに購入・ダウンロードした追加コンテンツを、アプリケーションを実行したまま利用できるようにするかどうかを指定します。
  • NX Addon 4.2.0 以降利用可能な設定です。
  • 要素名: RuntimeAddOnContentInstall
  • 設定値
    • AllowAppend
      • アプリケーション実行中に、新しくダウンロードした追加コンテンツを利用できるようになります。
    • Deny
      • アプリケーション実行中には、利用可能な追加コンテンツが変化しないようになります。
      • 新しくダウンロードした追加コンテンツを利用できるようになるのは、アプリケーションを再起動した場合です。
  • 規定値は Deny です。
  • 本設定のより詳しい説明は、アプリケーション実行中に、バックグラウンドでダウンロードした追加コンテンツを利用できるようにする方法 を参照してください。

クラッシュレポート

  • アプリケーションがクラッシュレポートを保存してよいかを設定します。
    • クラッシュレポートの詳細な説明は、「クラッシュレポート」を参考にしてください。
  • NX Addon 4.4.0 以降利用可能な設定です。
  • 要素名: CrashReport
  • 設定値
    • Allow
      • クラッシュレポートの保存を許可します
    • Deny
      • クラッシュレポートの保存を許可しません
  • デフォルト値は Deny です。

プレイ情報の取得権限

  • この設定項目が関わる機能を他社のアプリケーションに対し利用する場合には事前の申請が必要です。任天堂の窓口までご相談ください。
  • NX Addon 5.0.0 以降利用可能な設定です。
  • プレイ情報の取得権限を設定します。
    • プレイ情報の詳細については「PDM ライブラリ」を参照してください。
  • 要素名: PlayLogQueryCapability
  • 設定値
    • None
      • 自アプリケーション以外のプレイ情報の取得はできません。
    • WhiteList
      • 自アプリケーションに加えて後述する PlayLogQueryableApplicationId で指定したアプリケーションのプレイ情報が取得できます。
  • 規定値は None です。

プレイ情報の取得対象のアプリケーション ID

  • この設定項目が関わる機能を他社のアプリケーションに対し利用する場合には事前の申請が必要です。任天堂の窓口までご相談ください。
  • NX Addon 5.0.0 以降利用可能な設定です。
  • プレイ情報の取得対象とする自アプリケーション以外のアプリケーション ID を 設定します。
  • 要素名: PlayLogQueryableApplicationId
  • 設定値
    • アプリケーション ID を表す 64 bit ID 文字列
      • 最大 16 個まで設定できます。
      • 設定したアプリケーション ID のプレイ情報の取得には併せて PlayLogQueryCapability を WhiteList に設定する必要があります。
  • 規定値は設定無しです。

CardSpec の設定項目

参考：

CardSpec の設定項目は記述を省略することもできます。詳しくは NX 開発カード を参照してください。

容量

  • NX 開発カードの利用する容量を設定します。
  • 要素名: Size
  • 設定値
    • 1, 2, 4, 8, 16, 32 のいずれか
  • デフォルト値
    • アプリケーションデータのサイズに応じて自動設定されます。詳しくは NX 開発カード を参照してください。

動作周波数

  • NX 開発カードの動作周波数を設定します。
  • 要素名: ClockRate
  • 設定値
    • 25, 50 のいずれか
    • 容量の設定値に応じて、取り得る値に制限があります。
  • デフォルト値
    • アプリケーションデータのサイズに応じて自動設定されます。詳しくは NX 開発カード を参照してください。

NX 開発カード に記載されているとおり、アプリの容量が 4GB 以上かどうかによってゲームカードの物理的な速度が変わります。
自動設定から変更して速度を確認するパターンとして、例えば以下のようなもの考えられます。(設定が残らないようご注意ください)

4GB のアプリで 8GB 以上のカードの速度をエミュレートしたい場合
  <CardSpec>
    <Size>8</Size>
    <ClockRate>50</ClockRate>
  </CardSpec>
16GB のアプリで 4GB 以下のカードの速度をエミュレートしたい場合
  <CardSpec>
    <Size>16</Size>
    <ClockRate>25</ClockRate>
  </CardSpec>
