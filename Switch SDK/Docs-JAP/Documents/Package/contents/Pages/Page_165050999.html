はじめに

ポインティングモードは、 画面の付近に置いた専用の再帰性反射材バーをモーションIRカメラで検出して、画面をポインティングするモードです。

注意：

使用する再帰性反射材バーが、本ライブラリが要求する仕様を満たしていない場合、原則ポインティングモードのサポート対象外となります。
特に製品版で使用する場合は、マスタリングの際に任天堂のモーションIRカメラ開発チームの確認が必要となりますので、必ず事前に開発チームにお問い合わせください。

ポインティングモード

概要

反射材を 3 点並べた反射材バーを検出します。その 3 点の座標から中心点をポインティング座標として出力します。

参考：

入射した光を入射してきた方向に効率的に反射させる構造を持った素材のことで、モーションIRカメラのような点光源からの発光をその付近のIRカメラで捉える構成において、特徴点となるマーカーとして適しています。
ポインティング機能においては、反射効率がよいほど反射材バーから離れた位置からでも検出できるようになります。

出力されるデータの座標系については、「IR センサーのデータ座標系」を参照してください。

設定項目

ポインティングモードにはアプリ側から設定する項目がなく、コントローラと反射材バーの距離の変化などに伴うカメラの明るさ調整などの対応はシステム側で自動設定します。

 単純に nn::irsensor::RunPointingProcessor 関数を呼び出すことでポインティングモードを起動することができます。

出力フォーマット

ポインティングモードの出力は以下の様なフォーマットになります。

項目 説明 値域
samplingNumber 処理結果を受け取る度に増加する値です。 0 以上
timeStamp

前回取得したデータからの経過時間です。

 0 以上
pointingStatus 検出したポインティング座標の状態です。
正常でない場合はエラーの要因が出力されます。		 PointingStatus_DataValid,
PointingStatus_LackOfObjects

position

 正規化されたポインティング座標です。
中心を (0.0f, 0.0f) として右上を (1.0f, 1.0f), アスペクト比4:3 で正規化された値です。		 (-1.0,-1.0) ~ (1.0, 1.0)

各クラスタの詳細は以下のフォーマットになります。

これらの出力値は、 nn::irsensor::PointingProcessorState 構造体のメンバとして持っています。

struct PointingProcessorState
{
    int64_t samplingNumber;         //!< ポインティングプロセッサから処理結果を得る度に増加する値です（0 以上）。
    ::nn::TimeSpanType deltaTime;   //!< 処理開始時点から処理結果を取得した時点までの時間情報です。過去データの時間との差分をとって扱います（0 以上）。
    PointingStatus pointingStatus;  //!< 検出したポインティングデータの状態です。
    ::nn::util::Float2 position;    //!< 正規化されたポインティングの位置座標です。（中心を (0.0f, 0.0f) として、右上を (1.0f, 1.0f)、アスペクト比4:3 で正規化されています。)

これらの出力データは、 nn::irsensor::GetPointingProcessorStates に出力フォーマットのバッファを渡して、呼び出すことで取得することができます。

ポインティングモードの使用方法

ポインティングモードは以下のような流れで使用します。

  1. nn::hid::NpadIdType を元に、nn::irsensor::IrCameraHandle を取得し、モーションIRカメラを初期化します。

    #include <nn/irsensor.h>

nn::irsensor::IrCameraHandle handle = nn::irsensor::GetIrCameraHandle(npadIdType);
nn::irsensor::Initialize(handle);

  2. 取得した nn::irsensor::IrCameraHandle を引数に指定して、  nn::irsensor::RunPointingProcessor 関数を呼び、キャプチャを開始します。

     nn::irsensor::RunPointingProcessor(handle);

  3. 出力用の nn::irsensor::PointingProcessorState のバッファを準備して、 nn::irsensor::GetPointingProcessorStates 関数を呼びます。 
     nn::irsensor::GetPointingProcessorStates 関数では一度に最大 nn::irsensor::PointingProcessorStateCountMax 個の処理結果を取得できます。
    呼び出した時点の最新の処理結果を取得しますが、コントローラの BT 転送の都合上最速でも15ms 間隔での更新になるため、
    例えば、ゲームフレームなどと同期する形で、ポーリング方式で使用することを推奨します。
    一方、内部的には 5ms 間隔でデータの更新を行っていますので、理想的には 15ms 間隔で 3個 の新しいデータを取得することができます。
    軌跡の描画などを行う際には、より多くの情報を使用することを推奨します。

    nn::irsensor::PointingProcessorState state[3];
int count;
nn::irsensor::GetPointingProcessorStates(&state, &count, 3, handle);

  4. 停止したい nn::irsensor::IrCameraHandle を引数に指定して、 nn::irsensor::StopImageProcessor 関数を呼び、ポインティングモードを停止します。
    その後、 nn::irsensor::Finalize 関数を呼び、 モーションIRカメラを終了します。

    nn::irsensor::StopImageProcessor(handle);
nn::irsensor::Finalize(handle);
