カスタムフィールドノード

「カスタムフィールドノード」について説明したページを作成しました。

以下に、ゲームアプリケーションでカスタムフィールドとして「風」と「爆風」フィールドを実装したと仮定して説明します。
「風」フィールドは、ゲーム内の一定方向に力を加えたとき、パーティクルの移動速度にその影響を与えるカスタムフィールドです。
「爆風」フィールドは、爆発エフェクトの発生地点から放射状に力を加えたとき、パーティクルの移動速度にその影響を与えるカスタムフィールドです。

UIの例 ゲームにおける使用イメージ
   
A

「風の影響を受ける」 = ON

イメージのように「木の葉」パーティクルは「風」フィールドの影響を受け、→ の方向に移動します。
B 「爆風の影響を受ける」 = OFF イメージのように「木の葉」パーティクルは「爆風」フィールドの影響を受けず、→ の方向に移動します。
C
 		「風」の影響力 = 0.8 「木の葉」パーティクルの速度に対し、「風」の速度を 0.8 の割合で加算します。

カスタムアクションノード

「カスタムアクションノード」について説明したページを作成しました。

以下に、ゲームアプリケーションでアクションとして「衝突判定」を実装したと仮定したときのカスタムアクションの設定例です。
「衝突判定」は、ゲーム中の地形やオブジェクトのコリジョンと接触したときの挙動を定義したアクションです。

UI の 例 ゲームにおける使用イメージ

A アクションID = 「衝突判定を使用」 エミッタは、アクション「衝突判定」を利用します。
B マップと衝突判定する = ON
オブジェクトと衝突判定する = OFF		 イメージのように、パーティクルはマップ（地形）と衝突します。
オブジェクト（キャラクタ）とは衝突しません。
C 反射係数 = 0.8 パーティクルはマップとの衝突により、 0.8 の速度で反射します。

中間ファイルバージョンの更新

3D 中間ファイルやグラフィックス中間ファイルのバージョンを 3.7.0 に更新しました。
EffectMaker ( vfx  ) から参照されるプリミティブ( fmdb ) や テクスチャ( ftxb ) ファイルのアップデートが必要です。
 

以下の中間ファイルアップデータを利用することで、既存の中間ファイルのバージョンを更新できます。

Tools\Graphics\GraphicsTools\IntermediateFileUpdater.exe
