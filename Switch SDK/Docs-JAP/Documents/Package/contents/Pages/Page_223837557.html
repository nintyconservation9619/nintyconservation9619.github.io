親マテリアル参照パス

概要

親マテリアル参照パスは、親マテリアルの追加・変更ダイアログに表示されるマテリアルを持つモデルの検索に使われます。
親マテリアル参照パスが複数設定された場合は、設定順にモデルの検索が行われます。

親マテリアル参照パスにはフォルダまたはファイルのパスを指定してください。絶対パスを設定することができます。
パスの文字列には % で囲われた環境変数を含めることができます。3DEditor は % で囲われた環境変数を環境変数の値に置き換えて解釈します。
パスにはワイルドカード(*または?)を使用することが出来ます。

名前でマテリアル一覧に表示されるモデルをユーザーが絞り込むことが出来ます。

親マテリアルを持つモデルの検索

親マテリアルの追加・変更ダイアログで、「チーム設定」から設定された親マテリアルモデルはパスを持たないファイル名のみ保存されます。

子マテリアルモデルからファイル名しか持たない親マテリアルモデルを開くときは以下の順序で検索します。

  • 子マテリアルモデルと同じフォルダ
  • 親マテリアル参照パスを設定順に検索
    • 親マテリアル参照パスで見つかったファイル
    • 親マテリアル参照パスで見つかったフォルダ内のファイル
      • Recursive が true の場合はサブフォルダ内すべてのファイル

ワイルドカードによる名前の置き換え

名前の一部をパスのワイルドカードで一致した部分で置き換えることが出来ます。
置き換えはパスの先頭からワイルドカードの一致順に、\1, \2, \3... で指定します。

参考：
<ParentMaterialPath Name="TownModel(\1)(\2)">%NINTENDO_SDK_ROOT%\Samples\Sources\Applications\G3dDemo\Resources\Town\*\*</ParentMaterialPath>

チーム設定ファイルの <FileIo> 内に上のコードを追加すると、親マテリアルの追加ダイアログには以下のような名前が表示されます。 

TownModel(bg_WhiteTown)(bg_WhiteTown.fmdb)
TownModel(bg_WhiteTown)(bg_WhiteTown_cloth.fmdb)
TownModel(bg_WhiteTown)(sky_blueA.fmdb)
TownModel(Duck)(Duck.fmdb)
TownModel(FemaleA)(FemaleA.fmdb)
TownModel(Fish)(Fish.fmdb)
...
