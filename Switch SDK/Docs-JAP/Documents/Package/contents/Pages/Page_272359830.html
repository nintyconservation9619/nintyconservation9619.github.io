Low-Level Graphics Debugger

概要

Low-Level Graphics Debugger は、NVN 専用の低レベルグラフィックスデバッガです。NVN グラフィックスデバッガとよく似て、アプリケーションをデバッグするための、リソースやパイプラインステートを調査する機能を提供しますが、Low-Level Graphics Debugger はより低レベルに設計されています。

Low-Level Graphics Debugger はアプリケーションのデバッグ目的に使われることを意図しています。このライブラリを使用したROM を Nintendo Developer Portal (NDP) にマスター ROM としてアップロードすることはできません。

パッケージ

Low-Level Graphics Debugger は、NintendoSDK NX Addon とは別のパッケージで提供されます。また、他の NintendoSDK パッケージよりも遅れてリリースされる場合があります。

以下にパッケージを一覧します。

パッケージ名 説明
Low-Level Graphics Debugger NVN 専用の低レベルグラフィックスデバッガです。

配置

Low-Level Graphics Debugger は、以下のように NintendoSDK を配置したディレクトリ直下に配置されます。

NintendoSDK/
LowLevelGraphicsDebugger/

また、NintendoSDK の Libraries ディレクトリ以下に llgd-target.nso の名前で Low-Level Graphics Debugger 用のライブラリーファイルが配置されています。Low-Level Graphics Debugger を使用するには、対象のアプリケーションに追加の NSO ファイルとして llgd-target.nso をリンクする必要があります。
NintendoSDK の Include ディレクトリ以下には llgd.h の名前で Low-Level Graphics Debugger 用のヘッダーファイルが配置されています。Low-Level Graphics Debugger の API を使うには llgd.h をインクルードします。

マニュアル

パッケージの詳細については、パッケージ内のドキュメントを参照してください。

NVN グラフィックスデバッガとの違い

Low-Level Graphics Debugger はNVN グラフィックスデバッガと共通の機能を多く提供しており、その使用方法や使用目的はよく似ています。ここでは、実際に使おうとする際にどちらがより適切かを決定するための、両者のおおざっぱな違いを説明します。細かな機能の違いについてはそれぞれのドキュメントを参照してください。

NVN グラフィックスデバッガは NVN API の単位でキャプチャを行う一方で、Low-Level Graphics Debugger は GPU に送られるコマンドの単位でキャプチャを行います。そのため、キャプチャされる内容は実際のハードウェアのステートであり、アプリケーションが NVN API に渡したパラメータは完全には復元できない傾向にあります。また、利用できる機能もNVN グラフィックスデバッガと比べて減少する傾向にあります。

Low-Level Graphics Debugger は NX 実機上で動作しているアプリケーションのみに使用でき、NVN グラフィックスデバッガとは違って PC 上で動作しているアプリケーションをキャプチャすることはできません。

Low-Level Graphics Debugger は軽快な動作を目指して設計されており、キャプチャしていないときの CPU へのオーバーヘッドはほぼゼロで、キャプチャ時は 10MB 以下のメモリを使ってキャプチャします。そのかわり、NVN グラフィックスデバッガがもつリアルタイム解析のような機能はありません。また、ターゲット上で使用できる LLGD API が提供されており、たとえば、ツール上の UI からキャプチャをトリガーするかわりに、LLGD API を通してキャプチャ対象のアプリケーションからキャプチャをトリガーすることが可能です。
