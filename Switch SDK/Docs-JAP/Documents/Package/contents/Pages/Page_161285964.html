消費電力計測機能

機能概要

PCM ライブラリはボード上の消費電力を計測するインタフェースを提供します。

計測ポイントの一覧

計測ポイントは、古い開発機では構成が異なります。また、 EDEV には使える計測ポイントがありません。

計測された値の利用方法

計測値の変化をチェックしながら動作確認することで、アプリ内で相対的に消費電力が大きくなる（負荷の高い）シーンを知ることができます。

消費電力が大きいシーンが継続的に続くと、その分だけバッテリーの持続時間が短くなります。

消費電力は、 OE ライブラリで指定した性能の設定や、CPU / GPU の使用率によって大きく影響を受けます。

アプリの選択する性能設定と負荷のバランスのチューニングの参考としてください。

注意：

計測される消費電力は個体によってばらつきがあることに注意してください。

現時点では、これを補正する機能は提供していません。

実装方法

消費電力計測機能を使用する場合は以下の手順で使用してください。

  1. PCM ライブラリヘッダを include してください。

     #include <nn/pcm/pcm.h>

  2. PCM ライブラリを初期化してください。

    nn::pcm::Initialize();

  3. 計測対象が実装されているかを確認し、実装されているものについて消費電力を取得してください。

    if (nn::pcm::IsSupported(nn::pcm::MeasuringPoint_Cpu))
{
    int power = nn::pcm::ReadCurrentPower(nn::pcm::MeasuringPoint_Cpu);
    NN_LOG("CPU power consumption is %d mW\n", power);
}
else
{
    // 実行中のボードには対象の計測ポイントが実装されていません。
    // このような計測ポイントに対して nn::pcm::ReadCurrentPower() を呼び出すと、 0 mW が返されます。
}

実装サンプル

消費電力計測に関する開発支援機能

消費電力計測機能の無効化

注意：

NX Addon 0.12 系では本機能は提供していません。

DevMenuCommand の enable-pcm/disable-pcm コマンドを使用して、消費電力計測機能の有効/無効を設定することができます。

本機能を使って消費電力計測機能を無効にした状態では、アプリケーションが nn::pcm::Initialize() 関数を呼び出した時点でアプリケーションが内部でアボートするようになります。

PCM ライブラリの API は製品環境で使用できないため、アプリケーションがその API を誤って使用していないことを確認するために利用できます。
