Webアプレット・オフラインWebアプレット利用にあたっての注意

本体更新の影響

本体更新によって、Webアプレット・オフラインWebアプレットが更新されることがあります。Webブラウザーの性質上、更新の結果、同じWebページの表示や挙動が異なるものになる可能性があります。本ドキュメントには、これまでのWebアプレット・オフラインWebアプレットの Firmware バージョンの違いによる、仕様や挙動の変更についての情報が記載されています。将来も同様に本体更新の影響があることを理解の上、Webアプレット・オフラインWebアプレットを利用してください。

アプリケーションが行う処理のパフォーマンスへの影響

アプリケーションから呼び出されたWebアプレット・オフラインWebアプレットはアプリケーションが用いているのと同じCPUコアで動作します。もしWebアプレット・オフラインWebアプレットの起動中も、アプリケーションがスレッドなどを用いて継続して処理を行っていると、その影響でWebアプレット・オフラインWebアプレットのパフォーマンスが低下することがあります。

終了理由の値について

将来のWebアプレット・オフラインWebアプレットの仕様変更により下記の API で取得できる終了理由が、現在未定義の値を返すようになる可能性があります。終了理由が未定義の値を返す場合でもアプリケーションを異常終了させず、継続して動作させるようにしてください。

アプレット API
Webアプレット nn::web::WebPageReturnValue::GetWebExitReason()
オフラインWebアプレット nn::web::OfflineHtmlPageReturnValue::GetOfflineExitReason()

動画再生中の画面焼け軽減機能の無効化

Webアプレット・オフラインWebアプレットは動画プレイヤーを持ち、ウェブサイトに置かれた動画やオフラインHTMLに含まれる動画を再生する場合があります。動画再生中には画面焼け軽減機能は無効化されます。

JS拡張を有効にする

Webアプレット・オフラインWebアプレットで表示するWebページでは、様々なJS拡張を利用できます。WebページでJS拡張を利用するには、下記の API を使って機能を有効にしておく必要があります。

アプレット API
Webアプレット nn::web::ShowWebPageArg::SetJsExtensionEnabled()
オフラインWebアプレット nn::web::ShowOfflineHtmlPageArg::SetJsExtensionEnabled()

また、Webアプレットでは下記の条件も満たす必要があります。

  • WebサーバーとSSL/TLS通信で接続している。
  • 起動時に指定した URL と同じドメインのWebページを表示している。

JS拡張が有効になっている時、window オブジェクトに nx という名前のプロパティが定義されます。またその場合、Webページで window.nx を定義することはできません。

起動と終了

アプリケーションからパラメーターを渡す

アプリケーションの情報をWebアプレット・オフラインWebアプレットで表示するWebページに反映することができます。アプリケーション内の情報を元に WebアプレットではURLを、オフラインWebアプレットではオフラインHTMLのパスを変えて、表示するWebページを変えることができます。 また、URL やパスにURLパラメーターでアプリケーションの情報を付加して、同じWebページに対して反映させることもできます。

以下は、URLパラメーターでプレイヤーの位置情報をオフラインWebアプレットへ渡して、オフラインHTMLで地図を表示することを想定したサンプルコードです。

char path[nn::web::ShowOfflineHtmlPageArg::DocumentPathMaxLength];
snprintf(path, sizeof(path), "map.htdocs/index.html?x=%d&y=%d", x, y);

nn::web::ShowOfflineHtmlPageArg showOfflineHtmlPageArg(path);
nn::web::OfflineHtmlPageReturnValue offlineHtmlPageReturnValue;
nn::Result result = nn::web::ShowOfflineHtmlPage(&offlineHtmlPageReturnValue, showOfflineHtmlPageArg);

使用例は、WebDemoQuickHelp のサンプル、WebDemoRaceResult のサンプルを参照してください。

システムによって終了した時に復元する

アプリケーションからWebアプレット・オフラインWebアプレットを呼び出してWebページを表示しているとき、ユーザーがHOMEボタンを押してHOMEメニューからシステムアプリを起動すると、アプリケーションから呼び出していたWebアプレット・オフラインWebアプレットはシステムによって終了してしまいます。アプリケーションはこれを検出して、Webアプレット・オフラインWebアプレットを再び呼び出して復元することができます。

注意：

表示していたWebページの状態は保存されていないので、完全に復元することは出来ません。

以下に、アプリケーションから呼び出したWebアプレット・オフラインWebアプレットがシステムによって終了する例を示します。

  1. アプリケーションからWebアプレット・オフラインWebアプレットを呼び出してWebページを表示します。
  2. ユーザーがHOMEボタンを押してHOMEメニューへ遷移します。
  3. ユーザーがマイページ（システムアプリ）を起動します。
    1. ここで、アプリケーションから呼び出したWebアプレット・オフラインWebアプレットはシステムによって終了してしまいます。
  4. ユーザーがHOMEメニューへ戻ります。
  5. ユーザーがプレイ中のアプリケーションを選びます。
  6. アプリケーションから呼び出したWebアプレット・オフラインWebアプレットは終了してしまっています。
  7. アプリケーションはシステムによる終了を検出して、Webアプレット・オフラインWebアプレットを再び呼び出して復元することができます。

以下は、システムによってオフラインWebアプレットが終了した時に復元するサンプルコードです。

nn::Result result;
nn::web::OfflineHtmlPageReturnValue offlineHtmlPageReturnValue;
do
{
    nn::web::ShowOfflineHtmlPageArg showOfflineHtmlPageArg("htmlcontents0.htdocs/index.html");
    result = nn::web::ShowOfflineHtmlPage(&offlineHtmlPageReturnValue, showOfflineHtmlPageArg);
} while ( result.IsSuccess() && offlineHtmlPageReturnValue.GetOfflineExitReason() == nn::web::OfflineExitReason_ExitMessage )

使用例は、WebDemoRaceResult のサンプルを参照してください。

デザイン

背景にアプリケーションのキャプチャー画像を利用する

Webページの背景にアプリケーションのキャプチャー画像を適用して、Webページを透過させて重ね合わせて表示することができます。適用するキャプチャー画像には、ぼかし効果をかけることもできます。この機能を使って、ゲームアプリケーションのポーズ画面をオフラインWebアプレットで表示するオフラインHTMLで作ることができます。

以下は、サンプルコードです。

nn::web::ShowOfflineHtmlPageArg showOfflineHtmlPageArg("htmlcontents0.htdocs/index.html");
showOfflineHtmlPageArg.SetBackgroundKind(nn::web::OfflineBackgroundKind_ApplicationCaptureBlur);
nn::web::OfflinePageReturnValue offlinePageReturnValue;
nn::Result result = nn::web::ShowOfflineHtmlPage(&offlinePageReturnValue, showOfflineHtmlPageArg);

使用例は、WebDemoPauseMenu のサンプル、WebDemoQuickHelp のサンプルを参照してください。

空間ナビを独自のフォーカス表現に変更する

Webページに対して空間ナビの操作を行うと、標準のフォーカス枠が表示されます。標準のフォーカス枠の代わりに、Webページの CSS でアプリケーションのデザインに合った独自のフォーカス表現に変更することができます。

Webページに下記の <meta> 要素を記述すると、標準のフォーカス枠が表示されなくなります。

<meta name="focus-ring-visibility" content="hidden">

以下は、ボタン要素に独自のフォーカス枠を表示する CSS のサンプルです。

button:focus {
  outline: solid 4px rgb(0, 128, 255);
}


この他にも様々な表現の方法があります。詳細は、WebDemoSpatialNavigation のサンプルを参照してください。

標準のフォーカス枠は、Webページの最前面に描画されます。そのため、フルスクリーン状態を除いて、フォーカスしている要素が別の要素に遮蔽されている場合でも、フォーカス枠は遮蔽されません。標準のフォーカス枠の代わりに独自のフォーカス枠を表示することによって、この問題を回避して、正しく遮蔽されるようにすることができます。フォーカスしている要素がフルスクリーン状態で表示されている要素に遮蔽されている場合、標準のフォーカス枠は表示されません。

フッターの表示を制御する

ユーザーがWebページのリンクを決定したりスクロール操作を行うと、フッターは必要に応じて表示されたり隠れたりします。

フッターの表示状態を制御するための、いくつかの機能があります。

  • オフラインWebアプレットで、フッターを常に非表示にする。
  • フッターの表示を制御する方法を変更する。
  • JS拡張を使って、フッターの表示状態を一時的に固定する。

フッターを常に非表示にする

オフラインWebアプレットは、起動時に nn::web::ShowOfflineHtmlPageArg::SetFooterEnabled() を使って、フッターを常に非表示にしてオフラインHTMLを全画面に表示することができます。この機能を使って、オフラインWebアプレットで表示するオフラインHTMLをアプリケーションの一部のように見せることができます。Webアプレットには、フッターを常に非表示にする機能はありません。

使用例は、WebDemoInstructionManual のサンプルを参照してください。

フッターの表示を制御する方法を変更する

フッターの表示を制御する方法は、下記の API を使って変更することができます。標準でフッターは、必要に応じて表示されたり隠れたりします。制御する方法を変更して、フッターを常に表示することもできます。

アプレット API
Webアプレット nn::web::ShowWebPageArg::SetFooterFixedKind()
オフラインWebアプレット nn::web::ShowOfflineHtmlPageArg::SetFooterFixedKind()
参考：

NintendoSDK NXAddon 5.1.0 (Firmware for NX 5.0.0-1.0) 以降から利用できます。

JS拡張を使ってフッターの表示状態を一時的に固定する

下記のJS拡張を使って、フッターの表示状態を一時的に固定することができます。例えば、画像を全画面に表示したい場合にフッターを非表示にしたり、ユーザーに操作方法を常に案内したい場合にフッターを表示状態に固定することができます。固定した表示状態は、Webページを遷移すると解除され、制御方法によって表示状態が変わります。

window.nx.footer.setFixed(state)

フッターの表示状態を固定します。

  • パラメーター
    • state: String 固定する状態を表す文字列
  • 戻り値
    • 無し
  • 詳細
    • state に指定できる文字列。
文字列 表示状態
shown 表示
hidden 非表示

window.nx.footer.unsetFixed() 

window.nx.footer.setFixed()で固定した表示状態を標準の状態に戻します。

  • パラメーター
    • 無し
  • 戻り値
    • 無し
  • 詳細
    • 特になし

サウンド

BGMを鳴らす

Webアプレット・オフラインWebアプレットを呼び出す前にあらかじめ BGM を鳴らしておくと、Webページを表示中に BGM を鳴らすことができます。Webアプレット・オフラインWebアプレットを起動する API はアプレットが終了するまで処理をブロックし、呼び出したスレッドは停止してしまうので、BGM の処理は別のスレッドで行う必要があります。BGM の音量は、nn::oe::SetExpectedVolumeBalance() で設定できます。Webアプレット・オフラインWebアプレットで動画を再生しているとき、BGM は鳴りません。

SEを鳴らす

Webページに対して空間ナビの移動やボタンの押下などの操作を行うと、Webアプレット・オフラインWebアプレットが設定している標準のシステムSEが鳴ります。SE の音量は、nn::oe::SetExpectedVolumeBalance() で設定できます。JS拡張を使って、標準のシステムSEの代わりに簡易的に任意のシステムSEを鳴らすことができます。また、Web Audio を使ってアプリケーションのデザインに合った独自の SE を鳴らすことができます。

以下のように、WebページでHTML要素にnx-se-disabled属性を追加すると、この要素にフォーカスした時と、この要素を決定またはクリックした時に鳴る標準のシステムSEを無効にすることができます。

<button nx-se-disabled>

JS拡張の window.nx.footer.unsetAssign() や window.nx.footer.setAssign() を使うと、標準のコントローラー操作の機能やシステムSEを無効にしたり、カスタマイズすることができます。詳しくは、JS拡張を使ってコントローラー操作をカスタマイズする を参照してください。

JS拡張を使ってシステムSEを鳴らす

下記のJS拡張を使って、Webページで簡易的にシステムSEを鳴らすことができます。アプレットが鳴らす SE と合わせて4つまで同時に鳴らすことができます。この機能を使って鳴らした SE は、ページを遷移すると停止します。

window.nx.stopSystemSe([options])

再生中のシステムSEを停止します。

  • パラメーター

Web Audioを使ってSEを鳴らす

Web Audio を使ってアプリケーションのデザインに合った独自の SE を鳴らすことができます。WAVE_FORMAT_PCMタイプのWAVEコンテナフォーマットのファイル（.wav）、およびAAC-LCコーデックのMP4コンテナフォーマットのファイル（.mp4、.m4a）を再生することができます。Web Audio を利用するには、下記の API を使って機能を有効にしておく必要があります。

アプレット API
Webアプレット nn::web::ShowWebPageArg::SetWebAudioEnabled()
オフラインWebアプレット nn::web::ShowOfflineHtmlPageArg::SetWebAudioEnabled()

以下は、"sample_se.mp4" という名前のファイルを再生する HTML のサンプルです。

<!DOCTYPE html>
<meta charset="utf-8">
<button id='play-se-button' disabled nx-se-disabled>play</button>
<script>
window.AudioContext = window.AudioContext || window.webkitAudioContext;
var audioCtx = new AudioContext();
window.onload = function() {
  var request = new XMLHttpRequest();
  request.open('GET', 'sample_se.mp4', true);
  request.responseType = 'arraybuffer';
  request.onload = function () {
    audioCtx.decodeAudioData(request.response, function (buffer) {
      var element = document.getElementById('play-se-button');
      element.disabled = false;
      element.onclick = function() {
        var source = audioCtx.createBufferSource();
        source.buffer = buffer;
        source.connect(audioCtx.destination);
        source.start(0);
      };
    });
  };
  request.send();
};
</script>

使用例は、WebDemoPauseMenu のサンプルを参照してください。

Webアプレット・オフラインWebアプレットの WebAudio には以下のような制限があります。

AudioNode は、AudioBufferSourceNode、AudioDestinationNode、GainNode のみを使うことができます。

AudioBufferSourceNode.detune メソッドは正しく動作しません。

各 AudioNode で AudioParam を使うことも出来ますが、linearRampToValueAtTime と exponentialRampToValueAtTime メソッドは正しく動作しません。

Webページで複数の AudioContext を生成することができますが、再生される AudioContext は１つだけです。iframe要素を使って別のページを埋め込む場合に注意してください。

同時に再生する SE の数に制限はありませんが、ミキシングされた波形の振幅が最大値を超えた場合はクリッピングされるため、音量の大きい SE を同時に再生したり、多くの SE を同時に再生するとノイズが発生する場合があります。ノイズが発生する場合は、同時に SE を再生しないようにしてください。

ページを遷移する時、AudioContext は破棄され、再生中の SE は止まります。ページを遷移する時に SE を再生したい場合は、再生が終わるのを待ってからページを遷移してください。再生が終わるのを待つためにAudioBufferSourceNode.onendedイベントハンドラを使うことができますが、AudioBufferSourceNode.startメソッドで再生を開始した後にAudioBufferSourceNodeオブジェクトへの参照が無くなると、onendedイベントハンドラが呼ばれない場合があります。onendedイベントハンドラが確実に呼ばれるようにするには、AudioBufferSourceNodeオブジェクトへの参照を保持しておき、onendedイベントハンドラで参照を解放するようにしてください。

参考：

NintendoSDK NXAddon 4.3.0 (Firmware for NX 4.0.0-3.0) 以降から利用できます。

操作のカスタマイズ

コントローラー操作の機能をカスタマイズするための、いくつかの機能があります。

  • RスティックとRスティックボタンに割り当てられているズーム機能を無効にする。
  • タッチ操作を無効にする。
  • Lスティックの操作を空間ナビに限定する。
  • JS拡張を使って、標準の機能を無効にしたり、独自の機能を割り当てる。
  • コントローラー操作を読み取るため、Keyboard Events と Gamepad API を利用する。

ズーム機能を無効にする

常に画面にフィットした状態でWebページを表示したい場合、以下のような <meta> 要素を記述すると、ズーム機能を無効にすることができます。 

<meta name="viewport" content="user-scalable=no">

ズーム機能が無効になっているとき、その操作ガイドはフッターに表示されません。

使用例は、WebDemoQuickHelp のサンプルを参照してください。

タッチ操作を無効にする

Webページの操作を、常にコントローラー操作のみにしてタッチ操作を無効にしたい場合、下記のAPIを使います。

アプレット API
Webアプレット nn::web::ShowWebPageArg::SetTouchEnabledOnContents()
オフラインWebアプレット nn::web::ShowOfflineHtmlPageArg::SetTouchEnabledOnContents()

以下のような JavaScript を記述すると、Webページごとにタッチ操作を無効にすることができます。

document.addEventListener('touchstart', function(event) {
  event.preventDefault();
});

ただし、フッターやダイアログなどのWebアプレット・オフラインWebアプレットのUIや、ソフトウェアキーボードやエラービューアなどのアプレットのUIのタッチ操作は無効になりません。また、JavaScript を使った方法ではタッチによるスクロール操作を完全に無効にすることができないため、スクロールしないページに用いることができます。

使用例は、WebDemoSpatialNavigation のサンプルを参照してください。

参考：

常にタッチ操作を無効にする API は、NintendoSDK NXAddon 4.1.0 (Firmware for NX 4.0.0-1.0) 以降から利用できます。

Lスティックの操作を空間ナビに限定する

Webアプレット・オフラインWebアプレットをコントローラーで操作する方法には、空間ナビとポインターの二種類の方法があります。ユーザーは、Lスティックボタンで操作方法をいつでも切り替えることができます。

ポインターの操作方法は他の本体機能や多くのアプリケーションと異なっており、ユーザーにとって慣れていない操作である可能性があるので、Lスティックの操作を空間ナビに限定することができます。

注意：

本機能は、表示される全てのWebページが空間ナビで操作できるように作られたページである場合に限って、利用するようにしてください。

Webアプレットで汎用的なWebサービスや、様々なデバイスで表示されるWebページを利用する場合、注意が必要です。そのようなWebページはWebアプレットで表示されることを前提に作られておらず、空間ナビのみでは操作できなくなる恐れがあります。Webアプレットで表示するWebページにそのようなWebページがある場合、Lスティックの操作を空間ナビに限定せず、ポインターに切り替えられるようにしてください。

Lスティックの操作を空間ナビに限定するには、下記の API を使います。

アプレット API
Webアプレット nn::web::ShowWebPageArg::SetPointerEnabled()
オフラインWebアプレット nn::web::ShowOfflineHtmlPageArg::SetPointerEnabled()

Lスティックの操作が空間ナビに限定されているとき、Lスティックの操作を切り替える操作ガイドはフッターに表示されません。

使用例は、WebDemoInstructionManual のサンプルを参照してください。

JS拡張を使ってコントローラー操作をカスタマイズする

下記のJS拡張を使って、標準のコントローラー操作の機能を無効にしたり、独自の機能を割り当てることができます。この機能を使ってカスタマイズした機能は、ページを遷移すると標準の状態に戻ります。複数のページでカスタマイズしたい場合、各ページごとに関数を呼び出してください。

使用例は、WebDemoGamepadApi のサンプル、WebDemoRaceResult のサンプルを参照してください。

注意：

標準でＢボタンにはページ遷移を戻る機能、ＸボタンにはWebアプレット・オフラインWebアプレットを終了する機能が割り当たっています。これらのボタンをカスタマイズする場合は、他の機能を使ってWebページに対する操作を行うことができ、アプリケーションへ戻れるように注意してください。特にWebアプレットは、ネットワークの状態などによってJavascriptファイルをロードできず、開発者が意図しないカスタマイズ状態になってしまう場合があることに注意してください。

window.nx.footer.setAssign(key, label, function, [options])

指定されたキーの機能と操作ガイドを変更します。

  • パラメーター
    • key: String 変更するキー
    • label: String フッターに表示する操作ガイドの文字列
    • function: Function 割り当てる機能
    • options: Object 省略可 オプションのオブジェクト
      • se: String 割り当てた機能が実行された時に鳴らす SE
  • 戻り値
    • 無し
  • 詳細
    • key に、'B'、'X'、'Y'、'L'、'R' のいずれかのキーを指定できます。大文字でも小文字でも構いません。
    • label の最大文字数は 63 です。
    • key に 'L'、'R' を指定した場合、操作ガイドはフッターに表示されません。label には空文字列を指定してください。
    • options.se に指定できる文字列は、window.nx.playSystemSe() の label に指定できる文字列と同じです。
    • options.se に空文字列を指定した場合、SE は鳴りません。
参考：

NintendoSDK Firmware for NX 5.0.0-1.0 より前の label の最大文字数は 10 です。

window.nx.footer.unsetAssign(key)

指定されたキーの機能を無効にして、フッターの操作ガイドを非表示にします。

  • パラメーター
    • key: String 機能を無効にして、フッターの操作ガイドを非表示にするキー
  • 戻り値
    • 無し
  • 詳細
    • key に、'A'、'B'、'X'、'Y'、'L'、'R' のいずれかのキーを指定できます。大文字でも小文字でも構いません。

window.nx.footer.setDefaultAssign(key, [label])

指定されたキーの機能を標準に変更します。

  • パラメーター
    • key: String 標準に変更するキー
    • label: String 省略可 フッターに表示する操作ガイドの文字列
  • 戻り値
    • 無し
  • 詳細
    • key に、'A'、'B'、'X'、'Y'、'L'、'R' のいずれかのキーを指定できます。大文字でも小文字でも構いません。
    • label の最大文字数は 63 です。
    • label を指定しない場合、標準の操作ガイドの文字列がフッターに表示されます。
参考：

NintendoSDK Firmware for NX 5.0.0-1.0 より前の label の最大文字数は 10 です。

cookie

Webアプレットで表示するWebページでは、cookie を使うことができます。

サードパーティcookieは拒否されます。

オフラインWebアプレットで表示するオフラインHTMLでは、cookie を使うことはできません。

Web Storage

Webアプレットで表示するWebページでは、sessionStorage と localStorage を使うことができます。オフラインWebアプレットで表示するオフラインHTMLでは、sessionStorage を使うことができますが、localStorage を使うことはできません。

Viewport

ビューポートはページをレイアウトする際の仮想的なウィンドウで、<meta> 要素の name="viewport" 属性の指定で変更できます。
標準のサイズは width=1280, height=648 です。オフラインWebアプレットで、起動時に nn::web::ShowOfflineHtmlPageArg::SetFooterEnabled() を使ってフッターを常に非表示にした場合、サイズは width=1280, height=720 に設定されます。サイズに width=device-width, height=device-height を指定した場合も、それらと同じ値に設定されます。
ビューポートを任意の値に変更できますが、指定する値によってはドットバイドットで表示されなくなったり、ページが画面にフィットしなくなる場合があるので注意してください。

Page Cache

Webアプレット・オフラインWebアプレットには、Page Cache という機能が備わっています。Webページのリンクを進んだ時に、ページの状態が一時的にメモリに保存され、履歴を戻った時に、保存されていたページが再び表示されます。Page Cache によって、高速に履歴を戻ったり、ページの状態を復元することができます。しかし、履歴を戻った時に、Webサイトからページを再取得したい場合や、ページに加えた動的な変更を取り消したい場合があります。
常に Page Cache を無効にしたい場合、下記のAPIを使います。

アプレット API
Webアプレット nn::web::ShowWebPageArg::SetPageCacheEnabled()
オフラインWebアプレット nn::web::ShowOfflineHtmlPageArg::SetPageCacheEnabled()

Webページごとに Page Cache を無効にするには、以下の方法があります。

  • HTTPSサイトで、HTTPレスポンスヘッダの Cache-Control に no-store を指定して、ページがキャッシュされないようにする。
  • pageshowイベントハンドラで persisted プロパティを判定し、キャッシュされたページだったらリロードする。
onpageshow = function(e) {
  if (e.persisted) {
    location.reload();
  }
};
参考：

常に PageCache を無効にする API は、NintendoSDK NXAddon 4.3.0 (Firmware for NX 4.0.0-3.0) 以降から利用できます。

言語

Webアプレット・オフラインWebアプレットは、フッターやダイアログなどのテキストの言語にシステムUI言語を使用します。ここでは、システム言語のうち「TraditionalChinese」、「SimplifiedChinese」、「Korean」を「AmericanEnglish」に置き換えたものをシステムUI言語とします。
システム言語は、「本体機能ガイド」の「付録 > 表示言語の仕組み」を参照してください。
NavigatorLanguage.language には、システムUI言語に対応した言語コードが設定されます。
言語コードは、言語設定 を参照してください。
Webアプレットで表示するWebページの言語については、Accept-Language を参照してください。

User Agent

Webアプレット・オフラインWebアプレットの User Agent は下記の通りです。

アプレット User Agent
Webアプレット Mozilla/5.0 (Nintendo Switch; WebApplet) AppleWebKit/<WEBKIT_VERSION> (KHTML, like Gecko) NF/<ENGINE_VERSION> NintendoBrowser/<APPLICATION_VERSION>
オフラインWebアプレット Mozilla/5.0 (Nintendo Switch; OfflineWebApplet) AppleWebKit/<WEBKIT_VERSION> (KHTML, like Gecko) NF/<ENGINE_VERSION> NintendoBrowser/<APPLICATION_VERSION>

実際の User Agent の例は下記の通りです。

アプレット User Agent
Webアプレット Mozilla/5.0 (Nintendo Switch; WebApplet) AppleWebKit/601.6 (KHTML, like Gecko) NF/4.0.0.5.9 NintendoBrowser/5.1.0.13341
オフラインWebアプレット Mozilla/5.0 (Nintendo Switch; OfflineWebApplet) AppleWebKit/601.6 (KHTML, like Gecko) NF/4.0.0.5.9 NintendoBrowser/5.1.0.13341

以下は、JavaScript でWebアプレット・オフラインWebアプレットを判別するサンプルです。

function isNintendoSwitch() {
  return window.navigator.userAgent.match(/^.*\bNintendo Switch\b.+\bNintendoBrowser\/.*$/) != null;
}

UserAgentに文字列を付加する

Webアプレットは、起動時に nn::web::ShowWebPageArg::SetUserAgentAdditionalString() を使って、任意の文字列を UserAgent に付加することができます。Webサーバーは、クライアントの種類やバージョンを判別するために、UserAgent に付加された文字列を使うことができます。オフラインWebアプレットには、この機能はありません。

Webインスペクタ

Webコンテンツの開発のために、Webインスペクタを使うことができます。

Webインスペクタを利用するための手順は、以下の通りです。

  • 設定マネージャで Webインスペクタを有効にします。
    • Webインスペクタを有効にするを参照してください。
  • WebCallerCommandなどでWebアプレット・オフラインWebアプレットを起動します。
  • IPアドレスが表示されたダイアログが表示されます。
  • Webインスペクタのリソースは、$(NINTENDO_SDK_ROOT)/Tools/WebInspector/Resources に配置されています。
  • リソースに含まれている Main.html をPCのブラウザで開きます。
    • PCのブラウザのデフォルト設定では Webインスペクタが正しく動作しないため、以下の設定を行ってください。
      • Google Chrome の場合、--allow-file-access-from-files オプションをつけて起動してください。
        • Windows では、ショートカットを作成してリンク先の設定にオプションを記述します。
      • Mac Safari 10.x の場合、「開発」メニューの「クロスオリジンの制限を無効にする」にチェックを入れてください。
  • ファイルパスの Main.html の後ろに、「?ws=<ダイアログに表示されたIPアドレス>:9222/devtools/page/1」を付加して移動します。
  • Webインスペクタの利用を始めます。
参考：

NintendoSDK NXAddon 5.1.0 (Firmware for NX 5.0.0-1.0) 以降から利用できます。

console

Webページで consoleオブジェクトを使って、Webインスペクタのデバッグコンソールにアクセスできます。
console.log() メソッドの出力は、Target Manager などのログにも出力されます。
ログは、開発機を「開発機専用ファームウェア(DEV MODE)」で初期化したWebアプレット・オフラインWebアプレットで出力され、「microSDカードからのログ出力機能付きの製品機に近いファームウェア(ProdWithLog)」では出力されません。
