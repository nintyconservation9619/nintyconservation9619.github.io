EDEV の初期化

EDEV の電源を ON にする

EDEV 本体上部の電源ボタンを短押ししてください。

なお、電源 OFF の状態から EDEV を PC に接続した場合は自動で電源が入ります。

注意：

事前に EDEV を十分に充電（1時間以上を推奨）しておく必要があります。

初期化の途中で電池残量が足りなくなった場合、更新に失敗して故障する可能性があります。

EDEV を PC と接続する

「USB 充電ケーブル」を PC の USB ポートと EDEV 本体底面の「USB Type-C 端子」に接続します。

注意：

初期化の際はリレーボックスを使用しないでください。

リレーボックスを使用して初期化を行った場合、更新に失敗して故障する可能性があります。


EDEV と PC を接続した際、USB ドライバーのアップデートが行われることがあります。

EDEV と PC を接続した際は、EDEV の電源が ON の状態で一定時間待ってください。

初期化の途中に USBドライバーのアップデートが行われた場合、更新に失敗して故障する可能性があります。

EDEV を初期化する

専用のツールで EDEV を初期化します。（およそ 5 分で完了します。）

参考：

手順の途中で、Windows ファイアウォールに関する警告が数回表示されますが「アクセスを許可する」を選択してください。

手順 操作
1

EDEV 底面に貼ってあるシールに書かれた値を確認します。

（この値が EDEV の シリアル番号を示しています。以下の例では「XAW03121003905」です。少し離れて印字されている「5」も含みます）

参考：

ここで確認した値は、手順３で EDEV を選択する際に使用します。
2

NintendoSDK 管理画面の [EDEV を初期化する] ボタンをクリックします。


参考：

NintendoSDK 管理画面は、 Nintendo Dev Interface の「ようこそ」画面の「開発環境の構築・管理」を選択すると表示できます。
3

初期化する EDEV を選択し、初期化をおこないます。


「1. Select EDEV actual equipment」の下にある EDEV 接続リストから「Serial」が手順 1 で確認したものと一致する行をマウスで選択します。


「2. Select firmware variation」の下にあるコンボボックスから利用したいファームウェアのバージョンを選択します。

参考：

最初に選択されているファームウェアをそのまま選択することを強く推奨します。

「+ Prod」「+ ProdWithLog」を選択する場合は、「NintendoSDK ドキュメント」の「環境構築ガイド > より詳しい情報 > 製品機に近いファームウェア(PROD MODE)への初期化」を先に読んでください。

次に「3. Start EDEV initialization」の下にある「Start initialization」のボタンを押すと EDEV の初期化が始まります。


参考：

EDEV 本体がセーフモード等の状態にある場合はこのように Name 欄に (Safemode) と記載された行が表示されます。


この行を選択後「Start initialization」ボタンを押すと初期化が始まります。

参考：

途中、80～90 秒程度ログが止まっている状態になりますが、そのまま待機してください。

数分待ってもログが止まったままの場合は、失敗している可能性があります。

参考：

起動後も Re-Scan ボタンをクリックしても EDEV 接続リストに初期化したい EDEV 本体が表示されない場合は EDEV が不正規な状態になっている可能性があるので「EDEV の初期化が失敗するときは」のページも参照してください。
4

ログウィンドウの左下に「Success」と表示されたら、「Exit」ボタンを押してツールを終了します。


参考：

ファンの動作音

手順 3 が実行されている過程で、EDEV に内蔵されている冷却ファンが動作しはじめます。
少し大きな動作音がしますが、故障しているわけではありません。
参考：

他にもセットアップに関する追加情報を「NintendoSDK ドキュメント」の「環境構築ガイド > より詳しい情報 > 開発機セットアップに関する情報あれこれ」のページにまとめました。ご参照ください。

参考：

EDEV が正常に初期化されたかどうか確認するためのビルド済みサンプルが用意されています。

サンプルの実行方法は以下の通りです。

  1. NintendoSDK 管理画面で [サンプルを実行する] ボタンをクリックします。
  2. EDEV が正常に初期化され、VSI for NX が正しくインストールされている場合は、EDEV の液晶に次の画面が表示されます。

※このサンプルは自動的に終了しません。HOMEボタンでメニュー画面に戻ってください。

トラブルシューティング

EDEV の初期化に失敗する場合や EDEV 接続リストに初期化したい本体が見つからない場合は「EDEV の初期化が失敗するときは」のページを参照して何度か試してください。

それでも成功しない場合は、サポート窓口まで問い合わせてください。

EDEV 用のドライバについて

PC が EDEV を認識するには、EDEV 用のデバイスドライバがインストールされている必要があります。

通常、EDEV 用のデバイスドライバは NDI で NintendoSDK をインストールする際に自動でインストールされます。ただし、環境によっては自動インストールに失敗してしまう場合があります。

EDEV 用のデバイスドライバがインストールされているか確認するには、以下の手順に従ってください。

  1. EDEV の電源を ON にして、PC に USB ケーブルで接続する
  2. PC でデバイスマネージャーを起動する
  3. デバイスマネージャーに、NintendoSdkDebugger が表示されていることを確認する

EDEV 用のデバイスドライバがインストールされていない場合は、以下の手順に従って手動インストールを行ってください。
