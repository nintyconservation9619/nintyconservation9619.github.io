概要

Nintendo NX CPU プロファイラ（NXCP）は、多くの CPU リソースを占有する関数を検出するための統計的サンプリングプロファイラです。時間ベースの割り込みを使用して、割り込み時にアクティブになっている関数に関する情報を収集します。このツールは、アプリケーションによる CPU リソースおよび OS リソースの使用状況を把握できるように調整されています。リソースを多用しているロジックループを自動的に検出し、パフォーマンスの向上を図るための一般的なアドバイスを提供するほか、サンプリングされたデータをユーザーがさまざまな方法で検証できるようにします。

特長

  • 60Hz の 1 フレームあたり 1 〜 4000 倍のレート（60Hz 〜 240,000Hz）でコールスタックをサンプリング。
  • コールツリービューと部分コールツリービュー（選択した関数を中心として上位および下位）。
  • ユーザーによるカウンタデータの時間変化のロギング。
  • ユーザーによるマーク付きコードブロックのロギング。
  • ユーザーによるフレームレートおよびハートビートのロギング。
  • DirectX を使用したデータのグラフ表示による数百万点のデータポイント描画：
    • 「サンプルグラフ」：選択した関数ごとに未加工サンプリングデータを表示します。
    • アイシクルグラフ：システムによって色分けされた、各サンプルのコールスタックを表示します。
    • スタックグラフ：各サンプルのスタック深度を表示します。
    • 負荷グラフ：フレームごとに関数による負荷を表示します。
    • システム負荷：システム（物理的システム、グラフィック、AI など）ごとにフレームあたりの負荷を表示します。
    • ヒートマップモード：すべてのフレームをオーバーレイし、データを集約して表示します。
    • タイムライン：フレームレートとウィンドウビューを表示します。
    • ユーザーが記録したカウンタデータ、マーク付きコードブロック、およびハートビートのグラフ化。
  • 自動分析
    • 推測ハートビート：関数が呼び出されていた期間を検出し、コアごとに各種のハートビートを推測します。
    • システム分類：関数は、16 の異なるシステムのいずれかに属するものとして分類されます。
    • ストーリー分析：フレームごとに、そのフレームのストーリーを伝える主要な関数を特定します。
    • スパイク検出：1 つのフレーム上でスパイクしているか、複数のフレームにわたってバーストしている、異常な関数を特定します。
  • プロファイリングで確認された関数と確認されなかった関数のコードカバレッジ。
  • 正規表現を使用するオプションを備えた、関数、スレッド、およびカウンタの検索機能。
  • オペコードあたりのサンプルのアセンブリビュー（現時点では未実装）。

要件

Nintendo NX CPU プロファイラのクライアントが SDK のディレクトリに存在することを確認してください。このプロファイラをインストール済みであれば、NintendoSDK\Tools\NintendoNxCpuProfiler というディレクトリが存在します。このディレクトリが存在しない場合は、NDI クライアントを使用して Nintendo NX CPU プロファイラのパッケージをダウンロードします。

このプロファイラでは以下の各コンポーネントが必要です。

  • .NET 4.5 以降
  • DirectX End-User Runtime（June 2010）
    • https://www.microsoft.com/en-us/download/details.aspx?id=8109

制限事項

実行可能ファイルにプロファイラコードを組み込んでゲームを出荷することは禁止されています。マスター提出の際には、プロファイラに関連するコードをすべて無効にしてください。また、プロジェクトにプロファイラライブラリがリンクしていないようにします。

プロファイラでは、nn::os::HighestThreadPriority の優先度でスレッドが作成されます。プロファイラが正常に動作するようにするために、この優先度で他のスレッドを作成しないでください。

リソース

このプロファイラを使用するには、64MB 以上のバッファをゲームで割り当てている必要があります。

プロファイラが動作するために、合計 6 つのスレッドが作成されます。メインスレッドが 1 つ、PC 通信用スレッドが 2 つ、サンプリング用スレッドがコアごとに 1 つです。サンプリングスレッドは nn::os::HighestThreadPriority の優先度で起動します。プロファイラが正常に動作するようにするために、この優先度で他のスレッドを作成しないでください。プロファイラによって作成されるスレッドにはすべて、名前の前に "NX CPU Profiler:" が付けられます。
