ACCOUNT ライブラリ

はじめに

本ドキュメントの目的

本ドキュメントでは、NintendoSDK に収録されているアカウントライブラリについて説明します。

対象読者

NintendoSDK を使ってアプリケーションを開発される方を対象としています。

関連資料

Nintendo Developer Portal  の対象プラットフォーム向けアカウントガイドおよび、 アカウントライブラリの関数リファレンス と併せて参照してください。

アカウントライブラリの機能

アカウントライブラリは、デバイスの利用者を識別するためのユーザーアカウントとその周辺の機能を提供します。
ユーザーアカウントやその周辺の機能に関して、概要は Nintendo Developer Portal  に掲載される各プラットフォーム向けアカウントガイドを参照してください。 

提供する機能

アカウントライブラリでは、以下の機能を提供します。

  • ユーザーアカウントの利用
    • ユーザーアカウント ID の参照
    • ユーザーアカウントの状態の取得
  • ユーザーのプロフィール情報の利用
  • ネットワークサービスアカウントの利用 
    • ネットワークサービスアカウント ID の参照
    • ネットワークサービスアカウントの認証
    • ネットワークサービスアカウントの認証情報 (NSA ID トークン) の取得

それぞれの機能について、詳細は 関数リファレンス を参照してください。

エラーハンドリング

アカウントライブラリは期待する対処の異なるエラーを定義しています。
アカウントライブラリの返し得るエラーとそれぞれのエラーに関して期待する対処について、 エラーハンドリング に記載しています。

禁止事項

現在のバージョンでは、アカウントライブラリには次の禁忌があります。

ユーザーアカウント ID のネットワーク上での利用

ユーザーアカウント ID (nn::account::Uid) をネットワーク上で利用することを禁止します。
ユーザーアカウント ID は同一デバイス内でのユーザー識別のために使用することのみを想定しています。 

ネットワーク上でユニークな ID が必要な場合、次のいずれかの手段を用いてください。

  • ネットワーク サービスアカウント ID (nn::account::NetworkServiceAccountid) を使用する。
    • あるユーザーアカウントに関連づくネットワークサービスアカウント ID はユーザー操作で変更される可能性があることに注意してください。
  • ユーザーに対してアプリケーション独自の系列として ID を発行し、それを使用する。

制約事項

現在のバージョンでは、アカウントライブラリには次の制約があります。

ニンテンドーアカウントの属性に応じたネットワークサービスの利用可否

ユーザーアカウントにニンテンドーアカウントを連携している状態では、ニンテンドーアカウントの属性に応じてネットワークサービスアカウントの認証情報（NSA ID トークン）の取得可否が異なります。

Windows 環境でのインターネット通信を使用する機能の利用

現在のバージョンの SDK では、 Windows 環境においてインターネット通信を使用する機能を利用できません。
