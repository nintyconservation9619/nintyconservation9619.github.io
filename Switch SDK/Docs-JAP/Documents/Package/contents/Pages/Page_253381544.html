設定マネージャ

Webコンテンツの開発のために、設定マネージャを使ってWebアプレット・オフラインWebアプレットの動作をカスタマイズすることができます。

使用方法

設定マネージャの使い方については、設定マネージャを参照してください。
Webアプレット・オフラインWebアプレットに関連する web_settings カテゴリの設定項目は、下記の通りです。

項目 データ種別 初期値 キー値
ルートパスとしてマウントするホストPCのパス パスを表す文字列 空文字列 web_settings.host_root_path
SSL証明書エラーを無視して接続を許可するホスト ホスト名を表す文字列 - web_settings.allowed_ssl_hosts[].host_name
Webインスペクタが有効か否か 真偽値 false web_settings.use_web_inspector

設定項目

ルートパスとしてマウントするホストPCのパスを変更する

Webコンテンツの開発のため、Webアプレット・オフラインWebアプレットは、標準でホストPCの "C:/NXWebApplet" というパスをルートパスとしてマウントしています。
設定項目の host_root_path を使って、マウントするホストPCのパスを変更することができます。
この機能の使い方については、WebCallerCommand のオフラインHTMLのパスを参照してください。
以下は、マウントするホストPCのパスを標準の "C:/NXWebApplet" から "D:/Sample" へ変更する例です。

"host_root_path": {
    "current": "D:/Sample",
    "default": ""
}

正しく設定された場合、以下のようなログが出力されます。

<web>(0) [Hfio] hostRootPath: "D:/Sample"

設定に問題があった場合、以下のようなログが赤色で表示されます。

<web>(0) [Hfio] Failed to mount hostRootPath: "D:/Sample"

WebCallerCommand で開くオフラインHTMLのパスについては、オフラインHTMLのパスを参照してください。

参考：

NintendoSDK NXAddon 4.2.0 (Firmware for NX 4.0.0-2.0) 以降から利用できます。

SSL証明書エラーを無視して接続を許可するホストを指定する

設定項目の allowed_ssl_hosts を使って、SSL証明書エラーを無視して接続を許可するホストを指定することができます。
以下は、"www.nintendo.co.jp" と "www.nintendo.com" を指定する例です。

"allowed_ssl_hosts": [
    {
        "host_name": "www.nintendo.co.jp"
    },
    {
        "host_name": "www.nintendo.com"
    }
],

正しく設定された場合、以下のようなログが赤色で出力されます。

<web>(3) [AllowSslHost] www.nintendo.co.jp by SettingsManager
<web>(3) [AllowSslHost] www.nintendo.com by SettingsManager

SSL証明書エラーを無視して接続を許可するも参照してください。

参考：

NintendoSDK NXAddon 4.2.0 (Firmware for NX 4.0.0-2.0) 以降から利用できます。

Webインスペクタを有効にする

設定項目の use_web_inspector を使って、Webインスペクタを有効にすることができます。
以下は、Webインスペクタを有効にする例です。

"use_web_inspector": {
    "current": true,
    "default": false
},

正しく設定された場合、以下のようなログが赤色で出力されます。

<web>(0) [WebInspector] uses WebInspector by SettingsManager

Webインスペクタを使う手順については、Webインスペクタを参照してください。

参考：

NintendoSDK NXAddon 5.1.0 (Firmware for NX 5.0.0-1.0) 以降から利用できます。
