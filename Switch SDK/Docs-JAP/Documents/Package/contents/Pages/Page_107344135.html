グラフィックスランタイム共通の変更点

ビューア機能を別ライブラリに分離

ビューア機能(nn::*::viewer 名前空間の機能)を別ライブラリ libnn_*viewer に分離しました。
ヘッダの場所や名前空間に変更はありませんが、ビューア機能を利用しているアプリケーションは、別途 libnn_*viewer ライブラリへのリンクを追加する必要があります。
例えば、nn::vfx::viewer を利用している場合は libnn_vfxviewer ライブラリへのリンクを追加して下さい。

ビューアライブラリプロジェクトファイル ビューアライブラリファイル
NintendoSDK\Sources\Libraries\g3d\libnn_g3dviewer\libnn_g3dviewer-spec.XXX.vcxproj libnn_g3dviewer.lib
NintendoSDK\Sources\Libraries\ui2d\libnn_ui2dviewer\libnn_ui2dviewer-spec.XXX.vcxproj libnn_ui2dviewer.lib
NintendoSDK\Sources\Libraries\vfx\libnn_vfxviewer\libnn_vfxviewer-spec.XXX.vcxproj libnn_vfxviewer.lib

 

3D ランタイムの主な変更点

nn::g3d::ResMaterial にサンプラーおよびテクスチャのディスクリプタスロットを設定する機能を追加

nn::g3d::ResMaterial にサンプラーおよびテクスチャのディスクリプタスロットを設定することが出来るようになりました。詳しくは 以下のAPIリファレンスをご覧ください。

・nn::g3d::ResMaterial::SetTextureDescriptorSlot()
・nn::g3d::ResMaterial::GetTextureDescriptorSlot()
・nn::g3d::ResMaterial::SetSamplerDescriptorSlot()
・nn::g3d::ResMaterial::GetSamplerDescriptorSlot()

また、これに伴い以下のAPIの引数または返り値が変更になっています。

・nn::g3d::ResFile::BindTexture()
・nn::g3d::ResFile::ForceBindTexture()
・nn::g3d::ResModel::BindTexture()
・nn::g3d::ResModel::ForceBindTexture()
・nn::g3d::ResMaterial::BindTexture()
・nn::g3d::ResMaterial::ForceBindTexture()
・nn::g3d::ResMaterialAnim::BindTexture()
・nn::g3d::ResMaterialAnim::ForceBindTexture()
・nn::g3d::MaterialObj::SetTexture()
・nn::g3d::MaterialObj::GetTexture()
・nn::g3d::MaterialAnimObj::SetTexture()
・nn::g3d::MaterialAnimObj::GetTexture()

nn::g3d::ResShaderAssign::FindSamplerAssignTextureIndex() および nn::g3d::ResShaderAssign::GetSamplerAssignTextureIndex() の削除

nn::g3d::ResShaderAssign::FindSamplerAssignTextureIndex () およびnn::g3d::ResShaderAssign::GetSamplerAssignTextureIndex() を削除しました。これらはサンプラーが参照するテクスチャのインデックスを知るための関数でしたが、サンプラーが参照するテクスチャのインデックスはサンプラーのインデックスと同一のため、サンプラーのインデックスから参照するテクスチャのインデックスを知ることが可能です。

nn::g3d::ResTextureRef クラスの削除

nn::g3d::ResTextureRef クラスを削除しました。ResTextureRef クラスから取得出来ていた情報は、nn::g3d::ResMaterial クラスから取得可能になっています。詳しくは nn::g3d::ResMaterial の API リファレンスをご覧ください。

オブジェクトが持つリソースポインタを const に変更

オブジェクトがリソースを変更しないことを保証するため、オブジェクトが持つリソースポインタを const に変更しました。これにより、GetResource() で取得するポインタも const に変更されています。

ドキュメントに3Dガイド、プログラミングガイドを追加

ドキュメントに3D ガイドおよびプログラミングガイドを追加しました。

 

ui2d ランタイムの主な変更点

LayoutConverter や LayoutArchiver のプラットフォームオプションが必須に

LayoutConverter や LayoutArchiver のプラットフォームオプションが必須となりました。
対象プラットフォームに合わせて適切に設定を行ってください。

LayoutArchiver でテクスチャを統合する処理を導入

LayoutArchiver でテクスチャを統合する処理が導入されました。
それに伴って、アーカイブ中に任意のテクスチャを含める方法が変更されます。詳細は関連リンクを参照してください。

vfx ランタイムの主な変更点

vfx ランタイム内部で利用されるバッファ仕様の変更について

これまでは、エミッタ生成時に各エミッタが放出するパーティクル数応じてバッファを確保・メモリプール生成 を行っていましたが、 vfx システムがメモリプールを１つ生成し、そこから各エミッタがバッファを切り出すような仕様に変更しました。
vfx システムが生成するメモリプールのサイズは、デフォルトで 1MByte が設定され、サイズ変更を行う場合は、nn::vfx::Config::SetGpuBufferSize メソッドでサイズ変更できます。

エミッタ拡張パラメータ機能追加のお知らせ

ゲームアプリで利用できる固有のデータをエミッタに付加できる 「エミッタ拡張パラメータ」 機能が実装されました。
ゲームアプリ固有の機能として利用できる値を設定でき、その値に沿った形で EffectMaker に UI を付加できます。ユーザーは UI で設定した値を編集、エフェクトデータとして保存、バイナリ化できます。
