パーカッションリスト

パーカッションリスト

バンクタブは上下に分割されて表示されます。上領域はインストリストで、下領域にはサンプルマップとパーカッションリストのタブが配置されています。
この下領域で、パーカッションリストのタブを選択した時に表示されるのがパーカッションリストです。

パーカッションリストの役割

パーカッションリストはインストルメントを編集する専用のエディタです。
インストリストで選択したインストルメントの編集を行います。

パーカッションリストは、サンプルマップと同様にマルチリージョンのインストルメントを編集するエディタですが、サンプルマップとは対照的に鍵盤を縦にリストにして表示します。
パーカッションリストはサンプルマップに比べ、非常に狭い範囲に、小さなキーマップを沢山作るケースの多い、パーカッションタイプのインストルメントの作成に適しています。
パーカッションリストでは、ベロシティリージョンを扱うことはできません。

パーカッションリストの操作

ここではパーカッションリストで可能な操作を解説します。

キーリージョンの作成

キーリージョンの作成は、リスト上で右クリックにより現れるコンテキストメニューの「キーリージョンの分割」を選択して行います。

パーカッションリストは、上から順に低音から高音の鍵盤に対応しています。
キーリージョンは、分割操作を行った行から、より高い音に対して、次のキーリージョンがある行までの領域に作成されます。

ベロシティリージョンの作成

パーカッションリストではベロシティリージョンを作成することはできません。

フォルダを開く・参照ファイルを開く

右クリックして現れるコンテキストメニューから「フォルダを開く」または「参照ファイルを開く」を選択することで、そのファイルに対するそれぞれの動作を行えます。

パーカッションリストの編集

ここではパーカッションリストの各定義要素の意味を解説します。

圧縮方式

参照波形ファイルをコンバートする際の圧縮方式を指定します。

  • ADPCM
    波形ファイルを専用の ADPCM 形式にコンバートします。
  • PCM16
    16bit の PCM データにコンバートします。

ファイルパス

リージョンが参照する波形ファイルを指定します。

ファイル名、およびパス名に使用できる文字には制限があり、以下の文字が使えます。

  • ファイル名の先頭は、半角英字とアンダーバー [a-zA-Z_]
  • ファイル名の先頭以外およびパス名は、半角英数字とアンダーバー [a-zA-Z0-9_]

再生時間

サウンドの再生時間（波形データ長の時間）を表示します。
表示単位は［分：秒．ミリ秒］で、「ミリ秒」は 3 桁目を四捨五入し、2 桁表示となります。

ループ付き波形データでは、波形そのものの長さでは無く、ループエンドまでの時間を表示します。
また、セルの先頭に "L" が表示されます。

ティック数

必要なノートコマンドの長さを表示します。
ティックは、テキストシーケンスでノートや休符の長さを指定するための単位です。

ティック数は、テンポや四分音符分解能の値に依存して値が変わりますが、テンポのデフォルト値 120 (1 分間あたり四分音符 120 回) および、テキストシーケンスの四分音符分解能のデフォルト値 48 (これはプロジェクト設定の「SMFの四分音符分解能」で設定するものとは異なります)で考えた場合、

  • 1 分 ＝ 120 * 48      ＝ 5760 ティック
  • 1 秒 ＝ 120 * 48 / 60 ＝   96 ティック

にそれぞれ相当します。

テキストシーケンスを書く際に、ティック数がわかると有用です。

ただし、インストの音程を変えた場合、実際の波形再生時間が長くなったり短くなったりしますが、これにあわせて実際の再生ティック数も変化してしまうので、注意が必要です。

サンプルレート

波形ファイルのサンプルレートを表示します。

ビットレート

波形ファイルのビットレートを表示します。

量子化ビット数

波形ファイルの量子化ビット数を表示します。

チャンネル数

波形のチャンネル数が 1, 2 のときは Mono, Stereo と表示します。 チャンネル数が 3 以上の時は、数字をそのまま表示します。

データサイズ

圧縮変換後の波形データのサイズを表示します。
マルチリージョンのインストルメントなど、複数の波形ファイルを参照する場合は、参照ファイルの合計サイズを表示します。
表示内容は、コンバートのタイミングで更新されます。

オリジナルキー

リージョンのオリジナルキーを指定します。

Env A

エンベロープは、インストルメント単位またはリージョン単位で設定できます。
エンベロープについての解説はインストルメントのパラメータを参照してください。

マルチリージョンのインストルメントの場合のみ、リージョンに対してエンベロープのアタックタイムを 0 〜 127 で指定します。
以降のエンベロープの各フェーズの指定も同様です。

Env H

エンベロープのホールドタイムを 0 〜 127 で指定します。

Env D

エンベロープのディケイタイムを 0 〜 127 指定します。

Env S

エンベロープのサスティンレベルを 0 〜 127 で指定します。

Env R

エンベロープのリリースタイムを 0 〜 127 で指定します。

音量

リージョンの音量を 0 〜 255 の範囲で指定します。

サウンドデータの音量を 127 にすると、元データと等倍の音量 (1.0倍) で再生されます。

ただし、音量は、いろいろな場面 (データやプログラム) で設定されます。
すべての設定値が掛け合わされた結果、元音量から比べて、0.0 倍 〜 2.0 倍にクランプされます。
詳しくは、サウンドシステムマニュアルの「サウンドパラメータ制御の動作」の章をご覧ください。

ピッチ（半音）

リージョンのピッチを半音単位で指定します。

ピッチ（セント）

リージョンのピッチをセント単位（半音の 1 / 100）で指定します。

パン

リージョンのパンを指定します。

パーカッションモード

リージョンに対してパーカッションモードを指定します。
チェックが入った状態が、パーカッションモードが有効な状態を示します。

パーカッションモードを指定したリージョンでは、ノートオフは無視され、ノート長に関わらず波形ファイルの最後まで再生されます。
ノート高に対応したピッチの計算は、通常通り行われます。

ループする波形ファイルを設定したリージョンへの適用は、十分ご注意ください。

ループ波形をパーカッションモード ON に指定した場合は、以下のような再生動作となります。

  • ノートオフされないので、音が鳴り続けます。
  • ノートオフされないので、リリース状態に移行しません。
  • そのため、実質的に「リリース時優先固定 OFF」設定は参照されません。

このような経緯から、次のような結果を招きます。

  • 結果的に、優先度負けでしかこの音は停止しないので、高い優先度でこの音を再生していると、使える音数が減ってしまいます。

キーグループ

リージョンに対して、キーグループを 0 〜 15 の範囲で指定します。

値に 0 を指定した状態では、キーグループは動作を行いません。キーグループのデフォルトには、0 が設定されています。

0 以外の同じキーグループを設定したリージョンでは、新しいノートが発音される毎に、同じキーグループで発音中の再生を停止します。
この機能は、例えばクローズハイハットとオープンハイハットの演奏エミュレーションなどに利用することができます。

キーグループの動作は、シーケンス内の同一トラック内でのみ有効です。
複数のシーケンスから同じインストルメントを利用するケースや、シーケンス内の複数のトラックから同じインストルメントを同時に利用するケースでは、キーグループの機能を利用することはできません。

補間タイプ

インストルメント単位で補間タイプを指定します。

ボイスの補間方法を設定し、「4 点補間」,「線形補間」が選べます。デフォルト値は 4 点補間 です。

現状、こちらの補間タイプの設定を変更しても、補間方法には反映されませんのでご了承ください。
