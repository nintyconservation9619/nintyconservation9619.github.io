サイズが奇数ピクセル数のテクスチャの扱い

ピクチャペイン等でサイズが奇数ピクセル数のテクスチャを扱うことができますが、 注意点があります。

サイズが奇数ピクセルのテクスチャをドットバイドットでピクチャペインで表示させたい場合に おいて、以下の条件がそろうと、フィルタリングによってぼやけて見えることがあります。

  1. テクスチャサイズが奇数
  2. テクスチャサイズとペインサイズが同じ(ドットバイドット)
  3. ペイン中心位置の水平、垂直方向設定が「中央」

3 の条件で、左上座標値は以下のようにもとめられます。
左上座標 = ペイン位置 - (ペインサイズ / 2)

たとえば、ペイン位置が(100, 100) ペインサイズが(39, 39) の場合、左上座標は(80.5, 80.5)になります。

座標値に小数点以下の値が入り、 テクスチャのピクセルとフレームバッファのピクセルがずれることで、 複数ピクセルからサンプリングされ、ぼやけたような見た目になります。

これを回避するには、テクスチャサイズを偶数にしてもらうか、 ペイン位置の小数点以下を .5 にする必要があります。

インダイレクトテクスチャのテクスチャフォーマットによる挙動の違い

インダイレクトテクスチャは、テクスチャフォーマットによって（正確には、フォーマットのチャンネル数によって）挙動が変わります。

1チャンネルのテクスチャフォーマット（BC4、L8、A8）インダイレクトでずらす値は、st共に同じ値が使われます。2チャンネルのテクスチャフォーマット（BC5、LA8）インダイレクトでずらす値は、Rがs、Aがtとして使われます。3チャンネルのテクスチャフォーマット（BC1）インダイレクトでずらす値は、Rがs、Gがtとして使われ、Bの値はインダイレクトをかけたあとにアルファで抜くときの値として使われます。詳しくはインダイレクトテクスチャを使用したペインでマスクまたはアルファの処理を最適化したいを参照してください。4チャンネルのテクスチャフォーマット（BC2、BC3、RGBA8）インダイレクトでずらす値は、Rがs、Gがtとして使われ、Aの値はインダイレクトをかけたあとにアルファで抜くときの値として使われます。Bの値は使われません。詳しくはインダイレクトテクスチャを使用したペインでマスクまたはアルファの処理を最適化したいを参照してください。
