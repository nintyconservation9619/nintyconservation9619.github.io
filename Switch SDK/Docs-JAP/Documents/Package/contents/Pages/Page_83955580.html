エミッタタイプについて

NintendoWare エフェクトの３つのエミッタタイプについて説明します。

エフェクトでは以下３つの「エミッタタイプ」が利用できます。
エミッタタイプによって、パーティクルの 位置 や 速度 の処理方法か異なります。

CPU エミッタ CPU エミッタは、 CPU で計算します。
GPU エミッタ

GPU エミッタは、頂点シェーダを利用して GPU で計算します。
GPU+SO エミッタ

GPU+SO エミッタは、演算処理を得意とするコンピュートシェーダを利用して GPU で計算します。

エミッタ内で行われる処理

エミッタにでは CPU または GPU で以下 5 つの処理が行われます。

エミッタ処理 処理内容
エミッタ計算処理

エミッタ時間によるアニメーションの処理やエミッタマトリクス更新処理がおこなわれます。
これらの処理は CPU で行われます。
関連リンク 「エミッタ時間」については「エミッタ時間とパーティクル時間」を参照してください。
パーティクル放出処理 エミッタから放出されるパーティクルの放出量決定や放出するパーティクルの初期化処理が行われます。
この処理はCPU で行われます。
パーティクル挙動処理 パーティクルの挙動計算処理では、パーティクルの位置や速度の更新処理を行います。
エミッタタイプ によって計算処理を行うユニットが CPU または GPU になります。
ストリームアウト処理

GPU+SOエミッタ利用時のみ行われるパーティクル挙動計算をGPU上で行いメモリに書き出す処理です。

ここで書き出したメモリの内容に従ってパーティクル描画を行うため、GPU+SOエミッタは２パス描画になります。
描画コマンド発行処理

描画ステート設定やテクスチャの設定などを行い、パーティクルを描画します。

パーティクルソートを指定した場合、パーティクルソート処理(CPU)を行い、１パーティクルごとに描画処理を行います。
この処理は CPU で行われます。

GPU+SO エミッタ

GPU+SO エミッタは、パーティクルの位置や速度を GPU（コンピュートシェーダ） で計算します。以下にメリットとデメリットについて説明します。

以下は CPU と GPU で行う計算要素について説明した図です。
GPU でパーティクルの位置や速度、スケール、回転、カラー要素すべてを計算します。
さらに、パーティクルの位置や速度は演算処理専用のシェーダ「Compute Shader」で行う為、複雑な計算ができます。

メリット

  • CPU リソースをあまり消費しません。
  • コンピュートシェーダを利用することで、複雑なパーティクル挙動を計算できます。
    例えば以下のサンプルエフェクト「RockBreak」のように、炸裂した岩石を床のコリジョンと衝突判定させる場合に有効です。

  • １エミッタあたり、GPU が２回駆動するため、GPU処理リソースを消費します。
    例えば 以下のように仮想平面と衝突判定するフィールド「シンプルコリジョン」では、衝突判定の計算でGPU負荷がかかります。

エミッタタイプの詳しい説明

エミッタタイプについて、プログラマ向けにより詳しく説明します。

CPUエミッタ

CPUエミッタでは、パーティクルの位置や速度がCPUで処理されます。カスタムアクション利用してゲームアプリ側でパーティクル情報を取得したりカスタマイズする場合などは、EffectMaker上で自動的にCPUエミッタが選択されます。CPUエミッタの場合、エミッタ計算処理、パーティクル放出処理の後にCPUにてパーティクル挙動処理を行います。パーティクル挙動処理では、位置や速度以外にもゲームアプリ側で参照する為のカラーやスケールも計算されますが、実際のパーティクルの描画にはこれらの値は利用されず頂点シェーダ内で計算された値で描画されます。また、パーティクル挙動処理ですが、下図で示すように、入力の パーティクル位置 に対して、パーティクル速度 を足しこむような処理を CPU 上で行います。この パーティクル位置 と パーティクル速度 はマルチバッファ化されており、GPU が参照中のバッファをCPU側で書き換えないようになっています。

GPUエミッタ

パーティクル位置やパーティクル速度がパーティクル時間から計算可能な場合は GPUエミッタ が選択されます。パーティクル位置やパーティクル速度の計算は、パーティクル描画処理時に頂点シェーダ内でパーティクル初期位置やパーティクル初期速度をもとにパーティクル時間から計算されます。パーティクル位置やパーティクル速度がパーティクル時間から計算される、前フレームの位置や速度に対して操作するようなパーティクル挙動は計算や、逆に現フレームの計算結果を次フレームで参照するなどの柔軟性はありませんが、CPU/GPU速度面において一番コストの低いエミッタタイプになります。

GPU+SO エミッタ

パーティクル位置やパーティクル速度が時間から計算できない場合 ( フィールド / シンプルコリジョンを利用時など ) は、GPU+SO エミッタとして駆動します。GPU+SO エミッタは、各パーティクルの挙動計算を コンピュートシェーダを利用して行い、計算結果をメモリに出力し、それをもとにパーティクルの描画を行う為、２パス描画になります。
