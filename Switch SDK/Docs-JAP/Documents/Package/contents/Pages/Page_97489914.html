はじめに

本ドキュメントの目的

本ドキュメントでは NintendoSDK に含まれる zlib についての説明と、zlib を使用するための方法を説明します。

zlib について

zlib は deflate アルゴリズムを用いてデータを圧縮／展開するオープンソースライブラリです。http://www.zlib.net/ で配布されています。

対象読者

NintendoSDK を使ってアプリケーション・ミドルウェアを開発される方を対象としています。

NintendoSDK に含まれる zlib

NintendoSDK には zlib を NintendoSDK に合わせて改変したものが含まれています。
zlib のバージョンは、

  • NX Addon 5.1.0 以前 : 1.2.8
  • NX Addon 5.2.0 以降 : 1.2.11

です。

改変箇所

  • バージョン番号の変更 (ライセンス条項に基づく変更)
  • Z_PREFIX 有効時のプレフィックスを z から nnutilZlib へ変更
    • ただし、ビルド時には Z_PREFIX は定義していないため、プレフィックス付きの定義はリンカエラーとなります。
  • fcntl.h と sys/types.h の include を行わない。

なお、 gz から始まる API はコンパイル対象から除外しています。 (gzip file access functions は使えません)

UTIL ライブラリと ZLIB API の提供

NintendoSDK に含まれる zlib を利用するには次の 2つの方法があります。

UTIL ライブラリ経由での提供

nn::util ライブラリの Compress / Decompress API は内部で zlib を使用しています。これらの API を使用することで簡易的ながら zlib を使用することができます。

これらの API を利用するには以下のヘッダーを include します。

  • nn/util/util_Compression.h
  • nn/util/util_Decompression.h
  • nn/util/util_StreamingDecompression.h

zlib API 機能の提供

zlib の API について理解されている方は、本来の zlib の API を使用することもできます。

zlib API を直接利用する場合は zlib.h を include してください。

ZLIB ライブラリが提供する機能

UTIL ライブラリ経由で提供する機能

UTIL ライブラリ経由では簡易的ながら zlib を使用した圧縮・伸張機能が利用できます。

詳しくはリンク先の関数リファレンスをご覧ください。

ZLIB API の直接利用で提供する機能

zlib API の直接利用では zlib の各機能を zlib API のインターフェースのまま利用できます。

ただし、利用における制限事項及び注意事項があります。

制限事項

  • gzip file access functions は使用できません。

注意事項

  • NintendoSDK に含まれている zlib は Windows API を使用しない状態でコンパイルされています。
  • zlib.h のインクルード前に FAR マクロが空以外で定義されているとリンクエラーとなります。
    • 例えば nn_Windows.h では、 FAR が far として定義されており、これに該当します。
    • 従って、 zlib.h と nn_Windows.h は同じファイルからインクルードしないように注意してください。

ZLIB ライブラリのライセンスについて

zlib のライセンスは zlib ライセンスとなっています。

使用に制限や宣言は必要ありません。ライセンス条項に関しては http://www.zlib.net/ をご覧ください。
