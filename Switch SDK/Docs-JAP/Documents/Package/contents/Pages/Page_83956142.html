チーム設定ファイル

概要

チーム設定ファイルを使用することにより、3DEditor の動作を調整することができます。

3DEditor が読み込むチーム設定ファイルは 3DEditor のコマンドラインオプション --team-settings-file　により指定できます。

コマンドラインオプションにチーム設定ファイルが指定されない場合は、3dEditor.exe と同じフォルダにある 3dEditor.teamsettings が使用されます。このファイルもない場合は、既定の動作をします。

チーム設定ファイルは、3DEditor の起動時に読み込まれます。

参考：

3DEditor 起動中に設定ファイルを編集しても変更は反映されません。変更を反映するためには 3DEditor を再起動する必要があります。

 

ここでは、主な設定項目を紹介します。詳細は、チーム設定ファイルフォーマットを確認してください。

ファイル入出力設定 <FileIo>

ファイルの入出力にかかわる設定を記述します。

主な設定項目は以下の通りです。

  • スタートアップパス : 3DEditor が起動時に開くファイルを指定します。
  • 追加参照パス : 参照ファイルの検索やスタートアップパスの起点、プロジェクトに含まれるファイルのパスの解決に使われます。
  • ファイル操作時のコマンド : ファイルの保存前後とオープン前に呼び出すプログラムを指定します。
  • シェーダ最適化用のシェーダバリエーション生成コマンドを指定します。

プリセット設定 <Preset>

テクスチャフォーマットのプリセットが設定できます。

デフォルト値設定 <DefaultValue>

3DEditor が使用するデフォルト値等を記述します。

サンプラのミップマップのフィルタ等のデフォルト値を設定できます。

またシェーダパラメータの範囲の制限や 3DEditor で編集可能な項目を設定できます。

エラー設定 <Error>

エラー時に送るメールの宛先や件名等を設定できます。

プレビュー設定 <Preview>

実機での想定FPSやメッセージ送信間隔を設定できます。

カラー設定 <Color>

ガンマ補正を行うかどうかを設定できます。

プラットフォーム設定 <PlatformPresets>

プラットフォーム設定を確認できます。

チーム設定で <PlatformPresets> を指定しない場合は、3DEditor の標準のプラットフォーム設定が使われます。

プラットフォーム設定で追加したプラットフォームは、ツールバーのプラットフォームから選択できるようになります。
