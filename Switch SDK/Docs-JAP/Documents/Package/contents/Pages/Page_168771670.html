アプリケーションに付属可能なドキュメントの作成

アプリケーションに付属可能なドキュメントの種類

アプリケーションが設定できるドキュメントには以下の三つがあります。

  • オフライン HTML
    • オフライン Web アプレットが参照するデータです。
    • 使い方の詳細は オフラインWebアプレット を参照してください。
  • Web アプレットで参照できるインターネット上のサイトの URL リスト
    • 詳細は Webアプレット を参照してください。
  • ソフトリーガル情報
    • おもに法的に記載必須とされている内容です。
    • Nintendo Developer Portalにある「Software Legal Information System（SLIM）」というシステムで作成したものを利用します。

アプリケーションに付属可能なドキュメントの作成


  • AuthoringTool でアプリケーションの nsp を作成する時に各ドキュメントのディレクトリや zip ファイルを指定するか、 nmeta ファイルにパスを記述します。
    • nmeta ファイルでの記述方法は nmeta ファイルの書き方 を参照してください。
    • AuthoringTool のオプションと nmeta ファイルでの指定が重複した場合、 nmeta ファイルの記述が優先され使われます。

AuthoringTool の実行例

$ AuthoringTool.exe creatensp --type Application --desc Application.desc --meta Application.meta --program programDirectory dataDirectory


VSI を利用してアプリケーションをビルドする場合には、 「構成プロパティ」 > 「オーサリング」にそれぞれディレクトリや zip ファイルを指定してください。
