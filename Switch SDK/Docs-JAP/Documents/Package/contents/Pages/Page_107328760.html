はじめに

DevMenuCommand は DevMenu の機能をコマンドラインで実行するためのターゲット向けツールです。
以下の操作が実行可能です。

  • ファームウェア情報の確認
  • アプリケーションのインストール、リスト、起動、アンインストール
  • パッチのインストール、リスト、アンインストールGe
  • セーブデータのリストアップ、削除
  • コントローラペアリング設定の削除
  • コントローラの無線接続、切断
  • ユーザーアカウントの追加、削除
  • フレンド機能
  • データ配信機能
  • 時計の設定
  • 開発支援機能
  • アルバム保存ファイル管理機能
  • エラーレポート機能
  • NX 開発カードへのアプリケーションの書き込み、NX 開発カードの状態取得、NX 開発カードの初期化
  • NAND 空き容量のフィル
  • SD カードの状態取得・SD カードの初期化
  • ネットワーク情報の取得
  • Mii 開発支援機能
  • 保護者による使用制限(ペアレンタルコントロール)の設定
  • TV 設定, オートスリープ設定
  • 「ダウンロードできる本体」の登録
  • プレイ記録の取得
  • 上記コマンドのバッチ実行


参考：

DevMenuCommand は本体機能が使用するメモリを一部借りて動作するようになっているため、DevMenuCommand 実行中は本体機能（主に HOME メニューですが、DevMenu も含む）をあまり操作しないようにしてください。DevMenuCommand 実行中にこれらの操作を行うとメモリ不足で FATAL エラーが発生する場合があります。そのような場合には、一度本体をリセットして起動し直してください。

使用方法

RunOnTarget を利用してツールを起動します。

$ Tools\CommandLineTools\RunOnTarget.exe TargetTools\NX-NXFP2-a64\DevMenuCommand\Release\DevMenuCommand.nsp -- --help
usage: DevMenuCommand <command> [<args>]

The DevMenu commands are:
   firmware        Show firmware information
   application     Install, uninstall, list or launch applications
   savedata        List or delete savedata
[SUCCESS]

SDK 1.1.0 以降では、一度の DevMenuCommand 呼び出しで複数処理を順番に行うことも可能です。逐次 RunOnTarget.exe で DevMenuCommand.nsp を呼び出すのに比べて、呼び出し時間が少なくて済みます。

複数処理の指定は、特定のトークンでコマンドを区切ることで行います。

トークン 説明
; (セミコロン) 処理を逐次実行します
&&

直前の処理が成功 (SUCCESS) の場合に、次のコマンドを実行します
|| 直前の処理が失敗 (FAILURE) の場合に、次のコマンドを実行します
THEN ; (セミコロン)の代替記法です。ただし、前後にスペースが必要です。
AND && の代替記法です。ただし、前後にスペースが必要です。
OR || の代替記法です。ただし、前後にスペースが必要です。

bash などのシェルでは ; (セミコロン)、&&、|| はシェル側でパースされてしまうため、バックスラッシュなどでのエスケープを行うか、THEN、AND、OR の代替記法を利用してください。

利用例は下記のとおりです。

$ Tools\CommandLineTools\RunOnTarget.exe TargetTools\NX-NXFP2-a64\DevMenuCommand\Release\DevMenuCommand.nsp -- application list AND application uninstall 0x01004b9000490000 OR application uninstall 0x01004b9000490001
1 application(s) found.
id                          version
0x01004b9000490000            0.0.0
Not found.
[SUCCESS]


従来同様 batch コマンドも利用できるので、都合の良いほうを利用してください。

アプリケーションのインストール

アプリケーションのインストール方法については、 NX アプリケーションを開発機にインストールして実行する もあわせて参照してください。
nsp ファイルについては、アプリケーションイメージの作成 や 環境構築ガイド を参照してください。

$ Tools\CommandLineTools\RunOnTarget.exe TargetTools\NX-NXFP2-a64\DevMenuCommand\Release\DevMenuCommand.nsp -- application install "C:\application.nsp"
Preparing to install C:\application.nsp...
[SUCCESS]

--force オプションを付けることで、既に同じプログラム ID でインストールしている場合もインストールできます。
(元あったものは削除されます) 

$ Tools\CommandLineTools\RunOnTarget.exe TargetTools\NX-NXFP2-a64\DevMenuCommand\Release\DevMenuCommand.nsp -- application install "C:\application.nsp" --force
Preparing to install C:\application.nsp...
[SUCCESS]

-s オプションを付けることで、インストール先のストレージを指定することが出来ます。
オプションで指定できるストレージは以下のとおりです。

  • builtin: NAND にインストールします。
  • sdcard: SD カードにインストールします。
  • auto: インストール先を自動で選択します。

-s を指定しなかった場合は NAND にインストールされます。

$ Tools\CommandLineTools\RunOnTarget.exe TargetTools\NX-NXFP2-a64\DevMenuCommand\Release\DevMenuCommand.nsp -- application install "C:\application.nsp" -s sdcard
Preparing to install C:\application.nsp...
[SUCCESS]

インストール対象の nsp 名の先頭に "sdcard:\" を指定することで、SD カード上の nsp をインストールできます。

$ Tools\CommandLineTools\RunOnTarget.exe TargetTools\NX-NXFP2-a64\DevMenuCommand\Release\DevMenuCommand.nsp -- application install "sdcard:\application.nsp"
Preparing to install sdcard:\application.nsp...
[SUCCESS]

アプリケーション起動パラメータのインストール、アンインストール

参考：

NintendoSDK NX Addon 3.3.0 以降で使用可能です。

アプリケーション起動パラメータについては、起動パラメータ取得 もあわせて参照してください。

SD カードにインストールしたファイルを、アプリケーションで起動パラメータとして取得できます。

注意：

インストールしたパラメータは、メニューに DevMenu を選択しているときのみ取得できます。
HOME メニューを選択しているときは、パラメータがインストールされていても、アプリケーションに渡されないことに注意してください。

メニューを切り替える方法は、本ページの「メニューの切り替え」を参照してください。

開発機に SD カードを挿入してから、コマンドを実行してください。アプリケーションには、ファイルのデータがそのまま渡されます。
既にインストールされているパラメータがあるときは、--force オプションを使用して、パラメータを上書きする必要があります。

$ Tools\CommandLineTools\RunOnTarget.exe TargetTools\NX-NXFP2-a64\DevMenuCommand\Release\DevMenuCommand.nsp -- application install-nxargs "C:\parameter.nxargs" --application-id 0x01004b9000490000
[SUCCESS]

SD カードにインストールされた起動パラメータを、アンインストールします。

$ Tools\CommandLineTools\RunOnTarget.exe TargetTools\NX-NXFP2-a64\DevMenuCommand\Release\DevMenuCommand.nsp -- application uninstall-nxargs 0x01004b9000490000
[SUCCESS]

アプリケーション管理データのキャッシュの無効化

nmeta ファイルで指定するアプリケーション管理データのキャッシュを無効化します。

$ Tools\CommandLineTools\RunOnTarget.exe TargetTools\NX-NXFP2-a64\DevMenuCommand\Release\DevMenuCommand.nsp -- application invalidate-cache application-control
[SUCCESS]

アプリケーション管理データはシステムにより本体保存メモリーにキャッシュされます。以下のケースではゲームカード内のアプリケーション管理データではなくキャッシュが読み込まれるようになっており、HOME メニューや DevMenu のランチャーでキャッシュが読み込まれないようにするには本コマンドを実行する必要があります。

ケース 対処
同バージョンのアプリケーションが書き込まれたゲームカードを挿しかえる 本コマンドを実行後に新しいゲームカードを挿す。HOME メニューの場合は続けて開発機を再起動する。
同バージョンのアプリケーションを DevMenu または DevMenuCommand で再インストール

HOME メニューの場合のみ、開発機を再起動する。

アプリケーション起動時にキャッシュが更新されるため、アプリケーション起動後は本コマンドを実行せずとも ROM に含まれるアプリケーション管理データが使用されます。

必須バージョンのリセット

古いバージョンのアプリケーションが起動するのを防ぐ仕組みである、必須バージョンの情報をリセットします。

$ Tools\CommandLineTools\RunOnTarget.exe TargetTools\NX-NXFP2-a64\DevMenuCommand\Release\DevMenuCommand.nsp -- application reset-required-version 0x01004b9000490000
Reset: 0x01004b9000490000

注意：
  • EDEV で CPU オーバークロックを有効にしても効果はありません。無効時と同じ CPU クロックレートで動作します。
  • CPU オーバークロック有効時は無効時に比して SDEV の表面温度が上昇します。取り扱いにはご注意ください。
  • CPU オーバークロック有効時は PCM ライブラリによる消費電力測定結果および CPU プロファイラの結果は、無効時の結果と大きく異なる値になります。これらの値を製品環境の参考にする場合は CPU オーバークロックを無効にしてください。

  • スリープモードが pseudo の場合は、電源ボタンを押す等をしてもシステムは実際にはスリープせず、1秒程度でスリープから復帰します。いくつかの点で製品のスリープの挙動とは異なりますのでご注意ください。
    • スリープの前後でホスト PC とターゲットの接続を維持することができます。
      • スリープ通知等のアプリケーションに対するインタフェースは製品と同様に動作するため、スリープをまたいでアプリケーションをデバッグするのに有用です。
    • ネットワークやコントローラの接続済のセッションが切断されません。
    • 無操作で放置した際の自動スリープからも 1 秒程度で復帰しますが、画面焼け軽減は解除されません。

コマンド実行後のスリープから変更後の設定が有効となります。システムの再起動は不要です。

無操作状態検出モードの制御

開発用に、無操作状態を検出した際に働く自動スリープや画面焼け軽減機能の有効／無効を変更することができます。

この設定を有効にすると、本体設定（HomeMenu 環境の本体設定画面で変更できる自動スリープまでの時間設定など）に従って自動スリープ等が発動します。

この設定を無効にすると、本体設定を無視して自動スリープや画面焼け軽減機能がすべて無効になります。

本機能は現時点ではデフォルトで 無効 となっています。製品では常時有効です。

エージングテスト等の用途では、このコマンドを使って自動スリープを無効に設定する必要があります。

アルバム保存ファイルのダウンロード

開発機のアルバムに保存されている全てファイルを PC の指定したディレクトリにコピーします。コピー元となるアルバムのストレージと、コピー先の PC 上のディレクトリの指定が必要です。コピー先に同名のファイルが存在する場合は、デフォルト動作としてはスキップします。また、RunOnTarget.exe に --suppress-auto-kill オプションを付加して DevMenuCommand.nsp を起動することで、事前に起動していたアプリケーションを終了させることなくアルバムコマンドを実行することができます。

$ RunOnTarget.exe --suppress-auto-kill TargetTools\NX-NXFP2-a64\DevMenuCommand\Release\DevMenuCommand.nsp -- album download --storage <builtin|sdcard> --directory <path> [options]
[SUCCESS]

--storage オプションでダウンロード元となるストレージを builtin か sdcard で指定します（指定必須）。builtin は本体内蔵ストレージ、sdcard は SD カードとなります。

--directory オプションでダウンロード先となる PC 上ディレクトリの絶対パスを指定します（指定必須）。ドライブのルート（d:/ など）は指定できません。

--force オプションを付加すると、コピー先に同名ファイルが存在する場合に強制的に上書きします。

--check-empty オプションを付加すると、コピーを開始する前に、コピー先ディレクトリが空かどうかをチェックし、空でない場合にはダウンロード処理を停止します。

時間制限の設定

時間制限の設定は、製品ではローカル設定のみでは行えず、スマホ連携を行った場合にのみ設定することができますが、アプリケーション開発時の挙動確認(デバッグ)を目的として、DevMenuCommand にて時間制限の設定を行うことができます。

時間制限は「timer-set」で残り時間等の設定を行います。利用する場合、次の内容にご注意ください。

  • 事前に暗証番号の設定が必要です。
  • あらかじめネットワーク時計が有効な状態にする必要があります。このため、インターネット設定を完了させてインターネット通信ができる状態にする必要があります。
  • 実際に時間制限による通知や強制中断を発生させるには、本体起動時にHOMEメニューが起動される状態にする必要があります。
  • 「timer-start」で時間制限と残り時間計測を開始、「timer-stop」で停止しますが、HOMEメニューが起動される状態である場合は「timer-start」と「timer-stop」に相当する処理が自動で行われるため、通常はこの操作は不要です。
  • その他、本機能は開発時の利便性(デバッグ用途)を考慮して設けられたものであるため、一部製品版と異なる挙動になる場合があります。また、製品では起こりえない操作によって予期しない挙動になる場合があります。

timer-set サブコマンドの構文は以下の通りです。

usage: DevMenuCommand pctl timer-set [--timer-mode <mode>] [--using-week-settings <true|false>] [--week <week>] [--bedtime [<true|false>][,HH:mm]]  [--limit-time [<true|false>][,<minutes>]] 
    --timer-mode <mode> : set timer mode
      <mode> : Alarm (0), Suspend (1)
    --using-week-settings <true|false> : specify whether the week settings is used
    --week <week> : specify the week (or daily-settings) which to change settings
      <week> : Sunday (0), Monday (1), Tuesday (2), Wednesday (3),
               Thursday (4), Friday (5), Saturday (6), Daily (7)
      * if omitted, Daily settings will be changed
    --bedtime [<true|false>][,HH:mm] : set bedtime
      <true|false> : if true, the bedtime will be used (if omitted, the flag will not be changed)
      HH:mm : the bedtime to set (24h format)
      * example: --bedtime false / --bedtime 22:00 / --bedtime true,23:45
    --limit-time [<true|false>][,<minutes>] : set limit time
      <true|false> : if true, the limit-time will be used (if omitted, the flag will not be changed)
      <minutes> : the limit-time to set (minutes)
      * example: --limit-time false / --limit-time 60 / --limit-time true,90
  * At least one option except --week must be specified to change settings.
  * The omitted settings will not be changed.
オプション 説明
--timer-mode 時間制限のモードを設定します。「Alarm」は通知のみ、「Suspend」は時間切れの際に強制中断の割り込み表示を行います。
--using-week-settings 曜日ごとの設定値を用いるかどうかを指定します。曜日は本体設定の日付を元に利用されます。
--week

どの曜日(あるいは「毎日」の設定)の設定値を変更するかを指定します。設定値は曜日別と「毎日」(Daily)の計8セット存在します。

省略した場合は「毎日」の設定値を変更します。
--bedtime

指定した時刻を期限とする制限時間を設定します。指定時刻に達した場合「時間切れ」となります。なお、時刻は本体設定の時刻設定およびタイムゾーンに基づき、「00:00～05:59」は翌日扱いで計算されます。

時刻の設定のほか、「true」「false」を引数に付加することで、この値を用いるかどうかを指定します。省略した場合は「用いるかどうか」が不変となります。
--limit-time

指定した時分を「プレイ可能時間」として設定します。当日00:00～翌00:00の直前までの間に、timer-start して timer-stop (スリープや通常の電源断も含む) するまで、もしくは timer-stop されていない場合は現在までの時間を「プレイ時間」とし、「プレイ可能時間」-「プレイ時間」を残り時間として計算します。設定値を変更すると「プレイ可能時間」のみ変更となるため、「残り時間」がリセットされることにはならない点にご注意ください。(「プレイ時間」は日付を跨ぐとリセットされます。)

時間の設定のほか、「true」「false」を引数に付加することで、この値を用いるかどうかを指定します。省略した場合は「用いるかどうか」が不変となります。


--bedtime と --limit-time を両方有効にした場合、「残り時間」はそれぞれの設定から算出される値のうち小さい方が適用されます。

ショップ初回接続時の設定と「ダウンロードできる本体」の登録

shop start コマンドを使用して初回のショップアクセス時に行われる処理を実行出来ます。
ユーザーアカウントとニンテンドーアカウントをリンクした状態で shop start コマンドを実行すると、実行した本体が「ダウンロードできる本体」として登録されます。
初回のショップアクセス時には、「ダウンロードできる本体」の登録以外にもショップの利用に必要な設定が行われるため、1つの本体に対して1回は必ず shop start コマンドを実行する必要があります。

$ Tools\CommandLineTools\RunOnTarget.exe TargetTools\NX-NXFP2-a64\DevMenuCommand\Release\DevMenuCommand.nsp -- shop start 0
Registering as active console of the account user index 0 ...
[SUCCESS]

「ダウンロードできる本体」の登録

ユーザーアカウントとニンテンドーアカウントをリンクした状態で shop link-device コマンドを使うと「ダウンロードできる本体」の登録が出来ます。
すでに別の本体で「ダウンロードできる本体」が登録されているニンテンドーアカウントを指定した場合、登録に失敗します。
「ダウンロードできる本体」を変更する場合は、「ダウンロードできる本体」に登録した本体で登録を解除してから、目的の本体で「ダウンロードできる本体」の登録を実行してください。

$ Tools\CommandLineTools\RunOnTarget.exe TargetTools\NX-NXFP2-a64\DevMenuCommand\Release\DevMenuCommand.nsp -- shop link-device 0
Registering as active console of the account user index 0 ...
[SUCCESS]

[SUCCESS]

コマンドのバッチ実行

batch サブコマンドを使うことで、DevMenuCommand の各サブコマンド実行を一括で行なうことができます。

まず最初に以下のような、DevMenuCommand のサブコマンド以降を列挙したテキストファイルを用意します。ファイル名は仮に batch.txt としておきます。
行頭に # もしくは // を置くことでコメント行とすることが出来ます。

// アプリケーションのインストール
application install "C:\application1.nsp"
application install "C:\application2.nsp"
application install "C:\application3.nsp"
application install "C:\application4.nsp"

上記のテキストファイルを batch コマンドで指定します。

$ Tools\CommandLineTools\RunOnTarget.exe TargetTools\NX-NXFP2-a64\DevMenuCommand\Release\DevMenuCommand.nsp -- batch c:\batch.txt
Preparing to install C:\application1.nsp...
Preparing to install C:\application2.nsp...
Preparing to install C:\application3.nsp...
Preparing to install C:\application4.nsp... 
[SUCCESS]

--ignore-error オプションを付けると、エラーが出ても処理を続行します。
