Nintendo Switchガイドラインへの準拠の確認

別途提供されている「Nintendo Switchガイドライン」では、アプリケーションが順守すべき事項を定めています。すべてのアプリケーションはガイドライン項目に準拠していることを確認してください。アプリケーションが該当するすべてのガイドライン項目に準拠しなければ、マスター ROM が受理されません。

開発用の性能を使用していないことの確認

nn::oe::PerformanceConfiguration で設定できる性能には、開発時にしか利用できないものがあります。開発時にしか利用できない性能を使用しているアプリケーションは製品版の NX で動作しませんので、必ずこれらの性能が使用されていないことを確認してください。PROD MODE に設定した EDEV を使用し、nn::oe::SetPerformanceConfiguration で性能設定を更新するシーンの動作確認を行うことを推奨します。

nn::oe::PerformanceConfiguration で設定できる性能の詳細は、以下のページを参照してください。

  • 性能選択

アプリ起動時にチェックされる各種項目が正しく設定されていることの確認

VSI や RunOnTarget 経由でアプリケーションを実行した場合は、アカウント選択やセーブデータの作成、保護者による使用制限のチェックなどアプリケーション起動時のチェックがいくつか行われます。ただしすべてのチェックや処理が行われるわけではありません。VSI や RunOnTarget での起動は開発用の機能ですので、製品と相違点があります。

アプリケーションに各種メタ情報が正しく設定されているかは、HOMEメニューからアイコンを選んで起動することで確認してください。このとき、アプリケーションは本体にインストールしたものでも、NX 開発カードに書き込んだものでも構いません。

メタ情報については、「nmeta ファイルの書き方」を参照してください。

BG ダウンロードストレステストの有効化

BG ダウンロードストレステストとは、アプリケーションのバックグラウンドでシステムがデータをダウンロードしている状況を作り出す機能です。この機能を有効にしてアプリケーションを動作させ、バックグラウンドでデータがダウンロードされているときもアプリケーションの挙動に支障がないことを確認してください。

BG ダウンロードストレステストの設定については、以下のページを参照してください。

  • DevMenuCommand
  • 開発支援機能の設定

この機能とテストは、将来の本体更新によって不要になる予定です。

注意：

この項目は開発期間中からテストすることができます。DEV MODE でも PROD MODE でも有効化することができますが、最終動作確認環境としては PROD MODE との併用は推奨しません。

FS の速度エミュレーション機能の有効化

アプリケーションがアクセスする記録メディアは、その種類や経年劣化によってアクセス速度が異なります。そのような場合でもアプリケーションが支障なく動作するために、開発者は記録メディアのアクセス速度に依存しないよう実装する必要があります。このことを確認するため、「速度エミュレーション」という機能を有効にして動作確認してください。この動作確認は、パッケージ版のアプリケーションを開発する場合も実施する必要があります。

アプリケーションを本体保存メモリーにインストールした上で、速度エミュレーションの Slower Mode、Faster Mode、Random Mode のそれぞれのモードで動作確認をして下さい。速度エミュレーション機能は DevMenu の「Debug ＞ FS Speed Emulation Mode」から設定できます。この動作確認や速度エミュレーション機能の詳細については以下のページを参照して下さい。

  • アクセス速度依存確認
  • 開発支援機能の設定
参考：

FS の速度エミュレーション機能の Slower Mode と BG ダウンロードストレステストは併用可能です。

ただし、Faster Mode と Random Mode は BG ダウンロードストレステストと併用しないでください。BG ダウンロードストレスによりアクセスが遅くなり、定常的に速めなアクセス速度とランダムに変化するアクセス速度を意図した性能でエミュレーションできなくなるためです。

なお、Slower Mode でのテストはフリーズするなどの深刻な不具合がないかを確認するもので、遅めの SD カードで快適なアプリケーション動作を求めるものではありません。

注意：

この項目は開発期間中からテストすることができます。DEV MODE でも PROD MODE でも有効化することができますが、最終動作確認環境としては PROD MODE との併用は推奨しません。

動作モードと性能モードの組み合わせに対応していることの確認


アプリケーションが全ての動作モードと性能モードの組み合わせに対応していることを確認してください。（特定のプレイスタイルに対応しないアプリケーションは除きます。）
動作モードと性能モードの組み合わせは下表の通りです。


 

携帯モード

据置モード
ノーマルモード ドックからの抜去時にはこの組み合わせになります ドックへの挿入時、電池残量が不十分であるとこの組み合わせになります
ブーストモード 発生しません ドックへの挿入時、電池残量が十分であるとこの組み合わせになります

多くの場合、「携帯モードかつノーマルモード」もしくは「据置モードかつブーストモード」ですが、「据置モードかつノーマルモード」の状態も存在します。
全ての組み合わせでアプリケーションを期待通り動作させるためには、動作モード遷移と性能モード遷移を混同せずに、各遷移に対応する処理を適切に実装する必要があります。

「据置モードかつノーマルモード」の組み合わせは通常の操作では維持することが難しいので、この組み合わせのデバッグには DevMenuCommand の動作モードの制御ポリシーの設定機能、性能モードの制御ポリシーの設定機能を使用してください。

NX Addon 3.2.0 以降では 開発機の制御／DevMenu の開発支援機能（Debug）に動作モードの制御ポリシーの設定機能（Operation Mode Policy）、性能モードの制御ポリシーの設定機能（Performance Mode Policy）が追加されましたのでより容易に設定可能です。

キャプチャーボタン長押しによる動画撮影機能の確認

NX Addon 4.2.0（NX Addon 3 系では 3.6.0 以降）より、Switch の本体機能としてキャプチャーボタンの長押しによる動画撮影に対応しました。

アプリケーションではこの機能がデフォルトで有効化されていますが、nmeta ファイルの設定によって完全に無効化したり、API 呼び出しによって一時的に無効化することができます。

アプリケーションがインフォーカス状態の場合、キャプチャーボタンの長押しにより動画撮影処理を実行します。
機能の無効化や再有効化が設定できているか、あるいは撮影処理による処理負荷の増加によって動作に支障がないかを確認してください。
