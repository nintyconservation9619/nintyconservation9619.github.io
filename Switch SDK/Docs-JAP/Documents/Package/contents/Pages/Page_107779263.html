新規プロジェクトの作成

ここでは、NX アプリケーション開発のための Visual C++ プロジェクトの作成方法を説明します。

プロジェクトの作成方法は次の 2 つがあります。

  • 新規にプロジェクトを作成する
    • 新規アプリケーション作成
    • 新規スタティックライブラリ作成
    • 新規ダイナミックライブラリ作成
  • 既存の Visual C++ プロジェクトを NX アプリケーション開発に対応させる
  • 外部のビルドシステムと連携したアプリケーション開発を行う

新規にプロジェクトを作成する

参考：

NintendoSDK 及び NX Addon がサポートしている Visual Studio のバージョンは、動作環境 (NX Addon) をご確認ください。

新規アプリケーション作成

Visual Studio を起動し、[ファイル] - [新規作成] - [プロジェクト] を選びます。

[テンプレート] - [ウィザード] - [NintendoSDK] - [NintendoSDK アプリケーション] を選び、[名前] と [場所]、[ソリューション名] を適切に入力します。

参考：

これは Visual Studio の “既定の環境設定” が 「全般」「Visual C++」「Web 開発」のいずれかである場合です。それ以外の設定 (「Visual C#」等) である場合は [テンプレート] - [他の言語] - [ウィザード] - [NintendoSDK] - [NintendoSDK アプリケーション] にあります。

NintendoSDK のウィザードが起動します。

NDI でインストールした環境を上部のドロップダウンリストから選択します。

警告：

現在、こちらで選択した絶対パスがプロジェクトに追加される仕様になっています。
そのため、作成したプロジェクトを別のディレクトリにインストールした SDK 環境で利用する場合は、このプロパティシートに記述されたパスを書き換える必要があります。
今後絶対パス以外の方法でルートパスを参照できるように対応予定です。なお、2 つ目のラジオボタンの「環境変数もしくはパスを指定する」で環境変数を利用することで、回避できます。
これについては プロジェクトで使用する SDK のパスを変更するを参照してください。

 

API セットの選択画面が表示されます。ここで 「NX」 を選ぶと、NX API セットを利用できるようにしたプロパティシートをインポートしたプロジェクトファイルが作成されます。
（「汎用」 を選ぶと、NX Addon を含めない NintendoSDK 環境で作成したものと同じプロジェクトファイルが作成されます。）

参考：

NintendoSDK には API セットと呼ばれる概念があります。

NintendoSDK 単体で提供されている API を 「汎用 API セット」 と呼びます。
NintendoSDK NX Addon を導入することで利用できるようになる API を 「NX API セット」 と呼びます。

完了を押すと、プロジェクト一式が作成されます。
ウィザードから生成された Visual Studio プロジェクトは、「追加のインクルードディレクトリ」、「追加のライブラリディレクトリ」、「追加の依存ファイル」 などに NintendoSDK が提供するアイテムが参照されるよう、適切な値が入力されています。

新規スタティックライブラリ作成

新規アプリケーション作成の手順と同様に Visual Studio のプロジェクト新規作成を選択し、
[テンプレート] - [ウィザード] - [NintendoSDK] - [NintendoSDK ライブラリ] を選び、[名前] と [場所]、[ソリューション名] を適切に入力します。

参考：

これは Visual Studio の “既定の環境設定” が 「全般」「Visual C++」「Web 開発」のいずれかである場合です。それ以外の設定 (「Visual C#」等) である場合は [テンプレート] - [他の言語] - [ウィザード] - [NintendoSDK] - [NintendoSDK ライブラリ] にあります。

以降、新規アプリケーション作成の手順と同様、利用する SDK 環境と API セットを選択し、完了を押すと、プロジェクト一式が作成されます。

新規ダイナミックライブラリ作成

新規アプリケーション作成の手順と同様に Visual Studio のプロジェクト新規作成を選択し、
[テンプレート] - [ウィザード] - [NintendoSDK] - [NintendoSDK ライブラリ] を選び、[名前] と [場所]、[ソリューション名] を適切に入力します。

参考：

これは Visual Studio の “既定の環境設定” が 「全般」「Visual C++」「Web 開発」のいずれかである場合です。それ以外の設定 (「Visual C#」等) である場合は [テンプレート] - [他の言語] - [ウィザード] - [NintendoSDK] - [NintendoSDK ライブラリ] にあります。

新規スタティックライブラリ作成の時と同様にウィザードが表示されますので設定を入力していくのですが、以下のようなライブラリ詳細設定のページが表示された時に[ライブラリの種類]として「DLL」を選んでから完了ボタンを押して下さい。

するとプロジェクト一式が作成されます。

既存の Visual C++ プロジェクトを NX アプリケーション開発に対応させる

NX Addon を導入済みの環境では、既存の Visual C++ プロジェクトで NX アプリケーションの開発を行えるように設定することもできます。NX Addon 0.3.0 までに作成したプロジェクトも設定可能です。
手順は既存の Visual C++ プロジェクトに NX 開発機用のプラットフォームを追加するを参照してください。

外部のビルドシステムと連携したアプリケーション開発を行う

makefile といった外部のビルドシステムと連携した開発を行うことも可能です。
手順は Makefile の使用方法 を参照してください。
