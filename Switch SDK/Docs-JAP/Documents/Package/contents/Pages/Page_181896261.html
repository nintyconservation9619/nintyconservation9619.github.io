はじめに

本ドキュメントの目的

本ドキュメントでは、NintendoSDK に収録されている EC ライブラリの機能について説明します。

対象読者

NintendoSDK を使ってアプリケーションを開発される方を対象としています。

関連資料

EC の関数リファレンス と併せて参照して下さい。

EC ライブラリの提供形態

以下のページを参照してください。

  • ライブラリの提供形態と利用方法

EC ライブラリは現時点では NX 版のみが提供されています。

EC ライブラリが提供する機能

EC ライブラリは下記の機能を提供します。

  • ニンテンドーeショップで販売中の追加コンテンツを一覧表示・販売する画面にジャンプする機能
  • ニンテンドーeショップで販売中の期間型サービスアイテムを一覧表示・販売する画面にジャンプする機能
  • ニンテンドーeショップで販売中の消費型サービスアイテムを一覧表示・販売する画面にジャンプする機能
  • ニンテンドーeショップで扱うアプリケーションの詳細情報を表示し、アプリケーションを販売する画面にジャンプする機能
注意：

ショップの画面へのジャンプを行う際、ニンテンドーアカウントを連携させたユーザーアカウントが必要です。EC ライブラリはショップへのジャンプを行う前にこのチェックを行い、事前に選択されたユーザーアカウントがありニンテンドーアカウントを連携済みである場合は、そのユーザーアカウントを使用します。また、条件を満たすユーザーアカウントがない、または引数に指定された値が指すユーザーアカウントが条件を満たさない場合は、「アカウント管理アプレット」を呼び出して条件を満たすユーザーアカウントの選択を促します。

追加コンテンツ・期間型サービスアイテム・消費型サービスアイテムの一覧表示および販売

EC ライブラリでは、アプリケーション内から特定の追加コンテンツや期間型サービスアイテム、および消費型サービスアイテムの購入をスムーズに行えるようにするための、ショップ販売画面へのジャンプ機能を提供します。この機能はライブラリアプレットの1つである「EC アプレット」を利用して行われ、EC ライブラリの特定の関数を呼び出すことでEC アプレットへのジャンプを行います。

なお、ショップ販売画面へのジャンプ機能を利用するには、事前にアカウントライブラリ(ACCOUNT ライブラリ)の初期化が必要です。

アプリケーションの詳細情報表示および販売

また、EC ライブラリではアプリケーションそのものの詳細情報の表示や、アプリケーションの購入を行うことができる画面へのジャンプ機能を提供します。こちらも「EC アプレット」を利用して提供される機能であり、特定の関数を呼び出すことでジャンプを行います。例として、体験版アプリケーションの画面から、nn::ec::ShowShopApplicationInformation に製品版アプリケーションのアプリケーションIDを指定して呼び出すことで、製品版アプリケーションの詳細情報を表示する、といった使い方をすることが可能です。

なお、こちらの機能についても事前にアカウントライブラリの初期化が必要です。

その他

EC ライブラリで扱う追加コンテンツや期間型サービスアイテム、および消費型サービスアイテムの概要については「eコマースガイド」を、アプリケーション内で追加コンテンツを利用する際に利用するAOC ライブラリについては「AOC ライブラリ」のページをご覧ください。

また、販売画面へのジャンプ機能はライブラリアプレットの1つである「EC アプレット」を利用して行われます。ライブラリアプレットの概要および仕様は「本体機能ガイド」の「ライブラリアプレット」をご覧ください。
