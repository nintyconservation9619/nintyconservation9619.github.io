独自のビルドシステムを構築する (Windows 環境)

はじめに

このページでは、NintendoSDK が提供する Visual Studio 用プロパティシートを使用せずに独自のビルドシステムを構築しようとする方を対象として、Windows 環境用のアプリケーションを作成するために必要な情報を提供します。

SDK が必要とする Visual Studio の設定

ランタイム ライブラリ

NintendoSDK のライブラリが使用している Visual C++ ランタイム ライブラリ設定は、ライブラリのビルドタイプにより異なります。詳しくは ビルドターゲットとビルドタイプ を参照してください。

文字セット

NintendoSDK のライブラリはすべて Unicode 文字セットを使用する設定でビルドされています。

SDK のインクルードディレクトリ

NintendoSDK のインクルードディレクトリは次の場所にあります。コンパイル時にこれらのインクルードディレクトリを指定する必要があります。

  • %NINTENDO_SDK_ROOT%\Common\Configs\Targets\<ビルドターゲット>\Include
  • %NINTENDO_SDK_ROOT%\Include

ビルドターゲットについては ビルドターゲットとビルドタイプ を参照してください。

SDK のライブラリディレクトリ

NintendoSDK のライブラリディレクトリは次の場所にあります。リンク時にこれらのライブラリディレクトリを指定する必要があります。

  • %NINTENDO_SDK_ROOT%\Libraries\<ビルドターゲット>\<ビルドタイプ>

ビルドターゲットとビルドタイプについては ビルドターゲットとビルドタイプ を参照してください。

NintendoSDK が提供するライブラリの一覧は ライブラリの提供形態と利用方法 を参照してください。

SDK が必要とするプリプロセッサマクロ

NintendoSDK では提供する機能を切り替えるためにいくつかのプリプロセッサマクロを必要とします。以下のプリプロセッサマクロをコンパイラオプションとして与えるようにしてください。

必須マクロ

以下のプリプロセッサマクロは常に定義してください。

  • NN_NINTENDO_SDK

選択必須マクロ

以下のプリプロセッサマクロはビルド設定に応じていずれか 1 つを定義してください。

ビルドタイプ

使用する NintendoSDK ライブラリのビルドタイプに応じていずれか 1 つを定義してください。

  • NN_SDK_BUILD_DEBUG
  • NN_SDK_BUILD_DEVELOP
  • NN_SDK_BUILD_RELEASE
