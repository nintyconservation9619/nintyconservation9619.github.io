ディスプレイプロセス

概要

ディスプレイプロセスの主な目的は、物理ディスプレイとそこに送信するすべてのコンテンツを管理することにあります。この目的は、最終的にアプリケーションが関わるいくつかの領域の管理に行き着きます。アプリケーションは、VI ライブラリを通じてディスプレイプロセスとやりとりします。

システムポリシー

プラットフォームによっては、パフォーマンスを損なわずに物理ディスプレイに送信できるコンテンツに対して厳格な制限が存在することがあります。パフォーマンスの低下を引き起こす可能性のあるアクションやハードウェア上の制限に起因して実行できないアクションをアプリケーションで実行しようとすると、そのアクションは拒否されます。アプリケーションから見ると、この機能から該当のエラー結果が返されます。

ディスプレイプロセスは、論理ディスプレイの使用に伴うさまざまなポリシーに関する情報を提供します。これらの論理ディスプレイは、1 台以上の物理ディスプレイにマッピングされます。詳細については、このセクションを参照してください。

合成処理

複数のプロセスから物理ディスプレイにコンテンツを同時に送信できます。ディスプレイプロセスでは、これらすべてのソースからコンテンツを受け取り、それらを最終的に物理ディスプレイに提供する単一のイメージに合成します。

参考：

ディスプレイプロセスで使用する合成機能はアプリケーションからは使用できず、システムでの使用専用に確保されています。

ディスプレイプロセスでは、アプリケーションに代わって合成処理の詳細を管理します。この方法により、すべてのシステム機能を考慮した適切な出力をユーザに提供できるようになります。

フレームバッファの管理

ディスプレイに表示するすべてのコンテンツは、どこかの時点でディスプレイプロセスを通過します。アプリケーションでは、最終的なレンダーターゲットにマップするレイヤオブジェクトを作成します。レンダーターゲットとバックのテクスチャメモリはアプリケーション側で所有します。一方で、ディスプレイプロセスではレイヤオブジェクトを管理し、ディスプレイでも GPU でも使用していないテクスチャを要求に応じて提供します。

イベントを構成する処理体系全体についてはここに説明があります。この処理体系は、プラットフォームで使用できるさまざまなバッファモードの基礎を形成します。
