はじめに

ここでは、SDEVおよびEDEVでネットワーク機能を利用する方法を案内します。

SDEV/EDEVのネットワーク設定

SDEV/EDEVは無線もしくは有線接続でネットワークへ接続できます。接続先のアクセスポイントの設定は、「設定マネージャ」もしくはシステムアプリ上の本体設定で設定できます。

設定マネージャを利用する場合は、NIFMライブラリのサンプル（NifmNetworkConnectionSimple）に含まれる「ImportNetworkSettings-spec.*.bat」を利用する方法が簡単です。詳細は「NIFM ライブラリ」を参照してください。

有線接続を行う場合は、ドックとUSB Ethernet アダプターが必要になることに注意してください。SDEVの上面にある「Ethernet ポート」はPCとの接続専用です。このポートを直接ネットワークに接続して利用することはできません。設定方法についての詳細は「開発機における注意点」を参照してください。


ネットワークサービスアカウントの利用

独自サーバーやNEXなどのネットワーク機能を利用するためには、SDEV/EDEVにネットワークサービスアカウントを登録する必要があります。ネットワークサービスアカウントについての詳細は、Nintendo Developer Portalからダウンロードできる「アカウントガイド」を参照してください。ネットワークサービスアカウントの登録は、DevMenuの「Accounts」メニューもしくはシステムアプリ上の本体設定で行います。

DevMenuの「Accounts」メニューを利用した登録の手順については「ネットワークサービスアカウントの利用」を参照してください。参照先に記載してあるように、開発環境ではDevMenuの「Accounts」メニューで「Require Nintendo Account for use  of Network Service」の値を「Disabled」に変更する必要があることに注意してください。

システムアプリ上で設定を行う場合は、ネットワークサービスアカウントを作成するためにニンテンドーアカウントを登録する必要があります。開発環境では、ニンテンドーアカウントはNX上では作成できないため、PCのブラウザ上からニンテンドーアカウントを作成した上でNXと紐づけてください。PCのブラウザ上でニンテンドーアカウントを作成する方法についての詳細は「開発環境でのニンテンドーアカウント作成手順」を参照してください。

ネットワーク機能の開発準備

アプリケーションが独自サーバーやNEXなどのネットワーク機能を利用する場合、以下の準備を行う必要があります。

  1. 製品コードを取得し、OMASにタイトル登録を行う。
  2. OMASでネットワーク機能の利用申請を行う。
  3. OMAS登録時に発行されたアプリケーションIDをプロジェクトに設定する。

OMASへのタイトル登録と利用申請

OMAS（Online title MAnagement System）とは、ネットワークサービス対応のアプリケーションを開発する際に、ネットワークサービス使用を申請するWebサイトです。OMASへはNintendo Developer Portalのリンクからアクセスできます。

ネットワーク機能の利用に必要な任天堂のサーバー設定は、OMASの申請を元に行われます。また、アプリケーションIDの発行もOMASが行います。

アプリケーションはOMASで申請された内容が任天堂により承認され、任天堂のサーバーにアプリケーションの情報が登録された時点からネットワーク機能の利用が可能になります。

アプリケーションへのアプリケーションIDの設定

アプリケーションがネットワーク機能を利用する場合、任天堂のサーバーに設定されたアプリケーションIDと、アプリケーションのメタデータに設定されているアプリケーションIDが一致している必要があります。アプリケーションIDはnmetaファイルに記載し、アプリケーションに設定します。詳細は「アプリケーションの開発」を参照してください。

ネットワーク機能の開発

ネットワーク機能の開発を行う場合は、Nintendo Developer Portalからダウンロードできる「独自サーバー構築マニュアル」や「eコマースガイド」、「NEXガイド」などの関連する各種マニュアルを参照してください。
