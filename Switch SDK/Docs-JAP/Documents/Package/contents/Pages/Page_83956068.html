ガンマ特性とは？

画像における「明るさの特性」のことを「ガンマ特性」と呼び、以下に示すようなグラフで示されます。

fz
図のおいて、横軸が画像が持つ元々の明るさで、縦軸が実際にテレビ画面に出力される明るさです。

ガンマ特性 2.2（sRGB）

グラフに実線で書かれているのは、一般的なPCモニターやテレビのガンマ特性をあらわす ガンマ特性 2.2 のカーブです。
中間調～暗部が実際よりも暗めに表示されることが読み取れます。

リニア色空間

点線で描かれている明るさの特性は、元の明るさと実際に出力される明るさが完全に一致します。
これを「リニア色空間」と呼びます。

今までのガンマ特性の扱いかたとその問題点

従来の製作手法では、テレビのガンマ特性 2.2 にあわせてデータを作っていました。 テレビで見た目がちょうど良くなるように、デザイナがデータ側を適切に調整をしていたことになります。

おおむね問題なく開発できていたはずですが、ガンマ特性の影響で以下のような問題がありました。

  • 物理挙動に基づいた写実的なレンダリング計算を行っても、正しい表示結果にならない
  • ライト調整が困難な場合がある（暗い部分にあわせて調整すると、明るい部分が白とびしてしまう）

リニアワークフロー

前項で上げたような問題を解決するために、ゲームプログラム内部で、リニア色空間の画像を扱う制作手法をとります。
これをリニアワークフローと呼びます。


以下にリニアワークフローのデータの流れを示します。
ポイントは大まかに３つです。

  • フレームバッファにリニア色空間の画像を生成する(図中のA)。
  • シェーダなどに入力するテクスチャや、カラーには補正を行う(図中のB)。
    （デザイナ作業モニタのガンマ特性に合わせた調整を排除するため）
  • フレームバッファをテレビに出力する際に、テレビのガンマ特性を打ち消すような補正を行って出力する(図中のC)。
