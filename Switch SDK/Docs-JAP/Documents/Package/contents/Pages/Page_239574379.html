3D テクスチャーコンバーター

3D テクスチャーコンバーターの使用方法と注意事項を説明します。

  • 概要
  • コマンドラインオプション
  • 利用例

概要

3D テクスチャーコンバーターは GFX のテクスチャーコンバーターにテクスチャー中間ファイルを扱う機能を追加したコンバーターです。

画像ファイル (.tga、.png、.dds、.exr) からテクスチャー中間ファイル (.ftxb、.ftxa) を作成します。
1 つまたは複数のテクスチャー中間ファイルをテクスチャーバイナリーファイル（.bntx）に変換します。

テクスチャー中間ファイルを入力として、フォーマットなどを変更して再変換することもできます。
また、テクスチャー中間ファイルを画像ファイルに変換することもできます。

画像ファイルを各フォーマットの DDS ファイルに変換したり、1 つまたは複数の DDS ファイルをテクスチャーバイナリーファイルに変換することもできます。

ユーザーデータのみ

テクスチャーバイナリーファイルには対象プラットフォームに最適化した形式のデータが格納されます。

テクスチャー中間ファイル、テクスチャーバイナリーファイルでは常に画像の左上が原点です。
キューブマップのテクスチャーデータは右手座標系で +X、-X、+Y、-Y、-Z、+Z の順に格納されます。
以下の説明で、キューブマップの座標軸は右手座標系です。

入力ファイル

入力ファイルを指定します。 画像ファイル (.tga、.png、.dds、.exr) またはテクスチャー中間ファイルを指定します。
テクスチャーバイナリーファイルを出力する場合は複数のテクスチャー中間ファイルまたは DDS ファイルを指定できます。
（画像ファイル 1 つを直接テクスチャーバイナリーファイルに変換することも可能ですが、アライメントのためにデータ量が多くなるので推奨しません。）
--cubemap-back、--cubemap-left、--cubemap-right、--cubemap-top、--cubemap-bottom オプションを指定した場合、入力ファイルはキューブマップの前面 (+Z 方向) となります。 
16 bit/チャンネルの PNG ファイルは実数型として扱います。
DDS ファイルは、dxtc / bc / astc / 各 RGBA フォーマットに対応しています。入力ファイルが DDS の場合、-f、-i などのオプションを指定しなければ再エンコードしないで出力します（複数の DDS ファイルをテクスチャーバイナリーファイルに変換する場合は常に再エンコードしません）。
-t オプションを指定して 3D テクスチャーおよび配列テクスチャーを出力する場合、ファイル名の末尾が数値になっている必要があります（詳細は -t オプションの説明を参照）。

--args-file ""

引数ファイルに記述した内容を、引数として指定できます。
引数ファイルは複数行に渡って各オプションを記述できます。
行頭に # がある行はコメントとして扱います。
オプション値指定の "" によるエスケープに改行を含めることはできません。

--job-list ""

ジョブリストファイルを指定します。
ジョブリストを使用すると 1 回のプロセス起動で複数の変換処理を実行できます。
ジョブリストファイルには改行区切りで各処理の引数を記述します。
--job-list 以外の引数を同時に指定した場合、--job-list 以外の引数の後にジョブリストファイルの各行で指定した引数を追加した場合と同様の動作になります。
行頭に # がある行はコメントとして扱います。 
ジョブリストファイル内で --version、-h、-s、--job-list オプションは指定できません。

--version

バージョン情報を表示します。

-h --help

ヘルプを表示します。

-s --silent

処理中のメッセージ表示を抑制します。

--project-root ""

プロジェクトのルートフォルダーを指定します。
指定した場合、テクスチャー中間ファイルの <create> の src_path にルートフォルダーからの相対パスを出力します。 

--disable-file-info

入力ファイルがテクスチャー中間ファイル以外なら、テクスチャー中間ファイルに <file_info> を出力しません。
入力ファイルがテクスチャー中間ファイルなら、テクスチャー中間ファイルの <file_info> を更新しません。
テクスチャー中間ファイル以外を出力する際は、このオプションは影響しません。

--disable-original-image

テクスチャー中間ファイルにエンコード前の元画像データを含めません。
ファイルサイズが小さくなりますが、出力したファイルを再変換すると画質が劣化します。
入力ファイルが元画像データを含まないテクスチャー中間ファイルの場合、このオプションを指定しなくても出力ファイルに元画像データは含まれません。
テクスチャー中間ファイル以外を出力する際は、このオプションは影響しません。

--disable-memory-pool

テクスチャーバイナリーファイルを出力する際にメモリープールのアライメント制限を考慮せず、メモリープールのオブジェクト領域も確保しません。

外部のメモリープールを使用する場合に指定します。
テクスチャーバイナリーファイル以外を出力する際は、このオプションは影響しません。 

--tile-mode ""

テクスチャーバイナリーファイルを出力する場合にタイリングのモードを指定します。
linear を指定すると、リニアーなタイリングのデータを出力します。
linear 以外を指定する場合、対象プラットフォーム向けの Addon がインストールされている必要があります。
このオプションを指定しない場合、リニアーなタイリングのデータを出力します。

--tile-optimize ""

テクスチャーバイナリーファイルを出力する場合にタイリングの最適化モードを指定します。
パフォーマンスが高くなるように最適化するなら performance、サイズが小さくなるように最適化するなら size、データ削減率に応じて自動的にサイズ優先の最適化を選択するなら size_auto を指定します。
size_auto を指定した場合、--tile-size-threshold オプションでデータ削減率のしきい値を指定します。
このオプションを指定しない場合、パフォーマンス優先の最適化をします。
このオプションは --tile-mode オプションで特定のモードを指定した場合のみ影響します。

--tile-size-threshold ""

--tile-optimize オプションで size_auto を指定した場合に、サイズ優先の最適化をするか判断するためのしきい値 [%] を 0 以上 100 以下の整数で指定します。
（サイズ優先の場合のサイズ / パフォーマンス優先の場合のサイズ）x 100 がしきい値以下になるテクスチャーはサイズ優先で最適化し、それ以外のテクスチャーはパフォーマンス優先で最適化します。
しきい値を大きくするほど、サイズ優先が選択される可能性が高くなります。
このオプションを指定しない場合のしきい値は 75 です。

--sparse-tiled ""

テクスチャーバイナリーファイルを出力する場合に、スパーステクスチャー向けのタイリング変換をするなら true、通常テクスチャー向けのタイリング変換をするなら false を指定します。
このオプションを指定しない場合、通常テクスチャー向けのタイリング変換をします。 
現在、 スパーステクスチャーは特定プラットフォームでのみ使用できます。

-o "" --output ""

出力ファイルを指定します。テクスチャー中間ファイル、テクスチャーバイナリーファイル、画像ファイル（.tga、.png、.dds、.exr）を指定します。
このオプションを指定しない場合、入力ファイルが画像ファイルなら拡張子を .ftxb にしたファイルを出力します。
入力ファイルがテクスチャー中間ファイルなら同じファイルに上書きします。

テクスチャーバイナリーファイルを指定した場合、入力ファイルがテクスチャー中間ファイルまたは DDS ファイルなら -c とテクスチャーバイナリーファイル関連以外のオプションは無視されます。例外として、入力ファイルが DDS 1 つの場合、各オプションが反映されます。

画像ファイルを指定するとテクスチャー中間ファイルのエンコード前の画像データを TGA、PNG、DDS、EXR ファイルに出力できます。
画像ファイルをエンコードして各フォーマットの DDS ファイルを出力することもできます（-f オプションを指定）。
TGA ファイルは入力ファイルが整数型の場合のみ出力できます。
EXR ファイルは入力ファイルが実数型の場合のみ出力できます。
TGA、PNG、EXR ファイルを出力する場合、キューブマップは水平十字キューブマップの状態で出力します。
DDS ファイルを出力する際に -i オプションが指定されていれば、ミップマップありの DDS ファイルを出力できます。
その際、-l、--mip-gen-filter オプションも反映されます。
-f オプションで srgb_* フォーマットを指定すると、リニアー空間でミップマップを計算し、sRGB 空間に戻して出力します。
入力ファイルがテクスチャー中間ファイルの場合、-f、-l、--mip-gen-filter オプションが指定されていなければ、quantize_type、linear、mip_gen_filter が反映されます。

-m "" --merge ""

マージするテクスチャー中間ファイルを指定します。
指定したテクスチャー中間ファイルの <texture_info> の設定（hint、linear、dimension、
quantize_type、mip_level、mip_gen_filter、comp_sel、weighted_compress、initial_swizzle）と
<comment> の設定でテクスチャー中間ファイルを作成します。
マージするファイルの mip_level が 2 以上の場合、出力ファイルの最小レベルの幅と高さが、マージするファイルの最小レベルの幅と高さ以下になるレベルまでミップマップを作成します。
入力ファイルがテクスチャー中間ファイルで、<comment> 要素の text 属性が空でなければ、text はマージされません。
このオプションと -n、-l、-d、-f、-w、-i、--mip-gen-filter、--swizzle、--comment オプションを同時に指定した場合、-n、-l、-d、-f、-w、-i、--mip-gen-filter、--swizzle、--comment オプションの指定が優先されます。

--dcc-preset ""

DCC プラグインのプリセット名を指定します。テクスチャー中間ファイル以外を出力する際は、このオプションは影響しません。

-n "" --hint ""

ヒント情報を指定します。
このオプションを指定しない場合、フォーマットが snorm_bc5 / snorm_eac_11_11 なら normal、snorm_bc5 / snorm_eac_11_11 以外なら空文字で出力します。
ヒント情報が normal で入力ファイルが整数型の場合、法線の Z 成分が 0 以上かつ長さが 1.0 になるように調整したテクスチャーデータを出力します。 

-l "" --linear ""

カラーを sRGB からリニアーに変換するなら 1、変換しないなら 0 を RGBA の順に 4 文字で指定します。
たとえば -l 1110 とすると、RGB 成分はリニアーに変換しますが、A 成分は変換しません。
このオプションを指定しない場合、どの成分もリニアーに変換しません。
srgb_* フォーマットの場合、このオプションの RGB 成分の指定は無視され（RGB 成分は常にリニアーに変換）、A 成分の指定のみ反映されます。

-d "" --dimension ""

次元（1d / 2d / 3d / cube / 1d_array / 2d_array / cube_array）を指定します。
3d、1d_array、2d_array の場合、-t オプションで奥行きを指定する必要があります。
cube_array の場合、-t オプションでキューブ数を指定する必要があります。
1d、1d_array の場合、入力ファイルの上端の 1 ラインを出力します。
このオプションを指定しない場合、入力ファイルが TGA、PNG、EXR なら 2d で出力します。
入力ファイルが DDS、テクスチャー中間ファイルなら元の次元を維持します。
--cubemap-back、--cubemap-left、--cubemap-right、--cubemap-top、--cubemap-bottom オプションを指定した場合は、
-d cube を指定しなくても自動的に cube で出力します。

水平十字または垂直十字キューブマップを使用する場合

--cubemap-back、--cubemap-left、--cubemap-right、--cubemap-top、--cubemap-bottom オプションを指定しないで -d cube を指定します。
その際、入力ファイルの幅と高さから水平十字または垂直十字キューブマップを判別します。

unorm_4_4、unorm_8_8、unorm_bc5、unorm_eac_11_11 を指定した場合

入力ファイルが整数型の場合に snorm / sint フォーマットを指定すると、符号なし 8bit の値（0～255）から 128 を引いた値をエンコードします。ただし、符号なし 8bit の値 0 は -127 としてエンコードします。
astc フォーマットを指定した場合、 ヒント情報が normal であれば、 入力ファイルの R 成分をテクスチャーの RGB 成分に、入力ファイルの G 成分をテクスチャーの A 成分に反映します。
実数型の入力ファイルを astc フォーマットに変換する場合、入力ファイルのピクセル値が [0,1] の範囲外の場合はクランプします。

特定のアドオンがインストールされていない環境で bc1 / bc2 / bc3 フォーマットを指定した場合、ゆるやかにカラーが変化する領域が広い画像なら、ディザを使用してバンディングを抑制します。
次の条件をすべて満たす場合にディザを使用します。

  • 入力ファイルが整数型
  • 幅または高さが 256 ピクセル以上かつ、幅と高さのうち小さいほうが 8 ピクセル以上
  • 周辺ピクセルとの RGB 値の差の最大値が 1 のピクセルが 50% 以上
  • 周辺ピクセルとの RGB 値の差の最大値が 64 以上のピクセルが 20% 未満

周辺ピクセルは上下左右斜めの 8 方向、RGB 値は 8 bit（256 段階）で計算します。

-w "" --weighted-compress ""

bc1、bc2、bc3、etc1、etc2 の圧縮をする際に、RGB の誤差に対して人間の目に合わせた重み付けをするなら true、重み付けをしないなら false を指定します。 
このオプションを指定しない場合、入力ファイルが TGA、PNG、DDS、EXR なら重み付けをしません。 
入力ファイルがテクスチャー中間ファイルなら元の設定を維持します。

-t "" --depth ""

3D テクスチャーおよび配列テクスチャーの奥行きを指定します。 
キューブ配列テクスチャーの場合はキューブ数を指定します。
1D 配列テクスチャーの場合 -d 1d_array、2D 配列テクスチャーの場合 -d 2d_array、
水平十字／垂直十字キューブマップの場合 -d cube_array も同時に指定します。

このオプションを指定して -d オプションを省略した場合、入力ファイルが 2D なら 3D テクスチャー、入力ファイルがキューブマップならキューブ配列テクスチャーとなります。

入力ファイルのファイル名の末尾の数値を増やしたファイルを、このオプションで指定した枚数だけ入力ファイルとして使用します。
たとえば、入力ファイルが image1.tga で -t 3 とすると、image1.tga、image2.tga、image3.tga を使用します。

入力ファイルが image010.tga で -t 4 とすると、image010.tga、image011.tga、image012.tga、image013.tga を使用します。
キューブマップの各方向の画像を個別指定する場合、--cubemap-back、--cubemap-left、--cubemap-right、--cubemap-top、--cubemap-bottom オプションで指定したファイル名の末尾の数値を増やしたファイルも 入力ファイルとして使用します。

--no-encoding

再エンコードしないで変換するなら指定します。
配列テクスチャーを作成する際に指定すると、DDS ファイルまたはテクスチャー中間ファイルのエンコード済みデータを結合します。
テクスチャー中間ファイルを DDS ファイルに変換する際に指定すると、テクスチャー中間ファイルのエンコード済みデータを DDS ファイルに出力します。
このオプションを指定した場合、--version、-h、-s、-o、-t、-c およびテクスチャーバイナリーファイル関連以外のオプションは無視されます。

以下は配列テクスチャーを作成する場合の注意点です。
入力ファイルは DDS ファイルまたはテクスチャー中間ファイルにする必要があります。入力ファイルのミップマップデータも反映されます。
-t オプションと同時に指定します。
入力ファイルの次元は、1d、2d、cube のいずれかである必要があります。
各入力ファイルのヒント情報、リニアー変換フラグ、次元、幅、高さ、フォーマット、ミップマップ数、成分選択は同じである必要があります。
DCC プラグインのプリセット名、初期スウィズル値、ユーザーデータ、編集用コメント、ツールデータ、ユーザーツールデータは先頭の入力ファイルのものを出力します。
たとえば image1.dds、image2.dds、image3.dds の次元が 2d の場合、image1.dds -o array1.dds -t 3 --no-encoding とすると、エンコード済みデータを結合して array1.dds（次元は 2d_array） に出力します。

--gpu-encoding ""

特定のフォーマットに変換する際に GPU でエンコードするかどうかを指定します。
true を指定すると、現在の環境で GPU エンコーディングが可能なら GPU でエンコードし、不可能ならエラーを発生します。
false を指定すると CPU でエンコードします。
auto を指定すると、可能なら GPU でエンコードし、不可能なら CPU でエンコードします。
このオプションを指定しない場合は true を指定したときと同じ動作です。
現在、このオプションは bc1 / bc2 / bc3 / bc6 / bc7 / astc フォーマットに変換する場合のみ影響します。
GPU でエンコードする場合と CPU でエンコードする場合では、エンコード結果が同じにならない場合があります。
グラフィックスドライバーのバージョンが古いと GPU でエンコードできない場合があります。
特定のアドオンがインストールされていない場合、このオプションは無視され、常に CPU でエンコードします。

--check-gpu-encoding

現在の環境で GPU エンコーディングが可能か判定します。
GPU エンコーディングが可能なら 3dTextureConverter.exe の終了コード（%ERRORLEVEL%）は 0、不可能なら 0 以外になります。
終了コードが 0 でも、GPU エンコーディングの対象フォーマットでなければ CPU でエンコードします。
特定のアドオンがインストールされていて、GPU とグラフィックスドライバーが条件を満たしていれば、GPU エンコーディングが可能です。
このオプションを指定した場合、他のオプションは無視され、変換は実行されません。

-i "" --mip-level ""

ミップマップの最大レベル数を指定します。 
入力ファイルの幅と高さが小さくて指定したレベル数まで作成できない場合、幅と高さが 1 になるレベルまでミップマップを作成します。
このオプションを指定しない場合、入力ファイルが TGA、PNG、EXR なら 幅と高さが 1 になるレベルまでミップマップを作成します。
入力ファイルが DDS、テクスチャー中間ファイルなら元のレベル数を維持します。
入力ファイルが DDS の場合、-i / -m / --crop / -e / -x / -y オプションのいずれも指定しなければ、入力ファイルのミップマップ画像が反映されます。

--mip-gen-filter ""

ミップマップを生成する際のフィルター（point / linear / cubic）を指定します。 
このオプションを指定しない場合、入力ファイルが TGA、PNG、DDS、EXR なら linear を使用し、入力ファイルがテクスチャー中間ファイルなら元のフィルターを維持します。

 

-c "" --comp-sel ""

RGBA の各チャンネルに対して、どのカラー成分（r / g / b / a / 0 / 1）を選択するか指定します。
出力ファイルがテクスチャー中間ファイルまたはテクスチャーバイナリーファイルの場合に有効です。
たとえば -c rrr1 とすると、R、G、B チャンネルにすべて R 成分、A チャンネルに定数 1 が選択されます。
このオプションを指定しない場合、フォーマットによって次のいずれかが選択されます。

rrr1（フォーマットの成分数が 1） 
rrrg（フォーマットの成分数が 2 で符号なし） 
rg01（フォーマットの成分数が 2 で符号あり、または unorm_16_16、uint_*_*） 
rgb1（フォーマットの成分数が 3）
rgba（フォーマットの成分数が 4）
ra01（astc フォーマットでヒント情報が normal）

入力ファイルがテクスチャー中間ファイルで、このオプションを指定しない場合、フォーマットの成分数と符号あり／なしが変わらなければ元の comp_sel を維持します。
入力ファイルが RG の 2 成分で符号なしの DDS ファイルの場合、デフォルトの成分選択は rg01 となります。

--swizzle ""

初期スウィズル値を 0 以上 7 以下の整数で指定します。初期スウィズル値は特定プラットフォームでのみ、パフォーマンスに影響します。
このオプションを指定しない場合、初期スウィズル値 0 で出力します。

--cubemap-back ""

キューブマップの後面（-Z 方向）の画像ファイルを指定します。

--cubemap-left ""

キューブマップの左面（-X 方向）の画像ファイルを指定します。

--cubemap-right ""

キューブマップの右面（+X 方向）の画像ファイルを指定します。

--cubemap-top ""

キューブマップの上面（+Y 方向）の画像ファイルを指定します。

--cubemap-bottom ""

キューブマップの下面（-Y 方向）の画像ファイルを指定します。

--mip-imageM ""
（M はミップマップのレベルを表す 1 以上 14 以下の整数）

ミップマップ画像を個別指定する場合に各レベルの画像ファイルを指定します。
このオプションで指定しないレベルは解像度の高いレベルを縮小した画像になります。

たとえばコマンドラインオプションが base.tga -i 5 --mip-image2 sub.tga の場合
レベル 0 は base.tga の画像、
レベル 1 は base.tga を 1/2 に縮小した画像、
レベル 2 は sub.tga の画像、
レベル 3 は sub.tga を 1/2 に縮小した画像、
レベル 4 は sub.tga を 1/4 に縮小した画像、となります。
sub.tga の幅と高さが base.tga の幅と高さの 1/4 でないとエラーになります。

--cubemap-frontM ""
 --cubemap-backM ""
 --cubemap-leftM ""
 --cubemap-rightM ""
 --cubemap-topM ""
 --cubemap-bottomM ""
（M はミップマップのレベルを表す 1 以上 14 以下の整数）

--cubemap-back、--cubemap-left、--cubemap-right、--cubemap-top、--cubemap-bottom オプションでキューブマップの各方向の画像を個別指定して、かつミップマップ画像も個別指定する場合に各レベルの画像ファイルを指定します。
--cubemap-frontM、--cubemap-backM、--cubemap-leftM、--cubemap-rightM、--cubemap-topM、--cubemap-bottomM がそれぞれ前面、後面、左面、右面、上面、下面のレベル M に対応します。
このオプションで指定しないレベルは解像度の高いレベルを縮小した画像になります。

--comment ""

テクスチャー中間ファイルの編集用コメント文を指定します。

--crop ""

画像の一部分を切り抜いて出力する場合に、カンマで区切った 4 つの整数値で切り抜く矩形を指定します（X,Y,W,H の形式）。
X,Y は矩形の左上の座標を 0 以上の整数値で指定します。
W,H は矩形の幅と高さを 1 以上の整数値で指定します。
-e / -x / -y オプションと併用した場合、矩形を切り抜いてからサイズ変更します。 
入力画像が水平／垂直十字キューブマップの場合、単一のビュー方向の画像に対する矩形を指定してください。

-e "" --reduce-level ""

1 以上の整数値 N を指定すると、1 / (2 の N 乗) のサイズに縮小した画像を出力します。
（N = 1 なら 1/2、N = 2 なら 1/4、N = 3 なら 1/8）
このオプションを指定しない場合は縮小しません。
-x / -y オプションと併用した場合、このオプションは無効となります。

-x "" --resize-w ""
 -y "" --resize-h ""

画像サイズを変更して出力する場合に、サイズ変更後の幅（-x）と高さ（-y）をピクセル数（1 以上の整数）または元のサイズに対するパーセンテージ（実数 + %）で指定します。
-x / -y オプションの一方のみ指定した場合は縦横比を固定してサイズ変更します。
-x 100% -y 50% とすれば、幅を変えずに高さだけ半分にできます。
バッチファイルでパーセンテージを指定する場合、50%% のようにパーセント記号を 2 つ記述する必要があります。
入力画像が水平／垂直十字キューブマップの場合、単一のビュー方向の画像の幅と高さを指定してください。

--resize-filter ""

-e / -x / -y オプションで画像サイズを変更する際のフィルターを指定します。

point ポイントサンプリング
linear 線形補間
cubic 標準的なキュービック補間
cubic_smooth 滑らかなキュービック補間
cubic_sharp シャープなキュービック補間

このオプションを指定しない場合、線形補間でサイズ変更します。

--resize-gamma ""

-e / -x / -y オプションを指定していて、-l オプションや入力画像の設定によってガンマを sRGB からリニアーに変換する場合に、リニアー空間でサイズ変更計算するなら true、 カラー空間を変換せずにサイズ変更計算するなら false を指定します。
このオプションを指定しない場合はリニアー空間でサイズ変更計算します。

--ignore-alpha ""

入力画像ファイルの A 成分を無視して不透明として処理するなら true、無視しないなら false を指定します。
不要な A 成分が設定された画像ファイルを変換する場合や、unorm_bc5 フォーマットなどで A 成分が G 成分に反映されないようにする場合に true を指定します。
このオプションを指定しない場合は A 成分を無視しません。

-u "" --adjust-transparent-rgb ""

線形補間による透明ピクセルの色のにじみを防止するために透明ピクセルの RGB 成分を調整するなら true、調整しないなら false を指定します。
true を指定すると、透明ピクセル（A 成分が 0）の RGB 成分に、隣接する透明でないピクセル（A 成分 > 0）の RGB 成分の平均を設定します。
また true の場合、-e / -x / -y オプションによるサイズ変更時に透明ピクセルのRGB 成分が影響しないようにします。
ただし、サンプリング範囲がすべて透明ピクセルなら透明ピクセルの RGB 成分を平均します。 
このオプションを指定しない場合は透明ピクセルの RGB 成分を調整しません。 

-r "" --replace-r "" 
-g "" --replace-g "" 
-b "" --replace-b "" 
-a "" --replace-a ""

入力ファイルの R/G/B/A チャンネルを、別の画像ファイルで差し替えて変換する場合に、別の画像ファイルを指定します。
別の画像ファイル名の後に「:チャンネル名」（:r/:g/:b/:a）を記述すると、別の画像ファイルの指定したチャンネルを使用します。
「:チャンネル名」を省略した場合は R チャンネルを使用します。
たとえば -g image1.tga -b image2.tga:a とすると、入力ファイルの G チャンネルが image1.tga の R チャンネルで、入力ファイルの B チャンネルが image2.tga の A チャンネルで差し替えられます。
別の画像ファイルが ftx ファイルの場合、リニアー変換フラグも差し替えます。
別の画像ファイルの解像度が入力ファイルと異なる場合、別の画像を入力ファイルにあわせてサイズ変更して処理します。その際、--resize-filter オプションで指定したフィルターを使用します。
このオプションと -t、--cubemap-back、--cubemap-left、--cubemap-right、--cubemap-top、--cubemap-bottom、--mip-imageM オプションとの併用には対応していません。

--no-tga-rle

TGA ファイルを非圧縮で出力するなら指定します。
このオプションを指定しなければ RLE 圧縮で出力します。

利用例

画像ファイルをテクスチャー中間ファイルに変換

image.tga を変換して、srgb_bc1 フォーマット、ミップマップレベル数 8 のテクスチャー中間ファイル image.ftxb を作成します。

3dTextureConverter.exe image.tga -o image.ftxb -f srgb_bc1 -i 8

merge.ftxb をマージ指定して、merge.ftxb の設定（ヒント情報、リニアー変換、次元、フォーマット、ミップマップ、成分選択、初期スウィズル値）で、テクスチャー中間ファイル image.ftxb を作成します。

3dTextureConverter.exe image.tga -o image.ftxb -m merge.ftxb

image1.tga、image2.tga、image3.tga を変換して 2D 配列のテクスチャー中間ファイル array1.ftxb を作成します。

3dTextureConverter.exe image1.tga -o array1.ftxb -d 2d_array -t 3

ジョブリストファイルを用いてまとめて変換します。
image1.ftxb と image2.ftxb のヒント情報は albedo、image3.ftxb のヒント情報は specular となります。

list.txt

image1.tga -o image1.ftxb -f srgb_bc1
image2.tga -o image2.ftxb -f srgb_bc3
image3.tga -o image3.ftxb -f unorm_8 --hint specular
 
3dTextureConverter.exe --job-list list.txt --hint albedo

テクスチャー中間ファイルをテクスチャーバイナリーファイルに変換

imageA.ftxb、imageB.ftxb、imageC.ftxb をテクスチャーバイナリーファイル textures.bntx （リニアーなタイリング）に変換します。

3dTextureConverter.exe imageA.ftxb imageB.ftxb imageC.ftxb -o textures.bntx --tile-mode linear

テクスチャー中間ファイルの再変換

image.ftxb のフォーマットを unorm_5_6_5 に変更して保存します。
テクスチャー中間ファイルはエンコード前の画像データを持っているので、再エンコードによる画質の劣化はありません。

3dTextureConverter.exe image.ftxb -o image.ftxb -f unorm_5_6_5

テクスチャー中間ファイルを TGA ファイルに変換

image.ftxb のエンコード前の画像データを original.tga に出力します。

3dTextureConverter.exe image.ftxb -o original.tga

画像ファイルを各フォーマットの DDS ファイルに変換

image.tga を変換して、srgb_bc1 フォーマット、ミップマップレベル数 8 の DDS ファイル image.dds を作成します。

3dTextureConverter.exe image.tga -o image.dds -f srgb_bc1 -i 8

image1.tga、image2.tga、image3.tga を変換して 2D 配列の DDS ファイル array1.dds を作成します。

3dTextureConverter.exe image1.tga -o array1.dds -f srgb_bc1 -d 2d_array -t 3

ジョブリストファイルを用いてまとめて変換します。
image1.dds と image2.dds のフォーマットは srgb_bc1、image3.dds のフォーマットは unorm_5_6_5 となります。

list.txt

image1.tga -o image1.dds
image2.tga -o image2.dds
image3.tga -o image3.dds -f unorm_5_6_5
 
3dTextureConverter.exe --job-list list.txt -f srgb_bc1

DDS ファイルをテクスチャーバイナリーファイルに変換

image_dxt1.dds、image_bc3.dds、image_r5g6b5.dds をテクスチャーバイナリーファイル textures.bntx （リニアーなタイリング）に変換します。
DDS ファイルのデータを再エンコードせずにそのまま利用して変換します。 

3dTextureConverter.exe image_dxt1.dds.dds image_bc3.dds image_r5g6b5.dds -o textures.bntx --tile-mode linear
