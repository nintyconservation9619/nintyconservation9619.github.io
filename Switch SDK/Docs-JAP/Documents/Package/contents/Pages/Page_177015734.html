ERR ライブラリ

はじめに

本ドキュメントの目的

本ドキュメントでは、NintendoSDK に収録されているエラーライブラリについて説明します。

対象読者

NintendoSDK を使ってアプリケーションを開発される方を対象としています。

関連資料

 エラーライブラリの関数リファレンス と併せて参照してください。

エラーライブラリの機能

エラーライブラリは、下記のエラーを表示する機能を提供します。

  • NintendoSDK や任天堂が提供するライブラリが定義する nn::Result や nn::err::ErrorCode をもとにしたエラーメッセージの表示
  • アプリケーション独自のエラーメッセージの表示

提供形態

以下のページを参照してください。

  • ライブラリの提供形態と利用方法

アプリケーション独自のエラーの表示

アプリケーションは nn::err::ShowApplicationError 関数を利用することで独自のエラーメッセージをエラーコード付きで表示することができます。

この機能を使用するには、アプリケーションを作成する際の nmeta ファイルに ApplicationErrorCodeCategory を入力する必要があります。ApplicationErrorCodeCategory には通常はそのアプリケーションのイニシャルコードを指定します。例外として、シリーズ共通で同一の ApplicationErrorCodeCategory を使用したい場合には、他のアプリケーションのイニシャルコードを指定しても構いません。

nmeta ファイルの書き方について、詳しくは nmeta ファイルの書き方 を参照ください。

参考：

nn::err::ShowApplicationError 関数はエラーの表示のために用意された関数です。汎用的なメッセージ表示のための関数ではありません。警告やお知らせなど、エラー以外のメッセージ表示には使用しないでください。

制約事項

エラーライブラリには次の制約があります。

Windows 環境でのエラー表示

Windows 環境ではエラー表示は行われません。
