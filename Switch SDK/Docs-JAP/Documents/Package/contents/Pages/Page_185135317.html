はじめに

このパッチドキュメントは、ゲーム開発者向けに、

  • パッチの作成方法
  • パッチ開発時のワークフロー
  • パッチのサイズを小さくするための方法

などの内容を記載しています。

より企画寄りの、NX のパッチがどういうユーザ体験になるか、どういう修正が可能か、といった内容はパッチガイドで説明します。そちらのドキュメントも併せて参照してください。

提供する機能

NX のパッチは、下記の機能を提供します。

  • リリースしたカードアプリの不具合修正や機能追加
  • リリースしたダウンロードアプリの不具合修正や機能追加
  • 不具合を修正した状態のカードを生産するために、カード内にパッチを抱え込む機能 (オンカードパッチ)

どの用途でも、パッチは共通のものを利用できます。

ファイルの部分パッチ

NX のパッチでは、ファイルの一部を修正したような場合に、その修正分だけをパッチに抱え込むことができるようになりました。

過去プラットフォームでは、複数ファイルのアーカイブを作成して読み込み高速化などをしていた場合などに、その一部ファイルを書き換えたとしても、アーカイブを丸ごとパッチに抱え込む必要がありました。そのためにパッチサイズが大きくなってしまう場合がありました。

NX では部分パッチに対応することで、この問題を起こりづらくすることができます。ただし、差分で修正点を抱え込むためにはいくつか制約があります。詳細は 最適化情報 を参照してください。

パッチの更新

複数のパッチをリリースした場合、そのパッチ間の更新は修正量に応じた差分になります。例えば v1 パッチが 100MB、v2 パッチが 110MB、この間の修正量が 20MB の場合、v1 を落としていたユーザは 20MB のダウンロードで済みます。（あくまでも単純化した例であり、パッチのサイズ・修正内容などによりダウンロードサイズは変わります）

パッチの更新については、開発者は特に意識する必要はありません。

パッチ適用時のファイルシステムのパフォーマンス

アプリケーションにパッチを適用した場合、アプリケーションが参照するファイルシステムのパフォーマンスが変化します。影響については パッチ適用時のパフォーマンス を参照してください。

Firmware for NX 1 系では、読み込み速度が想定以上に遅くなる不具合があるため、特に注意が必要です。

ファイルシステムのパフォーマンスの変化がゲーム性に影響を与える場合、パッチを適用した状態での検証を行うようにしてください。

 

パッチを開発可能な SDK

パッチを作成するには、NintendoSDK NX Addon 1.0.0 以降に移行する必要があります。

パッチの修正対象となる元アプリケーションは、これより古いバージョンの NintendoSDK で作ったもので問題ありません。
