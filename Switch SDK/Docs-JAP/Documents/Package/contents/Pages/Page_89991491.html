オーディオ入力

参考：

オーディオ入力の機能は、オーディオ信号の向きを除けば オーディオ出力 と同様のものとなっています。

注意：

NX では本機能は標準では使用することができません。
使用を希望される場合には任天堂窓口までご相談ください。

機能概要

オーディオ入力は、オーディオデータをデバイスから録音するための機能です。データの転送機能のみを提供するため、ボリュームのコントロールなどを行うことはできません。

オーディオ入力は AudioIn 構造体により管理されます。オーディオ入力に関する操作は、大きく分けて以下の 3 種類があります。

  • 制御（初期化、録音開始、録音停止、終了）
  • 情報取得（サンプルレート、チャンネル数、etc.）
  • 波形バッファ操作

制御

各オーディオ入力は名前を持ちます。システム上で利用可能なオーディオ入力の情報については ListAudioIns() により取得可能です。

参考：

NX プラットフォームでは、本体のヘッドホンマイク端子が "BuiltinHeadset" という名前でリストされます。
将来的に ListAudioIn () で取得されるデバイスが増えた場合にも、デバイスと名前の対応関係は保証されます。

初期化は OpenAudioIn() または OpenDefaultAudioIn() により行います。引数には名前（とサンプルレート）を指定します。録音を開始するには StartAudioIn() を呼びます。録音停止、およびオーディオ入力の終了はそれぞれ StopAudioIn(), CloseAudioIn() により行います。

情報取得

以下の情報が取得可能です。

  • 状態
  • 名前
  • サンプルレート (Hz)
  • チャンネル数
  • サンプルフォーマット

波形バッファ操作

録音に利用するバッファは AudioInBuffer 構造体（オーディオ入力バッファ）により管理します。データバッファやそのサイズを指定した後に AppendAudioInBuffer() により AudioIn 構造体に登録します。登録されたバッファは、登録順に順次オーディオ入力よりデータが録音されます。データの録音の完了は GetReleasedAudioInBuffer() により知ることができます。

オーディオ入力バッファデータは、オーディオ入力の以下のパラメータに従った構成である必要があります。

  • サンプルレート (Hz)
  • サンプルフォーマット
  • チャンネル数 (ChannelCount)

オーディオ入力バッファに含まれる（チャンネル当たり）サンプル数は以下のように計算できます。

（オーディオ入力バッファの（チャンネルごとの）サンプル数）=（オーディオ入力バッファサイズ [bytes]）/（（チャンネル数）x（サンプルフォーマットから決まる 1 サンプルのデータサイズ））

「オーディオ入力バッファの（チャンネルごとの）サンプル数」と「サンプルレート」から、オーディオ入力バッファが取得可能となるタイミングの周期が以下のように計算できます。

（オーディオ入力バッファが取得可能となるタイミングの周期）=（オーディオ入力バッファの（チャンネルごとの）サンプル数）/（サンプルレート）

例えばサンプルレートが 48000 Hz で、オーディオ入力バッファが 480 サンプルである場合、周期は 480 / 48000 = 0.01（秒）= 10（ミリ秒）となります。

バッファが登録されていない、あるいは登録されたすべてのバッファへの録音が完了すると、オーディオ入力は録音波形を破棄します。オーディオ録音が途切れないようにするために、ライブラリ利用者は適切にオーディオ入力バッファの登録を行う必要があります。

注意点

PC 開発環境と開発機環境の相違点について

PC 開発環境におけるオーディオの入力は Windows OS のサウンドドライバを利用しており、特にハードウェア割り込みのタイミングは環境に大きく依存します（一般的には割り込みの間隔の揺らぎは開発機よりも大きくなります）。このタイミングの違いにより、短いサンプル数の AudioInBuffer を利用する場合に、環境によっては音途切れが発生する可能性があります。
