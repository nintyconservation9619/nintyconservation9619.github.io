以降では、それぞれの API について説明します。

GFX

GFX は、NVN, OpenGL といった低レベルグラフィックス API を抽象化したグラフィックス API です。詳細は GFX ライブラリ を参照してください。

本 API は、以下のような用途に適しています。

  • 将来の低レベルグラフィックス API への移植を容易にしつつ、高いパフォーマンスを発揮したい
  • NintendoWare を利用したい 

将来のプラットフォームでは、対象のプラットフォームに最適化された新しいグラフィックス API が提供される可能性が高いです。GFX は、そのような場合においても、サポートする低レベルグラフィックス API を増やす形で対応することを目指しています。これにより、共通のコードを用いて、プラットフォーム毎に提供される高速な低レベルグラフィックス API を利用することが可能です。

また、NintendoWare を利用する場合には、GFX ライブラリを一緒に利用する必要があります。

加えて、GFX は、低レベルグラフィックス API との相互運用をサポートしています。これは、GFX ライブラリのオブジェクトから低レベルグラフィックス API のオブジェクトへの変換と、その逆の操作を行うことができます。この機能を用いることで、GFX ライブラリ API と低レベルグラフィックス API を併用することができます。

NVN

NVN は NX でサポートされる低レベル 3D グラフィックス API です。詳細は NVN ライブラリ を参照してください。

本 API は、以下のような用途に適しています。

  • NX で最大パフォーマンスを発揮したい

NVN は、NX で最大パフォーマンスを発揮できることを目的としており、どのグラフィックス API よりも高いパフォーマンスを発揮することができます。しかし、将来のプラットフォームでは、対象のプラットフォームに最適化された新しいグラフィックス API が提供される可能性が高く、移植性は低いです。

また、NVN は、OpenGL との相互運用をサポートしています。これは、OpenGL のテクスチャオブジェクトから NVN のテクスチャオブジェクトへの変換を行うことができます。その逆はできません。この機能を用いることで、テクスチャを介した形で NVN ライブラリ API と OpenGL API を併用することができます。

シェーダのフォーマットとしては、GLSL、SPIR-V、バイナリ形式をサポートします。

OpenGL

Khronos Group が策定するグラフィックス API の一つです。詳細は OpenGL ライブラリ を参照してください。

本 API は、以下のような用途に適しています。

  • OpenGL を利用する既存のタイトル、フレームワークを移植したい

NX では、主に他のプラットフォームからの移植を目的として OpenGL をサポートしています。

Vulkan

Khronos Group が策定するグラフィックス API の一つです。詳細は Vulkan ライブラリ を参照してください。

本 API は、以下のような用途に適しています。

  • Vulkan を利用する既存のタイトル、フレームワークを移植したい

NX では、主に他のプラットフォームからの移植を目的として Vulkan をサポートしています。
