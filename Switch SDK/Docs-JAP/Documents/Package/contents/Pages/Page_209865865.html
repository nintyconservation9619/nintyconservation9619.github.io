顔輪郭・マスクテクスチャの描画フロー

似顔絵ライブラリでは、最終的に Mii を描画するガンマ空間の種類にかかわらず、必ず sRGB 空間で顔輪郭・マスクテクスチャを生成します。そのためリニアガンマ用のリソースでも、顔輪郭やマスクを構成する各パーツのテクスチャは sRGB データになっています。

以下の二つの図はそれぞれ、sRGB 空間とリニア空間での顔輪郭とマスクテクスチャの描画のフローを示しています。

sRGB 空間
リニア空間

レンダリング対象となる nn::gfx::ColorTargetView には必ず Unorm 形式のフォーマットを設定します。顔輪郭・マスクテクスチャは sRGB 空間でレンダリングするので sRGB ライトは行いません。そのためリニア空間でアイコンを生成する場合でも ColorTargetView のイメージフォーマットには Unorm 形式が使用されます。

顔輪郭・マスクテクスチャの生成時に使用する ColorTargetView に設定すべきフォーマットは nn::mii::GetDynamicTextureColorTargetFormat() 関数から取得可能です。nn::mii::TextureShader を使用して顔輪郭・マスクテクスチャを生成する場合は、nn::mii::FacelineGpuBuffer、nn::mii::MaskGpuBuffer の内部で ColorTargetView に適切なフォーマットを設定するので nn::mii::GetDynamicTextureColorTargetFormat() 関数を使用する必要はありません。独自のシェーダで顔輪郭・マスクテクスチャを生成する場合に使用してください。

sRGB 空間での Mii の顔モデルの描画フロー

sRGB 空間での Mii の顔モデルの描画フローは以下の通りです。番号が付いている要素は、似顔絵ライブラリ利用者がアプリケーションの実装に合わせて設定する項目です。上の表の番号と対応しています。

顔輪郭とマスクテクスチャ、リソースから取得する顔の各パーツテクスチャ全てが sRGB データです。シェーダに渡るデータは全て sRGB データになります。

リニア空間での Mii の顔モデルの描画フロー

リニア空間での Mii の顔モデルの描画フローは以下の図の通りです。番号が付いている要素は、似顔絵ライブラリ利用者がアプリケーションの実装に合わせて設定する項目です。上の表の番号と対応しています。

顔輪郭テクスチャとマスクテクスチャのデータは sRGB データとして生成されますが、リソースから取得する各顔のパーツテクスチャはリニアデータです。リニア空間での描画の場合は、シェーダは受け取る全てのデータをリニアデータとして扱うので、顔輪郭とマスクテクスチャのデータをリニアデータに変換する必要があります。そのため、顔輪郭とマスクテクスチャの TextureView のフォーマットを UnormSrgb に設定し、sRGB フェッチをします。nn::mii::CharModelInfo::SetDynamicTextureFormat() に設定した値が顔輪郭とマスクテクスチャの TextureView のフォーマットになります。

基本的なフローは sRGB 空間での Mii の顔モデルの描画と同じです。nn::mii::IconShader （nn::mii::VariableIconShader）は受け取るデータを sRGB データとして扱います。sRGB フェッチや sRGB ライトの必要はないので、TextureView や ColorTargetView のイメージフォーマットの形式は Unorm を設定します。

リニア空間でのアイコンの生成フロー

リニア空間でアイコンの生成フローは以下の図の通りになります。番号が付いている要素は、似顔絵ライブラリ利用者がアプリケーションの実装に合わせて設定する項目です。上の表の番号と対応しています。

基本的なフローはリニア空間での Mii の顔モデルの描画と同じです。最終的なアイコンのテクスチャデータを sRGB データとして保持したい場合は、アイコン用のテクスチャの ColorTargetView のフォーマットを UnormSrgb に設定します。IconShader が出力するデータはリニアデータなので、sRGB ライトをして sRGB データに変換します。アイコンテクスチャのデータをリニアデータとして保持したい場合は、ColorTargetView の設定を Unorm にしてください。

注意事項

リニア空間での半透明メガネの描画をする場合の注意点

半透明メガネをかけた Mii を sRGB 空間とリニア空間で描画した結果を比較すると、見た目が異なります。これはガンマによるブレンド計算の違いのためで、sRGB 空間に比べてリニア空間で描画した Mii はメガネのカラーによって半透明メガネの濃さ（透明度）が異なって見えます。

リニア空間でレンダリングした半透明メガネの見た目を sRGB 空間でレンダリングした時の見た目に近づけるため、似顔絵ライブラリではリニア設定での描画時のみ半透明メガネのアルファ値に補正をかけています。補正処理の詳細は、変調ページや API リファレンスの nn::mii::DrawParam::ModulateType の項目を参照してください。

アイコン生成時のガンマと異なるガンマ空間でアイコンを描画する場合の注意点

アイコン生成時と異なるガンマで、背景が半透明または透明のアイコンを描画する場合は注意が必要です。

sRGB 空間生成したアイコンをリニア空間で描画する場合

sRGB 空間で、以下のような背景が透明で顔からはみ出るような半透明メガネをつけた Mii のアイコンを作成します。

sRGB 空間で生成した背景なしのアイコン

以下は、背景ありのアイコンと各ガンマ空間で背景なしのアイコンを別のテクスチャにレンダリングした結果です。レンダリング先のテクスチャは、背景ありのアイコンと同じ色でクリアしてあります。

sRGB 空間で生成した背景ありのアイコン sRGB 空間でレンダリング リニア空間でレンダリング

sRGB 空間で作成したアイコンを sRGB 空間で描画する場合には見た目に問題ありません。（背景ありのアイコンと見た目が一致しています。）ですが、リニア空間で描画した場合は、メガネの見た目が Mii の顔から飛び出した部分と顔の内側に収まっている部分とで異なります。半透明メガネの Mii の顔にかかる部分は sRGB 空間で描画されますが、顔から飛び出した部分はリニア空間で描画されます。そのため、半透明部分の描画のガンマ空間が顔の内側と外側で異なり見た目に差が出ます。

リニア空間生成したアイコンを sRGB 空間で描画する場合

リニア空間で、以下のような背景が透明で顔からはみ出るような半透明メガネをつけた Mii のアイコンを作成します。

リニア空間で生成した背景なしのアイコン

以下は、背景ありのアイコンと各ガンマ空間で背景なしのアイコンを別のテクスチャにレンダリングした結果です。レンダリング先のテクスチャは、背景ありのアイコンと同じ色でクリアしてあります。

リニア空間で生成した背景ありのアイコン リニア空間でレンダリング sRGB 空間でレンダリング

sRGB 空間同様、アイコン生成時と同じガンマ空間でアイコンを描画する場合には問題ありません。ですが、 異なるガンマ空間でアイコンを描画すると、Mii の顔から飛び出した部分と顔の内側に収まっている部分とでメガネの見た目が異なります。これは、リニア空間で半透明メガネを描画したため、アルファ値に補正がかかったままになっているためです。上の例では、半透明メガネのはみ出した部分が濃くなっています。
