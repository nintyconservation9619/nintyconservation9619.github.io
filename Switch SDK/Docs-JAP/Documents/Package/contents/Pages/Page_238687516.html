アニメーション区間タグ

アニメーション区間タグ

ここでは、アニメーション区間タグ機能について説明します。
LayoutEditor では、アニメーションの一部分に名前をつけて管理することができ、これをアニメーション区間タグと呼びます。
アニメーション区間タグは、タグ名、フレーム区間（開始フレーム、終了フレーム）、バイナリ出力に関する設定などからなります。
例えば、別途紹介する「 部品機能 」でボタン挙動アニメーションを指定する場合にアニメーション区間タグが利用されています。

アニメーション区間タグの利点

アニメーション区間タグを使うと、ひとつのアニメーションファイルを区間で区切って、複数のアニメーションを管理することができるようになります。
バイナリコンバータは、タグ情報を解釈してタグごとに別のバイナリファイルとしてファイルを出力することが可能です。

アニメーション区間のモード

ひとつの長いアニメーションカーブを区間タグで範囲指定する「アニメーション区間の結合モード」、区間タグの範囲毎にアニメーションを別ウインドウで編集する「アニメーション区間の分割モード」があります。

( ビューアでの再生は「区間タグ」を参照しますので、どちらのモードを選択しても、同様に再生することができます )


参考：

※ モードを切り替える際にアニメーションの結合操作を行う場合、区間範囲外にキーが追加される場合があり、意図しない結果を生み出す可能性があります。
そのため、基本的には「分割モード」「結合モード」のどちらかに固定した状態で作業していただくことを推奨します。

結合モード、分割モードの違い

同じアニメーション「 Start 」「 Wait 」「 End 」を使用し、「結合モード」と「分割モード」のアニメーション表示の違いを比較します。

アニメーション区間の「結合モード」を使用した場合のアニメーション作成

アニメーションを一つの繋がったアニメーションとして製作し、アニメーション区間タグによって分類する方法です。

例：

  • 00 ～ 20 フレームに作られた、合計 20 フレームのアニメーション「 Start 」
  • 20 ～ 70 フレームに作られた、合計 50 フレームのアニメーション「 Wait 」
  • 70 ～ 100 フレームに作られた、合計 30 フレームのアニメーション「 End 」
アニメーション区間の「分割モード」を使用した場合のアニメーション作成

アニメーション区間タグによって管理されているアニメーションを、別々に 0 フレーム目から作成します。

例：

  • 00 ～ 20 フレームに作られた、合計 20 フレームのアニメーション「 Start 」
  • 00 ～ 50 フレームに作られた、合計 50 フレームのアニメーション「 Wait 」
  • 00 ～ 30 フレームに作られた、合計 30 フレームのアニメーション「 End」

「基本値」

分割モードで区間タグが選択されていないときのペインの値を「基本値」と呼びます。

区間タグを選択してアニメーションを設定する際、キーが設定されていない項目は「基本値」が適応されます。
基本値は区間タグを選択していない状態でしか変更することができません。
区間タグを選択した状態でキーが設定されていない項目の値を変更すると、その値はキーを追加するまで不確定な値となります。
不確定な値は「区間タグの切り替え」「タイムラインの時間操作」「ファイル保存」を行うと「基本値」または「カーブの値」に戻ります。

分割モードでのアニメーション作成

分割モードを使用してのアニメーション作成について説明します。

編集したいアニメーションの付いたペインを選択状態にして「 アニメーション区間タグ編集ウインドウ」から、編集したい範囲を示した「区間タグ」を選択すると、アニメーションウインドウに「区間タグの範囲のアニメーション」が表示されます。
「分割モード」では、区間タグごとにアニメーションカーブを編集します。
区間タグの選択を切り替えることで個別にアニメーションカーブを作成することができます。

分割モードで作成するアニメーションは、全て 0 フレームから始まります。
