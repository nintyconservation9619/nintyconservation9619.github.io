動作環境

atk は nn::audio に依存しており、動作プラットフォームによって、CPU or DSP のどちらで処理されるのか？が異なります。

CPU レンダリング or DSP レンダリング
NX アプリケーションを SDEV 上で実行する nn::audio (DSP) でレンダリングされます
NX アプリケーションを Windows 上で実行する nn::audio (CPU) でレンダリングされます

リソースコンバータ

atk のバイナリリソースファイル ( .bfsar, .bfstm など ) は、PC、NX (SDEV など) で共通のファイルを利用します。

※SoundProjectConverter や WaveConverter には、プラットフォーム指定オプションはありません。

SoundMaker

SoundMaker と NX アプリケーションの接続方法

SoundMaker のツールバーから「接続先」と「プラットフォーム」の組み合わせで、接続先アプリケーションを切り替えることができます。

プラットフォーム 接続先 説明
PC PC アプリケーション Windows 上で動作する NintendoSDK アプリケーションに接続します。 
NX PC アプリケーション

Windows 上で動作する NX アプリケーションに接続します。

NX アドオンがインストールされている場合にみ選択できます。
NX 実機 アプリケーション

SDEV などの実機上で動作する NX アプリケーションに接続します。

NX アドオンがインストールされている場合にみ選択できます。

SoundMaker と 実機版 AtkPlayer の接続

「プラットフォーム」で【NX】を選択することによって、以下の機能が使用できるようになります。

  • 「実機版 AtkPlayer の起動」 （メニュー項目およびツールボタンが出現します）
  • 「実機版 AtkPlayer の停止」 （メニュー項目のみ出現します）

また、「接続先」で【実機版アプリケーション】を選択することで、実機版 AtkPlayer が接続対象になります。
実機版 AtkPlayer が起動したら、「接続」を実行することにより、SoundMaker と接続します。
接続が確立すると画面上に "connected." と表示されます。

参考：

「プラットフォーム」が【NX】で「接続先」が【PC アプリケーション】では PC版 AtkPlayer は接続することができません。
PC版 AtkPlayer と接続する場合は、「プラットフォーム」が【PC】、および「接続先」が【PC アプリケーション】の設定にして接続を行ってください。

プレビュー再生するサウンドの情報は、コマンドプロンプトウィンドウ、および TargetManager ウィンドウのログに表示されます。

実機版 AtkPlayer の操作方法は、AtkPlayer ページの「操作方法」を参考にして下さい。

  • SoundArchivePlayer パネル
  • SoundParameters パネル
  • SoundSystem パネル

上記の各パネルに対応する選択状態表示は、テキスト表示されている左側に ' | ' でマーク表示されます。
