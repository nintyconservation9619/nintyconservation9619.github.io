MOVIE ライブラリ


はじめに

本ドキュメントの目的

このドキュメントでは、Nintendo SDK に付属する movie ライブラリについて説明します。

対象読者

このドキュメントは、アプリケーションでマルチメディア再生がサポートされるようにしたいと思っている開発者を対象としています。

関連資料

このドキュメントは、nn::movie 名前空間 API リファレンスと合わせて使用してください。

概略

movie ライブラリは、マルチメディアコンテンツのリアルタイム再生を可能にするためにアプリケーションに組み込むことのできるソフトウェア API です。ライブラリは、オーディオ/ビデオファイルまたはエレメンタリコンテンツストリームを、プレゼンテーションに適した圧縮されていないオーディオ/ビデオデータに抽出およびデコードするために使用できます。

このライブラリでは、次に挙げる 2 つのグループの API を使用できます。

  • Extractor API：マルチメディアファイルの圧縮されたオーディオ/ビデオデータを解析、逆多重化、および出力
  • Decoder APIs：オーディオ/ビデオ圧縮データを表示可能なコンテンツにデコード

データ処理

ムービーのデコードは、圧縮されたデータを入力として受け取って、圧縮されていないオーディオ/ビデオデータを出力として生成する、データ圧縮処理と考えることができます。アプリケーションでは、Extractor API を使用してローカルのファイルソースから、または外部のメカニズムを通じて、圧縮されたデータをデコーダに提供します。また、アプリケーションでは、デコーダから提供された出力データをレンダリングしたり、必要に応じてその他の後処理を実行したりします。

movie ライブラリのビデオデコーダでは、すべてが揃った画像データがフレームバッファに存在する必要があります。現在のところ、（スライスの）部分的なデータをデコードすることはできません。ムービーデコーダは、ITU-R BT.601 の YCbCr 色空間ピクセルを NV12 形式で出力します。ムービーオーディオデコーダは、16 ビットの PCM サンプルを出力します。

依存関係

movie ライブラリは、ダイナミックリンクライブラリ（.nso ファイルと .nss ファイル）としてパッケージ化されています。movie ライブラリが正しく機能するためには、アプリケーションでさらに以下のダイナミックライブラリにもリンクする必要があります。

  • nvn
  • nnSDK
