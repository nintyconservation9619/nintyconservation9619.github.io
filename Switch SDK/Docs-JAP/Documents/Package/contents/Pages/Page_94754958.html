Windows プラットフォームにおける制約事項

Windows.h の代わりに nn/nn_Windows.h をインクルードする必要があります

<Windows.h> の代わりに <nn/nn_Windows.h> をインクルードしてください。

<Windows.h> によって定義されるマクロの中に、 NintendoSDK のシンボルと名前が衝突しているものが存在するという問題があります （例: CreateFile)。 <nn/nn_Windows.h> では、この問題の回避を行っています。

nn/nn_Windows.h をインクルードする以前に、次のマクロを定義する必要があります: WIN32_LEAN_AND_MEAN、 NOMINMAX

<nn/nn_Windows.h> をインクルードする以前に、次のマクロを定義してください:

  • WIN32_LEAN_AND_MEAN
  • NOMINMAX

これらのマクロを定義しない状態で <nn/nn_Windows.h> をインクルードした場合の動作は保証されません。

独自の nninitStartup() 定義直後のビルドで LNK2005 エラーになることがあります

nninitStartup() をアプリケーションで独自に定義を行ない、その直後に行なった最初のビルドで以下のようなリンクエラーが発生することがあります （nninitStartup() 以外にも、SDK のライブラリの中にデフォルトの関数定義があるものについては同様のリンクエラーが発生することがあります）。

1> main.obj : error LNK2005: _nninitStartup は既に libnn_init.lib(init_Startup.obj) で定義されています。

これは、Visual Studio のインクリメンタルリンク機能の影響によるものです。

このリンクエラーが発生した場合は、再度「ビルド」もしくは「リビルド」を行なって頂くと正しく再ビルドされます。
