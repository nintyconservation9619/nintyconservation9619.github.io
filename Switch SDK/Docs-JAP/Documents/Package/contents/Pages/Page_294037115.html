GPU エンコーディングについての仕様変更

--gpu-encoding false を指定しない場合に GPU エンコーディング不可能ならエラーにするように仕様変更されています。
従来のように可能なら GPU でエンコードして、不可能なら CPU でエンコードしたい場合は --gpu-encoding auto を指定してください。

gpu-encoding は、プログラマーがコンバーターに対して指定するオプションです。
担当プログラマーに相談の上で、LayoutEditor のプロジェクト設定や、FontConverter のプリファレンス設定でプレビュー時にコンバーターに与えるオプションを指定してください。

通常は、表示結果が作業者によって変わってしまうことがないようにチーム内で統一した値を指定します。

LayoutEditor

LayoutEditor に新しく追加された機能について紹介します。

整列ペイン

「親子階層」ウィンドウで階層下にあるペインの位置やサイズを調整する「整列ペイン」を追加しました！

「 整列ペイン」は、 「親子階層」ウィンドウで階層下にあるペイン同士の配置状態を維持した状態で位置やサイズを調整するペインです。
ゲームアプリケーションでの実行時に処理が行われるため、プログラム上で動的にペインサイズを変更されるときに利用できます。

例えば以下のようにローカライズ対応等で「 テキストボックスペイン」の文字列が変わった場合にサイドに配置したアイコン同士の位置関係が保持されます。

A 「整列ペイン」未使用です。

B 「整列ペイン」を使用しています。「♠スペードアイコン」が文字列にぴったり沿って移動しています。

マスク

ペインに登録したテクスチャ画像を利用して切り抜き処理ができる「マスク」の機能を追加しました！

「 マスク」は登録したテクスチャファイルを切り抜き用画像「マスク画像」として使用します。
ペイン自身の形状でマスク画像を切り抜いたり、逆にマスク画像のアルファ値でペイン自身を切り抜くことができます。

以下は「 テキストボックスペイン」の形状で「 マスク」に登録したマスク画像を切り抜いた例です。
テクスチャ座標をアニメーションさせることで、文字列に雲模様を流しています。

fcpxMaker

「複合フォントファイル(.fcpx)」を GUI で作成できるツール「fcpxMaker」がリリースされました！
fcpxMaker を使うことで、以下のようにスケーラブルフォントを利用した「複合フォント定義ファイル」が簡単に作成できます。

  • フォントの使用サイズを変更
スケーラブルフォントサイズ = 40 スケーラブルフォントサイズ = 80

  • 縁取りの太さを変更
縁取りサイズ = 4 縁取りサイズ = 8
