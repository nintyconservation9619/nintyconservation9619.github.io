Windows 上で動作する VI

概要

VI ライブラリの最大の目標は、Nintendo プラットフォームを対象としてディスプレイの機能を使用する際に発生する数々の相違を最小限とすることにあります。このような相違について以下で詳しく説明します。

あらゆる汎用仕様向けおよび NX 仕様向けに VI ライブラリを使用できます。Windows と Horizon のオペレーティングシステムはいずれも全面的にサポートされています。

Windows 上で動作する NX

Windows および Horizon のどちらでも、NX アプリケーションをコンパイルする際は、動作上の以下の相違点に留意する必要があります。

  • 関数の中には必ず成功して終了するものがありますが、それらは何の効果も残していないように見えます。このような場合は、前提条件の確認のみが実行されています。Windows では、次の関数が全面的にはサポートされていません。
    • nn::vi::SetLayerScalingMode()
  • nn::vi::GetDisplayVsyncEvent()
    • NX は、リフレッシュレート 60 Hz で特定の解像度のみをサポートしますが、Windows の vsync イベントはモニターのリフレッシュレートに従ってシグナルを送信します。60 Hz シグナルはエミュレートしません。
    • プライマリモニターの vsync のみがイベントシグナルの送信に使用されます。Window がセカンダリモニターに移った場合、シグナルの時間は調整されません。プライマリモニターとセカンダリモニターのリフレッシュレートが異なる場合、vsync シグナルとテクスチャの取得時間の同期が取れなくなる可能性があります。
