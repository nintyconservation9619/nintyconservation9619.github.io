プログラマ向け変更点

NintendoSDK 0.14.0 で行われた NintendoWare オーディオ (nn::atk) の変更点について説明します。

目次

  • 複数の Opus ファイルを同時に再生する機能の追加
  • サウンドプロファイル機能の追加
  • SoundProjectConverter に出力先を指定するコマンドラインオプションを追加
  • サウンドアーカイブのバージョンを更新

複数の Opus ファイルを同時に再生する機能の追加

Opus ファイルを利用するサウンドの再生はこれまで同時に 1 つしか再生することができませんでしたが、
複数再生することができるようになりました。詳しくは AtkOpus サンプル や以下の API リファレンスをご確認ください。

  • nn::atk::InitializeOpusDecoder
  • nn::atk::GetRequiredOpusDecoderBufferSize

サウンドプロファイル機能の追加

オーディオフレーム毎のサウンドスレッドの CPU 処理負荷や、内部で使用する nn::audio のオーディオレンダラの処理負荷、再生中のボイスの数 などを取得するためのサウンドプロファイルの機能を追加しました。
 詳しくは  AtkProfile サンプル  やプログラミングガイドの 処理負荷の計測、以下の API リファレンスをご確認ください。 

  • nn::atk::SoundProfile
  • nn::atk::ProfileReader 

SoundProjectConverter に出力先を指定するコマンドラインオプションを追加

SoundProjectConverter にコマンドラインオプション「-o, --output」を指定することができるようになりました。
サウンドプロジェクト (fspj) の設定よりも、このオプションで指定した値が優先されます。

詳しくは、SoundProjectConverter を参照ください。

サウンドアーカイブのバージョンを更新

評価中の機能のために、バイナリサウンドアーカイブファイル (bfsar) のバージョンが 0.2.3.0 に更新されました。
今回の更新においては、互換性が維持されているため、既存機能に影響はなく再コンバートする必要はありません。
