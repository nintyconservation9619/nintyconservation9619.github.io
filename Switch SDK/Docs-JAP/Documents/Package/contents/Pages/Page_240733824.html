フィールドの不具合修正にともなうデータ修正について

EffectMaker で以下の設定でエミッタを回転させながらパーティクルを放出したさいの不具合を修正しました。

  • 「CPUで動作させる」
  • フィールド「位置に加算」を使用
    • 「加算値」に「パーティクル時間アニメ」を適用
    • 「ワールド座標系で適用」を ON

以下画像は修正前と修正後の比較画像です。
修正前は「位置に加算」の加算値がワールド座標系で適用されていませんでした。

修正前 修正後（正しい挙動）
エミッタが回転するとパーティクルの移動軌跡も連動して動いています。 エミッタが回転してもパーティクルの移動軌跡は変化しません。

補足：修正対象となる .eset の洗い出し方法

修正後の挙動が正しい挙動になります。これまでのデータで見え方が変わってしまう場合はお手数ですが修正をお願いいたします。

上記設定を持つ .eset を検索するための powershell スクリプトを掲載します。必要に応じてご利用ください。
コマンドプロンプトから実行することで、カレントディレクトリ以下の .eset をすべて走査し、条件に合致する .eset の絶対パスと、エミッタセット名、エミッタ名を列挙します。
このスクリプトは、 .eset の読み込みだけを行い、一切の書き込み・修正は行いません。
以下は powershell スクリプトの実行コマンドです。 

powershell -NoProfile -ExecutionPolicy unrestricted .\Grep-FieldAddCpu.ps1

#-------------------------------------------------------------------------------
# カレントディレクトリ以下の .eset を走査し、
# 条件を満たすエミッタセットの絶対パスと エミッタセット・エミッタ名を列挙するスクリプト
#-------------------------------------------------------------------------------

Function Eval-Condition( $Emitter )
{
    #-------------------------------------------------------------------------------
    # - 「CPU で動作させる」がON
    # - フィールド「位置に加算」が存在
    #   - 「加算値」にパーティクル時間アニメを適用
    #   - 「ワールド座標系で適用」を ON
    #-------------------------------------------------------------------------------
    if( $Emitter.EmitterBasicSettingData.EmitterBasicBasicData.ProcessType -eq "0" )
    {
        foreach ( $Field in $Emitter.FieldList )
        {
            if( $Field.AddLocationData -ne $null )
            {
                if( ( $Field.AddLocationData.FieldAddLocationParam.AddGlobal -eq "True" ) -and
                    ( $Field.AddLocationData.FieldAddLocationParam.EnableAnimation -eq "1" )
                )
                {
                    return $TRUE
                }
            }
        }
    }
    return $FALSE
}

Write-Host("EmitterSet Grep")

$SearchPath = "*.eset" -f $ResultPath
$Esets = Get-ChildItem $SearchPath -Recurse

foreach ( $e in $Esets )
{
    $xml = [xml](Get-Content -Encoding UTF8 $e)
    foreach ( $Emitter in $xml.EmitterSetData.EmitterList.EmitterData )
    {
        # 親エミッタ
        if( Eval-Condition( $Emitter ) -ne $TRUE )
        {
            Write-Host("{0}: {1}" -f $e.FullName, $Emitter.Name)
        }
        
        # チャイルドエミッタ
        foreach ( $ChildEmitter in $Emitter.EmitterList.EmitterData )
        {
            if( Eval-Condition( $ChildEmitter ) -eq $TRUE )
            {
                Write-Host("{0}: {1}.{2}" -f $e.FullName, $Emitter.Name, $ChildEmitter.Name)
            }
        }
    }
}

ブレンドプリセットに「乗算済みα」を追加

「ブレンドプリセット」に新しいプリセット「乗算済みα」が追加されました！
「一般的なαブレンド」はアルファ値が小さくなる部分で色も一緒に暗くなってしまうのに対し、「乗算済みα」は意図した明るさになります。

以下はプリセット「一般的なαブレンド」と「乗算済みα」の比較画像です。
テクスチャは RGB と アルファで同じ画を使用しています。
カラーは、 RGB = 1 , 1 , 0.5 ～ 10 , 10 ,1 の範囲でアニメーションしています。
R , G チャンネル 10 のタイミングでは、グレア効果がかかることを期待しています。

A：一般的なαブレンド

テクスチャのアルファ値が小さくなる部分が想定より暗くなっています。
B：乗算済みα

テクスチャのアルファ値が小さくなる部分でも正しい明るさになっています。 
グレアの効果も正しくなります。

プレビューノードに「マニュアルエミッタ放出」追加

ゲームアプリケーションでエミッタの放出を制御できる機能「マニュアルエミッタ放出」がプレビューノードに追加されました！

以下は「マニュアルエミッタ放出」の利用イメージです。
「マニュアルエミッタ放出」は、ゲームで複数配置されたキャラクタの足跡を残すさいに、たった 1 つのエミッタで実現できる為、高速化に効果的です。
本機能を使用することで、ゲームへの組み込み前にデザイナ側でイメージや問題点を把握できます。

1 エミッタで 1 キャラクター分（2 つ）の足跡を配置 1 エミッタで3 キャラクター分（6 つ）の足跡を配置

カラー編集のUIにカラー成分の表示形式追加

「カラー」タブで選択しているカラーやアルファの値を調整する UI に、カラー成分の表示形式「sRGB」が追加されました！
例えば以下のように四角い岩が破裂するようなエフェクトで、「岩」をノンリニア空間の DCC ツールで作成、「破片」をリニア空間の EffectMaker で作成したとします。


この場合、以下のように DCCツールで設定したカラーを EffectMaker に持ち込むことができます。

ノンリニア空間の DCC ツールでモデルを作成 リニア空間のエフェクトのカラーを作成したモデルに合わせる

モデルの頂点カラーは RGB = 0.5 です。
このモデルとエフェクトの色を一致させるとします。

カラー成分「RGB」で RGB = 0.5 と入力します。
色空間が異なる為、明度が一致していません。

 カラー成分「sRGB」で RGB = 0.5 と入力します。
色空間を合わせた為、明度が一致しました。
