Makefile の使用方法

  • 新規プロジェクトを作成する

新規プロジェクトを作成する

  1. VSI の「ファイル > 新規作成 > プロジェクト」を選択し、「テンプレート > Visual C++ > 全般 > メイクファイルプロジェクト」を選択します。
  2. 「デバッグ構成の設定」画面で各項目を指定します。
    • NX で実行可能なバイナリのビルド方法については、独自のビルドシステムを構築する (NX 開発機環境) を参照してください。
    • 出力（デバッグ用）には、最終生成物である、nsp ファイルのパスを指定します。
    • ここで指定する値は、作成後もプロジェクトプロパティ > NMake から変更することが可能です。
  3. リリースビルドの設定をデバッグビルドの設定とは異なるものにしたい場合は、「デバッグ構成と同じ」のチェックを外したうえで編集します。
  4. 「構成マネージャ」から、「アクティブソリューションプラットフォーム > <新規作成...>」を選択します。
  5. 「新しいプラットフォームを入力または選択してください」のボックスから、NX32 または NX64 を選択します。
  6. 「設定のコピー元」は「x86」を選択し、OK を押します。
  7. 「プロジェクトプロパティ > VC++ ディレクトリ > ライブラリディレクトリ」に、ビルド時にリンクされる nnSdk.nso と nnrtld.nso の存在するディレクトリのパスを指定します。
    • この設定をせずとも実行は可能ですが、設定しなかった場合はデバッグ情報を検索することができず、デバッグすることができません。
  8. 一般的なプロジェクトと同じようにデバッグを開始することで、NMake > 全般 > 出力 で指定してあるファイルのデバッグ実行が開始されます。
