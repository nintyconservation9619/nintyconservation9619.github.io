色空間を指定する

ここではレイアウトの色空間をリニアで扱うかどうかを設定する手順について説明します。

リニア空間に設定した場合、レイアウトで扱うデザイン素材にガンマ補正を行います。
以下に素材毎のおすすめの補正方法について説明します。

テクスチャ

デザイナーは通常通りテクスチャを作りレイアウトエディタで利用します。
リニア色空間への変換準備は、バイナリコンバータで行います。
コンバータへのオプション指定はプログラマが行います。
オプション指定が行われると、コンバータは、リニア変換を行いながらテクスチャを参照するハードウェア機能(sRGB フェッチ)を使うテクスチャを出力します。
(ただし、ハードウェア制限から、RGBA8、BC1、BC2、BC3以外のフォーマットのテクスチャでは、上記の sRGB フェッチ機能がつかえないため、コンバータ内部で補正処理を行います。)
なお、コンバータは、テクスチャがインダイレクト用途ならガンマ補正をしません。
インダイレクト用途かどうかは、自動判別するのでデザイナーが意識する必要が無いようになっています。

この運用では、デザイナーがテクスチャごとにガンマ特性に関する設定を行う必要はありません。
レイアウトではテクスチャの用途が限定的なのでコンバータによる自動判別できるためです。

頂点カラー、カラー補間

2通りの方法があります。

  • デザイナーがいままでどおりデータを作りバイナリ変換時に一括変換する
  • 最初からリニア特性でデータ作成する

いままでどおりデータを作る場合は、コンバータでリニア変換するオプションを利用します。(コンバータへのオプション指定はプログラマが行います)
アニメーションで結果が厳密に一致しません(スロープの変換は行わないので)。

リニアで作る場合は、ツール環境設定でリニア色空間モードの指定をします（[メニュー]→[編集]→[ツール環境設定]）。
レイアウトウィンドウと、ビューアの表示がリニア色空間に変わります。
レイアウトエディタやビューアで見える色を基に素材制作をすればリニア環境での絵作りが可能です。

バイナリ変換時に一括変換する場合は、[色パラメータをリニア色空間で指定する] のチェックを外して運用します。プレビュー時にコンバータへのオプション指定がされ、リニア変換が行われます。

関連リンク 「リニア色空間モード」については こちら を参照してください。

RGB 値の数値入力

リニア色空間モードで運用する場合、非リニア環境(Photoshopなどのツール)からスポイトツールで色を拾って、LayoutEditor に RGB数値を設定すると表示される色が食い違ってしまいます。
この場合はカラーピッカーにある以下の様なガンマ補正ボタンもあわせて利用してください。

数値入力してからガンマ変換ボタンを押すと、リニア環境に合わせたRGB値が簡単に設定できます。

関連リンク 「カラーピッカー」についてはこちらを参照してください。

フォントについて

形式によってそれぞれ別の対処を取ります。

  • 1チャンネル形式 ... 対処無し。 画素はアルファ値として扱うのでガンマ特性の影響を受けない。
  • 2チャンネル形式 ... 必要に応じて、フォントコンバータに入力する画像ファイルにガンマ補正をかけて対処。
  • フルカラー ... _SRGB と付いたリニア色空間用フォーマット(1.10.0 より追加されました)を指定してフォントを作る。
    従来は、フォントコンバータに入力する画像ファイルにガンマ補正をかけて対処していましたが、補正計算時に精度低下が起こるためリニア色空間用フォーマットの利用をお勧めします。

ビューアでの確認について

ツール環境設定でリニアモードと、sRGBモード(従来とおなじ動作)に変更が可能です。
プレビュー時のバイナリコンバートの設定も連動して変わります。
