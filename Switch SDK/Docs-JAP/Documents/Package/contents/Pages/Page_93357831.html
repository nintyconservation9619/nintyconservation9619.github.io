CPUで動作させる

ここではパーティクルの挙動計算方法を切り替える機能「CPUで動作させる」について説明します。

概要

パーティクルの挙動計算を GPU から　CPU に切り替える機能です。エミッタが使用している機能により利用できます。
以下は「 設定項目」の概要です。

CPUで動作させる

パーティクル挙動の計算方法を「 チェックボックス」ON/OFF で切り替えます。
カスタムアクションやストライプが追加されている場合は、強制的に　「CPUで動作させる」が ON になります。

ON

エミッタタイプは CPU で動作し、パーティクル演算はCPUで行われます。
OFF

エミッタタイプは設定内容により GPU もしくはGPU + SO で動作し、パーティクル演算はGPUで行われます。

以下はワークスペース上でのエミッタノードの表示と特徴についてまとめた表です。

CPU

パーティクルの位置や速度を CPU で処理します。
上記の様なパーティクル情報をゲームアプリ側から取得する場合や、ゲーム内情報でコントロールする場合に利用します。
パーティクル数が大量になる場合はCPU処理を大量に消費する為、「CPUで動作させる」を OFF にすることをおすすめします。
GPU パーティクルの位置や速度を GPU で処理します。
大量のパーティクルを放出してもCPU処理を消費しません。エミッタ処理ではCPU処理を消費します。
GPU+SO（GPU + StreamOut）

パーティクルの位置や速度を GPU で処理し、処理した結果をメモリに書き出し次のフレームで利用します。
 EffectMaker では「CPUで動作させる」が OFF の時、利用する機能により、自動で切り替わります。
大量のパーティクルを放出してもCPU処理を消費しませんが、書き出し先のワークメモリは大量に消費します。
