ライブラリファイルの提供形態

Windows 環境

  • ライブラリファイルは基本的に機能ごとの静的ライブラリファイル (.lib) として提供されます。
  • アプリケーションは、必要とする各ライブラリファイルを静的リンクして利用します。

実機環境

  • ライブラリファイルは基本的に SDK DLL として提供され、アプリケーションに自動的に動的リンクされます。
  • ただし、ライブラリ仕様ならび利用目的に応じて SDK DLL に含まれず、アプリケーションが明示的にリンクして使用する必要のあるライブラリが存在します。

ライブラリファイルの指定方法

プロジェクトで指定できるパラメータ、ビルドシステムのマニュアル のページを参照してください。

NintendoSDK で提供される各ライブラリを利用するために必要なライブラリファイルならび指定方法については、以下を参照してください。
