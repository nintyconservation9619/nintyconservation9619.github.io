プロジェクト

概要

プロジェクトは 3DEditor が起動中に、ひとつ存在して 3DEditor 作業状況を管理します。

プロジェクトは以下の情報を保持します。

  • 3DEditor 上に開かれているファイルのパス
  • アニメーションのバインド情報
  • アニメーションセットの構成とバインド情報
  • モデルのプレビュー用の情報
  • スケルタルアニメーションのリターゲット先モデル名
  • プロジェクトプロパティウィンドウに表示する編集用カラーと編集用コメント
参考：

3DEditor は起動時に Project という名前の空プロジェクトを作成します。

プロジェクトの保存

プロジェクトは、プロジェクトファイル ( .f3pj ) に保存します。
拡張子を除いたものがプロジェクト名です。

プロジェクトを保存する場合は、3DEditor 上で開かれている全てのファイルパスを確定させます。

プロジェクトファイルに保存されるファイルパス

  • ファイルが絶対パスで設定された追加参照パス経由で開かれている場合は、追加参照パスからの相対パスが保存されます。
  • ファイルがプロジェクトファイルと同じドライブにあるファイルは、相対パスが保存されます。
    ※違うドライブにあるファイルは、絶対パスが保存されます。
参考：

保存時に特定のファイルをプロジェクトから除外することはできません。
Startup フォルダからスタートアップパスで開かれたファイルは、プロジェクトファイルの相対パスではなく、Startup からの相対パスとして保存されます。

プロジェクトファイルのフォーマットについては、こちら を参照してください。

プロジェクトを開く

プロジェクトファイルを開くと、3DEditor 上に開かれているファイルは全て閉じられ、プロジェクトファイルに記述されたファイルを開きます。

※モデルが参照しているテクスチャは、プロジェクトファイルに記述されていなくても検索して読み込みます。

参考：

同時に複数のプロジェクトファイルや、その他のファイルを指定して開くことはできません。
プロジェクトファイルを開く場合は、単体で指定します。

プロジェクトファイルを開いた際に記述されたファイルが見つからなかった場合は、メッセージダイアログを表示します。
※見つからなかったファイルは、プロジェクトファイルから除外します。

複数の作業環境でプロジェクトファイルを共有する場合は、ファイルの相対的な配置や追加参照パスの設定と、追加参照パスの設定に含まれる環境変数の設定を適切に行う必要があります。

プロジェクトの新規作成

プロジェクトを新規作成する際は、既に開かれているプロジェクト以外のファイルを閉じるかの選択肢があります。

  • 閉じるを選択すると、空の新規プロジェクトが作成されます。
  • 閉じないを選択すると、ファイルやアニメーションとアニメーションセットのバインド情報が新規プロジェクトに引き継がれます。
