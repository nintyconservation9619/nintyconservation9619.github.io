関数

ここには、プロファイラの GUI に表示される特別な関数のリストが示されています。関数がモジュールから読み取られたものではなく、GUI の内部で作成されたものである場合、その関数は特別なものと見なされます。

UNRESOLVED（未解決）

関数が UNRESOLVED（未解決）とマーク付けされている場合、その原因はプロファイラで関数のアドレスをモジュール（NSS ファイルまたは NRS ファイルのいずれか）にマップできないことです。コールスタックプロファイルでは、関数が UNRESOLVED（未解決）の場合、通常はコールスタックフレームが有効でないか、フレームデータが破損していることが原因です。リーフのみプロファイルでは、通常こうした関数は発生しません。

GUI では、0x000000000 がこの関数の固定アドレスとなります。

<モジュール>_internal (unresolved)

この形式の関数（例えば、"nnSdk.nss_internal (unresolved)"）は、特定のモジュールに属すものの、プロファイラが関数シンボルを見つけることができなかった関数を表しています。これは、nnSdk.nss などの SDK 提供のモジュールでよく起こります。通常、これらのモジュールでは内部関数シンボルが削除されるからです。

System Idle Thread（システムアイドルスレッド）

関数が System Idle Thread（システムアイドルスレッド）とマーク付けされている場合は、プロファイラがサンプルを取得したときに、その時点で実行されていたスレッドがシステムレベルのアイドルスレッドであったことを意味しています。

GUI では、0x000000001 がこの関数の固定アドレスとなります。

SYSTEM THREAD（システムスレッド）

関数が SYSTEM THREAD（システムスレッド）とマーク付けされている場合は、サンプリングが発生したときに、System Idle Thread（システムアイドルスレッド）以外の何らかのシステムレベルのスレッドが実行されていたことを意味します。これは、以下に示す特別関数 "SYSTEM: <関数>" と似ていますが、プロファイラがシステムコールバックを特定の関数またはスレッドにマップできていません。

GUI では、0x000000004 がこの関数の固定アドレスとなります。

SYSTEM: <関数>

文字列 "SYSTEM:" で始まる関数は、システムコールを実行中の関数です。コールスタックプロファイルを取得する場合は、「コールツリー」タブを使用して、コールスタックでどのようなシステムコールが実行されているかを確認することができます。

Other Process（他のプロセス）

関数が Other Process（他のプロセス）とマーク付けされている場合は、コアで何らかの別のプロセスが実行されており、プロファイラでサンプルを取得したときに、その別のプロセスがまさに実行されていたことを意味しています。標準的なアプリケーションでは、こうした関数は通常発生しません。

GUI では、0x000000003 がこの関数の固定アドレスとなります。

BAD FRAME POINTER（問題フレームポインタ）

関数が BAD FRAME POINTER（問題フレームポインタ）とマーク付けされている場合は、コーススタックフレームをウォークしているときに、現在のスタックの外部にジャンプしたフレームポインタが存在していたことを意味します。通常、その原因は、コールスタックフレームが有効でないか、フレームデータが破損していることです。こうした関数は、コールスタックプロファイルでのみ発生します。コールツリーでこのノードの下位にある関数は、リーフ関数を除いてすべて無効になっている可能性があります。

GUI では、0x000000005 がこの関数の固定アドレスとなります。

SENTINEL SAVE（センチネル保存）

関数が SENTINEL SAVE（センチネル保存）とマーク付けされている場合、その原因はコールスタックフレームをウォークしているときに、現在のバッファのスペースが不足していたことです。こうした関数は、まれにコールスタックプロファイルでのみ発生します。コールツリーでこのノードの下位にある関数は、引き続き有効です。

GUI では、0x000000006 がこの関数の固定アドレスとなります。

UNKNOWN STACK DETAILS（不明なスタック詳細）

関数が UNKNOWN STACK DETAILS（不明なスタック詳細）とマーク付けされている場合は、PC でコールバックを読み取っているときに何らかの不明な問題が発生したことを意味しています。通常、こうした関数は発生しません。

GUI では、0x00000000 がこの関数の固定アドレスとなります。

スレッド

ここには、プロファイラの GUI に表示される特別なスレッドのリストが示されています。スレッドが、プロファイルされたアプリケーション内のアクティブなスレッドのいずれかではなく、GUI の内部で生成されたものである場合、そのスレッドは特別なものと見なされます。

System Idle Thread（システムアイドルスレッド）

System Idle Thread（システムアイドルスレッド）は、コア上の他のスレッドをスケジュールできないときに実行されるシステムレベルのスレッドです。

GUI では、0x000000000 がこの関数の固定アドレスとなります。

SYSTEM THREAD（システムスレッド）

SYSTEM THREAD（システムスレッド）とマーク付けされているスレッドは、System Idle Thread（システムアイドルスレッド）ではなかったシステムレベルのスレッドです。

GUI では、0x000000001 がこの関数の固定アドレスとなります。

Other Process（他のプロセス）

コア上で他のプロセスがスレッドを実行している場合、そのスレッドがサンプルされている可能性があります。他のプロセスが何であるかをプロファイラが判断することはできないため、他のプロセスはすべて、コアごとに単一のグループとして Other Process（他のプロセス）に含められます。標準的なアプリケーションでは、こうしたスレッドは通常発生しません。

GUI では、0x000000002 がこの関数の固定アドレスとなります。
