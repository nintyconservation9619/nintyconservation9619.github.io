アプリケーションイメージに含めるアイコンについて

アプリケーションイメージ作成に必要なアイコン

アプリケーションイメージを作成するためには下記の２種類のアイコン画像が必要です。

  • スマートデバイスで使用するための大きなサイズのアイコン画像
  • NX で使用するための小さなサイズのアイコン画像

このうち、AuthoringTool に入力する必要のある画像は前者の大きなサイズのアイコンです。

後者の小さなサイズのアイコン画像は AuthoringTool が自動生成してアプリケーションイメージに組み込みます。
自動生成結果に問題がある場合のみ、後述のオプションで手動で入力することが可能です。

AuthoringTool に入力するアイコン画像の仕様

AuthoringTool で扱うことのできるアイコンの仕様は下記に沿っている必要があります。

種類 形式 画像サイズ ファイルサイズ
大きなサイズのアイコン 24bit BMP 1024 × 1024 -
小さなサイズのアイコン（通常は指定しない） ベースライン Jpeg 256 × 256 100 KB 以下

アイコン画像のフォーマットや内容の詳細は「アプリアイコン制作マニュアル」を参照してください。

アイコンの指定方法

nmeta ファイルの Application 要素内に Icon 要素を定義して指定します。

nmeta ファイル全体の構成についてはこちらを参照してください。

 

要素名 記述する内容 備考
Language

下記の言語名のいずれかを指定します。

  • AmericanEnglish
  • BritishEnglish
  • Japanese
  • French
  • German
  • LatinAmericanSpanish
  • Spanish
  • Italian
  • Dutch
  • CanadianFrench
  • Portuguese
  • Russian
 必須。
IconPath

大きなサイズのアイコンへのパスを指定します。

相対パスを指定した場合、nmeta ファイルからの相対パスとなります。

 必須。
NxIconPath

小さなサイズのアイコンへのパスを指定します。

相対パスを指定した場合、nmeta ファイルからの相対パスとなります。

基本的には指定しないでください。


自動生成されたアイコンのクオリティに問題がある場合でのみ使用するようにしてください。

 

具体的には下記のように記述して指定します。

 

<Application>
  <Icon>
    <Language>Japanese</Language>
    <IconPath>Icon_Japanese.bmp</IconPath>
  </Icon>
  <Icon>
    <Language>AmericanEnglish</Language>
    <IconPath>Icon_AmericanEnglish.bmp</IconPath>
    <NxIconPath>Icon_AmericanEnglish_small.jpg</NxIconPath>
  </Icon>
</Application>

 

大きなサイズのアイコンについて

<IconPath> で指定された大きなサイズのアイコン画像は下記の仕様の Jpeg に変換され、アプリケーションイメージに格納されます。

  • Exif の Maker タグに "Nintendo co., ltd"、Software タグに "Nintendo AuthoringTool" が設定される。
  • JPEG の品質は 98。
  • ベースライン形式。

 

自動生成される小さなサイズのアイコンについて

大きなサイズのアイコンから小さなアイコンが自動生成されアプリケーションイメージに格納されます。

自動生成されるアイコンは下記の仕様です。

  • 大きなサイズのアイコンを .NET Framework の Graphics クラスを使って縮小したもの（補完モードは HighQualityBicubic ）
  • Exif の Maker タグに "Nintendo co., ltd"、Software タグに "Nintendo AuthoringTool" が設定される。
  • JPEG の品質は ファイルサイズが 100KB を超えない範囲でなるべく高くなるように調整される。
  • ベースライン形式。

 

手動入力した小さなサイズのアイコンについて

<NxIconPath> で指定された小さなサイズのアイコンは仕様に基づいたデータになっているかどうかが確認された後、Jpeg の Exif が下記のように編集されて、アプリケーションイメージに格納されます。

  • サムネイル画像が削除される。
  • Maker タグに "Nintendo co., ltd" が設定される。
