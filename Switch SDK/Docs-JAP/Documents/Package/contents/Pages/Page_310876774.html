概要

アプリケーションの起動に必要となる各種機能を提供します。

  • インストール、削除
  • リスト、起動、終了
  • ゲームカードへの書き込み
バックグラウンドアプリケーション無し バックグラウンドアプリケーション有り

使い方

リスト

開発機本体にインストールされているアプリケーション一覧を表示します。

VersionやApplication Nameを表示するには、アプリケーションの.nmetaファイルにディスプレイバージョンやタイトル名を設定する必要があります。
設定方法は nmeta ファイルの書き方 を参照してください。

参考：

アプリケーション管理データはシステムにより本体保存メモリーにキャッシュされます。ゲームカード内のアプリケーション管理データではなくキャッシュが読み込まれるようになっており、HOMEメニューやDevMenuのランチャーでキャッシュが読み込まれないようにするには DevMenuCommand を実行する必要があります。詳しくは DevMenuCommand の「アプリケーション管理データのキャッシュの無効化」を参照してください。

インストール

microSDカードに保存したnspファイル（アプリケーション本体、追加コンテンツ、パッチ）をインストールできます。
「Install from SD」ボタンを押すとmicroSDカード内のnspファイル一覧画面に移ります。

インストール対象デバイス

インストール対象のデバイスを以下から選択します。

デバイス 処理
Auto (Default) microSDカードを優先してインストールを試みます。
microSDカードにインストールできない場合は、本体保存メモリーにインストールを試みます。
System Memory 本体保存メモリーにインストールを試みます。
SD Card microSDカードにインストールを試みます。


一括インストール

｢Install All｣ ボタンを押すと一覧に表示されているmicroSDカード内の全nspファイルを一括でインストールできます。
｢Start｣ ボタンを押すとインストールを開始します。インストール完了後はnspファイル一覧画面に戻ります。

個別インストール

インストールしたいnspファイルを選択して A ボタンを押すと確認ダイアログが表示されます。｢Start｣ ボタンを押すとインストールを開始します。
インストール完了後はnspファイル一覧画面に戻ります。

注意：

インストール中はDevMenuの操作ができません。

ゲームカードへの書き込み

参考：

NX Addon 4.4.1以降で使用可能です。


microSDカードに保存した nspファイル（アプリケーション本体、パッチ）をインストールできます。
「Write to Game Card」ボタンを押すとゲームカードに書き込むmicroSDカード内のnspファイルを選択する画面に移ります。

起動、終了

Aボタン押下でアプリケーションを起動します。アプリケーション起動後、DevMenuは終了することなくバックグラウンドに常駐します。
アプリケーション内でHOMEボタンを押下すると DevMenuに制御が戻ります。

バックグラウンドでアプリケーションが動作している場合は、DevMenuにて以下の操作が可能になります。アプリケーションの状態は、画面下部の表示有無にて確認できます。

操作 実行される処理
HOMEボタン押下 アプリケーションに制御を戻す

Xボタン押下

 アプリケーションを終了する (Applicationページ以外でも機能します)
L, R, Xボタン同時押下

アプリケーションを強制終了する (Applicationページ以外でも機能します)

参考：

NX Addon 5.3.0以降で使用可能です。
別アプリケーションを起動
（Aボタン押下→Launch）		 起動中のアプリケーションを終了して、選択したアプリケーションを起動する
注意：

アプリケーションをインストールする前に追加コンテンツやパッチをインストールした場合、起動しようとしても失敗します。

起動パラメータのインストール、アンインストール

microSDカードにインストールしたファイルを、アプリケーションで起動パラメータとして取得できます。
アプリケーション起動パラメータについては、起動パラメータ取得 もあわせて参照してください。

DevMenuCommandを利用する方法

DevMenuCommand の「アプリケーション起動パラメータのインストール、アンインストール」を参照してください。

PCを利用する方法

  1. microSDカードをPCで開いて、 "NintendoSDK\ApplicationLaunchParameters"ディレクトリを作成してください
  2. パラメータを格納したファイルを以下の形式の名前で作成し、1.で作成したディレクトリに保存してください
    • アプリケーションには、ファイルのデータがそのまま渡されます。
<アプリケーションID>.nxargs

nxargsはファイルの拡張子です。アプリケーションIDは、半角大文字英数の16進数16桁で指定します。

01004B9000490000.nxargs
注意：

インストールしたパラメータは、Initial Boot MenuにDevMenuを選択しているときのみ取得できます。
HOMEメニューを選択しているときは、パラメータがインストールされていても、アプリケーションに渡されないことに注意してください。

メニューを切り替える方法は、本ページの「Initial Boot Menu」を参照してください。

注意：

起動パラメータの最大サイズは、4096バイトです。

詳細の表示

-/+ボタン押下でアプリケーションの詳細を表示します。

Launch Application

アプリケーションを起動します。

Manage

アプリケーションに関係するコンテンツ（アプリケーション本体、追加コンテンツ、パッチ）の削除や必須バージョンのリセットを行う機能を提供します。
詳しくは「コンテンツ管理」を参照してください。

Metadata Information

.nmetaファイルで設定した情報が表示されます。表示要素は今後追加される予定です。

Application Information

アプリケーション本体、追加コンテンツ、パッチの合計サイズを表示します。

コンテンツ管理

以下の機能を提供します。

  • 必須バージョンのリセット
  • アプリケーション本体、追加コンテンツ、パッチの削除
  • ROM識別ハッシュの表示
    • 詳細は ビルド成果物の識別について を参考にしてください

Reset RequiredVersion

アプリケーションの必須バージョンをリセットします。

Delete

選択したコンテンツを削除します。以下のケースでは、コンテンツは削除されずにシステムの管理データのみが削除されます。

  • ゲームカードを挿入後に抜いた
  • microSDカードにコンテンツをインストール後に抜いた
参考：

ゲームカードに書き込まれたコンテンツの個別削除はできません。

Delete All

リストされている全コンテンツを削除します。

参考：

ゲームカードを対象に削除を実行しても、書き込んだデータはイレースされません。システムの管理データのみが削除されます。

Show Hash

選択したコンテンツのROM IDハッシュを表示します。

自動起動アプリケーションの設定

exhibition modeが有効な場合、開発機の電源ON後にDevMenuの画面を表示せずに予め設定したアプリケーションを自動起動させることが可能です。
exhibition modeの有効／無効の切替えは「開発支援機能の設定 - Exhibition Mode」を参照してください。DevMenuCommand で切替えることも可能です。

自動起動対象のアプリケーションは、アプリケーションリスト上でY ボタンを押下することで設定できます。
自動起動対象のアプリケーションを変更したい場合は、開発機の電源ON時にNintendo Switchロゴが表示されて暫く経過後からアプリケーションが起動されるまでの間に、以下のコマンドを入力してください。

コントローラー 入力コマンド
Joy-Con ZLボタン, Rボタン, 方向ボタン下を同時入力
クラシックコントローラー PRO ZLボタン, Rボタン, 十字ボタン下を同時入力

PROコントローラー

exhibition modeでのアプリケーション自動起動では、以下の点が通常起動時と異なります。

  • 起動時にアカウント選択を要求するアプリケーションでは、accountライブラリで一番先頭にリストされたアカウントが自動選択される
  • 起動後にHOMEボタンを押下してもDevMenuに切り替わらない
  • 開発用オーバーレイ表示が一部しか実行されなくなる
  • アプリケーションが終了すると開発機が再起動される

ストレージの容量

デバイスに内蔵されているNANDと挿入しているmicroSDカードの容量を表示します。
microSDカードの容量を表示するには、カードを挿した状態で本体の電源を入れる必要があります。
