ニュースVer0.7.0

NintendoSDK 0.7.0 での NintendoWare 全体にかかわるニュースをお知らせします。

 

TargetManager を利用したHost / Target 間通信への切り替え

NintendoSDK 0.7.0 リリースにて HostPC と 各種ビューアやアプリ 間にて TargetManagerと呼ばれるWindowsツールを介して通信するようになりました。この変更により、3DEditor / LayoutEditor / EffectMaker などのWindows ツールを利用して 各種ビューアやアプリと通信する際に TargetManager を介して通信するようになりました。

警告：

TargetManager を利用してWindows ツール、ビューア 間通信を行うにはこれらの起動より先に TargetManager を起動し、ビューア、ツールの順に起動する必要があります。
NintendoSDK パッケージをNDI経由でインストールした場合、TargetManager はスタートアップに登録されます。

参考：

以下に記す ツール / ランタイム間 の接続については、NintendoSDK 0.7.0 では winsock を利用します。

  • SoundMaker と nn::atk 間の接続。　（次の SDK バージョンにて TargetManager 経由の接続に対応する予定です。）
  • SoundMaker と SoundPlayer / nw::snd 間の接続。
  • EffectMaker と EffectViewerNw4f/EffectViewer.exe / nw::eft2 間の接続。
  • LayoutEditor と LayoutViewer-Nw4f/Win32/lytviewer.exe / nw::lyt 間の接続。

 

nn::ige ライブラリの追加

TargetManager 経由のHost / Target 間通信は、 nn::ige ライブラリを利用して行われます。よって、各モジュールのビューアライブラリを利用してアプリ内ビューア等を実装する場合は、nn::ige ライブラリをリンクしてアプリケーションをビルドする必要があります。

  • nn::g3d::viewer
  • nn::ui2d::viewer
  • nn::vfx::viewer
  • nn::atk::viewer （次のバージョンで対応予定）
