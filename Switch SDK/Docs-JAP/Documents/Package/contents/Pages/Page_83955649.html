コンバート前後の自動処理

コンバート前後に起動するバッチファイルの設定

コンバートの前と後に、指定したバッチファイル（拡張子 .bat）を起動させることができます。

コンバートで出来上がったバイナリファイルを、常に別の場所にコピーしたい場合や、名前の変更を行いたい場合などに利用することができます。

このセクションでは、コンバート前後にバッチファイルを起動する手順を解説します。

起動するバッチファイルの設定

起動するバッチファイルの設定は、プロジェクト設定ダイアログで行います。

[プロジェクト]→[プロジェクトの設定...] でプロジェクト設定ダイアログを開きます。

プロジェクト設定ダイアログでコンバートイベントのタブを有効にし、ここで「コンバート前コマンド」と「コンバート後コマンド」に、各々コンバート前と後に実行したいバッチファイルを記入してください。
パスは、絶対パス・相対パスのどちらでも構いません。相対パスで指定した場合は、プロジェクトファイルのあるディレクトリからの相対パスとして解釈されます。

それぞれのチェックボックスを OFF にすることにより、一時的にコマンドを無効化することができます。
次回のコンバートから、チェックボックスが ON のコマンドについて、指定したバッチファイルが実行されます。

参考：

コンバート前コマンドおよびコンバート後コマンドでは、実行時に入力を求めるような対話的な入力には対応していませんので、ご注意ください。

 

出力ファイルパスリストの参照

コンバート後コマンドに環境変数 %OUTPUT_LIST_FILE_PATH% を指定することで、出力ファイルの一覧が書かれたファイル（出力ファイルパスリスト）への絶対パスを埋め込むことができます。

このファイルには、以下のコンバータが出力したファイルへの絶対パスが 1 行ずつ記載されます。

  • *.bfsar
  • *.bfstm
  • *.bfgrp
  • *.fsid
  • *.xml (マップファイル)
  • *.html (マップファイル、HTML サウンドリストファイル)

この環境変数は、コンバート後コマンド限定で、コンバート前コマンドには使用できません。
