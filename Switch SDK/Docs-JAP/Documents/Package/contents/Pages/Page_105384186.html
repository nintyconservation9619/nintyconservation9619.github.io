並列処理

大量の処理を分割して並列に処理することで、終了までの時間を短縮し、CPUリソースを最適に利用できます。

エフェクトランタイムでの並列処理について

エフェクトランタイムの並列処理には、３つのケースが考えられます。

  • ユーザーのエフェクトの操作を並列で行う
  • エフェクトの計算処理を並列で行う
  • エフェクトの描画処理を並列で行う

現状のエフェクトランタイムは以下のような対応状況になっています。

  具体的な操作 対応状況 nn::vfx::Config での設定が必要か？
エフェクトの操作の並列処理 エミッタセットの生成や破棄、各エミッタセットに対しての操作 対応済 不必要
エフェクト計算処理の並列処理 nn::vfx::System::BeginFrame / Calculate メソッドの呼び出し 対応予定 -
エフェクト描画処理の並列処理

nn::vfx::System::Draw / AddSortBuffer / DrawSortBuffer メソッドの呼び出し、

また、nn::vfx::System::SetViewParam など描画設定メソッドの呼び出し

 対応済 必要

 

ユーザー操作の並列処理

ユーザーが行うエミッタセットの作成や破棄、生成済みエミッタセットのへの操作は、特に制限なく並列な処理から呼び出すことが可能です。ただし、エミッタセットの生成や破棄のタイミングでエフェクトランタイムが内部で保有するリソースの確保の際に排他処理を行う必要があるため、内部でMutex による Lock 処理を行います。

 

エフェクト描画処理の並列処理

複数のコア、スレッドから描画コマンドを発行することができます。描画処理を並列で行う場合、nn:vfx::Config::SetProcessingCount にて並列処理させたい数を指定し、nn::vfx::System::Draw / AddSortBuffer / DrawSortBuffer メソッド呼び出し時に、引数にてどの並列処理インデックスで処理するのか指定する必要があります。

描画処理を並列で行う場合、書き込み先のコマンドバッファはそれぞれで異なるコマンドバッファを指定する必要があります。

// エフェクトシステムの初期化
nn::vfx::Config config;
config.SetProcessingCount( 2 );    // 並列数 2 を指定する
pEffectSystem = new nn::vfx::System( config );

// エフェクトの並列描画処理関数
void VfxMultiDrawProcess( int processIndex, int groupId )
{
    mVfxSys->SetViewParam( processIndex, &viewParam );
    mVfxSys->Draw( processIndex, groupId );
}

// スレッド0 からエフェクトの並列描画処理関数を呼び出す
VfxMultiDrawProcess( 0, gourp_0 );

// スレッド1 からエフェクトの並列描画処理関数を呼び出す
VfxMultiDrawProcess( 1, gourp_1 );
