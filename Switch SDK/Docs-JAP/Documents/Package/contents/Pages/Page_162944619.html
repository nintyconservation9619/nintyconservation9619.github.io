マイページアプレット

マイページアプレットは、ユーザーに関する情報や機能がまとめられているシステムアプリです。

その中に、フレンド一覧の確認やフレンドの追加といったフレンド関連の機能が含まれています。

マイページアプレットに含まれるフレンド機能

マイページアプレットが提供するフレンド関連の機能と、それらを呼び出すためのAPIの一覧は下記の通りです。

利用にあたっての注意

通信に対する影響

マイページアプレットは起動中にインターネット通信及びローカル通信を利用する可能性があります。
マイページアプレットが利用する可能性のある通信の種類については「マイページアプレットに含まれるフレンド機能」の表を参照ください。

パフォーマンスに対する影響

ネットワークサービスアカウントに関する処理（アカウント連携のためのログイン処理など）のために内部的に Web ブラウザーを使用しており、アプリケーションが使用している CPU コアを本体システム側からも使用することがあります。
 そのため、アカウント管理アプレットの表示中にアプリケーションがバックグラウンドで処理を継続している場合、本体システムでの UI 表示などのパフォーマンスが低下することがあります。

ヘッダファイルのインクルード

プログラム中に下記のコードを記述してください。

#include <nn/friends.h>
または
#include <nn/friends/friends_Applet.h>

マイページアプレットの起動

マイページアプレットの各機能は対応した API を呼び出すことで利用できます。
マイページアプレットを起動するAPIはアプレットが終了するまで処理をブロックします。 

nn::Result result = nn::friends::ShowFriendList( uid );

API の処理結果について

マイページアプレットの呼び出し API の処理結果に応じて実行しなければならない処理はありません。
必要であれば処理結果に応じた任意の処理を実行することができますが、起動中のマイページアプレットが外的な要因によって終了した場合にも失敗を返すため、失敗時にもう一度マイページアプレットを呼び出す実装はしないでください。

呼び出し API が失敗を返す条件

マイページアプレットの呼び出し API は下記の場合に失敗を返します。

  • マイページアプレットが外的な要因によって終了した
  • nn::friends::StartSendingFriendRequest() の呼び出しによるフレンドリクエストの送信に失敗した

マイページアプレットが外的な要因によって終了する

マイページアプレットが外的な要因によって終了する例を下記に示します。

  • ユーザー操作によって、マイページアプレットを開いている最中に HOME メニューに遷移し、別のライブラリアプレットを開いた
  • ユーザー操作によって、マイページアプレットのプレイ記録から起動中のアプリケーションを選択して実行した
