コントロール定義ファイル

はじめに

コントロール定義ファイルは、ユーザが独自に定義したコントロールの設定をレイアウトエディタで行えるようにするための情報を含むファイルです。本ドキュメントでは、コントロール定義ファイルの作り方について説明します。

コントロール定義ファイルの形式

コントロール定義ファイルはxmlとして記述します。スキーマは、LayoutEditor/XMLSchemata/ver_1_5/flct.xsd にあります。

コントロール定義ファイルを置く場所について

コントロール定義ファイルは、部品ルートフォルダより下の階層のフォルダに置くことによって読み込むことができます。
複数ある場合も、全てのファイルが読み込まれて追加されます。ファイルの拡張子は .flct としてください。

通常、コントロール定義ファイルで定義したコントロールとデフォルトコントロールは両方有効になります。
ただ、アプリケーションで定義したコント ロールのみを使用し、デフォルトコントロールは使いたくないという場合もあると思います。そのような場合は、部品ルートフォルダに NW_DefaultControl.flct という名前のコントロール定義ファイルを置いてください。
このファイルがある場合、レイアウトエディタはデフォルトコントロールを無効とします。

コントロール定義ファイルの記述方法

記述例

コントロール定義ファイルの記述例が、 $NINTENDO_SDK_ROOT/SampleData/Layout/sample_NW_DefaultControl.flct にあります。内容は以下のようになっています。

<?xml version="1.0" encoding="utf-8"?>
<nw4f_layout xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
    xmlns:xsd="http://www.w3.org/2001/XMLSchema"
    version="1.5.0" xmlns="http://www.nintendo.co.jp/NW4F/LayoutEditor">
<head>
    <create user="" host="" date="2009-09-10T18:38:14.84375+09:00" source="" />
    <title />
    <comment />
    <generator name="NW4C LayoutEditor Core" version="2007_07_00_000" />
</head>
<body>
    <lct>
        <controlForm name="TestButton" uiName="テストボタン">
            <description>カスタムボタンテスト用ボタンです。</description>
            <parameterAnimation name="On" isRequired="true">
                <description>[必須]OffからOn状態に切り替わるアニメです</description>
            </parameterAnimation>
            <parameterAnimation name="Off" isRequired="true">
                <description>[必須]OnからOff状態に切り替わるアニメです</description>
            </parameterAnimation>
            <parameterPane name="Hit" isRequired="true">
                <description>[必須]ボタンの当たりを示すペインです</description>
            </parameterPane>
            <parameterPane name="Cursor" isRequired="false">
                <description>[任意]選択カーソル用ペインです</description>
            </parameterPane>
            <constraintAnimationEqual>
                <animation name="On" position="End" />
                <animation name="Off" position="End" />
            </constraintAnimationEqual>
        </controlForm>
    </lct>
</body>
</nw4f_layout>

各要素の説明

nw4f_layout/body/lct/controlForm

コントロールの定義を示す要素です。この要素を並べていくことで、複数のコントロールを定義できます。

この要素はnameとuiNameという二つの属性を持っています。
nameは、プログラムからこのコントロールを扱うときの名前です。半角英数字とハイフンのみ使用できます。ここで指定した文字列が、ControlSrcのGetControlNameメソッドで取得できます。これは必須の属性となります。
uiNameは、レイアウトエディタのユーザーインターフェース上で表示される名前です。こちらには、各国の言語等のデザイナがわかりやすい名前で記述できます。これは必須の属性ではありまん。省略された場合は、代わりにnameが使われます。

nw4f_layout/body/lct/controlForm/description

コントロールの説明を記述する要素です。
この説明は、ランタイムのプログラムでは使われません。レイアウトエディタの部品設定画面で表示されます。

nw4f_layout/body/lct/controlForm/parameterAnimation

コントロールの機能アニメについて記述します。この要素を並べることで、機能アニメを複数指定することができます。レイアウトエディタで指定されたアニメタグ名を、ControlSrcのGetFunctionalAnimNameメソッドで取得することができます。
この要素を並べた順番が、そのままGetFunctionalAnimNameに指定するインデックスになりますので、順番にはご注意ください。

この要素は、nameとisRequiredという二つの属性を持っています。
nameは、レイアウトエディタで表示される名前です。ランタイムのプログラムでは使われません。(インデックスで指定するため。)これは必須の属性となります。
isRequiredは、この機能ペインを必ず設定しなければいけないかどうかです。この属性がtrueになっていたとき、未設定だと、部品設定のウィンドウに警告が表示されます。省略した場合は、falseを設定している(必須ではない)と見なされます。

nw4f_layout/body/lct/controlForm/parameterPane

コントロールの機能ペインについて記述します。この要素を並べることで、機能ペインを複数指定することができます。レイアウトエディタで指定されたペイン名を、ControlSrcのGetFunctionalPaneNameメソッドで取得することができます。
この要素を並べた順番が、そのままGetFunctionalPaneNameに指定するインデックスになりますので、順番にはご注意ください。

この要素は、nameとisRequiredという二つの属性を持っています。
nameは、レイアウトエディタで表示される名前です。ランタイムのプログラムでは使われません。(インデックスで指定するため。)これは必須の属性となります。
isRequiredは、この機能アニメを必ず設定しなければいけないかどうかです。この属性がtrueになっていたとき、未設定だと、部品設定のウィンドウに警告が表示されます。省略した場合は、false設定している(必須ではない)と見なされます。

nw4f_layout/body/lct/controlForm/ConstraintAnimationEqual

機能アニメーションの制約条件を表す設定で、 子要素 animation で列挙されるアニメーションの結果が同一であるという制約を定義します。
animation 要素の、name 属性には機能アニメーション名、 position 属性には Start(開始フレーム) か End(終端フレーム) を設定します。
制約条件はシステム側では利用されていませんが、 将来的には LayoutEditor での警告表示などのサポート機能に利用する予定です。
