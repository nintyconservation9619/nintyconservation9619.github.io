テクスチャービュー

概要

テクスチャービューは、テクスチャーの一部を参照するための機能を提供します。
テクスチャービュー自体はテクスチャーメモリーを管理しません。それは C++ 言語における参照と似ています。

テクスチャーは直接的にコマンドバッファーにセットできない一方、テクスチャービューはコマンドバッファーにセットすることができます。セットする際は、対応するサンプラーとペアでセットします。

また、テクスチャービューは元のテクスチャーの次元やフォーマットに対して異なった次元やフォーマットでそれを解釈するための機能も提供します。ただし、異なった解釈をするためにはそれらの間に互換性がある必要があります。どの次元やフォーマットが互換性があるかについては対象の低レベルグラフィックス API のマニュアルを参照してください。

情報

テクスチャービューの情報は nn::gfx::TextureViewInfo として定義されています。

次元

元のテクスチャーをどのような次元で解釈するかを SetImageDimension により指定します。 
ここで設定する次元は元のテクスチャーの次元と互換性がある必要があります。
元のテクスチャーと同じ次元を指定した場合は互換性があることが保証されます。

キューブマップのテクスチャービューを作成するには、次元としてキューブマップ、サブリソース範囲の配列長として 6 を指定します。このとき、元のテクスチャーの次元が 2D であり、配列長が 6 以上である必要があります。
キューブマップ配列のテクスチャービューを作成するには、次元としてキューブマップ配列、サブリソース範囲の配列長として 6 * n を指定します。このとき、元のテクスチャーの次元が 2D であり、 配列長が 6 * n 以上である必要があります。

フォーマット

 元のテクスチャーをどのようなフォーマットで解釈するかを SetImageFormat により指定します。
ここで設定するフォーマットは元のテクスチャーのフォーマットと互換性がある必要があります。
元のテクスチャーと同じフォーマットを指定した場合は互換性があることが保証されます。

 テクスチャーポインター

参照する元のテクスチャーを SetTexturePtr 関数により指定します。
元となるテクスチャーは、GPU がアクセスする方法として nn::gfx::GpuAccess_Texture または nn::gfx::GpuAccess_Image を含んで初期化されている必要があります。

要素マッピング

テクスチャーサンプリングの際、赤要素、緑要素、青要素、アルファ要素にそれぞれどの要素を対応させるかを SetChannelMapping により指定します。
要素の他に、0 または 1 の固定値を選択することも可能です。

サブリソース範囲

 元のテクスチャーのどの部分を参照するかを EditSubresourceRange により編集します。
部分は連続的であり、ミップマップ範囲と配列範囲により表現されます。

深度ステンシルモード

深度ステンシルテクスチャの場合に、深度とステンシルのどちらをフェッチするのかを SetDepthStencilTextureMode 関数により指定します。

オブジェクト

テクスチャービューのオブジェクトは nn::gfx::TextureView として定義されています。

初期化と破棄

テクスチャービューの初期化と破棄はオブジェクトの規則に従います。

テクスチャービューのセット

デスクリプタープールへのセット

nn::gfx::DescriptorPool クラスの SetTextureView によりサンプリングテクスチャーとして、SetImage によりイメージロード・ストア用のイメージとして、デスクリプタープールにテクスチャービューがセットされます。
セットしたテクスチャービューのデスクリプタースロットは nn::gfx::DescriptorPool クラスの GetDescriptorSlot により取得できます。

コマンドバッファーへのセット

nn::gfx::CommandBuffer クラスの SetTextureAndSampler に、サンプラーとペアでセットすることで、サンプリングテクスチャーをセットするためのコマンドが追加されます。イメージロード・ストア用のイメージをセットするコマンドを追加するには nn::gfx::CommandBuffer クラスの SetImage を使います。デスクリプタープールを使う場合はデスクリプタースロットを、使わない場合はテクスチャービューを引数として渡します。
このときのシェーダーのスロット番号には、シェーダーオブジェクトから動的に取得した値をセットするようにしてください（GLSL シェーダーにおけるユニフォームブロックとサンプラーの初期 binding の上書きを参照）。
