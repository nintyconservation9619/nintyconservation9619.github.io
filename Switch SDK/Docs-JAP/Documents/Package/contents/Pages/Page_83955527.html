テクスチャーパターンアニメーション

テクスチャーパターンアニメーションは、file ノードのイメージ番号（Image Number）（フレーム拡張子（Frame Extension）アトリビュート）をアニメーションさせて設定します。

まず、file ノードを選択してアトリビュートエディタを開きイメージシーケンスの使用（Use Image Sequence）（フレーム拡張子の使用（Use Frame Extension）アトリビュート）を ON にします。

このとき、イメージ番号（フレーム拡張子アトリビュート）にデフォルトのエクスプレッションが自動的に接続されるので、キーを打つ場合はエクスプレッションを削除してください。

次に、イメージ番号（フレーム拡張子アトリビュート）にアニメーションを設定します。ドリブンキー、エクスプレッション、アニメーションレイヤも使用できます。
ただし、エクスプレッションやアニメーションレイヤを使用する場合および直接キーを設定しない場合（ユーティリティノードを経由する場合など）はテクスチャーパターンアニメーション設定プラグインで出力するテクスチャーのリストを設定する必要があります。

フレームオフセット（Frame Offset）アトリビュートは 0 にしておいてください。
次の場合はベイクしたデータからアニメーションカーブデータを自動作成して出力します。

  • 接線タイプがステップ（Stepped）でないキーが 1 つでもある
  • ドリブンキー、エクスプレッション、アニメーションレイヤを使用している
  • イメージ番号（フレーム拡張子アトリビュート）に直接キーを設定していない

テクスチャーパターンアニメーション用のテクスチャーファイルは、"name.1.tga", "name.2.tga", "name.3.tga", ... のように拡張子の前にフレーム拡張子に対応する数字を付けます。
"name.01.tga", "name.001.tga" のように 0 から始まる数字にも対応しています（0 から始まる数字は最大 4 桁まで）。"name.1.tga" と "name.01.tga" の両方が存在する場合は、0 がついていないほうのテクスチャーが出力されます。

デフォルトでは、出力されるテクスチャーは file ノードのイメージの名前（Image Name）アトリビュートで指定したものと、フレーム拡張子アトリビュートに打たれたキーに対応するものだけです。

たとえば、イメージの名前が "name.1.tga" のとき、開始フレームでフレーム拡張子が 1、終了フレームでフレーム拡張子が 4 の 2 つだけキーが打たれているならば、"name.1.tga" と "name.4.tga" だけが出力されます。
アニメーションカーブの接線タイプがリニアであっても "name.2.tga" と "name.3.tga" は出力されません。
アニメーションカーブのフレーム拡張子に対応するテクスチャーが出力されていない場合は、一番近いフレーム拡張子に対応するテクスチャーがパターンアニメーションデータとして使用されます。

参考：
テクスチャーパターンアニメーション設定プラグインを使用すると、強制的に出力するテクスチャーを設定できます。 テクスチャーパターンアニメーション設定プラグインで設定したテクスチャーは、イメージ番号（フレーム拡張子アトリビュート）にアニメーションが設定されていなくても出力されます。

たとえば、強制的に出力するフレーム拡張子の範囲を "1-4"（1 から 4）のように設定しておけば、開始フレームでフレーム拡張子 が 1、終了フレームでフレーム拡張子が 4 の 2 つのキーだけを打ち、アニメーションカーブの接線タイプをリニアにして出力することが可能です。
1 つのモデルに対して複数の fma ファイル（ftp ファイル）を使用する場合、テクスチャーパターンアニメーション設定プラグインで必要なテクスチャーがすべて出力されるように設定してください。

エクスポートオプションのマテリアル圧縮が有効になっていると、マテリアルがまとめられた結果、テクスチャーパターンアニメーションが設定されていないマテリアルまでアニメーションしてしまう場合があります。 その場合はマテリアルアトリビュート設定プラグインで、テクスチャーパターンアニメーションを設定したマテリアルが圧縮されないようにしてください。
