カスタムフィールド

ここではゲームアプリが独自に用意したフィールドの設定ができる「カスタムフィールド」について説明します。
ゲームアプリの機能やワークフローで利用できます。

ビット

「 ビット」は、カスタムフィールドをフラグで設定できます。
以下はグループ内設定項目の一覧です。

ビット [0] ～ [15] 16 個のフラグを「 チェックボックス」ON/OFF で設定します。

数値

「 数値」は、カスタムフィールドをスライダで設定できます。
以下はグループ内設定項目の一覧です。

数値0 ～ 数値15  0 ～ 255 の 256 段階の範囲で により値を設定します。

「カスタムフィールド」は GPU（コンピュートシェーダ） で処理される為、ノードを作成すると「 GPU+SO エミッタ」に切り替わります。
関連リンク「 GPU+SO エミッタ」についてはこちらを参照してください。

設定例

以下に、ゲームアプリケーションでカスタムフィールドとして「風」と「爆風」フィールドを実装したと仮定して説明します。
「風」フィールドは、ゲームシーン内の一定方向に力を加えたとき、パーティクルの移動速度にその影響を与えるカスタムフィールドです。
「爆風」フィールドは、爆発エフェクトの発生地点から放射状に力を加えたとき、パーティクルの移動速度にその影響を与えるカスタムフィールドです。

「木の葉」エフェクトの UI の 例 ゲームにおける使用イメージ
A

「風の影響を受ける」 = ON

イメージのように「木の葉」パーティクルは「風」フィールドの影響を受け、→ の方向に移動します。
B 「爆風の影響を受ける」 = OFF イメージのように「木の葉」パーティクルは「爆風」フィールドの影響を受けず、→ の方向に移動します。
C
 		「風」の影響力 = 0.8 「木の葉」パーティクルの速度に対し、「風」の速度を 0.8 の割合で加算します。
