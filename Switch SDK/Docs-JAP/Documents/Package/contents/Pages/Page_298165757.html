マルチアプリケーションカード

概要

マルチアプリケーションカードは複数のアプリケーションが書き込まれたNXカードです。

マルチアプリケーションカードはロットチェックに合格したアプリケーションを集めて1つのNXカードに入れて販売することを想定しています。

以下では、マルチアプリケーションカードの制約やNintendo Developer Portal(NDP)にマルチアプリケーションカード用のROMをアップロードするために必要なことを説明します。


NX開発カードにマルチアプリケーションカードに含めるアプリケーションを書き込み起動する手順についてはこちらを参照してください。

NX Addon 5.2.0 からマルチアプリケーションカードの書き込みや動作確認を行うことが出来ます。


注意：

2018/01/25 時点で、 ニンテンドーeショップは複数のアプリケーションをひとまとめにして販売する「バンドル販売」にまだ対応していません。
したがって、マルチアプリケーションカードと同じ様な商品をニンテンドーeショップで売ることがまだ出来ないのでご注意ください。


制約

マルチアプリケーションカードに含めることができるアプリケーションの数は4つです。

もし5つ以上のアプリケーションを含めたい場合は任天堂窓口までご相談ください。


NDPにROMをアップロードするために必要なこと

注意：

2018/01/25 時点では、NDPでマルチアプリケーションカードのプロダクトを作成することはできません。

以下はNDPでマルチアプリケーションカードのプロダクトが作成できるようになった場合の手順を説明します。


アップロード用nspファイルの作成

AuthoringTool を使って、NDPにアップロードするためのnspファイルを作成します。

このときに、オプションにマルチアプリケーションカードを識別するためのアプリケーションIDとマルチアプリケーションカードに含めるnspファイルが必要になります。

  • アプリケーションID
    • NDPでマルチアプリケーションカードのプロダクトを作成すると取得できるアプリケーションIDを指定してください。
    • 開発中で動作検証を目的とする場合には、0x0100000003488000などの試作用IDを使ってください。
      • 試作用のアプリケーションIDについてはこちらを参照してください。
      • 試作用のIDを使って作成したアップロード用nspファイルはNDPにアップロードできません。
  • マルチアプリケーションカードに含めるnspファイル
    • アプリケーションとパッチのnspファイルを指定することが出来ます。
      • 注意：追加コンテンツは含めることが出来ません。
      • NX Addon 5.2.0 以前の SDK で作ったアプリケーションやパッチも含めることが出来ます。
    • NDPにアップロードして、ロットチェックに合格したnspファイルを利用してください。
      • 開発中で動作確認を目的とする場合には、開発途中のnspファイルも利用できます。

1: マルチアプリケーションカードに含める nsp ファイルをテキストファイルに列挙する

UTF-8 BOM付きのテキストファイルを作成してください。

1 行ごとに含めるアプリケーションやパッチのnspファイルの絶対パスを書いてください。

アプリケーションの nsp の書かれた順番通りにHomeMenuやDevMenuに並びます。

パッチの nsp ファイルの順番は関係ありません。


C:\application0.nsp
C:\application1.nsp


2: AuthoringTool でアップロード用nspファイルを作成

以下のようなコマンドでアップロード用nspファイルを作成します。

$ AuthoringTool.exe bundleup --id 0x0100000003488000 --nsp-list-file C:\multi-application-card-file-list.txt


NX開発カードを用いたデバッグ

NXカード上で実行する場合、動作周波数がダウンロード版とは異なるのでアプリケーションの挙動に影響を与える可能性があります。

カードサイズと動作周波数の関係については、こちらを参照してください。

複数のアプリケーションが1つのNXカードに含まれることになるので、それぞれのアプリケーションが必要とするカードサイズや動作周波数と異なる可能性があります。

アップロード用nspファイルを作成するときに自動的に必要なカードサイズが計算されますが、後でサイズが変わることが予想される場合や動作周波数を指定したい場合など、自分でカードサイズや動作周波数を指定したい場合にはAuthoringTool のオプションで指定することが出来ます。


提出されたアップロード用nspファイルの設定を元に生産されるNXカードの容量と動作周波数が決定されます。

必ず適切な設定をしたうえで動作確認を行ってください。


マルチアプリケーションカードの動作確認の方法はこちらを参照してください。
