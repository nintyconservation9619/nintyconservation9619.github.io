ユーザーセーブデータへの使用に注意を要する情報

  • 概要
  • 疑似本体 ID (PseudoDeviceId)
    • PseudoDeviceId が異なる本体にセーブデータを移動して問題がないかを確認する
  • ユーザーの識別子 (nn::account::Uid)
    • nn::account::Uid が異なるユーザーアカウント間でセーブデータを複製して問題がないかを確認する

概要

次の情報は、特に必要のない場合にユーザーセーブデータに記録したり、ユーザーセーブデータのファイル名として使用しないでください。
アプリケーションの企画上これらの情報の使用が必要な場合に限り、以下の注意事項を厳守して使用してください。

  • 疑似本体 ID (PseudoDeviceId)
  • ユーザーの識別子 (nn::account::Uid)

疑似本体 ID (PseudoDeviceId)

ユーザーセーブデータに、 nn::oe::GetPseudoDeviceId() 関数で取得可能な疑似本体 ID (PseudoDeviceId) を使用する場合、次のことに注意してください。

アプリケーションの企画上必要でない場合、この情報を使用しないでください。
使用する場合、「ユーザーアカウントの移行」や「セーブデータのコピー」で、ユーザーセーブデータが別の本体に移動しても問題ないことを確認してください。

「ユーザーアカウントの移行」や「セーブデータのコピー」の機能を使用すると、 セーブデータはもともと作成された環境と異なる PseudoDeviceId をもつ本体に移動ないし複製されます。
従ってアプリケーションがセーブデータを参照した際に、想定しない PseudoDeviceId を検出する可能性があります。
アプリケーションは、このような場合でも動作に問題のないように作成される必要があります。

PseudoDeviceId が異なる本体にセーブデータを移動して問題がないかを確認する

PseudoDeviceId が異なる本体にセーブデータを移動した場合にアプリケーションが正しく動作するかは、次の方法で確認できます。 

  1. DevMenu を起動でき、ユーザーアカウントを作成済の本体を準備

  2. アプリケーションを起動し、ユーザーアカウントのセーブデータを蓄積し、アプリケーションを終了する
  3. DevMenu を起動し「Debug」画面を表示し、次を行う
    1. Reset Pseudo Device ID の Reset ボタンを押す
  4. アプリケーションを起動し、2 で作成したセーブデータでアプリケーションが問題なく動作することを確認


注意：

この方法によるテストは DEV モードで初期化した本体でのみ可能です。

ユーザーの識別子 (nn::account::Uid)

ユーザーセーブデータにユーザーの識別子 (nn::account::Uid) を使用する場合、次のことに注意してください。
(ユーザーセーブデータの所有者であるユーザーアカウントの Uid を使用するか、そうでないユーザーアカウントの Uid を使用するかは関係ありません。) 

アプリケーションの企画上必要でない場合、この情報を使用しないでください。
使用する場合、「セーブデータのコピー」で、ユーザーセーブデータの所有者を別のユーザーアカウントに変更しても問題ないことを確認してください。

「ユーザーセーブデータのコピー」では、ユーザーセーブデータは異なる nn::account::Uid をもつユーザーアカウントを跨いで複製されます。
従って複製されるユーザーセーブデータは、その両方のユーザーアカウントについて、正しく動作する必要があります。

参考：

アプリケーションの仕様上ユーザーセーブデータが nn::account::Uid に強く依存する必要があり、所有者を別のユーザーアカウントに変更した際の問題を避けられない場合、任天堂窓口にお問い合わせください。

nn::account::Uid が異なるユーザーアカウント間でセーブデータを複製して問題がないかを確認する

ユーザーセーブデータが別のユーザーアカウントに複製された場合にアプリケーションが正しく動作するかは、次の方法で確認できます。

  1. DevMenu を起動でき、ユーザーアカウント 2 つ (A, B) を作成済の本体を準備

  2. アプリケーションを起動し、ユーザーアカウント A のセーブデータを蓄積し、アプリケーションを終了する
  3. DevMenu を起動し「SaveData」画面を表示し、次を行う
    1. ユーザーアカウント A のセーブデータを選択し、 microSD カードにエクスポート
    2. 画面上部のドロップダウンリストから「Backup in SD card」を選択
    3. 3-a でエクスポートしたユーザーセーブデータを選択。このときインポート対象のユーザーアカウントとして B を選択
  4. ユーザーアカウント A を削除する
  5. アプリケーションを起動し、ユーザーアカウント B のセーブデータでアプリケーションが問題なく動作することを確認
