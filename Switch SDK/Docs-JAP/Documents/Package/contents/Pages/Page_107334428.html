座標系

gfx では、3D 座標系、クリップ座標系、ウインドウ座標系など、座標系について独自の定義や制御するための機能をもっておらず、低レベルグラフィックス API 依存としています。よって、gfx を使用している際は、どの低レベルグラフィックス API を使っているかによって座標系に関する挙動が変わるため、注意する必要があります。特に、異なるウィンドウ座標系の低レベルグラフィックス API 間では、レンダリングしたテクスチャーを描画しようとした際に上下が揃わずに反転することになります。

座標系は大きく分けて OpenGL 系と DirectX 系があります。各低レベルグラフィックス API がどちらの座標系を定義しているかはその低レベルグラフィックス API のドキュメントを参照してください。

OpenGL 系の座標系を採用している低レベルグラフィックス API の多くは、いくつかの座標系を DirectX 系の定義に変更するための API をもっています。 たとえば OpenGL では、glClipControl 関数により、ウインドウ座標系の原点を左上または左下に、クリップ座標系の深度モードを -1 から 1 または 0 から 1 に設定できます。クリップ座標系の深度モードの変更は、正規デバイス座標系からウィンドウ座標系に変換される際の z の計算式を変更します。深度モードを 0 から 1 にしている場合、その計算式において浮動小数点の足し算が入らなくなり、精度が改善される場合があります。
