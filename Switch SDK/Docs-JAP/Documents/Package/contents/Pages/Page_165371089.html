スクリプト処理の概要

Windows PowerShell スクリプトファイルでプロファイラをロードして、自動化した GUI 制御を実行できます。これは、同じ組み合わせのプロファイルを複数回取得する場合に効果的です。

Windows PowerShell

Windows PowerShell のスクリプト処理機能では、かなり複雑な動作が可能です。ここでは、このスクリプト言語の背景については取り上げません。WindowsPowerShell によるスクリプト処理の詳細情報は Microsoft から入手できます。

コマンドレット

このプロファイラでは、使用する新しいコマンドレットを作成することでスクリプト処理を実現できます。組み込みコマンドレットもすべて使用できます。別途インストールした拡張コマンドレットは使用できない場合があります。

プロファイラのコマンドレットはすべてプレフィックス 'Profiler-' で始まります。

注意： Windows PowerShell では、コマンドレットの大文字と小文字は区別されません。

プロファイラのコマンドレットの引数はスペースで区切って指定します。引用符で囲まれていないテキストは、内部での比較のために大文字に変換されます。このため、関数名で問題が発生します。したがって、すべての関数名を引用符（""）で囲む必要があります。複数の引数を指定する C++ 関数名やスペースを使用するテンプレート情報でも、必要に応じて引用符を使用できます。

引数名が [] で囲まれている場合、その引数の指定は任意です。引数名が省略されている場合、各引数はここに示されている順序で渡されます。

Profiler-Analyze

プロファイルデータの分析を実行するようプロファイラに要求します。

ストーリー分析

このオプションは、「ストーリー分析」で説明したストーリー分析モードのいずれかを使用してデータを分析するようプロファイラに要求します。

Profiler-Analyze -Story<StoryType>

-Story <StoryType>

  • ストーリー分析オプションのタイプの 1 つです。オプションについては「ストーリータイプ」を参照してください。

スパイク検出

このオプションは、「スパイク検出」で説明したスパイク検出モードのいずれかを使用してデータを分析するようプロファイラに要求します。

Profiler-Analyze -SpikeDetection<SpikeDetectionType>

-SpikeDetection <SpikeDetectionType>

  • スパイク検出オプションのタイプの 1 つです。オプションについては「スパイク検出タイプ」を参照してください。


Profiler-Analyze -Story Full
Profiler-Analyze -SpikeDetection Rare

Profiler-Assembly

「アセンブリ」タブとの連係を図るためのオプションを制御します。

関数の設定

Profiler-Assembly -SetFunction<String>


-FunctionName <String>

  • 入力する文字列は、プロファイラに表示される関数名と正確に一致している必要があります。C++ を使用する場合は、引数およびテンプレート情報を指定します。

ソースの表示

Profiler-Assembly -ShowSource<Boolean>

-ShowSource <Boolean>

  • ソースコードを表示するかどうかを指定します。


Profiler-Assembly -FunctionName "TestFunction"
Profiler-Assembly -ShowSource $true

サンプリングされるプロファイルにコールスタックを記録するかどうかを設定します。

Profiler-Callstacks [-ShowCallstacks]<Boolean>

[-ShowCallstacks] <Boolean>

  • コールスタックを記録するかどうかを指定します。


Profiler-Callstacks $false

Profiler-Copy

指定されたタブの内容をクリップボードにコピーするようプロファイラに指示します。

Profiler-Copy [-Tab] <ScriptingTabs> [-ExtendedParam <ScriptingCopyExtensions>]

[-Tab] <ScriptingTabs>

  • タブの名前については「スクリプト処理」タブを参照してください。

[-ExtendedParam <ScriptingCopyExtensions>]

  • 任意指定。タブによっては、複数のコピーオプションのあるものがあります。拡張オプションについては「コピー拡張」を参照してください。


Profiler-Copy Functions
Profiler-Copy CallTree Selected
Profiler-Copy CodeCoverage -ExtendedParam Seen

Profiler-Filter

指定のタブにフィルタを設定します。

このコマンドレットにはいくつかの異なる書式があります。すべての書式にすべてのタブとの互換性があるわけではありません。

名前基準

このオプションは、Profiler-Filter のデフォルトの形式です。タブの「フィルタ/検索」ボックスが変更されます。

Profiler-Filter [-Tab] <ScriptingTabs> [-Negate] <Boolean> [-RegEx] <Boolean> [-FilterName] <String>

[-Tab] <ScriptingTabs>

  • タブの名前については「スクリプト処理」タブを参照してください。

[-Negate] <Boolean>

  • 結果を反転するかどうかを指定します。

[-RegEx] <Boolean>

  • フィルタが正規表現かどうかを指定します。

[-FilterName] <String>

  • フィルタに使用するテキストです。特に正規表現フィルタの場合には、このテキストを引用符で囲むことをお勧めします。

制限

このオプションでは、「関数」タブに表示する結果を制限するか、「関数」タブにすべての結果を表示するかを指定します。

Profiler-Filter [-Tab] <ScriptingTabs> -Limit <Boolean>

[-Tab] <ScriptingTabs>

  • タブの名前については「スクリプト処理」タブを参照してください。

Limit <Boolean>

  • 表示する結果を制限するかどうかの指定。

選択表示

このオプションでは、選択した関数のみが表示されるように「関数」タブを設定します。

Profiler-Filter [-Tab] <ScriptingTabs> -ShowSelected <Boolean>

[-Tab] <ScriptingTabs>

  • タブの名前については「スクリプト処理」タブを参照してください。

ShowSelected <Boolean>

  • 選択した項目のみを表示するかどうかの指定。

コア基準

このオプションでは、指定されたコアで得られた結果のみを表示することによって、現在表示されている結果をフィルタ処理します。

Profiler-Filter [-Tab] <ScriptingTabs> -Core<Object>

[-Tab] <ScriptingTabs>

  • タブの名前については「スクリプト処理」タブを参照してください。

Core <Object>

  • 表示するコアの番号を整数で指定するか、文字列「All」を指定してすべてのコアを表示します。

スレッド基準

このオプションは、指定されたスレッドで得られた結果のみを表示することによって、現在表示されている結果をフィルタ処理します。

Profiler-Filter [-Tab] <ScriptingTabs> -Thread <String>

[-Tab] <ScriptingTabs>

  • タブの名前については「スクリプト処理」タブを参照してください。

Thread <String>

  • フィルタ処理するスレッドの名前。すべてのスレッドを表示するには「All」を指定します。

グループ基準

Profiler-Filter [-Tab] <ScriptingTabs> -Group <String>

[-Tab] <ScriptingTabs>

  • タブの名前については「スクリプト処理」タブを参照してください。

Group <String>

  • フィルタ処理するグループの名前。すべてのグループを表示するには「All」を指定します。

システム基準

Profiler-Filter [-Tab] <ScriptingTabs> -System <String>

[-Tab] <ScriptingTabs>

  • タブの名前については「スクリプト処理」タブを参照してください。

System <String>

  • フィルタ処理するシステムの名前。この名前は「システム分類フィルタ」に表示される名前と同じです。このフィルタ結果のいずれとも一致しない名前を指定すると、デフォルトの「すべてのシステム」オプションが選択されます。

Profiler-Filter Functions $false $true "^OS"
Profiler-Filter Functions -Negate $false -RegEx $true -FilterName "^OS"
Profiler-Filter Functions -ShowSelected $true
Profiler-Filter Threads -Core 0
Profiler-Filter Functions -Thread "ThreadName"
Profiler-Filter Counters -Group "GroupName"
Profiler-Filter Functions -System "Graphics"


Profiler-Marked

マーク付きコードをプロファイルデータに記録するかどうかを設定します。

GUI では各タブで個別にこの値を設定できるようになっていても、このコマンドレットではマーキングされたコードをサンプルプロファイルと計装プロファイルの両方で記録するようにこのフラグが設定されます。

Profiler-Marked [-ShowMarkedCode]<Boolean>

[-ShowMarkedCode] <Boolean>

  • マーク付きコードをプロファイルデータに記録するかどうかを指定します。

Profiler-Marked $false

Profiler-Open

NPROF ファイルを開きます。

Profiler-Open [-FileName]<String>

[-FileName] <String>

  • 開く NPROF ファイルへのフルパス。このパス名は引用符で囲むことをお勧めします。

Profiler-Open "C:/Profiles/SavedProfile1.nprof"

Profiler-PerfCounters

サンプルプロファイルと計装プロファイルの両方に、どのパフォーマンスカウンタを記録するかを設定します。

GUI ではこれらを個別に設定できるようになっていても、このコマンドレットでは両方が同時に設定されます。

Profiler-PerfCounters [-PerfCounters]<ScriptingPerfCounters>

[-PerfCounters] <ScriptingPerfCounters>

  • パフォーマンスカウンタの値の詳細は、「パフォーマンスカウンタのスクリプト処理」を参照してください。

Profiler-PerfCounters Disabled

Profiler-Sample

サンプルプロファイルに対して、サンプリングストラテジーとサンプリングレートの両方を設定します。

Profiler-Sample [-Strategy] <ScriptingSampleStrategy> [-Rate] <Integer> [-NoWarning]

[-Strategy] <ScriptingSampleStrategy>

  • 戦略値の詳細は、「サンプル方法のスクリプト処理」を参照してください。

[-Rate] <Integer>

  • レート値の詳細は、「スクリプト処理」を参照してください。

[-NoWarning]

  • 無効な方法とレートを組み合わせて指定しても、このフラグでは警告が生成されません。

返り値

  • 方法とレートが正しく設定されたかどうかを示すブール値。

Profiler-Sample Time 1000
$success = Profiler-Sample -NoWarning -Strategy Time -Rate 100

NameDisplay

Profiler-SampleGraph -NameDisplay<Scripting.SampleGraph.FunctionNameDisplay>

Profiler-Save

現在のプロファイルデータを保存します。

Profiler-Save [[-FileName] <String>]

[[-FileName] <String>]

  • 任意指定。保存するファイルのフルパス。このパス名は引用符で囲むことをお勧めします。この値を指定しないと、日時をファイル名として、現在のディレクトリにファイルが保存されます。

Profiler-Save

Profiler-Select

指定のタブで項目を選択します。

TopGroup

50 個の要素で構成される複数のグループを同時に選択します。

Profiler-Select [-Tab] <ScriptingTabs> -Top [[-TopGroup] <Integer>]

[-Tab] <ScriptingTabs>

  • タブの名前については「スクリプト処理」タブを参照してください。

-Top

  • 50 個の要素で構成される複数のグループが同時に選択されるよう指定します。

[[-TopGroup] <Integer>]

  • どのグループを選択するかの指定。指定しないと、最初の 50 個の要素が選択されます。

List

指定のインデックスの要素をタブから選択します（インデックスは、最上部を 0 として、最上部から最下部に向かって割り当てられます）。

Profiler-Select [-Tab] <ScriptingTabs> -List <Integer[]>

[-Tab] <ScriptingTabs>

  • タブの名前については「スクリプト処理」タブを参照してください。

-List <Integer[]>

  • 選択するインデックスのリスト。

DeselectAll

現在選択されているすべての要素を選択解除します。

Profiler-Select [-Tab] <ScriptingTabs> -DeselectAll

[-Tab] <ScriptingTabs>

  • タブの名前については「スクリプト処理」タブを参照してください。

-DeselectAll

  • タブで選択されているすべての要素を選択解除するよう指定します。

Profiler-Select Functions -Top
Profiler-Select Functions -Top 3
Profiler-Select Functions -List 0 1 4 5
Profiler-Select Functions -DeselectAll
[int[]] items = 0, 3
Profiler-Select Functions -List @items

Profiler-Sync

指定された ELF を使用してランタイムアプリケーションと同期します。

自動化にはこの機能を使用せず、一般的にはコマンドラインオプションを使用して自動ロードのための ELF を指定することをお勧めします。

Profiler-Sync [[-FileName] <String>]

[[-FileName] <String>]

  • 任意指定。ロードする ELF へのフルパス。このパス名は引用符で囲むことをお勧めします。ELF を具体的に指定しないと、GUI は現在使用されている ELF を判別して、それをロードしようとします。

返り値

  • 同期が成功したかどうかを示すブール値。

Profiler-Sync
$success = Profiler-Sync "C:/Game/bin/Release/game.nss"

<Integer[]>

コピーの拡張

「コールツリー」タブおよび「コードカバレッジ」タブにはいくつかのコピー機能があります。

以下では、各オプションに続けて、そのオプションを使用できるタブを示します。

  • 「すべて」：「コールツリー」および「コードカバレッジ」で使用できます。デフォルト値です。タブのすべてのデータをコピーするときに使用します。
  • 「未確認」：「コードカバレッジ」で使用できます。未確認関数リストのみをコピーするときに使用します。
  • 「確認済み」：「コードカバレッジ」で使用できます。確認済みの関数リストのみをコピーするときに使用します。
  • 「選択済み」：「コールツリー」で使用できます。コールツリーから選択済みの関数のみコピーするときに使用します。

その他の便利なコマンド

スクリプト処理を最大限に有効活用するうえで便利なプロパティ、コマンド、および関数呼び出しを以下に示します。

[Environment]::CurrentDirectory

  • プロファイラアプリケーションの現在のディレクトリを取得します。これは通常、EXE を起動したフォルダです。

[Nintendo.CPU.Profiler.Scripting.ScriptingCore]::NProfFilename

  • 現在ロードされている NPROF ファイルのファイル名全体。NPROF がロードされていない場合、またはまだ保存されていない新しいプロファイルの場合、この値は空の文字列です。

[Nintendo.CPU.Profiler.Scripting.ScriptingCore]::ScriptDirectory

  • 現在実行しているスクリプトのロード元ディレクトリ。


[Nintendo.CPU.Profiler.Scripting.ScriptingGUI]::ELFDirectory

  • 現在同期している ELF のディレクトリ。
