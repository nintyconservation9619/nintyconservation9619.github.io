概要

AdpcmEncoder は 入力 PCM データを 任天堂独自の ADPCM フォーマットにエンコードする PC ツールです。Tools/Audio/AdpcmTools/ 以下に配置されています。

このツールは以下の DLL に依存しています。これらは NintendoSDK に含まれるものです。

  • Tools/Audio/AdpcmTools/AdpcmCore.dll
  • Tools/Audio/AdpcmTools/AudioFile.dll

ADPCM フォーマットは過去の任天堂プラットフォームのものと同等ではありますが、全く同じものではありません。nn::audio ライブラリで解釈が可能なデータを作成するためには、AdpcmEncoder を利用する必要があります。

使用方法

$ AdpcmEncoder.exe [-o|--output <output-filepath>] [--loop-begin <sample-index>] [--loop-end <sample-index>] [-h|--help] [-v|--verbose] <input-filepath>

引数一覧

input-filepath

入力ファイル名を指定します。以下のファイルフォーマットが指定可能です。

  • 16bit .wav
  • 16bit .aiff

2 以上のチャンネル数の音源を扱うことも可能ですが、出力ファイルは 1 チャンネルのものがチャンネル数分だけ生成されます。

オプション一覧

-o, --output 出力ファイル名を指定します。拡張子は .adpcm とする必要があります。
--loop-begin ループ開始位置を指定します。位置は先頭を 0 としたサンプル位置で表します。
このオプションは --loop-end と同時に指定する必要があります。
--loop-end

ループ終了位置を指定します。位置は先頭を 0 としたサンプル位置で表します。

このオプションは --loop-begin と同時に指定する必要があります。
-h, --help ヘルプを表示します。
-v, --verbose 詳細なログを出力します。

使用例

input.wav を output.adpcm にエンコードする

$ AdpcmEncoder.exe -o output.adpcm input.wav

ループ位置を指定してエンコードする

$ AdpcmEncoder.exe --output output.adpcm --loop-begin 1000 --loop-end 2000
