LDN ライブラリの初期化と終了

LDN ライブラリの初期化

LDN ライブラリにおける他のあらゆる機能を使用する前に、まず、LDN ライブラリを初期化する必要があります。

LDN ライブラリの初期化には nn::ldn::Initialize() を使用します。この API はローカル通信のために通信デバイスを占有するため、LDN ライブラリの使用を終了するまで他の通信機能を利用できなくなります。システムが通信機能を使用する場合もありますので、必要なシーン以外では LDN ライブラリを初期化しないでください。なお、ローカル通信の開始時に他の通信処理の終了を待機するため、nn::ldn::Initialize() が最大数秒間程度処理を返さないことがあります。

LDN ライブラリを初期化すると nn::ldn::GetState() で取得できる状態は nn::ldn::State_None から nn::ldn::State_Initialized に遷移します。

LDN ライブラリの終了

LDN ライブラリの使用を終了するためには nn::ldn::Finalize() を使用します。

nn::ldn::Initialize() に成功した後であれば、あらゆる状態から nn::ldn::Finalize() を呼び出すことができます。アクセスポイントとしてネットワークを構築した状態であれば自動的にネットワークが破棄され、ステーションとしてネットワークに接続した状態であれば自動的にネットワークから切断します。システムにローカル通信の使用状況を伝えるため、ローカル通信の使用が終わったら速やかに nn::ldn::Finalize() を実行してください。ローカル通信を完了したにも関わらず、LDN ライブラリを初期化したまま放置することがないように注意してください。

LDN ライブラリの使用を終了すると nn::ldn::GetState() で取得できる状態は nn::ldn::State_None に遷移します。

エラーハンドリング

nn::ldn::Initialize() は通信デバイスの占有に失敗した場合に失敗します。詳細についてはエラーハンドリングを参照してください。
