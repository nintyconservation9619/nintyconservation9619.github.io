はじめに

FONTLL ライブラリでは、スケーラブルフォントを扱う低レイヤー API を提供します。

対応フォントファイル形式

FONTLL が対応するフォントファイル形式は、以下の通りです。

  • 本体内蔵フォント用スケーラブルフォント(bfttf)
  • TrueType(ttf)
  • OpenType(otf)

bfttf

bfttf は本体内蔵フォント向けに作られた専用フォーマットです。

パッケージに付属する nintendo_NTLG-DB_002.bfttf のライセンスについて

Samples\Resources\font に含まれている、nintendo_NTLG-DB_002.bfttf はサンプル用途に限って利用してください。製品での利用は許諾されません。

SDK パッケージ同梱のライセンスファイル (Documents/Licenses/Fontworks/readme.txt)もあわせて確認してください。

nn::font との違い

nn::fontll と似た機能を提供する nn::font というモジュールがあります。

nn::fontll は最低限の機能だけを提供する下位モジュールです。
対して、nn::font は高度な機能をより簡単に利用できるようにした上位モジュールです（内部で nn::fontll を使っています）。

ゲーム画面にフォント表示するまでの処理段階を以下のように分類したとき、具体的に nn::fontll が提供するのは 1、2 の項目の機能です。

  1. (nn::fontll) フォントを読み込み、グリフ情報へのアクセスする
  2. (nn::fontll) メインメモリにグリフビットマップを展開する
  3. (nn::font) nn::font::TextureCache クラスでグリフビットマップをテクスチャに変換。GPU で画面表示できるようにする
  4. (nn::font) nn::font::TextureCache クラスで上記テクスチャの管理をする（キャッシュ構造）
  5. (nn::font) nn::font ライブラリ、nn::ui2d ライブラリと協調利用する(インタフェース整合、ヘルパ)

権利関連の補足・注意

FontConverter には、いかなるフォントのライセンスも付属しません。

フォントを利用したソフトを発売するには別途フォントメーカーとのライセンス契約が必要になります。

サンプルプログラム

Samples\Sources\Applications\FontllSaveToJpeg 以下に、フォントのグリフを取得して画像に変換するサンプルプログラムがありますので、そちらを参考にしてください。

グリフメンバーの詳細

グリフ情報（ScGlyphMap）の詳細を説明します。

原点(origine)

文字が定義される基点です。

文字送り量(i_dx, i_dy)

現在の文字原点と、次の文字原点の間隔です。
横書き文字の場合は、i_dy は常にゼロになります。
dx、dy で固定小数表現の値が取得できます(通常、利用する必要はありません)。

グリフ位置(lo_x, hi_y)

原点を基準にして、実際にグリフ画像(ピクセルが存在する)矩形の左上位置を示します。

グリフサイズ(width, height)

実際のグリフ画像(ピクセルが存在する)矩形の幅と高さです。
