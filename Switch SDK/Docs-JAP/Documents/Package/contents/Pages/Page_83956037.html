3D シェーダーコンバーター

3D シェーダーコンバーター(%NINTENDO_SDK_ROOT%\Tools\Graphics\3dTools\3dShaderConverter.exe)の使用方法と注意事項を説明します。

  • 概要
  • コマンドラインオプション
  • 仕様
  • 制限事項
  • 利用例

概要

シェーダー設定ファイル（.fsc）からシェーダー定義ファイル（.fsd）を生成します。また、シェーダー定義ファイルをランタイム用のバイナリーファイル (.bfsha) に変換します。その際、シェーダーバリエーションファイル (.fsv) により、生成するバリエーションを指定することもできます。

コマンドラインオプション

3dShaderConverter.exe [--args-file ""] [--version] [-h] [-s] [--verify]
[--root-config ""] [--root-definition ""] [--force-variation] [--dump-shader-source ""] [--define-macro] [--glsl-version] 
[--api-type ""] [--code-type ""] [--debug-info-directory ""] [--shader-cache-directory ""] [--debug-info-level ""] [--glsl-extension ""]
[--preprocess] [--jobs ""] [--output ""] [--gfx-shader-converter-options ""] [--skip-convert] [--variation-begin-ratio ""] [--variation-end-ratio ""]
[ファイル]... 

--args-file ""

引数ファイルを指定します。
引数ファイルに記述した内容を、引数として指定できます。
引数ファイルは複数行に渡って各オプションを記述できます。
行頭に # がある行はコメントとして扱います。
オプション値指定の "" によるエスケープに改行を含めることはできません。

--version

バージョン情報を表示します。

-h --help

ヘルプを表示します。

-s --silent

処理中のメッセージ表示を抑制します。

--verify

中間ファイルがバイナリーコンバート可能かどうかを調べます。

-o "", --output ""

出力ファイル名を指定します。
記述例： --output "output_dir\demo.bfsha"、-o "output_dir\demo.bfsha"

ファイル

シェーダー設定ファイルまたはシェーダー定義ファイルを指定します。

シェーダー定義ファイル(.fsd) 生成向けオプション

--unified-annotation

頂点シェーダー、フラグメントシェーダーなどステージ間でアノテーション表記が異なる場合に、
頂点シェーダー ＜ ジオメトリシェーダー ＜ フラグメントシェーダー < コンピュートシェーダー の順で上書きを行います。

--root-config ""

シェーダー定義ファイルに含まれるパスを指定したフォルダからの相対パスにします。
例）--root-config "D:\home" と指定するとシェーダー定義ファイルのパスから D:\home が除去されます。
このオプションはコンバーターの入力パスの指定には利用できません。

シェーダーアーカイブファイル(.bfsha) 生成向けオプション

--root-definition ""

バイナリーファイルに含まれるパスを指定したフォルダからの相対パスにします。
例）--root-definition "D:\home" と指定するとバイナリーファイルのパスから D:\home が除去されます。
このオプションはコンバーターの入力パスの指定には利用できません。

--force-variation

シェーダーオプションの branch を強制的に false とみなし、バリエーションとして扱うオプションです。

--dump-shader-source ""

コンパイル時の情報を出力するオプションです。
出力先フォルダのパスを指定します。

出力される情報については プラットフォーム固有の情報 も併せてご覧ください。

--glsl-version ""

ソースの先頭に挿入される GLSL のバージョン文字列を指定します。

--api-type ""

必須オプションです。使用するグラフィックスAPIの種類を指定します。設定値は GL です。
アドオンパッケージをインストールすることにより、設定できる値は追加されます。

--code-type ""

出力するシェーダーの種類を指定します。設定値は api-type によって異なります。
--api-type GL では下記を指定できます。
Source: シェーダーソースを出力ファイルに含めます。
Binary: シェーダーバイナリーを出力ファイルに含めます。
Binary_Source: シェーダーバイナリー、ソースを出力ファイルに含めます。

--debug-info-directory ""

GPU デバッガのデバグ情報を出力します。

--shader-cache-directory ""

シェーダーのコンパイル結果を指定したディレクトリにキャッシングします。 
2 度目のコンバートが高速になります。
--api-type GL ではキャッシングされません。

--debug-info-level ""

付加するデバッグ情報のレベルを指定します。
使用可能な値はGFXライブラリ／ツールリファレンス／シェーダーコンバーター (.exe)をご参照ください。

--preprocess

プリプロセス済みのシェーダーをコンパイルします。

--use-preprocessor-variation

プリプロセッサ定義を使用してバリエーション生成します。
未指定時はオプション変数にバリエーション定数が存在する場合はバリエーション定数が使用されます。
--api-type、シェーダーコードの長さ、バリエーション数、バリエーション定数の数によっては
本オプションと --jobs を使用した方が高速にコンバートできる場合があります。
プリプロセッサ定義、バリエーション定数についてはシェーダーコンパイラーをご参照ください。

--define-macro ""

プリプロセッサマクロを定義します。
値を指定する場合はマクロ名に = を続けて値を指定します。
= を記述せずマクロ名のみ指定した場合はそのマクロの値は 1 になります。
記述例: --define-macro "HOGE" --define-macro "FOO=2"
上記例では下記のコードがシェーダーコード内に埋め込まれます。

#define HOGE 1
#define FOO 2

--glsl-extension ""

glsl 拡張をシェーダーコードに埋め込みます。 
直接シェーダーソースコードに書いた場合はコンパイルエラーになります。

--jobs ""

並列処理の最大スレッド数を指定します。
1 を指定した場合はシングルスレッドで動作します 。

参考：

基本的にはコア数の数だけ高速化されますが一部の api-type では低レイヤコンパイラの仕様上高速化されない場合があります。

--gfx-shader-converter-options ""

シェーダーコンバーターへのオプションを指定します。
--jobs や --shader-cache-dirextory 等の 3D シェーダーコンバーターにも存在するオプションを同時指定した場合、
基本的に後に指定された値が採用されますが、複数値指定可能なオプションはすべての値が採用されます。
シェーダーコンバーターオプションで使用するダブルクオートのエスケープはシングルクオートで記述します。
(例) --gfx-shader-converter-options "--preprocessor-definition=' TEST' "

--skip-convert

シェーダーのコンパイルまでを行い、コンパイル以外の処理(シェーダーアーカイブへのシリアライズ等)をスキップします。
そのためシェーダーアーカイブは出力されませんが、コンパイルは実行されるため --shader-cashe-directory を使用したコンパイルキャッシュの出力は可能です。
「分散コンパイル」の仕組みを利用する場合等に効果的に使用することができます。

--variation-begin-ratio ""
--variation-end-ratio ""

ひとつのシェーダーアーカイブのコンバートを複数回に分割する場合に、生成するシェーダーバリエーション範囲の開始位置もしくは終了位置を比率で指定します。
--shader-cashe-directory オプションと合わせて使用することで「分散コンパイル」の仕組みを利用することができます。
生成される全シェーダーバリエーションの範囲を [0.0, 1.0] としたとき、[variation-begin-ratio, variation-end-ratio] の範囲にあるシェーダーバリエーションがコンパイルされます。
0.0 <= variation-begin-ratio <= variation-end-ratio <= 1.0 の範囲で指定可能です。
variation-begin-ratio を指定しない場合は [0.0, variation-end-ratio] の範囲のシェーダーバリエーションがコンパイルされます。
variation-end-ratio を指定しない場合は [variation-begin-ratio, 1.0] の範囲のシェーダーバリエーションがコンパイルされます。


プラットフォーム固有のオプション

プラットフォーム固有のオプションについては  プラットフォーム固有の情報 をご参照ください。

仕様

3D シェーダーコンバーターの仕様について説明します。

ソースコードの文字コード

シェーダーソースコードの文字コードを表すコードページは fsc の code_page 属性を用いて指定してください。code_page="0" とするとシステムロケールに従ってソースを解釈します。読み込むシェーダーソースと fsc に記述したコードページに食い違いがあるとコンパイルエラー時の表示が途中で途切れる事があるので注意して下さい。

自動定義されるマクロ

以下のマクロはシェーダーコンバート時に自動的に定義されます。

マクロ名 値
NN_G3D_VERTEX_SHADER 1
NN_G3D_GEOMETRY_SHADER 1
NN_G3D_FRAGMENT_SHADER 1
NN_G3D_COMPUTE_SHADER 1

自動定義されるオプション

以下のオプションは自動的に定義されます。

オプション名 値
system_id 0

fsc でバリエーションの記述を省略した際の挙動

基本的には、すべてのオプションはバリエーションとして fsc に記述することが想定されています。

バリエーションの記述を省略した際は、以下の扱いになります。

属性名 値
choice default と同じ
default (空)
branch false

各属性の詳細については、中間ファイル仕様書の <variation> を参照してください。

fsv でオプションの記述を省略した際の挙動

fsv によるバリエーションの絞り込みで、記述を省略したオプションについては、choice にデフォルトが指定されたものと見なされます。

制限事項

3D シェーダーコンバーターの制限事項を説明します。

使用可能なファイル名に関する制限

入力及び --output に指定するファイル名に使用可能な文字は次の文字のみです。

[0-9A-Za-z-_]

同名のソースファイル

ソースファイルの名前はユニークである必要があります。以下のように、同名のファイルが存在する場合、正常に動作しません。

./abc.glsl
./def/abc.glsl

include-path と ファイルの配置

3D シェーダーコンバーターは #include を解決する際に fsc からのパスと include-path からの相対パスの両方を用いて解決します。

このとき、fsc とソースファイルの記述に従い、一旦すべてのファイルをワークスペースに展開するために以下の制約があります。

  • include-path は包含できない
    include-path を包含する include-path は定義することができません。 このようなパスが指定された場合 fsd への変換は失敗します。
  • 同じ名前のファイルを登録できない
    例えば、include0/src/abc.glsl と include1/src/abc.glsl がソースとして登録されており、 include-path に include0 と include1 が指定された場合、両方とも相対パスが src/abc.glsl となり変換は失敗します。

  • include-path から上位ディレクトリを指定できない
    下記のように、上位ディレクトリのファイルを相対パスで参照することはできません。

    #include <../abc.glsl>

アノテーション

アノテーションについては シェーダーアノテーション をご覧ください。

特に注意するべき項目について記載します。

  • option アノテーションが付与されたマクロの #ifdef~#endif などの間で #include を記述しない
    下記のような記述を行うとマクロの初期値によっては #include が解決できず、ファイルを展開できない可能性があります。

    #define TEST // @@ id="option" choice="bool"

#ifdef TEST
    #include <abc.glsl>
#else
    #include <def.glsl>
#endif

  • 変数の後のアノテーションでは C スタイルコメント形式が使えない
    変数の後にアノテーションを付ける場合は C++ スタイルで記述してください。 もしアノテーションが複数行に渡る場合は C++ スタイルアノテーションと C スタイルによる遠隔指定がお勧めです。

     // OK
uniform sampler2D albedo0;     // @@ id="albedo0"
// NG
uniform sampler2D albedo1;     /* @@ id="albedo1" */
// OK
/* @@

sampler_id="albedo0" hint="_a0"
label="albedo diffuse texture"

*/

  • define-macro オプションにより値が定義されるマクロブロック内にアノテーションを記述しない
    マクロブロック内のアノテーションは無視されます。下記の例では "enable_vertex_color" は生成された fsdb に存在しなくなります。期待しない動作の原因になるためこの様な記述は避けて下さい。

    #if defined( FIXED_MACRO_USE_VERTEX_COLOR )               // FIXED_MACRO_USE_VERTEX_COLOR は define-macro オプションで値が定義されるマクロ
    #define ENABLE_VERTEX_COLOR          ( 0 ) // @@ id="enable_vertex_color" choice="0, 1" type="dynamic"
    #endif

シェーダー定義中間ファイルのテキスト中間ファイル（fsda）

シェーダー定義中間ファイルのテキスト中間ファイル（fsda）はサポートされていません。

利用例

3D シェーダーコンバーターの利用方法を説明します。

シェーダー定義ファイルの生成

shader.fsca を処理して、output.fsdb を生成します。

3dShaderConverter.exe --output output.fsdb shader.fsca

シェーダー定義ファイルの変換

shader.fsdb をコンバートして、output.bfsha に保存します。

3dShaderConverter.exe --glsl-version 430 --api-type GL --code-type Binary_Source --output output.bfsha shader.fsdb

引数ファイルを用いて shader.fsdb をコンバートして、output.bfsha に保存します。

convert.txt

--glsl-version 430
--api-type GL
--code-type Binary_Source
--output output.bfsha
shader.fsdb
 
3dShaderConverter.exe --args-file convert.txt

シェーダーバリエーションの絞り込み

シェーダー定義ファイルの変換では、通常はすべてのバリエーションが生成されますが、シェーダーバリエーションファイルを入力することで、生成するバリエーションを絞り込むことができます。

3dShaderConverter.exe --glsl-version 430 --api-type GL --code-type Binary_Source --output output.bfsha shader.fsdb shader.fsva
