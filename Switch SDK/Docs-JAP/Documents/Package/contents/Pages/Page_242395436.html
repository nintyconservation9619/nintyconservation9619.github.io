概要

SoundProjectUtility.exe (以下、SoundProjectUtility) は、NintendoWare SoundMaker プロジェクトファイルに対する雑多な処理を行うコマンドラインツールです。

ここでは SoundProjectUtility の使い方と機能について説明します。

SoundProjectUtility の使い方

SoundProjectUtility の実行方法

SoundProjectUtility を使うには、コマンドラインから以下の形式で実行します。

SoundProjectUtility.exe <サブコマンド> [オプション] <プロジェクトファイルパス>

指定できるオプションはサブコマンド毎に異なります。

例）list-files サブコマンドは指定したサウンドプロジェクトが参照しているファイルのリストを出力します。

$ cd %NINTENDO_SDK_ROOT%
$ ./Tools/Audio/SoundMaker/SoundProjectUtility.exe list-files --relative . Samples/Sources/Resources/Atk/Common/Common.fspj
Samples/Sources/Resources/Atk/Common/aif/bass_pull.e2.aiff
Samples/Sources/Resources/Atk/Common/aif/bass_slpthm.e2.aiff
Samples/Sources/Resources/Atk/Common/aif/bassdrum.aiff
Samples/Sources/Resources/Atk/Common/aif/cabasa.aiff
...

プロジェクトファイル (.fspj) のパスを指定します。

SoundProjectUtility の機能

list-files サブコマンド

サウンドプロジェクトが参照しているファイルの一覧を出力します。

オプション 説明
-o, --output <filepath>

標準出力の代わりに結果を書き込むファイルのパスを指定します。

--relative <basepath>

相対パスを出力します。相対パスの基準となるパスを指定します。
--backslash

出力のディレクトリセパレータ文字としてバックスラッシュを使います。

指定したサウンドプロジェクトファイル (.fspj) および、参照するサウンドセットファイル (.fsst) やバンクファイル (.fbnk)、またそれらが参照するテキストシーケンスファイル、標準 MIDI ファイル、波形ファイル等の参照ファイルのパスを標準出力に出力します。

  • --output オプションを指定すると、標準出力の代わりに指定のファイルにパスを出力します。
  • パスの形式を --relative および --backslash オプションで調整できます。

update-project サブコマンド

サウンドプロジェクトのバージョンを更新します。

オプション 説明
--backslash

出力のディレクトリセパレータ文字としてバックスラッシュを使います。
--backup

更新時にオリジナルのコピーを保存します。
--dry-run

実際には処理を行いません。

--relative <basepath>

相対パスを出力します。相対パスの基準となるパスを指定します。
-s, --silent

出力を抑制します。

指定したサウンドプロジェクトファイル (.fspj) および、参照するサウンドセットファイル (.fsst) やバンクファイル (.fbnk) の内容を、SoundMaker が対応するファイルの最新バージョンに更新します。

更新されたファイルのパスが標準出力に出力されます。

  • パスの形式を --relative および --backslash オプションで調整できます。
  • --silent オプションを指定するとパスは出力されません。
  • --dry-run オプションを指定すると、バージョンの更新されたファイルのパスが出力されますが、実際には更新は行われません。

追加情報

  • SoundProjectUtility.exe は単体では使用できません。"インストールディレクトリ"\Tools\Audio\SoundMaker フォルダごと必要ですのでご注意ください。
