VSI

NintendoSDK のウィザードが削除されない

NDI を使って NintendoSDK をインストールすると、Visual Studio のプロジェクトテンプレートとして、NintendoSDK に関連したウィザードが追加されます。

ただ、お使いの PC から NintendoSDK を削除しても上記のウィザードが削除されない問題があります。

ウィザードを削除するには、下記のフォルダを手動で削除してください。

  • %USERPROFILE%\Documents\Visual Studio 2017\Wizards\NintendoSDK
  • %USERPROFILE%\Documents\Visual Studio 2017\NintendoSDK\CustomWizard
  • %USERPROFILE%\Documents\Visual Studio 2015\Wizards\NintendoSDK
  • %USERPROFILE%\Documents\Visual Studio 2015\NintendoSDK\CustomWizard
  • %USERPROFILE%\Documents\Visual Studio 2013\Wizards\NintendoSDK
  • %USERPROFILE%\Documents\Visual Studio 2013\NintendoSDK\CustomWizard
  • %USERPROFILE%\Documents\Visual Studio 2012\Wizards\NintendoSDK
  • %USERPROFILE%\Documents\Visual Studio 2012\NintendoSDK\CustomWizard

NintendoSDK のウィザードが利用できない

NintendoSDK のウィザードを構成するファイルは、NDI で NintendoSDK をインストールしたときに利用したユーザーアカウントの %USERPROFILE% 以下に配置されます。

このため、「NintendoSDK をインストールしたときのユーザー」と「ウィザードを利用するときのユーザー」が異なる場合、 NintendoSDK のウィザードが Visual Studio のプロジェクトテンプレートに現れません。

ウィザードを利用するには、ウィザードを利用したいユーザーアカウントで NintendoSDK をインストールしてください。

プロジェクトのリビルド時に警告 LNK4099 が出力される

"NintendoSDK ライブラリ" プロジェクトで作成したライブラリにリンクする "NintendoSDK アプリケーション" プロジェクトを作成した場合リビルドを行うと警告 LNK4099 が出力される場合があります。

これはライブラリのデバッグ情報として pdb ファイル形式を使っているために発生する警告であり仕様となっています。リビルドではなくプロジェクトを一旦クリーンした後にビルドするとこの警告は発生しません。

次のどちらかの方法で、この警告を回避することができます。

  • プロジェクト作成時にオプション “ソリューションのディレクトリを作成” を有効にする
  • ライブラリの “デバッグ情報の形式” を “C7互換 (/Z7)” もしくは “なし” にする

SDK のライブラリで発生した例外を扱うことができない

NX32 及び NX64 用の SDK ライブラリは、C++ 例外をサポートしないようにビルドされています。
そのため、ライブラリ内で例外が発生した場合、Abort するような形になります。

アプリケーション側のみしか伝搬しないコードであれば、ビルド時に C++ 例外を有効にしてビルドすることで例外を扱うことはできます。

デバッグ中にスリープさせるとデバッグが中止される

スリープ時に VSI からのデバッグが中止されるのは仕様となります。
起床後にデバッグを再開したい場合は、ビルド済プログラムをデバッグする の、「既に起動してるアプリをデバッグする」 の項目を参考に、再度アタッチしてください。
スリープ前にブレークさせておくことで、起床直後の状態からアタッチすることも可能です。

Build Tools for Visual Studio 2017 と Visual Studio 2017 の両方がインストールされている場合 VSI for NX のインストールに失敗する

Build Tools for Visual Studio 2017 と Visual Studio 2017 の両方がインストールされている場合、 VSI for NX のインストールに失敗する可能性があります。

Visual Studio 2017 (バージョン 15.4) の VSIXInstaller には、Build Tools for Visual Studio 2017 がインストールされている場合に Visual Studio 拡張をインストールできない不具合があるためです。詳細は次のページを参照してください:  vsixinstaller.exe throws NullReferenceException when Visual Studio Build Tools 2017 are installed

この問題を回避するには、次の手順を試してください。

  1. Build Tools for Visual Studio 2017 をアンインストールする

Visual Studio Help

インターネット接続できない環境で Document パッケージをインストールすると失敗する

インターネット接続ができない環境で NintendoSDK Documents パッケージをインストールするとインストールに失敗する場合があります。
これはデジタル証明書の失効サーバーへの接続できないために署名の有効性が検証できずヘルプファイルのインストールができないためです。

この問題を回避するには、次の手順を試してください。

  1. インターネットに接続する
    or
    デジタル証明書の上位局が発行する中間証明書(CN = DigiCert SHA2 Assured ID Code Signing CA)をインストールする
  2. NintendoSDK Documents パッケージを再インストールする

NintendoSDK API リファレンスマニュアルの Result 値検索が Microsoft ヘルプビューアーから正しく動作しない

NintendoSDK API リファレンスマニュアルの Result 値検索が Microsoft ヘルプビューアーから利用すると、テキストボックスに値が入力できず Result 値が検索できません。

Result 値検索を利用する場合はウェブブラウザからご利用ください。

(独自のビルドシステム) cl.exe からの構文エラー

Visual Studio 2017 (バージョン 15.3) の cl.exe (バージョン 14.11.25503) には、 /permissive- オプションを与えた場合、クラステンプレートに対して誤った構文エラーが生成される不具合があります。

error C2187: syntax error: '<token>' was unexpected here
error C2187: 構文エラー: ここでは '<token>' は予期されていませんでした
error C2760: syntax error: expected ';' not 'identifier'"
error C2760: 構文エラー: ';' ではなく '識別子' が必要です

この問題を回避するには、次の手順を試してください。

  • cl.exe に対し、/permissive- オプションより後ろに /Zc:twoPhase- オプションを与える

この問題の詳細は次のページを参照してください。

  • Variable template compiler error - Developer Community
  • VS 2017 15.3 with -permissive- shows error C2187 in combaseapi.h - Developer Community

Build Tools for Visual Studio 2017 と Visual Studio 2017 の両方がインストールされている場合 NintendoSDK VsExtension のインストールに失敗する

  • Build Tools for Visual Studio 2017 をアンインストールする

Visual Studio 2017 (バージョン 15.1 以前) に NintendoSDK VsExtension がインストールされない

インストールされている Visual Studio 2017 のバージョンが 15.1 以前の場合、NintendoSDK VsExtension はインストールされません。これは仕様です。

この問題を回避するには、次の手順を試してください。

  1. Visual Studio 2017 (バージョン 15.2 以降) に更新する
  2. %NINTENDO_SDK_ROOT%\Installers\NintendoSdkVsExtension\Nintendo.NintendoSdkVsExtension.vsix をダブルクリックして手動でインストールを行う

FS

nn::fs::Commit() / nn::fs::CommitSaveData() が ResultTargetLocked で失敗する

エディタなど、ホスト PC 上のプログラムがセーブデータのファイルをオープンした状態では nn::fs::Commit() / nn::fs::CommitSaveData() が正しく動作しません。必ずほかのプログラムから開かれていない状態でアプリケーションを実行するようにしてください。

また、ウイルススキャン等のプログラムが自動でファイルアクセスをしてしまう場合があります。この場合は、アプリケーションが配置されているパスをウイルススキャンの対象外に設定する等の対策をしてください。


ドライブのルートディレクトリに nn::fs::ReadDirectory() を実行した際、1つ目のエントリが列挙されない場合がある

nn::fs::MountHost()/nn::fs::MountHostRoot() 経由でドライブ直下を nn::fs::OpenDirectory/nn::fs::ReadDirectory() した際に、対象がリムーバブルメディアなどの場合に、1つ目のエントリが列挙されない場合があります。

リムーバブルメディアのルートディレクトリの利用を避ける、ダミーのエントリを配置するなどして回避してください。

ネットワークに接続できない

nn::nifm::IsNetworkAvailable() が true にならない

設定の内容が意図通りになっていることを確認してください

インポートされた内容を確認する場合は、「NIFM ライブラリ-ネットワーク接続設定の登録」を参考に、設定をエクスポートして確認してください。

認証と暗号化の設定は、 SDK の対応している組み合わせになっている必要があります。

システムが自動で利用する対象に含めるには、自動接続に利用する設定を有効にする必要があります。

ステルス設定の無線アクセスポイントに接続する場合、ステルスのアクセスポイントに対して接続を試行する設定を有効にする必要があります。

Windows 環境では、Windows が利用しているネットワーク接続に合致する内容のネットワーク接続設定を登録する必要があります。

nn::nifm::IsNetworkAvailable() は true でも、外部ネットワークに接続できない

外部ネットワークに接続可能な回線を利用していることを確認してください

NintendoSDK 0.11.0 から、ネットワーク接続設定に登録したプロキシ設定の内容が自動で反映されるようになりました。これに伴って、curl_easy_setopt() で設定したプロキシの内容は無視されるようになります。「NIFM ライブラリ-ネットワーク接続設定の登録」を参考に接続設定を登録するか、curl_easy_setopt() で CURLOPT_PROXYAUTOCONFIG の値を 0 に設定して接続設定のプロキシ設定が反映されないようにしてください。

Nintendo Dev Interface(NDI)

環境の削除に失敗することがある

Nintendo Target Manager など、削除しようとする環境にインストールされたプログラムが起動したままになっていると、ファイルが削除できないため環境の削除に失敗します。
起動しているプログラムを終了することで問題を回避できます。


NINTENDO_SDK_ROOT 環境変数が設定されない

NDI で「環境」を作成すると、一番最後に作成した NintendoSDK のパスを値とする NINTENDO_SDK_ROOT 環境変数が設定されます。

意図した「環境」に NINTENDO_SDK_ROOT 環境変数が設定されない場合や、別の「環境」に NINTENDO_SDK_ROOT 環境変数を設定したい場合は、NDI で環境を選択後、[管理] → [Switch デフォルトに設定] を選択してください。

本体機能

「認証表示」の画像は仮画像になっている

製品版相当でない実機はぼやけています。

MP品未満の本体、Joy-Conでスティックの補正を行うと不具合が発生する

スティックの補正はMP品以上のみサポートされています。EP品などはサポートされていないのでMP品以上で行うようにお願いします。


HID

PC 環境で振動が発生しない

症状

NintendoSDK 0.15.0 以前を利用したことがある PC 上で、NintendoSDK 0.16.0 以降を利用して振動機能を利用する際に、振動が発生しない不具合があります。

HidVibrationBasic や HidVibrationPlayer のような振動機能を利用するプログラムを起動した際に、この不具合を確認できます。 

回避策

以下のファイルを手動で削除してください。

C:\Users\<ユーザー名>\AppData\Local\Temp\Nintendo_SystemSaveData_SystemSettings

その後、該当プログラムを PC 上で再度起動すると、上記のファイルが自動的に再生成されて、振動が発生するようになります。


もしくは、設定マネージャを使用して、設定ファイルを変更することでも回避可能です。

具体的には、設定ファイルをエクスポートして、hid_settings.vibration_master_volumeの値を1.0に変更し、設定ファイルをインポートします。

 

  1. 設定ファイルのエクスポート

    Tools\CommandLineTools\Settings\SettingsManager.exe Export "エクスポート先となるファイルの絶対パス"

  2. 編集するべき設定ファイルの値（vibration_master_volumeで検索ください）

    "hid_settings": {
    "vibration_master_volume": {
        "current": 1.0,
        "default": 1.0
    }
},

  3. 設定ファイルのインポート

    Tools\CommandLineTools\Settings\SettingsManager.exe Import "設定ファイルの絶対パス"

PC 環境でアプリ終了時に数秒間振動が継続することがある

症状

 PC 環境で振動機能を利用する際に、振動値を送信した状態でアプリを終了すると、コントローラが数秒間振動することがあります。 

回避策

PC 環境でアプリを終了する前に、振動を停止する処理をした上で、アプリを終了するようにしてください。

この不具合は PC 環境でしか発生しません。


オーディオツール

NintendoWare SoundMaker でインポートしたサウンドプロジェクトをコンバートすると、不正なサウンドアーカイブが出力される

NW4C / NW4F のサウンドプロジェクト（コンバート用のキャッシュファイルが残った状態）をインポートして通常のコンバートを実行すると、NintendoSDK 向けのバイナリサウンドアーカイブファイル (.bfsar) に、NW4C / NW4F 向けのバイナリリソースファイルが混ざってしまい、nn::atk ランタイムで正しく読み込めない問題が発生します。

古いキャッシュを利用しているのが問題であるため、サウンドプロジェクトをインポートした後は [再コンバート]を行ってください。

NintendoWare SoundMaker, SoundProjectWaveExporter でサウンドの波形出力をした際に、波形の頭に無音が入ることがある

NintendoWare のツール SoundMaker, SoundProjectWaveExporter でサウンドの波形出力を行うと、出力波形の頭に無音が入ることがあります。

もし波形の頭に無音が入ってしまった場合は、波形編集ソフトなどで波形から無音部分を削除してください。

NintendoWare AtkPlayer でサウンドアーカイブのリロードを行うとアプリケーションが停止することがある

AtkPlayer の起動後、以下の操作を行うことでアプリケーションが停止することがあります。

  • SoundMaker でストリームサウンドを編集してサウンドアーカイブファイルのコンバートを行い、AtkPlayer でサウンドアーカイブのリロードを行う
  • SoundMaker で別のサウンドプロジェクトを開き、AtkPlayer でサウンドアーカイブのリロードを行う

上記操作後は AtkPlayer を起動しなおすか、SoundMaker でのコンバート後しばらく待ってから AtkPlayer のリロードを行うことで問題を回避できることがあります。

nn::atk::SoundSystem::SetVoiceCountMax によって最大ボイス数を設定できない

nn::atk::SoundSystem::SetVoiceCountMax を用いてサウンドライブラリが使用する最大ボイス数を設定しようとした場合、その設定は無視されて nn::atk::SoundSystem::SoundSystemParam::voiceCountMax で設定した値が使用されます。

nn::atk::SoundSystem::SetVoiceCountMax は非推奨な機能であるため、推奨される機能である nn::atk::SoundSystem::SoundSystemParam::voiceCountMax を用いることをご検討ください。詳しくは nn::atk::SoundSystem::SoundSystemParam::voiceCountMax のリファレンスを参照ください。

NintendoWare AtkPlayer でサウンド再生前に変更したパラメータがサウンド再生時に適用されないことがある

AtkPlayer の起動後、1 度も再生していないサウンドに対してパラメータを変更し、サウンドを再生した場合にパラメータが適用されないことがあります。この問題に遭遇した場合は、再びサウンドの再生を行うことで回避することができます。
