「ホーム」タブの概要

「ホーム」タブには、開発機材への接続やプロファイルの操作を初めとするプロファイラの主要な機能が用意されています。

「接続」グループ

開発機材に接続するには、まず「接続」ボタンをクリックします。このボタンをクリックすると、以下の 3 ステップの接続プロセスが開始されます。

  1. 開発機材を選択します。利用可能な開発機材が 1 つのみの場合は、その開発機材に自動的に接続します。複数の開発機材を使用可能な場合は、使用可能な開発機材の一覧のドロップダウンメニューを備えたウィンドウが表示されます。
  2. プロファイルを取得する実行ファイルの選択を求められます。これにより、関数のアドレスと関数名が相互参照されます。
  3. 開発機材に対して ping が実行され、割り当てられたバッファのサイズが PC に送信されます。

この接続プロセスが完了すると、プロファイルの「開始」ボタンと「切断」ボタンが有効になります。

別の実行ファイルのプロファイルを取得する場合、または別の開発機材に接続する場合は、「切断」ボタンをクリックします。

現在の接続情報は、アプリケーションの左下にあるステータスバーに表示されます。

「ファイル」グループ

これまでに保存したプロファイルを開くには「開く」ボタンを使用します。

「保存」ボタンを使用すると、現在のプロファイルが保存されます。

「スクリプト」ボタンを使用すると、プロファイリングを自動化できるスクリプトが読み込まれ、実行されます。

「ヘルプ」グループ

現在は、「マニュアル」ボタンをクリックすると、プロファイルの対象ゲームを迅速に設定する方法を説明した readme が表示されます。今後、「マニュアル」ボタンをクリックすると、このマニュアルが表示されるようになる予定です。

「単位」グループ

これらのボタンを使用して、各関数の実行に要した時間を表示するときの単位を制御できます。ここで選択した単位は、「関数」タブや「コールツリー」タブのようなプロファイルデータが表示されるタブや、「サンプルグラフ」タブのツールチップに反映されます。複数の単位を使い分けることにより、さまざまな方法による関数の分析と比較が可能となります。

パーセントモード

選択した時間の中で、特定の関数の実行時間が他のすべての関数の実行時間に対して占める比率（パーセント値）です。例えば、合計 400 のサンプルのうち、ある関数が 100 回サンプリングされていたとすると、この関数が実行時間の 25% を占有していたことになります。「パーセント」は、関数の実行時間を相対的に比較する場合に役立ちます。値の誤差を確認するには「エラー」ボタンを有効にします。

時間モード

各関数で費やされた時間の近似値です。この値はタイマーを使用した実測値ではなく、統計に基づいて計算された値です。1 つまたは複数のフレームが選択されているか、特定の期間が選択されている場合、この時間はそれらの選択されたフレームや期間の全体にわたる近似値となります。フレームも時間も選択していない場合は、プロファイルの全期間にわたる推測値が示されます。

「時間」の値は、サンプリングレートを非常に低くして、サンプリングに非常に長い時間をかければ、より正確になります。これは、プロファイラのオーバーヘッドに起因するものです。

1 つの関数が 1 フレームあたり平均でどれくらいの時間を費やしているかを確認するには平均値モードを使用します。

平均値モード

各関数で費やされた時間を「フレーム数で割った」近似値です。対象のコアにフレームが存在しない場合は、時間モードと等しくなります（平均値は算出されません）。対象のコアにフレームが存在していても、フレームが選択されていない場合は、実行時間をそのコアで記録されたフレームの総数で割った値になります。対象のコアに選択したフレームが存在している場合は、それらの選択されたフレームで費やされた時間を、現在選択されているフレームの数で割った時間になります。誤差を確認するには「エラー」ボタンを有効にします。

関数の実行で実際に要した時間を知るには「平均値」を使用します。関数が一定の時間またはフレームレートの範囲で実行されるようにする必要がある場合に効果的です。「平均値」は、サンプリングレートを非常に低くして、サンプリングに非常に長い時間をかければ、より正確になります。プロファイラのオーバーヘッドが増加するにつれて「平均値」の精度は低下します。したがって、この測定基準を使用する場合は、サンプリングレートを低く設定することがきわめて重要になります。また、個別のフレームを選択すると誤差が大きくなるので、精度と確度が低下します。誤差を確認するには「エラー」ボタンを有効にします（このエラーの計算では、プロファイラのオーバーヘッドが考慮されず、統計上の偏差のみの計算になります）。

注意：「平均値」を選択すると、「関数」タブの「合計」列と「セルフ」列の並べ替えが機能していないように見えることがあります。この原因は、さまざまなコアの関数の実行時間をさまざまなフレーム数で割っているにもかかわらず、並べ替えは重要度（絶対時間）に基づいていることにあります。それでも、対象コアのすべての関数に対して、並べ替えが正しく実行されています（コア別にフィルタすると、並べ替えが正しく実行されていることがわかります）。

サンプルモード

関数で実際に取得したサンプルの数です。上記の各単位を計算するための基本単位です。誤差を確認するには「エラー」ボタンを有効にします。

注意：サンプルモードでは、関数の呼び出し回数が示されるわけではありません。関数がサンプリングされた回数が示されます。

「エラー」切り替え

このボタンは、他の「単位」ボタンとは別のものです。「パーセント」、「時間」、「平均値」、「サンプル」の各値の横に誤差を表示するかどうかを指定します。誤差は、結果の精度を判断するために役立ちます。例えば、フレームを 1 つだけ選択した場合は、誤差がきわめて大きくなる傾向があります。その結果、「パーセント」や「時間」の値が正確であるように見えることもありますが（例えば 8.2%）、正確な精度を示しているのは誤差です（例えば Â±5.8%）。値の比較やパフォーマンスの評価ではこの誤差が重要になります。「エラー」ボタンの状態はプロファイラの設定に保存されるので、次回プロファイラを起動するときには、前回選択した状態が保持されます。

注意：この誤差の計算では、プロファイラのオーバーヘッドが考慮されず、統計上の偏差のみが計算されます。
