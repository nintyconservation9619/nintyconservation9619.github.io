3DEditor

親マテリアルのカテゴリー分けができるように

親マテリアルの追加ウィンドウ、変更ウィンドウでチーム設定で指定したカテゴリーから親マテリアルを選択できるようになりました。

関連リンク 以下は関連ページへのリンクです。

マテリアルプロパティウィンドウ 親マテリアルページ

親マテリアル編集ウィンドウ / 親マテリアルのモデルや親マテリアルの追加と編集ができます。


チーム設定で以下のように設定することで特定のフォルダー以下のファイルをひとつのカテゴリーとして定義することができます。

<FileIo>
  <ParentMaterialPath Name="BOY" >%NINTENDO_SDK_ROOT%\Samples\Sources\Applications\G3dDemo\Resources\boy</ParentMaterialPath>
  <ParentMaterialPath Name="TOWN" Recursive="true">%NINTENDO_SDK_ROOT%\Samples\Sources\Applications\G3dDemo\Resources\town\</ParentMaterialPath>
</FileIo>

ワイルドカードなど特殊な記法を利用することができます。

 関連リンク 以下は関連ページへのリンクです。

チーム設定ファイルフォーマット チーム設定ファイルに含まれる要素を説明します。 <ParentMaterialPath> の項目をご覧ください。


チーム設定の <SamplerNamePreset> の利用制限を解除

これまでチーム設定の <SamplerNamePreset> は <FollowDccSamplerNameRule> が true のときにしか利用できない制限がありましたが、false の場合も利用できるようになりました。

既存のチーム設定が <FollowDccSamplerNameRule> が false で <SamplerNamePreset> が定義されている場合は、NintendoSDK 4.0.0 以降のバージョンで <SamplerNamePreset> に記述されているサンプラー名がサンプラー名候補として表示されるよう動作が変わりますので、ご注意ください。

コマンドラインオプションの --preview_target(-pt) オプションの廃止

コマンドラインオプションの --preview_target(-pt) オプションを廃止しました。このオプションを指定している場合、 3DEditor 起動時にエラーダイアログが表示されます。代替方法はありませんので、指定しないように起動時オプションを修正してください。
