Event クラス

機能概要

Event クラスは、1bit （二値）のシグナル状態／非シグナル状態によってスレッド間で同期を行なうための機能です。

Event クラスは、nn::os::EventType オブジェクトを利用した イベント機能 をラッピングしたもので以下のメンバ関数を持ちます。API リファレンスはリンク先を参照して下さい。

名前 説明 備考
Event コンストラクタ InitializeEvent() 同等の初期化を実行
~Event デストラクタ FinalizeEvent() 同等の破棄を実行
Wait イベントの待機 WaitEvent() 同等のイベントの待機
TryWait イベントのポーリング TryWaitEvent() 同等のイベントのポーリング
TimedWait イベントの時限付き待機 TimedWaitEvent() 同等のイベントの時限付き待機
Signal イベントのシグナル化

SignalEvent() 同等のイベントのシグナル化
Clear イベントの非シグナル化 ClearEvent() 同等のイベントの非シグナル化

 

Event クラスの使用例

ここでは例として、Event クラスを使った StickyEvent クラスを実装します。

#include <nn/os.h>

class StickyEvent
{
public:
    StickyEvent() : m_Event(nn::os::EventClearMode_ManualClear) {}

    void Wait()
    {
        m_Event.Wait();
    }

    void Set()
    {
        m_Event.Signal();
    }

    bool IsSignaled()
    {
        return m_Event.TryWait();
    }

private:
    nn::os::Event   m_Event;
};

 

StickyEvent クラスは、１度セットしたらクリアすることの出来ない 1bit のイベントフラグを提供する機能です。インスタンス化したオブジェクトが破棄されるまで状態が残ります。

コンストラクタにて m_Event を初期化しています。クリアモードを Manual にすることで自動的にクリアされないようにします。

Wait() はイベントがセットされるまで待機する関数です。複数のスレッドが同時に待機することも出来ます。イベントが既にセットされている状態では待機せずに即座にリターンします。

Set() はイベントをセットする関数です。一度セットするとこのイベントはもうクリアすることは出来ません。複数のスレッドが待機している状態で Set() を発行した場合は、それら全てのスレッドが待ち解除されます。

IsSignaled() はイベントの状態を返す関数です。イベントが既にセットされていれば true を、そうでなければ false を返します。

このような StickyEvent 機能は、オブジェクトやモジュールの初期化を行なう場合などに使用されることがあります。
