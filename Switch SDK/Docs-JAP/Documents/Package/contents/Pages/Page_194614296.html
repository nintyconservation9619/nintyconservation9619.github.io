プログラマ向け変更点

NintendoSDK 1.0.0 で行われた NintendoWare Spy の変更点について説明します。

目次

  • Release ビルドで Spy ライブラリを有効化する際、ライブラリを再ビルドする必要がなくなりました
  • NN_SPY_ENABLE マクロを廃止します
  • （評価中） nn::spy::DebugModule の追加

Release ビルドで Spy ライブラリを有効化する際、ライブラリを再ビルドする必要がなくなりました

これまでは Spy ライブラリを Release ビルドでも利用したいときには spy_Config.h を書き換えたうえでライブラリを再ビルドする必要がありました。

今リリースより Spy ライブラリの再ビルドは不要になり、spy_Config.h を書き換えるだけで Spy ライブラリを有効化できます。

注意：

Spy ライブラリの機能を呼び出しているアプリケーションコードは、spy_Config.h を書き換えた後に再ビルドが必要です。

NN_SPY_ENABLE マクロを廃止します

NintendoSDK のコーディング規約にあわせ、Spy ライブラリが有効化されていることを示すマクロを NN_BUILD_CONFIG_SPY_ENABLED に変更しました。

従来の NN_SPY_ENABLE マクロは NintendoSDK 1.1.0 で廃止する予定です。

（評価中） nn::spy::DebugModule の追加

注意：

この機能は現在、評価中です。アプリに組み込んで利用できますが、予告なく変更・削除されることがあります。

Spy ライブラリの動作状態を記録する nn::spy::DebugModule を追加しました。
