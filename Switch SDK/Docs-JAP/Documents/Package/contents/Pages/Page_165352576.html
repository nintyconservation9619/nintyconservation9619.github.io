「他の人との自由なコミュニケーション」が制限されているかどうかを確認する機能

PCTL ライブラリは、アプリケーションが「他の人との自由なコミュニケーション」の機能をガイドラインに基づいて制限する際に、その制限のチェックを行う関数を提供します。

アプリケーションは、「他の人との自由なコミュニケーション」の機能をアプリケーションに組み込む際、その機能の入り口で都度制限を行わなければならないかどうかチェックし、制限が必要であれば該当機能を利用できないようにする必要があります。

以下のコードは、PCTL ライブラリの関数を利用して制限のチェックを行う例です。

// 「他の人との自由なコミュニケーション」機能を利用するシーンの入り口で処理される関数
void OnCommunicationSceneEntered()
{
    // PCTL ライブラリのチェック関数を呼び出す
    // (該当シーンに入る際はその入り口で必ず毎回呼び出す)
    if (!nn::pctl::CheckFreeCommunicationPermission(true))
    {
        // 戻り値が false である場合は制限がかけられていることを意味するため
        // 該当機能には入らずに前のシーンに戻すなどの対応を行う
        BackToPreviousScene();
    }
    else
    {
        // 機能を利用可能であるため、機能の本処理に進める
        SetSceneState(SceneState_Running);
    }
}
注意：

アプリケーションを中断しているときなどに機能が制限されるかどうかが変化する可能性があるため、チェック関数の戻り値をキャッシュしないでください。

CheckFreeCommunicationPermission 関数の呼び出し引数にて、第1引数に true を指定して制限されている旨の表示を行うようにした場合、関数の内部で画面表示の処理を行います。表示中は呼び出し元のスレッドをブロックしますのでご注意ください。

NX 環境においては、画面表示はエラービューアによる表示となります。

開発環境において機能制限を確認したい場合、以下のいずれかの方法で設定変更を行うことができます。

  • DevMenu にて「Initial Boot Menu」を「HomeMenu」として、本体設定から制限を行う
  • DevMenuCommand にて直接設定の変更を行う

なお、Windows 環境では設定変更を行うことができないため確認を行うことができません。
