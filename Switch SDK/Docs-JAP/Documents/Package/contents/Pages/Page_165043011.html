はじめに

NGC ライブラリでは、下記のプラットフォーム固有の機能を提供します。

  • NG ワードリストを参照して、文字列に NG ワードが含まれるかどうかの判定
  • 文字列に含まれる、数を表す文字の個数

NG ワードチェック

UGC をほかのユーザーとやり取りするアプリケーションで、不適切な言葉が含まれていないかを確認することができます。

NG ワードチェックの実装方法

NG ワードチェックは以下のいずれかの関数で行います。

  • nn::ngc::ProfanityFilter::CheckProfanityWords 関数
  • nn::ngc::ProfanityFilter::MaskProfanityWordsInText 関数

CheckProfanityWords 関数により問題があると判定されたテキストは、アプリケーション自身が次のような対策をして、通信相手に表示しないようにする必要があります。

  • テキストをアスタリスクなどの代替文字に置き換える
  • テキストを送信しない/受信しても破棄する

一方、 MaskProfanityWordsInText 関数は、入力テキストのうち問題があると判断した部分をアスタリスクに置き換えます。そのため、アプリケーション自身が上記のような対処をする必要はありません。

なお、Mii のニックネームの NG ワードチェックは、NG ワードが含まれる Mii は作成できないため、不要です。

ngc ライブラリは、本体の言語設定に応じて異なる NG ワードのパターンリストを使用し、NG ワードチェックを行います。ngc ライブラリが使用する既定のリストに加えて、任意の言語向けのリストでのチェックを行うことも可能です。

NG ワードチェックは送信側・受信側いずれで行っても構いません。送信側と受信側で本体の言語設定が異なる場合でも、送信側・受信側どちらで確認するかアプリケーションが自由に決めて構いません。
送信側・受信側の両方でチェックしても構いません。また、テキスト入力時にチェックを行っても構いません。その際、NG ワードチェックをしてから送信するまでに言語設定が変更される可能性がありますが、特に何か対応する必要はありません。

任天堂が提供するライブラリを使って NG ワードチェックを行ったうえで独自の NG ワードチェックを行っても構いません。

パターンリストは ngc ライブラリ外に保存されているものを動的に参照します。
そのため、 SDK のバージョンやそれにより初期化された開発機によってチェック内容が変わる可能性があります。
互換性の観点から特定の文字列に対する結果が常に不変であることを想定したコードの記述は行わないようにしてください。
