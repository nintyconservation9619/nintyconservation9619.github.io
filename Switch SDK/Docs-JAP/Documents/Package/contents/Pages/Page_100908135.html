カスタムシェーダ設定

ここではゲームアプリで用意したプログラマブルシェーダが利用できる「カスタムシェーダ設定」について説明します。

概要

「カスタムシェーダ設定」は、ゲームアプリ側で用意したシェーダをエミッタに設定できる機能です。
「カスタムシェーダ」を利用することによって、エフェクトのグラフィックス表現を拡張できます。
以下は「 設定項目」の概要です。

「カスタムシェーダ設定」からシェーダを選択すると「 シェーダコードのコンパイル」と「 シェーダコードの表示」が表示されます。
以下はグループ内の設定項目です。

シェーダコードのコンパイル

「 シェーダコードのコンパイル」では選択したシェーダコードのエラーチェックを行います。
ユーザーがシェーダを編集した場合ボタンを押すことで更新内容をチェックできます。
以下はボタンの説明です。

シェーダ0 プロジェクト設定ファイルに記述された「描画設定」で Def1 , Def2 を使用しない場合に使用されます。
シェーダ1 プロジェクト設定ファイルに記述された「描画設定」で Def1 を使用した場合に使用されます。
シェーダ2 プロジェクト設定ファイルに記述された「描画設定」で Def2 を使用した場合に使用されます。

シェーダコードの表示

「 シェーダコードの表示」では頂点シェーダやフラグメントシェーダの情報をテキストエディタで表示します。

プリプロセスを有効にする

シェーダのコードを表示する際に、プリプロセス前後どちらの結果を表示するかを「 チェックボックス」ON/OFF で切り替えます。
設定を ON にするとプリプロセス後の結果を表示します。
この機能は「オプション設定」の影響は受けず、UI での選択状態がそのまま反映されます。

頂点シェーダ、フラグメントシェーダの表示

各ボタンの実行によりシェーダの情報を表示します。
以下はボタンの種類です。

  頂点シェーダ フラグメントシェーダ
Code 頂点シェーダのソースコードを表示します。 フラグメントシェーダのソースコードを表示します。
Assemble 頂点シェーダアセンブリのソースコードを表示します。 フラグメントシェーダアセンブリのソースコードを表示します。

カスタムシェーダ設定 カスタムシェーダ全般の設定ができます。
「プリプロセスを有効にする」の設定を ON にするとプリプロセス後の結果をバイナリ化する際に反映させます。

設定例

以下の場所にあるサンプルカスタムシェーダについて説明しています。
\Samples\Sources\Applications\VfxCustomShader\Resources\Eset\CustomShader.eset

A Color カラーの変更 「Cstmシェーダ」タブの「 カラー」で設定した RGB 値をエミッタに乗算します。
B Position 描画位置の変更 「Cstmシェーダ」タブの「 加算する位置」の X Y Z 値をエミッタ位置に加算することでエフェクトの描画位置を調整します。
C Light ライティング シンプルなライティング処理を適用します。
D Normal 法線カラー ビューアでパーティクルの法線を色として確認できます。
E NormalMap 法線マップ

法線マップを適用します。法線マップ用テクスチャは「 テクスチャ」タブの「 テクスチャ1」に登録してください。

関連リンク EffectMaker における法線マップについては こちら を参照してください。
F SimpleIBL ライトマップ 「Cstmシェーダ」タブの「 IBL カラー0」空のカラー、「 IBL カラー1」を地面のカラーとしたシンプルなライトマップを適用します。
