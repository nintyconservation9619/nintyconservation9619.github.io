概要

AuthoringEditor は nmeta ファイルおよび nsp ファイルを確認・編集するための GUI ツールです。

作成した nmeta ファイルは AuthoringTool の入力として与えることで nsp ファイルを作成できます。

配置場所

このツールは NintendoSDK に含まれており、以下の場所に配置されています。

Tools/AuthoringEditor/AuthoringEditor.exe

使用方法

AuthoringEditor.exe を実行すると起動画面が表示されます。

行いたい操作を選んで、nmeta ファイルの作成・編集、または nsp ファイルの編集を行います。

制限

AuthoringEditor が含まれる SDK とはメジャーバージョンが異なる SDK でビルドされた nsp について

  • 正しく nsp の内容を確認・編集することはできません。
  • 確認・編集したい nsp をビルドした SDK に付属する AuthoringEditor を使用してください。

できること

nmeta ファイルの作成、編集

  • nmeta ファイルの書き方 に記載されている各設定項目について、GUI ツール上から記入・選択ができます。
  • 入力内容に誤りがある箇所には赤い × 印で表示されます。
  • 誤りが 1 点でも含まれる場合、nmeta ファイルの書き出しが行えません。

nsp ファイルの確認

  • 作成済みの nsp ファイルをAuthoringEditor 上で開くことができます。
  • nsp の ROM 識別ハッシュ や使用している SDK やミドルウェア、 nmeta に設定したタイトルの情報を確認できます。
    • ROM 識別ハッシュについては ビルド成果物の識別について を参考にしてください。
    • アプリケーションビルド時のファイルパスが同一でなければ、ROM 識別ハッシュは異なる値になります。詳細は ビルド成果物の識別について を参考にしてください。

nsp ファイルの編集

  • nmeta ファイルの書き方 に記載されている一部の設定項目については変更した上で、別の nsp ファイルとして書き出すことができます。

nsp / nmeta ファイルの比較

  • 2 つの nsp もしくは nmeta ファイルのパスを与えることで、両者の構成に差異があるかを確認できます。

ユーザーアカウントセーブデータ領域のサイズシミュレート

  • サイズシミュレーターを使うと、ディレクトリ数とファイル数から必要となるセーブデータバイト数を計算できます。
  • セーブデータバイト数は セーブデータ ページ内の「セーブデータの特性」に基づくものです。
  • サイズシミュレーターの計算結果にはマージンとして 2 ブロック分のバイト数が含まれます。
