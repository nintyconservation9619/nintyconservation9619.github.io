再エクスポート時の注意点

再度エクスポートする場合の注意点を説明します。ここではモデリング時とテクスチャー作成時のワークフローの注意点を説明します。

以下の場合はマージエクスポートの設定を行う必要があります。

  • 3DEditor でマテリアル等を編集保存したモデル（fmd）を、再度 Maya から Model Data（fmd）をエクスポートする場合。
  • テクスチャーをPhotoshopからフォーマット変更やコメントを追加して ftx 形式でエクスポートした後に、再度 Maya から Texture Data（ftx）をエクスポートする場合。

正しくマージの設定を行わないで再度エクスポートして上書きすると、一部の編集内容が上書きされて消失します。

マージとは

モデル中間ファイルの中には大きく分けると、モデルの形状や構造を表すデータと、マテリアル設定(ゲームのシェーダーにおける設定)の2種類のデータが含まれます。前者は DCC ツールで作成し、後者は DCC ツールからエクスポートした後に 3DEditor を使って設定します。
マージエクスポートを行うと、エクスポート先のモデル中間ファイルのマテリアル設定を維持しつつ、モデルの形状や構造のデータだけが上書きされます。

下表は fmd をエクスポートした場合の例です。

通常のエクスポート モデルの形状や構造のデータ 上書きされる マテリアル設定(シェーダー設定) 上書きされる
マージエクスポート モデルの形状や構造のデータ 上書きされる マテリアル設定(シェーダー設定) 保持される

マージ設定

マージ設定を説明します。

下図と表の様に設定します。マージするファイルがなければ通常のエクスポートが行われるため、最初からマージ設定 ON で作業することができます。

  項目 説明
A Merge fmd File

ON にすると、fmd ファイルをマージします。

3DEditor でのマテリアル等の編集状態を維持したままモデルを更新します。

（OFF の場合はシェーダー定義をはじめとする 3DEditor での編集が無効になります）
B Merge fmd File パス

パスが空欄の場合、Output Folder で指定したフォルダーの Output File Name で指定した名前の fmd にマージします。

（fmd ファイルのパスを指定すると、指定したパスの fmd ファイルをマージして Output Folder 指定パスに書き出します）
C Merge ftx File

フォーマット設定やコメント記述等の編集状態を維持したままテクスチャーを更新します。

（OFF の場合はフォーマットをはじめとする Photoshop での ftx エクスポート時の設定が無効になります）
注意：

Merge fmd File を ON にすると、Maya での一部の設定が（fmd）で無効になります。3DEditor で設定する必要があります。

  • 例１　テクスチャーの張り替え
  • 例２　テクスチャーの 繰り返し UV
  • 例３　テクスチャーのミラー、ラップ
  • 例４　マテリアルのカラー（シェーダーによって変わります）

マテリアルのマージはマテリアル名が同じ場合に行われます。Maya でマテリアル名を変更するとマージされません。

プロジェクトによっては「Maya から Texture Data をエクスポートしない」というワークフローもあります。
その場合は Merge ftx File は OFF のままで問題有りません。
