はじめに

本ドキュメントの目的

本ドキュメントでは、NintendoSDK に収録されている プレイデータ管理（PDM）ライブラリの機能について説明します。

参考：

プレイデータ管理ライブラリは NX Addon 5.0.0 以降で利用可能です。

対象読者

NintendoSDK を使ってアプリケーションを開発される方を対象としています。

プレイデータ管理ライブラリが提供する機能

プレイデータ管理ライブラリは、下記の機能を提供します。

  • 本体上で起動されたことのあるアプリケーションのプレイ情報の取得

アプリケーションのプレイ情報の取得

アプリケーションは nn::pdm::QueryApplicationPlayStatistics 関数を利用することで、アプリケーション ID を入力としてそのアプリケーションのプレイ情報を取得することができます。

プレイ情報には

  • 起動回数
  • プレイ時間

が含まれます。ここで得られる情報は、取得対象のアプリケーション内で使用されたユーザーの区別などを行わない、デバイス単位の情報となります。

この機能を使用するには、アプリケーションを作成する際の nmeta ファイルの「PlayLogQueryCapability」に「WhiteList」を設定し、取得対象のアプリケーションのアプリケーション ID を「PlayLogQueryableApplicationId」に記載する必要があります。

nmeta ファイルの書き方について、詳しくは  nmeta ファイルの書き方  を参照ください。

参考：

システムにおけるプレイ情報の記録容量は有限のため、古い記録は消える可能性があります。そのため、プレイ情報の起動回数とプレイ時間が必ず単調増加する、ということは保証されません。また、本体初期化などによってもこの情報は削除されている可能性があります。

一般的な利用では記録容量の上限に達しないことを想定していますが、単調増加を前提とするような処理は行わないでください。

制約事項

プレイデータ管理ライブラリには次の制約があります。

Windows 環境でのプレイ情報の取得

Windows 環境ではプレイ情報を取得することはできません。
