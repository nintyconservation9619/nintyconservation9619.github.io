パッチの作成方法

はじめに

本ページでは VSI からパッチを作成する際の手順について説明します。
作成されたパッチを使用する方法については、パッチのビルド、実行方法 を参照してください。

準備

あらかじめ、パッチが適用される前のバージョンの nsp を用意しておきます。
もし今回作成するパッチが二版目以降のパッチである場合は、直前の版のパッチファイルも用意しておきます。

作成

「プロジェクトプロパティ > オーサリング > Patch」の項目を以下のように設定し、ビルドすることでパッチファイルが作成されます。
出力されるファイルは、$(ターゲット名) .patch.nsp となります。

  • Generate Patch を「はい」に
  • Initial NSP file にパッチが適用される前のバージョンの nsp を指定
  • 二版目以降のパッチの場合は、Previous NSP file に、直前の .patch.nsp ファイルを指定
警告：

パッチビルド時には中間生成ファイルとして、今回の版の nsp が $(OutDir)$(ターゲット名) .nsp として出力されます。
そのため、Initial NSP file や Previous NSP file に $(OutDir)$(ターゲット名) .nsp と同じパスが指定されている場合、意図したパッチファイルが生成されません。
Initial NSP file や Previous NSP file に指定するファイルは別のパスに置いておく必要があります。

キャッシュ

パッチ生成時には、中間生成物を「プロジェクトプロパティ > オーサリング > パッチ > Cache Directory」で指定されたパスに出力します。
この中間生成物があることで、次の生成にかかる時間が短縮されます。
このパラメータのデフォルト値は「%USERPROFILE%\AppData\Local\Nintendo\Cache\Patch」です。
