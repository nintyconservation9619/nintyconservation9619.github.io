ウェーブサウンド

ウェーブサウンド

プロジェクトツリーで、ウェーブサウンドのノードを選択することで、サウンドセットタブにウェーブサウンドのリストを表示することができます。
この時表示されるリストは、サウンドセットタブ内を上下に分割して表示されます。
分割されたタブのうち、上をウェーブサウンドセットリストと呼び、下をウェーブサウンドリストと呼びます。

ウェーブサウンドセットの役割

ウェーブサウンドは、参照する波形データをメモリにロードして再生します。
ウェーブサウンドは主に効果音に利用されることを想定していますが、このような用途では非常に沢山のウェーブサウンドが必要となり、メモリへのロードを個別に行うのは効率的ではありません。
ウェーブサウンドセットは、この時一度にメモリにロードする波形データの組み合わせを定義します。
通常、効果音の波形データは、使われるシーン毎にまとめてメモリへロードされます。ウェーブサウンドセットを使って、同じシーンで使われる効果音（ウェーブサウンド）を効率的に集めることで、このような動作を前提としたサウンドデータの作成ができます。

ウェーブサウンドセットの新規作成と追加

ウェーブサウンドセットを新規作成または追加する場合は、サウンドセットタブのリスト領域を選択して、[プロジェクト]→[アイテムの追加]→[ウェーブサウンドセット…] を実行します。

ウェーブサウンドセットリストで、右クリックを行うと現れるコンテキストメニューによる操作でも、ウェーブサウンドセットを新規作成または追加することができます。

ウェーブサウンドセットリストの定義要素

ここではウェーブサウンドセットリストの各定義要素の意味を解説します。

名前

ウェーブサウンドセットの名前を指定します。
名前に使用できる文字には制限があり、以下の文字が使えます。

  • 先頭は、半角英字とアンダーバー [a-zA-Z_]
  • 先頭以外は、半角英数字とアンダーバー [a-zA-Z0-9_]

波形アーカイブ

波形アーカイブとの紐付けの設定（共有したい波形アーカイブの選択）をします。

  • 自動（共有）
    【自動】で波形アーカイブが生成されます。
    また、同じ埋め込みグループ内のほかのウェーブサウンドセットやバンクの波形アーカイブが【1 つにまとめられます】。
    （埋め込みグループ内で使用するすべての波形をまとめた波形アーカイブが 1 つ生成されます。）
    波形アーカイブは波形データの【個別ロード】はできず、【一括ロード】されます。
  • 自動（個別）
    【自動】で波形アーカイブを生成します。
    しかし、グループに属してもグループ内のほかのウェーブサウンドセットやバンクとは波形アーカイブが【1 つにはまとめられません】。
    この波形アーカイブは、ほかのどのバンクとも共有されません。【一括ロード】となります。
  • 波形アーカイブを明示指定
    波形アーカイブ内の波形ファイルの【個別ロード】【一括ロード】は、波形アーカイブリストで指定できます。
    しかし、【個別ロード】かつグループに【埋め込み】の場合は、【一括ロード】になります。

グループの「出力方式」と組み合わせた波形アーカイブの解説は「波形アーカイブの自動(共有, 個別)設定」も参照してください。

ウェーブサウンドの新規作成と追加

ウェーブサウンドを新規作成または追加する場合は、最初にウェーブサウンドセットリストでウェーブサウンドを追加したいウェーブサウンドリストを選択します。
この状態で [プロジェクト]→[アイテムの追加]→[ウェーブサウンド…] を実行します。

ウェーブサウンドリストで、右クリックを行うと現れるコンテキストメニューによる操作でも、ウェーブサウンドを新規作成または追加することができます。

ウェーブサウンドの編集

ウェーブサウンドの編集はウェーブサウンドリストで行います。
ウェーブサウンドリストの各列はウェーブサウンドに必要な定義要素で、サウンドアーカイブのウェーブサウンドのデータに必要な各要素に対応しています。

ウェーブサウンドリストの定義要素

ここではウェーブサウンドリストの各定義要素の意味を解説します。

再生

ボタンをクリックすると、プレビュープレイヤーに登録され、ウェーブサウンドがトリガ再生されます。

名前

ウェーブサウンドの名前を指定します。この名前は最終的に作成されたサウンドアーカイブを利用して、プログラムが Atk ライブラリ でウェーブサウンドを鳴らす時の名前になります。

名前に使用できる文字には制限があり、以下の文字が使えます。

  • 先頭は、半角英字とアンダーバー [a-zA-Z_]
  • 先頭以外は、半角英数字とアンダーバー [a-zA-Z0-9_]

圧縮方式

コンバータが参照波形ファイルをコンバートする際の圧縮方法を指定します。

  • ADPCM 波形ファイルを専用の ADPCM 形式にコンバートします。
  • PCM16 16bit の PCM データにコンバートします。

ファイルパス

ウェーブサウンドを作成する際に参照する波形ファイルを指定します。

ファイル名、およびパス名に使用できる文字には制限があり、以下の文字が使えます。

  • ファイル名の先頭は、半角英字とアンダーバー [a-zA-Z_]
  • ファイル名の先頭以外およびパス名は、半角英数字とアンダーバー [a-zA-Z0-9_]

再生時間

サウンドの再生時間（波形データ長の時間）を表示します。
表示単位は［分：秒．ミリ秒］で、「ミリ秒」は 3 桁目を四捨五入し、2 桁表示となります。

ループ付き波形データでは、波形そのものの長さでは無く、ループエンドまでの時間を表示します。
また、セルの先頭に "L" が表示されます。

ダウンサンプル

ダウンサンプルの有効、無効を指定します。

有効にすると元の波形ファイルを変更することなく「サンプルレート」に指定された値でダウンサンプルしてサウンドアーカイブに含めることができます。

サンプルレート

「ダウンサンプル」が無効の場合は、波形ファイルのサンプルレートを表示します。

「ダウンサンプル」が有効の場合は、ダウンサンプルする値を 2000 ～ 192000 Hz の範囲で指定します。

ビットレート

波形ファイルのビットレートを表示します。

量子化ビット数

波形ファイルの量子化ビット数を表示します。

チャンネル数

波形のチャンネル数が 1, 2 のときは Mono, Stereo と表示します。
チャンネル数が 3 以上の時は、数字をそのまま表示します。

ダウンミックス

ダウンミックスの有効、無効を指定します。

有効にすると元の波形ファイルを変更することなくステレオをモノラルにダウンミックスしてサウンドアーカイブに含めることができます。

データサイズ

圧縮変換後の波形データのサイズを表示します。
表示内容は、コンバートのタイミングで更新されます。

音量

ウェーブサウンドを Atk ライブラリ が鳴らす時のボリュームを 0 〜 255 の範囲で指定します。

サウンドデータの音量を 127 にすると、元データと等倍の音量 (1.0 倍) で再生されます。

ただし、音量は、いろいろな場面 (データやプログラム) で設定されます。
すべての設定値が掛け合わされた結果、元音量から比べて、0.0 倍 〜 2.0 倍にクランプされます。

詳しくは、サウンドシステムマニュアルの「サウンドパラメータ制御の動作」の章をご覧ください。

平均ラウドネス値

アイテムを選択し、コンテキストメニューの「平均ラウドネス値の計測」を実行することで平均ラウドネス値を表示することができます。

チャンネル優先度

ウェーブサウンドのチャンネルプライオリティを 0 〜 127 の範囲で指定します。
数値が大きい程優先順位が高く（強く）なります。

チャンネルプライオリティはウェーブサウンドが実際に発音する際に、AX に渡されるボイス優先順位となります。

プレイヤー優先度

ウェーブサウンドのプレイヤープライオリティを 0 〜 127 の範囲で指定します。
数値が大きい程優先順位が高く（強く）なります。

プレイヤープライオリティは、プレイヤーのサウンドリミットの値を超えて再生しようとしたときや、サウンドやトラックなどについてシステムで設定した最大数を越えて再生しようとしたときに、再生を行う（止める）サウンドを選ぶ時に利用されます。

詳しくはサウンドシステムマニュアルをご覧ください。

プレイヤー

ウェーブサウンドが発音する際に使用するプレイヤーを指定します。

アクタープレイヤー

再生するアクタープレイヤーの 4 種類の番号を 0 〜 3 の値で指定します。

アクタープレイヤーは、実世界の音源、具体的にはボイスに対する声帯に相当し、アクタープレイヤーごとに同時に再生できるサウンド数を制限することができます。
プレイヤーと同様に、設定したサウンド数を超えた場合には、各サウンドに設定されたプレイヤープライオリティを比較して、最も優先度が低いサウンドが停止します。
各アクタープレイヤーのサウンド制限数の初期値は、0 番は無制限で、1 〜 3 番は 1 に設定されています。

アクタープレイヤーは、主に 3D サウンドのアクターと組み合わせて利用します。
例としては、同じサウンドを、3D サウンドの複数のアクターで同時に再生する可能性がある状況において、各アクター単位で、それらのサウンドの再生を制御したい場合などに有効です。

単一再生

単一再生の制御モードを指定します。

単一再生を利用すると、あるサウンドを同じプレイヤーで同時に 1 つだけ再生するように制御することができます。その制御方法として、あるサウンドを再生中にもう一度同じサウンドを再生したときの制御モードを以下の 5 つから指定します。

  • 使用しない (デフォルト)
    再生中のサウンドと後から再生したサウンドを共に再生します。
  • 後着優先
    再生中のサウンドを停止し、後から再生したサウンドを再生します。
  • 時間内後着優先
    再生中のサウンドの再生開始からの経過時間が「単一再生有効時間」カラムで設定した時間以内ならば、後着優先と同じ制御を行います。時間外ならば、単一再生を使用しない場合と同じ制御を行います。
  • 先着優先
    再生中のサウンドを再生し続け、後から再生しようとしたサウンドの再生を失敗させます。
  • 時間内先着優先
    再生中のサウンドの再生開始からの経過時間が「単一再生有効時間」カラムで設定した時間以内ならば、先着優先と同じ制御を行います。時間外ならば、単一再生を使用しない場合と同じ制御を行います。

単一再生有効時間

単一再生に「時間内後着優先」か「時間内先着優先」を指定した場合に、単一再生の制御を有効化する時間をミリ秒で指定します。

パン

ウェーブサウンドを Atk ライブラリ が鳴らす時のパンを 0 〜 127 の範囲で指定します。

Sパン

ウェーブサウンドを SoundLib が鳴らす時のサラウンドパンを 0 〜 127 の範囲で指定します。

ピッチ

ウェーブサウンドを Atk ライブラリ が鳴らす時のピッチを倍率で指定します。

入力可能な数値の下限は 0.00001 です。 また、一括編集で 0.00001 未満の数値になる時は、0.00001 に修正されます。

LPF

ローパスフィルタのカットオフ周波数を 0 〜 64 の範囲で指定します。
64 をセンター（フィルタがかかっていない）として 0 に近づくほどフィルタがかかります。

※ 数値はセンターから下方向（フィルタのかかる操作）のみ有効です。

また、 LPF と biquad フィルタは併用することができ、併用した場合は 2 つのフィルタの効果が重ね掛けされます。

biquad フィルタの種類

biquad フィルタの種類を指定します。

  • 使用しない (デフォルト値)
  • LPF (ローパスフィルタ)
  • HPF (ハイパスフィルタ)
  • BPF (中心周波数 512Hz) (バンドパスフィルタ)
  • BPF (中心周波数 1024Hz)
  • BPF (中心周波数 2048Hz)
  • ユーザー定義のフィルタ (64 種)

biquad フィルタ値

biquad フィルタのかかり具合を 0 〜 127 の範囲で指定します。
0 の時は効果がなく、127 の時に効果が最大になります。

また、 biquad フィルタ と LPF は併用することができ、併用した場合は 2 つのフィルタの効果が重ね掛けされます。

Env R

リリースエンベロープの、リリースタイムを 0 〜 127 の範囲で指定します。

センド AUX_A

補助バス A に対するセンド量を 0 〜 127 の範囲で指定します。

センド AUX_B

補助バス B に対するセンド量を 0 〜 127 の範囲で指定します。

センド AUX_C

補助バス C に対するセンド量を 0 〜 127 の範囲で指定します。

センド MAIN

メインバスのセンド量を 0 〜 127 の範囲で指定します。

各補助バスとメインバスに対するセンド量は、以下に示す概念図に従ってミキシングされます。
センド MAIN を 0 にすることで、補助バス（エフェクト）成分のみのサウンドを作成できます。

 

リリース時優先度固定

チェックを入れたサウンドは、リリース中も、発音プライオリティが下がらなくなります。

3D Vol

3D サウンドによるボリュームの制御を受けるかどうかを設定します。

このチェックを外すことにより、サウンドに対する 3D サウンドのボリューム制御を無効にすることができます。

3D Pan

3D サウンドによるパンの制御を受けるかどうかを設定します。

このチェックを外すことにより、サウンドに対する 3D サウンドのパン制御を無効にすることができます。

3D SPan

3D サウンドによるサラウンドパンの制御を受けるかどうかを設定します。

このチェックを外すことにより、サウンドに対する 3D サウンドのサラウンドパン制御を無効にすることができます。

3D Pri

3D サウンドによるプレイヤープライオリティの制御を受けるかどうかを設定します。

このチェックを外すことにより、サウンドに対する 3D サウンドのプレイヤープライオリティ制御を無効にすることができます。

3D Filter

3D サウンドによる Biquad フィルタの制御を受けるかどうかを設定します。

このチェックを入れることにより、サウンドに対する 3D サウンドの Biquad フィルタ制御を有効にすることができます。

3D 減衰カーブ

3D サウンドによって行われる、ボリューム減衰の方式を選択します。

3D 減衰率

3D サウンドによって行われる、ボリューム減衰の減衰率を設定します。

3D ドップラーファクター

3D サウンドによって行われる、ドップラー効果を設定します。
ドップラー効果による音程の変化度合いを 0 〜 255 の範囲で設定します。

デフォルト値は 0 で、値が大きくなると音程の変化が急になり、0 の時は全く変化しません。

パンモード

ステレオの波形に対する、パンの動作モードを、Dual と Balance から選択します。

Balance は左右の音量バランスを調整するモード、Dual は左右を個別のモノラルボイスとして、パンを設定するモードです。

パンモードの設定と動作についての詳細は、サウンドシステムマニュアルの「パンの動作」のセクションをご覧ください。

パンカーブ

パンの動作において、左右チャンネルの音量を決定する減衰カーブの種類を設定します。

パンカーブの設定と動作についての詳細は、サウンドシステムマニュアルの「パンの動作」のセクションをご覧ください。

ユーザーパラメータ

ユーザーが自由に利用可能な 32bit の数値を設定します。

[プロジェクト]→[プロジェクトの設定] からプロジェクト設定ダイアログを開き、「ユーザーパラメータ」タブで各ビットの構成を設定できます。
