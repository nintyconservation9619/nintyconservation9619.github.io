float カラー出力の使い方

はじめに

本ドキュメントではレイアウトデータで float カラーを扱う方法について説明します。

通常 LayoutEditor 上では 0 - 255 の値でカラーを設定しますが、float カラー出力を有効にすることでカラーの値を float で設定することができるようになります。
float カラーではこれまで 0 - 255 で設定した値が 0.0 - 1.0 に対応しており、また 1.0 を超えた値を設定することも可能です。

機能の有効化

この機能はデフォルトでは無効となっているため、使用するためには以下の設定が必要となります。

LayoutEditor で float カラーを有効にする

LayoutEditor で float カラー出力を有効にするにはプロジェクト設定ファイルの以下の項目を設定します。

<FloatColorWriteEnabled>true</FloatColorWriteEnabled>

プロジェクト設定ファイルに関しては「プロジェクト設定ファイルフォーマット」を参照してください。

LayoutEditor プレビューでのグレアエフェクト

LayoutEditor のビューアー(LayoutViewer)でプレビューする際にグレアエフェクトがかかる機能が含まれています。

float カラー機能を利用して LayoutViewer へグレアエフェクトを実装しているためで、float カラーを有効にすればグレアが表示されるわけではありません。

float カラーで出力されたデータを活用するためには、アプリケーション側でデータを利用したエフェクトなどの実装を行う必要があります。

 アプリケーションでの float カラー取り扱い時の注意点

float カラーデータをアプリケーションコードで内で扱う際は、カラーバッファのフォーマットを ImageFormat_R16_G16_B16_A16_Float にするなど、 1.0 を超えるカラー値を適切に扱えるフォーマットを選択する必要があります。

機能使用上の注意点

内部データの表現形式が変更されるため機能を有効化した状態で保存される中間ファイルデータについては十分に注意する必要があります。
機能を有効化した状態で float カラーを使用したデータを出力すると、これまで内部で 0 - 255 で保持していたデータを 0.0 - 1.0(もしくはそれ以上) の浮動小数点数で保存します。
データ変換時の誤差は発生しないためデータとしては同じ意味のものが出力されますが、保存されるタグが既存のタグとは違うものになるため float カラーに対応していないバージョンの LayoutEditor では正しく開くことができなくなります。
