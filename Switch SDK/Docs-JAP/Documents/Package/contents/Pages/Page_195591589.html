ネットワーク接続の動作ポリシー

はじめに

ここでは Nintendo Switch のネットワーク接続の動作ポリシーについて説明します。

アプリケーションは、ネットワーク利用要求を提出する以外の手段でデバイスのネットワーク接続に関する挙動に影響を及ぼすことができません。したがってアプリケーションは、ネットワーク利用要求を提出していない状態でのネットワーク接続の状態や、利用要求の提出時に暗黙的におこなわれるネットワーク接続の選択等について、特定の期待を持つべきではありません。ここでの説明は、システムの挙動を理解して、アプリケーションの検証時に適切な検証パターンを構築できるようにすることを目的におこなわれています。

ネットワーク接続の選択ポリシーはシステムのアップデートによっても変更される可能性があります。

基本ポリシー

自律接続

システムは原則として、いくつかの自律的に動作するネットワークサービスのために、随時ネットワーク接続を確立しようと試みます。これにより、アプリケーションが明示的にネットワーク利用要求を提出しなくても、ネットワーク接続が利用できる状況では、デバイスとしてはネットワーク接続を確立した状態になります。

ただし、新しい無線アクセスポイントの発見には、状況によって数十秒から数分程度の時間を要することがあります。

インターネット通信の接続先自動選択

システムがネットワーク接続先を自動選択する際、原則として以下の順で優先的にポリシーが適用されます。

  1. 利用中のネットワーク接続優先
    • すでに確立されている利用中のネットワーク接続があれば、それより優先度の高いネットワークがあっても切り替えをおこないません。
    • HOME メニューの「インターネット接続」内で手動で接続先を選択した場合、その接続が失われるまで選択した接続が維持されるので、その後自動選択による切り替わりは行われません。
  2. 有線優先
    • 利用可能な有線 LAN のネットワークと無線 LAN のネットワークがあった場合、有線 LAN を優先的に利用します。
  3. 最近使用した接続設定優先
    • 最後に使用したのが最近である接続設定ほど優先的に使用されます。
    • この結果として、利用可能なネットワークが複数ある環境でも、明示的な切り替えをおこなわないかぎり、常に一方が選択されます。
  4. 電波強度の強いアクセスポイント優先
    • 同じ接続設定が適用可能な無線アクセスポイントが複数ある場合、デバイスでの受信電波強度が最も高いアクセスポイントが選択されます。

例外的に、無線 LAN 接続の利用中に新しく利用可能な有線 LAN の接続を発見した場合は、アプリケーションなどが明示的にネットワーク接続を利用していない状態であれば、そのとき利用中の無線 LAN の接続を切断して有線 LAN の接続へ切り替えを試みます。明示的にネットワーク接続を利用していた場合、その利用が終了した後に有線 LAN の接続への切り替えがおこなわれます。

インターネット通信とローカル通信の共存

インターネット通信とローカル通信は共存しません。有線 LAN 接続を利用してインターネット通信をしている場合でも、ローカル通信の開始によってインターネット通信は切断されます。

個別シナリオにおける挙動

USB Ethernet アダプターや LAN ケーブルの接続（ドックへの接続）

無線 LAN の利用中に USB Ethernet アダプターや LAN ケーブルの接続によって Ethernet がリンクアップした場合、以下の挙動を取ります。

  • アプリケーションなどが明示的にネットワーク利用要求を提出していない場合、無線 LAN との接続を切断し、有線 LAN の利用に切り替えをおこないます。
  • アプリケーションなどが明示的にネットワーク利用要求を提出している場合、利用中の無線 LAN との接続を維持します。
    • ただし、明示的な利用要求が取り下げられたタイミングで無線 LAN との接続を切断し、有線 LAN の利用に切り替えをおこないます。

Ethernet のリンクアップを伴わない USB Ethernet アダプターや LAN ケーブルの接続は、ローカル通信を含むネットワーク接続に影響を与えません。

USB Ethernet アダプターが接続されたドックへの接続は、 USB Ethernet アダプターの接続と等価です。

USB Ethernet アダプターや LAN ケーブルの取り外し（ドックからの取り外し）

有線 LAN 利用中に、利用中の USB Ethernet アダプターや LAN ケーブルを取り外した場合、ネットワーク接続は切断されます。その後、無線 LAN の利用可能な接続設定と接続先があれば、自律動作によって自動で無線でネットワーク接続が再確立されますが、切断前に提出していたネットワーク利用要求が自動で再受理されることはありません。

利用されていない USB Ethernet アダプターの取り外しは、ローカル通信を含むネットワーク接続に影響を与えません。

USB Ethernet アダプターが接続されたドックからの取り外しは、 USB Ethernet アダプターの取り外しと等価です。

複数の USB Ethernet アダプターの接続

有効な USB Ethernet アダプターが1つもない状態で接続された USB Ethernet アダプターが有効になります。

  • 複数の USB Ethernet アダプターを接続した場合、先に接続した USB Ethernet アダプターが使用されます。
  • 複数の USB Ethernet アダプターを接続した状態で、有効な USB Ethernet アダプターを取り外した場合、無効だったほかの USB Ethernet アダプターが有効になることはありません。
  • 無効な USB Ethernet アダプターしか接続されていない状態で、新しく USB Ethernet アダプターを接続した場合、その USB Ethernet アダプターが有効になります。

起動時から複数の USB Ethernet アダプターが接続されている場合、有効になる USB Ethernet アダプターは予測できません。

有効になる USB Ethernet アダプターに、 LAN ケーブルの接続状況は影響を与えません。
