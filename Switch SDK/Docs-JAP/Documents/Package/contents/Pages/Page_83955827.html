StartSound が成功しているのに、サウンドが鳴らない

nn::atk::SoundArchivePlayer::StartSound 関数が成功しているのにサウンドが鳴らない場合は、以下のケースを疑ってみてください。

  • nn::atk::SoundArchivePlayer::Update 関数が呼ばれていない

メモリプールのデタッチが終了しない

ループするサウンドを再生したまま、そのサウンドをロードしたメモリを含む領域のメモリプールをデタッチしようとしてもデタッチに失敗します。
まずは、サウンドを停止させずにメモリプールのデタッチを試みていないかをご確認ください。

メモリプール管理が必要なリソースについては、下記のドキュメントもご確認ください。

  • AUDIO ライブラリのメモリプールのドキュメント
  • プログラミングガイドのメモリー管理のドキュメント

nn::atk::viewer ライブラリを組み込んだ PC 版アプリで SoundMaker と接続できない

  • Generic 版 viewer ライブラリを組み込んだアプリと、NX 版 viewer ライブラリを組み込んだアプリで SoundMaker の接続設定が異なります。
    NX 版 viewer ライブラリを組み込んだアプリの場合は、ツールバーの [接続先] が PC アプリケーション、[プラットフォーム] が NX に設定されている事をご確認ください。
