プロジェクト設定ファイル

概要

プロジェクト設定ファイルは、プロジェクト全体で共通する EffectMaker の設定を記述するファイルで、プログラマが記述・管理するファイルです。
アプリケーションで使用するエフェクトの各種設定を読み込むための起点となる設定ファイルです。
EffectMaker でのデータ編集時、バイナリコンバート時の両方で使用します。

設定できる内容は大まかには以下の通りです。

  • カスタム機能設定
    • カスタムシェーダ設定ファイルへのパス
    • カスタムアクション設定ファイルへのパス
    • エミッタ拡張パラメータ設定ファイルへのパス
  • UI ラベル設定
    • エミッタセットユーザーデータ
    • カスタムフィールド
  • シェーダコンパイル設定
    • 追加のプリプロセッサ
  • 描画パス設定
  • アプリケーションの各種描画設定
  • 各種ファイルパス
    • プリセットフォルダへのパス
    • バージョン管理ツールとの連携設定
  • その他
    • ドキュメントへのパス
    • 標準重力設定
    • プレビュー用バイナリの名前をファイルパスに変更

各設定項目の詳細については 設定項目 をご覧ください。

使用方法

EffectMaker のオプションダイアログから設定

EffectMaker のオプションダイアログの「プロジェクト設定ファイルパス」から設定できます。
 詳細は EffectMaker のドキュメントの 【プロジェクト設定】 をご覧ください。

オプション上で、

  • リニアワークフロー 
  • カスタムシェーダ
  • カスタムアクション

上記の設定を変更しても、EffectMaker 終了時にプロジェクト設定ファイルは更新されません。
これらの設定を変更する場合は、設定ファイル自体を編集してください。

参考：

EffectMaker のオプションダイアログの「プロジェクト設定」にて、現在設定されてる内容のプロジェクト設定ファイルをエクスポートできます。
プロジェクト設定ファイルのひな型の生成などにご利用ください。

EffectMaker の起動時引数から設定

--team-setting-file オプションを使用して、EffectMaker 起動時にプロジェクト設定ファイルを渡すことができます。

EffectMaker.exe --team-setting-file ProjectConfig.xml

詳細については EffectMaker の便利機能 のページをご覧ください。

バイナリコンバートの引数から設定

バイナリコンバート時においても、 --team-setting-file オプションでプロジェクト設定ファイルの指定が可能です。

EffectMaker.exe --team-setting-file ProjectConfig.xml --convert-list ConvertList.xml --output EffectBinary.ptcl --api-type Gl --code-type Source --tile-mode Linear

バイナリコンバートの詳細については バイナリコンバータ をご覧ください。

記述方法

ファイル形式

ファイル形式は以下の形式を満たす必要があります。

  • XML 形式
  • 文字コードは UTF-8
参考：

各種ファイルパスの記述には、

  • 絶対パス
  • プロジェクト設定ファイルからの相対パス

が使用できます。

また、環境変数も利用可能です。バッチファイルと同様の記述方法で、

%APPLICATION_ROOT%

の書式で記述してください。

シェーダコンパイル設定 追加のプリプロセッサ

シェーダコンパイル時に与えられる追加のインクルードディレクトリを指定します。

ここで定義したプリプロセッサは、シェーダコンパイル時に渡されるため、コンパイルされるすべてのシェーダに対して適用されます。

</AdditionalShaderPreprocessorDefinitions>

アプリケーションの各種描画設定

 リニアワークフロー 
LinearMode
 リニアモードのオンオフが指定できます。
リニアモードの詳細については リニア空間 のページもご覧ください。		 
<LinearMode>false</LinearMode>
テクスチャ原点設定 
TextureOriginMode

テクスチャ原点位置を設定します。設定可能な値は以下の通りです。

LowerLeft
 テクスチャ座標原点は左下 
UpperLeft
 テクスチャ座標原点は左上

設定によりシェーダ内で利用されるプロジェクトション変換行列が切り替わります。
アプリの設定に合わせて正しく設定しない場合、屈折シェーダ等のフレームバッファを利用する描画が崩れます。

参考：

NXAddon 4.1 以前で利用していた AdditionalShaderPreprocessorDefinitions を利用して原点を切り替えは、
互換性の観点から当面維持されますが、TextureOriginMode で指定する新しい方法への切り替えを推奨します。


関連リンク

  • フレームバッファ・デプスバッファの UV 原点位置の切り替え 
<TextureOriginMode>LowerLeft</TextureOriginMode>
デプスモード 
DepthMode

デプスモードを設定します。設定可能な値は以下の通りです。

NearIsMinusW

ニアクリップ面の Z 位置が -W
NearIsZero

ニアクリップ面の Z 位置が 0

設定によりシェーダ内で利用されるプロジェクトション変換行列が切り替わります。
アプリの設定に合わせて正しく設定しない場合、ソフトパーティクル等のデプスバッファテクスチャを利用する描画が崩れます。

 
<DepthMode>NearIsMinusW</DepthMode>
各種ファイルパス
  • プリセットフォルダへのパス
  • バージョン管理ツールとの連携設定
 
FilePaths

ユーザー指定の各種ファイルパスを指定します。

  • プリセットフォルダパス
    • プリセットフォルダのディレクトリへのパスです。

以下は、バージョン管理ツールなどと連携するためのフック処理のためのファイルパスです。

  • ファイル保存前コマンド
    • ファイル保存前に呼び出すバッチファイルへのパスです
  • ファイル保存後コマンド
    • ファイル保存後に呼び出すバッチファイルへのパスです
  • ファイル削除前コマンド
    • ファイル削除前に呼び出すバッチファイルへのパスです
  • ファイル削除後コマンド
    • ファイル削除後に呼び出すバッチファイルへのパスです
  • ファイルオープン前コマンドEffectMakerでの指定方法
    • ファイルを開く前に呼び出すバッチファイルへのパスです
 
<FilePaths>
  <!-- プリセットフォルダへのパス -->
  <PresetFolderPath>%PRESET_DIRECTORY_PATH%</PresetFolderPath>
  <!-- バージョン管理ツールとの連携設定 -->
  <PreSaveCommandFilePath>%EFFECT_HOOK_SCRIPT%\PreSaveCommand.bat</PreSaveCommandFilePath>
  <PostSaveCommandFilePath>%EFFECT_HOOK_SCRIPT%\PostSaveCommand.bat</PostSaveCommandFilePath>
  <PreDeleteCommandFilePath>%EFFECT_HOOK_SCRIPT%\PreDeleteCommand.bat</PreDeleteCommandFilePath>
  <PostDeleteCommandFilePath>%EFFECT_HOOK_SCRIPT%\PostDeleteCommand.bat</PostDeleteCommandFilePath>
  <PreOpenCommandFilePath>%EFFECT_HOOK_SCRIPT%\PreOpenCommand.bat</PreOpenCommandFilePath>
</FilePaths>
その他 ドキュメントパス DocumentPath ドキュメント連携用のパスを設定できます。
設定値が存在しない場合は、自動で NintendoSDK パッケージの、Documents/index.html の場所を参照しますが、それ以外の場所にドキュメントを配置した場合に、そのパスを指定することで連携先のドキュメントを切り替えることができます。		 
<DocumentPath>%NINTENDO_SDK_ROOT%\Documents\index.html</DocumentPath>
