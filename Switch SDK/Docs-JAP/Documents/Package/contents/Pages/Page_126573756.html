リソース

概要

リソースは、あらかじめツールにより作成されたバイナリーイメージを解釈するための機能を提供します。

GFX におけるリソースはプリミティブな機能ではなく、たとえばシェーダーリソースは内部にシェーダーの情報をもっているように、GFX のプリミティブな情報やオブジェクトを使うことによってさまざまな機能を提供しています。

特にバイナリーイメージそのものを表すトップレベルのクラスのことをリソースファイルクラスと呼ぶことにします。

構成

GFX のリソースに使われるバイナリーイメージは、必ず nn::util::BinaryFileHeader で始まっています。
よって、sizeof( nn::util::BinaryFileHeader ) だけを読み込み、 nn::util::BinaryFileHeader としてファイルのアライメントやサイズなどを確認することが可能です。

GFX のリソースクラスは、アクセサーとデータにより構成されています。
アクセサーは、対応するデータをテンプレート引数にとる nn::util::AccessorBase を継承しています。

名前

GFX においてリソースを扱うクラスは "Res" によってプレフィックスされます。
また、 リソースファイルクラスはさらに "File" のサフィックスをもちます。

共通機能

ここでは GFX のすべてのリソースファイルクラスに共通の機能を説明します。

シグネチャー

リソースファイルクラスはバイナリーイメージの先頭 8 バイトを表す静的変数 Signature をもっています。

バージョン

リソースファイルクラスはバージョンを表す静的変数 MajorVersion, MinorVersion, MicroVerion をもっています。

MajorVersion が異なるリソースファイルや、MinorVersion が大きいリソースファイルを使うことはできません。

バイナリーの有効性の判定

バイナリーイメージの先頭へのポインターを IsValid に渡すことによって、対象のバイナリーのバージョンなどをチェックし、リソースとして利用できるかどうかを判定できます。

リソースファイルクラスへのキャスト

バイナリーイメージの先頭へのポインタを ResCast に渡すことによってリソースファイルクラスへのポインターを取得することができます。

リソースファイルクラスへのキャスト時、内部的にポインターメンバーをオフセット値から実際のポインターに解決するためのリロケーションが行われます。

バイナリーヘッダーの取得

GetBinaryFileHeader により、nn::util::BinaryFileHeader へのポインターを取得することができます。
ファイルのアライメントやサイズの取得や、リロケーションの手動制御などに利用することができます。
