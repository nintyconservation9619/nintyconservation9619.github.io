カスタムアクション

「 カスタムアクション」はゲームプログラマがエフェクトシステムに登録した複数のアクションから １ つを選択し、実行する機能です。
ゲームアプリケーションの機能やワークフローで利用できます。
以下はグループ内設定項目の一覧です。

アクションID ゲームプログラマがエフェクトシステムに登録した複数のアクションから １ つを選択して実行できます。

ビット

「 ビット」は、カスタムフィールドをフラグで設定できます。
以下はグループ内設定項目の一覧です。

ビット [0] ～ [15] 16 個のフラグを「 チェックボックス」ON/OFF で設定します。

「 数値」は、カスタムフィールドをスライダで設定できます。

「アクション」とは、ゲームプログラマがゲームアプリケーションで独自に用意した機能群です。
ゲームプログラマがカスタムアクションフォルダ内にあらかじめ設定ファイルを用意することで使用できます。
アクションを使用することにより、例えばゲーム中のコリジョンと衝突判定を行い、衝突時のパーティクルの挙動を制御できます。

「カスタムアクション」は CPU で処理される為、ノードを作成すると「 CPUエミッタ」に切り替わります。
関連リンク「 CPUエミッタ」についてはこちらを参照してください。

設定例

以下に、ゲームアプリケーションでアクションとして「衝突判定」を実装したと仮定したときのカスタムアクションの設定例です。
「衝突判定」は、ゲーム中の地形やオブジェクトのコリジョンと接触したときの挙動を定義したアクションです。

UI の 例 ゲームにおける使用イメージ

A アクションID = 「衝突判定を使用」 エミッタは、アクション「衝突判定」を利用します。
B マップと衝突判定する = ON
オブジェクトと衝突判定する = OFF		 イメージのように、パーティクルはマップ（地形）と衝突します。
オブジェクト（キャラクタ）とは衝突しません。
C 反射係数 = 0.8 パーティクルはマップとの衝突により、 0.8 の速度で反射します。
