3dExcelToShaderAnnotation
参考：

このツールは試験中のツールです。今後予告なく動作を変更する場合があります。

3dExcelToShaderAnnotation(%NINTENDO_SDK_ROOT%\Tools\Graphics\3dTools\3dExcelToShaderAnnotation.exe)の使用方法を説明します。

概要

Excel から GLSL のソースコードとして使用できるシェーダーアノテーションを作成します。

コマンドラインオプション

3dExcelToShaderAnnotation.exe [--version] [-h] [-s] [ファイル]

--args-file ""

引数ファイルを指定します。
引数ファイルに記述した内容を、引数として指定できます。 
引数ファイルは複数行に渡って各オプションを記述できます。 
行頭に # がある行はコメントとして扱います。 
オプション値指定の "" によるエスケープに改行を含めることはできません。

--version

バージョン情報を表示します。

-h --help

ヘルプを表示します。

-s --silent

処理中のメッセージ表示を抑制します。

ファイル

アノテーションに変換する Excel ファイル(.xlsx)、もしくは Excel ファイルに変換するシェーダー定義(.fsd)ファイルを指定します。

利用例

3dExcelToShaderAnnotation の利用例です。

3dExcelToShaderAnnotation.exe Input.xlsx

Excel の書式

先頭のワークシート

一番先頭のワークシートは各シェーダーパラメーターのアノテーションを生成するための共通情報(ページやグループなど)を定義するワークシートです。具体的には以下の項目を定義します。

項目 省略時の扱い 説明
ソースコードのファイル名 省略不可 生成するソースコードのファイル名です。

ソースコードの出力フォルダーパス

 省略不可

生成するソースコードの出力先フォルダーパスです。%VAR% のように % で環境変数名を囲むと環境変数を利用することができます。相対パスを指定した場合は Excel ファイルからの相対パスとして扱われます。
オプション変数のユニフォームブロック用構造体の宣言書式 layout(std140) uniform Options ソースコード上でのオプション変数のユニフォームブロック用構造体の宣言書式です。ここに書いた書式がそのまま代入されます。
オプション変数のユニフォームブロック ID option

オプション変数のユニフォームブロック ID です。ユニフォームブロック の id に対応します。

マテリアルのユニフォームブロック用構造体の宣言書式

 layout(std140) uniform Material ソースコード上でのマテリアルのユニフォームブロック用構造体の宣言書式です。ここに書いた書式がそのまま代入されます。
マテリアルのユニフォームブロック ID material

マテリアルのユニフォームブロック ID です。ユニフォームブロック の id に対応します。 

ページの定義

(ページ名、ページラベル、ページの並び順)

 ページ名が省略された場合は無視、その他は空文字 ページ名は ページ の page、ページラベルは label に対応します。ページの並び順は記述した順番になります。

グループの定義

(グループ名、グループラベル、グループの表示条件式、グループの並び順)

 グループ名が省略された場合は無視、その他は空文字 グループ名は グループ の group、グループラベルは label、グループの表示条件式は condition に対応します。グループの並び順は記述した順番になります。

二番目以降のワークシート

二番目以降のワークシートは各ページにおける各シェーダーパラメーターのアノテーションの定義をするワークシートです。ひとつのワークシートがひとつのページに対応します。ワークシート名はページ名と同じ名前にします。ページがひとつもない場合は一番先頭のワークシートのグループ定義の下に記述することもできます。このワークシートでは以下の項目を定義します。

項目 省略時の扱い 説明
グループ名 該当セルから行番号が小さいセルに向かっていき、最初に見つかった空でないグループ名が代入されます。空でないセルが見つからなかった場合は空文字として扱われます。 シェーダーアノテーション の各種パラメーターの group に対応します。
種類 省略不可 パラメーターの種類です。オプション変数は option、マテリアルユニフォームブロック変数は ub_material、サンプラー変数は sampler、描画情報は renderinfo を記述します。
変数宣言 省略不可 ソースコード上での変数宣言を記述します。float weight のように型名も含めて記述します。オプション変数の場合は SKINNING (0) のようにマクロ名と値を記述します。オプション変数のユニフォームブロック変数の型は int、変数名は id と同じになります。ユニフォームブロック内の変数の宣言順序は Excel で記述した順番になります。
ID 省略不可 シェーダーアノテーション の各種パラメーターの id に対応します。描画情報の場合は name に対応します。
ラベル なし シェーダーアノテーション の各種パラメーターの label に対応します。
デフォルト値 なし シェーダーアノテーション の各種パラメーターの default に対応します。
その他のアノテーション なし その他追加したい任意のアノテーションです。ui_order アノテーションは Excel で記述した順番に従って自動で付与されるため、ここで記述しないでください。
#if ディレクディブによる除外の条件式 なし #if ディレクディブにより変数宣言を除外する条件式です。else を記述すると #else で条件が区切られます。
UI 上での並び順 省略不可 Excel で記述した順番でソートされるように シェーダーアノテーション の各種パラメーターの ui_order が自動で付与されます。

これらは Excel で以下の書式で記述します。赤色のセルは各設定項目のヘッダーとなるセルです。

セル番号 A B C D E F G H
1 group type variable id label default other_annotation preprocess
2 パラメータ A のグループ名 パラメータ A の種類 パラメータ A の変数宣言 パラメータ A の ID パラメータ A のラベル パラメータ A のデフォルト値 パラメータ A のその他のアノテーション パラメータ A の #if ディレクディブによる除外の条件式
3 パラメータ B のグループ名 パラメータ B の種類 パラメータ B の変数宣言 パラメータ B の ID パラメータ B のラベル パラメータ B のデフォルト値 パラメータ B のその他のアノテーション パラメータ B の #if ディレクディブによる除外の条件式
