Mii の追加・削除タイミング

ここでは、データベースの Mii がどのようなタイミングで追加・削除されるかについて説明します。

アプリケーションからアプレットを呼び出した場合

アプリケーションからMii作成・編集アプレットやamiibo設定アプレットを呼び出した場合に Mii の追加・削除が発生する可能性があります。

本体設定から Mii の作成・編集を呼び出した場合

アプリケーション実行中にホームボタンを押して、本体設定の Mii の作成・編集 や amiibo のオーナーとニックネームの登録より、 Mii の追加・削除が発生する可能性があります。

デバッグ API を使ってデータベースを操作した場合

アプリケーションから、 nn::mii::Database クラスの デバッグ用 API を呼び出した場合に Mii の追加・削除が発生する可能性があります。

追加・削除に関する注意事項

アプリケーション内で次のような操作を行う場合は、上記の追加・削除が発生することを考慮した上でご利用ください。

  • nn::mii::Database::IsUpdated() を使ってデータベースの更新を検出する場合
  • nn::mii::Database::IsFullDatabase() を使ってデータベース上が、満杯か判定する場合
  • nn::mii::Database::Get() を使ってデータベース上の Mii を取得する場合
  • nn::mii::Database::GetCount() を使ってデータベースの件数を取得する場合
  • nn::mii::Database::GetIndex() を使ってデータベース上の Mii を検索する場合
  • nn::mii::Database::UpdateLatest() を使ってデータベース上の Mii から最新を取得する場合
