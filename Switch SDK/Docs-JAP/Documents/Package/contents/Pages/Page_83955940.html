1. はじめに

以降では、レイアウトを利用するまでの流れを示します。

2. レイアウトデータ作成の手順

レイアウトデータを作成するには、付属のレイアウトエディタを用います。

作成したレイアウトデータは、バイナリコンバータでバイナリファイルに変換し、アーカイバで1つのアーカイブファイルにまとめたものをプログラムから読み込む流れになります。

2.1. レイアウトエディタの使い方

レイアウトエディタは、Tools\Graphics\LayoutEditor\LayoutEditor.exe を開くことで起動します。

図2. レイアウトエディタ

  • [ ファイル ] → [ 新規作成 ] → [ レイアウト ] から、レイアウトの新規作成ができます。
  • [ ファイル ] → [ バイナリ出力 ] で、作成したレイアウトを、バイナリコンバータでバイナリファイルに変換して出力することができます。
  • [ ペイン ] → [ ペイン作成 ] から、レイアウトにペインを追加することができます。
    ペインとは、テキストボックスなどのレイアウト上の要素のことを指します。
  • [ ウインドウ ] → [ カーブエディタ ] から、ペインにアニメーションを設定することができます。
  • 「 F5 キー」を押すことで、ビューアーに転送して、再生確認することができます。
    • [ ツール ] → [ ビューアーターゲット ] で、起動するビューアーを選ぶことができます。
    • PCビューアーの場合は、プレビューを実行すると 自動的に LayoutViewer が起動します。実機ビューアーの場合は、事前に [ ツール ] → [ 実機ビューアー を起動 ] から起動してください。

関連リンク レイアウトエディタの詳細については、「 LayoutEditor 」のページを参照してください。

参考：

具体的な使い方は、サンプルデータの simple.flyt ( Samples\Sources\Resources\Ui2d\simple\data ) を参考にすると良いでしょう。
データを開く前に、setup.bat ( Samples\Sources\Resources\Ui2d\Common ) でフォントファイルを生成してください。

2.2. バイナリコンバータ

レイアウトエディタ上でバイナリコンバータによるバイナリファイルへの変換が行えますので、バイナリコンバータ単体での使い方はここでは省略します。

関連リンク バイナリコンバータ単体の使い方は「バイナリコンバータ」のページを参照してください。

2.3. アーカイバ

出力したバイナリファイルは、アーカイバにより1つのアーカイブファイルにまとめる必要があり、コンソール上でコマンドを記述して行います。
例えば、出力したバイナリファイルが以下のような構成になっているとき、

./simple
./simple/anim
./simple/blyt
./simple/font
./simple/timg

カレントディレクトリ上で下記のコマンドを記述することによりアーカイブファイルが作成できます。最後の「.」を忘れないように注意してください。

Tools\Graphics\LayoutTools\LayoutArchiver.exe -D simple ../simple.arc . --tile-mode linear --api-type Gl --code-type Source

関連リンク アーカイバの詳細については、「アーカイバ」のページを参照してください。
