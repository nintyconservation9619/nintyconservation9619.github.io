プログラムの異常終了

  • はじめに
  • 異常終了時に出力される定型文字列

はじめに

いくつかの理由で、SDK ライブラリ内外でアプリケーションプログラムが異常終了する場合があります。

このドキュメントでは、アプリケーション異常終了時に出力される「定型文字列の種類」と「その理由」について解説します。

この内容を理解することで、異常終了の原因が探りやすくなります。
また、自動テストなどで監視すべき文字列の一覧としても利用できます。

異常終了時に出力される定型文字列

文字列 理由
SDK Assertion Failure SDK コード内で条件が満たされていません。 
Precondition not met ユーザーコード内で API を呼び出す際に、事前条件が満たされていません。
Assertion Failure ユーザーコード内の NN_ASSERT 系 API で条件が満たされていません。
Unexpected Default

SDK コードあるいはユーザーコード内の switch 文で、到達してはいけない default ラベルに到達しました。
Abort SDK コードあるいはユーザーコード内で NN_ABORT 系 API が呼び出されました。
UserException handler is called at pid=0x... SDK コードあるいはユーザーコード内で、なんらかの例外が発生しました。

下記のケースでは、ユーザー側で修正が可能です。それぞれ条件を満たすように修正してください。

  • Precondition not met
  • Assertion Failure
  • Unexpected Default のうち、ユーザーコード内の NN_UNEXPECTED_DEFAULT に到達した場合
  • Abort のうち、ユーザーコード内の NN_ABORT 系 API で条件を満たさなかった場合
  • UserException handler is called at pid=0x... のうち、ユーザーコード内で例外が発生した場合

これ以外のケース ( SDK Assertion Failure および、SDK コード内での条件違反・例外) はユーザー側で修正することができません。
