はじめに

Input Director は PC 上の HID 入力を開発機へ転送するためのツールです。 転送された値は開発機の HID 入力として扱われます。

PC 上の HID 入力による開発機の操作を可能にすることで、以下のことを実現します。

  • シームレスなデバッグ
  • 実デバイスを必要としないアプリケーションの開発


下記の HID 入力に対応しています。

  • XInput 対応ゲームパッド
  • キーボード
  • マウス

使用方法

プログラムの起動

下記のプログラムを開発機上でのデバッグを開始する前に起動しておきます。 

Tools/Hid/InputDirector/InputDirector.exe

常駐プログラムとして動作しますので、一度起動しておけば再起動の必要はありません。

起動後は、 Target Manager と開発機が接続状態にあれば、 Input Director との接続も自動的に確立します。 

開発機と接続が確立すると下記のように「接続」に開発機の名前が表示されてチェックマークがつきます。

初回起動時は、 ここから「設定」を選択し、 初期設定を行う必要があります。

上図のように「画面端へマウスポインタ移動で切り替え」と「ホットキーで切り替え」からチェックを外し、「OK」ボタンを押して設定の変更を反映してください。

XInput 対応ゲームパッドからの入力の転送

開発機との接続が確立した状態であれば、 XInput 対応ゲームパッドからの全ての入力が自動的に開発機へ転送されます。

転送された入力は BasicXpad または NpadStyleFullKey からの入力として扱われます。

ゲームパッドの対応関係は、 常駐プログラムの「設定」の「ゲームパッド設定」から変更可能です。

BasicXpad として操作する場合

「ゲームパッド」から設定の対象となる XInput 対応ゲームパッドを選択し、 転送先となる BasicXpad のプレイヤー番号を指定してください。

NpadStyleFullKey として操作する場合

「ゲームパッド」から設定の対象となる XInput 対応ゲームパッドを選択し、 転送先となる Npad のコントローラー番号を指定してください。
コントローラー番号を指定する際は、1引いた数を設定する必要があります。
例) NpadId::No1 の場合 -> プレイヤー番号に "0" を設定する。 

キーボードまたはマウスからの入力の転送

Input Director がキャプチャモードにあるとき、 キーボードやマウスからの入力は開発機へ転送されます。

転送された入力はそれぞれ Keyboard と Mouse からの入力として扱われます。

キャプチャモードの切り替えは、マウス操作やホットキーで行うことが可能です。 「設定」からお好みに合わせた切り替え方法を設定してください。
