フォント(nn::font)モジュールでできること

フォントモジュール（nn::font）を使うと、画面に文字を表示することができます。

文字は、ポリゴンにテクスチャを貼って表示するので沢山の文字を高速に表示することができます。
また、以下のような効果を付けて文字を表示するこができます。

  • 文字の色を付ける。またその色をグラデーションさせる
  • 斜体（イタリック）
  • 文字に影を付ける
  • 文字間隔や、行間隔を制御する
  • カーニング処理（美しく見えるように、文字種類ごとに間隔を調整すること）

 

フォントモジュールが表示できるフォントの形式は3種類あります。

  • ビットマップフォント（専用ツールによって作成する、表示する大きさが固定されているフォント）
  • スケーラブルフォント（otf/ttf 形式の表示する大きさを変えることのできるフォント）
  • 本体内蔵フォント（特殊な形式のスケーラブルフォント）

用語の説明

ビットマップフォント

フォントモジュールが扱うことができる、フォント形式です。
後述する、FontConverter という専用ツールで作成します。
作成するときに指定した「固定サイズ」で表示される点が、スケーラブルフォントとの大きな差異点です。

ファイル拡張子は、bffnt です。

スケーラブルフォント

フォントモジュールが扱うことができる、フォント形式です。
一般的なファイル形式である、ttf (ttc) や otf 形式のファイルを扱えます。
表示する際に、任意のサイズに拡大・縮小（スケール）することができます。

表示に必要なテクスチャを、実行時に用意するため、専用の事前処理が必要となります。そのためプログラムの作り方を工夫する必要があります。
一方で、事前に表示する文字内容が予想できない場合にも少ないメモリ量で対応できるというメリットにがあります。

本体内蔵フォント

ゲーム機本体から、読みだして利用することができるフォントです。

特殊な形式のスケーラブルフォントとなっています。

フォントの読出しには、「PL ライブラリ」を使います。

ファイル拡張子は bfttf です。この形式を ユーザーが作成することはできません。

FontConverter

ビットマップフォントを作成するためのツールです。GUI 版と、CUI 版があります。

入力形式は、特殊な形式のビットマップ画像か、スケーラブルフォント（ttf/otf）が選べます。

詳しい説明は、「こちら」をご覧ください。

フォントランタイム(nn::font)

フォントを読み込んで、画面に表示するためのランタイムライブラリ（プログラマーが、ゲームプログラムに組み込んで使う部品的なプログラムのこと）です。

関連モジュール

nn::fontll

nn::fontll は、スケーラブルフォントを扱うモジュールです。

フォントを読み込んで解釈し、フォントの幅や高さといった情報にアクセスしたり、メインメモリ上に 文字の画像をレンダリングするといった基本的な処理だけを行います。

nn::font はnn::fontllを下位モジュールとして利用しています。nn::fontll を 単体で利用することも可能です。

詳しい説明は、「こちら」をご覧ください。

nn::gfx::util::DebugFont

DebugFont はデバック用途に特化した文字描画モジュールです。

nn::font よりできることは少ないですが、ずっと簡単に利用することができます。

詳しい説明は、「こちら」をご覧ください。

nn::ui2d

ui2dは、ステータス表示やメニュー画面などの主に2Dで描画される部分を、効率的に開発できるように設計されたモジュールです。

ui2dは、文字列を表示する際に、nn::font を利用しています。

詳しい説明は、「こちら」をご覧ください。
