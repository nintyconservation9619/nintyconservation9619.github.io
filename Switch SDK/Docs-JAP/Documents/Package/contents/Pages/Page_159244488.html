EDEV の初期化が失敗するときは

EDEV の状態等によっては初期化が失敗する時があります。

  • 初期化が Fail で失敗する
  • InitializeEdevWin の EDEV 接続リストに初期化したい EDEV 本体が表示されない (されなくなった)

その場合は、次のことを確認してください。

InitializeEdev.exe (もしくは InitializeEdevWin.exe) を用いる

InitializeEdev.exe もしくは InitializeEdevWin.exe を直接呼び出すことを推奨します。

また以下のファイルもそれらを直接呼び出しています。

  • %NINTENDO_SDK_ROOT\InitializeEdev.bat
  • %NINTENDO_SDK_ROOT\Tools\CommandLineTools\DevKitSetup.bat
  • %NINTENDO_SDK_ROOT\Tools\CommandLineTools\DevKitSetup.ps1

指定パラメーターを確認する

コマンドラインツールである InitializeEdev.exe を使っている場合はパラメーターの指定が合っているかを確認してください。(なおシリアル番号はチェックディジットを含んでも含まなくても動作します)

(バージョンダウンの場合)制約のあるバージョンであるかを確認する

新しいバージョンの SDK に含まれる InitializeEdev.exe で初期化した EDEV を古いバージョンの SDK に含まれる InitializeEdev.exe で初期化する場合、バージョンによっては初期化できないケースや手動でセーフモードに入れる必要のあるケースがあります。

その制約のある古いバージョンとは以下のものです。

  • FW 2.2.0-0 および FW 2.2.0-1
  • NX Addon 1.5.0 (FW 2.1.0-0) 以前の全バージョン

何度か初期化する

EDEV を何度か初期化することで成功することがありますのでお試しください。

EDEV 本体を十分に充電する

EDEV 本体は十分に充電を行ってから初期化してください。

充電を行うためには ACアダプターと接続との接続が必要です。USB ケーブルと接続しただけでは充電できませんのでご注意ください。

USB ケーブルを再接続し電源を入れる

USB ケーブルをつなぎ直して電源が ON になった状態から EDEV を初期化してください。

特にうまくいかないような場合は以下のような手順でつなぎ直してください。

  1. EDEV 本体を USB ケーブルからはずす
  2. EDEV 本体左上部にある電源ボタンを長押し(約12秒)して電源を切る
  3. 電源が OFF 状態のまま EDEV 本体を USB ケーブルに再接続する
  4. 接続すると自動的に電源が入りますので EDEV を初期化してください

また、リレーボックスに接続して利用している場合はリレーボックスを用いず直接 PC に接続するか、もしくは正しい手順でリレーボックスと接続できているかを確認して下さい。

Target Manager が既に起動している場合は終了する

Target Manager が既に起動している場合は終了させてから EDEV を初期化してください。

ちなみに Target Manager はタスクトレイに常駐している場合もありますので、そちらもご確認ください。

デバイスマネージャーで EDEV 本体が認識されていることを確認する

EDEV がUSBデバイスと認識されていることをデバイスマネージャーで確認してください。
詳しくは「EDEV の初期化」 の「EDEV 用のドライバについて」をご確認ください。

セーフモードを手動操作で起動する

セーフモードを起動してから初期化することで成功する場合があります。

参考：

セーフモードとは最低限の機能で動作するモードです。

以下のような場合にセーフモードを利用してください。

  • 本体の動作がうまくいかないため復旧が必要な場合
  • EDEV 初期化が失敗する場合
  • 電源を入れてもメニュー画面が表示されない場合

セーフモードはユーザーの操作によって起動することが可能です。

セーフモードに一旦入ると初期化しない限り通常モードでは起動しない場合がありますので、 復旧や初期化が必要な時のみ利用することをお奨めします。

セーフモードに入る手順

セーフモードに入るには2つの方法があります。

(方法1) Joy-Con(R) を装着して行う方法

この方法がおすすめです。

  1. USB ケーブルを接続した状態で電源ボタンを長押し(約12秒)して EDEV 本体の電源を OFF にします。 (なお20秒押し続けると再度電源が入りますので12秒を守ってください)
  2. EDEV 本体に Joy-Con(R) を装着する。(Joy-Con(L) は装着しないようにすること)
  3. 音量ボタンの「-」を押しながら電源ボタンを押す
  4. 後述の「セーフモードに入ったことを確認する方法」を参照してセーフモード起動したかを確認し、うまくセーフモードに入っていない場合(特にメニュー画面が表示された場合)は手順1からやり直して下さい。

(方法2) Joy-Con を装着せずに行う方法

  1. USB ケーブルを接続した状態で電源ボタンを長押し(約12秒)して EDEV 本体の電源を OFF にします。 (なお20秒押し続けると再度電源が入りますので12秒を守ってください)
  2. 本体左上にあるボタンを以下の順番で操作します。
    1. 音量ボタンの「-」を押しながら電源ボタンを押す
    2. 電源ボタンを押したまま、音量ボタンの「-」を離して音量ボタンの「+」を押す
    3. 音量ボタンの「+」を押したまま、電源ボタンを離す
    4. 音量ボタンの「+」を離す
  3. 後述の「セーフモードに入ったことを確認する方法」を参照してセーフモード起動したかを確認し、うまくセーフモードに入っていない場合(特にメニュー画面が表示された場合)は手順1からやり直して下さい。
参考：

セーフモードに入れようとしても通常の起動シーケンスに入ってしまいメニューが表示されてしまう場合は、ボタンの操作間隔を調整するとうまくいく場合があります。（ボタン操作の間隔が速いと操作が受け付けされない場合があるためです。）

目安となる操作間隔は「0.6 秒ごと」です。これは「BPM=100 で 4 ビートの間隔」と同じでして、"BPM=100" で動画検索すると間隔の目安となる動画が見つかると思います。

セーフモードに入ったことを確認する方法

セーフモードに入ったことを確認する方法はファームウェアのバージョンに応じて色々存在します。

  1.  Re-Scan ボタンを押下すると接続一覧に “(Safe Mode)” と表示された本体が表示される。(ただし InitializeEdevWin.exe をお使いの場合)
  2. EDEV 本体の画面に "NX DevKit Safe Mode" と4秒間表示される。(ただしNX Addon 3.0.0 以降で初期化した EDEV をお使いの場合)
  3. EDEV 本体の画面が黒いままである。(ただし NX Addon 1.X 以前で初期化した EDEV をお使いの場合)
    1. なお、この時画面が一瞬だけ白色点灯する場合もあります。

また NX Addon 1.0 より前で初期化した EDEV をお使いの場合は後述の “セーフモード操作で赤画面になる場合があります” の場合もありますのでご参照ください。


注意：

SDK のバージョンによって手動でセーフモードに入れても InitializeEdev が認識しない場合があります。
それに対する修正は以下のバージョンの SDK で実施していますので、以下のバージョンのもので実行して頂くことを推奨します。

  • NXAddon 0.12 系のうち 0.12.17 以降 (Firmware 1.0.0-21 以降)
  • NXAddon 1.X 系のうち 1.4.0 以降 (Firmware 2.0.0-16 以降)
  • NXAddon 3.0.0 SS1 以降
参考：

NX Addon 1.0 以降で初期化したことのない EDEV をお使いの場合はセーフモードに入れる操作を行うと赤画面になることがあります。この状態になった場合は電源ボタン12秒長押しの後再度電源ボタンを押下するとセーフモードで起動できます。

参考：

NX Addon 0.10.0 以前 (DevMenu で表示されるファームウェアバージョンが 0.7.0-2 以前) の EDEV 本体でセーフモードに入れる場合は、PC に接続する EDEV 本体は一台のみにする必要があります。

他の EDEV を PC と接続している場合は外してください。

USB 3.0で接続して失敗している場合は USB 2.0 で試す

USB 3.0 コントローラーとの相性によって EDEV のセットアップに失敗する場合があります。
以下のことを試してください。

  • USB 3.0 コントローラーのドライバーを最新のものに更新する
  • USB 3.0 コントローラーのファームウェアを最新のものに更新する (公開されていない場合もあります)
  • USB 2.0 のポートに接続してセットアップを行う

なお以下の USB 3.0 コントローラーが搭載された PC でセットアップを行うと失敗することが確認されています

  • Renesas Electronics USB 3.0 Host Controller

PC に接続している EDEV を一台に減らす

NX Addon 0.10.0 以前 (DevMenu で表示されるファームウェアバージョンが 0.7.0-2 以前) のファームで初期化された EDEV 本体の場合は、PC と接続されている他の EDEV があるために失敗することがあります。
その時は PC に接続している EDEV 本体を一台にした後で再度電源を入れ直して初期化してください。

(なおこの方法は NX Addon 0.10.0 より新しいファームで初期化した場合は不要です)

サポートに問い合わせる

上記のことを実施しても初期化に成功しないようでしたらサポート窓口までお問い合わせください。

お問い合わせいただく時は以下のことも合わせて報告して頂くとサポートがスムーズになります。

  • 御使用頂いている EDEV のシリアル番号
  • 御使用頂いている SDK のバージョン
  • 初期化時のログ
    • InitializeEdevWin の場合はログウィンドウに出力されているログをコピー後テキストファイルに貼りつけて、そのファイルをお送りください。
    • InitializeEdev の場合は --verbose オプションをつけて実行したログをファイルに保存してお送りください。
