デバッグ用サンプルデータベース

開発の利便性のために、いくつかの似顔絵データベースのサンプルファイルを用意しています。

サンプルデータベースファイルは SDK の以下の場所にあります。

$(NINTENDO_SDK_ROOT)/Resources/Mii/Common/sampledb
注意：

これらのデータベースファイルはデバッグ目的でのみ使用するようにしてください。製品で使用しないように注意してください。

データベースファイルのインポート方法

サンプル用データベースは、デバッグ用の関数 nn::mii::Database::Import() を使用するもしくは、DevMenu や DevMenuCommand を使用することでインポート可能です。nn::mii::Database::Import 関数は DevMenu にある設定 Mii Testing Mode が有効になっていないと使用できません。 DevMenu　の Debug タブにある Mii Testing Mode を Enabled に設定してから使用してください。DevMenu とDevMenuCommand を使用したインポート方法は、それぞれ DevMenu の ImportDatabase の項目と DevMenuCommand の Mii データベースのインポートの項目を参照してください。

index 名前 概要
0 SHBS シェイプリソースのクオリティが High（nn::mii::ShapeQuality_High）、シェイプあごひげの時にメモリ使用量が最大になる組み合わせです。

3 SMBTMN

テクスチャリソースのクオリティ（Middle、Low）にかかわらず、メモリ使用量が最大となるパーツの組み合わせは同じです。シェイプリソースのクオリティが High の場合は、モデルタイプが通常の場合もかぶり物用の場合も、メモリ使用量が最大になるパーツの組み合わせは共通です。

テクスチャとして表現されるあごひげは種類によらず同じメモリサイズです。シェイプとして表現されるあごひげは種類によってメモリサイズが異なります。
