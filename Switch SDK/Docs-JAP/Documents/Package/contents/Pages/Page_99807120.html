ベクトル・行列

概要

3D グラフィックスでの利用を目的としたベクトルと行列の計算を行う機能を提供します。

要素型

単精度浮動小数点数をサポートします。倍精度浮動小数点数はサポートしません。

座標系

右手座標系を採用しています。

ベクトルと行列の積

ベクトルを行ベクトルとみなし、行列を右から掛けます。

SIMD 対応

SIMD ユニットを使える環境においては、これらを利用した実装に切り替わることを想定しています。

SIMD ユニットを使えない環境では、汎用の実装を提供します。

ベクトル

2 次元ベクトル、3 次元ベクトル、4 次元ベクトルをサポートします。

注意：

2 次元ベクトルは NXAddon 5.1.x 以降からサポートされます。


  • nn::util::Vector2fType
  • nn::util::Vector3fType
  • nn::util::Vector4fType


注意：

これらの型は計算のために最適化されています。このため、メモリレイアウトもサイズも環境によって異なる場合があります。よって、内部要素への直接のアクセスは禁止します。要素へのアクセスは、専用のマクロ、および、関数を利用してください。

また、 SIMD ユニットでの計算を最適化するため、本構造体は、メモリアライメントを要求します。これらの型をメモリ上に確保する場合には、要求されるアライメントを満たすよう注意してください。

行列

3 行 2 列の 3x2 行列、4 行 3 列の 4x3 行列、4 行 4 列の 4x4 行列をサポートします。

注意：

3 行 2 列の 3x2 行列は NXAddon 5.1.x 以降からサポートされます。

加えて、それぞれの行列について、内部メモリレイアウトとして行要素から順に若いアドレスに並べる行優先（RowMajor）と、列要素から順に若いアドレスに並べる列優先（ColumnMajor）の両方をサポートします。

  • nn::util::MatrixRowMajor3x2fType
  • nn::util::MatrixRowMajor4x3fType
  • nn::util::MatrixRowMajor4x4fType
  • nn::util::MatrixColumnMajor3x2fType
  • nn::util::MatrixColumnMajor4x3fType
  • nn::util::MatrixColumnMajor4x4fType

また、それぞれの行列について、別名を用意しています。

  • nn::util::Matrix3x2fType (nn::util::MatrixRowMajor3x2fType)
  • nn::util::Matrix4x3fType (nn::util::MatrixRowMajor4x3fType)
  • nn::util::Matrix4x4fType (nn::util::MatrixRowMajor4x4fType)
  • nn::util::MatrixT3x2fType (nn::util::MatrixColumnMajor3x2fType)
  • nn::util::MatrixT4x3fType (nn::util::MatrixColumnMajor4x3fType)
  • nn::util::MatrixT4x4fType (nn::util::MatrixColumnMajor4x4fType)


注意：

これらの型は計算のために最適化されています。このため、メモリレイアウトもサイズも環境によって異なる場合があります。よって、内部要素への直接のアクセスは禁止します。要素へのアクセスは、専用のマクロ、および、関数を利用してください。

クォータニオン

クォータニオンをサポートします。

クォータニオンの計算は、nn::util::Vector4fType に対する専用の関数として提供します。

決められた要素数の float 型のメンバを持つ構造体を定義します。

ベクトル・行列に対する値の設定、および、取得に用います。

  • nn::util::Float3
  • nn::util::Float4
  • nn::util::FloatRowMajor3x2

  • nn::util::FloatRowMajor4x4

想定する利用方法

nn::util::VectorType, nn::util::MatrixType は、ベクトル・行列の計算を行うために最適化されており、メモリレイアウトもサイズも環境によって異なる場合があります。

対して、nn::util::Float は、決められたメモリレイアウトを持ちます。

これらを用いて、以下のような流れで計算し、結果を利用することを想定しています。

  1. nn::util::VectorType, nn::util::MatrixType にデータを設定する
    • 初期化マクロを用いる
    • 各要素を設定する関数を用いる
    • nn::util::Float や 配列などを指定して読み込む
  2. nn::util::VectorType, nn::util::MatrixType に用意された関数を用いて計算を行う
  3. nn::util::VectorType, nn::util::MatrixType に収められた結果を得る
    • 各要素にアクセスする関数を用いる
    • nn::util::Float や float 配列などのアドレスを指定して書き出す

ベクトル・行列の初期化

nn::util::VectorType, nn::util::MatrixType は、ベクトル・行列の計算を行うために最適化されており、メモリレイアウトもサイズも環境によって異なる場合があります。

ゆえに、nn::util::VectorType, nn::util::MatrixType に対して初期化子を直接用いることは、環境に依存した実装になるために利用できません。

このため、ベクトル・行列に対し、初期化子として利用できるマクロを定義しています。

  • NN_UTIL_VECTOR_2F_INITIALIZER
  • NN_UTIL_VECTOR_3F_INITIALIZER
  • NN_UTIL_VECTOR_4F_INITIALIZER
  • NN_UTIL_FLOAT_ROW_MAJOR_3X2_INITIALIZER
  • NN_UTIL_FLOAT_ROW_MAJOR_4X3_INITIALIZER
  • NN_UTIL_FLOAT_ROW_MAJOR_4X4_INITIALIZER
  • NN_UTIL_FLOAT_COLUMN_MAJOR_3X2_INITIALIZER
  • NN_UTIL_FLOAT_COLUMN_MAJOR_4X3_INITIALIZER
  • NN_UTIL_FLOAT_COLUMN_MAJOR_4X4_INITIALIZER

また、それぞれの行列について、別名を用意しています。

  • NN_UTIL_FLOAT_3X2_INITIALIZER
  • NN_UTIL_FLOAT_4X3_INITIALIZER
  • NN_UTIL_FLOAT_4X4_INITIALIZER
  • NN_UTIL_FLOAT_T3X2_INITIALIZER
  • NN_UTIL_FLOAT_T4X3_INITIALIZER 
  • NN_UTIL_FLOAT_T4X4_INITIALIZER

本マクロは、環境に応じて適切な初期化リストとなり、以下のように記述することでベクトル・行列を初期化できます。

nn::util::Vector3fType vector = NN_UTIL_VECTOR_3F_INITIALIZER(0.f, 1.f, 2.f);

制約

提供するベクトル・行列計算についての制約をまとめます。

  • ベクトル・行列型の内部要素への直接のアクセス禁止
    • 環境に依存したメンバを持つため、内部要素への直接のアクセスは禁止します。
  • ベクトル・行列型のメモリレイアウトに依存した実装の禁止
    • 環境に依存したメモリレイアウトを持つため、これに依存した実装は禁止します。
