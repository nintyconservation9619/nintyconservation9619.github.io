プログラマ向け変更点

NintendoSDK 1.1.0 で行われた NintendoWare オーディオ (nn::atk) の変更点について説明します。

目次

  • ストリームサウンドのプリフェッチ機能の追加
  • エフェクトに設定できるチャンネル設定の追加

ストリームサウンドのプリフェッチ機能の追加

ストリームサウンドの先頭のデータだけを前もってメモリ上にロードし、再生開始時の読み込み待ちをなくすプリフェッチ機能を追加しました。

プリフェッチ機能を使用するには、まず SoundMaker でストリームサウンドのプリフェッチを有効に設定し、プリフェッチデータ(.bfstp)が生成されるようにします。設定の方法は SoundMaker マニュアルの ストリームサウンド を参照ください。

プログラム上でプリフェッチ機能を使用する方法は、ストリームサウンドの再生方法に応じて異なります。詳しくはプログラミングガイドの ストリームサウンド を参照ください。

  • 通常のストリームサウンド再生の場合
    • MemorySoundArchive を使用している場合、自動的にプリフェッチ機能が適用されます。
    • FsSoundArchiveArchive を使用している場合、プログラミングガイドの ストリームサウンド を参考にサウンドアーカイブやグループのプリフェッチデータを再生前にロードしてください。
  • StartInfo でパス指定を行うストリームサウンド再生の場合
    • nn::atk::SoundStartable::StartInfo::StreamSoundInfo::externalPath を使用したストリームサウンド再生では、
      プログラム上で .bfstp ファイルのロードを直接行った上で、nn::atk::SoundStartable::StartInfo::StreamSoundInfo::prefetchData に .bfstp をロードしたアドレスを設定してください。

エフェクトに設定できるチャンネル設定の追加

nn::atk::EffectDelay, nn::atk::EffectReverb, nn::atk::EffectI3dl2Reverb について、設定できるチャンネルが追加されました。
現状、設定できるチャンネル設定は以下のようになります。

  • nn::atk::EffectDelay : 1ch, 2ch, 4ch, 6ch(追加)
  • nn::atk::EffectReverb : 1ch(追加), 2ch, 4ch
  • nn::atk::EffectI3dl2Reverb : 1ch(追加), 2ch, 4ch, 6ch
