はじめに

SDEV の初期化 では、初回の SDEV セットアップについて説明しました。

このドキュメントでは、古い SDK でセットアップされた SDEV ファームウェアのアップデート方法を説明します。

SDEV ファームウェアのアップデート手順

SDEV ファームウェアのアップデート手順は次の通りです。 

  1. ファームウェアパッケージのインストール
  2. PC / SDEV の接続、SDEV の電源ケーブルの接続
  3. 本体設定のバックアップ（オプショナル）
  4. InitializeSdev.bat の実行

以下、詳細に説明します。 


ファームウェアパッケージのインストール

NintendoSDK SystemUpdater for NX をインストールします。

MegaZarf 形式でパッケージをインストールしている場合は、自動的にインストールされています。
NDI でサーバーにアクセスし、パッケージを選択してインストールする場合は、明示的に「NintendoSDK SystemUpdater for NX」を選択し、インストールしてください。

PC / SDEV の接続、SDEV の電源ケーブルの接続

通常使用時と同様に、PC と SDEV を接続し、SDEV に電源ケーブルを接続してください。

本体設定のバックアップ

アップデートを行うと本体設定は初期値にリセットされます。

設定マネージャを用いて設定値をバックアップしておくことでアップデート前の設定値をアップデート後に引き継ぐことが可能になります。

セーブデータのバックアップ

各種セーブデータも消去されますので、DevMenu のセーブデータバックアップ機能でバックアップを取ると初期化後に引き継ぐことができるようになります。
(ツール BackupSaveData 等を利用してバックアップすることも可能です)

InitializeSdev.bat の実行 

NintendoSDK フォルダ直下に配置されている InitializeSdev.bat を実行してください。

この手順で、新しいファームウェアが SDEV に書き込まれます。

実行手順は SDEV の初期化  で示されている内容と同様です。


【NX Addon 4.3.0以降限定】更新ツールを使った SDEV ファームウェアのアップデート

NX Addon 4.3.0 からは SDEV 本体更新ツールを用いてファームウェアのアップデートをすることもできるようになりました。

詳しくは &ldquo;開発機セットアップに関する情報あれこれ&rdquo; のページの “開発機の更新ツールに関して” の章をご覧ください。

制限事項

SDEV に記録された情報の保持に関して

SDEV のファームウェアをアップデートすると、以前のバージョンの SDK やファームウェアを利用して SDEV に記録された情報は、すべて削除されます。削除される情報は次の通りです：

  • インストールしたアプリケーション
  • セーブデータ
  • アカウント情報


NX Addon 0.5.0 より前のバージョンへのバージョンダウンを伴うアップデートに関して

NX Addon 0.5.0 以降のファームウェアに更新した SDEV をそれより前のバージョンのファームウェアにバージョンダウンすると InitializeSdev.bat の実行に失敗する場合があります。

その場合はSDEVの復旧を行ってから再度 InitializeSdev.bat を実行して下さい。 SDEV の復旧方法については「SDEV 取扱説明書」の「困ったときは」 - 「SDEV 本体の起動に失敗する」のページを参照して下さい。
