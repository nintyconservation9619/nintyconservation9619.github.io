概要

Maya プラグインは Autodesk Maya からグラフィックス中間ファイルを出力するためのプラグイン群です。

表記について

以降の記述において、次の簡略化表記を使用します。

正式名称 簡略表記
Autodesk Maya Maya
グラフィックス中間ファイル 中間ファイル

動作環境

Maya プラグインは以下の環境で動作を確認しています。

Maya Maya 2015 Service Pack 6
Maya 2016 Service Pack 6
Maya 2017 Update 5
Maya 2018.2
OS Microsoft Windows 7 Professional (64bit)
Microsoft Windows 10 Pro (64bit)

Maya を英語環境で使用する場合でも、日本語環境で使用する場合でも、Maya プラグインは動作します。

ただし、日本語環境の場合でも、プラグインのメニューやダイアログは英語で表示されます（一部、Maya 標準の UI を利用している部分は日本語で表示されます）。
このマニュアルでは Maya のメニュー名やアトリビュート名を「日本語環境での表示（英語環境での表示）」のように記述します。

注意：

Maya 2018.2 では、Output Window に表示されるエラー（警告）が文字化けする場合があります。


Maya プラグインの起動

Maya プラグインは、Tools\Graphics\MayaPlugins フォルダーにある起動用バッチファイルから Maya を起動することにより利用できます。
Maya のバージョンによって起動するバッチファイルが異なりますので、 利用している Maya に合ったバッチファイルを利用してください。

Maya のバージョン バッチファイル
Maya 2015 StartMaya-2015.bat
Maya 2016 StartMaya-2016.bat
Maya 2017 StartMaya-2017.bat
Maya 2018 StartMaya-2018.bat

Maya 起動後のメインメニューに Nintendo が追加されていれば、起動に成功しています。

Maya 起動用バッチファイルの引数にシーンファイル名を指定すると、Maya 起動時にシーンが開かれます。

StartMaya-2017.bat scene1.mb

シーンファイル名の後に「-proj プロジェクト名」を記述すると、指定したプロジェクトのシーンが開かれます。

StartMaya-2017.bat scene1.mb -proj C:\Project1

シーンファイル名の後に「-command MEL コマンド」を記述すると、シーンを開いた後に指定した MEL コマンドが実行されます。

StartMaya-2017.bat scene1.mb -command "select modelA"

Maya 起動用バッチファイルの最初の引数が「-batch」なら、 mayabatch.exe で Maya を起動します。

StartMaya-2017.bat -batch -proj "C:/Project1" -file "Scene1.mb" -command "NintendoExportByFdes \"C:/Data/Scene1.fdes\""

Maya 起動用バッチファイルで Maya を起動すると、環境変数 NINTENDO_MAYA_ROOT に Maya 起動用バッチファイルのあるフォルダーのパスが設定されます。

Maya 起動用バッチファイルの最初の引数が「--environment-only」なら、 プラグイン用の環境変数の設定だけを実行して、Maya は起動しません。
mayapy.exe でプラグインを利用する場合などに指定します。 

call StartMaya-2017.bat --environment-only
"%NINTENDO_MAYA_APP_ROOT%\bin\mayapy.exe" "UserScript.py"


Maya の環境設定

プリファレンス（「ウィンドウ > 設定/プリファレンス > プリファレンス（Windows > Settings/Preferences > Preferences）」）は次のように設定してください。

  • 「設定（Settings）」の「アップ軸（Up axis）」は「Y」にしてください。
  • 「設定（Settings）」の「角度（Angular）」は「度（degrees）」にしてください。
  • 「設定 > アニメーション（Settings > Animation）」の「ウェイト付き接線（Weighted tangents）」は OFF にすることを推奨します。
    ウェイト付き接線が ON の状態でアニメーションを作成した場合、常に各フレームの値をベイクしたデータからアニメーションカーブデータを自動作成して出力します。そのため、小数フレームにおける再生結果が Maya 上と異なる場合があります。
参考：

Maya 2017 以前では、シーンの状態によっては、シーンファイルを開きなおして fmd ファイルを出力すると、接線や従法線の値が変化することがあります。
その場合、環境変数 MAYA_NO_TBB に 1 を設定した状態で Maya を使用することによって、毎回同じ値が出力されるようになります。
ただし、副作用として Maya 上でのレンダリング速度が遅くなる場合があります。

使用文字の制限

ノード名やマテリアル名に使用する文字について説明します。

禁止文字

ノード名やマテリアル名には、半角の「英字（大文字・小文字）」「数字」「_（アンダーバー）」のみ使用可能です。

それ以外の文字や記号は使用禁止です。禁止文字が使用されていた場合、出力時に「_（アンダーバー）」に変換して出力します。
シーンの読み込みやリファレンスを使用した場合、ノード名などに「：（コロン）」が含まれることがあります。 コロンは禁止文字であり、中間ファイル出力時にアンダーバーに変換されるので注意してください。

参考：

Maya 上で「Test:Cube1」と「Test_Cube1」という名前のノードは異なる名前として扱われますが、NintendoWare では同じ名前として扱われるため中間ファイル出力時に名前が変更されます。

参考：

エクスポートプラグインの Remove Namespace オプションを ON にすると、ネームスペース部分を削除した名前を中間ファイルに出力できます。


大文字 / 小文字

Windows では表記上、大文字・小文字の入力が可能ですが、内部的な処理では「ABC.fmdb」「abc.fmdb」は同じファイルと認識されますので、出力すると上書きされてしまいます。

 一方、Maya 上では（たとえばノード名など）「ABC」と「abc」という名前は区別されて処理されます。つまり、共存することが可能です。 
Maya 上でのこのような命名はデータ切り出しの際に衝突する可能性があります。

外部に（OS側に）ファイルとして出力されるデータに関しては、「仮にすべて小文字にしたとしても衝突しない名前」を使用するようにしてください。
