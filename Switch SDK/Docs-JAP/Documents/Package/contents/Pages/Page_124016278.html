プロジェクト設定ファイルフォーマット

「プロジェクト設定ファイル」は、ゲームのプロジェクト共通の設定内容を記録する XML 形式のファイルです。
「部品ルートフォルダ」以下に作成されます。
 
以下は「プロジェクト設定ファイル」の設定で可能になる要素です。

特に断りが無い限り、ProjectSettings 要素の子供要素として記載をしてください。

ファイル保存時にユーザコマンドを実行 コマンドファイルパスを指定して、ファイル保存前後、ファイル削除前後で任意のコマンドを実行できます。

レイアウトにおけるリニア空間

ゲームプロジェクトにリニアワークフローを導入するかどうかを設定できます。
関連リンク 詳細については 「リニアワークフロー」 を参照してください。


ビューアーや、LayoutEditror 上でプレビューを行う際の設定を以下の要素によって制御することができます。
また、これらの要素の設定は、[メニュー]→[編集]→[ツール環境設定]から変更できます。

<SRGBWriteEnabled>true</SRGBWriteEnabled>
<DegammaTextureConversionEnabled>true</DegammaTextureConversionEnabled>
<DegammaParameterConversionEnabled>false</DegammaParameterConversionEnabled>
float カラー出力

以下のタグへ true/false を設定することで float カラー出力機能の有効/無効を切り替えることができます。
デフォルトの値は false になっています。

  <FloatColorWriteEnabled>false</FloatColorWriteEnabled>


ASTC テクスチャ

<AstcTextureEnabled> へ true/false を設定することで ASTC テクスチャフォーマットの有効/無効を切り替えることができます。

ASTC の詳細は「ASTC テクスチャ」を参照してください。
プレビュー時の解像度設定

<PreviewDisplayWidth><PreviewDisplayHeight> へ画面の縦横サイズを指定することで
プレビュー時の内部バッファの解像度を設定することができます。
プレビュー時のスケーリング指定　　　　

レイアウトデータをプレビューする際のスケーリング指定を <ViewMode> タグで指定することができます。

  • original 
    • LayoutEditor の「レイアウト環境設定」->「画面サイズ」で設定された大きさで表示します。
      0.9.0 以前と同じ挙動となります。
  • fit_screen
    • LayoutEditor の「レイアウト環境設定」->「画面サイズ」で設定された範囲を画面いっぱいに引き伸ばして表示します。

サムネール画像の保存

設定を有効にするとレイアウトファイルを保存したときや、一括同期を行ったときなどに、サムネール画像をレイアウトファイルの横の .NwThumbs フォルダに「レイアウトファイル名.flyt.png」に保存します。

またファイルメニューに「指定フォルダ以下のサムネール画像を表示」のメニューが追加されます。

設定を省略したときはサムネールは出力されません。

プロジェクト設定ファイルでは画像の最大サイズを指定することもできます。

保存された画像はサムネールエクスプローラ拡張を使用して表示することができます。

  <SaveThumbnailImage>true</SaveThumbnailImage>
  <ThumbnailMaximumWidth>256</ThumbnailMaximumWidth>
  <ThumbnailMaximumHeight>256</ThumbnailMaximumHeight>

スケーラブルフォントの使い方

スケーラブルフォント設定を記述することで、利用できます。
カスタムPCビューアとの連携通信

ゲームプロジェクトで独自に用意したカスタムPCビューアとの通信の際のヒント（ウィンドウタイトル文字列）を指定します。
このオプションは、互換性のために残されていますが、現在は利用されていません。

  <CustomPCViewerWindowTitleHint />
拡張ユーザデータのプリセット

「拡張ユーザデータ」の機能を向上できます。
アーカイバ実行前コマンド

LayoutEditor がアーカイバを実行する前に任意のコマンドを呼び出すことができます。

コマンドの第一引数には、アーカイブ対象フォルダが渡ります。

それ以降には、プラットフォーム設定の <PreArchiverCommandAdditionalArguments> で指定された値が渡ります。
(絶対パス、プロジェクトファイル相対パス、環境変数の利用が可能です)

<PreArchiverCommandPath>%COMMANDS%\PreArchiverCommand.bat</PreArchiverCommandPath>
 
rem %1 はアーカイブ対象ディレクトリ
rem backup にコピーする
robocopy %1 %~dp0\..\backup /MIR

rem 7 以下は正常終了
if %errorlevel% leq 7 exit 0
exit 1
テクスチャ素材のソースファイル(psd)検索パス

テクスチャ素材のソースファイル(psd)を検索するフォルダを指定します。
LayoutEditor の、テクスチャウィンドウで、テクスチャ素材のソースファイル(psd)を開く機能から利用されます。
(絶対パス、プロジェクトファイル相対パス、環境変数の利用が可能です)

 <TextureSourceFileSearchRootPath >../TexSrc</TextureSourceFileSearchRootPath>

新規ペイン名の指定

ペインの種類ごとに新規のペイン名の接頭語を指定できます。

<UseTextureNameAsPicturePaneName> を true に設定すると、<Picture> の設定は無視され、新規ピクチャペイン名にはテクスチャ名が使われるようになります。

プロジェクト設定ファイル内の記述例：

<DefaultPaneNames>
 <Null>N_null_</Null>
 <Picture>P_pict_</Picture>
 <Text>T_text_</Text>
 <Window>W_window_</Window>
 <Bounding>B_bounding_</Bounding>
 <Parts>L_parts_</Parts>
 <Group>G_group_</Group>
 <UseTextureNameAsPicturePaneName>false</UseTextureNameAsPicturePaneName>
</DefaultPaneNames>
 
テキストボックスペインのテキスト ID の指定

プロパティウィンドウのテキストボックスページのテキスト ID の設定ボタンを押したときに設定される文字列を指定することができます。

$(Pane) はペイン名に置き換わります。

<DefaultTextBoxPaneTextID>@$(Pane)</DefaultTextBoxPaneTextID>
 
デフォルトテクスチャフォーマットの指定

テクスチャフォーマットのデフォルト設定を指定できます。

グレースケール（アルファ有・無）、カラー（アルファ有・無）の計４種類を指定します。

プロジェクト設定ファイル内の記述例：

 <DefaultTextureFormat>    
    <GrayScale>BC4L</GrayScale>
    <GrayScaleWithAlpha>BC5</GrayScaleWithAlpha>
    <RGBColor>BC1</RGBColor>
    <RGBColorWithAlpha>BC3</RGBColorWithAlpha>
 </DefaultTextureFormat>
 
デフォルト新規作成ペインサイズの指定

新規作成ペインサイズの指定ができます。

<DefaultInitialPaneSize>
 <Width>30</Width>
 <Height>40</Height>
 </DefaultInitialPaneSize>

ユーザーコマンドの指定

任意のコマンドをツールメニューから実行できます。

Name を / で区切ることにより、メニューを階層表示させることができます。

Args で引数を指定することができます。

選択しているレイアウトファイルパスがマクロ $(LayoutFile) に代入され、選択しているペインが $(Panes) に代入されます。

ConfirmSave で true を指定すると未保存の変更がある状態でのコマンドの実行を抑制します。

ShortCut でショートカットキーの組み合わせを指定します。

内容には、コマンド名またはコマンドのファイルパスを指定します。% で囲われた環境変数を含めることができます。

ワイルドカードを指定すると一括でコマンドを登録することができます。

 <DefaultTextureFormat>
  <UserCommand Name="スクリプト" Args="$(LayoutFile)" ConfirmSave="true" ShortCut="">%SCRIPT_ROOT%\*.bat</UserCommand>
  <UserCommand Name="notepad" Args="$(LayoutFile)" ConfirmSave="true" ShortCut="Ctrl+Shift+N">notepad</UserCommand>
 </DefaultTextureFormat>
アニメーション区間タグの対象グループ

アニメーション区間タグの対象グループの未指定を許容するかどうかを設定します。
「対象グループを必ず指定する運用」が標準的な運用と考え、デフォルトの値は false （未指定を許容しない）になっています。

 <AllowNoGroupAnimationTag>true</AllowNoGroupAnimationTag>

プレビュー時のアニメーションキーのバイナリ変換方法の指定

プレビュー時のアニメーションキーのバイナリ変換方法の指定します（具体的には、バイナリコンバーターに --omit-nokey-all オプションを指定するかどうかをコントロールしています）。
true が指定された場合は、アニメーション区間タグで指定されたフレーム区間内にキーが無い場合は、アニメーションデータを出力しません。

デフォルトの値は false になっています。

 <PreviewConverterOptionOmitNoKeyAllEnabled>false</PreviewConverterOptionOmitNoKeyAllEnabled>

スケールを維持したままフォントを差し替える機能の指定

スケールを維持したままフォントを差し替える機能を指定します。

デフォルトの値は false になっています。

<KeepingSizeScaleEnabled>false</KeepingSizeScaleEnabled>

キャプチャテクスチャ機能

以下のタグへ true/false を設定することでキャプチャテクスチャ機能の有効無効が切り替えられます。
デフォルトの値は false になっています。

<CaptureTextureFunctionEnabled>false</CaptureTextureFunctionEnabled>


 整列ペイン機能

以下のタグへ true/false を設定することで整列ペイン機能の有効無効が切り替えられます。

デフォルトの値は true になっています。

<AlignmentPaneEnabled>true</AlignmentPaneEnabled>
