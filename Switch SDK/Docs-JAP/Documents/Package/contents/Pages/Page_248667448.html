概要

複数の開発機に一括して各種コマンド処理を要求する 統合 CUI アプリケーションです。

条件を設定して、処理を実行するターゲット（開発機）を選択することが可能です。

注意事項

  1. TargetShell の処理実行中に、開発機の挿抜（接続台数などの開発機構成変更）や、 Nintendo TargetManager による　接続／切断／登録／削除／既定のターゲット変更　は行わないでください。
    　　　既定のターゲットについては TaregetShell の実行前のものを終了時に戻します。Nintendo TargetManager に開発機が登録されておらず、既定のターゲットも設定されていない場合は、TargetShell として意図的に設定は行いません。
    　　　Nintendo TargetManager に登録されているが、PCと接続されていない状態にはしないでください。

場所

  • 実行ファイル 
Tools\CommandLineTools\TargetShell.exe
  • プラグイン 
Tools\CommandLineTools\TargetShellPlugins配下

使い方

  • 開発機の初期化を行う手順です。（下記の例では、 Nintendo targetManager に登録されている開発機全てを初期化します。）
1. PC と開発機を接続します。
2. Windowsコマンドプロンプトを起動します。
3. cd %NINTENDO_SDK_ROOT%\Tools\CommandLineTools
4. TargetShell.exe initialize


  • Joy-Con ファームウェアの更新を行う手順です。（下記の例では、 Nintendo targetManager に登録されている開発機にジョイントしている Joy-Con 全てのファームウェアを更新します。）
1. 開発機にJoy-Conをハンドヘルドで接続します。
2. PC と開発機を接続します。
3. Windowsコマンドプロンプトを起動します。
4. cd %NINTENDO_SDK_ROOT%\Tools\CommandLineTools
5. TargetShell.exe update-joycon

TargetShell のログ表示抑制解除（ debug-log オプション）

  • TargetShell のデバイス情報の詳細ログを表示させるオプションとなります。
    他のサブコマンドに debug-log を付加する形で使用します。
    単独での使用はできません。

使用例

initialzie サブコマンドでの使用例

debug-log は initialize の前でも問題ありません。

TargetShell.exe initialize debug-log

開発機の初期化（ initialize サブコマンド）

オプション

ターゲット指定なしオプション

名前 説明 
-v, --verbose

設定することで初期化の詳細ログを出力します。（ログの出力先は --output の項目を参照ください。）

未指定の場合は、簡易ログとなります。

 
--fwtype <arg>

開発機のファームウェアを設定することで、指定されたファームウェアタイプのイメージを使用して初期化します。

設定しない場合は、通常の開発機専用ファームウェア（DEV MODE）が選択されます。

製品機に近いファームウェア（PROD MODE）　： Prod

microSD カードからのログ出力機能付き　　　　 ： ProdWithLog


例） TargetShell.exe initialize --fwtype=Prod

 
--output <arg>

ログの出力先フォルダを設定します。

未設定の場合は、ログ出力を行いません。


例） TargetShell.exe initialize --output=C:\templog

ターゲット指定ありオプション

※ターゲットを指定するオプションが複数指定された場合には、指定された全ての条件に合致するターゲットでコマンドが実行されます。

※ターゲットを指定するオプションは、再度指定されない限りは後続のサブコマンドにも適用されます。

名前 説明 
--detect-device

指定すると、PC に接続されている全ての開発機が対象となります。

指定しない場合には、Nintendo TargetManager に登録されているものだけをコマンドの実行対象とします。

 
--name <arg>

指定した名称に合致するターゲットをコマンドの実行対象とします。

デバイス名にはワイルドカード（*）を指定することができ、複数の試験機を指定することができます。


例） TargetShell.exe initialize --name=XAW02*

 
--serial-number <arg>

指定したシリアル番号に合致するターゲットをコマンドの実行対象とします。

シリアル番号にはワイルドカード（*）を指定することができ、複数の試験機を指定することができます。


例） TargetShell.exe initialize --serial-number=XAL*

 
--ip-address <arg>

指定した IP アドレスに合致するターゲットをコマンドの実行対象とします。

IP アドレスにはワイルドカード（*）を指定することができて複数の試験機を指定することができます。

（IP アドレスを設定すると USB 接続の EDEV に対しては実行されません）


例） TargetShell.exe initialize --ip-address=169.254.*

       TargetShell.exe initialize --ip-address=169.254.11.111 --ip-address=169.254.11.112

 
--hw-type <arg>

指定した開発機種別に合致するターゲットをコマンドの実行対象とします。

設定できる開発機種別は下記の通りです。

指定しない： NotSpecified （デフォルト）

SDEV　　　： Sdev

EDEV　　　： Edev


例） TargetShell.exe initialize --hw-type=Sdev

 
--connection-path <arg>

指定した接続方法に合致するターゲットをコマンドの実行対象とします。

設定できる接続方法は下記です。

指定しない　　： NotSpecified （デフォルト）

ホストブリッジ ： HostBridge

USB          　　： Usb


例） TargetShell.exe initialize --connection-path=HostBridge

Nintendo TargetManager に登録されている開発機だけを全て初期化します。

TargetShell.exe initialize

IP アドレスを指定

IP アドレスを指定して条件にマッチする開発機のみを初期化します。

TargetShell.exe initialize --ip-address=169.254.*

開発機種別を指定

開発機種別を指定して SDEV のみを初期化します。

TargetShell.exe initialize --hw-type=Sdev

複数の条件を指定

複数の条件を指定して初期化します。（この例では対象 PC に接続されているデバイス全てのうち、シリアル番号が "XAL" で始まり、かつ IP アドレスが ”169.254.*” の開発機を指定となります。）

TargetShell.exe initialize --detect-device --serial-number=XAL* --ip-address=169.254.*


Joy-Con ファームウェアの更新（ update-joycon サブコマンド）

  • Joy-Con ファームウェアの更新を行います。
  • 更新対象の Joy-Conはハンドヘルドのみになります。
    左右のうち、片方のみ接続している場合は、接続している Joy-Con が更新対象になります。
    Bluetooth 接続の Joy-Con は更新対象外です。
  • 複数台の SDEV 、EDEV 等を使って、 Joy-Con ファームウェアの更新を行う場合は並列で行います。

使用例

本体へアプリケーションをインストール

Nintendo TargetManager に登録されている開発機だけを対象に、強制フラグを有効にして本体へアプリケーションをインストールします。

TargetShell.exe install --type=application --path=C:\sdk\Samples\Outputs\NX-NXFP2-a64\Applications\FsSaveData\Develop\FsSaveData.nsp --force --location=builtin

SD カードへアプリケーションをインストール

Nintendo TargetManager に登録されている開発機だけを対象に、 SD カードへアプリケーションをインストールします。

TargetShell.exe install --type=application --path=C:\sdk\Samples\Outputs\NX-NXFP2-a64\Applications\FsSaveData\Develop\FsSaveData.nsp --location=sdcard

パッチのインストール

Nintendo TargetManager に登録されている開発機だけを対象に、パッチをインストールします。

TargetShell.exe install --type=patch --path=C:\sdk\patch-v1.nsp

AoCのインストール

Nintendo TargetManager に登録されている開発機だけを対象に、AoC をインストールします。

TargetShell.exe install --type=addoncontent --path=C:\sdk\Samples\Sources\Applications\AocSimple\Binaries\SampleAoc.nsp

アプリケーション、パッチ、AoC を開発機からアンインストール（ uninstall サブコマンド）

  • 指定したアプリケーション、パッチ、AoC を、指定した開発機から並列でアンインストールします。

オプション

ターゲット指定なしオプション

名前 説明 
--type <arg>

アンインストールのタイプを設定します。

application ：アプリケーションのアンインストールを行う。

path ：パッチのアンインストールを行う。

addoncontent ： AoC のアンインストールを行う。

設定されない場合は、アプリケーションIDに紐付くものをすべて消去します。		 
--application-id <arg>
 アンインストールするIDを設定します。 
--all-uninstall

設定を行うと、全てのアプリケーションをアンインストールします。

本オプションを設定すると、 --application-id の設定は無視されます。

--type に application を設定する時のみ有効、その他を設定すると無視します。

 
--index <arg>

AoC のインデックスを指定する。

--type に addoncontent を設定する時のみ設定するオプションです。

未設定時は ID に紐付く全ての AoC をアンインストールします。

 
-v, --verbose

設定することで詳細ログを出力します。（ログの出力先は --output の項目を参照ください。）

未指定の場合は、簡易ログとなります。

 
--output <arg>

ログの出力先フォルダを設定します。

未設定の場合は、ログ出力を行いません。


例） TargetShell.exe uninstall --output=C:\templog

ターゲット指定ありオプション

※ターゲットを指定するオプションが複数指定された場合には、指定された全ての条件に合致するターゲットでコマンドが実行されます。

※ターゲットを指定するオプションは、再度指定されない限りは後続のサブコマンドにも適用されます。

名前 説明 
--detect-device

指定すると、PC に接続されている全ての開発機が対象となります。

指定しない場合には、Nintendo TargetManager に登録されているものだけをコマンドの実行対象とします。

 
--name <arg>

指定した名称に合致するターゲットをコマンドの実行対象とします。

デバイス名にはワイルドカード（*）を指定することができ、複数の試験機を指定することができます。


例） TargetShell.exe uninstall --name=XAW02*

 
--serial-number <arg>

指定したシリアル番号に合致するターゲットをコマンドの実行対象とします。

シリアル番号にはワイルドカード（*）を指定することができ、複数の試験機を指定することができます。


例） TargetShell.exe uninstall --serial-number=XAL*

 
--ip-address <arg>

指定した IP アドレスに合致するターゲットをコマンドの実行対象とします。

IP アドレスにはワイルドカード（*）を指定することができて複数の試験機を指定することができます。

（IP アドレスを設定すると USB 接続の EDEV に対しては実行されません）


例） TargetShell.exe uninstall --ip-address=169.254.*

       TargetShell.exe uninstall --ip-address=169.254.11.111 --ip-address=169.254.11.112

 
--hw-type <arg>

指定した開発機種別に合致するターゲットをコマンドの実行対象とします。

設定できる開発機種別は下記の通りです。

指定しない： NotSpecified （デフォルト）

SDEV　　　： Sdev

EDEV　　　： Edev


例） TargetShell.exe uninstall --hw-type=Sdev

 
--connection-path <arg>

指定した接続方法に合致するターゲットをコマンドの実行対象とします。

設定できる接続方法は下記です。

指定しない　　： NotSpecified （デフォルト）

ホストブリッジ ： HostBridge

USB　　　　　　： Usb


例） TargetShell.exe uninstall --connection-path=HostBridge

                             addoncontent)
                                (e.g. --type=application)
  --application-id <arg>     Set application-id. This option is invalidated

  --all-uninstall            All uninstall.
                                (e.g. --all-uninstall)
  --index <arg>              This option is the AoC of index.
                             Valid only when AoC is set for type.
                             All AoC to be related application-id is
                             uninstalled if this option does not set.
                                (e.g. --index=1)
  -v, --verbose              Log verbosely.
                                (e.g. --verbose)

                                (e.g. --name=XXX01*)
  --serial-number <arg>      Specify target by serial number matching.

  --ip-address <arg>...      Specify target by ip address matching.
                                (e.g. --ip-address=169.254.*)
  --hw-type <arg>            Specify target by hardware type matching.
                                (e.g. --hw-type=Device)
  --connection-path <arg>    Specify target by connection path matching.
                                (e.g. --connection-path=HostBridge)
  --output <arg>             Specify output directory.
                                (e.g. --output=C:\logdir)

使用例

IDを指定してアプリケーションをアンインストール

Nintendo TargetManager に登録されている開発機だけを対象に、 ID を指定してアプリケーションをアンインストールします。

TargetShell.exe uninstall --type=application --application-id=0x0123456789abcdef

全てのアプリケーションをアンインストール

Nintendo TargetManager に登録されている開発機だけを対象に、 全てのアプリケーションをアンインストールします。

TargetShell.exe uninstall --type=application --all-uninstall

パッチのアンインストール

Nintendo TargetManager に登録されている開発機だけを対象に、 パッチをアンインストールします。

TargetShell.exe uninstall --type=patch --application-id=0x0123456789abcdef

AoCのアンインストール

Nintendo TargetManager に登録されている開発機だけを対象に、 AoC をアンインストールします。

TargetShell.exe uninstall --type=addoncontent --application-id=0x0123456789abcdef --index=1

登録済み開発機のみ

Nintendo TargetManager に登録されている開発機だけをミュート設定の対象とします。

TargetShell.exe mute-audio

IP アドレスを指定

IP アドレスを指定して条件にマッチする開発機のみをミュート設定の対象とします。

TargetShell.exe mute-audio --ip-address=169.254.*

開発機種別を指定

開発機種別を指定して SDEV のみミュート設定の対象とします。

TargetShell.exe mute-audio --hw-type=Sdev

複数の条件を指定

複数の条件を指定してミュート設定を行います。（この例では対象 PC に接続されているデバイス全てのうち、シリアル番号が "XAL" で始まり、かつ IP アドレスが ”169.254.*” の開発機を指定となります。）

TargetShell.exe mute-audio --detect-device --serial-number=XAL* --ip-address=169.254.*

未指定の場合は、簡易ログとなります。

IP アドレスを指定して条件にマッチする開発機だけを対象にアプリケーションのリストを取得します。

TargetShell.exe get-application-list --ip-address=169.254.*

開発機種別を指定

開発機種別を指定して SDEV のみを対象にアプリケーションのリストを取得します。

TargetShell.exe get-application-list --hw-type=Sdev

複数の条件を指定

複数の条件を指定してアプリケーションのリストを取得します。（この例では対象 PC に接続されているデバイス全てのうち、シリアル番号が "XAL" で始まり、かつ IP アドレスが ”169.254.*” の開発機を指定となります。）

TargetShell.exe get-application-list --detect-device --serial-number=XAL* --ip-address=169.254.*

セーブデータのリスト取得（ get-savedata-list サブコマンド）

  • 開発機に保存されているセーブデータのリストを取得します。

ターゲット指定なしオプション

名前 説明 
-v, --verbose

設定することで詳細ログを出力します。（ログの出力先は --output の項目を参照ください。）

未指定の場合は、簡易ログとなります。

 
--output <arg>

ログの出力先フォルダを設定します。

未設定の場合は、ログ出力を行いません。


例） TargetShell.exe get-savedata-list --output=C:\templog

ターゲット指定ありオプション

※ターゲットを指定するオプションが複数指定された場合には、指定された全ての条件に合致するターゲットでコマンドが実行されます。

※ターゲットを指定するオプションは、再度指定されない限りは後続のサブコマンドにも適用されます。

名前 説明 
--detect-device

指定すると、PC に接続されている全ての開発機が対象となります。

指定しない場合には、Nintendo TargetManager に登録されているものだけをコマンドの実行対象とします。

 
--name <arg>

指定した名称に合致するターゲットをコマンドの実行対象とします。

デバイス名にはワイルドカード（*）を指定することができ、複数の試験機を指定することができます。


例） TargetShell.exe get-savedata-list --name=XAW02*

 
--serial-number <arg>

指定したシリアル番号に合致するターゲットをコマンドの実行対象とします。

シリアル番号にはワイルドカード（*）を指定することができ、複数の試験機を指定することができます。


例） TargetShell.exe get-savedata-list --serial-number=XAL*

 
--ip-address <arg>

指定した IP アドレスに合致するターゲットをコマンドの実行対象とします。

IP アドレスにはワイルドカード（*）を指定することができて複数の試験機を指定することができます。

（IP アドレスを設定すると USB 接続の EDEV に対しては実行されません）


例） TargetShell.exe get-savedata-list --ip-address=169.254.*

 
--hw-type <arg>

指定した開発機種別に合致するターゲットをコマンドの実行対象とします。

設定できる開発機種別は下記の通りです。

指定しない： NotSpecified （デフォルト）

SDEV　　　： Sdev

EDEV　　　： Edev


例） TargetShell.exe get-savedata-list --hw-type=Sdev

 
--connection-path <arg>

指定した接続方法に合致するターゲットをコマンドの実行対象とします。

設定できる接続方法は下記です。

指定しない　　： NotSpecified （デフォルト）

ホストブリッジ ： HostBridge

USB　　　　　　： Usb


例） TargetShell.exe get-savedata-list --connection-path=HostBridge
