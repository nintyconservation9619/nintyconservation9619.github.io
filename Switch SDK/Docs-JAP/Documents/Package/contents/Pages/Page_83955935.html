概要

  • NintendoWare オーディオ開発環境で使用するファイルフォーマットの一覧です。
    • 「SoundMaker で使用するファイル」と「Atk ライブラリで使用するファイル」の、大きく分けて２種類が存在します。
    • 「コンバート時に一時生成されるファイル」は、最終的にバイナリサウンドアーカイブ (.bfsar) に含まれ、
      通常は単体で使用することはありません。

 

SoundMaker で使用するファイル

fspj

SoundMaker のプロジェクトファイル (SoundMaker Project) です。

そのプロジェクトに含まれているサウンドセットファイル (.fsst) の情報、
および SoundMaker の [プロジェクト] - [プロジェクトの設定] に書かれている情報が XML 形式で記述されています。

SoundMaker でプロジェクトを新規作成すると、作成されます。
新規作成時に、作成フォルダを指定することができます。

SoundMaker を起動し、[ファイル] - [プロジェクトを開く] あるいは、 [プロジェクトツリー] へのドラッグ＆ドロップで、このファイルを 開くことができます。

 

fsst

SoundMaker のサウンドセットファイル (SoundSet) です。

そのサウンドセットに含まれるサウンド、ウェーブ/シーケンスサウンドセット、
またバンク、波形アーカイブ、グループ、プレイヤーの情報が XML 形式で記述されています。

SoundMaker のプロジェクトツリーでサウンドセットを新規作成すると、作成されます。
新規作成時に、作成フォルダを指定することができます。

SoundMaker の [プロジェクトツリー] のプロジェクトノード下にあるサウンドセットノードから、サウンドセットに含まれる各種情報を確認できます。

 

fbnk

SoundMaker のバンクファイル (Bank) です。

そのバンクに含まれるインストルメントの情報が XML 形式で記述されています。

SoundMaker のバンクリストでバンクを新規作成すると、作成されます。
新規作成時に、作成フォルダを指定することができます。

SoundMaker の [プロジェクトツリー] のサウンドセットノード下にあるバンクノードを選択すると表示されるバンクリストや、
子ノードを選択すると表示される [バンクタブ] で各種情報を確認できます。

 

fseq

SoundMaker のテキストシーケンスファイル (Sequence) です。

テキストシーケンスの情報が NintendoWare 独自のフォーマットで記述されています。

テキストシーケンスファイルはテキストエディタ等で独自に作成してください。
もしくは SoundMaker のシーケンスサウンドリストで SMF 形式のアイテムから変換する事も可能です。
この場合は変換時に作成フォルダを指定することができます。

テキストシーケンスファイルに使用可能な文字コードは、ANSI と Shift_JIS です。

テキストシーケンスの詳細につきましては、シーケンスデータマニュアルの「テキストシーケンス」を参照ください。

 

finl

SoundMaker のインストリストファイル (Inst List) です。

バンクに含まれるインストラベルの ID のリストが定義されています。

バンクファイル (.fbnk) のコンバート時に、バンクファイルと同じ階層に作成されます。 バンクファイル１つにつき、インストリストファイルが１つ作成されます。

テキストシーケンスファイル (.fseq) の先頭でこのファイルをインクルードすることで、インストラベルを使用することができます。
インクルードの例につきましては、「シーケンスデータマニュアル - プリプロセッサ命令 - #include の使用例」を参照ください。

 

コンバート時に一時生成されるファイル

これらのファイルは最終的にサウンドアーカイブファイル (.bfsar) にまとめられます。
そのため、通常このファイルを単体で使用することはありません。

 

bfwav

NintendoWare オーディオのバイナリ波形ファイルです。

波形ファイル (.wav, .aiff) をバイナリ化した情報が含まれます。

プロジェクトファイル (.fspj) を SoundMaker あるいは SoundProjectConverter でコンバートすると、作成されます。
通常、プロジェクトファイルと同じ階層の cache フォルダ以下に作成されます。

 

bfbnk

NintendoWare オーディオのバイナリバンクファイルです。

バンクファイル (.fbnk) をバイナリ化した情報が含まれます。

プロジェクトファイル (.fspj) を SoundMaker あるいは SoundProjectConverter でコンバートすると、作成されます。
通常、プロジェクトファイルと同じ階層の cache フォルダ以下に出力され、
サウンドセットファイル (.fsst) に登録されたバンク１つにつき、バイナリバンクファイルが１つ作成されます。

 

bfseq

NintendoWare オーディオのバイナリシーケンスファイルです。

シーケンスファイル (.fseq) をバイナリ化した情報が含まれます。

プロジェクトファイル (.fspj) を SoundMaker あるいは SoundProjectConverter でコンバートすると、作成されます。
通常、プロジェクトファイルと同じ階層の cache フォルダ以下に出力され、
シーケンスファイル (.fseq or .mid) １つにつき、バイナリシーケンスファイルが１つ作成されます。

(シーケンスサウンドのファイルタイプが「SMF」の場合、一度テキストシーケンスファイル (.fseq) に変換された後で
バイナリシーケンスファイルにコンバートされます。この中間生成ファイルは cache/(テキストシーケンスファイル名).fseq に出力されます。 )

 

bfwsd

NintendoWare オーディオのバイナリウェーブサウンドファイルです。

プロジェクトファイル (.fspj) で指定したサウンドセットファイル (.fsst) に含まれる
ウェーブサウンドに関する情報をバイナリ化した情報が含まれます。

プロジェクトファイルを SoundMaker あるいは SoundProjectConverter でコンバートすると、作成されます。
通常、プロジェクトファイルと同じ階層の cache フォルダ以下に出力され、
サウンドセットファイルに登録されたウェーブサウンドセット１つにつき、バイナリウェーブサウンドファイルが１つ作成されます。

 

bfwar

NintendoWare オーディオのバイナリ波形アーカイブファイルです。

プロジェクトファイル (.fspj) で指定したサウンドセットファイル (.fsst) に含まれる
波形アーカイブに関する情報をバイナリ化した情報が含まれます。

プロジェクトファイルを SoundMaker あるいは SoundProjectConverter でコンバートすると、作成されます。
通常、プロジェクトファイルと同じ階層の cache フォルダ以下に出力され、
サウンドセットファイルに登録された波形アーカイブ１つにつき、バイナリ波形アーカイブファイルが１つ作成されます。

バンク・ウェーブサウンドセットの波形アーカイブ設定で「自動(共有・個別)」を指定している場合、
バンク・ウェーブサウンドセットファイル１つにつき、波形アーカイブファイルが１つ作成されます。

ただし、「自動(共有)」を指定していて、かつグループに属している場合、
グループファイル１つにつき、バイナリ波形アーカイブファイルが１つ作成される場合があります。
バンク・ウェーブサウンドセットの「波形アーカイブ」設定とグループの関係の詳細についてはグループも参照ください。

 

Atk ライブラリで使用するファイル

 

fsid

Atk ライブラリで利用されるサウンド ID ファイルです。

サウンドアーカイブに含まれる、全てのアイテムの名前が C/C++ 形式で記述されています。

プロジェクトファイル (.fspj) を SoundMaker あるいは SoundProjectConverter でコンバートすると、作成されます。
通常、プロジェクトファイルと同じ階層の output フォルダ以下に出力され、
プロジェクトファイル１つにつき、サウンド ID ファイルが１つ作成されます。

アプリケーションプログラムでインクルードし、サウンドの再生やデータのロードにここで定義されている ID を指定することができます。

(ラベル文字列データを使用することで、サウンド ID ファイルをインクルードせずに
Atk ライブラリを使用することもできます。詳しくは「プログラミングガイド」を参照ください。)

 

bfsar

Atk ライブラリで利用されるバイナリサウンドアーカイブファイルです。

プロジェクトファイル (.fspj) で指定した サウンドセットファイル (.fsst) に含まれるアイテムに関する情報をバイナリ化した情報が含まれます。

プロジェクトファイルを SoundMaker あるいは SoundProjectConverter でコンバートすると、作成されます。
通常、プロジェクトファイルと同じ階層の output フォルダ以下に出力され、
プロジェクトファイル１つにつき、バイナリサウンドアーカイブファイルが１つ作成されます。

各種 SoundArchive クラスを利用する事で、サウンドやバンクなどの各種アイテムの情報を取得する事ができます。
また SoundDataManager クラスを利用することで、内部に含まれるサウンドデータをロードすることができます。

 

bfstm

Atk ライブラリで利用されるバイナリストリームファイルです。

プロジェクトファイル (.fspj) で指定したサウンドセットファイル (.fsst) に含まれるストリームサウンドに関する情報をバイナリ化した情報が含まれます。

プロジェクトファイルを SoundMaker あるいは SoundProjectConverter でコンバートすると、作成されます。
通常、プロジェクトファイルと同じ階層の output/stream フォルダ以下に出力され、
サウンドセットファイルに登録されたストリームサウンド１つにつき、バイナリストリームサウンドファイルが１つ作成されます。

ストリームサウンドを再生する際に、含まれるサウンドデータやパラメータがサウンドライブラリで利用されます。

 

bfgrp

Atk ライブラリで利用されるバイナリグループファイルです。

プロジェクトファイル (.fspj) で指定したサウンドセットファイル (.fsst) に含まれる出力方式が [ユーザー管理] のグループの情報をバイナリ化した情報が含まれます。

プロジェクトファイルを SoundMaker あるいは SoundProjectConverter でコンバートすると、作成されます。
通常、プロジェクトファイルと同じ階層の output/userManagementFiles フォルダ以下に出力され、
サウンドセットファイルに登録されたグループのうち出力方式が「ユーザー管理」のグループ１つにつき、
バイナリグループファイルが１つ作成されます。

アプリケーションプログラムで読み込むことで、複数の関連するサウンドを一度に読み込むことができます。

 

SoundMaker でのコンバートの流れ

NintendoWare オーディオ開発環境で使用するファイルフォーマットの概要図です。

ユーザは通常、左端のファイルと右端のファイルを使用します。

SoundMaker のコンバートでは、最終的に「ライブラリで使用するファイル」を生成しますが
その途中で一時的に「コンバート時に一時生成されるファイル」を生成します。

最終的に .bfsar がどのようなバイナリを含んでいるのかは
output/***Map.html ファイルに各種情報が出力されますので、そちらでご確認ください。
