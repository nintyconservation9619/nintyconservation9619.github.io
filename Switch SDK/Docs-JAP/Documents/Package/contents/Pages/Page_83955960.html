プレイヤーヒープ

このセクションでは、プレイヤーヒープの動作と使い方について解説を行います。

プレイヤーヒープとは？

NintendoWare オーディオ開発環境では、Atk ライブラリが行うメモリ管理の方法として、通常サウンドヒープを使います。
サウンドヒープを使う場合、あらかじめシーン毎に必要なサウンドデータをグループという単位でまとめておく必要があります。
例えば、あるシーンで再生される可能性がある BGM 用シーケンスデータが数十曲あった場合、 グループに全てのシーケンスデータを含めておく必要があり、大量のメモリを必要とします。
プレイヤーヒープを使えば、この様な場合に、メモリ消費量を必要最低限に押さえることができます。

サウンドヒープが、サウンドデータを手動でロードするのに対して、 プレイヤーヒープでは、サウンドの再生開始時に自動的にロードを行い、再生が終了した時点で自動的に解放します。
解放されたヒープは、次に再生するサウンド用のデータをロードするために使われます。

プレイヤーヒープの使い方

プレイヤーヒープの設定は、プレイヤーに対して行います。
プレイヤーリストのプレイヤーヒープの項目で、プレイヤーヒープのサイズを指定してください。

シーケンスサウンドを、プレイヤーヒープが設定されたプレイヤーで再生しようとすると、 シーケンスデータが、自動的にプレイヤーヒープにロードされます。
従って、シーケンスデータをグループに含めて、あらかじめサウンドヒープにロードしておく必要は無くなります。

なお、プレイヤーヒープが設定されていても、シーケンスデータが既にサウンドヒープにロードされている場合は、 サウンドヒープ上のデータが使用され、プレイヤーヒープへのロードは行われません。

プレイヤーヒープの注意点

ロードできるデータの種類

プレイヤーヒープには、シーケンスデータのほか、すべてのサウンドデータをロードさせることができます。

消費メモリサイズ

プレイヤーヒープ用のメモリ領域は、サウンドリミットの個数分確保されます。
例えば、サウンドリミットが 3 だった場合、プレイヤーヒープで指定したサイズの 3 倍、メモリを消費します。
また、プレイヤーヒープ用のメモリ領域は、初期化時に、常駐領域として確保されます。

再生開始遅延

プレイヤーヒープを使うと、再生開始時にデータをロードします。
実際に音が鳴り始めるのは、ロードが完了した後なので、音の鳴り始めに遅延が発生します。

再生開始時に即座に音を鳴らしたい場合は、プログラム側で、再生準備処理（Prepare 処理）を行うことで、 前もってプレイヤーヒープにデータをロードしておくことができます。
