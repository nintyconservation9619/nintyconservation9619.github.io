動作確認環境

マスター ROM を提出する前の最終的な動作確認は、以下の環境で行ってください。

  • マスターROM提出可能な SDK とファームウェアの使用
  • 動作確認が必要なコントローラーファームウェアの使用
  • PROD MODE に設定した EDEV の使用
  • NX開発カードの使用

マスターROM提出可能な SDK とファームウェアの使用

マスター ROM を提出できる SDK や動作確認が必要なファームウェアのバージョンは、その時点でリリースされている SDK や時期によって変わります。
Nintendo Developer Portal の「マスター提出関連情報」ページに記載されている SDK やファームウェアを使用して動作確認を行ってください。

動作確認が必要なコントローラーファームウェアの使用

動作確認が必要なコントローラーファームウェアのバージョンは、その時点でリリースされているファームウェアや時期によって変わります。
Nintendo Developer Portal の「マスター提出関連情報」ページに記載されているファームウェアの中の最新バージョンに含まれているものを使用して動作確認してください。

コントローラーファームウェアについては、以下のページを参照してください。

  • コントローラーファームウェアのリリースノート
  • DevMenu／コントローラー管理

IRSENSOR ライブラリを使用しているアプリケーションは、使用する SDK のバージョンに応じて必要となるコントローラファームウェアバージョンが異なるため、アプリケーションの動作中にコントローラーファームウェアの更新が発生する場合があります。コントローラーファームウェアの更新が発生してもアプリケーションが問題なく動作することを確認するために、製品ローンチ時点のファームウェアバージョンのコントローラーを接続した場合の挙動も確認してください。

コントローラーファームウェアを製品ローンチ時点のバージョンに巻き戻す方法については、「デバッグを目的としたコントローラーファームウェアの巻き戻し」を参照してください。

PROD MODE に設定した EDEV の使用

EDEV は、デバッガの使用や TargetManager からの接続など開発用途の機能をいくつか使用することができますが、これらの機能を無効化して製品版とほぼ同じ動作をする「PROD MODE」というモードに切り替えることができます。アプリケーションはこの PROD MODE に設定した EDEV を使って最終動作確認を行うようにしてください。

PROD MODE の詳細や設定方法については、以下のページを参照して下さい。

  • 製品機に近いファームウェア(PROD MODE)への初期化

PROD MODE でのみアプリケーションが止まるなどの問題が起こる場合、製品では使用できない開発用機能 (HTC ライブラリ や PCM ライブラリ など) を呼び出してしまっていないか、確認してください。

NX開発カードの使用

カード版で販売するアプリケーションを開発する場合は、必ず NX開発カードを使用して動作確認を行ってください。

ダウンロード版のみで販売するアプリケーションの場合はこの動作確認は必須ではありませんが、ダウンロード版とカード版を併売する場合は必須です。
