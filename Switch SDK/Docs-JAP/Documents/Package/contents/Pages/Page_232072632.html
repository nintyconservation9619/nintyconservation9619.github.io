はじめに

WebCallerCommand は Webアプレット・オフラインWebアプレット をコマンドラインで呼び出すためのターゲット向けツールです。
以下の操作が実行可能です。

  • Webアプレットの起動
  • オフラインWebアプレットの起動

使用方法

RunOnTarget を利用してツールを起動します。

$ Tools\CommandLineTools\RunOnTarget.exe TargetTools\NX-NXFP2-a64\WebCallerCommand\Release\WebCallerCommand.nsp -- --help
usage: WebCallerCommand <command> [<args>]

The WebCallerCommand commands are:
   web <requestUrl>       ShowWebApplet
   offline <documentPath> ShowOfflineApplet
[SUCCESS]

Webアプレットの呼び出し

web コマンドを使用して「Webアプレットの呼び出し」を行うことが出来ます。

$ Tools\CommandLineTools\RunOnTarget.exe TargetTools\NX-NXFP2-a64\WebCallerCommand\Release\WebCallerCommand.nsp -- web --help
usage: WebCallerCommand web <requestUrl>
   [--callback-url <callbackUrl>]
   [--callbackable-url <callbackableUrl>]
   [--boot-as-media-player <true|false>]
   [--media-player-auto-close-enabled <true|false>]
   [--web-audio-enabled <true|false>]
   [--pointer-enabled <true|false>]
   [--left-stick-mode <SpatialNavi|StickPointer>]
   [--user-index <userIndex>]
   [--boot-display-kind <Default|White|Black>]
   [--background-kind <Normal|ApplicationCapture|ApplicationCaptureBlur>]
   [--js-extension-enabled <true|false>]
   [--display-url-kind <FullPath|Domain>]
   [--user-agent-additional-string <userAgentAdditionalString>]] 
   [--touch-enabled-on-contents <true|false>]
   [--page-cache-enabled <true|false>]

   [--page-fade-enabled <true|false>]
   [--page-scroll-indicator-enabled <true|false>]
[SUCCESS]

コマンド、オプションで指定したパラメータは nn::web::ShowWebPageArg の各 API を介し、呼び出し時のパラメータとして指定されます。

コマンド、オプションで指定できるパラメータの詳細については、呼び出される API をご参照ください。

各コマンドの引数は次の表の通りです。

requestUrl nn::web::ShowWebPageArg() 開くページの URL を指定してください。

実例

http://www.nintendo.com を 0番目のインデックスのユーザーで開きます。

Tools\CommandLineTools\RunOnTarget.exe TargetTools\NX-NXFP2-a64\WebCallerCommand\Release\WebCallerCommand.nsp -- web http://www.nintendo.com --user-index 0

http://www.nintendo.com を開き http://www.nintendo.com/countryselector に到達したら終了するよう指定し、開きます。

Tools\CommandLineTools\RunOnTarget.exe TargetTools\NX-NXFP2-a64\WebCallerCommand\Release\WebCallerCommand.nsp -- web http://www.nintendo.com --callback-url http://www.nintendo.com/countryselector

オフラインHTMLのパス

WebCallerCommand では ROM に付加されたオフラインHTML を開くかわりに、PC または SD カードのいずれかに置かれたオフラインHTML を開くことができます。

Webアプレット・オフラインWebアプレットは、標準でホストPCの "C:/NXWebApplet" というパスをルートパスとしてマウントしています。マウントするホストPCのパスは、設定マネージャを使って変更することができます。詳しくはルートパスとしてマウントするホストPCのパスを変更するを参照してください。オフラインHTMLを PC に置く場合は、マウントしているホストPCのパスに html-document フォルダを作成し、そのフォルダ以下に置いてください。

SD カードに 置く場合は SD カード直下に html-document フォルダを作成し、そのフォルダ以下に置いてください。

指定するオフラインHTML のパスは上記のフォルダからの相対パスを指定してください。例えば sample.htdocs/index.html を上で記載したフォルダに置くと、documentPath に指定するパスは PC , SD カードどちらも sample.htdocs/index.html となります。PC , SD カードどちらにも同じファイルがある場合、PC に置かれたオフラインHTML が優先されます。

実例

sample.htdocs フォルダにある index.html を開きます。

Tools\CommandLineTools\RunOnTarget.exe TargetTools\NX-NXFP2-a64\WebCallerCommand\Release\WebCallerCommand.nsp -- offline sample.htdocs/index.html

sample.htdocs フォルダにある index.html を 左スティックモードをポインターに切り替える事を禁止、フッター非表示 に指定して開きます。

Tools\CommandLineTools\RunOnTarget.exe TargetTools\NX-NXFP2-a64\WebCallerCommand\Release\WebCallerCommand.nsp -- offline sample.htdocs/index.html --pointer-enabled false --footer-enabled false
