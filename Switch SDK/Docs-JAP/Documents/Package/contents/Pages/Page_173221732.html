プロジェクトで使用する SDK のバージョンを取得する

はじめに

このページでは、NintendoSDK アプリケーションプロジェクト・ライブラリプロジェクトにおいて、参照する SDK のバージョン情報を取得する方法を説明します。

バージョン情報を取得する

Visual Studio のウィザードから新規作成したプロジェクト、および NintendoSDK が提供するプロパティシートをインポートしているプロジェクトでは、プロジェクトが参照する NintendoSDK のバージョン情報を取得することができます。バージョン情報は次のプロパティにより定義されます。

  • NintendoSdkVersionMajor
  • NintendoSdkVersionMinor
  • NintendoSdkVersionMicro

NX Addon パッケージをインストールしている場合は、次のプロパティも定義されます。

  • NintendoSdkNXAddonVersionMajor
  • NintendoSdkNXAddonVersionMinor
  • NintendoSdkNXAddonVersionMicro

各プロパティの詳細な定義については NintendoSDK のプロパティシート を参照してください。

バージョン情報を使用する

NintendoSDK のバージョン情報によってリンクするライブラリを自動的に切り替えるプロパティシートの例を示します。この例では、NintendoSDK のバージョンが 1.0.0 以降であれば、架空のライブラリ example をアプリケーションに自動的にリンクするよう設定しています。

<?xml version="1.0" encoding="utf-8"?>
<Project ToolsVersion="4.0" xmlns="http://schemas.microsoft.com/developer/msbuild/2003">
  <ItemDefinitionGroup>
    <Link>
      <AdditionalDependencies Condition="$(NintendoSdkVersionMajor) &gt; 0">libnn_example$(NintendoSdkLibSuffix);%(AdditionalDependencies)</AdditionalDependencies>
    </Link>
  </ItemDefinitionGroup>
</Project>
