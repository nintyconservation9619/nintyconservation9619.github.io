拡張ユーザ情報のプリセット

メリット

拡張ユーザ情報のプリセット定義を使うと、一覧リストから選択して拡張ユーザ情報を簡単に入力できます。
また、名前の打ち間違えを避けられたり、プリセット定義の作成者が書いた説明文章から拡張ユーザ情報の仕様を確認することができます。

プリセット定義の書き方

部品ルートフォルダ以下に存在する、「プロジェクト定義ファイル(NW_LytProject.flpj)」に下記のように記述します。

<?xml version="1.0" encoding="utf-8"?>
<ProjectSettings xmlns:xsd="http://www.w3.org/2001/XMLSchema" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance">
<ExUserDataPresets>
  <ExUserDataPreset>
    <Name>DataName</Name>
    <Type>String</Type>
    <Descriptions>このユーザデータの説明文章です。</Descriptions>
  </ExUserDataPreset>
</ExUserDataPresets>
</ProjectSettings>

設定したプリセットが、「プロパティウィンドウ」の「拡張ユーザ情報ページ」で一覧リストに表示されます。

ExUserDataPreset 要素

Name 要素と、Type 要素を持ちます。
Type 要素には、Float、Int、String のいづれかを指定します。

対象ペインを限定する

PaneKindLimitations 要素を記述するとプリセットの対象を特定種類のペインに限定することができます。
記述を省略すると、すべてのペイン種類が対象になります。
ただし、同時に ControlNameLimitations が記述されている場合、コントロール向けのプリセットとみなしてペインを対象としません。
PaneKind には Picture, Textbox, Window, Parts, Bounding, Null のいずれかを指定します。

<ExUserDataPresets>
  <ExUserDataPreset>
    <Name>DataName</Name>
    <Type>String</Type>
    <Descriptions>このユーザデータの説明文章です。</Descriptions>
    <PaneKindLimitations> <PaneKind>Null</PaneKind> <PaneKind>Picture</PaneKind> </PaneKindLimitations>
  </ExUserDataPreset>
</ExUserDataPresets>

対象コントロールを限定する

ControlNameLimitations 要素を記述するとプリセットの対象を特定のコントロールに限定することができます。
string要素に、コントロール名を指定します。
ただし、記述省略時に、同時に PaneKindLimitations が記述されている場合、ペイン向けのプリセットとみなしてすべてのコントロールを対象としません。

<ExUserDataPresets>
  <ExUserDataPreset>
    <Name>DataName</Name>
    <Type>String</Type>
    <Descriptions>このユーザデータの説明文章です。</Descriptions>
    <ControlNameLimitations> <string>Locator</string> <string>NormalButton</string> </ControlNameLimitations>
  </ExUserDataPreset>
</ExUserDataPresets>

レイアウトやアニメーション区間タグに適用できるようにする。

ApplicableToLayout 要素によりレイアウトへ適用するかどうかを指定できます。ApplicableToAnimTag 要素によりアニメーション区間タグへ適用するかどうかを指定できます。それぞれ、true または false を設定します。

1

<ExUserDataPresets>
  <ExUserDataPreset>
    <Name>DataName</Name>
    <Type>String</Type>
    <Descriptions>このユーザデータの説明文章です。</Descriptions>
    <ApplicableToLayout>true</ApplicableToLayout>
    <ApplicableToAnimTag>true</ApplicableToAnimTag>
  </ExUserDataPreset>
</ExUserDataPresets>

注意

PaneKindLimitations と ControlNameLimitations を同時に指定すると、特定ペインと特定コントロールの両方を対象にするプリセットが設定できます。

ダミーの ControlNameLimitations の活用

<ControlNameLimitations> <string>__Dummy__</string> </ControlNameLimitations> のような存在しないコントロールを対象とする記述でレイアウトやアニメーション区間タグだけに適用できるプリセットを作ることができます。

選択肢リスト

EnumValues要素を使うと、あらかじめ用意しておいた選択肢リストから値を入力することができます。
簡単に入力ができ、書き間違えを無くすことができます。
EnumValue要素に、選択肢を文字列で指定します。(数字を指定してもかまいません。) Description属性には各選択子の意味などの説明を記載します（省略可能です）。
MultiSelect が true のときは、一つ以上の要素が選択できます。複数の要素を選択した場合、flyt ファイルには Type が String の値は改行区切りで保存され、Type が Float, Int の値は空白区切りで保存されます。

<ExUserDataPresets>
  <ExUserDataPreset>
  <Name>DataName</Name>
    <Type>String</Type>
    <Descriptions>選択肢リストを使ったプリセットです。</Descriptions>
    <MultiSelect>false</MultiSelect>
    <EnumValues>
        <EnumValue Descriptions="This is an ItemA's descriptions.">ItemA</EnumValue>
        <EnumValue>ItemB</EnumValue>
    </EnumValues>
  </ExUserDataPreset>
</ExUserDataPresets>

拡張ユーザ情報プリセット定義ファイル(flup)

flpj ファイル以外にも、flup ファイルにプリセットを書くことができます。定義ファイルを複数に分けて管理したい場合にご利用ください。

部品ルートフォルダ(flpj ファイルと同じフォルダ)にある flup 形式のファイルがすべて読み込まれます。

同名のプリセットが発見された場合は後から読み込まれた設定が採用されます。
例えば、全体ルールを決める flup と、チーム独自のルールを決める flup のようにファイルを分割して運用することで、後から読み込んだ独自ルールによって全体ルールをオーバーライドするような利用をすることができます。

ファイル名のアルファベット順（大文字・小文字の区別無し）に読み込まれるので、順番調整は、ファイル名でおこなってください。
