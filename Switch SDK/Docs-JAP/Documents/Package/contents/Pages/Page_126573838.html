はじめに

NintendoSDK 構成マネージャとは

NintendoSDK 構成マネージャは、Visual C++ プロジェクトから NintendoSDK を使用するために必要な各種設定を行うことができる Visual Studio 拡張機能です。

Visual C++ プロジェクトから NintendoSDK を使用するためには、NintendoSDK が提供する特定のプロパティシートをインポートしたり、使用する NintendoSDK の機能をプロパティで定義したりといった作業が必要となります。NintendoSDK 構成マネージャは、これらの設定を Visual Studio 上で簡単に行うための機能を提供します。具体的に提供される機能は以下のとおりです。

  • プロジェクトで NintendoSDK を使用するかどうかの切り替え
  • 使用する NintendoSDK のルートパスの変更
  • 使用する NintendoSDK の API セット、ビルドタイプの変更

対象読者

Visual Studio をビルドシステムとして使用するすべての開発者を対象としています。特に以下の Visual Studio プロジェクトを管理している開発者は、NintendoSDK 構成マネージャを使用してプロジェクトを構成することを推奨します。

  • NintendoSDK を使用していない一般的な Windows 用プロジェクト
  • NintendoSDK の新規プロジェクト作成ウィザードを使用して作成されたプロジェクト

なお、プロジェクトを構成するために NintendoSDK 構成マネージャは必須ではありません。開発者自身の手で必要なプロパティの定義やプロパティシートのインポート等を行ってプロジェクトを構成することも可能です。

インストール方法

NDI を使用して NintendoSDK VsExtension パッケージをインストールしてください。インストール済みの Visual Studio 2013, 2015, 2017 の各バージョンに NintendoSDK 構成マネージャが追加されます。インストール時に Visual Studio を起動していた場合は、再起動を行ってください。

拡張機能が正常に追加されたかどうかは、Visual Studio の [ツール(T)] → [拡張機能と更新プログラム...(U)] を選択することで確認できます。「NintendoSDK Visual Studio Extension」という項目が存在すれば正常に追加されています。

参考：

環境によっては NintendoSDK VsExtension パッケージのインストールに失敗することがあります。トラブルシューティング のページにいくつかの問題について解決方法を掲載していますのでご確認ください。

アンインストール方法

NDI を使用して NintendoSDK VsExtension パッケージをアンインストールしてください。

使用方法

NintendoSDK 構成マネージャを起動するには、Visual Studio 上で [ツール(T)] → [NintendoSDK] → [NintendoSDK Configurations...] を選択してください。以下のようなウインドウが開きます。

現在 Visual Studio に開かれているソリューションのプロジェクトごと、およびプロジェクト構成ごとに、NintendoSDK に関する設定項目が表示されます。項目は以下の通りです。

項目 説明
NintendoSDK 使用 NintendoSDK を使用する場合にチェックを入れます。チェックを入れると NintendoSDK のプロパティシートがインポートされ、インクルードパスやライブラリパスなどの設定がプロジェクトに追加されます。
SDK パス 使用する NintendoSDK のルートパスです。
API セット 使用する NintendoSDK の API セットです。ここで選択可能な項目は、SDK パスに指定された NintendoSDK の Addon パッケージ構成により異なります。
SDK ビルドタイプ 使用する NintendoSDK のビルドタイプです。詳細は ビルドターゲットとビルドタイプ を参照してください。

[OK] を選択すると、設定が各プロジェクトに反映されます。

参考：
  • 設定を正しく反映するため、NintendoSDK 構成マネージャは [OK] を選択した直後に現在開かれているソリューションを一度閉じて再度開きなおします。編集中のファイルが存在する場合は自動的に保存されます。
  • 設定の反映時、NintendoSDK 構成マネージャは各プロジェクトファイルと同じディレクトリにいくつかのファイルを生成します。詳細は NintendoSDK 構成マネージャがプロジェクトに行う設定 を参照してください。

トラブルシューティング

「現在のプロジェクトは、古い NintendoSDK の方式で構成されています」というダイアログが表示される

NintendoSDK 構成マネージャを開いた直後に、以下のダイアログが表示される場合があります。

このダイアログは、NintendoSDK 0.15.0 以前の古い SDK を使用して作成されたプロジェクトに対し表示される場合があります。

NintendoSDK 構成マネージャは、開かれているプロジェクトとは異なる方式で SDK パスなどの設定を保存するため、NintendoSDK 構成マネージャが編集可能なプロジェクト形式に更新する必要があります。[OK] を選択してプロジェクトを新しい形式に更新してください。

「プロジェクトに NintendoSDK の構成を適用できませんでした」というダイアログが表示される

NintendoSDK 構成マネージャのウインドウで [OK] を選択すると、以下のダイアログが表示される場合があります。

このダイアログは、NintendoSDK 構成マネージャが行った設定の変更が正常に反映されていないプロジェクトが存在する場合に表示されます。

設定が反映されない原因として、NintendoSDK 構成マネージャがプロジェクトファイル内に追加するプロパティの値を開発者が上書き定義している場合が挙げられます。ダイアログに表示されたプロパティが上書き定義されていないかどうかご確認ください。

NintendoSDK 構成マネージャが定義するプロパティの詳細については、NintendoSDK 構成マネージャがプロジェクトに行う設定 を参照してください。

NintendoSDK 構成マネージャがプロジェクトに行う設定

NintendoSDK 構成マネージャのウインドウで [OK] を選択すると、NintendoSDK 構成マネージャは現在開かれている各プロジェクトに対して以下を行います。

古いプロパティシートのインポートを削除

プロジェクトファイル内に NintendoSDK が過去に提供していた以下のプロパティシートがインポートされていた場合、それらはインポートされなくなります。

  • NintendoSdkSpec_*.props
  • NintendoSdkBuildType_*.props
  • NintendoSdkVcProjectSettings.props

(* に入る文字列はプロジェクトファイルにより異なります)

インポートされなくなったプロパティシートは、プロジェクトから削除して問題ありません。

プロパティの定義

「NintendoSDK 使用」にチェックを入れた場合は、プロジェクトファイル内で次のプロパティを定義します。

  • NintendoSdkRoot
  • NintendoSdkSpec
  • NintendoSdkBuildType

「NintendoSDK 使用」のチェックを外した場合は、これらのプロパティは定義されなくなります。各プロパティの仕様については NintendoSDK のプロパティシート を参照してください。

プロパティシートのインポート

「NintendoSDK 使用」にチェックを入れた場合は、プロジェクトファイルに ImportNintendoSdk.props というプロパティシートがインポートされるようになります。

ImportNintendoSdk.props は NintendoSDK を使用するための各種設定を行う機能を持ったプロパティシートであり、NintendoSDK 構成マネージャによってプロジェクトファイルと同じディレクトリに生成されます。生成された ImportNintendoSdk.props は削除せず、常にプロジェクトファイルと同じディレクトリに配置するようにしてください。

「NintendoSDK 使用」のチェックを外した場合は、プロパティシートがインポートされなくなります。

NintendoSDK 構成マネージャ設定ファイルの生成

プロジェクトファイルと同じディレクトリに「<プロジェクトファイル名>.nnsdk.xml」というファイル名で NintendoSDK 構成マネージャの設定ファイルが生成されます。このファイルは、「NintendoSDK 使用」のチェックを外したプロジェクト構成の設定値を保存するために使用されます。
