プログラマ向け グラフィックスの変更点

NintendoSDK 0.17.0 で行われた NintendoWare グラフィックスのプログラマ向け変更点について説明します。
NintendoWare グラフィックスは、nn::g3d、nn::ui2d、nn::font、nn::vfx モジュールのことを指します。

  • プログラマ向け グラフィックスの変更点
  • nn::ui2d、nn::font ニュース
    • LayoutArchiver に、テクスチャのタイリング方法を指定するオプション（tile-optimize と tile-size-threshold）を追加

nn::ui2d、nn::font ニュース

LayoutArchiver に、テクスチャのタイリング方法を指定するオプション（tile-optimize と tile-size-threshold）を追加

LayoutArchiver に、テクスチャのタイリング方法を指定するオプション（tile-optimize と tile-size-threshold）を追加しました。

仕様は、{{DocLink=83956254}} テクスチャコンバータ の同じオプションに準拠しています。

サイズが小さく、２のべき乗ではないサイズのテクスチャでサイズの差が出ます。

Samples\Sources\Resources\Ui2d\cannonButton\CannonButton_00.flyt の全テクスチャで tile-optimize=size_auto を指定して比較した場合、以下のようになります。

  • tile-size-threshold=5 の場合、96.1 KB
  • tile-size-threshold=100 の場合 72.1 KB

(tile-size-thresholdは大きな値ほど、サイズを優先して変換をおこなうようになります)
