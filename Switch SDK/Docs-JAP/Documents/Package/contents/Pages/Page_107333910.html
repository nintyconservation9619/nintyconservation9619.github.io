プラットフォームプリセットファイル

レイアウトエディタ用プラットフォームプリセットファイルとは

プラットフォームプリファレンスファイルでは、対象プラットフォームやビューアの指定などを行います。

Tools\Graphics\LayoutEditor\Platforms\ 内に .preset ファイルが配置されます。

 

─ Platformss ┬ Generic.preset
               └ PlatformAAA.preset

レイアウトエディタ用プラットフォームプリセットファイルフォーマット

プラットフォームプリセットファイル (.preset) はXML形式のファイルです。

 <PlatformPreference>

プラットフォームプリセットファイルのルート要素です。

内容

以下の要素を含みます。

  • <PlatformName> : プラットフォーム名の設定です。最大1個出現します。
  • <UiPlatformName> : UI プラットフォーム名の設定です。最大1個出現します。
  • <ArchiverPlatformName> : アーカイバで指定するプラットフォーム名の設定です。最大1個出現します。
  • <AcceptableBntxPlatformNames> : レイアウトエディタで読み込む bffnt 内の bntx のプラットフォーム名の設定です。最大1個出現します。
  • <PcViewerPath> : PC ビューアのパスの設定です。最大1個出現します。
  • <TargetViewerRunnnerPath> : 実機ビューアのパスの設定です。最大1個出現します。
  • <PreviewStarterPath> : プレビュースターターのパスの設定です。最大1個出現します。
  • <PcTransferProtocol> : PC ビューアとの通信プロトコルの設定です。最大1個出現します。
  • <TargetTransferProtocol> : 実機ビューアとの通信プロトコルの設定です。最大1個出現します。
  • <PcPeerType> : PC ビューアとの通信に使うピアタイプの設定です。最大1個出現します。
  • <TargetPeerType> : 実機ビューアとの通信に使うピアタイプの設定です。最大1個出現します。
  • <PCConverterPath> : PC ビューアへ送信するレイアウトバイナリを作成するためのコンバータのパスの設定です。最大1個出現します。
  • <TargetConverterPath> : 実機ビューアへ送信するレイアウトバイナリを作成するためのコンバータのパスの設定です。最大1個出現します。
  • <ArchiverPath> : レイアウトアーカイバのパスの設定です。最大1個出現します。
  • <PreArchiverCommandAdditionalArguments>  : レイアウトアーカイバ実行前コマンドの追加引数です。最大1個出現します。
  • <FontConverterConsolePath> : コンソール版のフォントコンバータのパスの設定です。最大1個出現します。
  • <SearchPCViewerByTitle> : PC ビューアの探索方法の設定です。
  • <IsNW4FMode> : NW4F モードの設定です。最大1個出現します。

<PlatformName>

プラットフォーム名の設定です。

内容

プラットフォーム名を指定します。

<UiPlatformName>

UI プラットフォーム名の設定です。

内容

UI プラットフォーム名を指定します。他の .preset ファイル内の UiPlatformName と重複してはいけません。

<ArchiverPlatformName>

アーカイバで指定するプラットフォーム名の設定です。

内容

アーカイバで指定するプラットフォーム名を指定します。

<AcceptableBntxPlatformNames>

レイアウトエディタで読み込む bffnt 内の bntx のプラットフォーム名の設定を含みます。

内容

以下の要素を含みます。

  • <AcceptableBntxPlatformName> : レイアウトエディタで読み込む bffnt 内の bntx のプラットフォーム名の設定です。

<AcceptableBntxPlatformName>

レイアウトエディタで読み込む bffnt 内の bntx のプラットフォーム名の設定です。

内容

レイアウトエディタで読み込む bffnt 内の bntx のプラットフォーム名を指定します。

 

<PcViewerPath>

PC ビューアのパスの設定です。

内容

PC ビューアを起動する実行ファイルまたはバッチファイルのパスを指定します。

<TargetViewerRunnnerPath>

実機ビューアのパスの設定です。

内容

実機ビューアを起動する実行ファイルまたはバッチファイルのパスを指定します。

<PreviewStarterPath>

プレビュースターターのパスの設定です。

内容

プレビュースターターを起動する実行ファイルまたはバッチファイルのパスを指定します。

<PcTransferProtocol>

PC ビューアとの通信プロトコルの設定です。

内容

PC ビューアとの通信プロトコルを指定します。HTCS または HIO を指定します。

<TargetTransferProtocol>

実機 ビューアとの通信プロトコルの設定です。

内容

実機 ビューアとの通信プロトコルを指定します。HTCS または HIO を指定します。

<PcPeerType>

PC ビューアとの通信に使うピアタイプの設定です。

内容

PC ビューアとの通信に使うピアタイプを指定します。

<TargetPeerType>

ターゲットビューアとの通信に使うピアタイプの設定です。

内容

ターゲットビューアとの通信に使うピアタイプを指定します。

<PCConverterPath>

PC ビューアへ送信するためのレイアウトバイナリを作成するためのコンバータのパスの設定です。

内容

PC ビューアへ送信するためのレイアウトバイナリを作成するためのレイアウトコンバータのパスを指定します。

<TargetConverterPath>

実機ビューアへ送信するレイアウトバイナリを作成するためのコンバータのパスの設定です。

内容

実機ビューアへ送信するレイアウトバイナリを作成するためのレイアウトコンバータのパスを指定します。

<ArchiverPath>

レイアウトアーカイバのパスの設定です。

内容

レイアウトアーカイバのパスを指定します。

<PreArchiverArchiverCommandAdditionalArguments>

レイアウトアーカイバ実行前コマンドの追加引数の設定です。

内容

レイアウトアーカイバ実行前コマンドの追加引数を指定します。レイアウトアーカイバ実行前コマンドはレイアウトプロジェクトファイル (.flpj) の <PreArchiverCommandPath> で指定します。コマンドの第一引数はアーカイブ対象フォルダが渡され、第二引数以降には、追加引数が渡されます。

<FontConverterConsolePath>

コンソール版フォントコンバータのパスの設定です。

内容

コンソール版フォントコンバータのパスを指定します。

<SearchPCViewerByTitle>

PC ビューアの探索方法の設定です。

内容

PC ビューアの探索方法を指定します。true の場合は、<PcViewerPath> で指定された PC ビューアに限らず、ウィンドウのタイトルを参考にして PC ビューアへの接続を試みます。

<IsNW4FMode>

 NW4F モードの設定です。

内容

true または false を指定します。

Generic.preset の例

<?xml version="1.0"?>
<PlatformPreference>
  <PlatformName>Generic</PlatformName>
  <UiPlatformName>Win</UiPlatformName>
  <ArchiverPlatformName>Win</ArchiverPlatformName>
  <AcceptableBntxPlatformNames>
    <AcceptableBntxPlatformName>Gen</AcceptableBntxPlatformName>
  </AcceptableBntxPlatformNames>
  <PcViewerPath>LayoutViewer\Win64\lytviewerGfx.exe</PcViewerPath>
  <TargetViewerRunnnerPath>LayoutViewer\Cafe\run_viewer.bat</TargetViewerRunnnerPath>
  <PcTransferProtocol>HTCS</PcTransferProtocol>
  <PcPeerType>Generic-Windows</PcPeerType>
  <TargetTransferProtocol>HIO</TargetTransferProtocol>
  <TargetConverterPath>LayoutTools\LayoutConverter.exe</TargetConverterPath>
  <PCConverterPath>LayoutTools\LayoutConverter.exe</PCConverterPath>
  <ArchiverPath>LayoutTools\LayoutArchiver.exe</ArchiverPath>
  <FontConverterConsolePath>FontConverter\FontConverterConsole.exe</FontConverterConsolePath>
  <IsNW4FMode>false</IsNW4FMode>
  <PcConverterAdditionalArguments></PcConverterAdditionalArguments>
  <ArchiverAdditionalArguments>--tile-mode linear --api-type Gl --code-type Source</ArchiverAdditionalArguments>
  <FontConverterAdditionalArguments>-tile-mode linear</FontConverterAdditionalArguments>
</PlatformPreference>
