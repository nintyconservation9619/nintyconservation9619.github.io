INIT ライブラリについて

はじめに

本章では、NintendoSDK における 初期化ライブラリ（以後 INIT ライブラリ）について説明します。

対象読者

NintendoSDK を使ってアプリケーションを開発される方を対象としています。

関連資料

nn::init 名前空間のリファレンス と併せて参照して下さい。

INIT ライブラリの機能

INIT ライブラリは、アプリケーションを実装する上で必要となるプログラム起動時の初期化に必要な機能を提供します。

現時点では主に、メモリヒープの設定 や メモリアロケータの初期化 を行なうのに必要な機能を提供しています。

INITライブラリの提供形態

INIT ライブラリの機能は以下の２つのファイル群によって提供されます。

  • ヘッダファイル
  • ライブラリファイル

ヘッダファイルには、INIT ライブラリを利用する上で必要となる型や定数、関数プロトタイプ宣言などが含まれます。アプリケーションのソースコードからは以下のように記述することで、これらのヘッダファイルをインクルードすることが出来ます。

#include <nn/init.h>

 

ライブラリファイルには、INIT ライブラリ関数の実行バイナリ本体が含まれます。ライブラリは libnn_init という名前で提供されています。

INIT ライブラリが提供する機能

INIT ライブラリは、以下に示す機能を提供しています。それぞれの関数の詳細機能については、個々のリファレンスを参照して下さい。

スタートアップ初期化ルーチン

以下に示す関数が提供されています。

関数名 説明
nninitStartup()

スタートアップ初期化用ルーチンです。C リンケージで定義されます。

libnn_init ライブラリには、デフォルトの nninitStartup() が定義されています。

詳細は「プログラムの起動と初期化」を参照して下さい。

メモリアロケータ初期化機能

以下に示す関数が提供されています。

関数名 説明
nn::init::InitializeAllocator()

メモリアロケータを初期化します。

詳細は「プログラムの起動と初期化」も合わせて参照して下さい。

 

INIT ライブラリの機能に関するサンプルプログラム

サンプル名 説明
nninitStartup サンプルプログラムその 1

アプリケーションが独自の nninitStartup() 関数を定義する場合のサンプルです。
nninitStartupWithMalloc サンプルプログラムその 2 アプリケーションが独自の nninitStartup() 関数を定義し、かつ malloc(), free(), calloc(), realloc() も独自の実装を行なう場合のサンプルです。
