プログラマ向け グラフィックスの変更点

NintendoSDK 1.1.0 で行われた NintendoWare グラフィックスのプログラマ向け変更点について説明します。
NintendoWare グラフィックスは、nn::g3d、nn::ui2d、nn::font、nn::vfx モジュールのことを指します。

nn::g3d ニュース

NXAddon0.12.x で作成されたリソースデータのサポート

NXAddon0.12.x で作成されたシェーダーアーカイブファイル (.bfsha) はバイナリのメジャーバージョンが異なるため互換性がありませんでしたが、リソースデータとして扱えるように対応しました。これにより NXAddon1.0.0 で作成したシェーダーアーカイブファイルが扱えなくなっています。新しい 3D シェーダーコンバータで再コンバートが必要です。このパッケージの g3d ランタイムで扱えるリソースデータは以下になります。 

  • NXAddon0.12.x で作成されたリソースデータ 
  • NXAddon1.1.0 作成されたリソースデータ

nn::vfx ニュース

VfxBinarySegmentation サンプルの追加

エフェクトバイナリ内に含まれる テクスチャ、G3dプリミティブ、シェーダ を出力時に個別ファイルとして出力し、ランタイム上で組み合わせて利用するサンプルを追加しました。

G3dバイナリを分割して出力する 3Dバイナリをファイル分割出力した場合の運用方法について説明があります。

ShaderConverter を利用する際に ジョブ数 を指定できるように修正

EffectMaker を利用してバイナリコンバートを行う際に、 -j で指定されるジョブ数を ShaderConverter に渡すよう修正しました。

参考：

ShaderConverter に渡されるジョブ数が正しく動作するかは、--api-type で指定する api に依存します。
また、ジョブ数を指定しない場合、ShaderConverter 側の仕様に依存します。
詳しくは、ShaderConverter ドキュメントを参照ください。

 

ストライプの機能改善に伴う互換性の問題と回避方法

本リリース以前にストライプ機能を使用して作成されたデータを再生する際に以下の点に注意が必要です。

  • ストライプ利用時に カラータブ / ソフトパーティクル / 遮蔽チェック が動作するようになりました。
  • ストライプ利用時に 回転タブ / Ｚ軸 が動作するようになりました。

この変更により、これまでに作成したデータが意図せず描画結果が変わってしまう可能性があります。

import glob
from xml.etree.ElementTree import *

print("");
print("");
print("-------------------------------------------------------");
files = glob.glob('C:/data/*.eset');
print("find eset using StripeHistory [StripeType == 1 or 2]");
for file in files:
    print("-----------------------");
    print(file);
    tree = parse(file);
    elem = tree.getroot();
    for emitter in elem.findall("./EmitterList/EmitterData"):
        item = emitter.find(".//ReservedShader/PageData/ContentsData");
        if(item == None):
            continue;
        print("  " + item.tag);
        prop = item.get('{http://www.w3.org/2001/XMLSchema-instance}type')
        if(prop == None):
            print("not found xsi:type");
            continue;
        if(prop != "EPStripeHistoryData" and prop != "EPStripeHistoryTailData"):
            continue;
        print("    xsi:type = " + prop);
        stripeTypeNode = item.find(".//StripeType");
        if(stripeTypeNode == None):
            print("-----");
            continue;
        print("    StripeType = " + stripeTypeNode.text);
        if(stripeTypeNode.text != "1" and stripeTypeNode.text != "2"):
            continue;
        print("found file = " + file);

nn::ui2d ニュース

リソースアクセサから指定したタイプのリソース名をすべて検索する機能の追加

nn::ui2d::ArcResourceAccessor および nn::ui2d::MultiArcResourceAccessor クラスから指定したタイプのリソース名をすべて検索するメソッド FindResourceByType を追加しました。

nn::ui2d::ResourceAccessor に純粋仮想関数として FindResourceByType が追加されていますので、アプリ側で nn::ui2d::ResourceAccessor を継承している場合はオーバーライドしてください。
