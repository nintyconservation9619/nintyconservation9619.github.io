Natvis の使用方法

Visual Studio では、NatVis フレームワークを使用した、変数ウィンドウ内の変数のカスタム視覚化が可能です。

VSI for NX は、この機能のサブセットをサポートします。サポートされるノード、属性、および高度な機能について以降の各セクションで説明します。

高度な機能

NatVis ファイルの場所

ユーザは、vcxproj を右クリックし、「追加」->「新しい項目...」を選択して新しい NatVis ファイルを作成できます。その後、Visual C++ メニューで「ユーティリティ」を選択します。次に、「デバッガー視覚化ファイル（.natvis）」を選択します。NX デバッガは次のディレクトリでも NatVis ファイルを検索します。

  • %USERPROFILE%\Documents\Nintendo\NX\Visualizers.
  • %USERPROFILE%\Documents\Visual Studio 201X\Visualizers.
  • ユーザのソリューションの任意の vcxproj プロジェクトに含まれている natvis ファイル。

NatVis の式のコンテキスト

Oasis 3.0 でサポートされています。Oasis 4.0 以降では “this” を含む式のサポートが向上しました。NatVis フレームワークのドキュメントからの引用を言い換えました。LinkedListItems の “NextPointer” と “ValueNode”、TreeItems の “LeftPointer”、“RightPointer”、および “ValueNode” は、親の型ではなくノードのコンテキストで評価されます。
その結果、それらのコンテキスト内の NatVis の式は、親メンバのデータにアクセスできません。これらの NatVis の式のローカル変数にアクセスすることもできません。グローバル変数にはアクセスできる必要がありますが（::g_GlobalInt）、まだサポートされていません。

「出力」ウィンドウの「Oasis 出力」からの NatVis エラーレポート

Oasis 4.0 以降でサポートされるようになりました。ノードに対する NatVis の式が失敗した場合、デバッグ
文字列が「出力」ウィンドウに表示される必要があります。Oasis では、失敗した NatVis の式は「出力」ウィンドウ内の
「Oasis 出力」セクションに表示されます。

継承された NatVis の型の型マッチング

Oasis 4.0 以降でサポートされるようになりました。親の型に対する NatVis 定義を持つ派生型はいずれも、それらの親 NatVis 定義によって指定されたとおりに表示されます。派生型に NatVis で定義された親の型が複数含まれている場合は、最も近い親の型が優先されます。派生型が定義済みの NatVis 定義を持つ場合、この定義は定義済みのすべての親の型よりも優先されます。

テンプレート

Oasis 3.0 以降でサポートされるようになりました。テンプレート化された型がサポートされており、NatVis のノード式で “$T1”、“$T2” などを使用して、テンプレート化された型を式内で使用することができます。NatVis のテンプレート化された <型> 定義では、テンプレートのパラメータの代わりに「*」を使用して、指定されたテンプレートのパラメータのすべての型に一致させることができます。例えば、テンプレートの引数に関係なくすべての “list” 型に一致させるには、次のようにします。

<Type Name="list&lt;*&gt;">

構造体またはクラス内の NatVis の型

Oasis 4.0 以降で部分的にサポートされるようになりました。NatVis の型を含む構造体またはクラスの変数では、表示文字列がそれぞれの子の適切な視覚化によりフォーマットされることはありません。Oasis はこの機能をサポートしていません。ただし、NatVis の定義を持つ構造体内で展開されるすべての子要素は、NatVis の定義で定義されたとおりに正しくフォーマットされます。

配列およびベクトルのインデックス化（MyArray[0] など）

サポートされていません。NatVis で定義された型の、変数式内でのインデックス化は、新しい VS 2015 NatVis フレームワークの機能です。この機能はまだサポートされていません。

NatVis の定義の動的再読み込み（“.natvisreload”）

サポートされていません。NatVis フレームワークは、NatVis の定義が編集されるときの変数式の動的再評価をサポートしています。このフレームワークは、ウォッチウィンドウに “.natvisreload” と入力することによる手動再評価もサポートします。これらの機能は Oasis ではまだサポートされていません。

同じ型名に対する複数の定義

サポートされていません。複数の一致が検出された場合、最初に読み込まれた定義が使用されます。NatVis フレームワークでは、Priority 属性を使用することによって複数の一致をサポートしています。この属性は Oasis ではまだサポートされていません。

用語

NatVis の構文を記述するために使用される用語は、次のように定義されています。

用語 例
Type <Type Name="MyArrayType">
Node <DisplayString>The display string for MyArrayType</DisplayString>
Node <Expand>
Node w/Attribute <Item Condition="m_bInitialized == true" Name="[Size]">m_Size</Item>
Node <ArrayItems>
Node <Size>m_End - m_Begin</Size>
Node <ValuePointer>m_Begin</ValuePointer>
End Node </ArrayItems>
End Node </Expand>
End Type </Type>

NatVis フレームワークの構文、機能、および要件について詳しくは、https://msdn.microsoft.com/en-us/library/jj620914.aspx を参照してください。

ノード

NatVis の型のノードは、指定された型の表示および動作全般を定義します。サポートされているノードとサポートされていないノードを以下に示します。

<DisplayString>

Oasis 3.0 以降でサポートされるようになりました。DisplayString は、変数ウィンドウの「値」列に表示される文字列を定義します。“Condition” 属性は、変数の状態に応じて、複数の DisplayStrings を定義するために使用できます。

<DisplayString Condition="m_pID == 0">Invalid ID</DisplayString>
<DisplayString Condition="m_pID != 0">Valid ID</DisplayString>

<AlternativeType>

Oasis 3.0 以降でサポートされるようになりました。AlternativeType は、変数の代替型名を定義するために使用されます。これは、typedef エイリアスまたは宣言の使用などの指定に使用できます。この型名およびすべての代替型名は、指定された変数のビジュアライザを検索するときに使用されます。例えば、SparseArray という名前のテンプレート化された typedef にも一致させるには、次のようにします。

<AlternativeType Name="SparseArray&lt;*&gt;"/>

<Version>

サポートされていません。Version は、特定のモジュール（dll）の視覚化について詳しく調べるために使用されます。さらに、dll の最小バージョンと最大バージョンも指定できます。近い将来にサポートする予定はありません。

<StringView>、<UIVisualizer>、<CustomVisualizer>

サポートされていません。これらのノードは、別のカスタムビジュアライザで型が視覚化されるよう指定するために使用されます。例えば、StringView ノードでは、変数が組み込みテキストビジュアライザの VS を使用するよう指定します。UIVisualizer と CustomVisualizer も同様の機能であり、VSPackage または VSIX のいずれかの guid を使用してユーザ定義のビジュアライザを指定します。

<HResult>

サポートされていません。HResult ノードは、HResult のそれぞれに対してカスタム エラー文字列を定義するために使用されます。
これはサポートされていませんが、同様の nn::result を作成できます。

<Expand>

Oasis 3.0 以降でサポートされるようになりました。Expand ノードは、次のサブノードカテゴリのいずれかに属す
複数の子要素をリストするために使用されます。

<Item>

Oasis 3.0 以降でサポートされるようになりました。Item は、NatVis の型の子を作成するために使用できます。
このノードは、子の名前と値を指定します。以下に例を示します。

<Expand>
<Item Name="[Size]">m_Size</Item>
</Expand>

<ArrayItems>

Oasis 3.0 以降でサポートされるようになりました。ArrayItems は、構造体のような配列を繰り返すために使用されます。NatVis フレームワークでは、Size サブノードと ValuePointer サブノードを使用して、配列を繰り返す方法を定義します。以下に例を示します。

<Expand>
<ArrayItems>
<Size>m_End - m_Begin</Size>
<ValuePointer>m_Begin</ValuePointer>
</ArrayItems>
</Expand>

<TreeItems>

Oasis 3.0 以降でサポートされるようになりました。TreeItems は、構造体のようなバイナリツリーを繰り返すために使用されます。NatVis フレームワークでは、Size、HeadPointer、LeftPointer、RightPointer、および ValueNode の各サブノードを使用して、ツリーを繰り返す方法を定義します。以下に例を示します。

<Expand>
<TreeItems>
<Size>m_Size</Size>
<HeadPointer>m_pHead</HeadPointer>
<LeftPointer>m_pLeft</LeftPointer>
<RightPointer>m_pRight</RightPointer>
<ValueNode>m_Value</ValueNode>
</TreeItems>
</Expand>

<LinkedListItems>

Oasis 3.0 以降でサポートされるようになりました。LinkedListItems は、構造体のようなリンクされたリストを繰り返すために使用されます。NatVis フレームワークでは、Size、HeadPointer、NextPointer、および ValueNode の各サブノードを使用して、リストを繰り返す方法を定義します。以下に例を示します。

<Expand>
<LinkedListItems>
<HeadPointer>m_pHead</HeadPointer>
<NextPointer>m_pNext</NextPointer>
<ValueNode>m_Value</ValueNode>
</LinkedListItems>
</Expand>

<Synthetic>

Oasis 4.0 以降でサポートされるようになりました。Synthetic は、型の直接のメンバではない可能性のある追加データを表示するために使用されます。これは、変数の状態を説明するために使用される文字列の場合も、その他のノード（ArrayItems ノード、ネストされた Synthetic ノードなど）を含む構造体全体の場合もあります。以下に例を示します。

<Expand>
<Synthetic Name="Additional debug data">
<DisplayString>Expand for more information</DisplayString>
<Expand>
<Item Name="Name">m_pName</Item>
<Item Name="ID">m_pID</Item>
<Item Name="Next variable name">m_pNext-&gt;m_pName</Item>
</Expand>
</Synthetic>
</Expand>

<IndexListItems>

Oasis 3.0 以降でサポートされるようになりました。IndexListItems は、配列の繰り返しに何らかのインデックス操作が必要なときに、構造体のような配列を繰り返すために使用されます。以下に例を示します。

<Expand>
<IndexListItems Condition="m_Size != 0">
<Size>m_Size</Size>
<ValueNode>m_Array[($i + m_Start)]</ValueNode>
</IndexListItems>
</Expand>

<ExpandedItem>

Oasis 4.0 以降でサポートされるようになりました。ExpandedItem は、現在の展開レベルで子の展開を表示するために使用されます。これは、メンバが型の直接の子であるかのようにして、子構造体全体を表示するために使用できます。例えば、m_Array の子に含まれるすべての項目を NatVis の型の直接の子として表示するには、次のようにします。

<Expand>
<ExpandedItem>m_Array</ExpandedItem>
</Expand>

<CustomListItems>

サポートされていません。CustomListItems は、上記の定義済みノードタイプのいずれにも当てはまらない構造体を繰り返すために使用されます。これは VS 2015 に含まれた新機能でした。CustomListItems の子ノードと属性を以下に示します。

  • <Variable>
  • <Loop>
  • <If>
  • <Elseif>
  • <Else>
  • <Break>
  • <Exec>
  • <Item>
  • MaxItemsPerView=

属性

Name=

Oasis 3.0 以降でサポートされるようになりました。Name はサポートされており、Item ノードでは必須です。これは、変数ウィンドウの「名前」列に使用される表示名の定義に使用されます。以下に例を示します。

<Item Name="[Size]">m_Size</Item>

Condition=

Oasis 3.0 以降でサポートされるようになりました。Condition は、ノードをブール式に基づいて評価するかどうかを決めるために使用されます。それぞれ異なる Condition 式を含む複数の DisplayStrings を定義できます。以下に例を示します。

<DisplayString Condition="m_pID == 0">Invalid ID</DisplayString>
<DisplayString Condition="m_pID != 0">Valid ID</DisplayString>

Optional=

Oasis 3.0 以降でサポートされるようになりました。NatVis 式の評価に失敗した場合（子メンバが存在しないなど）、NatVis フレームワークは以降の型の評価を中止し、変数が視覚化されずに表示されます。Optional は、指定されたノードの評価が失敗した場合に、そのノードをスキップするために使用されます。以下に例を示します。

<Item Name="[Name]" Optional="true">m_pID-&gt;m_Name</Item>

Priority=

サポートされていません。Priority は、複数の定義が見つかった場合に、この視覚化をいつ使用するかを指定するために使用されます。この属性は最上位の型のノードにのみ適用できることに注意してください。

IncludeView=、ExcludeView=

サポートされていません。IncludeView および ExcludeView は、NatVis の視覚化に項目を含めるか、NatVis の視覚化から項目を除外するために使用されます。例えば、ウォッチウィンドウに “MyVar,view(simple)” と入力した場合、結果として得られる変数は NatVis の定義によって定義されたとおりに MyVar 変数を表示し、IncludeView=”simple” というフラグが立っている NatVis ノードを追加し、ExcludeView=”simple” というフラグの立ったノードを除外します。

Inheritable=

サポートされていません。NatVis フレームワークは、指定された型、およびその型から継承されたすべての派生型の、定義済みの視覚化を使用します。Inheritable は、派生型が親 NatVis 定義を使用しないようにするために使用されます。派生型が定義済みの視覚化を持つ場合、そのフレームワークはこの属性に関係なくその定義済みの視覚化を使用します。この属性は、視覚化が定義されていない型など、派生型に親の視覚化を使用してはならない場合にのみ必要です。
