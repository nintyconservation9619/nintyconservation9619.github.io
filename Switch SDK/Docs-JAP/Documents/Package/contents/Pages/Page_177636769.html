はじめに

本ページでは NintendoSDK NX Addon において使用される各種IDを紹介します。

なお、このページで使用されている用語の意味や用法は、一般的に知られているものと異なる場合があります。

事務手続き関連

製品コード

製品分類コード - 形態コード - イニシャルコード （仕向地コード）
HAC - x - xxxxx (xxx)
  • 製品分類コード
    • 「HAC」が割り当てられます。
  • 形態コード
    • 製品の種類に基づき、アルファベットおよび0から9までの数字の中から対応する1文字が割り当てられます。
  • イニシャルコード
    • アルファベットおよび0から9までの数字で構成される5文字の文字列です。
  • 仕向地コード
    • 地域を表す3文字の文字列です。

プログラム関連

アプリケーション ID

アプリケーションごとに割り当てられた固有の ID です。
0x0100 0000 0001 0000～0x01FFF FFFF FFFF FFFFで指定します。
製品アプリケーションではNintendo Developer Portalで指定された値を指定します。
開発ツールや試作用には、0x0100 XXX0 0348 8000で、XXXは000～FFFまで任意の値をお使いください。

 アプリケーション ID は NX 全体として以下の識別子として使用されています。
そのため、他の製品のアプリケーション IDを間違って使用した場合は各々以下の状況になります。 

  • NX 本体へインストールする際のコンテンツのID
    • 既にインストールされたコンテンツのIDと同じIDのコンテンツをインストールすると上書きされます。
  • Nintendo Developer Portal へROMを提出する際のID
    • NDID単位で提出可能なIDが管理されているため、間違ったIDで提出することはできません。
  • ネットワークサービスへのアクセスする際のアプリのID
    • アプリごとに別途アクセスキーを発行しているため、正しいアクセスキーを持たないアプリが別のアプリのネットワークサービスにアクセスすることはできません。
