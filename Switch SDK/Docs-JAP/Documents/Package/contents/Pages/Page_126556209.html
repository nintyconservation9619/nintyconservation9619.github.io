ビットマップフォントとスケーラブルフォントの表示サイズが異なる

質問

ビットマップフォントとスケーラブルフォントの表示サイズが微妙にずれてしまいます。

回答

カーニング計算方法に差異がある為です。
あるスケーラブルフォントから、ビットマップフォント（bffnt）を作成し 描画した結果と、同じサイズでスケーラブルフォントをそのまま ScalableFont で描画した結果が微妙に異なってしまう場合があります。

ビットマップフォントを作る際に、FontConverter の 「カーニング抽出元に scfont を使う」オプションを指定します。
FontConverterConsole では、-isck を指定します。
