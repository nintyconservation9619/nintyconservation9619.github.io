NintendoSDK のビルドシステム

NintendoSDK では開発機環境、Windows 環境ともに Visual Studio を使用した開発を前提としています。NintendoSDK が提供するビルドシステムも、基本的に Visual Studio での開発に必要なコンポーネントが中心となります。具体的に提供するものとして以下のものがあります。

  • Visual Studio 用プロパティシート
  • Visual Studio の新規 NintendoSDK アプリケーション／ライブラリプロジェクト作成ウィザード
  • Visual Studio 拡張機能 (VSI デバッガ等を含む)

また、使用している Addon パッケージによっては以下のものが提供されることがあります。

  • コンパイラツールチェイン
  • 各種 PC ツールなど

NintendoSDK では Visual Studio に依存しないビルドシステムを独自に構築するための手段も開発者に提供します。ただし、これはドキュメントによる情報の提供のみであり、特定のビルドツールを用いた実装例などは提供されません。

ビルド環境の構築手順

NintendoSDK では、開発者が NintendoSDK を使用したアプリケーションやライブラリをビルドするための手段として、以下のいずれかを想定しています。

  1. Visual Studio のプロジェクト作成ウィザードを使用して新しいプロジェクトファイルを作成する
  2. 既存の Visual Studio プロジェクトファイルに新たにプロジェクト構成を追加し、NintendoSDK を使用するよう設定する
    1. NintendoSDK が提供する Visual Studio 拡張機能を使用して設定する
    2. NintendoSDK が提供するプロパティシートをインポートする
    3. 必要なビルド設定を開発者自身が行う
  3. ビルド環境を一から構築する
    1. Visual Studio プロジェクトを一から作成する
    2. Visual Studio を使用せず独自のビルドシステムを構築する

初めて NintendoSDK を使用する場合は、以下のいずれかの手順でビルド環境を構築することを推奨します。最適な手順は、上記のどのビルド手段を選択するかによって異なります。

新しい Visual Studio プロジェクトを作成する場合

Visual Studio から NintendoSDK の新規プロジェクト作成ウィザードを使用して “NintendoSDK アプリケーション” プロジェクトを作成してください。

細かな設定作業は省略し、まずは NintendoSDK を使用したい場合に適しています。

既に Visual Studio プロジェクトが存在する場合

既存の Visual C++ プロジェクトで NintendoSDK を使用する のページに従い必要なプロジェクト設定を行うことで、NintendoSDK を使用した開発を始めることができます。具体的な Visual Studio プロジェクトの設定方法を チュートリアル のページにも掲載していますので、あわせてお読みください。

独自のビルドシステムを構築する のページを参考にしながら、NintendoSDK が提供する Visual Studio 拡張機能やプロパティシートを使用することなく開発者自身の手で必要な設定を行うこともできます。

既存のライブラリやミドルウェア等の Visual Studio プロジェクトが存在し、それを NintendoSDK 向けに移植したい場合に適しています。

ビルド環境を一から構築する場合

独自のビルドシステムを構築する のページを参考にしてください。

必要なビルド設定をすべて開発者自身の手で管理したい場合に適しています。

その他の情報

NintendoSDK のビルドシステムに関する情報は マニュアル、及び リファレンス から入手することができます。ビルド環境の構築時や調整時に不明な点がありましたら、これらのドキュメントを適宜お読みください。

また、コンパイラに関する情報は コンパイラ から、VSI デバッガに関する情報は VSI からそれぞれ入手することができます。
