シェーダー定義プロパティウィンドウ ソースコードページ

文字列からソースコードを検索するメニューバーです。

A 検索する文字列を入力するフィールドです。
B 前を検索します。
C 次を検索します。
D 全てのソースを検索対象とします。
E 大文字と小文字を区別して検索します。
F 正規表現を使用して検索します。

選択したソースコード ( .glsl ) を確認、編集するウィンドウです。

A シェーダーソースのインクルードパスが表示されます。
B シェーダーソースのインクルードパスからの相対パスです。
C シェーダーソースを編集します。
編集ボタンを押すと、外部エディタが開いてシェーダーソースの編集ができます。
D 強制的にインクルードするかどうかを表示します。
E シェーダーソースが表示され確認できます。

シェーダーソースの編集

3DEditor をランタイムに接続することにより、シェーダーソースの編集が行えます。

シェーダーソースの編集の手順は以下になります。

  1. 3DEditor にモデルとシェーダー定義ファイルを読み込ませます。
  2. モデルのマテリアルにシェーダー定義ファイルを設定します。
    関連リンク シェーダー定義ファイルの設定は、マテリアルプロパティウィンドウのシェーダーページから行います。
  3. ランタイムと接続します。
  4. 関連リンク ファイルビューからシェーダー定義プロパティウィンドウを開きます。
    シェーダー定義プロパティウィンドウは、シェーダー定義ファイル ( .fsdb ) をダブルクリックするか、ファイルメニューの[表示] → [プロパティウィンドウ]で開けます。
  5. 関連リンク シェーダー定義プロパティウィンドウのソースコードで、編集したいシェーダーソース ( .glsl ) を選択します。
  6. ソースコードページで、編集ボタンを押すと、5 で選択したシェーダーソース ( .glsl ) が外部エディタで編集可能になります。
    外部エディタは、「3dEditor.user.config 」で任意のエディタに変更できます。
  7. 外部エディタで、シェーダーソースを編集して保存すると編集内容が反映されます。
参考：

編集内容は、3DEditor に読み込んだオリジナルのシェーダー定義ファイルには反映されません。
編集した内容を残したい場合は、別途保存する必要があります。（シェーダーソースのコピーや、外部エディタで別名で保存します。）

参考：

シェーダーソースの編集は、ランタイム側のシェーダーにアタッチしても可能です。
関連リンク アタッチを実行するには、ランタイム側で QueueAttachShaderArchiveCommand() の実装が必要になります。
