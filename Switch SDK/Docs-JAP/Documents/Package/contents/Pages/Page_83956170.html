低レベルグラフィックス API のバリエーション

低レベルグラフィックス API

ここでは、GFX ライブラリーにおける低レベルグラフィックス API のバリエーションの扱いについて説明します。

これらは通常、気にかける必要のない機能のため、特別用途がない限りは使用しないようにしてください。

低レベルグラフィックス API の選択方法については低レベルグラフィックス API の選択を参照してください。

低レベルグラフィックス API の定義

低レベルグラフィックス API は各低レベルグラフィックス用の gfx_Variation-api.*.h ヘッダーにおいて定義されています。

低レベルグラフィックス API の種類

低レベルグラフィックス API の種類は、nn::gfx::ApiType 構造体によって表現されます。

nn::gfx::ApiType 構造体は、列挙値として低レベルグラフィックス API を表す value 静的変数をもちます。
これにより、低レベルグラフィックス API の一致比較などが可能です。

低レベルグラフィックス API のバージョン

低レベルグラフィックス API のバージョンは nn::gfx::ApiVersion 構造体によって表現されます。

nn::gfx::ApiVersion 構造体は、数値としてバージョン番号を表す value 静的変数をもちます。
これにより、低レベルグラフィックス API のバージョン比較などが可能です。

低レベルグラフィックス API のバリエーション

低レベルグラフィックス API のバリエーションは、低レベルグラフィックス API の種類と低レベルグラフィックス API のバージョンを合わせた nn::gfx::ApiVariation 構造体によって表現されます。

nn::gfx::ApiVariation 構造体は、nn::gfx::ApiType 構造体と nn::gfx::Version 構造体を型定義しており、これらを参照することによって低レベルグラフィックス API の種類とバージョンの情報を得ることができます。

対象の低レベルグラフィックス API

対象の低レベルグラフィックス API に関する情報は、nn::gfx::TargetConfig 構造体として表現されます。

nn::gfx::TargetConfig 構造体から、バリエーション、種類、バージョンを取得することができます。

関数のバリエーション

GFX においてわずかな機能が関数として提供されます。

低レベルグラフィックス API によって処理が異なる関数は、バリエーションをもっています。

ここでは例として、nn::gfx::Finalize 関数を使って説明します。

対象のバリエーション

対象としている低レベルグラフィックス API のバリエーションの関数を呼び出すには、なにも接尾辞をつけずに呼び出します。

nn::gfx::Finalize();

バリエーションの指定

呼び出す関数のバリエーションを明示的に指定するには、低レベルグラフィックス API の接尾辞をつけて呼び出します。

以下は GL4 を指定する場合です。

nn::gfx::FinalizeGl4();

バージョンを省略することもでき、その場合は対象バージョンが選択されます。

nn::gfx::FinalizeGl();

追加されていない低レベルグラフィックス API のバリエーションを指定することはできません。

パラメーターによる指定

nn::gfx::ApiVariation 構造体をテンプレートパラメーターとしてとることもできます。

シンボル名の違いで指定せずにパラメーターとして指定したい場合はこちらを利用します。

nn::gfx::Finalize< nn::gfx::ApiVariation< nn::gfx::ApiType< nn::gfx::LowLevelApi_Gl >, nn::gfx::ApiVersion< 4 > >();

クラスのバリエーション

GFX におけるほとんどの機能はクラスとして提供されます。

オブジェクトクラスは、バリエーションをもっています。

ここでは例として、nn::gfx::Device クラスを使って説明します。

対象のバリエーション

対象としている低レベルグラフィックス API のバリエーションのクラスを使うには、なにも接尾辞をつけずに記述します。

nn::gfx::Device

バリエーションの指定

使うクラスのバリエーションを明示的に指定するには、低レベルグラフィックス API の接尾辞をつけて記述します。

以下は GL4 を指定する場合です。

nn::gfx::DeviceGl4

バージョンを省略することもでき、その場合は対象バージョンが選択されます。

nn::gfx::DeviceGl

追加されていない低レベルグラフィックス API のバリエーションを指定することはできません。

パラメーターによる指定

nn::gfx::ApiVariation 構造体をテンプレートパラメーターとしてとる、接頭辞 "T" のついたバージョンもあります。 

シンボル名の違いで指定せずにパラメーターとして指定したい場合はこちらを利用します。

nn::gfx::TDevice< nn::gfx::ApiVariation< nn::gfx::ApiType< nn::gfx::LowLevelApi_Gl >, nn::gfx::ApiVersion< 4 > >

この機能は主に静的なプログラミングのために利用できます。
たとえば、可変長引数テンプレートを使うことで、すべてのバリエーションに対して静的な操作を行うことなどが可能です。 

バリエーションの取得

バリエーションをもつクラスは、共通して Target 型定義をもっています。
Target は nn::gfx::ApiVariation 構造体であるため、そこから低レベルグラフィックス API の バリエーション情報を静的に取得することができます。

例:

const int version = nn::gfx::Device::Target::ApiVersion::value;
