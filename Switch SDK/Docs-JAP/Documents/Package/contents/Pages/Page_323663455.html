振動ノードの使い方

振動ノードの構成例

振動ノードを具体例を紹介します。

#include <nn/hid/hid_Vibration.h>
 
nn::hid::VibrationDeviceHandle vibrationDeviceL;
nn::hid::VibrationDeviceHandle vibrationDeviceR;
 
/* VibrationDeviceHandle を取得する処理 */
 
// 振動ノードのインスタンスを生成
nn::hid::VibrationPlayer player1;
nn::hid::VibrationPlayer player2;
nn::hid::VibrationPlayer player3;
nn::hid::VibrationMixer mixer;

nn::hid::VibrationPlayer playerA;
nn::hid::VibrationPlayer playerB;
nn::hid::VibrationTarget targetA(vibrationDeviceL);
nn::hid::VibrationTarget targetB(vibrationDeviceR);
 
// 接続のインスタンスを生成
// Player1、Player2、Player3 は MixerA に接続
nn::hid::VibrationNodeConnection connectionFromPlayer1ToMixer(&player1, &mixer);
nn::hid::VibrationNodeConnection connectionFromPlayer2ToMixer(&player2, &mixer);
nn::hid::VibrationNodeConnection connectionFromPlayer3ToMixer(&player3, &mixer);

// PlayerA と Miexr は TargetA に接続
nn::hid::VibrationNodeConnection connectionFromPlayerAToTargetA(&playerA, &targetA);
nn::hid::VibrationNodeConnection connectionFromMixerToTargetA(&mixer, &targetA);

// PlayerB と Miexr は TargetB に接続
nn::hid::VibrationNodeConnection connectionFromPlayerBToTargetB(&playerB, &targetB);
nn::hid::VibrationNodeConnection connectionFromMixerToTargetB(&mixer, &targetB);


この時点で、以下の図のような有向グラフが構築されたことになります。



PlayerA は TargetA だけ、PlayerB は TargetB だけに振動が伝わります。

playerA と playerB は特定のコントローラだけを振動させたいときに使用します。

Mixer から出力されたものは TargetA および TargetB に振動が伝わります。

connectionFromMixerToTargetA と connectionFromMixerToTargetB に設定する、 nn::hid::VibrationModulation の gainLow と gainHigh を調整することでパンニングが可能です。

※パンニングの実装については、SDKに含まれている「振動を使ったインタラクティブな技術デモ （Samples/Sources/Applications/HidVibrationRollingBall）」を参照ください。


例えば、このグラフ構成を FPS であれば次のように使用します。

各 nn::hid::VibrationPlayer には、次の振動波形をセットします。

  • Player1：敵の攻撃による振動波形（砲撃音）
  • Player2：敵の被弾による振動波形（爆発音）
  • player3：敵の移動による振動波形（車両のエンジン音）
  • playerA：プレイヤの左側に障害物があることを表す振動波形
  • playerB：プレイヤの右側に障害物があることを表す振動波形

各 nn::hid::VibrationPlayer は、それぞれのアクションに応じて nn::hid::VibrationPlayer::Play() を呼び出します。

Player1 から player3 には、同一の敵に関する様々な振動波形をセットしておきます。これらの振動波形は Mixer に送られミックスされます。

このように Mixer を使うことで、出力を敵からの振動というグループでまとめられます。

プレイヤと敵の距離や方向に応じて、connectionFromMixerToTargetA と connectionFromMixerToTargetB を設定することで、

振動によって距離の距離を表現できます。このように振動を使うことで、ゲームの臨場感を高めることができます。


振動値のミックス方式の使い分け

 nn::hid::VibrationMixer には設定できるミックス方式（ nn::hid::VibrationMixMode ）として大きく分けて、

最も振幅が強い振動値を採用する nn::hid::VibrationMixMode_MaxAmplitude と

すべての振幅値を合算する nn::hid::VibrationMixMode_AmplitudeSum が存在します。

この2つの使い分けについて説明します。


VibrationMixMode_AmplitudeSum

この方式は入力されたすべての振幅値を合算します。

小さい振動が重なり合いを表現したい場面で有効です。



例えば、地面に小さい岩がたくさん落ちてくるシーンで、衝突の振動を表現したい場合を考えます。

この方式を使うと、すべての小さな衝突の振動を合算できます。

１つだけ振動するときは小さな振動になりますが、複数同時に振動する場合、その分だけ強い振動になります。


SDKに含まれている「パラメータによる簡易な振動波形生成（Samples/Sources/Applications/HidVibrationGenerator）」

というサンプルでは、線に触れるたびに振動するのですが、複数の線に同時に触れると振動が重なり合って強い振動になることを体験できます。


しかしながら、強い振動が重なり合う場合では、振幅が上限に達してしまうので、

２つの大きな振動が起きているのか、１つの大きな振動が起きているかを区別することができません。

そのような場合は次の nn::hid::VibrationMixMode_MaxAmplitude 方式が有効です。


VibrationMixMode_MaxAmplitude 

この方式は、入力されたすべての振動値の中で、最も振幅値が高いものを採用します。

そのときに、最も強い振動だけを取捨選択したい場面で有効です。



例えば、振動を使用してダメージ表現と、乗り物で移動している表現をする場合を考えます。

どちらも強い振動の場合、nn::hid::VibrationMixMode_MaxAmplitude では両者が同時に発生した場合、

振幅が上限に達してしまい、どの振動由来のものか分からなくなります。

この方式を使うと、そのときに一番強い振動だけが採用されて再生されます。


「ダメージの振動の強さ」＞「乗り物の振動の強さ」の関係の場合、

移動中は「乗り物の振動」だけが、ダメージを受けたときは「ダメージの振動」だけが再生されます。

このようにして、一番注目すべきものを目立たせることが可能です。


どちらを使うか迷われたときは、はじめは nn::hid::VibrationMixMode_AmplitudeSum を使用して、

問題があれば nn::hid::VibrationMixMode_MaxAmplitude を使用することをお勧めします。
