コマンドラインオプションの概要

プロファイラ GUI の動作の一部をより詳細に制御できるよう、いくつかのコマンドラインオプションが用意されています。コマンドラインの書式を以下に示します。

コマンドラインの書式

"Nintendo CPU Profiler.exe" [options] [NPROF] [ELF] [Executable]

[options]

  • 0 個以上のオプション（下記参照）。

[NPROF]

  • 前回保存した NPROF（C:\path\file.nprof）のファイル名を記述し、プロファイラを起動したときに自動的にロードされるようにします。
  • 重要：NPROF パラメータは、ELF パラメータや Executable パラメータと共に指定しないでください。

[ELF]

  • 同期処理で使用する ELF ファイルへのパスを入力します（C:\path\example.nss）。このオプションは、同期オプションが使用可能になったら直ちに、指定したファイルとの同期を実行するよう GUI に指示します。Executable が指定されていない場合は、プロファイラが自動的に探そうとします。

[Executable]

  • 同期に使用する実行可能ファイルへのパス（C:\path\example.nsp）を入力します。このオプションは、同期オプションが使用可能になったら直ちに、指定したファイルとの同期を実行するよう GUI に指示します。ELF が指定されていない場合は、プロファイラが自動的に探そうとします。
  • 重要：NPROF パラメータは、ELF パラメータや Executable パラメータと共に指定しないでください。

オプション

--autosave[=filename]

  • これは、ボタンを押すよう要求するのではなく、プロファイルの結果をすぐに保存するよう GUI に指示します。このオプションを使用すると、NPROF のみが保存されます。
  • コマンドラインのファイル名は任意指定です。ファイル名を指定しない場合、現在ロードされている ELF に基づいた名前のファイルが、ELF と同じフォルダに保存されます。使用される最終的なファイル名は次の形式になります。ファイル名-YYYYMMDD-hhmmss.nprof

--script=filename

  • プロファイラが起動すると、指定のスクリプトがプロファイラにより自動的に実行されます。この操作は、指定の NPROF がロードされた後に実行されますが、--loadelf が完了する前に実行される場合もあります。

--shutdown

  • このオプションは、プロファイルの受信後にプロファイラの GUI をシャットダウンします。このオプションは、--autosave オプションと共にに使用することをお勧めします。
