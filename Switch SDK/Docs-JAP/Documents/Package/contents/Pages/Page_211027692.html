テクスチャキャッシュのワークメモリの要求量

はじめに

本ページでは、スケーラブルフォントのテクスチャキャッシュで要求される nn::font::TextureCache::InitializeArg::workMemorySize および nn::font::TextureCache::InitializeArg::noPlotWorkMemorySize について解説します。

nn::font::TextureCache::InitializeArg::workMemorySize

nn::font::TextureCache::InitializeArg::workMemorySize は、nn::font::TextureCache::RegisterGlyph() などで実際にプロットした文字を扱う際に使われるメモリサイズです。

プロットされる文字数が増えるほど、またフォントサイズが大きくなるほど多くのメモリを必要とします。

nn::font::TextureCache::InitializeArg::noPlotWorkMemorySize

nn::font::TextureCache::InitializeArg::noPlotWorkMemorySize は、文字幅やカーニング情報などを、文字をプロットせずに取得する際に使われるメモリサイズです。

プロットされる文字数やフォントサイズが増えてもメモリ要求量は変わらず、常に一定のメモリサイズを必要とします。
