プログラマ向け グラフィックスの変更点

NintendoWare グラフィックスは、nn::g3d、nn::ui2d、nn::font、nn::vfx モジュールのことを指します。

  • nn::ui2d ニュース

    • 詳細なコンバイナモードの実装

    • BC7 フォーマットのサポート

nn::vfx ニュース

NintendoSDK 0.14.0 で行われた nn::vfx 関連のプログラマ向けニュースです。

ユーザーが用意したメモリプールへエフェクトバイナリを配置する機能に対応

ユーザー側で用意されたメモリプールに対して、エフェクトバイナリを配置した上でエフェクトバイナリの初期化が行えるようになりました。

 関連リンク 詳細については下記リンク先を参照ください。

メモリプールについて

vfxランタイム内部で生成されるメモリプールの説明や、
ユーザーが用意したメモリプールを利用してエフェクトバイナリを配置する方法についての説明しています。

コンピュートシェーダのバリエーション化によるGPU処理の最適化を実施

コンピュートシェーダのバリエーション化によるGPU処理の高速化に対応しました。利用されている機能の組み合わせ数だけコンピュートシェーダバイナリが生成され、GPU処理が最適に行われるようになり、バリエーション対応前後でコンピュートシェーダ処理が 約 10% ~ 30% ほど高速化しました。ただ、最終的に出力されるエフェクトバイナリサイズは生成されるコンピュートシェーダバリエーション数によって増加します。

nn::ui2d ニュース

NintendoSDK 0.14.0 で行われた nn::ui2d 関連のプログラマ向けニュースです。

nn::ui2d::Material クラスのサイズを最適化

nn::ui2d::Material クラス内部のメンバー変数を整理しクラスサイズを最適化しました。この最適化により nn::ui2d::Material クラスのサイズが約 60 ~ 70% ほど削減されました。また、最適化の影響により一部の API のインターフェイスが変更されておりアプリケーションコードの変更が必要となる場合があります。変更の詳細については変更履歴を参照してください。

詳細なコンバイナモードの実装

LayoutEditor で詳細なコンバイナモードへの切替を行い、編集した内容をランタイムで確認する事が出来ます。

テクスチャコンバイナ、カラー合成設定に、比較（暗）,比較（明）を追加

テクスチャコンバイナ、カラー合成設定に、比較（暗）,比較（明）を追加しました。

BC7 フォーマットのサポート

BC7 フォーマットを利用できるように拡張しました。

BC7 は、RGBA カラーフォーマットで、1ピクセルあたり、1バイトのテクスチャフォーマット(RGBA8 の1/4、BC1 の倍)となっています。
BC3 に比べると、圧縮ノイズが少なく高品質な表示となっています。一方で、変換に長い時間がかかる（特に大きなサイズの場合）のでご留意ください。

LayoutConverter で出力するたびに、異なるバイナリが出力される不具合を修正

LayoutConverter で出力するたびに、異なるバイナリが出力される不具合がありこれを修正しました。*.arc ファイルや、*.bflyt 、*.bflan ファイルに差分が発生していました。
