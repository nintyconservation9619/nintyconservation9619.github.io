ボーンビジビリティーアニメーション

ボーンビジビリティーアニメーションの設定について説明します。

ボーンビジビリティーアニメーション設定の種類

ボーンビジビリティーアニメーションを設定するには、transform ノードの可視性（Visibility）アトリビュートをアニメーションさせる方法と、mesh ノードの可視性（Visibility）アトリビュートをアニメーションさせる方法の 2 種類があります。

transform ノードの可視性アトリビュートをアニメーションさせた場合

下の階層のノードすべてに可視性の ON ／ OFF が影響します。
下の階層のノードの可視性が ON で固定の場合、上の階層と同じアニメーションが出力されます。
下の階層のノードの可視性もアニメーションしている場合、 上の階層のアニメーションと合成したアニメーションが出力されます。

mesh ノードの可視性アトリビュートをアニメーションさせた場合

その階層のポリゴンにだけ可視性の ON ／ OFF が影響します。

mesh ノードの可視性アトリビュートにキーを打つには、mesh ノードを選択してチャンネルコントロールを開き、 可視性（Visibility）をキー設定可（Keyable）に設定する必要があります。

ボーンビジビリティーアニメーションの注意事項

ボーンビジビリティーアニメーションは、必ずベイクしたデータからアニメーションカーブデータを自動作成して出力します。

参考：

ボーンビジビリティーアニメーションはノードに対して設定するので、エクスポートオプションの Compress Bone を Cull、Merge、Unite にした場合は、 ボーンビジビリティーアニメーションを設定したノードが圧縮されないようしてください（ノード圧縮不可設定プラグインで圧縮不可フラグを設定できます）。

ボーンビジビリティーアニメーションを使用する際は、Compress Bone を Unite All にしないでください。

Unite Child を ON にする場合は、ボーンビジビリティーアニメーションを設定したノードがまとめられないようにしてください。

デフォルトではすべてのノードのボーンビジビリティーアニメーションをバイナリーファイルに出力しますが、エクスポートプラグインのコンフィグファイルで、可視性にアニメーションが設定されているノードのみ出力するように設定できます。
