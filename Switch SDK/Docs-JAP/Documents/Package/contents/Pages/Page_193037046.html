nn::g3d ニュース

cmdl コンバータの追加

NintendoWare for CTR のモデル中間ファイル (.cmdl) ファイルを NintendoSDK のモデル中間ファイル (.fmdb) に変換するコンバータを追加しました。

NintendoWare for CTR からの移行の際にご利用ください。

シェーダーアノテーションの group の condition で uniform や renderinfo を条件判定で利用できるように

シェーダーアノテーションの group の condition ではこれまで option しか条件判定で利用できませんでしたが、uniform や renderinfo も利用できるようになりました。

// @@ group="render_state_opaque_group" label="不透明モード" condition="$renderinfo.render_state_mode == Opaque"

書式についてはシェーダーアノテーションのドキュメントをご覧ください。

nn::vfx ニュース

ビューアランタイム上にて 同名エミッタセットの衝突を回避する機能を追加

ビューアランタイムを利用してアプリ内エフェクトデータを編集する際、アプリ内に存在する編集対象と同名のエミッタセットも編集対象として認識してしまう不具合がありましたが、バイナリコンバータにエミッタセットファイルパスをバイナリ化する機能を追加し、これを回避できるようになりました。

この機能を利用するには、２点の作業が必要になります。

  • ゲーム内で読み込むエフェクトバイナリをパス付に変更する
  • EffectMaker から転送するエフェクトバイナリをパス付に変更する

エフェクトバイナリをパス付に変更するには、アプリ内で読み込むエフェクトバイナリコンバートオプションに --include-emitter-set-file-path-array を指定します。

EffectMaker から転送するバイナリをパス付に変更するには、EffectMaker の プロジェクト設定ファイル 上で <IncludeEmitterSetFilePathArrayInEditorMode> タグの記述が必要になります。この設定は、バイナリコンバートとは連動しないことにご注意ください。

注意：

エミッタセットファイルパスをバイナリに含めるのは、基本的に開発中のバイナリについて行うようにし、製品に含めるバイナリをコンバートする際には、本機能を使用しないでください。
エフェクトバイナリを解析された場合、開発環境のディレクトリ構成やエミッタセット名などの情報が漏えいするおそれがあります。

プリミティブのLOD操作メソッドを追加

プリミティブ( *.fmdb )内に含まれる LOD メッシュを利用して描画が行えるようになりました。

エフェクトランタイム上からのみ操作 void Emitter::SetPrimitiveLodLevel( uint8_t lodLevel ) にて LOD レベルの設定を行うことが可能です。
