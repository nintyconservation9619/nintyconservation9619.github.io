中間ファイルバージョンの更新

3D 中間ファイルバージョンを 3.7.0 から 4.0.0 へ更新しました。主な変更点については 3D 中間ファイル仕様書 の更新履歴をご覧ください。要素の削除や必須属性の追加など、中間ファイルフィルタースクリプトに影響する変更も含まれています。

既存の中間ファイルは 中間ファイルアップデーター で更新して下さい。この中間ファイルバージョン更新により、3D バイナリーコンバーターの出力するバイナリーリソースファイル(.bfres)に影響はありません。

特定条件で孫ペインが消えていた不具合の修正

以下の条件で、描画されるべき孫ペインがランタイム上で消える問題が発生していました。

NintendoSDK 5.0.0 で修正されますが、この修正によって消えていたペインが表示される可能性があります。

以下の条件をすべて満たす場合、NintendoSDK 5.0.0 より前のバージョンではランタイム上で孫ペインが消えていました。

  • 親ペインの「子階層に透明度の影響を与える」がオン
  • 親ペインの透明度が 255 以外
  • 子ペインの「子階層に透明度の影響を与える」がオフ
  • 子ペインの透明度が 0 もしくは非表示
  • 孫ペインが表示

下図は、上記の条件を満たすレイアウトを 4.4.0 のランタイムで描画した例です。

表示されるべき孫ペインが非表示になっています。

影響が出るペインを検出する方法

影響が出る可能性があるペインを検出するには、「Tools/Graphics/LayoutTools/SearchInfluenceAlphaChild.exe」を使います。

「ルートフォルダ」にすべての flyt と部品の flyt を含むフォルダを指定すると、以下の条件を満たす孫ペインがすべて列挙されます。

  • 親ペインの「子階層に透明度の影響を与える」がオン
  • 子ペインの「子階層に透明度の影響を与える」がオフ

透明度や非表示の有無に関しては、アニメーションやプログラム上で変更されうるため無視しており、影響が出ないペインが列挙されることがあります。

検出されたペインのうち透明度や非表示の有無が問題になるケースになっていないかをご確認ください。

実際には問題になる条件を合致するケースは稀なため、列挙されたペインのほとんどは問題ないことが多いです。

修正方法

問題となるペインの修正方法としては、列挙されたペインの「子階層に透明度の影響を与える」を有効化して見た目が変わらなければその解決方法が最も簡単です。

ただし「子階層に透明度の影響を与える」を有効化すると、その子ペインの透明度に影響が出る場合がありますのでご注意ください。

また、列挙されたペインの親ペインの「子階層に透明度の影響を与える」を無効化する方法もあります。

この場合も、その親ペインの下にあるペインの透明度に影響が出ますのでご注意ください。

バスルーティング設定を柔軟に行うための機能の追加（評価用）  

従来 6ch のバス 4 つで固定だったバスルーティングの設定を、「サブミックス」を用いてユーザーが柔軟に変更できる機能を評価用の機能として追加しました。
この機能を使用することで、バスの数をユーザーで増減させ、 4 つよりも多いバスを使用したルーティング設定を構築することが出来るようになります。 

詳しくは プログラミングガイド／サブミックス のページや、 AtkSubMix サンプル、サンプル解説書、以下の API リファレンスを参照してください。

  • nn::atk::SubMix
  • nn::atk::FinalMix
  • nn::atk::OutputReceiver
  • nn::atk::SoundSystem::SoundSystemParam::enableCustomSubMix
  • nn::atk::SoundArchivePlayer::SetDefaultOutputReceiver()
  • nn::atk::SoundHandle::SetSend()

ストリームサウンドバイナリのコンバート仕様の変更

ストリームサウンド用の波形ファイルにリージョン情報とループ情報が両方設定されていた場合、
これまではループ情報が無効化される仕様でしたが、今回の変更後はリージョン情報とループ情報の両方が出力されるようになります。 

これはストリームジャンプの機能を利用せずに、ループ再生して使うストリームサウンドから、
nn::atk::SoundArchivePlayer::ReadStreamSoundRegionDataInfo() でリージョン情報を読み取りたいようなケースにおいて、
以前の仕様では、ループ情報だけを持つ波形と、リージョン情報だけを持つ波形を用意する必要がありましたが、今回の変更により同じ波形を利用することが可能になります。

ただし、ストリームジャンプとループ再生の併用はこれまで通りできませんのでご注意ください。
今回の修正後は、ループ情報を含んだ状態で、ストリームジャンプ機能を利用した場合には、再生時にアサートで停止するようになります。
具体的には、ループ情報を持つストリームサウンドを再生するときにリージョンコールバックを指定した場合には、アサートで停止します。

一部のバイナリファイルのバージョンを更新

以下のバイナリファイルのバージョンを更新しました。

  • バイナリストリームファイル ( bfstm ) のバージョンを 0.6.2.0 に更新しました。
  • バイナリプリフェッチストリームバイナリ ( bfstp ) のバージョンを 0.5.2.0 に更新しました。

今回の更新においては、互換性が維持されているため、バイナリを再コンバートする必要はありませんが、「ストリームサウンドバイナリのコンバート仕様の変更」で触れているようにリージョン情報とループ情報の両方を利用したい場合には、再コンバートが必要になります。
