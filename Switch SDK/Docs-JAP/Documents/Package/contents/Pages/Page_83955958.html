LayoutEditor でのスケーラブルフォントの使い方

スケーラブルフォントの使い方

スケーラブルフォントを使ってレイアウトデータを作る 3 つの方式について説明します。
関連リンク スケーラブルフォントについての詳細は こちら を参照してください。

複合フォントを使う方式(推奨)

複合フォントを使うと、スケーラブルフォントを LayoutEditor やプログラム上から直接読み込むことができます。

複合フォントとは、複数のフォントを組み合わせて一つのフォントとする機能ですが、組み合わせない単体のスケーラブルフォントを扱う場合もお使い頂けます。

複合フォントを使うには「複合フォント」のページを参照してください。

プロジェクト設定ファイルでエディタに読み込む方式(従来手法)

スケーラブルフォントをプロジェクト設定ファイルによってエディタに読み込む手順について説明します。

<ScalableFontDescSet>
    <ScalableFontDesc>
        <FontName>ScFontA</FontName>
        <FilePath>.\commonRes\font\ttfont.ttf</FilePath>
        <FontSize>64</FontSize>
        <BoldWeight>0</BoldWeight>
        <BorderWidth>0</BorderWidth>
    </ScalableFontDesc>
    <ScalableFontDesc>
        <FontName>ScFontB</FontName>
        <FilePath>.\commonRes\font\nintendo_NTLG-DB_002.bfttf</FilePath>
        <FontSize>64</FontSize>
        <BoldWeight>0</BoldWeight>
        <BorderWidth>0</BorderWidth>
    </ScalableFontDesc>
</ScalableFontDescSet>

FontName は、重複しないようにしてください。
FilePath は、プロジェクトファイル相対でファイルパスを記述してください。ttf, otf, bfttf 形式が指定できます。
BoldWeight は 0.0 - 1.0 でボールドウェイトを指定します。(0.0で通常の太さです)
BorderWidth は 縁取りの太さを指定します。(0で縁取り描画がOFFになります。)

関連リンク プロジェクト設定ファイルについては「プロジェクトファイルフォーマット」を参照してください。

大きなサイズのフォントを利用する場合に、テクスチャキャッシュサイズが不足すると「?」と正しく表示されなくなります。
そうなった場合には、テクスチャキャッシュサイズを調整してください。

プログラム側での対応

実際にプログラムで表示する際にフォント名に応じてスケーラブルフォントに差し替えて表示を行うようにします。

現状の制限

ビューアでの表示確認に制限があります。

  • ふち付きフォントや、ボールドウェイトは表示に反映されない

スケーラブルフォントから作ったビットマップフォントを使う方式(従来手法)

レイアウトにビットマップフォント、実際にゲームで表示する際にスケーラブルフォントを利用する手順について説明します。

  • １．いったん FontConverter を利用してビットマップフォント(bffnt)化した仮フォントを使ってレイアウトを作成します。
  • ２．実際にプログラムで表示する際にスケーラブルフォントに差し替えて表示を行うようにします。

仮フォントと、実フォントでの表示差異はゼロではありませんが、ほぼ一致しています。

なお、仮フォントを作成する際に、カーニング情報を付加するようにご注意ください。
カーニングとは、前後の文字の組み合わせによって、自然に見えるように文字間隔を調整する機能です。
これを有効にしないと表示が食い違ってしまいます。

関連リンク FontConverter(GUI版)を使用したビットマップフォントの作成方法については「ビットマップフォントを作成する」を参照してください。
