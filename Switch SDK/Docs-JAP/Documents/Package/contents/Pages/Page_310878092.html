概要

セーブデータの削除、microSDカードへのバックアップ、microSDカードからのリストアを行えます。
セーブデータの作成については セーブデータ やFSライブラリのリファレンスを参照してください。

セーブデータはアプリケーション単位でリストされます。アプリケーション名は以下の場合には"Unable to load Application Name"が表示されます。

  • ホストPC上のアプリケーションを起動してセーブデータを作成した場合
    • VSI, Target Manager, RunOnTargetからの起動時が対象
  • アプリケーション名が設定されていないアプリケーションのセーブデータの場合
  • セーブデータ作成後にアプリケーションを削除した場合

使い方

アプリケーションセーブデータの表示

－/+ボタン押下でアプリケーションの全セーブデータを表示します。

詳細の表示

-/+ボタン押下でセーブデータの詳細を表示します。

ベリファイ

「Verify」ボタンを押すと、セーブデータが壊れていないかチェックします。

削除

「Delete」ボタンを押すと削除できます。

バックアップ

アプリケーションセーブデータの一覧で対象のセーブデータにフォーカスを合わせてAボタンを押すか、
もしくは詳細画面で「Export」ボタンを押すことでmicroSDカードにセーブデータをバックアップできます。

バックアップセーブデータは日付やアプリケーションIDを含む以下のパスにバイナリデータとして保存されます。

sd:/NintendoSDK/BackupSaveData/<yyyymmddhhmmss>_<applicationId>/<type>/<applicationId>/<xxxx>/

<type>にはセーブデータの種類に応じて以下の文字列が入ります。

種類 文字列
ユーザーアカウントセーブデータ user
BCAT bcat
キャッシュストレージ cache

リストア

ドロップダウンリストで「Backup in SD card」を選択し、対象のセーブデータにフォーカスを合わせてAボタンを押すことでリストアできます。
ユーザーアカウントセーブデータの場合は、リストア時に紐づけるアカウントを選択する画面が表示されます。 本体にアカウントが存在しない状態だとリストアはできません。
