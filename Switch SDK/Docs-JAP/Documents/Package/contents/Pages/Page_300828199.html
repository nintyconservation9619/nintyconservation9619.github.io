逆引きガイド

はじめに

nn::audio で実現できる機能について、具体的な事例から、その詳細な手順を調べるための逆引きリファレンスです。

オーディオレンダラ

DSP の処理負荷を PC 上で動作するツール上から見たい

SDK 付属の Spy ツールを使用することで実現できます。詳しくは Spy 及び SpyAudio サンプルをご覧ください。

1つのボイスから複数の SubMix / FinalMix に接続したい

1つのボイスからの複数の SubMix / FinalMix への接続は nn::audio::SplitterType を使うことで実現できます。詳しくは nn::audio::SplitterType の API リファレンス、及び AudioSplitter サンプルをご覧ください。

DSP 上で実行されるエフェクト処理の負荷を軽減したい

DSP 上で実行されるエフェクト処理の負荷を下げるために CPU 上でエフェクト処理を実行することができます。詳しくは nn::audio::AuxI3dl2ReverbType、nn::audio::AuxReverbType の API リファレンス、及び AudioEffectAuxUtility サンプルをご覧ください。
また、オーディオレンダラのレンダリング処理を全て CPU 上で行う仕組みも用意されています。詳しくは nn::audio::AudioRendererHandle の API リファレンス、及び AudioCpuRenderer サンプルをご覧ください。

オーディオ出力

オーディオ出力が対応していないサンプルレートの波形データを再生したい

nn::audio::ResamplerType を使用してリサンプル処理を行うことで実現できます。詳しくは nn::audio::ResamplerType の API リファレンス、及び AudioOutWithResampler サンプルをご覧ください。
