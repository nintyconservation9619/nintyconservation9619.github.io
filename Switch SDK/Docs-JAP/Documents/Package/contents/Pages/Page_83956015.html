機能説明

ここでは、OS ライブラリが提供する各機能を、API の使用方法と合わせて説明します。

機能一覧

OS ライブラリは以下に示す機能を提供していますが、これらの各機能に対してクラスユーティリティが提供されているものもあります。
詳細は クラス を参照して下さい。

機能 概要 クラスユーティリティ

スレッド

プロセッサ上で複数の処理を同時に実行するための実行単位を提供する機能です。 -----
スレッド固有領域 スレッド毎に異なる内容を記憶できる小規模な記憶領域を提供する機能です。 ThreadLocalStorage クラス
ファイバ スレッド上で更に複数のコンテキストを切替え可能な機能です。 -----
セマフォ 計数カウンタを利用して共有リソースを排他する機能です。 Semaphore クラス
イベント 二値のシグナル状態／非シグナル状態によってスレッド間同期機能です。 Event クラス
軽量イベント 二値のシグナル状態／非シグナル状態によってスレッド間同期機能です（軽量版）。 LightEvent クラス
メッセージキュー スレッド間で任意のデータを送受信するための同期機能です。 MessageQueue クラス
タイマーイベント イベント機能に、タイマーによる遅延シグナル化機能を追加した同期機能です。 TimerEvent クラス
多重待ち 複数の同期オブジェクトのいずれかのシグナル状態を一度に待機する同期機能です。 -----
ミューテックス 複数スレッド間で使用する共有リソースを排他制御するための同期機能です。 Mutex クラス
条件変数 ミューテックスと併用し、共有リソースの特定条件を待機するための同期機能です。 ConditionVariable クラス
Readers-Writer ロック 共有リソースを読込み目的もしくは書込み目的に応じた排他制御するための同期機能です。 ReaderWriterLock クラス
バリア 複数スレッド間で待ち合わせを行なうための同期機能です。 Barrier クラス
システムイベント システム通知イベントのシグナル化を待ち受けるための同期機能です。 SystemEvent クラス
時間管理 チックカウンタを提供する機能です。 -----
メモリ管理 動的メモリ確保などメモリに関する機能です。 -----
ユーザ例外ハンドラ ユーザ例外の発生に対して、独自の関数でハンドリングする機能です。 -----
