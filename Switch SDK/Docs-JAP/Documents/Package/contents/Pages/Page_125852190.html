2 サンプルの読み込み

シェーダー、モデル、アニメーションを読み込む手順を説明します。
パッケージ内にあるサンプルの各ファイル ( モデル、スケルタルアニメーション ) を読み込みます。

このチュートリアルでは以下の UI に触れています。

ドラッグ＆ドロップ

 ファイルを読み込みます。


モデルの読み込み

下記場所のサンプルモデル SampleTripod.fmdb を読み込みます。
「 SampleTripod.fmdb 」をフォルダから 3DEditor の画面にドラッグ＆ドロップします。
ファイルビューに「 SampleTripod.fmdb 」が表示されます。

モデル

%NINTENDO_SDK_ROOT%\Samples\Sources\Applications\G3dDemo\Resources\SampleTripod\SampleTripod.fmdb
参考：

チーム設定の追加参照パスに fsdb があるパスを指定しておくと fmdb を読み込んだ時に必要な fsdb が自動で読み込まれます。
このチュートリアルで起動する Start3dEditor.bat は自動でシェーダー（demo.fsdb , town.fsdb）が読み込まれるよう設定されています。

関連リンク 詳しくはチーム設定ファイルを参照してください。

スケルタルアニメーションの読み込み

下記場所の 3 個のサンプルアニメーションを読み込みます。
3 個のファイルをフォルダから 3DEditor の画面にドラッグ＆ドロップします。
ファイルビューに表示されます。

スケルタルアニメーション（左移動）

%NINTENDO_SDK_ROOT%\Samples\Sources\Applications\G3dDemo\Resources\SampleTripod\SampleTripod_MoveLeft.fskb

スケルタルアニメーション（右移動）

%NINTENDO_SDK_ROOT%\Samples\Sources\Applications\G3dDemo\Resources\SampleTripod\SampleTripod_MoveRight.fskb

スケルタルアニメーション（待機）

%NINTENDO_SDK_ROOT%\Samples\Sources\Applications\G3dDemo\Resources\SampleTripod\SampleTripod_Wait.fskb

すべて読み込むと下図のようになります。

シェーダーの確認

マテリアルにシェーダーが割り当てられているか確認します。

スケマティック表示からマテリアルをダブルクリックしてプロパティウインドウを開きます。

「BodyMt」と「EyeMt」のシェーダー定義が下図のように「demo」になっていることを確認します。

「未設定」の場合は赤枠をクリックして「demo」を選択します。

 

注意：

シェーダー定義が未設定の場合はビューアで何も表示されません。

このチュートリアルで使用する 3DEditor の起動バッチは、シェーダー（demo.fsdb , town.fsdb）が自動的に読み込まれます。
