NintendoWare グラフィックスは、以下のモジュールのことを指します。

  • nn::ui2d
  • nn::font
  • nn::vfx

NintendoSDK 0.8.0 で行われた NintendoWare グラフィックス の大規模な変更に関しては、NintendoWare グラフィックス の変更点 0.8.0 を参照ください。

Fontランタイムの変更点

古いカーニングファイル(*.bffkn)が読み込めない不具合の修正

古いカーニングファイル(NintendoSDK 0.10.0 より前に出力した *.bffkn)が、FontConverter で壊れて読み込まれる不具合を修正しました。

それに伴ってカーニングファイルのフォーマットの見直しを行いましたので、0.10.0 から 0.12.0 に含まれる FontConverter でカーニングファイルを出力していた場合は再出力をお願いいたします。

詳細は下記の表の通りです。

bffkn を生成した NintendoSDK のバージョン 格納できる文字コード 0.13.0 の FontConverter で読み込んだ場合
0.10.0 より前 16bit 正しく読み込める
0.10.0 から 0.12.0 32bit 壊れて読み込まれる(再出力してください)
0.13.0 32bit 正しく読み込める

ui2dランタイムの変更点

レイアウトと、アニメーション区間タグに、拡張ユーザー情報を追加

レイアウトと、アニメーション区間タグに、拡張ユーザー情報が設定できるようになりました。
以下のようなメソッドでアクセスします。

  • nn::ui2d::AnimResource::GetExtUserDataList
  • nn::ui2d::Layout::GetExtUserDataList

ツールでの設定については、「こちら」を参照ください。

アーカイブシェーダを 1 つに統合

従来の ui2d では、アーカイブファイルにシェーダバイナリが必要に応じて複数個含められていましたが、これらを 1 つに統合するように処理を変更しました。

シェーダバイナリの個数が増えるとアライメントによるメモリ消費量が増加してしまいますが 1 つに統合することで削減されます。
