プログラムの終了

本章ではプログラムの終了について説明します。

終了要求

開発機および製品機の環境では、ユーザ操作により HOME メニューからアプリケーションに対していつでも終了が要求されることがあります。このような場合には「プログラムへの通知メッセージ」ページの「終了要求の通知」に記載の方法で、アプリケーションを終了させて下さい。

自発的終了

製品機環境（および、HOME メニューが動作している開発機環境）では、アプリケーションは自発的に終了しないで下さい。何かしらの原因でアプリケーションが終了した場合は、HOME メニュー上でアプリケーションの終了を告知するようなシーンが表示されます。

開発機環境で DevMenu が動作している場合では、アプリケーションは nnMain() からリターンするか、std::quick_exit(0) を発行することでアプリケーションを自発的に終了できます。この場合は、単純に DevMenu の画面に復帰するという振舞いになります。

参考：

アプリケーションプログラムは、 nnMain() 関数からリターンすると std::quick_exit(0) を発行して終了します。そのため、NX32 および NX64 実機環境においては、静的オブジェクトのデストラクタなどは呼ばれず、リソースの破棄は全てシステムに任されます。

ただし、Win32 や x64 プラットフォーム向けにビルドされたプログラムは、std::quick_exit(0) の代わりに _exit(0) の呼び出しで終了します。

std::exit() 関数の禁止

std::exit() 関数を用いたプログラムの終了は動作保証されません。使用しないで下さい。
