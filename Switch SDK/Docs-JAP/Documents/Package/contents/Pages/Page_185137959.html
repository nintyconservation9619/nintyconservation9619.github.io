初版アプリケーション

ユーザにリリースされたアプリケーション、およびその nsp を指します。パッチのビルドを行うためには、リリースしたアプリケーションを正しく指定する必要があります。

修正版アプリケーション

パッチで修正したい内容を含んだ状態で、普通のアプリケーションのビルド方法で作成したアプリケーション、およびその nsp を指します。パッチは、一度修正版アプリケーションを作り、それと初版アプリケーションを指定することで作成します。

直前のパッチ

ユーザにリリースされたパッチのうち、一番最近リリースしたパッチ、およびその nsp を指します。2 番目以降のパッチをビルドする際には、直前のパッチを正しく指定する必要があります。

ファイルの部分パッチ

初版アプリケーションに存在していたファイルに対し、ファイルの一部のみの修正を行った場合に、その修正箇所のみをパッチに抱え込ませる機能を指します。NX では、条件を満たした場合にこれが自動的に有効になります。

オンカードパッチ

ゲームカード内に初版アプリケーションと一緒にパッチを抱え込ませることで、あらかじめ修正が当たった状態のカードを生産するための機能です。

必須バージョン

アプリケーションを起動するのに必須となるバージョンです。このバージョンに満たない状態では、本体システムによってそのアプリケーションの起動はできなくなります。この仕組みにより、パッチ開発時にセーブデータの前方互換性を気にする必要がなくなります。

必須バージョンは本体保存メモリーに保存されます。
