概要

NintendoSDK 向けライブラリおよびアプリケーションは、通常の Visual C++ プロジェクトと同じステップでビルド・実行できます。

ここでは、NintendoSDK が提供しているサンプルアプリケーションのひとつを例に取り、ビルド方法・実行方法を説明します。

サンプルプログラムの位置

サンプルプログラムは、NintendoSDK Samples パッケージに入っています。

NintendoSDK Samples パッケージをインストールすると、NintendoSDK\Samples\Sources\Applications 下にサンプルプログラムが配置されます。

アプリケーションのビルド

GfxPrimitiveRenderer サンプルを例に取り、サンプルプログラムのビルド手順を説明します。

手順 操作
1

NintendoSDK\Samples\Sources\Applications\GfxPrimitiveRenderer\GfxPrimitiveRenderer-spec.NX.vs2015.sln を開きます。

参考：

上記は、 Visual Studio 2015 を使用する場合の手順です。

Visual Studio 2013 を使用する場合は、同じフォルダに存在する GfxPrimitiveRenderer-spec.NX.vs2013.sln を開いてください。
2

Visual Studio の「ソリューションプラットフォーム」で「NX64」を選択します。

 
3

[ビルド] - [ソリューションのビルド] を実行します。

 

アプリケーションの実行

引き続き、GfxPrimitiveRenderer サンプルを例に取り、サンプルプログラムの実行手順を説明します。

手順 操作
1 前項の手順でサンプルプログラムをビルドしておきます。
2

Visual Studio の「ソリューションプラットフォーム」で「NX64」が選択されていることを確認します。

 
3

Visual Studio の[デバッグ] - [デバッグ開始] あるいは [デバッグ] - [デバッグなしで開始] を実行します。

 
4

Target Manager（後述）が起動し、サンプルが実行されると SDEV の液晶画面に次のような映像が表示されます。

 

補足：Target Manager

Target Manager は、PC に接続されている SDEV/EDEV を管理するソフトウェアです。

SDEV/EDEV を PC に登録したり、SDEV/EDEV 上で「ビルド済みのアプリケーション」を実行することができます。

 

Target Manager は、Visual Studio から SDEV/EDEV 上でアプリを実行する時に自動で起動します。

Visual Studio を使わずに SDEV/EDEV の操作を行う場合は、スタートメニューから手動で起動してください。

参考：

Target Manager の詳細な機能は、「NintendoSDK ドキュメント」の「機能 > 開発機の制御 > Target Manager」を確認してください。

この続きは、NintendoSDK ドキュメントの「チュートリアル」の章を確認してください。
