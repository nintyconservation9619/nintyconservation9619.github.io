アクセス速度依存確認

概要

記録メディアへのアクセス速度が変動しても正常に動作するようにすることがガイドラインで定められています。
 FS の速度エミュレーション機能により、3 つのパターンのアクセス速度でアプリケーションをデバッグすることが出来ます。

確認方法

注意：

NintendoSDK Firmware for NX 5.0.0-4 (NX Addon 5.3.0) 以上で初期化した開発機を使用してください。

  1. 開発機から SD カードを抜いてください。
  2. 動作確認するアプリケーション、追加コンテンツ、パッチを本体保存メモリーにインストールします。
    1. アプリケーションのインストール方法は アプリケーションの開発機へのインストール を参照してください。
    2. 追加コンテンツのインストール方法は 追加コンテンツの管理 を参照してください。
    3. パッチのインストール方法は パッチのビルド、実行方法 を参照してください。
  3. DevMenu の Debug メニュータブを選択し、FS Speed Emulation Mode を設定します。
    NX Addon 1.1.0 からは、DevMenuCommand からも設定ができます。詳細は DevMenuCommand を参照してください。
  4. アプリケーションを一通りプレイし、動作を確認します。

FS Speed Emulation Mode の設定は、開発機材の電源入れ直しやリセットにて初期状態に戻ります。


参考：

アプリケーション本体、パッチ、追加コンテンツの全てを開発機の本体保存メモリーにインストールする際に本体保存メモリーの空き容量が足りなくなる場合は、UHS-I（Ultra High Speed Phase I）対応（読み込み速度 60～95MB/秒）の SD カードを使用してください。

速度エミュレーション機能は記録メディアへのアクセスを遅くすることで実現しているため、アクセス速度が速い記録メディアを使用した方がより最適なテストが行えます。テスト対象は必ず本体保存メモリーに配置するようにし、できるだけその他のデータも本体保存メモリーに配置するようにしてください。そして、できるだけ速めの SD カードを使用するようにしてください。

速度エミュレーション機能について

速度エミュレーション機能は、記録メディアの中で最もアクセス速度が速い本体保存メモリーに対するアクセス速度を遅くすることで、アクセス速度が遅い記録メディアをエミュレーションします。
速度エミュレーション機能には以下のモードがあります。

  • Faster mode
    本体保存メモリーに対するアクセス速度を遅くせず、製品状態のセキュリティ処理を簡略化して動作します。
    開発機で可能な限りアクセス速度が速い動作を実現し、より速い環境（個体差による速さや将来のシステム改善による速さなど）で動作する状況を可能な範囲でカバーします。
    NintendoSDK SystemUpdater for NX 2.0.0-3 (NX Addon 1.0.0) からは、Faster mode は製品状態のセキュリティ処理を簡略化している分、製品状態に近い処理で動作する OFF より速く動作します。
    NintendoSDK SystemUpdater for NX 2.0.0-3 (NX Addon 1.0.0) より前では、インストールされるアプリケーションに対して製品状態のセキュリティ処理に対応していないため、Faster mode と OFF は同じ挙動です。
  • Slower mode
    本体保存メモリーに対するアクセスについて、そのアクセス方法に応じて一定の遅延を発生させます。
    市場で動作させられる可能性がある遅めの外部ストレージに相当する、定常的にアクセス速度が遅い記録メディアをエミュレーションします。
    NintendoSDK SystemUpdater for NX 5.0.0-1 (NX Addon 5.1.0) より前では、本体保存メモリーのみに配置されるデータへのアクセスも遅めの外部ストレージ相当の遅さになる制約があります。 
  • Random mode
    本体保存メモリーに対するアクセスについて、そのアクセス方法に応じてランダムな遅延を発生させます。
    記録メディア内部の応答性の変動をエミュレーションします。

Faster mode と Slower mode の実測値は パフォーマンス を参照してください。

Slower mode と  バックグラウンドで行うダウンロードストレステスト  は併用可能です。
ただし、Faster Mode と Random Mode はダウンロードストレステストと併用しないでください。ダウンロードストレスによりアクセスが遅くなり、定常的に速めなアクセス速度とランダムに変化するアクセス速度を意図したテストができなくなるためです。

速度エミュレーション機能は、将来にわたって NX の最高速度と最低速度を保証するものではないことをご留意ください。将来、記録メディアのシュリンクやシステムの改良により、アプリケーションからみたアクセス速度は検証時より速くなる可能性がありますし、ユーザーがより低速な市販の外部記録メディアを利用することも想定されます。
