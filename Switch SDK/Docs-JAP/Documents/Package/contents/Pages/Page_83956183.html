注意：

サンプルを確認する時の注意点

  • フォントファイルを使用しているサンプルデータを開く時は、データを開く前に、setup.bat ( Samples\Sources\Resources\Ui2d\Common ) でフォントファイルを生成してください。
    setup.bat は、使用するプラットフォームに合ったものを使用して下さい。
  • 部品ルートフォルダが、使用するバージョンの「 \NintendoSDK\Samples\Sources\Resources\Ui2d 」に設定されていることを確認して下さい。
  • レイアウトエディタウインドウのツールバーで、指定する「プラットフォーム」を確認して下さい。

PictureFont :

テキストボックスペインで以下のカラーフォントの表示を比較したサンプルです。

  • BC3（圧縮フォーマット）

Ui2dDemo

Ui2dDemo の「CapturePane」から起動する、プログラマ向けデモ Ui2dDemo の デモ画面です。


このデモではキャプチャペインの以下の機能が一覧できます。

  • DynamicCapture : 「初回フレームのみキャプチャ」機能をオフにし、全フレームをキャプチャしています。
  • CaptureFirstFrameOnly : 「初回フレームのみキャプチャ」機能を使用しています。
  • UserFilterShader : フレームバッファに描画されたモデルを含めてプログラム側で「ぼかし」フィルタをかけています。
  • TextureScale : 「テクスチャスケール」を強めにかけ、低解像度にしています。
  • FrameBufferCaptureTexture : ピクチャペインに「フレームバッファキャプチャテクスチャ」を使用しています。
  • ClearColor(Color,Alpha) : 背景がヌキのテクスチャを使用し、ヌキ部分にクリアカラーとアルファを指定しています。

キャプチャペインを使用した表現サンプルです。

\CapturePane\data\CapturePane_Expression.flyt

「｜赤線」左側は「 キャプチャペイン」階層下に「 テキストボックスペイン」や「 ピクチャペイン」を配置した例です。

「｜赤線」右側は キャプチャペインによって生成されたテクスチャを「 ピクチャペイン」で利用しています。


以下は利用方法に関する解説です。

A 「 ピクチャペイン」の詳細コンバイナ設定で、テキストの中にテクスチャのパターンを流しています。

B 「ピクチャペイン」にマスク用テクスチャを登録、テクスチャ座標をアニメーションし、テキストを左右から消しています。

C フラグメントオペレーションで加算合成した結果をキャプチャし、「 ピクチャペイン」でマスクしています。

ColorAndHighlight :

体力ゲージ量の加減で、カラーを変化させる動きです。

普通に 1 個のペインで製作した場合、白のハイライトを入れたりすると、ハイライトもカラーが変化してしまいます。

2 個のペインを使用しないで、1 個のペインで再現しています。

コンスタントカラーに乗せたい色を入れて、グレースケールのテクスチャに掛け算して、白のテクスチャを加算しています。

DistanceField :　

ディスタンスフィールドの欠点を補う為に「詳細なコンバイナモード」を使用したサンプルです。


使用テクスチャ

左 : 右 :


左 : ディスタンスフィールドの特性上、三角形の ◯ 部分が欠けてしまいます。

右 : 以下の方法で、三角形の ◯ 部分をシャープにしています。

  1. テクスチャの各カラーチャンネルごとに三角形 3 辺の画像を用意します。
    テクスチャ画像のチャンネルは、 R , G , B = , , のようにします。
  2. 用意したテクスチャを LayoutEditor に登録します。
  3. 「詳細なコンバイナモード」を使用し、各チャンネル画像ごとに倍率をかけてコントラストを上げます。
  4. 最後に「詳細なコンバイナモード」の最後の段で合成しています。

Flashing :

ボタンを決定した時に、少しだけ白くする動きです。

1 個のペインで設定しています。

白黒カラー補完だとうまく表現できないような、少しだけ色を変えたい動きを表現します。

2 個のペインを使用する時よりも、調整にまとまりがあります。

Mask :

金属的なものと別な箇所をくり抜きたい、一部分だけスペキュラを入れたい等の表現です。

テクスチャ２枚でスクロールしています。

マスクと乗せるもののテクスチャのサイズは違うものを使用しています。



左の領域 と右の領域でハイライトの入り方を変えたサンプルです。

1段目と2段目を別に計算した後、その結果を3段目で加算しています。

このとき前段のバッファと前段の出力を使用しています。
