3DEditor で LOD モデルをプレビューして確認する

3DEditor と 3dSampleViewer を使って、リダクションした LOD モデルをプレビューし、デフォルトパラメータ適用後の問題点を探します。

以下に LOD モデルプレビューまでの手順を示します。

3DEditor、3dSampleViewer を起動する

3DEditor (%NINTENDO_SDK_ROOT%\Tools\Graphics\3dEditor\3dEditor.exe)、3dSampleViewer(%NINTENDO_SDK_ROOT%\Samples\BuiltApplications\3dSampleViewer\3dSampleViewer.exe) を起動します。

起動方法の詳細については、3DEditor チュートリアルの「準備」ページの「3dEditor の起動」、「3dSampleViewer の起動」の項目をご覧ください。

3DEditor と 3dSampleViewer を接続する

3DEditor と 3dSampleViewer を起動します。

接続方法の詳細については、3DEditor チュートリアルの「ビューアと連携」ページの「ビューアに接続」の項目をご覧ください。

3DEditor にリダクションしたモデル中間ファイルを読み込む

3DEditor にステップ 1 でリダクションしたモデル中間ファイル bg_WhiteTown.fmdb を読み込みます。

3DEditor の GUI にファイルをドラッグ & ドロップすることで読み込むことができます。

ファイルを読み込むと、3dSampleViewer 上に読み込んだモデルが表示されることが確認できます。

モデルのプロパティウィンドウのプレビューページを開く

ファイルビューで bg_WhiteTown.fmdb をダブルクリックして bg_WhiteTown.fmdb のプロパティウィンドウを開きます。

プロパティウィンドウを開くとプレビューページが標準で開かれている状態になります。

プレビューする LOD レベルを指定する

プレビューページの「LOD プレビュー」の項目を 1 や 2 に変更すると、指定したレベルの LOD モデルが 3dSampleViewer でプレビューされます。

実行結果から、問題を探す

図２　Town を、オーソドックスな設定で実行した結果 （LOD レベル 1）（34365 ポリゴン）

青枠で囲まれた部分は、リダクションの結果が良くなかった個所を示す

（図2）に、デフォルトで実行した結果を示しています。この時、目立つ問題点が既にあります。

青枠内　黒っぽく見えます（＝UV 座標値が歪んでいるか、法線の問題か）。また、下段の青枠内の柱上のオブジェクトの形状の変化が大きいようです。次に、この問題を、パラメータを追加してゆくことで解決します。

このモデルはUVアトラス（MayaではUVシェルと呼ばれる）が複数存在しているため、デフォルトでの実行結果にもUVが考慮されています。
