リソース管理

リソース管理

概要

Atk によるオーディオリソース管理について説明します。

バイナリリソースファイル

Atk は、サウンドプロジェクトファイル、アセットファイルなどの編集用ファイルをコンバート、最適化して次のバイナリリソースファイルを作成します。
nn::atk は、編集用ファイルではなく最適化されたバイナリファイルを要求することで、効率よくリソースデータにアクセスできます。

Atk が取り扱うバイナリリソースファイルは、以下の通りです。

バイナリサウンドアーカイブ ファイル
(.bfsar) 

Atk の基本となるアーカイブで、Atk 用のサウンドリソースや音量などのメタデータが格納されます。

nn::atk は、このサウンドアーカイブ単位でサウンドの再生制御を行います。

nn::atk でサウンドを再生するためには、必ず１つ以上のサウンドアーカイブを作成する必要があります。

バイナリグループ ファイル

(.bfgrp)

Atk の基本的なリソースロードの単位となるファイルです。

埋め込みグループ、リンクグループの2種類は、サウンドアーカイブに内包されて一括管理されます。

 ユーザー管理グループは、外部ファイルとしてユーザーが任意の場所にファイル配置、管理します。

ユーザーは、サウンドアーカイブをまるごと一括でロードするだけでなく、このグループ単位でロードすることができます。

バイナリ波形アーカイブ ファイル

(.bfwar)

後述するバイナリ波形ファイル (.bfwav) のアーカイブファイルです。

バンクやウェーブサウンドセットが参照する波形ファイルをまとめることで、nn::atk が効率的にリソース管理できるようにします。

バイナリウェーブサウンド ファイル

(.bfwsd)

ウェーブサウンドのメタデータが格納されます。

nn::atk は、バイナリ波形アーカイブファイルと組み合わせて利用します。

バイナリバンク ファイル

(.bfbnk)

バンクのメタデータが格納されます。

nn::atk は、バイナリ波形アーカイブファイルと組み合わせて利用します。

バイナリシーケンスサウンド ファイル

(.bfseq)

シーケンスサウンドのコマンド情報が格納されます。

nn::atk は、バイナリバンクファイル、バイナリ波形アーカイブファイルと組み合わせて利用します。

バイナリ波形 ファイル

(.bfwav)

 オンメモリーで扱うための波形データが格納されます。

バイナリストリームサウンド ファイル

(.bfstm)

ストリーム再生用の波形データが格納されます。

このファイルはバイナリサウンドアーカイブファイルには含まれず、外部ファイルとして参照されます。

バイナリストリームサウンドプリフェッチ ファイル

(.bfstp)

ストリーム再生のための初回バッファ充填を不要にするための波形データが格納されます。

nn::atk は、バイナリストリームサウンドファイルと組み合わせて利用します。

各バイナリリソースファイルがどのファイルからコンバートされるかは ファイルフォーマット一覧 の 「SoundMaker でのコンバートの流れ」を参照ください。

バイナリリソースの分割ロード

nn::atk は、サウンド再生の際に必ずサウンドアーカイブを必要としますが、サウンドアーカイブに含まれるリソースデータは、分割してロードすることができます。

ここでは、そのバイナリリソースファイルを任意の粒度でロードする方法について説明します。

グループを分割し、グループ単位でロードする

リソースを分割ロードしたい単位に合わせて複数のグループを作成しておくと、プログラムから任意のタイミングでグループ単位のリソースロードが可能になります。

例えば、ゲームのシーンやアクターごとにグループを作成して、運用します。

グループの作成方法や、グループの種類についての詳細は、SoundMaker のグループ解説ページ を参照ください。

グループをプログラム上でロードする方法については、プログラミングガイドのリソース読み込み や サンプル解説書の AtkGroup サンプルの説明 を参照ください。

サウンドアーカイブを分割し、サウンドアーカイブ単位でロードする

リソースを分割ロードしたい単位に合わせてサウンドアーカイブを分割することで、プログラムから任意のタイミングでサウンドアーカイブ単位のリソースロードが可能になります。

ただし、nn::atk は、サウンドアーカイブ単位で再生制御を行いますので、サウンドアーカイブをまたいでサウンドに再生制限をかけたり、バイナリバンクファイルを共有したりすることはできません。
グループによるリソースの分割ロードで十分な場合は、そちらを利用することをおすすめします。
