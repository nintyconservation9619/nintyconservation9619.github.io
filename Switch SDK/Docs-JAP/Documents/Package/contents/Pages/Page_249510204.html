NintendoSDK 4.1.0


本バージョンの既知の不具合は以下の通りです。

  • 全般
    • OS のシステム ロケールをデフォルト値以外に変更した場合の動作は保証外
    • API リファレンスの関数などのアルファベット一覧が正しく表示されない
  • libcurl
    • 1 台の PC で複数のアプリケーションからの HTTPS が実行できない
  • Socket
    • nn::socket::Recv() で MSG_DONTWAIT フラグを使用できない
    • nn::socket::Select() もしくは、nn::socket::Poll() が EBADF を返すことがある
  • PERF
    • 特定の PC の環境において、サンプル PerfSimple が停止する
  • SSL
    • nn::ssl::Initialize() と nn::ssl::Finalize() は、メインスレッドで呼んでください
  • オーディオツール
    • nn::atk::viewer ライブラリを使ったインゲーム編集中に、シーケンスサウンドのファイルパスを変更するとプログラムが停止することがある
    • nn::atk::viewer ライブラリを使ってインゲーム編集のストリームサウンドを再生中に稀にプログラムが停止する
    • SoundMaker で cspj ファイルのインポートに失敗する
  • 3D-G3D
    • G3dDemo サンプルの Town シーンの実行中に不正アクセスが発生することがある
  • エフェクト-VFX
    • エフェクトデータのチャイルドエミッタの継承設定「パーティクルにエミッタ割当」が OFF の状態で「履歴式ストライプ」「履歴式ストライプ2」のいずれかを設定すると VFX ランタイムが停止する
  • レイアウト-Ui2d
    • 「結合モード」でエクスポートしたアニメーション区間タグを「分割モード」でインポートすると、区間の開始時間が0以外に設定される場合があり、編集もできない。
  • Mii
    • nn::mii::Database::UpdateLatest() が nn::mii::ResultBroken を返さない
  • DevMenu
    • マイページアプレットから「このソフトをはじめる」を実行すると、HOME ボタン以外効かなくなる

全般

OS のシステム ロケールをデフォルト値以外に変更した場合の動作は保証外

 

OS のシステム ロケールをデフォルト値以外に変更した場合の動作は保証外です。(例えば日本語版 Windows 上でシステム ロケールとして“日本語(日本)”以外を選択した場合)

API リファレンスの関数などのアルファベット一覧が正しく表示されない

API リファレンスで、「名前空間」→「名前空間メンバ」 の下の、「全て」 と 「関数」 と 「列挙値」で表示されるアルファベット一覧を選択しても正しいリストが表示されません。

libcurl

1 台の PC で複数のアプリケーションからの HTTPS が実行できない

症状

1 台の PC で複数のアプリケーションから HTTPS 通信を開始すると、エラーが発生する可能性があります。

回避策

回避策はありません。

修正予定

次回以降のリリースで修正予定です。

Socket

nn::socket::Recv() で MSG_DONTWAIT フラグを使用できない

症状

nn::socket::Recv() で MSG_DONTWAIT フラグをセットしても、ノンブロックモードになりません。

回避策

nn::socket::Fcntl() で O_NONBLOCK をセットすることで同等の動作にすることができます。

修正予定

次回以降のリリースで修正予定です。

症状

nn::socket::Select() もしくは、nn::socket::Poll() を呼ぶ際に ソケットディスクリプタが閉じられていると EBADF を返すことがあります。

回避策

回避策はありません。

PERF

特定の PC の環境において、サンプル PerfSimple が停止する

特定の PC の環境においてサンプル PerfSimple がアサートで停止する不具合があります。

この問題は今後のリリースで修正予定です。

SSL

nn::ssl::Initialize() と nn::ssl::Finalize() は、メインスレッドで呼んでください

症状

nn::ssl::Initialize() と nn::ssl::Finalize() をメインスレッド以外のスレッドで使用すると、動作が停止します。

回避策

nn::ssl::Initialize() と nn::ssl::Finalize() はメインスレッドで使用するようにしてください。

修正予定

次回以降のリリースで修正予定です。 

オーディオツール

nn::atk::viewer ライブラリを使ったインゲーム編集中に、シーケンスサウンドのファイルパスを変更するとプログラムが停止することがある

インゲーム編集の対象にしたシーケンスサウンドのファイルパスを変更すると、アサートが発生してプログラムが停止することがあります。現状以下のアサートが発生することを確認しています。

SDK Assertion Failure: 'velocity >= 0 &amp;&amp; velocity <= 127' in nn::atk::detail::driver::SequenceTrack::NoteOn()

本件は次回以降のリリースで改善予定です。

nn::atk::viewer ライブラリを使ってインゲーム編集のストリームサウンドを再生中に稀にプログラムが停止する

インゲーム編集対象のストリームサウンドの再生を繰り返したり、インゲーム編集でパラメータを高速に変更し続けたりしていると稀に以下のアボートが発生してプログラムが停止することがあります。 

Abort: 'delimiter != nullptr' in nn::fs::detail::GetMountName()

本件は次回以降のリリースで改善予定です。

SoundMaker で cspj ファイルのインポートに失敗する

症状

SoundMaker で cspj ファイルのインポートに失敗します。

回避策

NintendoSDK 3.5.0 以前の SoundMaker で cspj ファイルをインポートし、生成された fspj ファイルを使用するようにしてください。

修正予定

次回以降のリリースで修正予定です。

3D-G3D

G3dDemo サンプルの Town シーンの実行中に不正アクセスが発生することがある

環境によって Generic 版の G3dDemo サンプルの Town シーンの実行中に不正アクセスが発生することがあります。

この不具合は nn::perf の使用をやめたり、描画するモデル数を変更すると起きなくなることが確認されていますが、詳しい原因はわかっていません。

エフェクト-VFX

エフェクトデータのチャイルドエミッタの継承設定「パーティクルにエミッタ割当」が OFF の状態で「履歴式ストライプ」「履歴式ストライプ2」のいずれかを設定すると VFX ランタイムが停止する

症状

エフェクトデータのチャイルドエミッタの継承設定「パーティクルにエミッタ割当」が OFF の状態で「履歴式ストライプ」「履歴式ストライプ2」のいずれかを設定すると、VFX ランタイム内で不正アクセスが発生する不具合があります。

回避策

エフェクトデータの設定の「パーティクルにエミッタ割当」を ON に設定することで回避できます

修正予定

本件は次回以降のリリースで修正予定です。

レイアウト-Ui2d

「結合モード」でエクスポートしたアニメーション区間タグを「分割モード」でインポートすると、区間の開始時間が0以外に設定される場合があり、編集もできない。

症状

「分割モード」ではタグ区間の開始時間は0固定となっていますが、「結合モード」でエクスポートした区間タグをインポートすると0以外の値を設定出来てしまいます。また「分割モード」では区間タグの開始時間を編集出来ない仕様のため0に設定し直すこともできません。

回避策

分割モードで区間タグをエクスポートすることで開始時間を0にすることができます。開始時間が0以外の区間タグを読み込んでしまった場合はアニメーション区間タグウインドウから区間タグの削除を行ってください。

修正予定

本件は次回以降のリリースで修正予定です。

Mii

nn::mii::Database::UpdateLatest() が nn::mii::ResultBroken を返さない

症状

破損した nn::mii::CharInfo を nn::mii::Database::UpdateLatest() に渡しても、 nn::mii::ResultBroken 以外が返ります。

回避策

事前に nn::mii::CharInfoAccessor::IsValid() で破損判定を行ってください。

修正予定

本件は次回以降のリリースで修正予定です。

回避策

DevMenu ではなく HOME メニューを使用してください。
