別プロジェクトのインポート

NintendoWare SoundMaker プロジェクト以外の別プラットフォームのプロジェクトファイルをインポートすることができます。

参考：

ファイルのインポート後は、[プロジェクト]→[再コンバート] を行ってください。

別プラットフォームプロジェクトのインポートの方法

[ファイル]→[インポート]→[別プラットフォームのプロジェクト] からファイル選択ダイアログを開き、「ファイルの種類」で "別プラットフォームプロジェクト (*.cspj;*.rspj)" を選択することにより、NintendoWare 以外のサウンドプロジェクトを読み込ませることができます。

 

NintendoWare for CTR SoundMaker からのインポート

対象ファイルは、プロジェクトファイル (cspj)、サウンドセットファイル (csst)、バンクファイル (cbnk)、テキストシーケンスサウンドファイル (cseq) です。

インポート前のファイルはそのまま保持され、インポート後のファイルは同じフォルダに拡張子を変更して保存されます。

テキストシーケンスサウンドファイルインポートの具体的な処理内容は以下の通りです。

  • ファイル内に含まれる文字列 "cseq", "cinl" を "fseq", "finl" に一括置換します。
  • "cseq"ファイルについては、再帰的にインポート処理を行います。

「プロジェクト設定ダイアログ」の「サウンドアーカイブの出力先」が空欄の設定だった場合は、"output" が自動的に付加されます。

インストルメントの補間タイプが "補間なし" だった場合、インポート時に "線形補間" に変更されます。

NintendoWare for Revolution SoundMaker からのインポート

対象ファイルは、プロジェクトファイル (rspj)、サウンドセットファイル (rsst)、バンクファイル (rbnk)、テキストシーケンスサウンドファイル (rseq) です。

インポート前のファイルはそのまま保持され、インポート後のファイルは同じフォルダに拡張子を変更して保存されます。

テキストシーケンスサウンドファイルインポートの具体的な処理内容は以下の通りです。

  • ファイル内に含まれる文字列 "rseq", "rinl" を "fseq", "finl" に一括置換します。
  • "rseq"ファイルについては、再帰的にインポート処理を行います。

「プロジェクト設定ダイアログ」の「サウンドアーカイブの出力先」が空欄の設定だった場合は、"output" が自動的に付加されます。

バンクファイルのインポート方法

バンクリストがアクティブになっている状態で、[ファイル]→[インポート]→[バンクファイル] メニューを選択すると、各プロジェクトでのバンクファイル (*.cbnk;*.rbnk;*.bnk) のみをインポートすることができます。

 

NW4C のバンクファイルで、インストルメントの補間タイプが "補間なし" だった場合、インポート時に "線形補間" に変更されます。

サウンドセットファイルのインポート方法

プロジェクトファイルを開いている状態で、[ファイル]→[インポート]→[サウンドセット] メニューを選択すると、サウンドセットファイル (*.rsst;*.csst) をインポートすることができます。

 

関連したバンクファイルも同時にインポートされます。

stxt ファイルのインポート方法

stxt ファイルは、GAMECUBE 向けに提供されていた開発環境である sysdolphin のオーディオライブラリが利用していた効果音定義ファイルです。
この機能は sysdolphin で作られたゲームプロジェクトを移植する際に利用します。

サウンドセットタブにウェーブサウンドセットが開かれ、フォーカスがある時にのみ実行が可能です。 [ファイル]→[インポート]→[Stxt...] を実行して、インポートしたい stxt ファイルを指定してください。

 

stxt ファイルのインポートの動作

指定された stxt ファイルのファイル名を使って、ウェーブサウンドセットを作成します。 stxt ファイルの中に定義された効果音名は、その効果音名をウェーブサウンド名としてウェーブサウンドに変換します。

ウェーブサウンドには stxt ファイルに記述された再生情報から、以下の情報をインポートします。

  • 参照波形ファイル名とパス
  • ボリューム (AVOL コマンド)
  • プライオリティ (APRI コマンド)
    stxt ファイルに記載されたプライオリティは、チャンネルプライオリティに変換されます。
  • ピッチ (ATUNE コマンド)
  • パン (APAN コマンド)
  • サラウンドパン (ASPAN コマンド)
  • エフェクトミックス (AMIXA, AMIXB コマンド)
  • コメント (// で始まる記述)
    stxt ファイルにおいて、効果音の記述を行った {} 内の、一番最初のコメントのみをインポートします。 
参考：

stxt ファイルにおいて設定されていた各パラメータの値は、SoundMaker の値に自動的にスケーリングされます。 ただしプライオリティの値のみは、スケーリングを行わず、stxt ファイルの値がそのまま使用されます。

本リリースの時点では、stxt ファイルのインポートは、stxt ファイル内で効果音の先頭に指定されたパラメータのみに対応しています。
