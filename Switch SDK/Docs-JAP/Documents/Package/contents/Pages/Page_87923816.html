チーム設定ファイルフォーマット

チーム設定ファイルはXML形式のファイルです。

ここでは、チーム設定ファイルに含まれる要素を説明します。

<TeamSettings>

ファイルのルート要素です。

属性

名前 説明
Version チーム設定ファイルのバージョンです。現在のバージョンは 0.0.0.0 です。

内容

以下の要素を含みます。

  • <FileIo> : ファイルの読み込み時と書き込み時の動作の設定です。最大一つ出現します。
  • <Preset> : プリセットの設定です。最大一つ出現します。
  • <DefaultValue> : デフォルト値の設定です。最大一つ出現します。
  • <Error> : エラー の設定です。最大一つ出現します。
  • <Preview> : プレビューの設定です。最大一つ出現します。
  • <Color> : カラーの設定です。最大一つ出現します。

<FileIo>

ファイルの入出力にかかわる設定です。

内容

以下の要素を含みます。

  • <SearchPath> : 追加参照パスの設定です。0個以上出現します。
  • <StartUpPath> : スタートアップパスの設定です。0個以上出現します。
  • <AttachPath> : アタッチパスの設定です。0個以上出現します。
  • <ParentMaterialPath> : 親マテリアル参照パスの設定です。0個以上出現します。
  • <PreSaveCommand> : ファイルの保存前に実行するプログラムの設定です。最大一つ出現します。
  • <PostSaveCommand> : ファイルの保存後に実行するプログラムの設定です。最大一つ出現します。
  • <PreOpenCommand> : ファイルのオープン前に実行するプログラムの設定です。最大一つ出現します。
  • <PostCloseCommand> : ファイルのクローズ後に実行するプログラムの設定です。最大一つ出現します。
  • <PreBinarizeCommand> : ランタイム転送用にバイナリ化する前に実行するプログラムの設定です。最大一つ出現します。
  • <LogShaderConvertTime> : 最適化シェーダー生成時間をログバーへ表示する機能の設定です。最大一つ出現します。
  • <CreateShaderVariationCommand>: 最適化シェーダー生成用の fsva を生成するコマンドを指定します。最大一つ出現します。
  • <ShaderConverterAdditionalArgs> : シェーダコンバータに追加指定するコンバータオプションの設定です。最大一つ出現します。
  • <ShaderConverterAdditioanlArgsForOptimizedShader> : 最適化シェーダ生成時にシェーダコンバータに追加指定するコンバータオプションの設定です。最大一つ出現します。
  • <UserCommand> : 中間ファイルに対して実行する任意のプログラムの設定です。0 個以上出現します。
  • <RuntimeUserScript> : 接続先アプリケーションに送信する任意のスクリプトの設定です。0 個以上出現します。

<SearchPath>

追加参照パスの設定を含む要素です。

属性

名前 説明
Name 追加参照パス名です。相対パス指定の場合は省略可能です。
Recursive 参照パス以下のフォルダのサブフォルダも検索対象にするか指定します。true または false を設定します。省略した場合は false として扱われます。

内容

フォルダの相対パスまたは絶対パスです。% で囲われた環境変数を含めることができます。相対パスは参照元ファイルからの相対パスを指定します。

<StartUpPath>

スタートアップパスの設定です。

属性

名前 説明
Base 起点となる追加参照パスの名前です。スタートアップパスが追加参照パスからの相対パスでない場合は省略します。

内容

ファイルまたはフォルダの相対パスまたは絶対パスです。% で囲われた環境変数を含めることができます。相対パスは 追加参照パス からの相対パスを指定します。

<AttachPath>

アタッチパスの設定を含む要素です。

属性

名前 説明
Filter アタッチ可能なファイルの拡張子を指定します。複数指定する場合はカンマ区切りで記述します。
省略した場合はすべてのファイルが対象になります。

内容

フォルダの絶対パスです。% で囲われた環境変数を含めることができます。

<ParentMaterialPath>

親マテリアル参照パスの設定を含む要素です。

親マテリアルの追加・変更ダイアログで表示されます。

属性

名前 説明
Name 親マテリアル参照パスの名前です。名前の一部をパスのワイルドカードで一致した部分で置き換えることが出来ます。
Recursive 親マテリアル参照パス以下のフォルダのサブフォルダも検索対象にするか指定します。true または false を設定します。省略した場合は false として扱われます。

内容

ファイル、フォルダの絶対パスです。ワイルドカード(*または?)を使用することが出来ます。% で囲われた環境変数を含めることができます。

<PreSaveCommand>

ファイル保存前に実行するプログラムの設定を含む要素です。

対象は中間ファイル、プロジェクトファイルです。
指定されたプログラムには、保存先のファイルパスが行ごとに記述されたテキストファイル(UTF-8 BOM無し)のパスが引数として渡されます。

内容

ファイル保存前に実行するプログラムのパスを指定します。% で囲われた環境変数を含めることができます。

<PostSaveCommand>

ファイル保存後に実行するプログラムの設定を含む要素です。

対象は中間ファイル、プロジェクトファイルです。
指定されたプログラムには、保存先のファイルパスが行ごとに記述されたテキストファイル(UTF-8 BOM無し)のパスが引数として渡されます。

内容

ファイル保存後に実行するプログラムのパスを指定します。% で囲われた環境変数を含めることができます。

<PreOpenCommand>

ファイルオープン前に実行するプログラムの設定を含む要素です。

対象は中間ファイル、プロジェクトファイルです。
指定されたプログラムには、ファイルパスが行ごとに記述されたテキストファイル(UTF-8 BOM無し)のパスが引数として渡されます。

参考：

ファイルオープンのリドゥ前やファイルクローズのアンドゥ前にも実行されます。
既に開かれているファイルを開き直した時には、実行されません。 

内容

ファイルオープン前に実行するプログラムのパスを指定します。% で囲われた環境変数を含めることができます。

<PostCloseCommand>

ファイルクローズ後に実行するプログラムの設定を含む要素です。

対象は中間ファイル、プロジェクトファイルです。
指定されたプログラムには、ファイルパスが行ごとに記述されたテキストファイル(UTF-8 BOM無し)のパスが引数として渡されます。

参考：

ファイルクローズのリドゥ後やファイルオープンのアンドゥ後にも実行されます。
既に開かれているファイルを開き直した時には、実行されません。

内容

ファイルクローズ後に実行するプログラムのパスを指定します。% で囲われた環境変数を含めることができます。

<PreBinarizeCommand>

ランタイム転送用にバイナリ化する前に実行するプログラムの設定を含む要素です。

対象は中間ファイルです。
指定されたプログラムには、保存先のファイルパスが行ごとに記述されたテキストファイル(UTF-8 BOM無し)のパスが引数として渡されます。

属性

名前 説明
Filter プログラムに渡すファイルの拡張子を指定します。複数指定する場合はカンマ区切りで記述します。
省略した場合はすべてのファイルが対象になります。

内容

ランタイム転送前にバイナリ化する前に実行するプログラムのパスを指定します。% で囲われた環境変数を含めることができます。

<LogShaderConvertTime>

 最適化シェーダー生成時間をログバーへ表示する機能の設定を含む要素です。

内容

true または false を指定します。省略時は false として扱われます。

<CreateShaderVariationCommand>

 シェーダー最適化用のシェーダーバリエーション生成コマンドを指定します。省略時は、3DEditor の内部でシェーダーバリエーションを生成し、最適化シェーダーを生成します。

属性

名前 説明
AllowNoOutput

シェーダーアーカイブに対応するシェーダーバリエーションの一部のみを生成することを許容するかどうかを指定します。

true または false を設定します。省略した場合は false として扱われます。


シェーダーバリエーションを出力しなかった場合、通常はエラーになりますが、この属性を true に設定している場合、シェーダーバリエーションの作成を意図的にスキップしたとみなされ、接続先アプリケーションにシェーダーバリエーション作成がスキップされたことが通知されます。

具体的には nn::g3d::viewer::CallbackType_ShaderAssignUpdated コールバックが呼ばれる際に、シェーダーバリエーション作成がスキップされたマテリアルについては、nn::g3d::viewer::ShaderAssignUpdatedArg::MaterialShaderInfo::pResShaderArchive が NULL、nn::g3d::viewer::ShaderAssignUpdatedArg::MaterialShaderInfo::isOptimized が true、nn::g3d::viewer::ShaderAssignUpdatedArg::MaterialShaderInfo::isOptimizationSkipped が false で引数が渡されます。


この条件に一致するマテリアルについて、3DEditor が作成するシェーダーアーカイブの代わりに、アプリケーションが保有するシェーダーアーカイブをコールバック内で割り当てることができます。

これにより、シェーダー最適化時の 3DEditor のシェーダーコンバート時間を短縮することができます。

内容

コマンドを指定します。3DEditor はコマンドの第一引数 %1 に、処理対象のファイルを含むディレクトリのパスを渡します。コマンドはバッチファイルのように複数行にわたって記述できます。% で囲われた環境変数を含めることができます。

3DEditor はコマンド呼び出し前に処理対象のシェーダー定義 (.fsdb) とモデル (.fmdb) を第一引数のディレクトリに出力します。最適化シェーダーを生成する対象のマテリアル以外のマテリアルのシェーダー割り当ては空の状態でモデルが出力されます。シェーダーバリエーション生成コマンドは、第一引数のディレクトリに「シェーダーアーカイブ名.fsva」の形で、シェーダーバリエーションを出力するようにしてください。モデルに複数のシェーダーアーカイブが割り当てられている場合は、複数のシェーダーバリエーション (.fsva) を出力してください。

<UserCommand>

ファイルビューやオブジェクトビューのコンテキストメニューから選択中の中間ファイルに対して任意のプログラムを実行します。

現在サポートしているのはモデル中間ファイルのみです。

属性

名前 説明
Name

3DEditor 上で表示するコマンドの名前を指定します。"/" で区切ることにより、メニューを階層表示させることができます。

Filter プログラムに渡すファイルの拡張子を指定します。複数指定する場合はカンマ区切りで記述します。
省略した場合はすべてのファイルが対象になります。
Reload

プログラムで処理後の一時中間ファイルを 3DEditor に再読み込みしたい場合は true を指定します。省略した場合は false として扱われます。
Args

実行する UserCommand の引数を指定します。

Args には下記マクロがあり、3DEditor での選択状態などを取得できます。

マクロ 説明
$(ArgFile) 引数ファイルのパス(一時ファイルパスを書いたテキスト)が格納されます。
$(Materials) 実行対象がモデル中間ファイルの場合、実行対象中の選択中のマテリアル名が「Mat1 Mat2 Mat3」のようにスペース区切りで格納されます。
$(Bones) 実行対象がモデル中間ファイルの場合、実行対象中の選択中のボーン名が格納されます。格納形式は $(Materials) と同様です。
$(Shapes) 実行対象がモデル中間ファイルの場合、実行対象中の選択中のシェイプ名が格納されます。格納形式は $(Materials) と同様です。
$(Models) 実行対象がモデル中間ファイルの場合、モデル名が格納されます。

省略した場合は Args="$(ArgFile)" として扱われます。
Temporary

一時ファイルを出力せずに実ファイルパスを渡してユーザプログラムを実行したい場合は false を指定します。省略した場合は true として扱われます。

実ファイルを編集すると Reload の設定に関わらず、自動再読み込みが行われます。
ShortCut UserCommnad を実行するショートカットキーを指定します。ShortCut="Shift+A" のような形式で指定します。
FileTreeShortCut ファイルビューで選択しているファイルをショートカットキーの対象にしたい場合は true を指定します。false を指定するとオブジェクトビューで選択しているモデルが対象になります。

省略した場合は false として扱われます。

内容

実行するコマンドを指定します。% で囲われた環境変数を含めることができます。

ワイルドカードを指定すると一括でコマンドを登録することができます。

<UserCommand Name="フィルター" Filter="fmdb, fmda" Reload="true">%SCRIPT_ROOT%\Filters\*.bat</UserCommand>

<RuntimeUserScript>

3DEditor とアプリケーション接続中に任意の文字列と適用対象をアプリケーションに送信して、それをアプリケーションが実行します。
アプリケーションは文字列を好きなように解釈して、何を実行するかを g3d のビューアライブラリのコールバックで指定することができます。アプリケーション側の実装については 3DEditor から特定のモデルに対して任意の処理を実行する を参照してください。

現在サポートしているのはモデル中間ファイルのみです。

属性

名前 説明
Name

3DEditor 上で表示するスクリプトの名前を指定します。"/" で区切ることにより、メニューを階層表示させることができます。

内容

実行するコマンドを指定します。% で囲われた環境変数を含めることができます。

<Preset>

3DEditor で使用するプリセットの設定です。

内容

以下の要素を含みます。

  • <SettingName>: ユーザー設定及びアプリケーション設定の名前の設定です。
    最大一つ出現します。 
  • <EnableMaterialAnimCreation>: マテリアルアニメーション作成のメニューの表示設定です。
    最大一つ出現します。 
  • <SeparateMaterialAnimCreation>: マテリアルアニメーションを種類ごとに区別するかの設定です。
    最大一つ出現します。  
  • <ShowOnlyBinarizeEnabledBoneVisibilityAnimCurve>: バイナリ化するボーンビジビリティアニメーションのカーブのみ表示するかの設定です。
    最大一つ出現します。  
  • <TexturePresets> : テクスチャのプリセットの設定です。
    最大一つ出現します。
  • <FollowDccSamplerNameRule> : DCC ツールの命名規則に従ったサンプラー名を使用するかの設定です。
    最大一つ出現します。
  • <DisableEditTextureSrtWithOriginal> : DCC ツールで設定されたテクスチャ SRT を優先するかの設定です。
    最大一つ出現します。
  • <SamplerNamePresets> : サンプラー名のプリセットの設定です。
    最大一つ出現します。
  • <LightAnimPresets> : ライトアニメーションのプリセットの設定です。
    最大一つ出現します。
  • <LightDistAttnPresets> : ライトにおける距離減衰関数のプリセットの設定です。
    最大一つ出現します。
  • <LightAngAttnPresets> : ライトにおける角度距離減衰関数のプリセットの設定です。
    最大一つ出現します。
  • <FogDistAttnPresets> : フォグにおける距離減衰関数のプリセットの設定です。
    最大一つ出現します。
  • <PlatformPresets> : プラットフォーム毎の接続設定です。
    最大一つ出現します。

<SettingName>

ユーザー設定ファイル及びアプリケーション設定ファイルの名前を変更するための設定です。この設定は、チーム設定毎にアプリケーション設定ファイルやユーザー設定ファイルを切り替えるために使うことができます。

内容

ユーザー設定ファイル及びアプリケーション設定ファイルの名前を変更するための設定です。「3dEditor.application.config」「3dEditor.user.config」の「3dEditor」部分を指定した文字で変更することができます。

<EnableMaterialAnimCreation>

マテリアルアニメーション作成のメニューの表示設定です。この設定は、シェーダーパラメータアニメーション (.fspb, .fspa) 、テクスチャパターンアニメーション (.ftpb, .ftpa) 、マテリアルビジビリティアニメーション (.fvmb, .fvma) が廃止され、マテリアルアニメーションのみがサポートされるようになった時点で、廃止される予定です。

内容

マテリアルアニメーション (.fmab, fmaa) 作成のメニューを表示するか指定します。 false のときは、シェーダーパラメータアニメーション (.fspb, .fspa) 、テクスチャパターンアニメーション (.ftpb, .ftpa) 、マテリアルビジビリティアニメーション (.fvmb, .fvma) の作成メニューが表示されます。

<SeparateMaterialAnimCreation>

マテリアルアニメーション (.fmab, fmaa) を種類ごとに区別するかの設定です。区別する場合は、以下のように拡張子 (.fmab, .fmaa) を除いたファイル名の末尾で区別されます。

マテリアルアニメーションの種類 拡張子を除いたファイル名の末尾
シェーダーパラメータ _fsp
カラー _fcl
テクスチャ SRT _fts
テクスチャパターン _ftp
マテリアルビジビリティ _fvm

内容

マテリアルアニメーションを種類ごとに区別するかを指定します。区別する場合、カーブエディタには種類に対応したカーブが表示されます。アニメーション作成メニューには種類ごとの項目が表示されます。また種類が変更を伴う形のファイル名の変更はできなくなります。

<ShowOnlyBinarizeEnabledBoneVisibilityAnimCurve>

バイナリ化するボーンビジビリティアニメーションのカーブのみ表示するかを設定します。

内容

バイナリ化するボーンビジビリティアニメーションのカーブのみ表示するかを設定します。true の時は <bone_vis_bone_anim> の binarize_visibility が false のカーブを表示しません。

<TexturePresets>

テクスチャのプリセットの設定です。

内容

以下の要素を含みます。

  • <TexturePreset> : テクスチャのプリセットの項目です。0個以上出現します。

<TexturePreset>

テクスチャのプリセットの項目です。

属性

名前 説明
Name 項目の名前です。
quantize_type テクスチャフォーマットです。中間ファイルの texture_info の quantize_type に対応します。

<FollowDccSamplerNameRule>

DCC ツールの命名規則に従ったサンプラー名を使用するかの設定です。

内容

DCCツールが出力するヒントが設定されたサンプラーについて、命名規則に従ったサンプラー名を使用するかを設定します。
true または false を指定します。省略時は false として扱われます。

この設定の true、false でおおまかに以下のような違いがあります。


 true のとき false のとき
DCC ツールで追加されたサンプラーの編集 不可 可
サンプラー名の入力

制限あり

 制限なし

<DisableEditTextureSrtWithOriginal>

DCC ツールで設定されたテクスチャ SRT を優先するかの設定です。

内容

DCC ツールで設定されたテクスチャ SRT を 3DEditor で編集不可にするかの設定です。true または false を指定します。省略時は false として扱われます。

true の場合は、original_hint が空でない type が texsrt または texsrt_ex の <shader_param> は編集できなくなります。

サンプラー名のプリセットの設定です。
マテリアルプロパティウィンドウのサンプラーページのサンプラー名は選択肢になります。

内容

以下の要素を含みます。

  • <SamplerNamePreset> : サンプラー名のプリセットの項目です。0個以上出現します。

<SamplerNamePreset>

サンプラー名のプリセットの項目です。

属性

名前 説明
Name サンプラーの名前です。

ライトアニメーションのプリセットの設定です。

内容

以下の要素を含みます。

  • <LightAnimPreset> : ライトアニメーションのプリセットの項目です。0個以上出現します。

<LightAnimPreset>

ライトアニメーションのプリセットの項目です。

属性

名前 説明
Type ライトアニメーション種類を表す名前です。中間ファイルに出力されます。
Label ライトアニメーションのUI上に表示される名前です。省略した場合は Type と同じ値になります。

内容

 <ShaderConverterAdditionalArgs>, <ShaderConverterAdditionalArgsForOptimizedShader>

3DEditor がシェーダコンバータを呼ぶ際に追加指定するコンバータオプションの設定です。3DEditor が指定するオプションには、3DEditor が内部で指定するもの、<DeviceType> 内の <ShaderConverterAdditionalArgs> または<ShaderConverterAdditionalArgsForOptimizedShader> で指定するもの、<FileIo> 内の <ShaderConverterAdditionalArgs> または <ShaderConverterAdditionalArgsForOptimizedShader> で指定するものがあります。最適化シェーダ生成時は、<ShaderConverterAdditionalArgsForOptimizedShader> が使用されます。ただし、<DeviceType> 内で <ShaderConverterAdditionalArgsForOptimizedShader> を指定せず、<ShaderConverterAdditionalArgs> を指定した場合は、<ShaderConverterAdditionalArgs> が使用されます。<FileIo> 内の指定も同様です。最適化シェーダ生成時以外は <ShaderConverterAdditionalArgs> が使用されます。追加指定したオプションが 3DEditor が内部で指定するオプションと重複した場合は、3DEditor が内部で指定したオプションの値が優先されます。% で囲われた環境変数を含めることができます。

参考：

値をとるオプションを指定するときは、-- と = を使用してください。例 --dump-shader-source=DirectoryPath
なおこの制約は、将来廃止される予定です。 

参考：

3DEditor が内部で fsdb ファイルを生成するためにシェーダコンバータを呼ぶ際は、<ShaderConverterAdditionalArgsForOptimizedShader> や <DeviceType> 内の <ShaderConverterAdditionalArgs> は使用されません。関連する仕様は、将来変更される可能性があります。

属性

名前 説明

属性

名前 説明

min テクスチャ縮小時のフィルタを指定します。中間ファイルの filter の min に対応します。point, linear のいずれかを指定してください。
mip テクスチャミップマップ間のフィルタを指定します。中間ファイルの filter の mip に対応します。none, point, linear のいずれかを指定してください。

属性

名前 説明

max LOD の最大値を指定します。中間ファイルの lod の max に対応します。0 ～ 13 を指定してください。
bias LOD のバイアス値を指定します。中間ファイルの lod の bias に対応します。-32 ～ 31.984375 を指定してください。

<UIRange>

シェーダーパラメータの範囲の設定です。

属性

名前 説明
Clamp シェーダーパラメータとして入力できる値を ui_min, ui_max 内に制限するかどうかを指定します。制限するときは ui_default_min, ui_default_max がスライダーバーの範囲の初期値になります。制限しないときは ui_min, ui_max がスライダーバーの範囲の初期値になり、ui_default_min, uid_default_max は無視されます。省略時は true として扱われます。
