VSI

はじめに

本ドキュメントの目的

本ドキュメントでは、NintendoSDK に収録されている VSI (Visual Studio Integration) について説明します。

VSI では、Visual Studio を利用した NX アプリケーションのビルド・デバッグ機能を提供します。

対象読者

NintendoSDK を使ってアプリケーションを開発される方を対象とします。

各章の説明

VSI を使った新規プロジェクト作成のチュートリアルについては、新規プロジェクトの作成 をご覧ください。

本章で説明している内容は以下の通りになります。

ページ 内容
マニュアル VSI の基本的な仕様について説明します。
プロジェクトで指定できるパラメータ NX アプリケーションプロジェクトで指定できるパラメータを説明します。
ビルド済プログラムをデバッグする ビルド済の NX アプリケーションをデバッグする方法を説明します。
パッチの作成方法 パッチの作成方法について説明します。
動的ライブラリの使用方法 VSI から動的ライブラリを使用する方法を説明します。
Makefile の使用方法 外部ビルドシステムと連携したプロジェクトの使用方法を説明します。
ダンプ機能の使用方法 ダンプ機能の使用方法を説明します。
Natvis の使用方法 Visuao Studio の Natvis フレームワークの使用方法を説明します。
シンボルサーバーの使用方法 シンボルサーバーの使用方法を説明します。
ビルド・デバッグに関するトラブルシューティング ビルドやデバッグ実行に問題が発生した場合の解決手段を説明します。


関連資料

VSI のトラブルシューティングについてはトラブルシューティング (NX Addon)を参照してください。

VSI は Target Manager と連携して動作しますので、そちらのマニュアルもご覧ください。
