マニュアル

ここでは、GFX ライブラリーが提供する機能の使い方を説明します。

まずはじめに、GFX ライブラリーの配置について説明します。

GFX ライブラリー全般に渡る内容はプログラミング概要を、それぞれの機能の個別の詳細については機能説明を参照してください。

名前空間

GFX ライブラリーの提供する機能はすべて nn::gfx:: 名前空間に属しています。

nn::gfx::details 名前空間は内部的な機能であり、使用されることを意図していません。特別な理由がない限り使用しないでください。

ヘッダー

nn/gfx.h ヘッダーをインクルードすることにより、gfx ライブラリーの全機能を使うことが可能です。

nn/gfx.h は以下に配置されています。

$NINTENDO_SDK_ROOT/Include/nn/gfx.h

その他の GFX のヘッダーファイルは以下に配置されており、nn/gfx.h はこれらをインクルードしています。

$NINTENDO_SDK_ROOT/Include/nn/gfx/

ヘッダーは、C++03 で記述されています。C++11 以上の機能は使われていません。

C++03 以上の環境でこれらのヘッダーを使用することができます。

ライブラリーファイルの構成

libnn_gfx という名前のライブラリーファイルに、低レベルグラフィックス API に依存しない共通の機能と、対象の環境についてのデフォルトの低レベルグラフィックス API 用の機能が入っています。
デフォルトでない低レベルグラフィックス API を利用したい場合は、libnn_gfx に加えて、その専用の libnn_gfx* をリンクする必要があります。
たとえば、OpenGL がデフォルトの低レベルグラフィックス API でない環境で OpenGL を利用する場合、libnn_gfx と libnn_gfxGl をリンクする必要があります。

ソース

リビルドとデバッグを可能にする目的で、ソースコードを配置しています。

GFX ライブラリーのすべてのソースコードは以下に配置されています。

 $(NINTENDO_SDK_ROOT)/Sources/Libraries/gfx/

ソースコードもヘッダーと同様に C++03 で記述されています。

また、プロジェクトファイルは以下に配置されています。

$(NINTENDO_SDK_ROOT)/Sources/Libraries/gfx/libnn_gfx/libnn_gfx.vcxproj

Visual Studio からこのプロジェクトファイルを開くことで、GFX ライブラリーをリビルドすることが可能です。

特に理由がない限りは、リビルドしていない、配布されているライブラリーを使用してください。

依存ライブラリー

低レベルグラフィックス API のバリエーションとして OpenGL を使用している場合、NintendoSDK の提供するライブラリー以外に、ビルド時に OpenGL32.lib と glew32.lib に依存しており、実行時に glew32.dll に依存しています。glew ライブラリーは $(NINTENDO_SDK_ROOT)/Externals/glew に配置されています。gfx を使ったアプリケーションを作成する際はこれらをリンク・配置するようにしてください。
