fsp(fcl、fts)、ftp、fvm 中間ファイルフォーマットの統合

fsp(fcl、fts)、ftp、fvm は fma (マテリアルアニメーションファイル)フォーマットに統合されました。

fma フォーマットへの移行期間の間は fsp(fcl、fts)、ftp、fvm フォーマットも今までと同様に扱うことができますが、移行期間終了後は利用できなくなります。

移行期間終了のタイミングは未定ですが、終了する前のバージョンのリリースで次回バージョンでの移行期間終了を予告します。

以下に fma フォーマットへの移行に必要な情報を記載します。

fsp(fcl、fts)、ftp、fvm ファイルを fmab ファイルに変換する

%NINTENDO_SDK_ROOT%/Tools/Graphics/3dTools/ConvertToFmab.bat を利用することで fsp(fcl、fts)、ftp、fvm を fmab に変換することが可能です。

入力には変換したファイルパス、もしくはファイルを含むフォルダパスを指定します。フォルダを入力した場合は、フォルダ以下の変換可能な拡張子のファイルが再帰的に変換されます。

変換後のファイルは "[拡張子を除く入力ファイル名]_[旧拡張子].fmab" という名前で入力ファイルと同じフォルダに出力されます。例えば、human.fspb を入力した場合、human_fsp.fmab というファイルが出力されます。

ConvertToFmab.bat human.fsp human.ftp human.fvm ./ResourceDirectory

ConvertToFmab.bat に変換したいファイルやフォルダをドラッグ & ドロップすることでも変換することができます。

Maya プラグイン、3ds Max プラグインで fcl、fts、ftp ファイルから fma ファイルを出力するように切り替える

Maya プラグイン、3ds Max プラグインでは標準で fcl、fts、ftp が出力されるようになっていますが、コンフィグファイル(Export.ini)で fma を出力するように設定を切り替えることができます。 

use_fcl_fts_ftp="false"

fma フォーマットで出力するように切り替えた場合、標準ではテクスチャーパターンアニメーション、カラーアニメーション、テクスチャー SRT アニメーションがひとつの fma ファイルとして出力されます。

これを以前と同様に各種類のアニメーションを個別の fma ファイルとして書き出したい場合は、コンフィグファイルで以下の項目を指定することで、別々に出力されるように切り替えることができます。

separate_material_anim="true"

このとき、各種類のアニメーションは "[出力名]_[旧拡張子].fmab" という名前で出力されます。例えば、出力名が "Human" だった場合、カラーアニメーションは "Human_fcl.fmab"、テクスチャー SRT アニメーションは "Human_fts.fmab"、テクスチャーパターンアニメーションは "Human_ftp.fmab" というファイル名で出力されます。

3DEditor で新規作成される fsp(fcl、fts)、ftp、fvm ファイルを fma ファイルに切り替える

3DEditor では特に設定をしなくても、fsp(fcl、fts)、ftp、fvm と fma のどちらも読み込み、保存、編集が可能です。

ファイルビューで右クリックしたときに表示されるアニメーションの新規作成メニュにおいて、標準では下記の項目が表示されるようになっています。

  • シェーダパラメータ
  • カラー
  • テクスチャー SRT
  • テクスチャーパターン
  • ボーンビジビリティ
  • マテリアルビジビリティ

これを fma ファイルを作成するメニュに切り替えるために、チーム設定で以下の設定を行います。

<Preset>
    <EnableMaterialAnimCreation>true</EnableMaterialAnimCreation>
</Preset>

この設定をすると、新規作成メニュでは下記の項目が表示されるようになります。

  • マテリアルアニメーション
  • ボーンビジビリティ

fma ファイルを作成する際に、これまでと同様に各種類のアニメーションを個別のアニメーションとして作成できるようにしたい場合は、チーム設定で下記のような設定を行います。

<Preset>
    <EnableMaterialAnimCreation>true</EnableMaterialAnimCreation>
    <SeparateMaterialAnimCreationMenu>true</SeparateMaterialAnimCreationMenu>
</Preset>

この設定を行うと、3DEditor のアニメーションの新規作成メニュには下記の項目が表示されます。

  • マテリアル(シェーダパラメータ)
    • [モデル名]_fsp という名前がデフォルト名として挿入されます
  • マテリアル(カラー)
    • [モデル名]_fcl という名前がデフォルト名として挿入されます
  • マテリアル(テクスチャー SRT)
    • [モデル名]_fts という名前がデフォルト名として挿入されます
  • マテリアル(テクスチャーパターン)
    • [モデル名]_ftp という名前がデフォルト名として挿入されます
  • マテリアル(マテリアルビジビリティ)
    • [モデル名]_fvm という名前がデフォルト名として挿入されます
  • ボーンビジビリティ

複数の fma ファイルをひとつの fma ファイルにまとめる

中間ファイルオプティマイザのコンバイン機能を利用することで、複数の fma ファイルをひとつの fma ファイルにまとめることができます。

3dIntermediateFileOptimizer.exe Human_ShaderParamAnim.fmab  -o Human.fmab --combine --combine-options "Human_TexturePatternAnim.fmab Human_MaterialVisibilityAnim.fmab"

中間ファイルとしてはこれまでと同じようにシェーダパラメータアニメーション、テクスチャーパターンアニメーション、マテリアルビジビリティアニメーション毎にファイルを分けて管理しておき、バイナリ化する直前にひとつのマテリアルアニメーションにまとめてからバイナリ化するということも可能になります。

バイナリコンバート時の注意

3D バイナリコンバーターは fsp(fcl、fts)、ftp、fvm フォーマットのファイルと fma フォーマットの同時入力をサポートしていませんので、fma フォーマットへの移行時にこれらのフォーマットを混ぜて入力しないようご注意ください。

Use 3DEditor の廃止

エクスポートプラグインのオプションウィンドウの Use 3DEditor オプション(use_3deditor、lock_use_3deditor)は廃止になりました。

Use 3DEditor がなくても 3DEditor に読み込み済みの中間ファイルであれば、エクスポート後で上書きした際に自動で再読み込みされます。

また、コンフィグファイルの user_custom_ui を用いて 3DEditor の起動や任意の中間ファイルの 3DEditor への読み込みなどを行うボタンをエクスポートプラグインのオプションウィンドウに追加することができます。

3DEditor の起動や 3DEditor へのファイル読み込みは 3DEditorBridge を使用することで可能です。

もし Use 3DEditor と同等の機能が必要な場合にも  user_custom_ui と 3DEditorBridge を使用することで実現可能です。

以下に Use 3DEditor と同等の機能を実現する例を示します。

  1. user_custom_ui に Use 3DEditor と同等のチェックボックスを作成する MEL プロシージャを指定します

    global proc CreateUse3dEditorUi()
{
    columnLayout;
        checkBox -l "Use 3DEditor" use3dEditorCheckBox;
    setParent ..; // columnLayout
} 
    		 
    user_custom_ui="CreateUse3dEditorUi"

  2. ポストエクスポートスクリプトで 3DEditorBridge で 3DEditor の起動、3DEditor への出力ファイル読み込みを行います

    global proc Use3dEditorOnPostExportScript()
{
    int $use3dEditor = `checkBox -ex use3dEditorCheckBox`&&`checkBox -q -v use3dEditorCheckBox`;
    if ($use3dEditor)
    {
        string $bridge = "%NINTENDO_SDK_ROOT%/Tools/Graphics/3dEditor/3dEditorBridge.exe";
        // チーム設定を指定して 3DEditor を起動します
        system($bridge + " start-3deditor --team-settings-file %MY_3DEDITOR_TEAM_SETTINGS_FILE_PATH%");
        string $temp_output = `getenv NINTENDO_EXPORT_TEMP_FMD_FILE_PATH`;
        string $output = `getenv NINTENDO_EXPORT_FMD_FILE_PATH`;
        if ($temp_output != "")
        {
            // use_temp_folder_for_script が true の場合は出力先に一時ファイルをコピーしてから 3DEditor に読み込む
            sysFile -copy $output $temp_output;
        }
        system($bridge + " load-files " + $output);
    }
}

マテリアルアニメーションに関する制約事項
  • fsp(fcl, fts)、ftp、fvm 中間ファイルをひとつのアーカイブに含める場合、今までは同じ中間ファイル名でコンバートができましたが、それぞれ異なる中間ファイル名を割り当てる必要があります。例えば、今までは human.fspb、human.ftpb、human.fvmb をひとつのアーカイブに含めることができましたが、human_fsp.fspb、human_ftp.ftpb、human_fvm.fvmb のように異なるファイル名を割り当てないとひとつのアーカイブにまとめることができなくなりました。
  • fma と従来の fsp(fcl, fts)、ftp、fvm ファイルを同時に入力する事はできません。これらのファイルを同時に入力するとエラーになります。
