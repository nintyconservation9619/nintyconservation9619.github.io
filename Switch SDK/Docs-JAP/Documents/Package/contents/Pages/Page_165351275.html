NX 固有の変更がある API

以下の各関数の中には、OS とのインターフェースに起因して標準の API とは異なる動作を示すものがありますが、それ以外の点では標準の API 同様に機能します。

stdlib.h int clearenv(void);

説明

外部変数 environ の値を NULL に設定しますが、その変数を解放することはありません。

putenv() を初めて呼び出すと、malloc（） が呼び出されて environ にメモリが割り当てられます。

clearenv() を呼び出しても environ は解放されないので、プログラムを終了する前に "free(environ) を呼び出す必要があります。


インターフェース関数

該当なし

string.h errno_t memset_s(void *dest, rsize_t destsz, int ch, rsize_t count);

説明

memset_s() を使用するためには、プログラマは、以下のヘッダをインクルードすると共に、対応するマクロを以下に示す順序で定義する必要があります。


#include <features.h>

#if defined(__STDC_LIB_EXT1__) || defined(__NN_STDC_LIB_EXT1_MEMSET_S__)
#define __STDC_WANT_LIB_EXT1__ 1
#include <string.h>
#endif


…
<use memset_s()>
…

*_s 関数が完全にサポートされていないため、現在 Rynda では __STDC_LIB_EXT1__ と __STDC_WANT_LIB_EXT1__ は定義されていません。


現在 RSIZE_MAX、errno_t および rsize_t は string.h で定義されています。これらは標準の記述とは異なっています。 これらのマクロは、サポートされる *_s 関数が増えると、特別なヘッダに移動する可能性があります。


外部情報

http://en.cppreference.com/w/c/string/byte/memset
