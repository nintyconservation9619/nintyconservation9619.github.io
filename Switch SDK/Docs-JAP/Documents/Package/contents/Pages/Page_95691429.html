デスクリプタープール

デスクリプタープール

概要

デスクリプタープールは、デスクリプターを格納し管理するための機能です。

ここで管理されるデスクリプターは、そのひとつひとつが GPU によって参照されるひとつのリソースを記述しています。デスクリプターのサイズは動的に問い合わせることができますが、その内容は不透明です。

デスクリプタープールにデスクリプターをセットする際は、まず更新を開始する必要があります。このとき、デスクリプタープールには種類があり、ある種類のデスクリプタープールに対してはその種類のデスクリプターしかセットすることができません。もはやデスクリプタープールにデスクリプターをセットする必要がなくなれば、更新を終了する必要があります。更新は何度でも可能ですが、GPU によって使用されている間に更新してはいけません。

デスクリプタープールにデスクリプターをセットするそれぞれの領域はデスクリプタースロットとして抽象化されます。低レベルグラフィックス API のバリエーションによってはいくつかのデスクリプタースロットがシステムによって予約されていることがあることに注意してください。

情報

デスクリプタープールの情報は nn::gfx::DescriptorPoolInfo として定義されています。

デスクリプタープールの種類

デスクリプタープールの種類は SetDescriptorPoolType により指定されます。
ここで設定した種類以外のデスクリプターをセットする試みはエラーを引き起こします。

デスクリプタースロットの数

SetSlotCount によりデスクリプタースロットの数を指定します。
スロット数を多く設定するほど、オブジェクト側で必要なメモリープールの量も多くなります。

オブジェクト

 デスクリプタープールのオブジェクトは nn::gfx::DescriptorPool として定義されています。

 初期化と破棄

デスクリプタープールの初期化と破棄は通常のオブジェクトの方法に従います。

メモリープール

必要なメモリープールのサイズは、初期化するつもりの情報を引数として CalculateDescriptorPoolSize を呼ぶことによって計算できます。

デスクリプタープールを初期化する際に指定するメモリープールへのオフセットにはアライメント制限があり、その値は nn::gfx::DescriptorPool::GetDescriptorPoolAlignment により取得できます。

更新の開始と終了

デスクリプタープールの更新の開始は BeginUpdate により、更新の終了は EndUpdate によりそれぞれ行われます。これらは必ず対になって呼ばれる必要があります。デスクリプタープールにデスクリプターをセットする際は、デスクリプタープールの更新が開始されている状態である必要があります。

デスクリプターのセット

デスクリプターをセットするための関数はデスクリプタープールの種類によって異なります。
以下はデスクリプタープールの種類と対応する関数の一覧です。

デスクリプタープールの種類 関数
nn::gfx::DescriptorPoolType_BufferView SetBufferView
nn::gfx::DescriptorPoolType_TextureView SetTextureView
nn::gfx::DescriptorPoolType_Sampler SetSampler

関数には、セットするスロット番号と、セットする対象のリソースを指定します。

デスクリプタースロットの取得

GetDescriptorSlot により指定したスロット番号のデスクリプタースロットを取得します。
デスクリプタースロットは、コマンドバッファーにおけるさまざまな関数に利用されます。

デスクリプタープールのセット

nn::gfx::CommandBuffer::SetDescriptorPool を呼ぶことにより、使用するデスクリプタープールをセットします。デスクリプタープールの切り替えは低レベルグラフィックス API によってはとてもコストの高い操作として扱われることに注意してください。

デスクリプタースロット

デスクリプタースロットは、デスクリプタープールにおける特定のスロットを表現します。

デスクリプタースロットは nn::gfx::DescriptorPool の GetDescriptorSlot 関数により取得できます。
コマンドバッファーへのリソースのバインディングはデスクリプタースロットを通して行われます。 よって、一度デスクリプタースロットを取得すれば、リソースのセットのために元のデスクリプタープールまで参照しにいく必要はありません。

デスクリプタースロットは他の多くのオブジェクトとは異なり、オブジェクト自身のコピーを許可するよう設計されています。

デスクリプタースロットのオフセット

デスクリプタープールにおけるデスクリプターの並びを知っている場合は、元のデスクリプタープールから毎回デスクリプタースロットを取得することなく、取得したデスクリプタースロットをオフセットすることで別のデスクリプタースロットを参照することができます。デスクリプターのサイズは低レベルグラフィックス API やその種類によって異なるため、次のデスクリプタースロットを参照するためのオフセット値は、nn::gfx::DescriptorPool::GetDescriptorSlotIncrementSizeにより取得する必要があります。たとえばこの取得された値の 10 倍の値をオフセットすると、10 スロット先のデスクリプタースロットを参照するようになります。

デスクリプタースロットの有効性

デスクリプタースロットは無効な状態として構築されます。nn::gfx::DescriptorPool::GetDescriptorSlot により取得されたデスクリプタースロットは有効な状態となります。
デスクリプタースロットの有効性は nn::gfx::DescriptorSlot::IsValid により判定できます。また、nn::gfx::DescriptorSlot::Invalidate 関数を呼ぶことで明示的に無効な状態にすることもできます。

デスクリプタープールにおけるインデクスの取得

デスクリプタースロットがデスクリプタープールにおいて何番目のスロットなのかを取得するには nn::gfx::DescriptorPool::GetDescriptorSlotIndex を使います。デスクリプタースロットをオフセットしてある場合はオフセットを考慮したインデックスが取得されます。問い合わせるデスクリプタースロットは対象のデスクリプタープールから取得されたデスクリプタースロットである必要があり、そうでない場合に取得される値は未定義です。
