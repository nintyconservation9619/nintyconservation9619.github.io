概要

ユーザーセーブデータについて、本体システムはユーザーに次の機能を提供します。

  • ユーザーアカウントの移行
    • ユーザーセーブデータを別の本体に移動する機能
    • System version 4.0.0 よりユーザーに提供済み
  • ユーザーセーブデータのバックアップ
    • ユーザーセーブデータを本体内蔵ストレージ以外の場所に複製する機能
    • 提供時期未定
  • ユーザーセーブデータのコピー
    • ユーザーセーブデータを別のユーザーアカウントに対して複製する機能
    • 提供時期未定

このセクションでは、それぞれの機能がもつ性質について説明します。

ユーザーアカウントの移行

ユーザー単位で一括して、ユーザーセーブデータを別の本体に移動する機能を提供します。
この機能を「ユーザーアカウントの移行」と呼びます。

ユーザーアカウントの移行では、ユーザーアカウント (nn::account::Uid やプロフィール情報) と、そのユーザーのすべてのユーザーセーブデータが別の本体に移動します。
移行の前後では、移行したユーザーセーブデータに関して、所有者であるユーザーアカウントは変更されません。
一方で、ユーザーアカウント以外の情報（本体設定や一時領域等）は移行先のものが使用されます。 

ユーザーセーブデータのバックアップ

参考：

この機能の提供時期は未定です。


あるユーザーセーブデータを本体内蔵ストレージ以外の場所に複製する機能を提供します。
この機能を「ユーザーセーブデータのバックアップ」と呼びます。
アプリケーションがこの機能をサポートするかは、アプリケーションごとに選択できます。

ユーザーセーブデータのバックアップでは、ユーザーは随時ユーザーセーブデータのバックアップを作成することができます。
バックアップされたユーザーセーブデータに対してユーザーは自由に、同じユーザーアカウントのユーザーセーブデータとして復元できます。

ユーザーセーブデータのバックアップ・復元の前後ではユーザーセーブデータの所有者 (nn::account::Uid) は変更されませんが、ユーザーセーブデータの巻き戻りが発生する場合があります。

ユーザーセーブデータのコピー

参考：

この機能の提供時期は未定です。

あるユーザーセーブデータを別のユーザーアカウントに対して複製する機能を提供します。
この機能を「ユーザーセーブデータのコピー」 と呼びます。
アプリケーションがこの機能をサポートするかは、アプリケーションごとに選択できます。

このとき複製先のユーザーアカウントが、同じ本体に存在するものか別の本体上のものかは、区別されません。
従ってユーザーセーブデータのコピー前後で、本体システムから取得可能なすべての情報が変化する可能性があります。 
（アプリケーションが本体システムから取得した情報に依存していなければ、この影響はありません。） 

ユーザーセーブデータのコピーのサポートの推奨

ユーザーセーブデータのコピー機能をサポートすることで、アプリケーションは、「セーブデータを、ユーザーアカウントと関係なく自由に管理したい」という欲求をもつユーザーに対して、付加価値を提供できます。
また、ユーザーセーブデータのコピー処理は本体システムが行うため、アプリケーションが特別な機能を実装する必要はありません。 

もしアプリケーションに、ユーザーセーブデータのコピーを避ける仕様上の理由（他のユーザーとの間でユニーク性が必要、等）がない場合、この機能をサポートすることを推奨します。

注意：

ユーザーセーブデータのコピーをサポートする場合、ユーザーセーブデータに記録するにあたり特に注意が必要な情報が存在します。
詳細は「ユーザーセーブデータへの使用に注意を要する情報」を参照してください。

注意：

ユーザーセーブデータのコピーをサポートすると、コピー先でもまたバックアップと復元が行われるようになります。
この場合、バックアップから復元可能な期間に対する制限は、コピー先とコピー元とでは別個に検証されます。
従って実質的には、セーブデータのコピーをサポートする場合、アプリケーションの「ユーザーセーブデータのバックアップに対する制限」は実効性を失います。

ユーザーセーブデータのコピーをサポートする場合は、バックアップからの復元に対して制限がないことを確認してください。
