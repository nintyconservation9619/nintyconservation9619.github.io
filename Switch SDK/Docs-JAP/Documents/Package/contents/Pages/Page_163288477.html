VfxViewerSimple サンプルについて

はじめに

ビューアランタイムを利用した VfxViewerSimple サンプルの説明です。ビューアランタイムに関する詳しい情報は下記リンクを参照ください。

関連リンク 以下は関連ページへのリンクです。

インゲーム編集のワークフロー エフェクトをゲームアプリケーション上で直接確認しながら作成するワークフロー「インゲーム編集」について説明します。
ランタイム組み込みチュートリアル アプリにエフェクトランタイムを組み込んで再生するまでのチュートリアルです。

サンプルの実行について

サンプルを実行すると、炎のエフェクト と g3d を利用してキャラクタモデルが再生・描画されます。

NintendoSDK\Samples\Sources\Applications\VfxViewerSimple

この状態から EffectMaker を起動し、起動したサンプルに合わせて EffectMaker 上部の プラットフォーム / 接続先 を設定することで、サンプルと EffectMaker が接続状態になります。

NintendoSDK\Tools\Graphics\EffectMaker\EffectMaker.exe

このサンプルを仮想制作中アプリケーションと見立てて、ビューアランタイムの以下の機能について説明します。

  • アプリ上でエフェクトの作成や編集
  • アプリに組み込まれたエフェクトの編集
  • アプリ内モデルとエフェクトの連動
参考：

ビューアランタイムは、Debug / Develop 版のみEffectMakerと接続可能です。

アプリ上でエフェクトの作成や編集

サンプルと接続状態になった上で、EffectMaker からエミッタセットの新規作成を実行すると、下記画像のようにEffectMaker で作成したエミッタセットがサンプル上で再生されます。

青枠 青枠に表示されている炎のエフェクトは、サンプル上でエフェクトバイナリを読み込んで再生しています。
赤枠 赤枠に表示されている 四角いパーティクル は、ビューアランタイムが再生するエフェクトです。
ビューアランタイムが再生するエフェクトは、サンプルとEffectMakerを接続・転送したバイナリ です。
緑枠 緑枠に表示されている キャラクタモデルは、サンプル上で g3d モデルバイナリを読み込んで再生・描画しています。

アプリに組み込まれたエフェクトの編集

次に、サンプルが再生する 炎のエフェクト を EffectMaker から調整します。まずは、サンプルが再生する炎のエフェクトファイル( TorchFire.eset )を EffectMaker で開きます。

NintendoSDK\Samples\Sources\Applications\VfxViewerSimple\Resources\Eset\TorchFire.eset

サンプルによって再生される TorchFire.eset が編集状態になると、下の画像のように「 エミッタセット」のアイコンに編集状態を表す「 鎖 のマーク」がつきます。

編集状態になると下記画像のように、パーティクルのカラーを編集すると、サンプルが再生している TorchFire のカラーも変更されます。

編集状態では、アプリが再生しているエフェクトバイナリ と EffectMakerが転送したエフェクトバイナリ が差し替わり、EffectMakerで編集した内容が、アプリで再生中のエフェクトに反映されるようになります。

アプリ内モデルとエフェクトの連動

最後に、アプリ内モデルとEffectMakerから転送したエフェクトの連動について説明します。

サンプルでは、再生している モデルの情報を  nn::vfx::viewer::UpdateModelInfo メソッドを利用して EffectMakerに転送しています。転送された モデルの情報 はプレビューノードの 「SRT制御」 タブ の「接続モデル」「接続ボーン」UIに列挙され、EffectMaker から転送したエフェクトは、そこで指定した モデル、ボーン名マトリクスで描画されるようになります。
