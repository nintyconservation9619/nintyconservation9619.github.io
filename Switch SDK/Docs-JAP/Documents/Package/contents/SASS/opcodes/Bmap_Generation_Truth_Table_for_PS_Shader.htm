 Definitions:
     A '1' in the PS Bmap location indicates the physical memory is present in the TRAM.
     i is an enabled shader slot, whcih is always 5 for the PS.
     p is the last enabled VTG shader slot.
     PsBmap[i,*] controls the buffer at the input to the pixel shader (i.e., the TriRam);
    Idmap[i,n] and Odmap[i,n] are the values in the Attribute Address Map and Default Table, where i picks the column 
             and n picks the row.
 IPA operation as affectd by PsBmap[i,n]:
     if PsBmap[i,n]==0, then the IPA always reads a zero.
     if PsBmap[i,n]==1, then the IPA reads "from" the TRAM, which could be any of:
             an attribute from the last VTG shader;
             a HW-generated attribute; or
             a default value.
 Note that the HW can additionally force some Bmap locations to '1' or '0'.  Here are the cases:
     If primitive IDs are needed by any shader stage (i.e., any Imap is '1'), then HW will put a '1' in every shader's Bmap 
             location that correspond to primitive ID.  This is done to provide a mechanism for PrimitiveID
             to be passed from shader to shader by the hardware.
     In GF1xx, for PS default inputs, the HW inserts a default value by setting a '1' in a Bmap location and putting 
             the default value into the buffer.
 Notes:
     (1) As seen in the table above, PsMap[i,n] is independent of the VTG Mpipes Omap[p,n] and Odmap[p,n].
